# ⏩ Sekventielle Agentarbejdsgange med Azure OpenAI (Responses API) (.NET)

## 📋 Avanceret Sekventiel Procesvejledning

Denne notesbog demonstrerer **sekventielle arbejdsgangsmønstre** ved brug af Microsoft Agent Framework for .NET og Azure OpenAI (Responses API). Du vil lære, hvordan man bygger sofistikerede, trin-for-trin behandlingspipelines, hvor agenter udfører i en specifik rækkefølge, hvor hvert trin bygger videre på resultaterne fra det foregående trin.

## 💻 Kørsel af Koden

Den komplette implementering findes i `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Denne fil demonstrerer en **tretrins møbelanalysearbejdsgang**:

1. **Fase 1 - Salgsagent**: Analyserer møbelbilleder og giver købsforslag
2. **Fase 2 - Prisagent**: Giver detaljerede prisopdelinger og budgetmuligheder
3. **Fase 3 - Tilbudsagent**: Genererer et professionelt tilbudsdokument i Markdown-format

### 🏗️ **Arbejdsgangsarkitektur**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Hver agent:
- Modtager output fra forrige trin som kontekst
- Bygger videre på tidligere analyse med specialiseret ekspertise
- Opretholder arbejdsgangens kontinuitet gennem tilstandsadministration

### 🚀 Kørsel af Eksemplet

**Forudsætninger:**
- Placer et møbelbillede ved `../imgs/home.png` (eller opdater `imgPath` variablen)
- Konfigurer din `.env` fil med din Azure OpenAI-endpoint og deployment, og log derefter ind med `az login`

```bash
# Gør scriptet eksekverbart (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Kør den sekventielle arbejdsgang
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Eller på Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Forventet Output

Arbejdsgangen vil:
1. **Salgsagent**: Identificere møbelgenstande fra billedet og give anbefalinger
2. **Prisagent**: Tilføje detaljeret prisanalyse med budgetniveauer og købsanbefalinger
3. **Tilbudsagent**: Generere et formateret tilbudsdokument med al information sammenfattet

Det endelige output vil være et omfattende, professionelt møbeltilbud baseret på billedanalyse.

### 🔍 Forståelse af Tilstandsflow

Hver agent i rækkefølgen modtager:
- **Originalt Input**: Den oprindelige brugermeddelelse (billede + tekst)
- **Tidligere Agentoutput**: Alle tidligere agenters svar i samtalehistorikken
- **Akkumuleret Kontekst**: Fuldstændig tilstand vedligeholdt gennem hele arbejdsgangen

Dette muliggør sofistikeret flerstegsbehandling, hvor hver agent bygger videre på omfattende kontekst fra alle tidligere stadier.
