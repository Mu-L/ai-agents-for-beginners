# ⏩ Sekventielle Agentarbejdsgange med Azure OpenAI (Responses API) (.NET)

## 📋 Avanceret Sekventiel Behandlingsvejledning

Denne notesbog demonstrerer **sekventielle arbejdsgangsmønstre** ved brug af Microsoft Agent Framework for .NET og Azure OpenAI (Responses API). Du vil lære at opbygge sofistikerede, trin-for-trin behandlingspipelines, hvor agenter udfører i en bestemt rækkefølge, hvor hver fase bygger videre på resultaterne fra den foregående fase.

## 💻 Kørsel af Kode

Den komplette implementering findes i `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Denne fil demonstrerer en **tre-trins møbelanalysearbejdsgang**:

1. **Fase 1 - Salgsagent**: Analyserer møbelbilleder og giver købsforslag
2. **Fase 2 - Prisagent**: Giver detaljerede prisopdelinger og budgetmuligheder
3. **Fase 3 - Tilbudsagent**: Genererer et professionelt tilbudsdokument i Markdown-format

### 🏗️ **Arbejdsgangsarkitektur**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Hver agent:
- Modtager output fra den foregående fase som kontekst
- Bygger videre på tidligere analyser med specialiseret ekspertise
- Opretholder arbejdsgangens kontinuitet gennem tilstandsstyring

### 🚀 Kørsel af Eksemplet

**Forudsætninger:**
- Placer et møbelbillede på `../imgs/home.png` (eller opdater `imgPath` variablen)
- Konfigurer din `.env` fil med din Azure OpenAI-endpoint og deployment, og log derefter ind med `az login`

```bash
# Gør scriptet eksekverbart (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Kør den sekventielle arbejdsgang
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Eller på Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Forventet Output

Arbejdsgangen vil:
1. **Salgsagent**: Identificere møbelelementer fra billedet og give anbefalinger
2. **Prisagent**: Tilføje detaljeret prisanalyse med budgetniveauer og indkøbsanbefalinger
3. **Tilbudsagent**: Generere et formateret tilbudsdokument med al samlet information

Det endelige output vil være et omfattende, professionelt møbeltilbud baseret på billedanalysen.

- **Ejendom**: Ejendomsanalyse → Vurdering → Opretning af annoncer

### 🔍 Forståelse af Tilstand Flow

Hver agent i sekvensen modtager:
- **Originalt Input**: Den oprindelige brugermeddelelse (billede + tekst)
- **Tidligere Agent Outputs**: Alle tidligere agenters svar i samtalehistorikken
- **Akkumuleret Kontekst**: Fuld tilstand bevaret gennem hele arbejdsgangen

Dette muliggør sofistikeret flerstadiebehandling, hvor hver agent bygger videre på den omfattende kontekst fra alle tidligere faser.
