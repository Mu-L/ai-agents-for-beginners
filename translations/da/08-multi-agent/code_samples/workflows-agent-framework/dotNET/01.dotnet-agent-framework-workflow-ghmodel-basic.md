# 🔄 Grundlæggende Agentarbejdsgange med Azure OpenAI (Responses API) (.NET)

## 📋 Arbejdsgang Orkestrering Tutorial

Denne notesbog demonstrerer, hvordan man bygger sofistikerede **agentarbejdsgange** ved hjælp af Microsoft Agent Framework for .NET og Azure OpenAI (Responses API). Du lærer at skabe flertrins-forretningsprocesser, hvor AI-agenter samarbejder om at udføre komplekse opgaver gennem strukturerede orkestreringsmønstre.

## 🎯 Læringsmål

### 🏗️ **Grundlæggende om Arbejdsgangsarkitektur**
- **Workflow Builder**: Design og orkestrer komplekse flertrins AI-processer
- **Agentkoordination**: Koordiner flere specialiserede agenter inden for arbejdsgange
- **Azure OpenAI (Responses API)**: Udnyt Azure OpenAI Responses API i arbejdsgange
- **Visuelt Arbejdsgang Design**: Skab og visualiser arbejdsgangsstrukturer for bedre forståelse

### 🔄 **Process Orkestreringsmønstre**
- **Sekventiel Processtyring**: Kæd flere agentopgaver i logisk rækkefølge
- **Tilstandsadministration**: Vedligehold kontekst og dataflow gennem arbejdsgangsstadier
- **Fejlhåndtering**: Implementer robust fejlgendannelse og arbejdsgangsmodstandsdygtighed
- **Ydelsesoptimering**: Design effektive arbejdsgange til virksomhedsskala operationer

### 🏢 **Erhvervsarbejdsgangsapplikationer**
- **Forretningsprocesautomatisering**: Automatiser komplekse organisatoriske arbejdsgange
- **Indholdsproduktionspipeline**: Redaktionelle arbejdsgange med gennemgangs- og godkendelsesfaser
- **Kundeserviceautomatisering**: Flertrins løsning af kundehenvendelser
- **Databehandlingsarbejdsgange**: ETL-arbejdsgange med AI-drevet transformation

## ⚙️ Forudsætninger & Opsætning

Denne arbejdsgangdemonstration bruger flere nøglepakker til .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI Konfiguration**

**Miljøopsætning (.env fil):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI Adgang:**
1. Opret en Azure OpenAI-ressource i Azure-portalen
2. Udrul en model (for eksempel `gpt-4.1-mini`) og noter udrulningsnavnet
3. Log ind med `az login` og konfigurer miljøvariabler som vist ovenfor

## 💻 Kørsel af Koden

Den komplette implementering findes i `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Denne fil demonstrerer:

1. **Miljøkonfiguration** - Indlæsning af Azure OpenAI-konfiguration fra `.env` fil
2. **Azure OpenAI Client Opsætning** - Konfiguration af klienten til at bruge Azure OpenAI Responses API
3. **Agentoprettelse** - Definition af specialiserede agenter (Front Desk og Concierge)
4. **Workflow Builder** - Oprettelse af en fleragentarbejdsgang med sekventiel behandling
5. **Arbejdsudførelse** - Kørsel af arbejdsgangen med streamende resultater

### 🚀 Kørsel af Eksemplet

```bash
# Gør scriptet eksekverbart (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Kør arbejdsgangen
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Eller på Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Forventet Output

Arbejdsgangen vil:
1. Acceptere din rejsemålsanmodning ("Jeg vil gerne rejse til Paris")
2. Front Desk-agenten giver en indledende anbefaling
3. Concierge-agenten gennemgår og forbedrer anbefalingen
4. Det endelige output viser hele samtalestrømmen
