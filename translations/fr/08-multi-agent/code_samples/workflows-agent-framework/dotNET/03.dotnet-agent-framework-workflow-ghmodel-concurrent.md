# ⚡ Flux de Travail Concurrent avec Agents et Azure OpenAI (Responses API) (.NET)

## 📋 Tutoriel sur le Traitement Parallèle à Haute Performance

Ce carnet montre **des modèles de flux de travail concurrentiels** utilisant le Microsoft Agent Framework pour .NET et Azure OpenAI (Responses API). Vous apprendrez à construire des flux de travail performants et parallèles qui maximisent le débit en exécutant plusieurs agents IA simultanément tout en maintenant la coordination et la cohérence des données.

## 💻 Exécution du Code

L’implémentation complète est disponible dans `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Ce fichier démontre un **flux de travail concurrent Fan-Out/Fan-In** pour la planification de voyage :

### 🏗️ **Architecture du Flux de Travail**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Composants Clés :**

1. **ConcurrentStartExecutor** : Diffuse la demande utilisateur à tous les agents simultanément
2. **Agent Chercheur** : Analyse simultanément les destinations et attractions
3. **Agent Planificateur** : Crée des plans de voyage détaillés en parallèle
4. **ConcurrentAggregationExecutor** : Collecte et fusionne les résultats des deux agents

### 🎯 **Modèle Fan-Out/Fan-In**

Ce flux de travail illustre le modèle classique **Fan-Out/Fan-In** :
- **Fan-Out** : Un message d’entrée est diffusé à plusieurs agents simultanément
- **Traitement Concurrent** : Plusieurs agents travaillent en parallèle sur la même tâche
- **Fan-In** : Les résultats de tous les agents sont collectés et agrégés en une sortie unique

### 📝 Résultat Attendu

Le flux de travail va :
1. **Diffuser la demande** : Envoyer "Planifie un voyage à Seattle en décembre" aux deux agents
2. **Traitement concurrent** : Les deux agents travaillent simultanément :
   - Le chercheur identifie les attractions et détails

3. **Agrégation** : Combiner les deux réponses en une sortie complète

### 🔍 Comprendre les Executors Personnalisés

**ConcurrentStartExecutor :**
- Implémente `IMessageHandler<string>` pour accepter des entrées string
- Diffuse les messages à tous les agents connectés
- Envoie le `TurnToken` pour déclencher le traitement concurrent

**ConcurrentAggregationExecutor :**
- Implémente `IMessageHandler<ChatMessage>` pour recevoir les réponses des agents
- Collecte les messages de manière thread-safe
- Agrège quand toutes les réponses attendues sont reçues
- Produit la sortie finale via `context.YieldOutputAsync()`

### ⚡ Bénéfices de Performance

**Concurent vs Séquentiel :**
- Séquentiel : Agent1 (30s) → Agent2 (30s) = **60 secondes au total**
- Concurrent : Agent1 (30s) || Agent2 (30s) = **30 secondes au total**

**Amélioration du débit** : Jusqu’à N× plus rapide pour N agents concurrents (selon la charge et les ressources)

### 🛡️ Gestion des erreurs

Le flux gère gracieusement les échecs des agents individuels :
- Si un agent échoue, les autres continuent le traitement
- L’agrégateur peut implémenter une logique de timeout
- Des résultats partiels peuvent être retournés si nécessaire

### 📊 Fonctionnalités Avancées

**Nombre dynamique d’agents :**
Modifier la logique d’agrégation pour supporter un nombre variable d’agents :

```csharp
private int _expectedAgentCount;
private readonly List<ChatMessage> _messages = [];

public async ValueTask HandleAsync(ChatMessage message, IWorkflowContext context)
{
    this._messages.Add(message);
    if (this._messages.Count == _expectedAgentCount)
    {
        // Process aggregation
    }
}
```

Ce modèle de flux concurrent est essentiel pour construire des systèmes d’agents IA performants et scalables !
