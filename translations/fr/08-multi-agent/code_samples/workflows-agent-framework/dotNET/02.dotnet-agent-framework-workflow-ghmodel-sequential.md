# ⏩ Flux de travail séquentiels avec Agents et Azure OpenAI (Responses API) (.NET)

## 📋 Tutoriel avancé de traitement séquentiel

Ce cahier de travail démontre des **modèles de flux de travail séquentiels** en utilisant le Microsoft Agent Framework pour .NET et Azure OpenAI (Responses API). Vous apprendrez comment construire des pipelines de traitement sophistiqués, étape par étape, où les agents s'exécutent dans un ordre précis, chaque étape s'appuyant sur les résultats de l'étape précédente.

## 🎯 Objectifs d'apprentissage

### 🔄 **Architecture de traitement séquentiel**
- **Conception de flux linéaires** : Créer des pipelines de traitement étape par étape avec des dépendances claires
- **Gestion de l'état** : Maintenir le contexte et le flux de données à travers les étapes séquentielles du flux de travail
- **Azure OpenAI (Responses API)** : Exploiter les modèles Azure OpenAI dans des flux de travail .NET multi-étapes
- **Modèles de pipeline d'entreprise** : Construire des systèmes de traitement séquentiel prêts pour la production

### 🏗️ **Modèles séquentiels avancés**
- **Traitement par étapes (Stage-Gate)** : Implémenter des points de validation entre les étapes du flux de travail
- **Préservation du contexte** : Maintenir l'état et les connaissances accumulées à travers toutes les étapes
- **Propagation des erreurs** : Gérer les échecs avec grâce dans les chaînes de traitement séquentiel
- **Optimisation des performances** : Exécution séquentielle efficace avec une surcharge minimale

### 🏢 **Applications séquentielles d'entreprise**
- **Pipeline de traitement documentaire** : Analyse, transformation et validation multi-étapes de documents
- **Flux de travail d'assurance qualité** : Processus séquentiels de révision, validation et approbation
- **Pipeline de production de contenu** : Recherche → Rédaction → Édition → Révision → Publication
- **Automatisation des processus métier** : Flux de travail métier multi-étapes avec dépendances claires entre étapes

```env

AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Gestion de la configuration :**

### 🏗️ **Architecture du flux de travail séquentiel**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Agent d'analyse]
    B --> C[Point de contrôle 1]
    C --> D[Stage 2: Agent de traitement]
    D --> E[Point de contrôle 2]
    E --> F[Stage 3: Agent de validation]
    F --> G[Résultat final]
    
    H[Contexte d'état] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API des réponses)] --> B
    I --> D
    I --> F
```

**Composants clés :**
- **Agents séquentiels** : Agents spécialisés pour chaque étape de traitement
- **Contexte d'état** : Maintient les données accumulées et décisions à travers les étapes
- **Points de contrôle** : Points de validation entre étapes pour assurer qualité et cohérence
- **Client Azure OpenAI** : Accès cohérent au modèle IA à travers toutes les étapes du flux

## 💻 Exécution du code

L'implémentation complète est disponible dans `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Ce fichier démontre un **flux de travail d'analyse de mobilier en trois étapes** :

1. **Étape 1 – Agent commercial** : Analyse des images de mobilier et suggestions d'achat
2. **Étape 2 – Agent de prix** : Fournit des détails de prix et des options budgétaires
3. **Étape 3 – Agent de devis** : Génère un document de devis professionnel au format Markdown

### 🏗️ **Architecture du flux de travail**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Chaque agent :
- Reçoit la sortie de l’étape précédente comme contexte
- S’appuie sur l’analyse précédente avec une expertise spécialisée
- Maintient la continuité du flux de travail grâce à la gestion d’état

### 🚀 Exécution de l'exemple

**Prérequis :**
- Placez une image de mobilier dans `../imgs/home.png` (ou mettez à jour la variable `imgPath`)
- Configurez votre fichier `.env` avec votre endpoint Azure OpenAI et votre déploiement, puis connectez-vous avec `az login`

```bash
# Rendre le script exécutable (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Exécuter le flux de travail séquentiel
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Ou sous Windows :
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Sortie attendue

Le flux de travail va :
1. **Agent commercial** : Identifier les éléments de mobilier sur l’image et fournir des recommandations
2. **Agent de prix** : Ajouter une analyse détaillée des prix avec des niveaux budgétaires et recommandations d'achat
3. **Agent de devis** : Générer un document de devis formaté avec toutes les informations synthétisées

Le résultat final sera un devis complet et professionnel pour le mobilier basé sur l’analyse d’image.

### 🎯 Applications concrètes

Ce modèle séquentiel est idéal pour :
- **Commerce électronique** : Analyse de produit → Tarification → Génération de devis
- **Immobilier** : Analyse de propriété → Évaluation → Création d'annonce

### 🔍 Compréhension du flux d'état

Chaque agent dans la séquence reçoit :
- **Entrée originale** : Le message initial de l'utilisateur (image + texte)
- **Sorties des agents précédents** : Toutes les réponses des agents précédents dans l'historique de conversation
- **Contexte accumulé** : État complet maintenu tout au long du flux de travail

Cela permet un traitement multi-étapes sophistiqué où chaque agent s'appuie sur un contexte complet de toutes les étapes précédentes.
