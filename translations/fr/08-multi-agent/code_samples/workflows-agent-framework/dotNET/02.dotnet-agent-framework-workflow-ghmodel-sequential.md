# ⏩ Flux de travail séquentiels avec les agents Azure OpenAI (API Réponses) (.NET)

## 📋 Tutoriel avancé de traitement séquentiel

Ce notebook démontre des **modèles de flux de travail séquentiels** en utilisant le Microsoft Agent Framework pour .NET et Azure OpenAI (API Réponses). Vous apprendrez à construire des pipelines de traitement sophistiqués, étape par étape, où les agents s’exécutent dans un ordre spécifique, chaque étape s'appuyant sur les résultats de l'étape précédente.

## 💻 Exécution du code

L’implémentation complète est disponible dans `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Ce fichier illustre un **flux de travail d’analyse de meubles en trois étapes** :

1. **Étape 1 - Agent commercial** : Analyse les images de meubles et fournit des suggestions d'achat
2. **Étape 2 - Agent tarification** : Fournit des détails sur la décomposition des prix et des options budgétaires
3. **Étape 3 - Agent devis** : Génère un document de devis professionnel au format Markdown

### 🏗️ **Architecture du workflow**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Chaque agent :
- Reçoit la sortie de l’étape précédente comme contexte
- S’appuie sur l’analyse précédente avec une expertise spécialisée
- Maintient la continuité du workflow par la gestion d’état

### 🚀 Exécution de l'exemple

**Prérequis :**
- Placez une image de meuble dans `../imgs/home.png` (ou mettez à jour la variable `imgPath`)
- Configurez votre fichier `.env` avec votre point de terminaison Azure OpenAI et votre déploiement, puis connectez-vous avec `az login`

```bash
# Rendre le script exécutable (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Exécuter le flux de travail séquentiel
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Ou sous Windows :
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Résultat attendu

Le workflow va :
1. **Agent commercial** : Identifier les meubles sur l’image et fournir des recommandations
2. **Agent tarification** : Ajouter une analyse détaillée des prix avec des paliers budgétaires et recommandations d’achat
3. **Agent devis** : Générer un document de devis formaté avec toutes les informations synthétisées

Le résultat final sera un devis complet et professionnel basé sur l’analyse d’image des meubles.

### 🔍 Comprendre le flux d’état

Chaque agent dans la séquence reçoit :
- **Entrée originale** : Le message initial de l’utilisateur (image + texte)
- **Sorties des agents précédents** : Toutes les réponses des agents précédents dans l’historique de la conversation
- **Contexte accumulé** : État complet maintenu tout au long du workflow

Cela permet un traitement multi-étapes sophistiqué où chaque agent s’appuie sur un contexte complet de toutes les étapes précédentes.
