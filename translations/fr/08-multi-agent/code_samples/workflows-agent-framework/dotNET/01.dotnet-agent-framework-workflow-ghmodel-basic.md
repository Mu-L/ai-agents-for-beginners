# 🔄 Flux de travail de base des agents avec Azure OpenAI (API Responses) (.NET)

## 📋 Tutoriel d'orchestration des flux de travail

Ce notebook démontre comment construire des **flux de travail d'agents** sophistiqués en utilisant le Microsoft Agent Framework pour .NET et Azure OpenAI (API Responses). Vous apprendrez à créer des processus métier à plusieurs étapes où des agents IA collaborent pour accomplir des tâches complexes via des modèles d'orchestration structurés.

## 🎯 Objectifs d'apprentissage

### 🏗️ **Fondamentaux de l'architecture des flux de travail**
- **Workflow Builder** : Concevoir et orchestrer des processus IA complexes à plusieurs étapes
- **Coordination des agents** : Coordonner plusieurs agents spécialisés au sein des flux de travail
- **Azure OpenAI (API Responses)** : Exploiter l'API Azure OpenAI Responses dans les flux de travail
- **Conception visuelle des flux de travail** : Créer et visualiser les structures des flux pour une meilleure compréhension

### 🔄 **Modèles d'orchestration des processus**
- **Traitement séquentiel** : Enchaîner plusieurs tâches d'agents dans un ordre logique
- **Gestion de l'état** : Maintenir le contexte et le flux de données entre les étapes des flux
- **Gestion des erreurs** : Implémenter une récupération robuste des erreurs et une résilience des flux
- **Optimisation des performances** : Concevoir des flux efficaces pour des opérations à l'échelle entreprise

### 🏢 **Applications des flux de travail en entreprise**
- **Automatisation des processus métier** : Automatiser des flux organisationnels complexes
- **Pipeline de production de contenu** : Flux éditoriaux avec étapes de révision et d'approbation
- **Automatisation du service client** : Résolution multi-étapes des requêtes clients
- **Flux de traitement des données** : Flux ETL avec transformation assistée par IA

```xml
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Configuration Azure OpenAI**

**Configuration de l'environnement (fichier .env) :**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Accès Azure OpenAI :**
1. Créez une ressource Azure OpenAI dans le portail Azure
2. Déployez un modèle (par exemple, `gpt-4.1-mini`) et notez le nom du déploiement
3. Connectez-vous avec `az login` et configurez les variables d'environnement comme indiqué ci-dessus

## 💻 Exécution du code

L'implémentation complète est disponible dans `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Ce fichier démontre :

1. **Configuration de l'environnement** - Chargement de la configuration Azure OpenAI à partir du fichier `.env`
2. **Configuration du client Azure OpenAI** - Configuration du client pour utiliser l'API Azure OpenAI Responses
3. **Création d'agents** - Définition d'agents spécialisés (Réception et Concierge)
4. **Workflow Builder** - Création d'un flux multi-agents avec traitement séquentiel

```bash

chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Exécuter le flux de travail
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Ou sous Windows :
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Sortie attendue

Le flux va :
1. Accepter votre demande de destination de voyage ("Je voudrais aller à Paris")
2. L'agent de réception fournit une recommandation initiale
3. L'agent concierge examine et affine la recommandation
4. La sortie finale affiche le flux complet de la conversation

### 🔧 Personnalisation

Vous pouvez personnaliser le flux en :
- Modifiant les instructions des agents pour changer leur comportement
- Ajoutant plus d'agents pour créer des flux multi-étapes complexes
- Changeant le message utilisateur pour tester différents scénarios
- Ajustant les connexions du flux pour créer différents modèles d'exécution
