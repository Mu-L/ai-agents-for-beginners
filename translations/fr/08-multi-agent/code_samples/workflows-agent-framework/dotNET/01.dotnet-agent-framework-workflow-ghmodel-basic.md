# 🔄 Flux de travail basiques des agents avec Azure OpenAI (API Responses) (.NET)

## 📋 Tutoriel d'orchestration des flux de travail

Ce carnet montre comment construire des **flux de travail d'agents** sophistiqués en utilisant le Microsoft Agent Framework pour .NET et Azure OpenAI (API Responses). Vous apprendrez à créer des processus métier en plusieurs étapes où des agents IA collaborent pour accomplir des tâches complexes via des modèles d'orchestration structurés.

## 🎯 Objectifs d'apprentissage

### 🏗️ **Fondamentaux de l'architecture de flux de travail**
- **Concepteur de flux** : Concevoir et orchestrer des processus IA complexes en plusieurs étapes
- **Coordination des agents** : Coordonner plusieurs agents spécialisés dans les flux de travail
- **Azure OpenAI (API Responses)** : Exploiter l'API Responses Azure OpenAI dans les flux
- **Conception visuelle des flux** : Créer et visualiser les structures de flux pour une meilleure compréhension

### 🔄 **Modèles d'orchestration de processus**
- **Traitement séquentiel** : Enchaîner plusieurs tâches d'agents dans un ordre logique
- **Gestion d'état** : Maintenir le contexte et le flux de données entre les étapes du flux
- **Gestion des erreurs** : Implémenter une récupération robuste des erreurs et une résilience des flux
- **Optimisation des performances** : Concevoir des flux efficaces pour les opérations à l'échelle entreprise

### 🏢 **Applications des flux métier en entreprise**
- **Automatisation des processus métier** : Automatiser les flux organisationnels complexes
- **Pipeline de production de contenu** : Flux éditoriaux avec étapes de révision et d'approbation
- **Automatisation du service client** : Résolution de demandes clients en plusieurs étapes
- **Flux de traitement de données** : Flux ETL avec transformation assistée par IA

## ⚙️ Prérequis & Installation

### 📦 **Packages NuGet requis**

Cette démonstration de flux utilise plusieurs packages .NET clés :

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Configuration Azure OpenAI**

**Configuration de l'environnement (fichier .env) :**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Accès Azure OpenAI :**
1. Créez une ressource Azure OpenAI dans le portail Azure
2. Déployez un modèle (par exemple, `gpt-5-mini`) et notez le nom du déploiement
3. Connectez-vous avec `az login` et configurez les variables d'environnement comme montré ci-dessus

## 💻 Exécution du code

L'implémentation complète se trouve dans `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Ce fichier montre :

1. **Configuration de l'environnement** – Chargement de la configuration Azure OpenAI depuis le fichier `.env`
2. **Configuration du client Azure OpenAI** – Configuration du client pour utiliser l'API Responses Azure OpenAI
3. **Création des agents** – Définition des agents spécialisés (réception et concierge)
4. **Concepteur de flux** – Création d'un flux multi-agents avec traitement séquentiel
5. **Exécution du flux** – Exécution du flux avec résultats en streaming

### 🚀 Exécution de l'exemple

```bash
# Rendre le script exécutable (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Exécuter le flux de travail
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Ou sur Windows :
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Résultat attendu

Le flux fera :
1. Acceptera votre demande de destination de voyage (« Je voudrais aller à Paris »)
2. L'agent de la réception fournira une première recommandation
3. L'agent concierge examinera et affinera la recommandation
4. Le résultat final affichera la conversation complète en flux

### 🔧 Personnalisation

Vous pouvez personnaliser le flux en :
- Modifiant les instructions des agents pour changer leur comportement
- Ajoutant plus d'agents pour créer des flux complexes en plusieurs étapes
- Changeant le message utilisateur pour tester différents scénarios
- Ajustant les connexions du flux pour créer différents modèles d'exécution
