# ⏩ Sekvencijalni radni tijekovi agenata s Azure OpenAI (Responses API) (.NET)

## 📋 Napredni vodič za sekvencijalnu obradu

Ovaj bilježnik prikazuje **sekvencijalne obrasce radnih tijekova** koristeći Microsoft Agent Framework za .NET i Azure OpenAI (Responses API). Naučit ćete kako izgraditi sofisticirane, korak-po-korak procese obrade u kojima agenti izvršavaju zadatke u određenom redoslijedu, pri čemu se svaka faza nadovezuje na rezultate prethodne.

## 🎯 Ciljevi učenja

### 🔄 **Arhitektura sekvencijalne obrade**
- **Dizajn linearnih radnih tijekova**: Kreirajte korak-po-korak procese obrade s jasnim ovisnostima
- **Upravljanje stanjem**: Održavajte kontekst i protok podataka kroz sekvencijalne faze radnog tijeka
- **Azure OpenAI (Responses API)**: Iskoristite Azure OpenAI modele u višefaznim .NET radnim tijekovima
- **Obrasci za poslovne procese**: Izgradite proizvodne sustave za sekvencijalnu obradu

### 🏗️ **Napredni sekvencijalni obrasci**
- **Obrada sa kontrolnim točkama (Stage-Gate)**: Implementirajte provjere valjanosti između faza radnog tijeka
- **Čuvanje konteksta**: Održavajte stanje i akumulirano znanje kroz sve faze
- **Propagacija pogrešaka**: Rukujte kvarovima na sustavan način u sekvencijalnim lancima obrade
- **Optimizacija performansi**: Učinkovito sekvencijalno izvršavanje s minimalnim režijskim troškovima

### 🏢 **Sekvencijalne aplikacije za poduzeća**
- **Cjevovod za obradu dokumenata**: Višefazna analiza, transformacija i validacija dokumenata
- **Radni tijekovi osiguranja kvalitete**: Sekvencijalni procesi pregleda, potvrde i odobrenja
- **Cjevovod za stvaranje sadržaja**: Istraživanje → Pisanje → Uređivanje → Pregled → Objavljivanje
- **Automatizacija poslovnih procesa**: Višekorakni poslovni tijekovi s jasnim ovisnostima među fazama

```xml

<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

```env

AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Upravljanje konfiguracijom:**
```csharp
// Load environment variables securely
```

### 🏗️ **Arhitektura sekvencijalnog radnog tijeka**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Analitički agent]
    B --> C[Kontrolna točka 1]
    C --> D[Stage 2: Obradni agent]
    D --> E[Kontrolna točka 2]
    E --> F[Stage 3: Validacijski agent]
    F --> G[Konačni rezultat]
    
    H[Kontekst stanja] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Responses API)] --> B
    I --> D
    I --> F
```

**Ključne komponente:**
- **Sekvencijalni agenti**: Specijalizirani agenti za svaku fazu obrade
- **Kontekst stanja**: Održava akumulirane podatke i odluke kroz faze
- **Kontrolne točke**: Provjere valjanosti između faza za osiguranje kvalitete i konzistentnosti
- **Azure OpenAI klijent**: Dosljedan pristup AI modelima kroz sve faze radnog tijeka

## 💻 Pokretanje koda

Potpuna implementacija dostupna je u datoteci `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Ova datoteka prikazuje **troslojni radni tijek analize namještaja**:

1. **Faza 1 - Prodajni agent**: Analizira slike namještaja i daje prijedloge za kupnju
2. **Faza 2 - Agent za cijene**: Pruža detaljne razrade cijena i opcije budžeta
3. **Faza 3 - Agent za ponude**: Generira profesionalni dokument s ponudom u Markdown formatu

### 🏗️ **Arhitektura radnog tijeka**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Svaki agent:
- Prima izlaz iz prethodne faze kao kontekst
- Nadograđuje prethodne analize sa specijaliziranim znanjem
- Održava kontinuitet radnog tijeka upravljanjem stanja

### 🚀 Pokretanje primjera

**Preduvjeti:**
- Postavite sliku namještaja na `../imgs/home.png` (ili ažurirajte varijablu `imgPath`)
- Konfigurirajte vaš `.env` s Azure OpenAI krajnjom točkom i implementacijom, zatim se prijavite s `az login`

```bash
# Napravite skriptu izvršnom (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Pokrenite sekvencijski tijek rada
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Ili na Windowsu:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Očekivani rezultat

Radni tijek će:
1. **Prodajni agent**: Prepoznati stavke namještaja sa slike i dati preporuke
2. **Agent za cijene**: Dodati detaljnu analizu cijena s razinama budžeta i preporukama za kupnju
3. **Agent za ponude**: Generirati formatirani dokument s ponudom sa svim sintetiziranim informacijama

Konačni rezultat bit će sveobuhvatna, profesionalna ponuda za namještaj na temelju analize slike.
