# ⏩ Sekvencijski agentni tijekovi rada s Azure OpenAI (Responses API) (.NET)

## 📋 Napredni vodič za sekvencijsku obradu

Ovaj bilježnik demonstrira **uzorke sekvencijskih tijekova rada** koristeći Microsoft Agent Framework za .NET i Azure OpenAI (Responses API). Naučit ćete kako izgraditi sofisticirane, korak-po-korak obrade u kojima agenti izvršavaju u određenom redoslijedu, a svaka faza se nadovezuje na rezultate prethodne.

```xml

AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

### 🏗️ **Arhitektura sekvencijskog tijeka rada**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Agent analize]
    B --> C[Kontrolna točka 1]
    C --> D[Stage 2: Agent obrade]
    D --> E[Kontrolna točka 2]
    E --> F[Stage 3: Agent validacije]
    F --> G[Završni rezultat]
    
    H[Kontekst stanja] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Responses API)] --> B
    I --> D
    I --> F
```

**Ključne komponente:**
- **Sekvencijski agenti**: Specijalizirani agenti za svaku fazu obrade
- **Kontekst stanja**: Održava akumulirane podatke i odluke kroz faze
- **Kontrolne točke**: Točke validacije između faza za osiguravanje kvalitete i dosljednosti
- **Azure OpenAI klijent**: Dosljedan pristup AI modelu preko svih faza tijeka rada

## 💻 Pokretanje koda

Kompletna implementacija dostupna je u `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Ova datoteka demonstrira **tijek analize namještaja u tri faze**:

1. **Faza 1 - Agent prodaje**: Analizira slike namještaja i daje prijedloge za kupnju
2. **Faza 2 - Agent cijena**: Daje detaljne razrade cijena i opcije budžeta
3. **Faza 3 - Agent ponude**: Generira profesionalni dokument ponude u Markdown formatu

### 🏗️ **Arhitektura tijeka rada**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Svaki agent:
- Prima izlaz iz prethodne faze kao kontekst
- Nadovezuje se na prethodnu analizu sa specijaliziranim znanjem
- Održava kontinuitet tijeka rada kroz upravljanje stanjem

### 🚀 Pokretanje primjera

**Preduvjeti:**
- Postavite sliku namještaja u `../imgs/home.png` (ili promijenite varijablu `imgPath`)
- Konfigurirajte svoj `.env` s Azure OpenAI endpointom i deploymentom, zatim se prijavite putem `az login`

```bash
# Napravite skriptu izvršnom (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Pokrenite sekvencijalni tijek rada
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Ili na Windowsu:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Očekivani izlaz

Tijek rada će:
1. **Agent prodaje**: Prepoznati stavke namještaja sa slike i dati preporuke
2. **Agent cijena**: Dodati detaljnu analizu cijena s kategorijama budžeta i savjete za kupnju
3. **Agent ponude**: Generirati strukturirani dokument ponude sa svim sintetiziranim informacijama

Krajnji rezultat bit će sveobuhvatna, profesionalna ponuda za namještaj temeljem analize slike.
