# 🔄 Osnovni tijekovi rada agenata s Azure OpenAI (Responses API) (.NET)

## 📋 Tutorijal za orkestraciju tijeka rada

Ovaj bilježnik prikazuje kako izgraditi sofisticirane **tijekove rada agenata** koristeći Microsoft Agent Framework za .NET i Azure OpenAI (Responses API). Naučit ćete kako kreirati višestupanjske poslovne procese u kojima AI agenti surađuju kako bi izvršili složene zadatke kroz strukturirane obrasce orkestracije.

## 🎯 Ciljevi učenja

### 🏗️ **Osnove arhitekture tijeka rada**
- **Graditelj tijeka rada**: Dizajniranje i orkestriranje složenih višestupanjskih AI procesa
- **Koordinacija agenata**: Koordinacija više specijaliziranih agenata unutar tijekova rada
- **Azure OpenAI (Responses API)**: Korištenje Azure OpenAI Responses API-ja u tijekovima rada
- **Vizualni dizajn tijeka rada**: Kreiranje i vizualizacija struktura tijeka rada radi boljeg razumijevanja

### 🔄 **Obrasci orkestracije procesa**
- **Sekvencijalna obrada**: Lančanje više zadataka agenata u logičkom redoslijedu
- **Upravljanje stanjem**: Održavanje konteksta i protoka podataka kroz faze tijeka rada
- **Rukovanje pogreškama**: Implementacija robusnog oporavka od pogrešaka i otpornosti tijeka rada
- **Optimizacija performansi**: Dizajn učinkovitih tijekova rada za poslovne operacije velikih razmjera

### 🏢 **Poslovne primjene tijekova rada**
- **Automatizacija poslovnih procesa**: Automatizacija složenih organizacijskih tijekova rada
- **Proizvodni tijek sadržaja**: Urednički tijekovi rada s fazama recenzije i odobrenja
- **Automatizacija korisničke službe**: Višestupanjsko rješavanje korisničkih upita
- **Tijekovi obrade podataka**: ETL tijekovi rada s AI-pokretanom transformacijom

## ⚙️ Preduvjeti i postavljanje

### 📦 **Potrebni NuGet paketi**

Ova demonstracija tijeka rada koristi nekoliko ključnih .NET paketa:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Konfiguracija Azure OpenAI**

**Postavljanje okruženja (datoteka .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Pristup Azure OpenAI:**
1. Kreirajte Azure OpenAI resurs u Azure portalu
2. Postavite model (na primjer, `gpt-5-mini`) i zabilježite naziv implementacije
3. Prijavite se s `az login` i konfigurirajte varijable okruženja kao što je prikazano gore

### 🏗️ **Pregled arhitekture tijeka rada**

```mermaid
graph TD
    A[Graditelj tijeka rada] --> B[Registar agenata]
    B --> C[Pokretač izvršenja tijeka rada]
    C --> D[Agent 1: Generator sadržaja]
    C --> E[Agent 2: Recenzent sadržaja] 
    D --> F[Rezultati tijeka rada]
    E --> F
    G[Azure OpenAI (API odgovora)] --> D
    G --> E
```

**Ključne komponente:**
- **WorkflowBuilder**: Glavni orkestracijski mehanizam za dizajn tijekova rada
- **AIAgent**: Pojedinačni specijalizirani agenti s određenim sposobnostima
- **Azure OpenAI klijent**: Integracija s Azure OpenAI Responses API-jem
- **Kontekst izvršavanja**: Upravljanje stanjem i protokom podataka između faza tijeka rada

## 💻 Pokretanje koda

Cjelovita implementacija dostupna je u `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Ova datoteka prikazuje:

1. **Konfiguracija okruženja** - Učitavanje Azure OpenAI konfiguracije iz `.env` datoteke
2. **Postavljanje Azure OpenAI klijenta** - Konfiguracija klijenta za korištenje Azure OpenAI Responses API-ja
3. **Kreiranje agenata** - Definiranje specijaliziranih agenata (Front Desk i Concierge)
4. **Graditelj tijeka rada** - Kreiranje višestrukog tijeka rada s uzastopnom obradom
5. **Izvršenje tijeka rada** - Pokretanje tijeka rada s rezultatima u streamingu

### 🚀 Pokretanje primjera

```bash
# Napravite skriptu izvršnom (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Pokrenite tijek rada
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Ili na Windowsu:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Očekivani ishod

Tijek rada će:
1. Prihvatiti vaš zahtjev za destinaciju putovanja ("Želio/la bih ići u Pariz")
2. Agent na recepciji pruža početnu preporuku
3. Agent concierge pregledava i usavršava preporuku
4. Završni ispis prikazuje cijeli razgovor u tijeku

- Mijenjanjem uputa agentima za promjenu njihovog ponašanja

- Mijenjanjem korisničke poruke za isprobavanje različitih scenarija
- Prilagodbom veza tijeka rada za stvaranje različitih obrazaca izvršenja
