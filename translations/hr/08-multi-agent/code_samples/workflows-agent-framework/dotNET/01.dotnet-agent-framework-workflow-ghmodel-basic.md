# 🔄 Osnovni tijekovi rada agentâ s Azure OpenAI (Responses API) (.NET)

## 📋 Vodič za orkestraciju tijekova rada

Ovaj bilježnik demonstrira kako izgraditi sofisticirane **tijekove rada agenata** koristeći Microsoft Agent Framework za .NET i Azure OpenAI (Responses API). Naučit ćete kako kreirati višestepene poslovne procese u kojima AI agenti surađuju kako bi izvršili složene zadatke kroz strukturirane obrasce orkestracije.

## 🎯 Ciljevi učenja

### 🏗️ **Osnove arhitekture tijekova rada**
- **Graditelj tijekova rada (Workflow Builder)**: Dizajniranje i orkestracija složenih višestepenih AI procesa
- **Koordinacija agenata**: Koordinacija više specijaliziranih agenata unutar tijekova rada
- **Azure OpenAI (Responses API)**: Iskorištavanje Azure OpenAI Responses API-ja u tijekovima rada
- **Vizualni dizajn tijekova rada**: Kreiranje i vizualizacija struktura tijekova rada za bolje razumijevanje

### 🔄 **Obrasci orkestracije procesa**
- **Sekvencijalna obrada**: Povezivanje više zadataka agenata u logičkom redoslijedu
- **Upravljanje stanjem**: Održavanje konteksta i protoka podataka kroz faze tijeka rada
- **Rukovanje pogreškama**: Implementacija robusnog oporavka od pogrešaka i otpornosti tijeka rada
- **Optimizacija performansi**: Dizajniranje učinkovitih tijekova rada za korporativne operacije velikih razmjera

### 🏢 **Primjene korporativnih tijekova rada**
- **Automatizacija poslovnih procesa**: Automatizacija složenih organizacijskih tijekova rada
- **Cjevovod za proizvodnju sadržaja**: Urednički tijekovi rada s fazama pregleda i odobrenja
- **Automatizacija korisničke službe**: Višestepeno rješavanje upita korisnika
- **Tijekovi obrade podataka**: ETL tijekovi rada s AI-pogonjenom transformacijom

## ⚙️ Preduvjeti i postavljanje

### 📦 **Potrebni NuGet paketi**

Ova demonstracija tijeka rada koristi nekoliko ključnih .NET paketa:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Konfiguracija Azure OpenAI**

**Postavljanje okoline (.env datoteka):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Pristup Azure OpenAI-u:**
1. Kreirajte Azure OpenAI resurs u Azure portalu
2. Postavite model (na primjer, `gpt-4.1-mini`) i zabilježite naziv implementacije
3. Prijavite se s `az login` i konfigurirajte varijable okoline kao što je prikazano gore

### 🏗️ **Pregled arhitekture tijeka rada**

```mermaid
graph TD
    A[Graditelj tijeka rada] --> B[Registar agenata]
    B --> C[Pokretač izvršavanja tijeka rada]
    C --> D[Agent 1: Generator sadržaja]
    C --> E[Agent 2: Recenzent sadržaja] 
    D --> F[Rezultati tijeka rada]
    E --> F
    G[Azure OpenAI (Responses API)] --> D
    G --> E
```

**Ključne komponente:**
- **WorkflowBuilder**: Glavni mehanizam orkestracije za dizajn tijekova rada
- **AIAgent**: Pojedinačni specijalizirani agenti s određenim sposobnostima
- **Azure OpenAI Client**: Integracija Azure OpenAI Responses API-ja
- **Execution Context**: Upravljanje stanjem i protokom podataka između faza tijeka rada

## 💻 Pokretanje koda

Potpuna implementacija dostupna je u `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Ova datoteka demonstrira:

1. **Konfiguracija okoline** - Učitavanje Azure OpenAI konfiguracije iz `.env` datoteke
2. **Postavljanje Azure OpenAI klijenta** - Konfiguriranje klijenta za korištenje Azure OpenAI Responses API-ja
3. **Kreiranje agenata** - Definiranje specijaliziranih agenata (Front Desk i Concierge)
4. **Graditelj tijekova rada** - Kreiranje višedobnih tijekova rada s sekvencijalnom obradom
5. **Izvršavanje tijeka rada** - Pokretanje tijeka rada s prijenosom rezultata uživo

### 🚀 Pokretanje primjera

```bash
# Napravite skriptu izvršnom (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Pokrenite tijek rada
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Ili na Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Očekivani rezultat

Tijek rada će:
1. Prihvatiti vaš zahtjev za destinaciju putovanja ("Volio bih ići u Pariz")
2. Agent Front Desk-a daje početnu preporuku
3. Agent Concierge pregledava i usavršava preporuku
4. Završni ispis prikazuje cijeli tok razgovora
