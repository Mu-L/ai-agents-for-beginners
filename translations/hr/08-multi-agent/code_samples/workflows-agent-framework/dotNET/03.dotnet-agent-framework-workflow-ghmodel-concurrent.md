# ⚡ Istovremeni agentni radni tokovi s Azure OpenAI (Responses API) (.NET)

## 📋 Vodič za visokoučinkovitu paralelnu obradu

## 💻 Pokretanje koda

Cjelovita implementacija dostupna je u `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Ova datoteka demonstrira **Fan-Out/Fan-In istovremeni radni tok** za planiranje putovanja:

### 🏗️ **Arhitektura radnog toka**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Ključne komponente:**

1. **ConcurrentStartExecutor**: Emitira korisnički zahtjev svim agentima istovremeno
2. **Researcher Agent**: Paralelno analizira destinacije i atrakcije
3. **Planner Agent**: Paralelno izrađuje detaljne putne planove
4. **ConcurrentAggregationExecutor**: Prikuplja i spaja rezultate oba agenta

### 🎯 **Fan-Out/Fan-In obrazac**

Ovaj radni tok prikazuje klasični **Fan-Out/Fan-In** obrazac:
- **Fan-Out**: Jedna ulazna poruka emitira se višestrukim agentima istovremeno
- **Istovremena obrada**: Više agenata paralelno radi na istom zadatku
- **Fan-In**: Rezultati svih agenata prikupljaju se i agregiraju u jedinstveni izlaz

### 📝 Očekivani izlaz

Radni tok će:
1. **Emitirati zahtjev**: Poslati "Planiraj putovanje u Seattle u prosincu" oboma agentima
2. **Istovremena obrada**: Oba agenta rade istovremeno:

### 🔍 Razumijevanje prilagođenih izvršitelja

**ConcurrentStartExecutor:**
- Implementira `IMessageHandler<string>` za prihvat unosa tipa string
- Emitira poruke svim povezanim agentima
- Šalje `TurnToken` za pokretanje istovremene obrade

**ConcurrentAggregationExecutor:**
- Implementira `IMessageHandler<ChatMessage>` za primanje odgovora agenata
- Prikuplja poruke na thread-safe način
- Agregira kad stignu svi očekivani odgovori
- Vraća konačni izlaz pomoću `context.YieldOutputAsync()`

### ⚡ Prednosti performansi

**Istovremeno vs Sekvencijalno:**
- Sekvencijalno: Agent1 (30s) → Agent2 (30s) = **ukupno 60 sekundi**
- Istovremeno: Agent1 (30s) || Agent2 (30s) = **ukupno 30 sekundi**

**Poboljšanje propusnosti**: Do N× brže za N konkurentnih agenata (ovisno o opterećenju i resursima)

### 🛡️ Rukovanje pogreškama

Radni tok elegantno rukuje pojedinačnim kvarovima agenata:
- Ako jedan agent zakaže, ostali nastavljaju obradu
- Agregator može implementirati logiku timeouta
- Djelomični rezultati se mogu vratiti po potrebi
