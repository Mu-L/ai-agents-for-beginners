# ⚡ Istovremeni Agentni Radni Procesi s Azure OpenAI (Responses API) (.NET)

## 📋 Vodič za Paralelnu Obradu Visokih Performansi

Ovaj bilježnički dokument prikazuje **uzorke istovremenih radnih procesa** koristeći Microsoft Agent Framework za .NET i Azure OpenAI (Responses API). Naučit ćete kako izgraditi radne procese visokih performansi i paralelne obrade koji maksimalno povećavaju propusnost izvođenjem više AI agenata istovremeno uz održavanje koordinacije i konzistentnosti podataka.

## 💻 Pokretanje Koda

Potpuna implementacija dostupna je u `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Ova datoteka prikazuje **Fan-Out/Fan-In istovremeni radni proces** za planiranje putovanja:

### 🏗️ **Arhitektura Radnog Procesa**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Glavne Komponente:**

1. **ConcurrentStartExecutor**: Objavljuje korisnički zahtjev svim agentima istovremeno
2. **Researcher Agent**: Istovremeno analizira destinacije i atrakcije
3. **Planner Agent**: Istovremeno kreira detaljne planove putovanja
4. **ConcurrentAggregationExecutor**: Prikuplja i spaja rezultate oba agenta

### 🎯 **Fan-Out/Fan-In Obrazac**

Ovaj radni proces prikazuje klasični **Fan-Out/Fan-In** obrazac:
- **Fan-Out**: Jedna ulazna poruka se šalje više agenata istovremeno
- **Istovremena Obrada**: Više agenata radi paralelno na istom zadatku
- **Fan-In**: Rezultati svih agenata se prikupljaju i agregiraju u jedan izlaz

### 📝 Očekivani Izlaz

Radni proces će:
1. **Emitirati Zahtjev**: Poslati "Planiraj putovanje u Seattle u prosincu" oba agenta
2. **Istovremena Obrada**: Oba agenta rade istovremeno:
   - Istraživač identificira atrakcije i detalje

3. **Agregacija**: Spojiti oba odgovora u sveobuhvatan izlaz

### 🔍 Razumijevanje Prilagođenih Izvršitelja

**ConcurrentStartExecutor:**
- Implementira `IMessageHandler<string>` za prihvat ulaznih stringova
- Emitira poruke svim povezanim agentima
- Šalje `TurnToken` za pokretanje istovremene obrade

**ConcurrentAggregationExecutor:**
- Implementira `IMessageHandler<ChatMessage>` za primanje odgovora agenata
- Prikuplja poruke na thread-safe način
- Agregira kad stignu svi očekivani odgovori
- Izlaz proizvodi korištenjem `context.YieldOutputAsync()`

### ⚡ Prednosti Performansi

**Istovremeno vs Sekvencijalno:**
- Sekvencijalno: Agent1 (30s) → Agent2 (30s) = **60 sekundi ukupno**
- Istovremeno: Agent1 (30s) || Agent2 (30s) = **30 sekundi ukupno**

**Poboljšanje propusnosti**: Do N× brže za N istovremenih agenata (ovisno o opterećenju i resursima)

### 🛡️ Rukovanje Greškama

Radni proces elegantno rukuje pojedinačnim kvarovima agenata:
- Ako jedan agent zakaže, ostali nastavljaju obradu
- Agregator može implementirati logiku tajm-auta
- Djelomični rezultati se mogu vratiti po potrebi
