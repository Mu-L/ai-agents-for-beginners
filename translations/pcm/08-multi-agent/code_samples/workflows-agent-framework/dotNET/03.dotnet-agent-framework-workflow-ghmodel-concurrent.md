### 🔍 Wetin Be Custom Executors

**ConcurrentStartExecutor:**
- E dey use `IMessageHandler<string>` to accept string input
- E dey broadcast messages to all connected agents
- E dey send `TurnToken` to start concurrent processing

**ConcurrentAggregationExecutor:**
- E use `IMessageHandler<ChatMessage>` to receive agent responses
- E dey collect messages thread-safe way
- E dey aggregate when all expected results don come
- E go yield final output by `context.YieldOutputAsync()`

### ⚡ Performance Benefits

**Concurrent vs Sequential:**
- Sequential: Agent1 (30s) → Agent2 (30s) = **60 seconds total**
- Concurrent: Agent1 (30s) || Agent2 (30s) = **30 seconds total**

**Throughput better pass before**: Fit get N× faster for N concurrent agents (depend on workload and resources)

### 🛡️ Error Handling

The workflow fit handle single agent failure well:
- If one agent fail, others still dey work
- Aggregator fit run timeout logic
- E fit return partial results if e need am
