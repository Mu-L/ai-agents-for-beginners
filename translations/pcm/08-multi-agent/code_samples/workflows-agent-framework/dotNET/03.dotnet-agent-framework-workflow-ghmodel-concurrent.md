- **Concurrent Processing**: Multiple agents dey work parallel for one work
- **Fan-In**: All agent results dem collect and join am into one output

### 🔍 Wetin Custom Executors Mean

**ConcurrentStartExecutor:**
- Implemnet `IMessageHandler<string>` to accept string input
- Broadcast messages to all connected agents
- Send `TurnToken` to trigger concurrent processing

**ConcurrentAggregationExecutor:**
- Implement `IMessageHandler<ChatMessage>` to receive agent replies
- Collect messages safely inside thread
- Join when all expected answers don come
- Return final output with `context.YieldOutputAsync()`

### ⚡ Benefit For Performance

**Concurrent vs Sequential:**
- Sequential: Agent1 (30s) → Agent2 (30s) = **60 seconds total**
- Concurrent: Agent1 (30s) || Agent2 (30s) = **30 seconds total**

**Throughput improvement**: N× faster for N concurrent agents (if workload and resources allow)
