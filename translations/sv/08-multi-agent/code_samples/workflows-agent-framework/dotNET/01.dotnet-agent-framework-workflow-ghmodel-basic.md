# 🔄 Grundläggande Agentarbetsflöden med Azure OpenAI (Responses API) (.NET)

## 📋 Handledning för arbetsflödesorkestrering

Denna anteckningsbok visar hur du bygger avancerade **agentarbetsflöden** med Microsoft Agent Framework för .NET och Azure OpenAI (Responses API). Du lär dig att skapa flerstegs affärsprocesser där AI-agenter samarbetar för att utföra komplexa uppgifter genom strukturerade orkestreringsmönster.

## 🎯 Lärandemål

### 🏗️ **Grundläggande arbetsflödesarkitektur**
- **Workflow Builder**: Designa och orkestrera komplexa flerstegs AI-processer
- **Agentkoordination**: Koordinera flera specialiserade agenter inom arbetsflöden
- **Azure OpenAI (Responses API)**: Utnyttja Azure OpenAI Responses API i arbetsflöden
- **Visuell arbetsflödesdesign**: Skapa och visualisera arbetsflödesstrukturer för bättre förståelse

### 🔄 **Orkestreringsmönster för processer**
- **Sekventiell bearbetning**: Kedja ihop flera agentuppgifter i logisk ordning
- **Statushantering**: Behåll kontext och dataflöde över arbetsflödets steg
- **Felhanteing**: Implementera robust felåterställning och arbetsflödets motståndskraft
- **Prestandaoptimering**: Designa effektiva arbetsflöden för företagsnivå

### 🏢 **Företagsarbetsflödesapplikationer**
- **Affärsprocessautomation**: Automatisera komplexa organisationsarbetsflöden
- **Innehållsproduktionspipeline**: Redaktionella arbetsflöden med granskning och godkännande
- **Automatisering av kundservice**: Flerstegs kundförfrågningslösning
- **Databearbetningsarbetsflöden**: ETL-arbetsflöden med AI-drivna transformationer

## ⚙️ Förutsättningar & Setup

### 📦 **Nödvändiga NuGet-paket**

Denna arbetsflödesdemonstration använder flera viktiga .NET-paket:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI-konfiguration**

**Miljöinställning (.env-fil):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI-åtkomst:**
1. Skapa en Azure OpenAI-resurs i Azure-portalen
2. Distribuera en modell (till exempel `gpt-4.1-mini`) och notera distributionsnamnet
3. Logga in med `az login` och konfigurera miljövariabler som visas ovan

## 💻 Köra koden

Den kompletta implementeringen finns i `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Denna fil demonstrerar:

1. **Miljökonfiguration** - Laddar Azure OpenAI-konfiguration från `.env`-fil
2. **Azure OpenAI-klientinställning** - Konfigurera klienten för att använda Azure OpenAI Responses API
3. **Agentskapande** - Definiera specialiserade agenter (Front Desk och Concierge)
4. **Workflow Builder** - Skapa ett fleragenters arbetsflöde med sekventiell bearbetning
5. **Arbetsflödeskörning** - Köra arbetsflödet med strömmande resultat

### 🚀 Köra exemplet

```bash
# Gör skriptet körbart (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Kör arbetsflödet
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Eller på Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Förväntad utdata

Arbetsflödet kommer att:
1. Ta emot din resebegäran ("Jag vill åka till Paris")
2. Front Desk-agenten ger en första rekommendation
3. Concierge-agenten granskar och förfinar rekommendationen
4. Slutlig utdata visar hela konversationsströmmen

### 🔧 Anpassning

Du kan anpassa arbetsflödet genom att:
- Modifiera agentinstruktioner för att ändra deras beteende
- Lägga till fler agenter för att skapa komplexa flerstegsarbetsflöden
- Ändra användarmeddelandet för att testa olika scenarier
- Justera arbetsflödets kanter för att skapa olika exekveringsmönster
