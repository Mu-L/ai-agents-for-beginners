# ⚡ Samtidiga Agentarbetsflöden med Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial för högpresterande parallell bearbetning

Denna anteckningsbok demonstrerar **samtidiga arbetsflödesmönster** med Microsoft Agent Framework för .NET och Azure OpenAI (Responses API). Du kommer att lära dig hur man bygger högpresterande arbetsflöden för parallell bearbetning som maximerar genomströmning genom att köra flera AI-agenter samtidigt samtidigt som samordning och datakonsistens upprätthålls.

## 💻 Köra koden

Den kompletta implementeringen finns i `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Denna fil visar ett **Fan-Out/Fan-In samtidigt arbetsflöde** för reseplanering:

### 🏗️ **Arbetsflödesarkitektur**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Nyckelkomponenter:**

1. **ConcurrentStartExecutor**: Sänder användarförfrågan till alla agenter samtidigt
2. **Researcher Agent**: Analyserar destinationer och sevärdheter parallellt
3. **Planner Agent**: Skapar detaljerade reseplaner parallellt
4. **ConcurrentAggregationExecutor**: Samlar in och sammanställer resultat från båda agenterna

### 🎯 **Fan-Out/Fan-In-mönster**

Detta arbetsflöde demonstrerar det klassiska **Fan-Out/Fan-In**-mönstret:
- **Fan-Out**: Ett inmatningsmeddelande skickas till flera agenter samtidigt
- **Samtidig bearbetning**: Flera agenter arbetar parallellt med samma uppgift
- **Fan-In**: Resultat från alla agenter samlas in och aggregeras till en enda utdata

### 📝 Förväntad utdata

Arbetsflödet kommer att:
1. **Sända förfrågan**: Skicka "Planera en resa till Seattle i december" till båda agenterna
2. **Samtidig bearbetning**: Båda agenter arbetar samtidigt:
   - Researcher identifierar sevärdheter och detaljer
   - Planner skapar resplan och logistik
3. **Aggregation**: Kombinera båda svaren till en omfattande output
4. **Visa resultat**: Visa den sammanslagna reseplanen med all information

### 🔍 Förstå anpassade exekutorer

**ConcurrentStartExecutor:**
- Implementerar `IMessageHandler<string>` för att ta emot strängindata
- Sänder meddelanden till alla anslutna agenter
- Skickar `TurnToken` för att trigga samtidig bearbetning

**ConcurrentAggregationExecutor:**
- Implementerar `IMessageHandler<ChatMessage>` för att ta emot agentrespons
- Samlar meddelanden trådsäkert
- Aggregerar när alla förväntade svar kommit
- Returnerar slutgiltig output via `context.YieldOutputAsync()`

### ⚡ Prestandafördelar

**Samtidig vs sekventiell:**
- Sekventiell: Agent1 (30s) → Agent2 (30s) = **totalt 60 sekunder**
- Samtidig: Agent1 (30s) || Agent2 (30s) = **totalt 30 sekunder**

**Genomströmningsförbättring**: Upp till N× snabbare för N samtidiga agenter (beroende på arbetsbelastning och resurser)

### 🛡️ Felhantering

Arbetsflödet hanterar individuella agentfel smidigt:
- Om en agent misslyckas fortsätter andra bearbetningen
- Aggregeraren kan implementera timeout-logik
- Partiella resultat kan returneras vid behov
