## 💻 Köra koden

Den kompletta implementationen finns i `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Denna fil demonstrerar ett **Fan-Out/Fan-In samtidigt arbetsflöde** för reseplanering:

### 🏗️ **Arbetsflödesarkitektur**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Viktiga komponenter:**

1. **ConcurrentStartExecutor**: Sänder användarförfrågan till alla agenter samtidigt
2. **Researcher Agent**: Analyserar destinationer och attraktioner parallellt
3. **Planner Agent**: Skapar detaljerade reseplaner parallellt
4. **ConcurrentAggregationExecutor**: Samlar och sammanställer resultat från båda agenterna

### 🎯 **Fan-Out/Fan-In-mönster**

Detta arbetsflöde demonstrerar det klassiska **Fan-Out/Fan-In**-mönstret:
- **Fan-Out**: Ett indata-meddelande sänds ut till flera agenter samtidigt
- **Samtidig bearbetning**: Flera agenter arbetar parallellt på samma uppgift
- **Fan-In**: Resultaten från alla agenter samlas in och aggregeras till en enda output

### 🔍 Förstå anpassade exekutorer

**ConcurrentStartExecutor:**
- Implementerar `IMessageHandler<string>` för att acceptera strängindata
- Sänder meddelanden till alla anslutna agenter
- Skickar `TurnToken` för att trigga samtidiga bearbetningar

**ConcurrentAggregationExecutor:**
- Implementerar `IMessageHandler<ChatMessage>` för att ta emot agenters svar
- Samlar meddelanden trådsäkert
- Sammanställer när alla förväntade svar har kommit
- Returnerar slutgiltigt output med `context.YieldOutputAsync()`

### ⚡ Prestandafördelar

**Samtidigt vs Sekventiellt:**
- Sekventiellt: Agent1 (30s) → Agent2 (30s) = **60 sekunder totalt**
- Samtidigt: Agent1 (30s) || Agent2 (30s) = **30 sekunder totalt**

**Genomströmningsförbättring**: Upp till N× snabbare för N samtidiga agenter (beroende på arbetsbelastning och resurser)

### 🛡️ Felhantering

Arbetsflödet hanterar individuella agentfel smidigt:
- Om en agent misslyckas fortsätter de andra bearbetningen
- Aggregator kan implementera timeout-logik
- Delvisa resultat kan returneras vid behov
