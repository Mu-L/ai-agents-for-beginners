# ⏩ Sekventiella agentarbetsflöden med Azure OpenAI (Responses API) (.NET)

## 📋 Avancerad handledning för sekventiell bearbetning

Denna anteckningsbok demonstrerar **sekventiella arbetsflödesscheman** med Microsoft Agent Framework för .NET och Azure OpenAI (Responses API). Du kommer att lära dig hur man bygger sofistikerade, steg-för-steg bearbetningspipelines där agenter utför i en specifik ordning, där varje steg bygger på resultaten från föregående steg.

## 🎯 Lärandemål

### 🔄 **Arkitektur för sekventiell bearbetning**
- **Linjeärt arbetsflödesdesign**: Skapa steg-för-steg bearbetningspipelines med tydliga beroenden
- **Tillståndshantering**: Behåll kontext och dataflöde över sekventiella arbetsflödesskeden
- **Azure OpenAI (Responses API)**: Utnyttja Azure OpenAI-modeller i flerstegs .NET-arbetsflöden
- **Företagspipelinescheman**: Bygg produktionsklara sekventiella bearbetningssystem

### 🏗️ **Avancerade sekventiella mönster**
- **Stage-Gate-bearbetning**: Implementera valideringskontroller mellan arbetsflödesskeden
- **Kontextbevarande**: Behåll tillstånd och ackumulerad kunskap genom alla steg
- **Felförmedling**: Hantera fel på ett snyggt sätt i sekventiella bearbetningskedjor
- **Prestandaoptimering**: Effektiv sekventiell exekvering med minimal overhead

### 🏢 **Sekventiella företagsapplikationer**
- **Dokumentbearbetningspipeline**: Flestegs dokumentanalys, transformation och validering
- **Kvalitetssäkringsarbetsflöden**: Sekventiell granskning, validering och godkännandeprocesser
- **Innehållsproduktionspipeline**: Forskning → Skrivande → Redigering → Granskning → Publicering
- **Automatisering av affärsprocesser**: Flerstegs affärsarbetsflöden med tydliga stegberoenden

## 💻 Köra koden

Den fullständiga implementeringen finns i `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Denna fil demonstrerar ett **trestegs arbetsflöde för möbelanalys**:

1. **Steg 1 - Försäljningsagent**: Analyserar möbelbilder och ger köpförslag
2. **Steg 2 - Prisagent**: Ger detaljerade prisuppdelningar och budgetalternativ
3. **Steg 3 - Offertagent**: Genererar ett professionellt offertdokument i Markdown-format

### 🏗️ **Arkitektur för arbetsflöde**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Varje agent:
- Tar emot utdata från föregående steg som kontext
- Bygger vidare på tidigare analys med specialiserad expertis
- Behåller arbetsflödets kontinuitet genom tillståndshantering

### 🚀 Köra exemplet

**Förutsättningar:**
- Placera en möbelbild på `../imgs/home.png` (eller uppdatera `imgPath`-variabeln)
- Konfigurera din `.env`-fil med din Azure OpenAI-endpoint och distribution, logga sedan in med `az login`

```bash
# Gör skriptet körbart (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Kör det sekventiella arbetsflödet
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Eller på Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Förväntad utdata

Arbetsflödet kommer att:
1. **Försäljningsagent**: Identifiera möbelobjekt från bilden och ge rekommendationer
2. **Prisagent**: Lägga till detaljerad prisanalys med budgetnivåer och inköpsrekommendationer
3. **Offertagent**: Generera ett formaterat offertetdokument med all information sammanställd

Slutresultatet blir en omfattande, professionell möbeloffert baserad på bildanalys.

### 🎯 Verkliga tillämpningar

Detta sekventiella mönster är idealiskt för:
- **E-handel**: Produktanalys → Prissättning → Offertgenerering
- **Fastigheter**: Fastighetsanalys → Värdering → Annonsskapande

### 🔍 Förstå tillståndsflödet

Varje agent i sekvensen tar emot:
- **Ursprunglig indatan**: Det inledande användarmeddelandet (bild + text)
- **Föregående agents utdata**: Alla tidigare agenters svar i konversationshistoriken
- **Ackumulerad kontext**: Fullständigt tillstånd som upprätthålls genom arbetsflödet

Detta möjliggör sofistikerad flerstegsbehandling där varje agent bygger vidare på omfattande kontext från alla föregående steg.
