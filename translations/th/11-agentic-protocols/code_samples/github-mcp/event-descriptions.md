ผ่านการเดินโค้ดอย่างละเอียด คุณจะเห็นวิธีการออกแบบและใช้งานเวิร์กโฟลว์ที่ขับเคลื่อนด้วยเอเย่นต์ที่ผสานการดึงข้อมูลและการสร้างโดยเสริม (RAG), การเก็บความจำถาวร, และการเปลี่ยนสถานะไดนามิก เราจะครอบคลุม:

การทำงานร่วมกันของเอเย่นต์ด้วย LangGraph สำหรับการคิดวิเคราะห์แบบมีโครงสร้าง
การเก็บประวัติการแชทแบบเรียลไทม์ด้วย Azure Cosmos DB - ฐานข้อมูลเดียวกับที่ใช้เก็บประวัติแชทใน ChatGPT แอปเอเย่นต์ AI ที่เติบโตเร็วที่สุดในประวัติศาสตร์
การค้นหาเวกเตอร์เพื่อดึงความรู้ด้วยการสนับสนุน embeddings ดั้งเดิมของ Cosmos DB
ความสามารถ async ของ FastAPI เพื่อให้การโต้ตอบมีความตอบสนองและขยายตัวได้
เมื่อจบเซสชันนี้ คุณจะมีแผนภาพชัดเจนในการสร้างและแจกจ่ายแอป AI หลายเอเย่นต์ที่ขยายตัวได้และเนทีฟกับคลาวด์ที่ใช้พลังของ AI สมัยใหม่และโครงสร้างพื้นฐานคลาวด์ ไม่ว่าคุณจะเป็นวิศวกร AI, สถาปนิกคลาวด์ หรือผู้พัฒนา Python การบรรยายนี้จะมอบความรู้เชิงปฏิบัติและรูปแบบที่ผ่านการทดสอบเพื่อสร้างแอป AI รุ่นถัดไป

## คำอธิบาย

jQuery เคยทำให้การพัฒนาเว็บง่ายขึ้นโดยการแยกความซับซ้อนออก ช่วยให้นักพัฒนามุ่งเน้นการสร้างแทนการจัดการกับปัญหาตัวแปรของเบราว์เซอร์ วันนี้ การพัฒนา AI เผชิญกับความท้าทายคล้ายกัน รูปแบบใหม่เกิดขึ้นตลอดเวลาและการตามให้ทันอาจท่วมท้น โดยเฉพาะอย่างยิ่งเมื่อเครื่องมือ AI โดยเฉพาะเอเย่นต์กลายเป็นทรงพลังและซับซ้อนกว่าเดิม ถ้าคุณสามารถใช้ความสามารถ AI ขั้นสูงเพื่อทำงานอัตโนมัติด้วยนามธรรม JavaScript ที่เรียบง่ายและคุ้นเคย? ขอแนะนำ GenAIScript — วิธีผสมผสาน AI เข้ากับเวิร์กโฟลว์ของคุณอย่างง่ายดาย โดยปฏิบัติ prompt เป็นเหมือนโค้ดซ้ำใช้ได้ ในบทพูดคุยนี้ เราจะสำรวจว่า GenAIScript ทำให้เอเย่นต์อัตโนมัติ AI รู้สึกเป็นธรรมชาติในการเขียน JavaScript อย่างไร ช่วยคุณลดงานซ้ำซ้อนโดยไม่ต้องมีความเชี่ยวชาญ AI ลึกๆ

## ชื่องาน: Semantic Kernel กับ C# เพื่อสร้างแอป AI หลายเอเย่นต์ใช้พลังจาก Azure Cosmos

## คำอธิบาย

เราจะนำทางคุณผ่านแอปหลายเอเย่นต์ใน C# ที่สร้างบน Semantic Kernel framework คุณจะเข้าใจแนวคิดเบื้องหลังแอปเอเย่นต์ เข้าใจรายละเอียดการใช้งานและความละเอียดอ่อน และเรียนรู้วิธีผนวก Azure Cosmos DB เป็นฐานข้อมูลสำหรับกรณีการใช้งานต่างๆ

## URL
<https://developer.microsoft.com/en-us/reactor/events/25455/>
