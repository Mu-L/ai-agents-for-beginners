# หลักการออกแบบ AI Agentic

## บทนำ

มีหลายวิธีในการคิดเกี่ยวกับการสร้างระบบ AI Agentic เนื่องจากความไม่แน่นอนเป็นคุณสมบัติไม่ใช่ข้อบกพร่องในการออกแบบ Generative AI จึงบางครั้งวิศวกรจึงยากที่จะรู้ว่าจะเริ่มต้นอย่างไร เราได้สร้างชุดหลักการออกแบบ UX ที่เน้นมนุษย์ เพื่อช่วยให้นักพัฒนาสามารถสร้างระบบ agentic ที่มุ่งเน้นลูกค้าเพื่อตอบสนองความต้องการทางธุรกิจของพวกเขา หลักการออกแบบเหล่านี้ไม่ใช่สถาปัตยกรรมที่บังคับใช้แต่เป็นจุดเริ่มต้นสำหรับทีมที่กำลังกำหนดและสร้างประสบการณ์ agent

โดยทั่วไป ตัวแทนควร:

- ขยายและเพิ่มขีดความสามารถของมนุษย์ (ระดมความคิด แก้ปัญหา อัตโนมัติ ฯลฯ)
- เติมเต็มช่องว่างความรู้ (ช่วยให้ฉันเข้าใจเรื่องเฉพาะได้เร็วขึ้น การแปล ฯลฯ)
- สนับสนุนและช่วยในการทำงานร่วมกันในวิธีที่เราแต่ละคนชอบทำงานร่วมกับผู้อื่น
- ทำให้เราเป็นเวอร์ชันที่ดีขึ้นของตัวเอง (เช่น โค้ชชีวิต/ผู้ควบคุมงาน ช่วยให้เราเรียนรู้การควบคุมอารมณ์และทักษะการมีสติสร้างความยืดหยุ่น ฯลฯ)

## สิ่งที่จะครอบคลุมในบทเรียนนี้

- หลักการออกแบบ Agentic คืออะไร
- แนวทางบางอย่างที่ควรปฏิบัติเมื่อใช้หลักการออกแบบเหล่านี้
- ตัวอย่างการใช้หลักการออกแบบ

## เป้าหมายการเรียนรู้

หลังจากจบบทเรียนนี้ คุณจะสามารถ:

1. อธิบายว่าหลักการออกแบบ Agentic คืออะไร
2. อธิบายแนวทางการใช้หลักการออกแบบ Agentic
3. เข้าใจวิธีสร้างตัวแทนโดยใช้หลักการออกแบบ Agentic

## หลักการออกแบบ Agentic

![หลักการออกแบบ Agentic](../../../translated_images/th/agentic-design-principles.1cfdf8b6d3cc73c2.webp)

### Agent (พื้นที่)

นี่คือสภาพแวดล้อมที่ตัวแทนทำงาน หลักการเหล่านี้แจ้งวิธีออกแบบตัวแทนสำหรับการมีส่วนร่วมในโลกทั้งทางกายภาพและดิจิทัล

- **เชื่อมต่อ ไม่ใช่ยุบรวม** – ช่วยเชื่อมผู้คนกับคนอื่น เหตุการณ์ และความรู้ที่นำไปปฏิบัติได้ เพื่อเอื้อต่อการทำงานร่วมกันและการเชื่อมต่อ
- ตัวแทนช่วยเชื่อมเหตุการณ์ ความรู้ และผู้คน
- ตัวแทนนำผู้คนเข้ามาใกล้กันมากขึ้น ไม่ได้ถูกออกแบบมาเพื่อแทนที่หรือดูถูกผู้คน
- **เข้าถึงง่ายแต่บางครั้งมองไม่เห็น** – ตัวแทนทำงานส่วนใหญ่เบื้องหลังและจะแสดงขึ้นเมื่อตรงกับบริบทและเหมาะสมเท่านั้น
  - ตัวแทนค้นหาและเข้าถึงได้ง่ายสำหรับผู้ใช้ที่ได้รับอนุญาตบนอุปกรณ์หรือแพลตฟอร์มใด ๆ
  - ตัวแทนรองรับการป้อนข้อมูลและผลลัพธ์แบบหลายโหมด (เสียง พูด ข้อความ ฯลฯ)
  - ตัวแทนสามารถเปลี่ยนสถานะระหว่างหน้าจอและเบื้องหลัง ระหว่างเชิงรุกและตอบสนอง ได้อย่างไร้รอยต่อ ขึ้นอยู่กับการรับรู้ความต้องการผู้ใช้
  - ตัวแทนอาจทำงานในรูปแบบที่มองไม่เห็น แต่กระบวนการเบื้องหลังและการทำงานร่วมกับตัวแทนอื่นโปร่งใสและควบคุมได้โดยผู้ใช้

### Agent (เวลา)

นี่คือวิธีที่ตัวแทนทำงานตามเวลา หลักการเหล่านี้แจ้งวิธีออกแบบตัวแทนที่โต้ตอบข้ามอดีต ปัจจุบัน และอนาคต

- **อดีต**: สะท้อนประวัติศาสตร์ที่รวมทั้งสถานะและบริบท
  - ตัวแทนให้ผลลัพธ์ที่เกี่ยวข้องมากขึ้นโดยอิงวิเคราะห์ข้อมูลประวัติศาสตร์ที่หลากหลายมากกว่าสิ่งที่เกิดขึ้น ผู้คน หรือสถานะเพียงอย่างเดียว
  - ตัวแทนสร้างการเชื่อมโยงจากเหตุการณ์ในอดีตและสะท้อนความทรงจำอย่างกระตือรือร้นเพื่อตอบสนองสถานการณ์ปัจจุบัน
- **ปัจจุบัน**: สะกิดมากกว่าการแจ้งเตือน
  - ตัวแทนเป็นตัวแทนของวิธีการครอบคลุมในการโต้ตอบกับผู้คน เมื่อเกิดเหตุการณ์ ตัวแทนจะเกินกว่าการแจ้งเตือนแบบคงที่หรือลักษณะทางการแบบอื่น ตัวแทนสามารถทำให้กระบวนการง่ายขึ้นหรือสร้างเบาะแสแบบไดนามิกเพื่อดึงดูดความสนใจผู้ใช้ในเวลาที่เหมาะสม
  - ตัวแทนส่งข้อมูลโดยอิงสภาพแวดล้อมบริบท การเปลี่ยนแปลงทางสังคมและวัฒนธรรม และปรับแต่งตามเจตนาของผู้ใช้
  - การโต้ตอบกับตัวแทนอาจเป็นแบบค่อยเป็นค่อยไป พัฒนา/เพิ่มความซับซ้อนเพื่อเสริมพลังให้ผู้ใช้ในระยะยาว
- **อนาคต**: ปรับตัวและพัฒนา
  - ตัวแทนปรับตัวกับอุปกรณ์ แพลตฟอร์ม และรูปแบบต่าง ๆ
  - ตัวแทนปรับตัวเข้ากับพฤติกรรมผู้ใช้ ความต้องการการเข้าถึง และสามารถปรับแต่งได้อย่างอิสระ
  - ตัวแทนถูกกำหนดรูปแบบและพัฒนาอย่างต่อเนื่องผ่านการโต้ตอบกับผู้ใช้

### Agent (แกนกลาง)

เหล่านี้คือองค์ประกอบสำคัญในแกนกลางของการออกแบบตัวแทน

- **ยอมรับความไม่แน่นอนแต่สร้างความไว้วางใจ**
  - ระดับหนึ่งของความไม่แน่นอนของตัวแทนเป็นสิ่งที่คาดหวัง ความไม่แน่นอนเป็นองค์ประกอบสำคัญของการออกแบบตัวแทน
  - ความไว้วางใจและความโปร่งใสเป็นชั้นพื้นฐานของการออกแบบตัวแทน
  - มนุษย์ควบคุมเมื่อเปิด/ปิดตัวแทน และสถานะตัวแทนจะมองเห็นได้ชัดเจนตลอดเวลา

## แนวทางในการนำหลักการเหล่านี้ไปใช้

เมื่อคุณใช้หลักการออกแบบก่อนหน้า ให้ใช้แนวทางต่อไปนี้:

1. **ความโปร่งใส**: แจ้งผู้ใช้ว่า AI เข้ามามีส่วนร่วมอย่างไร ทำงานอย่างไร (รวมถึงการกระทำในอดีต) และวิธีให้ข้อเสนอแนะและแก้ไขระบบ
2. **การควบคุม**: เปิดโอกาสให้ผู้ใช้ปรับแต่ง ระบุความชอบและปรับเปลี่ยนตนเอง และควบคุมระบบและคุณลักษณะต่าง ๆ (รวมถึงความสามารถในการลืม)
3. **ความสม่ำเสมอ**: มุ่งหวังประสบการณ์ที่สม่ำเสมอและหลายโหมดข้ามอุปกรณ์และปลายทาง ใช้องค์ประกอบ UI/UX ที่คุ้นเคยเมื่อเป็นไปได้ (เช่น ไอคอนไมโครโฟนสำหรับการโต้ตอบด้วยเสียง) และลดภาระการรับรู้ของลูกค้าให้มากที่สุด (เช่น ตอบสั้นกระชับ ใช้ภาพช่วย และเนื้อหา ‘เรียนรู้เพิ่มเติม’)

## วิธีออกแบบตัวแทนการเดินทางโดยใช้หลักการและแนวทางเหล่านี้

สมมติว่าคุณกำลังออกแบบตัวแทนการเดินทาง นี่คือวิธีคิดเกี่ยวกับการใช้หลักการและแนวทางออกแบบ:

1. **ความโปร่งใส** – แจ้งให้ผู้ใช้ทราบว่าตัวแทนการเดินทางเป็นตัวแทนที่ใช้ AI ให้คำแนะนำพื้นฐานเกี่ยวกับวิธีเริ่มต้น (เช่น ข้อความ “สวัสดี” คำสั่งตัวอย่าง) เอกสารนี้ชัดเจนในหน้าผลิตภัณฑ์ แสดงรายการคำสั่งที่ผู้ใช้เคยส่งในอดีต แสดงวิธีให้ข้อเสนอแนะอย่างชัดเจน (ไอคอนถูกใจและไม่ถูกใจ ปุ่มส่งข้อเสนอแนะ ฯลฯ) ระบุชัดเจนถ้าตัวแทนมีข้อจำกัดในการใช้งานหรือหัวข้อ
2. **การควบคุม** – ทำให้แน่ใจว่าผู้ใช้ทราบวิธีแก้ไขตัวแทนหลังจากสร้างด้วยสิ่งต่าง ๆ เช่น System Prompt ให้ผู้ใช้เลือกความยาวของคำตอบ รูปแบบการเขียน และข้อจำกัดเกี่ยวกับสิ่งที่ไม่ควรพูด อนุญาตให้ผู้ใช้ดูและลบไฟล์หรือข้อมูลที่เกี่ยวข้อง คำสั่ง และบทสนทนาในอดีตได้
3. **ความสม่ำเสมอ** – ให้แน่ใจว่าไอคอนสำหรับ Share Prompt เพิ่มไฟล์หรือรูปภาพ และแท็กคนหรือสิ่งต่าง ๆ เป็นมาตรฐานและจดจำได้ ใช้ไอคอนคลิปหนีบกระดาษเพื่อระบุการอัปโหลด/แชร์ไฟล์กับตัวแทน และไอคอนรูปภาพเพื่อบ่งชี้การอัปโหลดกราฟิก
