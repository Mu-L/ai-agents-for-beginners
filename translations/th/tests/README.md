# การทดสอบ Smoke ของเอเจนต์

โฟลเดอร์นี้เก็บ **แคตตาล็อกการทดสอบ smoke** สำหรับเอเจนต์ที่คุณสร้างในหลักสูตร
การทดสอบ smoke คือการตรวจสอบอย่างรวดเร็วและราคาถูกว่า **เอเจนต์ที่โฮสต์โดย Microsoft Foundry ที่ถูกติดตั้ง**
นั้นสามารถเข้าถึง ตอบสนอง และปฏิบัติตามความคาดหวังพื้นฐานที่สุดของพรอมต์ได้หรือไม่ เป็นเกตแรก — ไม่ใช่การแทนที่สำหรับกระบวนการประเมินผลแบบเต็ม
ที่คุณเรียนรู้ใน [บทที่ 10](../10-ai-agents-production/README.md) และ
[บทที่ 16](../16-deploying-scalable-agents/README.md)


แคตตาล็อกเหล่านี้ถูกใช้งานโดย [AI Smoke Test](https://github.com/marketplace/actions/ai-smoke-test)
GitHub Action ผ่านเวิร์กโฟลว์ [`.github/workflows/smoke-test.yml`](../../../.github/workflows/smoke-test.yml)


## วิธีการรัน

1. **ติดตั้งเอเจนต์ของบทเรียน** เป็นเอเจนต์ที่โฮสต์ใน Microsoft Foundry (ดู
   บทที่ 16 สำหรับเวิร์กโฟลว์การติดตั้ง) จดจำ **ชื่อเอเจนต์** และ
   **จุดสิ้นสุดโครงการ Foundry** ของคุณ
2. เพิ่มความลับของที่เก็บนี้ (การตั้งค่า → ความลับและตัวแปร → การดำเนินการ):
   `AZURE_CLIENT_ID`, `AZURE_TENANT_ID`, `AZURE_SUBSCRIPTION_ID` สิทธิ์ตัวตนแบบฟีดเดอเรต
   ต้องมีบทบาท **Azure AI User** ในขอบเขตโครงการ Foundry
3. จากแท็บ **Actions** รัน **Smoke-test hosted agents** และเลือก
   `tests_file` สำหรับบทเรียน จากนั้นจึงระบุ `agent_name` และ
   `project_endpoint` ที่ตรงกัน

## บทเรียนใดบ้างที่มีการทดสอบ smoke?

การทดสอบ smoke ใช้กับบทเรียนที่คุณ **ติดตั้งเอเจนต์** ซึ่งข้อความตอบกลับสามารถ
ถูกตรวจสอบกับเนื้อหาที่ทราบได้ บทเรียนที่เน้นแนวคิด ดำเนินการเฉพาะในเครื่องของคุณ
หรือสร้างผลงานที่ไม่สามารถกำหนดได้อย่างแน่นอน จะถูกยกเว้นโดยเจตนา:

- **บทที่ 17 (การสร้างเอเจนต์ AI ในเครื่อง)** ทำงานทั้งหมดบนเวิร์กสเตชันของคุณด้วย
  Foundry Local และ **ไม่** เปิดเผยจุดสิ้นสุด Foundry Responses ดังนั้น
  การดำเนินการนี้จึงไม่ใช้กับบทนี้ ใช้การตรวจสอบโดยการรันโน้ตบุ๊กในเครื่องแทน
- บทเรียนแนวรูปแบบการออกแบบและทฤษฎี (02, 03, 06, 07, 09, 12) ไม่ได้ส่งมอบ
  เอเจนต์ที่ติดตั้งได้ตัวใดเลยเพื่อทดสอบ smoke

## โครงสร้างแคตตาล็อก (อ้างอิงด่วน)

แต่ละแคตตาล็อกเป็นเอกสาร JSON ที่มีอาร์เรย์ `tests` ชั้นบนสุด แต่ละรายการส่ง POST
พรอมต์หนึ่งรายการและตรวจสอบการตอบกลับ:

| ฟิลด์ | ความหมาย |
|-------|---------|
| `id` | ตัวระบุขั้นตอนที่ไม่ซ้ำซ้อนแสดงในบันทึก |
| `description` | วัตถุประสงค์ที่อ่านเข้าใจได้โดยมนุษย์ |
| `prompt` | ข้อความที่ส่งถึงเอเจนต์ |
| `assertions.status` | สถานะ HTTP ที่คาดหวัง (ค่าเริ่มต้น 200) |
| `assertions.contains_any` | ผ่านถ้าคำตอบมีสตริงย่อยใดๆ เหล่านี้ |
| `assertions.contains_all` | ผ่านถ้าคำตอบมีสตริงย่อยทุกอัน |
| `assertions.contains_none` | ผ่านถ้าคำตอบไม่มีสตริงย่อยใดๆ เหล่านี้ |
| `save_response_id_as` | เก็บ id การตอบกลับเพื่อใช้ในขั้นตอนหลายรอบถัดไป |
| `use_previous_response_id` | ส่งรอบนี้โดยต่อเนื่องกับ id การตอบกลับที่เก็บไว้ |

การตรวจสอบเป็นการเช็คสตริงย่อยแบบไม่สนใจตัวพิมพ์ ดู
[เอกสารการใช้งาน](https://github.com/marketplace/actions/ai-smoke-test) สำหรับ
โครงสร้างเต็ม รวมถึงทรัพยากรการสนทนาที่จัดการโดย Foundry
