# 🔍 Enterprise RAG กับ Microsoft Foundry (.NET)

## 📋 วัตถุประสงค์การเรียนรู้

โน้ตบุ๊กนี้สาธิตวิธีการสร้างระบบ Retrieval-Augmented Generation (RAG) ระดับองค์กรโดยใช้ Microsoft Agent Framework ใน .NET กับ Microsoft Foundry คุณจะได้เรียนรู้การสร้างเอเย่นต์ที่พร้อมใช้งานในผลิตภัณฑ์ซึ่งสามารถค้นหาผ่านเอกสารและให้คำตอบที่แม่นยำตามบริบท พร้อมด้วยความปลอดภัยและการขยายตัวระดับองค์กร

**ความสามารถของ Enterprise RAG ที่คุณจะสร้าง:**
- 📚 **ความชาญฉลาดของเอกสาร**: การประมวลผลเอกสารขั้นสูงด้วยบริการ Azure AI
- 🔍 **การค้นหาเชิงความหมาย**: การค้นหาเวกเตอร์ประสิทธิภาพสูงพร้อมคุณสมบัติระดับองค์กร
- 🛡️ **การผนวกรวมความปลอดภัย**: การเข้าถึงตามบทบาทและรูปแบบการปกป้องข้อมูล
- 🏢 **สถาปัตยกรรมที่ปรับขนาดได้**: ระบบ RAG ที่พร้อมผลิตพร้อมระบบติดตาม

## 🎯 สถาปัตยกรรม Enterprise RAG

### องค์ประกอบหลักระดับองค์กร
- **Microsoft Foundry**: แพลตฟอร์ม AI ระดับองค์กรที่มีการจัดการพร้อมความปลอดภัยและการปฏิบัติตามกฎระเบียบ
- **เอเย่นต์ถาวร**: เอเย่นต์ที่มีสถานะพร้อมประวัติการสนทนาและการจัดการบริบท
- **การจัดการร้านเวกเตอร์**: การจัดทำดัชนีและดึงข้อมูลเอกสารระดับองค์กร
- **การผนวกรวมตัวตน**: การตรวจสอบสิทธิ์ Azure AD และการควบคุมการเข้าถึงตามบทบาท

### ประโยชน์ของ .NET ระดับองค์กร
- **ความปลอดภัยของประเภท**: การตรวจสอบความถูกต้องในเวลาคอมไพล์สำหรับการดำเนินงาน RAG และโครงสร้างข้อมูล
- **ประสิทธิภาพแบบอะซิงโครนัส**: การประมวลผลเอกสารและการค้นหาที่ไม่บล็อก
- **การจัดการหน่วยความจำ**: การใช้ทรัพยากรอย่างมีประสิทธิภาพสำหรับคอลเล็กชันเอกสารขนาดใหญ่
- **รูปแบบการผนวกรวม**: การผนวกรวมบริการ Azure แบบเนทีฟด้วยการฉีดพึ่งพา

## 🏗️ สถาปัตยกรรมทางเทคนิค

### สายงาน Enterprise RAG
```
Document Upload → Security Validation → Vector Processing → Index Creation
                      ↓                    ↓                  ↓
User Query → Authentication → Semantic Search → Context Ranking → AI Response
```

### องค์ประกอบหลักของ .NET
- **Azure.AI.Agents.Persistent**: การจัดการเอเย่นต์ระดับองค์กรพร้อมการรักษาสถานะ
- **Azure.Identity**: การตรวจสอบสิทธิ์ที่ผนวกรวมเพื่อการเข้าถึงบริการ Azure อย่างปลอดภัย
- **Microsoft.Agents.AI.AzureAI**: การใช้งานเฟรมเวิร์กเอเย่นต์ที่ปรับแต่งสำหรับ Azure
- **System.Linq.Async**: การปฏิบัติการ LINQ แบบอะซิงโครนัสที่มีประสิทธิภาพสูง

- **การบันทึกการตรวจสอบ**: การติดตามกิจกรรมอย่างครบถ้วนเพื่อความต้องการในการปฏิบัติตามกฎระเบียบ

### ประสิทธิภาพและการขยายตัว
- **การจัดการพูลการเชื่อมต่อ**: การบริหารจัดการการเชื่อมต่อบริการ Azure อย่างมีประสิทธิภาพ

- **กลยุทธ์แคชชิ่ง**: การแคชอย่างชาญฉลาดสำหรับเอกสารที่เข้าถึงบ่อย
- **การปรับสมดุลโหลด**: การประมวลผลแบบกระจายสำหรับการปรับใช้งานขนาดใหญ่

### การจัดการและการติดตาม
- **การตรวจสอบสุขภาพ**: การติดตามระบบขององค์ประกอบ RAG ในตัว
- **เมตริกประสิทธิภาพ**: การวิเคราะห์เชิงลึกเกี่ยวกับคุณภาพการค้นหาและเวลาตอบสนอง
- **การจัดการข้อผิดพลาด**: การจัดการข้อยกเว้นอย่างครอบคลุมพร้อมนโยบายการลองใหม่
- **การจัดการการกำหนดค่า**: การตั้งค่าสิ่งแวดล้อมเฉพาะพร้อมการตรวจสอบความถูกต้อง

## 📊 แบบแผน Enterprise RAG

### แบบแผนการจัดการเอกสาร
- **การอัปโหลดจำนวนมาก**: การประมวลผลอย่างมีประสิทธิภาพของคอลเล็กชันเอกสารขนาดใหญ่

- **การจัดการข้อมูลเมตา**: คุณลักษณะเอกสารที่หลากหลายและการจัดระบบหมวดหมู่

- **การค้นหาแบบไฮบริด**: การผสมผสานการค้นหาเชิงความหมายและคำสำคัญเพื่อผลลัพธ์ที่ดีที่สุด

- **การจัดอันดับผลลัพธ์**: การจัดอันดับขั้นสูงพร้อมการผนวกรวมตรรกะธุรกิจ

- **ความปลอดภัยระดับเอกสาร**: การควบคุมการเข้าถึงอย่างละเอียดสำหรับแต่ละเอกสาร
