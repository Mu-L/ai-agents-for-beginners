# 🤝 ระบบเวิร์กโฟลว์หลายเอเจนต์สำหรับองค์กร (.NET)

## 📋 วัตถุประสงค์การเรียนรู้

โน้ตบุ๊กนี้แสดงวิธีการสร้างระบบหลายเอเจนต์ระดับองค์กรที่ซับซ้อนโดยใช้ Microsoft Agent Framework ใน .NET ร่วมกับ Azure OpenAI (Responses API) คุณจะได้เรียนรู้การประสานงานของเอเจนต์เฉพาะทางหลายตัวที่ทำงานร่วมกันผ่านเวิร์กโฟลว์ที่มีโครงสร้าง โดยใช้คุณสมบัติของ .NET สำหรับองค์กรเพื่อโซลูชันที่พร้อมใช้งานจริง

**ความสามารถของระบบหลายเอเจนต์สำหรับองค์กรที่คุณจะสร้าง:**
- 👥 **การประสานงานเอเจนต์**: การประสานงานเอเจนต์ที่ปลอดภัยด้วยประเภทที่ตรวจสอบในเวลาคอมไพล์
- 🔄 **การจัดการเวิร์กโฟลว์**: การกำหนดเวิร์กโฟลว์แบบประกาศด้วยรูปแบบ async ของ .NET
- 🎭 **ความเชี่ยวชาญเฉพาะบทบาท**: บุคลิกและโดเมนความเชี่ยวชาญของเอเจนต์ที่มีประเภทเข้มงวด
- 🏢 **การผสานรวมสำหรับองค์กร**: รูปแบบพร้อมใช้งานจริงพร้อมการตรวจสอบและจัดการข้อผิดพลาด

## ตัวอย่างโค้ด

โค้ดทำงานครบถ้วนสำหรับบทเรียนนี้สามารถหาได้ในไฟล์ C# ที่แนบมาชื่อ: [`08-dotnet-agent-framework.cs`](../../../../08-multi-agent/code_samples/08-dotnet-agent-framework.cs)

ในการรันตัวอย่าง:


หรือใช้ .NET CLI:

```bash
dotnet run 08-dotnet-agent-framework.cs
```

## สิ่งที่ตัวอย่างนี้สาธิต

ระบบเวิร์กโฟลว์หลายเอเจนต์นี้สร้างบริการแนะนำการเดินทางโรงแรมโดยมีเอเจนต์เฉพาะทางสองตัว:

1. **เอเจนต์ FrontDesk**: เอเจนต์ท่องเที่ยวที่แนะนำกิจกรรมและสถานที่
2. **เอเจนต์ Concierge**: ตรวจสอบคำแนะนำเพื่อให้มั่นใจในประสบการณ์ที่แท้จริงและไม่ใช่นักท่องเที่ยวมากเกินไป

เอเจนต์ทำงานร่วมกันในเวิร์กโฟลว์ที่:
- เอเจนต์ FrontDesk รับคำขอการเดินทางเริ่มต้น
- เอเจนต์ Concierge ตรวจสอบและปรับปรุงคำแนะนำ
- เวิร์กโฟลว์สตรีมการตอบสนองแบบเรียลไทม์

## แนวคิดสำคัญ

### การประสานงานเอเจนต์
ตัวอย่างแสดงการประสานงานเอเจนต์ที่ปลอดภัยด้วยประเภทโดยใช้ Microsoft Agent Framework พร้อมการตรวจสอบในเวลาคอมไพล์

### การจัดการเวิร์กโฟลว์
ใช้การกำหนดเวิร์กโฟลว์แบบประกาศด้วยรูปแบบ async ของ .NET เพื่อเชื่อมต่อเอเจนต์หลายตัวในสายงาน

### การสตรีมการตอบสนอง
นำเสนอการสตรีมแบบเรียลไทม์ของการตอบสนองจากเอเจนต์โดยใช้ async enumerables และสถาปัตยกรรมอิงเหตุการณ์

### การผสานรวมสำหรับองค์กร
แสดงรูปแบบพร้อมใช้งานจริงรวมถึง:
- การกำหนดค่าตัวแปรสภาพแวดล้อม
- การจัดการข้อมูลรับรองอย่างปลอดภัย
- การจัดการข้อผิดพลาด
- การประมวลผลเหตุการณ์แบบอะซิงโครนัส
