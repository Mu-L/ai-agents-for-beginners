# ⏩ การทำงานของเอเย่นต์แบบลำดับด้วย Azure OpenAI (Responses API) (.NET)

## 📋 บทเรียนการประมวลผลแบบลำดับขั้นสูง

สมุดบันทึกนี้สาธิต **รูปแบบการทำงานแบบลำดับ** โดยใช้ Microsoft Agent Framework สำหรับ .NET และ Azure OpenAI (Responses API) คุณจะได้เรียนรู้วิธีสร้างสายการประมวลผลที่ซับซ้อนเป็นขั้นตอน ซึ่งเอเย่นต์จะทำงานตามลำดับที่กำหนด โดยแต่ละขั้นตอนจะสร้างขึ้นจากผลลัพธ์ของขั้นตอนก่อนหน้า

## 🎯 วัตถุประสงค์การเรียนรู้

### 🔄 **สถาปัตยกรรมการประมวลผลแบบลำดับ**
- **การออกแบบเวิร์กโฟลว์เชิงเส้น**: สร้างสายการประมวลผลเป็นขั้นตอนโดยมีความสอดคล้องชัดเจน
- **การจัดการสถานะ**: รักษาเนื้อหาและการไหลของข้อมูลข้ามขั้นตอนของเวิร์กโฟลว์แบบลำดับ
- **Azure OpenAI (Responses API)**: ใช้ประโยชน์จากโมเดล Azure OpenAI ในเวิร์กโฟลว์แบบหลายขั้นตอนบน .NET
- **รูปแบบสายงานระดับองค์กร**: สร้างระบบประมวลผลแบบลำดับที่พร้อมใช้งานในงานจริง

### 🏗️ **รูปแบบขั้นสูงของการทำงานแบบลำดับ**
- **การประมวลผลแบบ Stage-Gate**: ใช้จุดตรวจสอบการตรวจสอบระหว่างขั้นตอนเวิร์กโฟลว์
- **การเก็บรักษาเนื้อหา**: รักษาสถานะและความรู้ที่สะสมในทุกขั้นตอน
- **การแพร่กระจายข้อผิดพลาด**: จัดการความล้มเหลวอย่างราบรื่นในโซ่การประมวลผลแบบลำดับ
- **การเพิ่มประสิทธิภาพการทำงาน**: การดำเนินการแบบลำดับอย่างมีประสิทธิภาพด้วยภาระงานน้อยที่สุด

### 🏢 **แอปพลิเคชันแบบลำดับในองค์กร**
- **สายการประมวลผลเอกสาร**: การวิเคราะห์เอกสารหลายขั้นตอน การแปลงข้อมูล และการตรวจสอบความถูกต้อง
- **เวิร์กโฟลว์การประกันคุณภาพ**: กระบวนการตรวจสอบ ตรวจสอบความถูกต้อง และอนุมัติโดยเป็นขั้นตอน
- **สายงานผลิตเนื้อหา**: การวิจัย → การเขียน → การแก้ไข → การตรวจทาน → การเผยแพร่
- **อัตโนมัติกระบวนการธุรกิจ**: เวิร์กโฟลว์ธุรกิจหลายขั้นตอนที่มีความสัมพันธ์ชัดเจนระหว่างขั้นตอน

## ⚙️ สิ่งที่ต้องมีและการตั้งค่า

### 📦 **แพ็กเกจ NuGet ที่จำเป็น**

แพ็กเกจที่จำเป็นสำหรับเวิร์กโฟลว์แบบลำดับบน .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **การตั้งค่า Azure OpenAI**

**การตั้งค่าสภาพแวดล้อม (.env ไฟล์):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**การจัดการการตั้งค่า:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **สถาปัตยกรรมเวิร์กโฟลว์แบบลำดับ**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: ตัวแทนวิเคราะห์]
    B --> C[จุดตรวจสอบ 1]
    C --> D[Stage 2: ตัวแทนประมวลผล]
    D --> E[จุดตรวจสอบ 2]
    E --> F[Stage 3: ตัวแทนตรวจสอบความถูกต้อง]
    F --> G[ผลลัพธ์ขั้นสุดท้าย]
    
    H[บริบทสถานะ] --> B

    I[Azure OpenAI (Responses API)] --> B
    I --> D
```

**ส่วนประกอบสำคัญ:**
- **เอเย่นต์แบบลำดับ**: เอเย่นต์เฉพาะแต่ละขั้นตอนการประมวลผล
- **บริบทสถานะ**: เก็บข้อมูลที่สะสมและการตัดสินใจข้ามขั้นตอน
- **จุดตรวจสอบ**: จุดตรวจสอบความถูกต้องระหว่างขั้นตอนเพื่อความคุณภาพและความสม่ำเสมอ
- **ไคลเอนต์ Azure OpenAI**: การเข้าถึงโมเดล AI อย่างสม่ำเสมอในทุกขั้นตอนของเวิร์กโฟลว์

## 🏢 **ประโยชน์ของระบบแบบลำดับในองค์กร**

### 🎯 **ความน่าเชื่อถือ & คุณภาพ**
- **การประมวลผลที่กำหนดได้แน่นอน**: ผลลัพธ์ที่สม่ำเสมอและทำซ้ำได้ผ่านขั้นตอนที่กำหนดโครงสร้าง
- **ประตูคุณภาพ**: จุดตรวจสอบที่รับประกันคุณภาพในแต่ละขั้นตอน
- **การแยกข้อผิดพลาด**: ปัญหาในขั้นตอนหนึ่งจะไม่ลามไปขั้นตอนถัดไป
- **ร่องรอยการตรวจสอบ**: การติดตามการตัดสินใจและการเปลี่ยนแปลงอย่างครบถ้วนในแต่ละขั้นตอน

### 📈 **ความสามารถในการปรับขนาด & ประสิทธิภาพ**
- **การออกแบบแบบโมดูลาร์**: แต่ละขั้นตอนสามารถเพิ่มประสิทธิภาพได้อย่างอิสระ
- **การจัดการทรัพยากร**: การจัดสรรทรัพยากรโมเดล AI อย่างมีประสิทธิภาพในแต่ละขั้นตอน
- **การเพิ่มประสิทธิภาพสถานะ**: การถ่ายโอนสถานะระหว่างขั้นตอนให้น้อยที่สุดเพื่อประสิทธิภาพสูงสุด
- **กลุ่มขั้นตอนคู่ขนาน**: เวิร์กโฟลว์แบบลำดับหลายชุดสามารถทำงานพร้อมกันได้

### 🔒 **ความปลอดภัย & การปฏิบัติตามข้อกำหนด**
- **ความปลอดภัยระดับขั้นตอน**: นโยบายความปลอดภัยต่างกันสำหรับแต่ละขั้นตอนการประมวลผล
- **การตรวจสอบข้อมูล**: รับรองความสมบูรณ์และการปฏิบัติตามข้อกำหนดของข้อมูลในแต่ละจุดตรวจสอบ
- **การควบคุมการเข้าถึง**: สิทธิ์เฉพาะเจาะจงสำหรับขั้นตอนเวิร์กโฟลว์แตกต่างกัน
- **การปฏิบัติตามระเบียบข้อบังคับ**: ตรงตามข้อกำหนดระเบียบผ่านการประมวลผลที่มีโครงสร้าง

### 📊 **การตรวจสอบ & การวิเคราะห์**
- **เมตริกระดับขั้นตอน**: การตรวจสอบประสิทธิภาพสำหรับแต่ละขั้นตอนของเวิร์กโฟลว์
- **การระบุจุดคับข้อง**: ตรวจจับและปรับปรุงขั้นตอนที่ช้า
- **เมตริกคุณภาพ**: ติดตามคุณภาพและอัตราความสำเร็จในแต่ละขั้นตอน
- **การเพิ่มประสิทธิภาพกระบวนการ**: ปรับปรุงอย่างต่อเนื่องโดยใช้การวิเคราะห์ระดับขั้นตอน

มาสร้างสายการประมวลผล AI แบบลำดับที่แข็งแกร่งกันเถอะ! 🚀

2. **เอเย่นต์ราคา**: เพิ่มการวิเคราะห์ราคาโดยละเอียดพร้อมระดับงบประมาณและคำแนะนำการซื้อ

### 🔧 ตัวเลือกการปรับแต่ง

**ปรับเปลี่ยนพฤติกรรมเอเย่นต์:**
```csharp
// Adjust agent instructions to change their focus
const string SalesAgentInstructions = "Your custom instructions...";
```

**เปลี่ยนการไหลแบบลำดับ:**
```csharp
// Add or reorder workflow stages
var workflow = new WorkflowBuilder(salesagent)
    .AddEdge(salesagent, priceagent)
    .AddEdge(priceagent, quoteagent)
    .AddEdge(quoteagent, newAgent)  // Add another stage
    .Build();
```

**ใช้ข้อมูลอินพุตที่แตกต่างกัน:**
```csharp
// Process text instead of images
ChatMessage userMessage = new ChatMessage(ChatRole.User, [
    new TextContent("Analyze pricing for a modern living room set")
]);
```

### 🎯 การใช้งานในโลกจริง

รูปแบบแบบลำดับนี้เหมาะสำหรับ:
- **อีคอมเมิร์ซ**: การวิเคราะห์สินค้า → การตั้งราคา → การสร้างใบเสนอราคา
- **อสังหาริมทรัพย์**: การวิเคราะห์ทรัพย์สิน → การประเมินค่า → การสร้างรายการขาย
- **ประกันภัย**: การวิเคราะห์เคลม → การประเมิน → การสร้างใบเสนอราคา
- **การสร้างเนื้อหา**: การวิจัย → การเขียน → การแก้ไข → การเผยแพร่

### 🔍 ความเข้าใจการไหลของสถานะ

เอเย่นต์แต่ละตัวในลำดับจะได้รับ:
- **อินพุตดั้งเดิม**: ข้อความจากผู้ใช้เริ่มต้น (ภาพ + ข้อความ)
- **ผลลัพธ์เอเย่นต์ก่อนหน้า**: การตอบกลับทั้งหมดจากเอเย่นต์ก่อนหน้าในประวัติการสนทนา
- **บริบทที่สะสม**: สถานะที่ครบถ้วนซึ่งรักษาผ่านเวิร์กโฟลว์ทั้งหมด
