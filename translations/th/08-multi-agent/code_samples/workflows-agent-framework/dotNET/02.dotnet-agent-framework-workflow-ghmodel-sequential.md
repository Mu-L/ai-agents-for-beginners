# ⏩ โฟลว์งานลำดับขั้นของเอเย่นต์ด้วย Azure OpenAI (Responses API) (.NET)

## 📋 บทช่วยสอนการประมวลผลลำดับขั้นขั้นสูง

โน้ตบุ๊กนี้แสดงตัวอย่าง **รูปแบบโฟลว์งานลำดับขั้น** โดยใช้ Microsoft Agent Framework สำหรับ .NET และ Azure OpenAI (Responses API) คุณจะได้เรียนรู้วิธีสร้างสายการประมวลผลแบบทีละขั้นตอนที่ซับซ้อน ซึ่งเอเย่นต์จะทำงานตามลำดับที่กำหนด โดยแต่ละขั้นตอนจะสร้างผลลัพธ์ต่อจากขั้นตอนก่อนหน้า

## 🎯 วัตถุประสงค์การเรียนรู้

### 🔄 **สถาปัตยกรรมการประมวลผลลำดับขั้น**
- **การออกแบบโฟลว์งานเชิงเส้น**: สร้างสายงานประมวลผลทีละขั้นตอนโดยมีความสัมพันธ์ชัดเจน
- **การจัดการสถานะ**: รักษาบริบทและการไหลของข้อมูลตลอดขั้นตอนโฟลว์งานลำดับขั้น
- **Azure OpenAI (Responses API)**: ใช้ประโยชน์จากโมเดล Azure OpenAI ในโฟลว์งานหลายขั้นตอนบน .NET
- **รูปแบบสายงานสำหรับองค์กร**: สร้างระบบประมวลผลลำดับขั้นพร้อมใช้งานในงานผลิตจริง

### 🏗️ **รูปแบบลำดับขั้นขั้นสูง**
- **การประมวลผลแบบ Stage-Gate**: ติดตั้งจุดตรวจสอบการตรวจสอบระหว่างขั้นตอนในโฟลว์งาน
- **การเก็บรักษาบริบท**: รักษาสถานะและความรู้สะสมตลอดขั้นตอนทั้งหมด
- **การกระจายข้อผิดพลาด**: จัดการกับความล้มเหลวอย่างมีประสิทธิภาพในสายโซ่การประมวลผลลำดับขั้น
- **การเพิ่มประสิทธิภาพประสิทธิผล**: การทำงานเรียงลำดับอย่างมีประสิทธิภาพโดยมีค่าใช้จ่ายต่ำสุด

### 🏢 **แอปพลิเคชันลำดับขั้นสำหรับองค์กร**
- **สายงานประมวลผลเอกสาร**: การวิเคราะห์ เเปลี่ยนเเปลง และตรวจสอบเอกสารหลายขั้นตอน
- **โฟลว์งานประกันคุณภาพ**: การตรวจสอบ ทบทวน และอนุมัติแบบลำดับขั้น
- **สายงานผลิตเนื้อหา**: การวิจัย → การเขียน → การแก้ไข → การทบทวน → การเผยแพร่
- **กระบวนการธุรกิจอัตโนมัติ**: โฟลว์งานธุรกิจหลายขั้นตอนที่มีความสัมพันธ์ชัดเจนแต่ละขั้น

## ⚙️ สิ่งที่ต้องมี & การตั้งค่า

### 📦 **แพ็กเกจ NuGet ที่ต้องใช้**

แพ็กเกจพื้นฐานสำหรับโฟลว์งานลำดับขั้นบน .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **การกำหนดค่า Azure OpenAI**

**การตั้งค่าสภาพแวดล้อม (.env file):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**การจัดการการกำหนดค่า:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: ตัวแทนวิเคราะห์]
    B --> C[จุดตรวจสอบ 1]

    D --> E[จุดตรวจสอบ 2]
    E --> F[Stage 3: ตัวแทนตรวจสอบ]
```

**องค์ประกอบหลัก:**
- **เอเย่นต์ลำดับขั้น**: เอเย่นต์เฉพาะทางสำหรับแต่ละขั้นตอนประมวลผล
- **บริบทสถานะ**: รักษาข้อมูลสะสมและการตัดสินใจตลอดขั้นตอนต่างๆ
- **จุดตรวจสอบ**: จุดยืนยันระหว่างขั้นตอนเพื่อรักษาคุณภาพและความสอดคล้อง
- **ไคลเอนต์ Azure OpenAI**: การเข้าถึงโมเดล AI ที่สม่ำเสมอในทุกขั้นตอนของโฟลว์งาน

## 🏢 **ประโยชน์ของโฟลว์งานลำดับขั้นสำหรับองค์กร**

### 🎯 **ความน่าเชื่อถือ & คุณภาพ**
- **การประมวลผลที่กำหนดแน่นอน**: ผลลัพธ์ที่สม่ำเสมอและทำซ้ำได้ผ่านขั้นตอนที่มีโครงสร้าง
- **ประตูคุณภาพ**: จุดตรวจสอบเพื่อให้มั่นใจในคุณภาพทุกขั้น
- **การแยกข้อผิดพลาด**: ปัญหาในขั้นตอนเดียวไม่ส่งผลกระทบต่อขั้นตอนถัดไป
- **บันทึกตรวจสอบ**: การติดตามครบถ้วนของการตัดสินใจและการเปลี่ยนแปลงในแต่ละขั้น

### 📈 **ความสามารถในการขยาย & ประสิทธิภาพ**
- **การออกแบบแบบโมดูลาร์**: แต่ละขั้นตอนสามารถเพิ่มประสิทธิภาพได้อย่างอิสระ
- **การจัดการทรัพยากร**: การจัดสรรทรัพยากรของโมเดล AI อย่างมีประสิทธิภาพตลอดขั้นตอน
- **การเพิ่มประสิทธิภาพสถานะ**: การถ่ายโอนสถานะน้อยที่สุดระหว่างขั้นตอนเพื่อประสิทธิภาพที่ดีที่สุด
- **กลุ่มขั้นตอนแบบขนาน**: โฟลว์งานลำดับขั้นหลายชุดสามารถทำงานพร้อมกันได้

### 🔒 **ความปลอดภัย & การปฏิบัติตามกฎระเบียบ**
- **ความปลอดภัยระดับขั้นตอน**: นโยบายความปลอดภัยแตกต่างกันไปตามแต่ละขั้นตอนประมวลผล
- **การตรวจสอบข้อมูล**: รักษาความถูกต้องและการปฏิบัติตามที่จุดตรวจสอบแต่ละแห่ง
- **การควบคุมการเข้าถึง**: สิทธิ์อย่างละเอียดสำหรับขั้นตอนต่างๆ ของโฟลว์งาน
- **การปฏิบัติตามกฎระเบียบ**: ปฏิบัติตามข้อกำหนดด้วยการประมวลผลที่มีโครงสร้าง

### 📊 **การตรวจสอบ & การวิเคราะห์**
- **ตัวชี้วัดระดับขั้นตอน**: การตรวจสอบประสิทธิภาพในแต่ละขั้นตอนของโฟลว์งาน
- **การหาจุดคับคั่ง**: ระบุและเพิ่มประสิทธิภาพขั้นตอนที่ช้า
- **ตัวชี้วัดคุณภาพ**: ติดตามคุณภาพและอัตราความสำเร็จในแต่ละขั้น
- **การเพิ่มประสิทธิภาพกระบวนการ**: การปรับปรุงอย่างต่อเนื่องจากการวิเคราะห์ระดับขั้นตอน

มาสร้างสายการประมวลผล AI แบบลำดับขั้นที่แข็งแกร่งกันเถอะ! 🚀

### 📝 ผลลัพธ์ที่คาดหวัง

โฟลว์งานจะ:
1. **เอเย่นต์ขาย**: ระบุรายการเฟอร์นิเจอร์จากภาพและให้คำแนะนำ
2. **เอเย่นต์ราคา**: เพิ่มการวิเคราะห์ราคาละเอียดพร้อมระดับงบประมาณและคำแนะนำการช็อปปิ้ง
3. **เอเย่นต์ใบเสนอราคา**: สร้างเอกสารใบเสนอราคาที่จัดรูปแบบพร้อมข้อมูลทั้งหมดที่สังเคราะห์มา

ผลลัพธ์สุดท้ายจะเป็นใบเสนอราคาฟอร์มครบถ้วนและเป็นมืออาชีพสำหรับเฟอร์นิเจอร์จากการวิเคราะห์ภาพ

### 🔧 ตัวเลือกการปรับแต่ง

**แก้ไขพฤติกรรมเอเย่นต์:**
```csharp
// Adjust agent instructions to change their focus
const string SalesAgentInstructions = "Your custom instructions...";
```

**เปลี่ยนโฟลว์ลำดับขั้น:**
```csharp
// Add or reorder workflow stages
var workflow = new WorkflowBuilder(salesagent)
    .AddEdge(salesagent, priceagent)
    .AddEdge(priceagent, quoteagent)
    .AddEdge(quoteagent, newAgent)  // Add another stage
    .Build();
```

```csharp
]);
```

### 🎯 การใช้งานในโลกความจริง

รูปแบบลำดับขั้นนี้เหมาะสำหรับ:
- **อีคอมเมิร์ซ**: การวิเคราะห์ผลิตภัณฑ์ → การตั้งราคา → การสร้างใบเสนอราคา
- **อสังหาริมทรัพย์**: การวิเคราะห์ทรัพย์สิน → การประเมินค่า → การสร้างรายการประกาศ
- **ประกันภัย**: การวิเคราะห์เคลม → การประเมิน → การสร้างใบเสนอราคา
- **การสร้างเนื้อหา**: การวิจัย → การเขียน → การแก้ไข → การเผยแพร่

### 🔍 การทำความเข้าใจการไหลของสถานะ

เอเย่นต์แต่ละตัวในลำดับจะได้รับ:
- **ข้อมูลนำเข้าเดิม**: ข้อความผู้ใช้ดั้งเดิม (ภาพ + ข้อความ)
- **ผลลัพธ์เอเย่นต์ก่อนหน้า**: การตอบกลับเอเย่นต์ทั้งหมดในประวัติการสนทนา
- **บริบทสะสม**: สถานะครบถ้วนที่ถูกเก็บไว้ตลอดโฟลว์งาน

สิ่งนี้ช่วยให้การประมวลผลหลายขั้นตอนที่ซับซ้อนโดยเอเย่นต์แต่ละตัวสร้างขึ้นบนบริบทครบถ้วนจากทุกขั้นตอนก่อนหน้า
