# 🔄 โฟลว์งานพื้นฐานของเอเจนต์ด้วย Azure OpenAI (Responses API) (.NET)

## 📋 บทแนะนำการประสานงานโฟลว์งาน

โน้ตบุ๊กนี้สาธิตวิธีสร้าง **โฟลว์งานเอเจนต์** ที่ซับซ้อนโดยใช้ Microsoft Agent Framework สำหรับ .NET และ Azure OpenAI (Responses API) คุณจะได้เรียนรู้การสร้างกระบวนการทางธุรกิจหลายขั้นตอนที่เอเจนต์ AI ร่วมมือกันเพื่อทำงานที่ซับซ้อนให้สำเร็จผ่านรูปแบบการประสานงานที่เป็นระบบ

## 🎯 จุดมุ่งหมายการเรียนรู้

### 🏗️ **พื้นฐานสถาปัตยกรรมโฟลว์งาน**
- **Workflow Builder**: ออกแบบและประสานงานกระบวนการ AI หลายขั้นตอนที่ซับซ้อน
- **Agent Coordination**: ประสานงานเอเจนต์หลายตัวที่มีความเชี่ยวชาญเฉพาะภายในโฟลว์งาน
- **Azure OpenAI (Responses API)**: ใช้ประโยชน์จาก Azure OpenAI Responses API ในโฟลว์งาน
- **Visual Workflow Design**: สร้างและแสดงภาพโครงสร้างโฟลว์งานเพื่อความเข้าใจที่ดีขึ้น

### 🔄 **รูปแบบการประสานงานกระบวนการ**
- **Sequential Processing**: ลำดับงานเอเจนต์หลายงานตามลำดับตรรกะ
- **State Management**: รักษาบริบทและการไหลของข้อมูลตลอดช่วงโฟลว์งาน
- **Error Handling**: นำเสนอการกู้คืนข้อผิดพลาดที่แข็งแกร่งและความทนทานของโฟลว์งาน
- **Performance Optimization**: ออกแบบโฟลว์งานที่มีประสิทธิภาพสำหรับการดำเนินงานระดับองค์กร

### 🏢 **แอปพลิเคชันโฟลว์งานระดับองค์กร**
- **Business Process Automation**: อัตโนมัติโฟลว์งานองค์กรที่ซับซ้อน
- **Content Production Pipeline**: โฟลว์งานด้านบรรณาธิการพร้อมขั้นตอนการตรวจสอบและอนุมัติ
- **Customer Service Automation**: การแก้ไขคำถามลูกค้าหลายขั้นตอน
- **Data Processing Workflows**: โฟลว์งาน ETL พร้อมการแปลงโดย AI

## ⚙️ ข้อกำหนดเบื้องต้นและการตั้งค่า

### 📦 **แพ็กเกจ NuGet ที่จำเป็น**

การสาธิตโฟลว์งานนี้ใช้แพ็กเกจ .NET สำคัญหลายตัว:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **การตั้งค่า Azure OpenAI**

**การตั้งค่าสภาพแวดล้อม (.env file):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**การเข้าถึง Azure OpenAI:**
1. สร้างทรัพยากร Azure OpenAI ในพอร์ทัล Azure
2. ปรับใช้โมเดล (ตัวอย่างเช่น `gpt-5-mini`) และจดชื่อการปรับใช้ไว้
3. ลงชื่อเข้าใช้ด้วย `az login` และตั้งค่าตัวแปรสภาพแวดล้อมตามที่แสดงข้างต้น

### 🏗️ **ภาพรวมสถาปัตยกรรมโฟลว์งาน**

```mermaid
graph TD
    A[ตัวสร้างโฟลว์งาน] --> B[บัญชีตัวแทน]
    B --> C[เครื่องยนต์รันโฟลว์งาน]
    C --> D[Agent 1: เครื่องสร้างเนื้อหา]
    C --> E[Agent 2: ผู้ตรวจสอบเนื้อหา] 
    D --> F[ผลลัพธ์โฟลว์งาน]
    E --> F
    G[Azure OpenAI (API การตอบกลับ)] --> D
    G --> E
```

**ส่วนประกอบหลัก:**
- **WorkflowBuilder**: เครื่องยนต์ประสานงานหลักสำหรับออกแบบโฟลว์งาน
- **AIAgent**: เอเจนต์เฉพาะที่มีความสามารถเฉพาะตัว
- **Azure OpenAI Client**: การผสาน API Responses ของ Azure OpenAI
- **Execution Context**: จัดการสถานะและการไหลของข้อมูลระหว่างช่วงโฟลว์งาน

## 🎨 **รูปแบบการออกแบบโฟลว์งานระดับองค์กร**

### 📝 **โฟลว์งานผลิตเนื้อหา**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **สายการประมวลผลเอกสาร**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **โฟลว์งานธุรกิจอัจฉริยะ**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **ระบบอัตโนมัติฝ่ายบริการลูกค้า**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **ประโยชน์สำหรับองค์กร**

### 🎯 **ความน่าเชื่อถือและการปรับขนาด**
- **Deterministic Execution**: ผลลัพธ์โฟลว์งานที่สม่ำเสมอและทำซ้ำได้
- **Error Recovery**: การจัดการข้อผิดพลาดอย่างมีเกียรติในทุกช่วงโฟลว์งาน
- **Performance Monitoring**: ติดตามเมตริกการดำเนินการและโอกาสในการปรับแต่ง
- **Resource Management**: การจัดสรรและใช้ทรัพยากรโมเดล AI อย่างมีประสิทธิภาพ

### 🔒 **ความปลอดภัยและการปฏิบัติตามข้อกำหนด**
- **Secure Authentication**: การตรวจสอบสิทธิ์ Microsoft Entra ID ผ่าน `az login` (AzureCliCredential)
- **Audit Trails**: การบันทึกล็อกอย่างครบถ้วนของการดำเนินโฟลว์งานและจุดตัดสินใจ
- **Access Control**: สิทธิ์การเข้าถึงอย่างละเอียดสำหรับการดำเนินโฟลว์งานและการตรวจสอบ
- **Data Privacy**: การจัดการข้อมูลที่ละเอียดอ่อนอย่างปลอดภัยตลอดโฟลว์งาน

### 📊 **การสังเกตการณ์และการจัดการ**
- **Visual Workflow Design**: การแสดงภาพการไหลของกระบวนการและความสัมพันธ์อย่างชัดเจน
- **Execution Monitoring**: การติดตามความคืบหน้าและประสิทธิภาพของโฟลว์งานแบบเรียลไทม์
- **Error Reporting**: การวิเคราะห์ข้อผิดพลาดและการดีบักอย่างละเอียด
- **Performance Analytics**: เมตริกสำหรับการปรับแต่งและการวางแผนกำลังการผลิต

มาสร้างโฟลว์งาน AI พร้อมสำหรับองค์กรครั้งแรกของคุณกันเถอะ! 🚀

## 💻 การรันโค้ด

โค้ดตัวเต็มมีในไฟล์ `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` ไฟล์นี้สาธิต:

1. **การตั้งค่าสภาพแวดล้อม** - โหลดการตั้งค่า Azure OpenAI จากไฟล์ `.env`
2. **การตั้งค่า Azure OpenAI Client** - กำหนดค่าไคลเอนต์ให้ใช้ Azure OpenAI Responses API
3. **การสร้างเอเจนต์** - กำหนดเอเจนต์เฉพาะ (Front Desk และ Concierge)
4. **Workflow Builder** - สร้างโฟลว์งานหลายเอเจนต์โดยใช้การประมวลผลแบบลำดับ
5. **การรันโฟลว์งาน** - รันโฟลว์งานพร้อมผลลัพธ์แบบสตรีมมิ่ง

### 🚀 การรันตัวอย่าง

```bash
# ทำให้สคริปต์สามารถรันได้ (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# รันเวิร์กโฟลว์
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

หรือใน Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 ผลลัพธ์ที่คาดหวัง

โฟลว์งานจะ:
1. รับคำขอจุดหมายปลายทางการเดินทางของคุณ ("ฉันอยากไปปารีส")
2. เอเจนต์ Front Desk ให้คำแนะนำเบื้องต้น
3. เอเจนต์ Concierge ตรวจสอบและปรับปรุงคำแนะนำ
4. แสดงผลลัพธ์สุดท้ายเป็นสตรีมเต็มของการสนทนา

### 🔧 การปรับแต่ง

คุณสามารถปรับแต่งโฟลว์งานโดย:
- แก้ไขคำสั่งเอเจนต์เพื่อเปลี่ยนพฤติกรรม
- เพิ่มเอเจนต์มากขึ้นเพื่อสร้างโฟลว์งานหลายขั้นตอนที่ซับซ้อน
- เปลี่ยนข้อความผู้ใช้เพื่อทดสอบสถานการณ์ที่แตกต่างกัน
- ปรับขอบโฟลว์งานเพื่อสร้างรูปแบบการทำงานที่แตกต่างกัน
