# 🔄 โฟลว์งานตัวแทนพื้นฐานกับ Azure OpenAI (Responses API) (.NET)

## 📋 แบบฝึกหัดการจัดการโฟลว์งาน

โน้ตบุ๊กนี้สาธิตวิธีสร้าง **โฟลว์งานตัวแทน** ที่ซับซ้อนโดยใช้ Microsoft Agent Framework สำหรับ .NET และ Azure OpenAI (Responses API) คุณจะได้เรียนรู้วิธีสร้างกระบวนการธุรกิจหลายขั้นตอนที่ตัวแทน AI ร่วมมือกันเพื่อทำงานที่ซับซ้อนให้สำเร็จผ่านรูปแบบการจัดการแบบมีโครงสร้าง

## 🎯 วัตถุประสงค์การเรียนรู้

### 🏗️ **พื้นฐานสถาปัตยกรรมโฟลว์งาน**
- **ตัวสร้างโฟลว์งาน**: ออกแบบและจัดการกระบวนการ AI หลายขั้นตอนที่ซับซ้อน
- **การประสานงานตัวแทน**: ประสานงานตัวแทนเฉพาะทางหลายตัวในโฟลว์งาน
- **Azure OpenAI (Responses API)**: ใช้ประโยชน์จาก Azure OpenAI Responses API ในโฟลว์งาน
- **การออกแบบโฟลว์งานเชิงภาพ**: สร้างและแสดงโครงสร้างโฟลว์งานเพื่อความเข้าใจที่ดีขึ้น

### 🔄 **รูปแบบการจัดการกระบวนการ**
- **กระบวนการแบบลำดับขั้น**: เชื่อมโยงงานตัวแทนหลายงานตามลำดับตรรกะ
- **การจัดการสถานะ**: รักษาบริบทและการไหลของข้อมูลในทุกขั้นตอนของโฟลว์งาน
- **การจัดการข้อผิดพลาด**: ดำเนินการฟื้นฟูข้อผิดพลาดอย่างมั่นคงและความทนทานของโฟลว์งาน
- **การเพิ่มประสิทธิภาพการทำงาน**: ออกแบบโฟลว์งานที่มีประสิทธิภาพสำหรับการดำเนินงานในระดับองค์กร

### 🏢 **การประยุกต์ใช้โฟลว์งานในองค์กร**
- **ระบบอัตโนมัติกระบวนการทางธุรกิจ**: อัตโนมัติโฟลว์งานองค์กรที่ซับซ้อน
- **สายการผลิตเนื้อหา**: โฟลว์งานบรรณาธิการพร้อมขั้นตอนการตรวจสอบและอนุมัติ
- **ระบบอัตโนมัติฝ่ายบริการลูกค้า**: การแก้ไขข้อสงสัยลูกค้าแบบหลายขั้นตอน
- **โฟลว์งานการประมวลผลข้อมูล**: โฟลว์ ETL พร้อมการเปลี่ยนแปลงด้วย AI

## ⚙️ ข้อกำหนดเบื้องต้น & การตั้งค่า

### 📦 **แพ็กเกจ NuGet ที่จำเป็น**

การสาธิตโฟลว์งานนี้ใช้งานแพ็กเกจ .NET หลักหลายตัว:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **การตั้งค่า Azure OpenAI**

**การตั้งค่าสภาพแวดล้อม (.env file):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**การเข้าถึง Azure OpenAI:**
1. สร้างทรัพยากร Azure OpenAI ใน Azure portal
2. ปรับใช้โมเดล (เช่น `gpt-4.1-mini`) และจดชื่อการปรับใช้
3. ลงชื่อเข้าใช้ด้วย `az login` และตั้งค่าตัวแปรสภาพแวดล้อมตามตัวอย่างข้างต้น

### 🏗️ **ภาพรวมสถาปัตยกรรมโฟลว์งาน**

```mermaid
graph TD
    A[ตัวสร้างเวิร์กโฟลว์] --> B[สมุดทะเบียนเอเย่นต์]
    B --> C[เอนจินการดำเนินการเวิร์กโฟลว์]
    C --> D[Agent 1: ตัวสร้างเนื้อหา]
    C --> E[Agent 2: ผู้ตรวจสอบเนื้อหา] 
    D --> F[ผลลัพธ์เวิร์กโฟลว์]
    E --> F
    G[Azure OpenAI (Responses API)] --> D
    G --> E
```

**ส่วนประกอบหลัก:**
- **WorkflowBuilder**: เครื่องยนต์จัดการหลักสำหรับการออกแบบโฟลว์งาน
- **AIAgent**: ตัวแทนเฉพาะทางแต่ละตัวที่มีความสามารถเฉพาะ
- **Azure OpenAI Client**: การรวม Responses API ของ Azure OpenAI
- **Execution Context**: จัดการสถานะและการไหลของข้อมูลระหว่างขั้นตอนโฟลว์งาน

## 🎨 **รูปแบบการออกแบบโฟลว์งานองค์กร**

### 📝 **โฟลว์งานการผลิตเนื้อหา**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **สายการประมวลผลเอกสาร**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **โฟลว์งานธุรกิจอัจฉริยะ**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **ระบบอัตโนมัติฝ่ายบริการลูกค้า**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **ประโยชน์สำหรับองค์กร**

### 🎯 **ความน่าเชื่อถือ & ความสามารถในการปรับขนาด**
- **การดำเนินงานแบบกำหนดผลลัพธ์ได้**: ผลลัพธ์โฟลว์งานที่สม่ำเสมอและทำซ้ำได้
- **การฟื้นฟูข้อผิดพลาด**: การจัดการความล้มเหลวในทุกขั้นตอนโฟลว์งานอย่างราบรื่น
- **การติดตามประสิทธิภาพ**: ติดตามตัวชี้วัดการดำเนินงานและโอกาสในการปรับปรุง
- **การจัดการทรัพยากร**: การจัดสรรและใช้งานทรัพยากรโมเดล AI อย่างมีประสิทธิภาพ

### 🔒 **ความปลอดภัย & การปฏิบัติตามข้อกำหนด**
- **การยืนยันตัวตนอย่างปลอดภัย**: การยืนยันตัวตน Microsoft Entra ID ผ่าน `az login` (AzureCliCredential)
- **บันทึกการตรวจสอบ**: บันทึกอย่างครบถ้วนของการดำเนินงานโฟลว์งานและจุดตัดสินใจ
- **การควบคุมการเข้าถึง**: สิทธิ์ในระดับละเอียดสำหรับการดำเนินการและการติดตามโฟลว์งาน
- **ความเป็นส่วนตัวของข้อมูล**: การจัดการข้อมูลที่ละเอียดอ่อนอย่างปลอดภัยตลอดโฟลว์งาน

### 📊 **การสังเกตการณ์ & การจัดการ**
- **การออกแบบโฟลว์งานเชิงภาพ**: การแสดงภาพกระบวนการและความสัมพันธ์อย่างชัดเจน
- **การติดตามการดำเนินงาน**: การติดตามความคืบหน้าและประสิทธิภาพแบบเรียลไทม์
- **รายงานข้อผิดพลาด**: การวิเคราะห์ข้อผิดพลาดอย่างละเอียดและความสามารถในการดีบัก
- **การวิเคราะห์ประสิทธิภาพ**: ตัวชี้วัดสำหรับการปรับปรุงและการวางแผนความจุ

มาเริ่มสร้างโฟลว์งาน AI สำหรับองค์กรของคุณกัน! 🚀

## 💻 การรันโค้ด

การใช้งานครบถ้วนมีอยู่ในไฟล์ `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` ไฟล์นี้สาธิต:

1. **การตั้งค่าสภาพแวดล้อม** - โหลดการตั้งค่า Azure OpenAI จากไฟล์ `.env`
2. **การตั้งค่าลูกค้า Azure OpenAI** - กำหนดค่าลูกค้าเพื่อใช้ Azure OpenAI Responses API
3. **การสร้างตัวแทน** - กำหนดตัวแทนเฉพาะทาง (Front Desk และ Concierge)
4. **ตัวสร้างโฟลว์งาน** - สร้างโฟลว์งานหลายตัวแทนพร้อมการประมวลผลแบบลำดับ
5. **การดำเนินโฟลว์งาน** - รันโฟลว์งานพร้อมผลลัพธ์แบบสตรีม

### 🚀 การรันตัวอย่าง

```bash
# ทำให้สคริปต์สามารถรันได้ (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# รันเวิร์กโฟลว์
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

หรือใน Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 ผลลัพธ์ที่คาดหวัง

โฟลว์งานจะ:
1. รับคำขอปลายทางการเดินทางของคุณ ("ฉันต้องการไปปารีส")
2. ตัวแทน Front Desk ให้คำแนะนำเบื้องต้น
3. ตัวแทน Concierge ตรวจสอบและปรับปรุงคำแนะนำ
4. แสดงผลสุดท้ายของการสนทนาแบบสตรีมเต็มรูปแบบ

### 🔧 การปรับแต่ง

คุณสามารถปรับแต่งโฟลว์งานโดย:
- แก้ไขคำสั่งของตัวแทนเพื่อเปลี่ยนพฤติกรรม
- เพิ่มตัวแทนมากขึ้นเพื่อสร้างโฟลว์งานหลายขั้นตอนที่ซับซ้อน
- เปลี่ยนข้อความของผู้ใช้เพื่อทดสอบสถานการณ์ต่างๆ
- ปรับแต่งขอบของโฟลว์งานเพื่อสร้างรูปแบบการดำเนินงานที่แตกต่างกัน
