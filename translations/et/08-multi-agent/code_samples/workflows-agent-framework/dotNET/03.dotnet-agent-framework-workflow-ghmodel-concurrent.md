# ⚡ Koosolevad agendi töövood Azure OpenAI-ga (Responses API) (.NET)

## 📋 Kõrge jõudlusega paralleeltöötluse õpituba

See märkmik demonstreerib **koosolevaid töövoomustreid**, kasutades Microsoft Agent Frameworki .NET jaoks ja Azure OpenAI-d (Responses API). Õpid, kuidas ehitada kõrge jõudlusega, paralleeltöötlusega töövooge, mis maksimeerivad läbilaset täites mitut tehisintellekti agenti samaaegselt, hoides samal ajal koordineeritust ja andmete järjepidevust.

## 🎯 Õpieesmärgid

### 🚀 **Koosoleva töötluse alused**
- **Paralleelne agendi täitmine**: Käivita mitu tehisintellekti agenti samaaegselt maksimaalse jõudluse jaoks
- **Async/Await mustrid**: Kasuta .NET-i asünkroonset programmeerimismudelit tõhusa samaaegse töö jaoks
- **Azure OpenAI (Responses API)**: Koordineeri mitut samaaegset kõnet Azure OpenAI Responses API-le
- **Ressursside haldus**: Halda tõhusalt tehisintellekti mudelite ressursse samaaegsete toimingute vahel

### 🏗️ **Täpsem samaaegsusarhitektuur**
- **Ülesandepõhine paralleelsus**: Kasuta .NET Task Parallel Library-t optimaalseks koosolevaks täitmiseks
- **Sünkroniseerimismustrid**: Koordineeri samal ajal töötavaid agente, vältides võidusõidutingimusi
- **Koormuse tasakaalustamine**: Töö efektiivne jagamine saadaoleva samaaegse töötlusvõimsuse vahel
- **Tõrketaluvus**: Töötle individuaalseid agendi rikkeid ilma kogu töövoogu peatamata

### 🏢 **Ettevõtte samaaegsed rakendused**
- **Suurte dokumentide töötlemine**: Töötle samaaegselt mitut dokumenti
- **Reaalajas sisuanalüüs**: Sissetulevate andmevoogude samaaegne analüüs
- **Pakkide töötlemise optimeerimine**: Maksimeeri läbilase suurtes mahus andmetöötluse operatsioonides
- **Mitme modaliteediga analüüs**: Paralleeltöötlus erinevat tüüpi ja formaadiga sisule

## 💻 Koodi käivitamine

Täielik teostus on saadaval failis `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. See fail demonstreerib **Fan-Out/Fan-In samaaegset töövoogu** reisiplaani koostamiseks:

### 🏗️ **Töövoo arhitektuur**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Põhikomponendid:**

1. **ConcurrentStartExecutor**: Edastab kasutajapäringu kõigile agentidele samaaegselt
2. **Researcher Agent**: Analüüsib sihtkohti ja atraktsioone samaaegselt
3. **Planner Agent**: Koostab üksikasjalikke reisiplaane samaaegselt
4. **ConcurrentAggregationExecutor**: Kogub ja ühendab mõlema agendi tulemused

### 🎯 **Fan-Out/Fan-In muster**

See töövoog demonstreerib klassikalist **Fan-Out/Fan-In** mustrit:
- **Fan-Out**: Üks sisendsõnum levitatakse mitmele agentidele samaaegselt
- **Samaaegne töötlemine**: Mitmed agendid töötavad paralleelselt sama ülesande kallal
- **Fan-In**: Kõigi agentide tulemused kogutakse ja ühendatakse üheks väljundiks

### 🔍 Kohandatud täitjate mõistmine

**ConcurrentStartExecutor:**
- Rakendab `IMessageHandler<string>` stringi sisendi vastuvõtmiseks
- Edastab sõnumeid kõigile ühendatud agentidele
- Saadab `TurnToken` samaaegse töötluse käivitamiseks

**ConcurrentAggregationExecutor:**
- Rakendab `IMessageHandler<ChatMessage>` agendi vastuste vastuvõtmiseks
- Kogub sõnumeid lõimohutult
- Kogub vastused kui kõik oodatud vastused saabuvad
- Tagastab lõpliku väljundi kasutades `context.YieldOutputAsync()`

### ⚡ Jõudluse eelised

**Samaaegne vs järjestikune:**
- Järjestikune: Agent1 (30s) → Agent2 (30s) = **kokku 60 sekundit**
- Samaegne: Agent1 (30s) || Agent2 (30s) = **kokku 30 sekundit**

**Läbilaske paranemine**: Kuni N korda kiirem N samaaegse agendi jaoks (sõltuvalt töökoormusest ja ressurssidest)
