# ⚡ Mitmeagentuuriline töövoog Azure OpenAI (Responses API) abil (.NET)

## 📋 Kõrge jõudlusega paralleeltöötluse juhend

See märkmik demonstreerib **mitmerealisi töövoogude mustreid** Microsoft Agent raamistiku abil .NET-ile ja Azure OpenAI (Responses API). Õpid, kuidas luua kõrge jõudlusega paralleeltöötluse töövooge, mis maksimeerivad läbilaskevõimet, käivitades samaaegselt mitut tehisintellekti agenti, säilitades samal ajal koordineerimise ja andmete järjepidevuse.

## 🎯 Õpieesmärgid

### 🚀 **Mitmerealise töötlemise alused**
- **Mitme agendi paralleelkäitamine**: Käivita samaaegselt mitu tehisintellekti agenti maksimaalse jõudluse saavutamiseks
- **Async/Await mustrid**: Kasuta .NET-i asünkroonse programmeerimise mudelit tõhusa samaaegsuse jaoks
- **Azure OpenAI (Responses API)**: Koordineeri mitut samaaegset kõnet Azure OpenAI Responses API-le
- **Ressursside haldus**: Jõuline tehisintellekti mudelite ressursside haldamine samaaegsete operatsioonide korral

### 🏗️ **Täiustatud samaaegsuse arhitektuur**
- **Ülesandepõhine parallelism**: Kasuta .NET Task Parallel Library optimaalsete samaaegsete käivituste jaoks
- **Sünkroonimismustrid**: Koordineeri samaaegseid agente, vältides võistlusolukordi
- **Koormuse tasakaalustamine**: Töö efektiivne jaotamine olemasoleva samaaegse töötluse mahtu arvestades
- **Vea taluvus**: Üksikute agentide rikete käitlemine ilma kogu töövoogu peatamata

### 🏢 **Ettevõtete samaaegsed rakendused**
- **Suurte dokumentide töötlemine**: Töötle mitut dokumenti samaaegselt
- **Reaalajas sisuanalüüs**: Sissevoolavate andmevoogude samaaegne analüüs
- **Pakkide töötlemise optimeerimine**: Läbilaskevõime maksimeerimine suurte andmetöötlusoperatsioonide jaoks
- **Mitmemodaalne analüüs**: Paralleeltöötlus erinevate sisutüüpide ja vormingute puhul

Täielik rakendus on saadaval failis `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. See fail demonstreerib **Fan-Out/Fan-In samaaegset töövoogu** reisiplaani koostamiseks:

### 🏗️ **Töövoo arhitektuur**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Põhikomponendid:**

1. **ConcurrentStartExecutor**: Saadab kasutaja päringu kõigile agentidele samaaegselt
2. **Researcher Agent**: Analüüsib sihtkohti ja vaatamisväärsusi samaaegselt
3. **Planner Agent**: Koostab üksikasjalikud reisiplaanid samaaegselt
4. **ConcurrentAggregationExecutor**: Kogub ja ühendab mõlema agendi tulemused

### 🎯 **Fan-Out/Fan-In muster**

See töövoog demonstreerib klassikalist **Fan-Out/Fan-In** mustrit:
- **Fan-Out**: Üks sisend-sõnum saadetakse samaaegselt mitmele agendile
- **Samaaegne töötlemine**: Mitmed agentid töötavad paralleelselt sama ülesande kallal
- **Fan-In**: Kõigi agentide tulemused kogutakse ja koondatakse üheks väljundiks

### 🔍 Kohandatud täitjate mõistmine

**ConcurrentStartExecutor:**
- Rakendab `IMessageHandler<string>` string-sisendi vastuvõtmiseks
- Levita sõnumeid kõikidele ühendatud agentidele
- Saada `TurnToken` samaaegse töötlemise käivitamiseks

**ConcurrentAggregationExecutor:**
- Rakendab `IMessageHandler<ChatMessage>` agendi vastuste vastuvõtmiseks
- Kogub sõnumeid niiditurvaliselt
- Kogub kokku siis, kui kõik oodatud vastused saabuvad
- Andmete väljastamine `context.YieldOutputAsync()` abil

### ⚡ Jõudluse eelised

**Samaaegne vs järjepidev:**
- Järjepidev: Agent1 (30s) → Agent2 (30s) = **kokku 60 sekundit**
- Samaaegne: Agent1 (30s) || Agent2 (30s) = **kokku 30 sekundit**

**Läbilaskevõime parandus**: Kuni N× kiirem N samaaegse agendi puhul (sõltuvalt töökoormusest ja ressurssidest)

- Vajadusel saab tagastada osalised tulemused
