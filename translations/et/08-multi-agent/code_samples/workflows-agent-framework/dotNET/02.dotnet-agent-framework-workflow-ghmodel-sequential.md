# ⏩ Järjestikused agendi töövood Azure OpenAI-ga (Responses API) (.NET)

## 📋 Täiustatud järjestikuse töötlemise juhend

See märkmik demonstreerib **järjestikuseid töövoomustreid** Microsoft Agent Framework for .NET ja Azure OpenAI (Responses API) abil. Õpid, kuidas luua keerukaid samm-sammult töötlemise torujuhtmeid, kus agendid täidavad ülesandeid kindlas järjekorras, kus iga etapp tugineb eelmise etapi tulemustele.

- **Vigade leviku haldamine**: Käitu vigadega järjestikuste töötlemisahelate korral graatsiliselt

## ⚙️ Eeltingimused ja seadistamine

### 📦 **Nõutavad NuGet paketid**

Olulised paketid .NET järjestikuste töövoogude jaoks:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Azure OpenAI seadistus**

**Keskkonna seadistamine (.env fail):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Konfiguratsiooni haldus:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **Järjestikuse töövoo arhitektuur**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Analüüsiagent]
    B --> C[Kontrollpunkt 1]
    C --> D[Stage 2: Töötlejaagent]
    D --> E[Kontrollpunkt 2]
    E --> F[Stage 3: Kinnituseagent]
    F --> G[Lõplik väljund]
    
    H[Oleku kontekst] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Responses API)] --> B
    I --> D
    I --> F
```

- **Azure OpenAI klient**: Järjepidev AI mudelite kasutus kõigis töövoo etappides

## 💻 Koodi käivitamine

Täielik teostus on saadaval failis `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. See fail demonstreerib **kolmeetapilist mööblianalüüsi töövoogu**:

1. **1. etapp - müügiaagent**: Analüüsib mööbli pilte ja pakub ostusoovitusi
2. **2. etapp - hinnakujundusagent**: Andmetab üksikasjalikud hinnapakkumised ja eelarvevalikud
3. **3. etapp - pakkumise agent**: Genereerib professionaalse pakkumisdokumendi Markdown-formaadis

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Iga agent:
- Saab kontekstiks eelmise etapi väljundi
- Arendab eelnevat analüüsi edasi spetsialiseeritud teadmistega
- Säilitab töövoo järjepidevuse oleku haldamise kaudu
