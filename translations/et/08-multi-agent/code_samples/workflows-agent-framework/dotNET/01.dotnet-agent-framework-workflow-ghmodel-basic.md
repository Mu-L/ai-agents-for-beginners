# 🔄 Põhilised volituste töövood Azure OpenAI-ga (Responses API) (.NET)

## 📋 Töövoo orkestreerimise juhend

See märkmik demonstreerib, kuidas luua keerukaid **volituste töövooge** Microsoft Agent Frameworki abil .NET-i ja Azure OpenAI (Responses API) jaoks. Õpid looma mitmeastmelisi äriprotsesse, kus tehisintellekti agendid töötavad koos keerukate ülesannete lahendamiseks struktureeritud orkestreerimise mustrite kaudu.

## 🎯 Õpieesmärgid

### 🏗️ **Töövoo arhitektuuri alused**
- **Workflow Builder**: Kujunda ja orkestreeri keerukaid mitmeastmelisi tehisintellekti protsesse
- **Agentide koordineerimine**: Koordineeri mitut spetsialiseerunud agenti töövoogudes
- **Azure OpenAI (Responses API)**: Kasuta Azure OpenAI Responses API-d töövoogudes
- **Visuaalne töövoo kujundus**: Loo ja visualiseeri töövoo struktuurid paremaks mõistmiseks

### 🔄 **Protsesside orkestreerimise mustrid**
- **Järjestikune töötlemine**: Seosta mitmeid agentide ülesandeid loogilises järjekorras
- **Oleku haldamine**: Säilita kontekst ja andmevoog töövoo etappide vahel
- **Vigade käsitlemine**: Rakenda paindlikku vea taastumist ja töövoo vastupidavust
- **Tulemuslikkuse optimeerimine**: Kujunda tõhusaid töövooge ettevõttekulu mastaapimiseks

### 🏢 **Ettevõtte töövoo rakendused**
- **Äriprotsesside automatiseerimine**: Automaatne keerukate organisatsiooni töövoogude juhtimine
- **Sisuloome torujuhe**: Toimetuslikud töövood koos ülevaatamise ja kinnituse etappidega
- **Klienditeeninduse automatiseerimine**: Mitmeastmeline kliendipäringute lahendamine
- **Andmetöötluse töövood**: ETL-protsessid koos AI-põhise teisendusega

## ⚙️ Eeltingimused ja seadistamine

### 📦 **Nõutavad NuGet paketid**

Selle töövoo demonstreerimiseks kasutatakse mitut olulist .NET paketti:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI seadistus**

**Keskkonna seadistamine (.env fail):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Azure OpenAI ligipääs:**
1. Loo Azure portaalis Azure OpenAI ressurss
2. Käivita mudel (näiteks `gpt-5-mini`) ja märgi käivituse nimi
3. Logi sisse käsuga `az login` ja seadista keskkonnamuutujad nagu eespool näidatud

### 🏗️ **Töövoo arhitektuuri ülevaade**

```mermaid
graph TD
    A[Töölaualoome tööriist] --> B[Agendi registri]
    B --> C[Töövoo täitmise mootor]
    C --> D[Agent 1: Sisu generaator
    C --> E[Agent 2: Sisu ülevaataja 
    D --> F[Töövoo tulemused]
    E --> F
    G[Azure OpenAI (Responses API)] --> D
    G --> E
```

**Peamised komponendid:**
- **WorkflowBuilder**: Peamine orkestreerimise mootor töövoogude disainimiseks
- **AIAgent**: Individuaalsed spetsialiseerunud agendid kindlate võimetega
- **Azure OpenAI klient**: Azure OpenAI Responses API integratsioon
- **Täideviimise kontekst**: Halda olekut ja andmevoogu töövoo etappide vahel

## 🎨 **Ettevõtte töövoo kujunduse mustrid**

### 📝 **Sisuloome töövoog**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **Dokumendi töötlemise torujuhe**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **Ärianalüüsi töövoog**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **Klienditeeninduse automatiseerimine**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **Ettevõtte eelised**

### 🎯 **Usaldusväärsus ja skaleeritavus**
- **Determineeritav täideviimine**: Järjepidevad ja korduvad töövoo tulemused
- **Vigade taastumine**: Etappidest sõltumatu vigade sujuv käsitlemine
- **Tulemuslikkuse monitooring**: Jälgi täideviimise mõõdikuid ja optimeerimisvõimalusi

- **Juurdepääsu kontroll**: Üksikasjalikud õigused töövoo täideviimiseks ja jälgimiseks
- **Andmekaitse**: Tundlike andmete turvaline käsitlemine kogu töövoo vältel

### 📊 **Jälgitavus ja haldus**
- **Visuaalne töövoo kujundus**: Selge protsessivoogude ja sõltuvuste esitlus
- **Täideviimise jälgimine**: Töövoo edenemise ja tulemuslikkuse reaalajas jälgimine
- **Veateadetus**: Üksikasjalik veaanalüüs ja silumisvõimalused
- **Tulemuslikkuse analüüs**: Mõõdikud optimeerimiseks ja mahtude planeerimiseks

## 💻 Koodi käivitamine

Täielik rakendus on saadaval failis `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. See fail demonstreerib:

1. **Keskkonna seadistamine** - Azure OpenAI seadistuse laadimine `.env` failist
2. **Azure OpenAI kliendi seadistamine** - kliendi haldamine Azure OpenAI Responses API kasutamiseks
3. **Agentide loomine** - spetsialiseerunud agentide määratlemine (vastuvõtt ja concierge)
4. **Töövoo ehitaja** - mitme-agendi töövoo loomine järjestikuse töötlemisega
5. **Töövoo täideviimine** - töövoo käivitamine voogesitatavate tulemuste kuvamisega

### 🚀 Näite käivitamine

```bash
# Tee skript käivitatavaks (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Käivita töövoog
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Või Windowsil:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Eeldatav väljund

Töövoog:
1. Võtab vastu sinu reisisihtkoha taotluse ("Soovin minna Pariisi")

3. Concierge agent vaatab soovituse üle ja täiustab seda
4. Lõplik väljund kuvab kogu vestluse voo
