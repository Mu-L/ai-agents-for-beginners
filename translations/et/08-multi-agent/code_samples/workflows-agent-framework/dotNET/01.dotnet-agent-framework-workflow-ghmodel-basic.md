# 🔄 Põhilised agendi töövood Azure OpenAI-ga (Responses API) (.NET)

## 📋 Töövoogude orkeestreerimise juhend

See märkmik demonstreerib, kuidas luua keerukaid **agendi töövooge** Microsoft Agent Frameworki .NET jaoks ja Azure OpenAI (Responses API) abil. Õpid looma mitmeastmelisi äriprotsesse, kus tehisintellekti agendid teevad koostööd keerukate ülesannete täitmiseks struktureeritud orkestreerimismustrite kaudu.

## 🎯 Õpieesmärgid

### 🏗️ **Töövoo arhitektuuri alused**
- **Töövoo koostaja**: Disaini ja orkestreeri keerukaid mitmeastmelisi tehisintellekti protsesse
- **Agendi koordineerimine**: Koordineeri mitut spetsialiseerunud agenti töövoogudes
- **Azure OpenAI (Responses API)**: Kasuta Azure OpenAI Responses API-t töövoogudes
- **Visuaalne töövoo disain**: Loo ja visualiseeri töövoo struktuure paremaks arusaamiseks

### 🔄 **Protsessi orkestreerimismustrid**
- **Järjestikune töötlus**: Järjestage mitut agendi ülesannet loogilises järjekorras
- **Oleku haldamine**: Säilita kontekst ja andmevoog töövoo etappide vahel
- **Veakäsitlus**: Rakenda tugev viga taastumine ja töövoo vastupidavus
- **Tõhususe optimeerimine**: Disaini tõhusaid töövooge ettevõtte tasandi toiminguteks

### 🏢 **Ettevõtte töövoo rakendused**
- **Äriprotsesside automatiseerimine**: Automatiseeri keerukaid organisatsiooni töövooge
- **Sisu tootmise torujuhe**: Toimetuslikud töövood ülevaatuse ja kinnitusetappidega
- **Klienditeeninduse automatiseerimine**: Mitmeastmeline kliendipäringute lahendus
- **Andmetöötluse töövood**: ETL töövood tehisintellektiga juhtitud andmetransformatsiooniga

## ⚙️ Eeltingimused & seadistus

### 📦 **Nõutavad NuGet paketid**

See töövoo demonstratsioon kasutab mitmeid peamisi .NET pakette:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI konfiguratsioon**

**Keskkonna seadistamine (.env fail):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI ligipääs:**
1. Loo Azure portaalis Azure OpenAI ressurss
2. Käivita mudel (näiteks `gpt-4.1-mini`) ja märgi üles juurutuse nimi
3. Logi sisse `az login` abil ja seadista keskkonnamuutujad vastavalt ülaltoodule

### 🏗️ **Töövoo arhitektuuri ülevaade**

```mermaid
graph TD
    A[Töövoo looja] --> B[Agentide register]
    B --> C[Töövoo täitmise mootor]
    C --> D[Agent 1: Sisu generaator
    C --> E[Agent 2: Sisu ülevaataja 
    D --> F[Töövoo tulemused]
    E --> F
    G[Azure OpenAI (Responses API)] --> D
    G --> E
```

**Põhikomponendid:**
- **WorkflowBuilder**: Peamine orkestreerimismootor töövoogude disainimiseks
- **AIAgent**: Individuaalsed spetsialiseerunud agendid konkreetsete võimetega
- **Azure OpenAI klient**: Azure OpenAI Responses API integratsioon
- **Täitmise kontekst**: Halda olekut ja andmevoogu töövoo etappide vahel

## 🎨 **Ettevõtte töövoo disainimustrid**

### 📝 **Sisu tootmise töövoog**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **Dokumendi töötlemise torujuhe**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **Ärianalüüsi töövoog**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **Klienditeeninduse automatiseerimine**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 💻 Koodi käivitamine

Täielik rakendus on saadaval failis `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Selles failis demonstreeritakse:

1. **Keskkonna konfiguratsioon** - Azure OpenAI seadistuse laadimine `.env` failist
2. **Azure OpenAI kliendi seadistamine** - Kliendi konfigureerimine Azure OpenAI Responses API kasutamiseks
3. **Agendi loomine** - Spetsialiseerunud agentide (esiposti ja konssierži) defineerimine
4. **Töövoo koostaja** - Mitme agendiga töövoo loomine järkjärgulise töötlemisega
5. **Töövoo täitmine** - Töövoo käivitamine voogedastusega tulemustega

### 🚀 Näite käivitamine

```bash
# Tee skript täidetavaks (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Käivita töövoog
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Või Windowsis:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Oodatav väljund

Töövoog teeb järgmist:
1. Võtab vastu sinu reisisihtkoha soovi („Tahaksin minna Pariisi“)
2. Esiposti agent pakub algset soovitust
3. Konssierž agent vaatab soovituse üle ja täiustab seda
4. Lõpptulemus kuvab täieliku vestlusvoo

### 🔧 Kohandamine

Saad töövoogu kohandada järgmiselt:
- Muutes agentide juhiseid nende käitumise muutmiseks
- Lisades rohkem agente keerukate mitme sammuga töövoogude loomiseks
- Muutes kasutaja sõnumit erinevate stsenaariumide testimiseks
- Kohandades töövoo servasid erinevate täitmismustrite loomiseks
