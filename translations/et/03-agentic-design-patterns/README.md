# Tehisintellekti Agentide Kujundamise Põhimõtted

## Sissejuhatus

On palju viise, kuidas mõelda tehisintellekti agentsete süsteemide loomisele. Kuna ebamäärasus on genereeriva tehisintellekti disainis omadus, mitte viga, on mõnikord inseneridel keeruline isegi otsustada, kust alustada. Oleme loonud inimkeskse kasutajakogemuse disainipõhimõtete komplekti, mis võimaldab arendajatel luua kliendikeskseid agente süsteeme, mis lahendavad nende ärilisi vajadusi. Need disainipõhimõtted ei ole etteantud arhitektuur, vaid pigem lähtepunkt meeskondadele, kes määratlevad ja arendavad agentide kogemusi.

Üldiselt peaksid agendid:

- Laiendama ja suurendama inimvõimeid (ajurünnakud, probleemide lahendamine, automatiseerimine jne)
- Täitma teadmiste lünki (tõstma teadmistel silma peal hoidmise taset, tõlkimine jne)
- Lihtsustama ja toetama koostööd viisil, kuidas me eelistame teistega töötada
- Muutma meid paremaks versiooniks iseendast (nt elunõustaja/tööde juhendaja, aidates omandada emotsionaalseid regulatsiooni ja teadveloleku oskusi, vastupidavuse kasvatamine jne)

## Selle õppetunni sisu

- Näited disainipõhimõtete rakendamisest

## Agentsete disainipõhimõtete kirjeldus

![Agentsete disainipõhimõtete skeem](../../../translated_images/et/agentic-design-principles.1cfdf8b6d3cc73c2.webp)

### Agent (ruum)

See on keskkond, milles agent toimib. Need põhimõtted juhivad, kuidas me kujundame agente, kes tegutsevad füüsilises ja digitaalses maailmas.

- **Ühendamine, mitte kokkuvarisemine** – aidata inimestel ühenduda teiste inimestega, sündmuste ja rakendatava teadmistega, et võimaldada koostööd ja sidet.
- Agendid aitavad sündmusi, teadmisi ja inimesi ühendada.
- Agendid toovad inimesed lähemale. Neid ei ole loodud inimesi asendama ega alavääristama.
- **Lihtsalt ligipääsetav, kuid aeg-ajalt nähtamatu** – agent tegutseb enamasti taustal ja hoiatab meid vaid siis, kui see on asjakohane ja sobiv.
  - Agent on volitatud kasutajatele igal seadmel või platvormil hõlpsasti leitav ja ligipääsetav.
  - Agent toetab multimodaalseid sisendeid ja väljundeid (heli, hääl, tekst jne).
  - Agent suudab sujuvalt liikuda esiplaanilt taustale; proaktiivselt ja reageerivalt, sõltuvalt kasutaja vajaduste tajumisest.
  - Agent võib toimida nähtamatus vormis, kuid selle taustaprotsess ja koostöö teiste agentidega on kasutajale läbipaistvad ja juhitavad.

### Agent (aeg)

See on, kuidas agent ajas toimib. Need põhimõtted juhivad, kuidas me kujundame agente, kes suhtlevad mineviku, oleviku ja tulevikuga.

- **Minevik**: peegeldades ajalugu, mis sisaldab nii seisundit kui ka konteksti.
  - Agent pakub asjakohasemaid tulemusi, analüüsides rikkalikumaid ajaloolisi andmeid, mitte ainult sündmust, inimesi või seisundeid.
  - Agent loob ühendusi minevikusündmuste vahel ja peegeldab aktiivselt mälu, et suhelda praeguste olukordadega.
- **Praegune aeg**: torkamine rohkem kui teavitamine.
  - Agent kehastab terviklikku meetodit suhtlemiseks inimestega. Kui midagi juhtub, läheb agent staatilisest teavitamisest kaugemale. Agent suudab lihtsustada protsesse või dünaamiliselt genereerida vihjeid, et suunata kasutaja tähelepanu õigel hetkel.
  - Agent edastab teavet vastavalt kontekstuaalsele keskkonnale, sotsiaalsetele ja kultuurilistele muutustele ning kasutaja kavatsustele.
  - Agenti suhtlus võib olla järkjärguline, arenedes ja kasvades keerukuses, et pikemas perspektiivis kasutajaid võimestada.
- **Tulevik**: kohandumine ja areng.
  - Agent kohandub erinevate seadmete, platvormide ja modaalustega.
  - Agent kohandub kasutajakäitumise, ligipääsetavuse vajadustega ning on vabalt kohandatav.
  - Agent kujuneb ja areneb pideva kasutajasuhtluse kaudu.

### Agent (tuum)

Need on võtmeelemendid agendi tuuma kujundamisel.

- **Aktsepteeri ebakindlust, kuid loo usaldus**.
  - Agendi ebakindluse tase on oodatud. Ebakindlus on agendi kujunduse oluline osa.
  - Usaldus ja läbipaistvus on agendi kujunduse aluspõhjad.
  - Inimesed juhivad, millal agent on sisse või välja lülitatud ning agendi olek on igal ajal selgelt nähtav.

## Juhised nende põhimõtete rakendamiseks

Kui kasutad eelnevaid disainipõhimõtteid, järgi järgmisi juhiseid:

1. **Läbipaistvus**: Teavita kasutajat, et AI on kaasatud, kuidas see toimib (sh varasemad tegevused) ning kuidas anda tagasisidet ja süsteemi muuta.
2. **Kontroll**: Võimalda kasutajal süsteemi kohandada, määrata eelistusi ja personaliseerida ning omada kontrolli süsteemi ja selle omaduste üle (ka võimalus unustada).
3. **Järjepidevus**: Püüdle järjepidevate, mitmemodaalsete kogemuste poole erinevatel seadmetel ja otsapunktides. Kasuta tuntud UI/UX elemente võimalusel (nt mikrofonisümbol häälinteraktsiooniks) ja vähenda kliendi kognitiivset koormust nii palju kui võimalik (nt lühikesed vastused, visuaalsed abivahendid ja „Loe veel“ sisu).

## Kuidas kujundada reisibürooakent selle põhimõtete ja juhiste alusel

Kujutle, et kujundad Reisibürooakenti – siin on, kuidas võiksid kasutada disainipõhimõtteid ja -juhiseid:

1. **Läbipaistvus** – Teavita kasutajat, et Reisibürooagent on AI-toega agent. Anna mõned põhilised juhised alustamiseks (nt „Tere“ sõnum, näidispäringud). Dokumenteeri see selgelt tootelehel. Näita kasutaja varasemaid küsimusi sisaldavat päringute nimekirja. Selgita, kuidas anda tagasisidet (näpud üles/alla, „Saada tagasisidet“ nupp jne). Too selgelt välja, kui agendil on kasutus- või teemapiirangud.
2. **Kontroll** – Veendu, et oleks selge, kuidas kasutaja saab agendi loomise järel muuta süsteemi päringutega. Võimalda valida, kui kõnekas agent on, tema kirjutamisstiil ja millest agent ei tohiks rääkida. Luba kasutajal vaadata ja kustutada seotud faile, andmeid, päringuid ja varasemaid vestlusi.
3. **Järjepidevus** – Veendu, et ikoonid jagamiseks, faili või foto lisamiseks ning kellegi või millegi märgistamiseks oleksid standardiseeritud ja äratuntavad. Kasuta paperclip'i ikooni faili üleslaadimise/jagamise näitamiseks agentiga ja pildi ikoone graafika üleslaadimiseks.
