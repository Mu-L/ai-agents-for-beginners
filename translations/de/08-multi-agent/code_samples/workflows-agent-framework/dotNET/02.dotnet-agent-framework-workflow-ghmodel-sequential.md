# ⏩ Sequentielle Agenten-Workflows mit Azure OpenAI (Responses API) (.NET)

## 📋 Fortgeschrittenes Tutorial zur sequentiellen Verarbeitung

Dieses Notebook zeigt **sequentielle Workflow-Muster** unter Verwendung des Microsoft Agent Frameworks für .NET und Azure OpenAI (Responses API). Sie lernen, wie man ausgeklügelte, schrittweise Verarbeitungspipelines erstellt, bei denen Agenten in einer bestimmten Reihenfolge ausgeführt werden und jede Stufe auf den Ergebnissen der vorherigen aufbaut.

## 🎯 Lernziele

### 🔄 **Architektur der sequentiellen Verarbeitung**
- **Lineares Workflow-Design**: Erstellen Sie schrittweise Verarbeitungspipelines mit klaren Abhängigkeiten

- **Azure OpenAI (Responses API)**: Nutzen Sie Azure OpenAI-Modelle in mehrstufigen .NET-Workflows
- **Unternehmens-Pipeline-Muster**: Bauen Sie produktionsreife sequentielle Verarbeitungssysteme auf

### 🏗️ **Fortgeschrittene sequentielle Muster**
- **Stage-Gate-Verarbeitung**: Implementieren Sie Validierungspunkte zwischen Workflow-Stufen
- **Kontextbewahrung**: Bewahren Sie Zustand und kumuliertes Wissen über alle Stufen hinweg
- **Fehlerweitergabe**: Gehen Sie mit Fehlern in sequentiellen Verarbeitungsketten sorgsam um
- **Leistungsoptimierung**: Effiziente sequentielle Ausführung mit minimalem Overhead

### 🏢 **Sequentielle Anwendungen für Unternehmen**
- **Dokumentenverarbeitungspipeline**: Mehrstufige Dokumentenanalyse, -transformation und -validierung

### 🏗️ **Architektur des sequentiellen Workflows**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Analyse-Agent]
    B --> C[Kontrollpunkt 1]
    C --> D[Stage 2: Verarbeitungs-Agent]
    D --> E[Kontrollpunkt 2]
    E --> F[Stage 3: Validierungs-Agent]
    F --> G[Endausgabe]
    
    H[Zustandskontext] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Antworten API)] --> B
    I --> D
    I --> F
```

**Wesentliche Komponenten:**
- **Sequentielle Agenten**: Spezialisierte Agenten für jede Verarbeitungsstufe
- **Zustandskontext**: Bewahrt kumulierte Daten und Entscheidungen über die Stufen hinweg
- **Checkpoints**: Validierungspunkte zwischen den Stufen zur Sicherung von Qualität und Konsistenz
- **Azure OpenAI Client**: Konsistenter Zugriff auf KI-Modelle über alle Workflow-Stufen hinweg

## 💻 Code ausführen

Die vollständige Implementierung befindet sich in `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Diese Datei demonstriert einen **dreistufigen Analyse-Workflow für Möbel**:

1. **Stufe 1 - Vertriebsagent**: Analysiert Möbelbilder und gibt Kaufempfehlungen
2. **Stufe 2 - Preisagent**: Liefert detaillierte Preisaufschlüsselungen und Budgetoptionen
3. **Stufe 3 - Angebotsagent**: Erstellt ein professionelles Angebotsdokument im Markdown-Format

### 🏗️ **Workflow-Architektur**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Jeder Agent:
- Erhält die Ergebnisse der vorherigen Stufe als Kontext
- Baut mit spezialisierter Expertise auf der vorherigen Analyse auf
- Bewahrt die Kontinuität des Workflows durch Zustandsverwaltung

### 🚀 Beispiel ausführen

**Voraussetzungen:**
- Platzieren Sie ein Möbelbild unter `../imgs/home.png` (oder aktualisieren Sie die Variable `imgPath`)
- Konfigurieren Sie Ihre `.env`-Datei mit Ihrem Azure OpenAI-Endpunkt und Deployment, melden Sie sich dann mit `az login` an

```bash
# Machen Sie das Skript ausführbar (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Führen Sie den sequentiellen Arbeitsablauf aus
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Oder unter Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Erwartete Ausgabe

Der Workflow wird:
1. **Vertriebsagent**: Möbelstücke im Bild identifizieren und Empfehlungen geben
2. **Preisagent**: Detaillierte Preisanalyse mit Budgetstufen und Einkaufsempfehlungen hinzufügen
3. **Angebotsagent**: Ein formatiertes Angebotsdokument mit allen gesammelten Informationen erstellen

Die finale Ausgabe wird ein umfassendes, professionelles Möbelangebot basierend auf der Bildanalyse sein.

### 🔍 Verständnis des Zustandsflusses

Jeder Agent in der Sequenz erhält:
- **Ursprünglichen Input**: Die ursprüngliche Benutzeranfrage (Bild + Text)
- **Frühere Agentenantworten**: Alle vorherigen Antworten der Agenten im Gesprächsverlauf
- **Kumulierten Kontext**: Den während des gesamten Workflows gepflegten vollständigen Zustand

Dies ermöglicht eine ausgeklügelte mehrstufige Verarbeitung, bei der jeder Agent auf einem umfassenden Kontext aller vorherigen Stufen aufbaut.
