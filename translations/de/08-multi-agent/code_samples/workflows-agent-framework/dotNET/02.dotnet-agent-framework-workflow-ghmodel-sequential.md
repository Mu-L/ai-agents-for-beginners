# ⏩ Sequenzielle Agenten-Workflows mit Azure OpenAI (Responses API) (.NET)

## 📋 Fortgeschrittenes Tutorial zur sequenziellen Verarbeitung

Dieses Notizbuch demonstriert **sequenzielle Workflow-Muster** mit dem Microsoft Agent Framework für .NET und Azure OpenAI (Responses API). Sie lernen, wie man anspruchsvolle Schritt-für-Schritt-Verarbeitungspipelines erstellt, bei denen Agenten in einer bestimmten Reihenfolge ausgeführt werden, wobei jede Stufe auf den Ergebnissen der vorherigen aufbaut.

### 🏗️ **Architektur des sequenziellen Workflows**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Analyse-Agent]
    B --> C[Kontrollpunkt 1]
    C --> D[Stage 2: Verarbeitungs-Agent]
    D --> E[Kontrollpunkt 2]
    E --> F[Stage 3: Validierungs-Agent]
    F --> G[Endausgabe]
    
    H[Zustandskontext] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Antworten API)] --> B
    I --> D
    I --> F
```

**Schlüsselkomponenten:**
- **Sequenzielle Agenten**: Spezialisierte Agenten für jede Verarbeitungsstufe
- **Zustandskontext**: Verwaltet akkumulierte Daten und Entscheidungen über alle Stufen hinweg
- **Checkpoints**: Validierungspunkte zwischen Stufen zur Sicherstellung von Qualität und Konsistenz
- **Azure OpenAI Client**: Konsistenter Zugang zu KI-Modellen über alle Workflow-Stufen hinweg

## 💻 Code ausführen

Die vollständige Implementierung ist in `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs` verfügbar. Diese Datei demonstriert einen **dreistufigen Möbelanalyse-Workflow**:

1. **Stufe 1 - Vertriebsagent**: Analysiert Möbelbilder und gibt Kaufempfehlungen
2. **Stufe 2 - Preis-Agent**: Liefert detaillierte Preisaufschlüsselungen und Budgetoptionen
3. **Stufe 3 - Angebots-Agent**: Erstellt ein professionelles Angebotsdokument im Markdown-Format

### 🏗️ **Workflow-Architektur**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Jeder Agent:
- Erhält die Ausgabe der vorherigen Stufe als Kontext
- Basiert auf vorherigen Analysen mit spezialisiertem Fachwissen auf
- Erhält Workflow-Kontinuität durch Zustandsverwaltung

### 🚀 Beispiel ausführen

**Voraussetzungen:**
- Legen Sie ein Möbelbild unter `../imgs/home.png` ab (oder aktualisieren Sie die Variable `imgPath`)
- Konfigurieren Sie Ihre `.env`-Datei mit Ihrem Azure OpenAI-Endpunkt und Deployment, dann melden Sie sich mit `az login` an

```bash
# Machen Sie das Skript ausführbar (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Führen Sie den sequentiellen Arbeitsablauf aus
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Oder unter Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Erwartete Ausgabe

Der Workflow wird:
1. **Vertriebsagent**: Möbelstücke im Bild erkennen und Empfehlungen aussprechen
2. **Preis-Agent**: Detaillierte Preisanalysen mit Budgetstufen und Einkaufsempfehlungen hinzufügen
3. **Angebots-Agent**: Ein formatiertes Angebotsdokument mit allen gesammelten Informationen erzeugen

Die finale Ausgabe ist ein umfassendes, professionelles Möbelangebot basierend auf Bildanalyse.

### 🔍 Verständnis des Zustandsflusses

Jeder Agent in der Sequenz erhält:
- **Originalinput**: Die ursprüngliche Benutzeranfrage (Bild + Text)
- **Vorherige Agentenantworten**: Alle vorherigen Agentenantworten im Gesprächsverlauf
- **Akkumulierten Kontext**: Vollständig gepflegter Zustand über den gesamten Workflow

Dies ermöglicht eine anspruchsvolle mehrstufige Verarbeitung, bei der jeder Agent auf umfassendem Kontext aller vorherigen Stufen aufbaut.
