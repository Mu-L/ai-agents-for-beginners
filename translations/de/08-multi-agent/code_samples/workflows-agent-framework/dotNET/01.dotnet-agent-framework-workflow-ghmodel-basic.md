# 🔄 Grundlegende Agenten-Workflows mit Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial zur Workflow-Orchestrierung

Dieses Notebook zeigt, wie man anspruchsvolle **Agenten-Workflows** mit dem Microsoft Agent Framework für .NET und Azure OpenAI (Responses API) erstellt. Sie lernen, mehrstufige Geschäftsprozesse zu gestalten, bei denen KI-Agenten zusammenarbeiten, um komplexe Aufgaben durch strukturierte Orchestrierungsmuster zu bewältigen.

## 🎯 Lernziele

### 🏗️ **Grundlagen der Workflow-Architektur**
- **Workflow Builder**: Entwerfen und orchestrieren komplexer mehrstufiger KI-Prozesse
- **Agenten-Koordination**: Koordination mehrerer spezialisierter Agenten innerhalb von Workflows
- **Azure OpenAI (Responses API)**: Nutzung der Azure OpenAI Responses API in Workflows
- **Visuelle Workflow-Gestaltung**: Erstellen und Visualisieren von Workflow-Strukturen für besseres Verständnis

### 🔄 **Orchestrierungsmuster für Prozesse**
- **Sequenzielle Verarbeitung**: Verkettung mehrerer Agentenaufgaben in logischer Reihenfolge
- **Statusverwaltung**: Erhaltung von Kontext und Datenfluss über Workflow-Phasen hinweg
- **Fehlerbehandlung**: Implementierung robuster Fehlererholung und Workflow-Resilienz
- **Leistungsoptimierung**: Gestaltung effizienter Workflows für Unternehmensmaßstab

### 🏢 **Anwendungen von Enterprise-Workflows**
- **Automatisierung von Geschäftsprozessen**: Automatisierung komplexer organisatorischer Abläufe
- **Content-Produktions-Pipeline**: Redaktionelle Workflows mit Überprüfungs- und Genehmigungsstufen
- **Automatisierung des Kundenservices**: Mehrstufige Bearbeitung von Kundenanfragen
- **Datenverarbeitungs-Workflows**: ETL-Workflows mit KI-gestützter Transformation

### 🔑 **Azure OpenAI Konfiguration**

**Umgebung einrichten (.env-Datei):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Zugriff auf Azure OpenAI:**
1. Erstellen Sie eine Azure OpenAI-Ressource im Azure-Portal
2. Stellen Sie ein Modell bereit (z. B. `gpt-4.1-mini`) und merken Sie sich den Bereitstellungsnamen
3. Melden Sie sich mit `az login` an und konfigurieren Sie die Umgebungsvariablen wie oben gezeigt

### 🏗️ **Übersicht zur Workflow-Architektur**

```mermaid
graph TD
    A[Workflow-Builder] --> B[Agenten-Register]
    B --> C[Workflow-Ausführungs-Engine]
    C --> D[Agent 1: Inhaltsersteller]
    C --> E[Agent 2: Inhaltsprüfer] 
    D --> F[Workflow-Ergebnisse]
    E --> F
    G[Azure OpenAI (Antworten API)] --> D
    G --> E
```

**Schlüsselkomponenten:**
- **WorkflowBuilder**: Zentrale Orchestrierungs-Engine zur Gestaltung von Workflows
- **AIAgent**: Einzelne spezialisierte Agenten mit spezifischen Fähigkeiten
- **Azure OpenAI Client**: Integration der Azure OpenAI Responses API
- **Execution Context**: Verwalten des Zustands und Datenflusses zwischen Workflow-Phasen

## 💻 Code-Ausführung

Die vollständige Implementierung ist in `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` verfügbar. Diese Datei demonstriert:

1. **Umgebungskonfiguration** - Laden der Azure OpenAI-Konfiguration aus der `.env`-Datei
2. **Azure OpenAI Client Setup** - Konfiguration des Clients zur Nutzung der Azure OpenAI Responses API
3. **Agentenerstellung** - Definition spezialisierter Agenten (Front Desk und Concierge)
4. **Workflow Builder** - Erstellung eines Multi-Agenten-Workflows mit sequentieller Verarbeitung
5. **Ausführung des Workflows** - Durchführung des Workflows mit Streaming-Ergebnissen

### 🚀 Beispiel ausführen

```bash
# Machen Sie das Skript ausführbar (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Führen Sie den Workflow aus
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Oder unter Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Erwartete Ausgabe

Der Workflow wird:
1. Ihre Reiseanfrage annehmen ("Ich möchte nach Paris reisen")
2. Der Front Desk Agent gibt eine erste Empfehlung
3. Der Concierge Agent überprüft und verfeinert die Empfehlung
4. Die endgültige Ausgabe zeigt den vollständigen Gesprächsverlauf

### 🔧 Anpassung

Sie können den Workflow anpassen durch:
- Ändern der Agentenanweisungen, um deren Verhalten zu verändern
- Hinzufügen weiterer Agenten für komplexe mehrstufige Workflows
- Ändern der Benutzeranfrage, um verschiedene Szenarien zu testen
- Anpassung der Workflow-Kanten, um unterschiedliche Ausführungsmuster zu erstellen
