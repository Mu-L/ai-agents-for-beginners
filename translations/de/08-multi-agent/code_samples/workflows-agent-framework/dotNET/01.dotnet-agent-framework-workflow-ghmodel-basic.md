# 🔄 Grundlegende Agenten-Workflows mit Azure OpenAI (Responses API) (.NET)

## 📋 Workflow-Orchestrierung Tutorial

Dieses Notebook zeigt, wie man anspruchsvolle **Agenten-Workflows** mit dem Microsoft Agent Framework für .NET und Azure OpenAI (Responses API) erstellt. Sie lernen, mehrstufige Geschäftsprozesse zu gestalten, bei denen KI-Agenten zusammenarbeiten, um komplexe Aufgaben durch strukturierte Orchestrierungsmuster zu bewältigen.

## 🎯 Lernziele

### 🏗️ **Grundlagen der Workflow-Architektur**
- **Workflow Builder**: Entwerfen und orchestrieren komplexer mehrstufiger KI-Prozesse
- **Agenten-Koordination**: Koordinieren mehrerer spezialisierter Agenten innerhalb von Workflows
- **Azure OpenAI (Responses API)**: Nutzung der Azure OpenAI Responses API in Workflows
- **Visuelles Workflow-Design**: Erstellen und Visualisieren von Workflow-Strukturen für bessere Verständlichkeit

### 🔄 **Prozess-Orchestrierungsmuster**
- **Sequenzielle Verarbeitung**: Verketten mehrerer Agentenaufgaben in logischer Reihenfolge
- **Statusverwaltung**: Kontext und Datenfluss über Workflow-Stufen hinweg beibehalten
- **Fehlerbehandlung**: Robuste Fehlerwiederherstellung und Workflow-Resilienz implementieren
- **Leistungsoptimierung**: Effiziente Workflows für Unternehmensmaßstab entwerfen

### 🏢 **Unternehmens-Workflow-Anwendungen**
- **Automatisierung von Geschäftsprozessen**: Automatisieren komplexer organisatorischer Workflows
- **Content-Produktionspipeline**: Redaktionelle Workflows mit Prüfungs- und Freigabestufen
- **Automatisierung im Kundenservice**: Mehrstufige Bearbeitung von Kundenanfragen
- **Datenverarbeitungs-Workflows**: ETL-Workflows mit KI-gestützter Transformation

### 🔑 **Azure OpenAI Konfiguration**

**Umgebungseinrichtung (.env Datei):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Azure OpenAI Zugriff:**
1. Erstellen Sie eine Azure OpenAI-Ressource im Azure-Portal
2. Stellen Sie ein Modell bereit (z.B. `gpt-5-mini`) und notieren Sie den Bereitstellungsnamen
3. Melden Sie sich mit `az login` an und konfigurieren Sie die Umgebungsvariablen wie oben gezeigt

**Schlüsselkomponenten:**
- **WorkflowBuilder**: Haupt-Orchestrierungsmaschine zur Gestaltung von Workflows
- **AIAgent**: Einzelne spezialisierte Agenten mit spezifischen Fähigkeiten
- **Azure OpenAI Client**: Integration der Azure OpenAI Responses API
- **Execution Context**: Verwalten von Status und Datenfluss zwischen Workflow-Stufen

## 💻 Ausführung des Codes

Die vollständige Implementierung finden Sie in `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Diese Datei demonstriert:

1. **Umgebungskonfiguration** - Laden der Azure OpenAI-Konfiguration aus der `.env` Datei
2. **Azure OpenAI Client Setup** - Konfigurieren des Clients zur Nutzung der Azure OpenAI Responses API
3. **Agenten-Erstellung** - Definition spezialisierter Agenten (Rezeption und Concierge)
4. **Workflow Builder** - Erstellen eines Multi-Agenten-Workflows mit sequenzieller Verarbeitung
5. **Workflow-Ausführung** - Ausführen des Workflows mit Streaming-Ergebnissen

### 🚀 Ausführen des Beispiels

```bash
# Machen Sie das Skript ausführbar (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Führen Sie den Arbeitsablauf aus
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Oder unter Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Erwartete Ausgabe

Der Workflow wird:
1. Ihre Reiseanfrage annehmen („Ich möchte nach Paris reisen“)
2. Der Rezeption-Agent gibt eine erste Empfehlung
3. Der Concierge-Agent überprüft und verfeinert die Empfehlung
4. Die finale Ausgabe zeigt den kompletten Gesprächsverlauf

- Ändern der Benutzernachricht, um verschiedene Szenarien zu testen
