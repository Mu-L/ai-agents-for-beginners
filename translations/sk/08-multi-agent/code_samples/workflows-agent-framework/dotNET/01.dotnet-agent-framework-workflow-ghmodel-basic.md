# 🔄 Základné pracovné toky agentov s Azure OpenAI (Responses API) (.NET)

## 📋 Návod na orchestráciu pracovného toku

Tento poznámkový blok demonštruje, ako vybudovať sofistikované **pracovné toky agentov** pomocou Microsoft Agent Framework pre .NET a Azure OpenAI (Responses API). Naučíte sa vytvárať viacstupňové podnikové procesy, kde AI agenti spolupracujú na zvládnutí zložitých úloh prostredníctvom štruktúrovaných vzorov orchestrácie.

## 🎯 Výučbové ciele

### 🏗️ **Základy architektúry pracovných tokov**
- **Workflow Builder**: Navrhujte a orchestrujte zložité viacstupňové procesy AI
- **Koordinácia agentov**: Koordinujte viacerých špecializovaných agentov v pracovných tokoch
- **Azure OpenAI (Responses API)**: Využívajte Azure OpenAI Responses API v pracovných tokoch
- **Vizualizácia workflowu**: Vytvárajte a vizualizujte štruktúry pracovných tokov pre lepšie porozumenie

### 🔄 **Vzory orchestrácie procesov**
- **Sekvenčné spracovanie**: Reťazte viaceré úlohy agentov v logickom poradí
- **Správa stavu**: Udržiavajte kontext a tok dát cez etapy workflowu
- **Spracovanie chýb**: Implementujte robustné zotavenie z chýb a odolnosť workflowu
- **Optimalizácia výkonu**: Navrhujte efektívne pracovné toky pre prevádzku na podnikovej úrovni

### 🏢 **Podnikové aplikácie pracovných tokov**
- **Automatizácia podnikových procesov**: Automatizujte zložité organizačné pracovné toky
- **Výrobný obsahový kanál**: Redakčné pracovné toky s etapami revízie a schválenia
- **Automatizácia zákazníckeho servisu**: Viacstupňové riešenie zákazníckych dopytov
- **Pracovné toky spracovania dát**: ETL pracovné toky s transformáciami pomocou AI

### 🔑 **Konfigurácia Azure OpenAI**

**Nastavenie prostredia (.env súbor):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Prístup k Azure OpenAI:**
1. Vytvorte Azure OpenAI zdroj v Azure portáli
2. Nasadte model (napríklad `gpt-5-mini`) a zapíšte si názov nasadenia
3. Prihláste sa pomocou `az login` a nakonfigurujte premenné prostredia ako je uvedené vyššie

**Kľúčové súčasti:**

- **Azure OpenAI Client**: Integrácia Azure OpenAI Responses API
- **Execution Context**: Spravuje stav a tok dát medzi etapami workflowu

## 💻 Spustenie kódu

Kompletná implementácia je k dispozícii v `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Tento súbor demonštruje:

1. **Konfigurácia prostredia** - Načítanie konfigurácie Azure OpenAI zo súboru `.env`
2. **Nastavenie Azure OpenAI klienta** - Konfigurácia klienta pre použitie Azure OpenAI Responses API
3. **Vytvorenie agentov** - Definovanie špecializovaných agentov (recepcia a concierge)
4. **Workflow Builder** - Vytvorenie workflowu s viacerými agentmi a sekvenčným spracovaním
5. **Spustenie workflowu** - Behu workflowu so streamovanými výsledkami

### 🚀 Spustenie príkladu

```bash
# Nastavte skript ako spustiteľný (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Spustite pracovný postup
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Alebo na Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Očakávaný výstup

Workflow:
1. Prijme vašu požiadavku na cieľ cesty ("Chcel by som ísť do Paríža")
2. Agent recepcie poskytne počiatočné odporúčanie
3. Concierge agent prehodnotí a vylepší odporúčanie
4. Konečný výstup zobrazí kompletný tok konverzácie

### 🔧 Prispôsobenie

Workflow môžete prispôsobiť:
- Úpravou inštrukcií agentov na zmenu ich správania
- Pridaním ďalších agentov pre vytvorenie komplexných viacstupňových workflowov
- Zmenou používateľskej správy na testovanie rôznych scenárov
- Úpravou hrán workflowu na vytvorenie rôznych vzorov vykonávania
