# 🔄 Základné pracovné postupy agentov s Azure OpenAI (Responses API) (.NET)

## 📋 Návod na orchestráciu pracovného postupu

Tento zápisník demonštruje, ako vytvárať sofistikované **pracovné postupy agentov** pomocou Microsoft Agent Framework pre .NET a Azure OpenAI (Responses API). Naučíte sa vytvárať viacstupňové obchodné procesy, kde AI agenti spolupracujú na dosahovaní komplexných úloh cez štruktúrované vzory orchestrácie.

## 🎯 Ciele učenia

### 🏗️ **Základy architektúry pracovných postupov**
- **Staviteľ pracovného postupu**: Navrhujte a orchestrujte komplexné viacstupňové AI procesy
- **Koordinácia agentov**: Koordinujte viacerých špecializovaných agentov v rámci pracovných postupov
- **Azure OpenAI (Responses API)**: Využívajte Azure OpenAI Responses API v pracovných postupoch
- **Vizuálny dizajn pracovných postupov**: Vytvárajte a vizualizujte štruktúry pracovných postupov pre lepšie pochopenie

### 🔄 **Vzory orchestrácie procesov**
- **Sekvenčné spracovanie**: Reťazte viacero úloh agentov v logickom poradí
- **Správa stavu**: Udržiavajte kontext a tok dát naprieč etapami pracovného postupu
- **Riešenie chýb**: Implementujte robustné zotavenie z chýb a odolnosť pracovného postupu
- **Optimalizácia výkonu**: Navrhujte efektívne pracovné postupy pre podnikové operácie vo veľkom rozsahu

### 🏢 **Podnikové aplikácie pracovných postupov**

- **Automatizácia zákazníckych služieb**: Viacstupňové riešenie zákazníckych dopytov

### 🔑 **Konfigurácia Azure OpenAI**

**Nastavenie prostredia (.env súbor):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Prístup k Azure OpenAI:**
1. Vytvorte zdroj Azure OpenAI v Azure portáli
2. Nasadte model (napríklad `gpt-4.1-mini`) a zaznamenajte názov nasadenia
3. Prihláste sa pomocou `az login` a nastavte premenné prostredia podľa uvedeného príkladu

### 🏗️ **Prehľad architektúry pracovného postupu**

```mermaid
graph TD
    A[Tvorca pracovných tokov] --> B[Register agentov]
    B --> C[Motor vykonávania pracovných tokov]
    C --> D[Agent 1: Generátor obsahu]
    C --> E[Agent 2: Recenzent obsahu] 
    D --> F[Výsledky pracovných tokov]
    E --> F
    G[Azure OpenAI (Responses API)] --> D
    G --> E
```

**Kľúčové komponenty:**
- **WorkflowBuilder**: Hlavný orchestrátor pre návrh pracovných postupov
- **AIAgent**: Individuálni špecializovaní agenti so špecifickými schopnosťami
- **Azure OpenAI Client**: Integrácia Azure OpenAI Responses API
- **Execution Context**: Spravuje stav a tok dát medzi etapami pracovného postupu

## 💻 Spustenie kódu

Kompletná implementácia je dostupná v `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Tento súbor demonštruje:

1. **Konfigurácia prostredia** - Načítanie konfigurácie Azure OpenAI zo súboru `.env`
2. **Nastavenie Azure OpenAI klienta** - Konfigurácia klienta na použitie Azure OpenAI Responses API
3. **Vytvorenie agenta** - Definovanie špecializovaných agentov (Front Desk a Concierge)
4. **Staviteľ pracovného postupu** - Vytváranie viacagentových pracovných postupov so sekvenčným spracovaním
5. **Vykonávanie pracovného postupu** - Spustenie pracovného postupu s prúdujúcimi výsledkami

### 🚀 Spustenie príkladu

```bash
# Nastavte skript ako spustiteľný (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Spustite pracovný postup
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Alebo na Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Očakávaný výstup

1. Prijme vašu požiadavku na cestovnú destináciu ("Chcel by som ísť do Paríža")

3. Agent Concierge odporúčanie preskúma a vylepší
