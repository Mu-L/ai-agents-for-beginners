## 💻 Spustenie kódu

Kompletná implementácia je dostupná v `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Tento súbor demonštruje **Fan-Out/Fan-In súbežný pracovný postup** pre plánovanie ciest:

### 🏗️ **Architektúra pracovného postupu**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Kľúčové komponenty:**

1. **ConcurrentStartExecutor**: Súčasne vysiela užívateľskú požiadavku všetkým agentom
2. **Researcher Agent**: Súbežne analyzuje destinácie a atrakcie
3. **Planner Agent**: Súbežne vytvára podrobné cestovné plány
4. **ConcurrentAggregationExecutor**: Zbiera a spája výsledky od oboch agentov

### 🎯 **Fan-Out/Fan-In vzor**

Tento pracovný postup demonštruje klasický **Fan-Out/Fan-In** vzor:
- **Fan-Out**: Jedna vstupná správa je súčasne vysielaná viacerým agentom
- **Súbežné spracovanie**: Viacerí agenti pracujú paralelne na tej istej úlohe
- **Fan-In**: Výsledky všetkých agentov sú zhromaždené a agregované do jedného výstupu

### 🔍 Pochopenie vlastných executorov

**ConcurrentStartExecutor:**
- Implementuje `IMessageHandler<string>` pre prijímanie reťazcových vstupov
- Vysiela správy všetkým pripojeným agentom
- Odosiela `TurnToken` na spustenie súbežného spracovania

**ConcurrentAggregationExecutor:**
- Implementuje `IMessageHandler<ChatMessage>` pre prijímanie odpovedí agentov
- Zhromažďuje správy bezpečne pre vlákna
- Agreguje po príchode všetkých očakávaných odpovedí
- Vráti finálny výstup pomocou `context.YieldOutputAsync()`

### ⚡ Výhody výkonu

**Súbežné vs Sekvenčné:**
- Sekvenčne: Agent1 (30s) → Agent2 (30s) = **60 sekúnd spolu**
- Súbežne: Agent1 (30s) || Agent2 (30s) = **30 sekúnd spolu**

**Zlepšenie priepustnosti**: Až N× rýchlejšie pre N súbežných agentov (v závislosti od zaťaženia a zdrojov)

### 🛡️ Spracovanie chýb

Pracovný postup elegantne rieši zlyhania jednotlivých agentov:
- Ak jeden agent zlyhá, ostatní pokračujú v spracovaní
- Aggregátor môže implementovať logiku časového limitu
- Čiastočné výsledky môžu byť vrátené podľa potreby
