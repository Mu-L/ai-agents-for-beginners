## 💻 Spustenie kódu

Kompletná implementácia je dostupná v `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Tento súbor demonštruje **Fan-Out/Fan-In súbežný pracovný postup** pre plánovanie cestovania:

### 🏗️ **Architektúra pracovného postupu**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Kľúčové komponenty:**

1. **ConcurrentStartExecutor**: Súčasne vysiela požiadavku používateľa všetkým agentom
2. **Researcher Agent**: Súbežne analyzuje destinácie a atrakcie
3. **Planner Agent**: Súbežne vytvára podrobné cestovné plány
4. **ConcurrentAggregationExecutor**: Zhromažďuje a spája výsledky od oboch agentov

### 🎯 **Vzorec Fan-Out/Fan-In**

Tento pracovný postup demonštruje klasický vzorec **Fan-Out/Fan-In**:
- **Fan-Out**: Jedna vstupná správa je súčasne vysielaná viacerým agentom
- **Súbežné spracovanie**: Viac agentov pracuje paralelne na rovnakej úlohe
- **Fan-In**: Výsledky od všetkých agentov sú zhromaždené a agregované do jedného výstupu

### 🔍 Pochopenie vlastných vykonávateľov

**ConcurrentStartExecutor:**
- Implementuje `IMessageHandler<string>` na prijímanie vstupu ako reťazec
- Vysiela správy všetkým pripojeným agentom
- Posiela `TurnToken`, aby spustil súbežné spracovanie

**ConcurrentAggregationExecutor:**
- Implementuje `IMessageHandler<ChatMessage>` na prijímanie odpovedí agentov
- Zhromažďuje správy bezpečne pre vlákna
- Agreguje po prijatí všetkých očakávaných odpovedí
- Vracia konečný výstup cez `context.YieldOutputAsync()`

### ⚡ Výhody výkonu

**Súbežné vs Sekvenčné:**
- Sekvenčné: Agent1 (30s) → Agent2 (30s) = **60 sekúnd celkovo**
- Súbežné: Agent1 (30s) || Agent2 (30s) = **30 sekúnd celkovo**

**Zlepšenie priepustnosti**: Až N× rýchlejšie pre N súbežných agentov (v závislosti od záťaže a zdrojov)

### 🛡️ Spracovanie chýb

Pracovný postup elegantne rieši chyby jednotlivých agentov:
- Ak jeden agent zlyhá, ostatní pokračujú v spracovaní
- Agregátor môže implementovať logiku časového limitu
- V prípade potreby môžu byť vrátené čiastočné výsledky
