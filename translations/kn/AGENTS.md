## ಸೆಟಪ್ ಕಮಾಂಡ್‌ಗಳು

### ಪೂರ್ವಾಪೇಕ್ಷಿತಗಳು
- ಪೈಥಾನ್ 3.12 ಅಥವಾ ಅದಕ್ಕೆ ಮೇಲು
- ಅಜ್ಯೂರ್ ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್ (ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌಂಡ್ರಿಗೆ)
- ಅಜ್ಯೂರು CLI ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಿ ಮತ್ತು ಪ್ರಮಾಣೀಕರಿಸಿ (`az login`)

### ಆರಂಭಿಕ ಸೆಟಪ್

1. **ರೆಪೊಸಿಟರಿಯನ್ನು ಕ್ಲೋನ್ ಅಥವಾ ಫೋರ್ಕ್ ಮಾಡಿ:**
   ```bash
   gh repo fork microsoft/ai-agents-for-beginners --clone
   # ಅಥವಾ
   git clone https://github.com/microsoft/ai-agents-for-beginners.git
   cd ai-agents-for-beginners
   ```

2. **ಪೈಥಾನ್ ವರ್ಚುಯಲ್ ಪರಿಸರವನ್ನು ರಚಿಸಿ ಮತ್ತು ಸಕ್ರಿಯಗೊಳಿಸಿ:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # ವಿಂಡೋಸ್‌ನಲ್ಲಿ: venv\Scripts\activate
   ```

3. **ಅವಶ್ಯಕತೆಗಳನ್ನೂ ಸ್ಥಾಪಿಸಿ:**
   ```bash
   pip install -r requirements.txt
   ```

4. **ಪರಿಸರ变量ವನ್ನೂ ಸೆಟ್ ಮಾಡಿ:**
   ```bash
   cp .env.example .env
   # ನಿಮ್ಮ API ಕೀಗಳು ಮತ್ತು ಅಂತಿಮ ಬಿಂದುಗಳನ್ನು ಹೊಂದಿಸಿ .env ತಿದ್ದುಪಡಿ ಮಾಡಿ
   ```

### ಅಗತ್ಯವಿರುವ ಪರಿಸರ变量ಗಳು

**ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌಂಡ್ರಿಗೆ** (ಅಗತ್ಯ):
- `AZURE_AI_PROJECT_ENDPOINT` - ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌಂಡ್ರಿ ಯೋಜನಾ ಅಂತಿಮಬಿಂದು
- `AZURE_AI_MODEL_DEPLOYMENT_NAME` - ಮಾದರಿ ನಿಯೋಜನೆಯ ಹೆಸರು (ಉದಾ: gpt-4.1-mini)

**ಅಜ್ಯೂರು AI ಶೋಧನಗೆ** (ಪಾಠ 05 - RAG):
- `AZURE_SEARCH_SERVICE_ENDPOINT` - ಅಜ್ಯೂರು AI ಶೋಧನೆ ಅಂತಿಮಬಿಂದು
- `AZURE_SEARCH_API_KEY` - ಅಜ್ಯೂರು AI ಶೋಧನೆ API ಕೀ

ಪ್ರಮಾಣೀಕರಣ: ನೋಟ್ಬುಕ್ಸ್ ಚಲಿಸುವ ಮೊದಲು `az login` ನಡಿಸುವುದು (`AzureCliCredential` ಉಪಯೋಗಿಸುವುದು).

## ಪರೀಕ್ಷಣೆ ಸೂಚನೆಗಳು

ಇದು ಸ್ವಯಂ ಪರಿಚಯ ಜ್ಞಾನಕೋಶ ಮತ್ತು ಸ್ವಯಂಚಾಲಿತ ಪರೀಕ್ಷೆಗಳೊಂದಿಗೆ ಉತ್ಪಾದನಾ ಕೋಡ್ ಅಲ್ಲ. ನಿಮ್ಮ ಸೆಟಪ್ ಮತ್ತು ಬದಲಾವಣೆಗಳನ್ನು ಪರಿಶೀಲಿಸಲು:

### ಕೈಯಿಂದ ಪರೀಕ್ಷಣೆ

1. **ಪೈಥಾನ್ ಪರಿಸರವನ್ನು ಪರೀಕ್ಷಿಸಿ:**
   ```bash
   python --version  # 3.12 ಅಥವಾ ಅದಕ್ಕಿಂತ ಹೆಚ್ಚಿನದಾಗಿರಬೇಕು
   pip list | grep -E "(agent-framework|azure-ai|azure-identity)"
   ```

2. **ನೋಟ್ಬುಕ್ ಕಾರ್ಯಾಚರಣೆಯನ್ನು ಪರೀಕ್ಷಿಸಿ:**
   ```bash
   # ನೋಟ್ಬುಕ್ ಅನ್ನು ಸ್ಕ್ರಿಪ್ಟ್‌ಗೆ ಪರಿವರ್ತಿಸಿ ಮತ್ತು ಚಾಲನೆ ಮಾಡಿ (ಟೆಸ್ಟ್‌ಗಳು ಇಂಪೋರ್ಟ್‌ಗಳನ್ನು ಪರೀಕ್ಷಿಸುತ್ತಿವೆ)
   jupyter nbconvert --to script <lesson-folder>/code_samples/<notebook>.ipynb --stdout | python
   ```

3. **ಪರಿಸರ变量ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ:**
   ```bash
   python -c "import os; from dotenv import load_dotenv; load_dotenv(); print('✓ AZURE_AI_PROJECT_ENDPOINT' if os.getenv('AZURE_AI_PROJECT_ENDPOINT') else '✗ AZURE_AI_PROJECT_ENDPOINT missing')"
   ```

### ವೈಯಕ್ತಿಕ ನೋಟ್ಬುಕ್ಸ್‌ಗಳನ್ನು ಚಲಿಸುವುದು

ಜೂಪಿಟರ್‌ನಲ್ಲಿ ನೋಟ್ಬುಕ್ಸ್ ತೆರೆಯಿರಿ ಮತ್ತು ಕ್ರಮವಾಗಿ ಸೆಲ್‌ಗಳನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸಿ. ಪ್ರತಿ ನೋಟ್ಬುಕ್ ಸ್ವತಃ ಸಂಪೂರ್ಣವಾಗಿದ್ದು ಈ ಕೆಳಗಿನವುಗಳನ್ನು ಒಳಗೊಂಡಿದೆ:
- ಆಮದು ಪ್ರಕಟಣೆಗಳು
- ಸಂರಚನೆ ಲೋಡ್ ಮಾಡುವುದು
- ಉದಾಹರಣೆ ಏಜೆಂಟ್ ಅನುಷ್ಟಾನಗಳು
- ಮಾರ್ಕ್‌ಡೌನ್ ಸೆಲ್‌ಗಳಲ್ಲಿ ನಿರೀಕ್ಷಿತ ಔಟ್‌ಪುಟ್‌ಗಳು

### ನಿಯೋಜಿಸಲಾದ ಏಜೆಂಟ್ಗಳ ಧೂಮಪಾನ-ಪರೀಕ್ಷಣೆ

ಪಾಠಗಳಲ್ಲಿ ಏಜೆಂಟ್ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌಂಡ್ರಿ ಹೋಸ್ಟ್ಗೆ‌ನಾಗಿ (01, 04, 05, 16) ನಿಯೋಜಿಸಲ್ಪಟ್ಟಾಗ, `tests/` ಅಡಿಯಲ್ಲಿ ಧೂಮಪಾನ-ಪರೀಕ್ಷಣಾ ಕಾಟಲಾಗ್‌ಗಳನ್ನು ರೆಪೊ ಕಳುಹಿಸುತ್ತದೆ, ಅವು `.github/workflows/smoke-test.yml` ಕಾರ್ಯಪ್ರವಾಹದ ಮೂಲಕ [AI Smoke Test](https://github.com/marketplace/actions/ai-smoke-test) ಕ್ರಿಯೆಯ ಮೂಲಕ ಚಲಿಸಲಾಗುತ್ತದೆ. ಇವು ಬೆಳಗಿನ ನಿಯೋಜನೆಗೆ后的 ಬಾಗಿಲು (ಏಜೆಂಟ್ ತಲುಪಬಹುದೋ ಮತ್ತು ಮೂಲ ಪ್ರಾಂಪ್ಟ್ ನಿರೀಕ್ಷೆಗಳನ್ನು ಅನುಸರಿಸುತ್ತವೆಯೋ?), ಪಾಠ 10 ಮತ್ತು 16 ರ ಮೌಲ್ಯಮಾಪನ ಪೈಪ್ಲೈನ್ ಅನ್ನು ಪೂರ್ಣಗೊಳಿಸುತ್ತವೆ. ಕಾಟಲಾಗ್-ನಿಂದ-ಪಾಠ-ನಡೆಗೊರವುಗೂ ಸಂಬಂಧಿಸಿದ ವಿವರಗಳಿಗೆ [tests/README.md](./tests/README.md) ನೋಡಿ. ಪಾಠ 17 ಅನ್ನು ಸ್ಥಳೀಯವಾಗಿ ಫೌಂಡ್ರಿ ಲೋಕಲ್‌ೊಂದಿಗೆ ಚಲಿಸಲಾಗುತ್ತದೆ ಮತ್ತು ಹೋಸ್ಟಿಂಗ್ ಅಂತಿಮಬಿಂದುವಿಲ್ಲ, ಆದ್ದರಿಂದ ನೇರವಾಗಿ ಅದರ ನೋಟ್ಬುಕ್ ಚಾಲನೆಯಿಂದ ಪರಿಶೀಲಿಸಲಾಗುತ್ತದೆ.

1. ರೆಪೊನನ್ನು ಫೋರ್ಕ್ ಅಥವಾ ಕ್ಲೋನ್ ಮಾಡಿ
2. ಸ್ಥಳೀಯವಾಗಿ ಅಥವಾ GitHub ಕೊಡ್ಸ್ಪೇಸಿನಲ್ಲಿ ನೋಟ್ಬುಕ್ಸ್ ಚಾಲನೆ ಮಾಡಿ
3. ಉದಾಹರಣೆಗಳನ್ನು ತಿದ್ದುಪಡಿ ಮತ್ತು ಪ್ರಯೋಗಮಾಡುವುದರಿಂದ ಕಲಿತುಕೊಳ್ಳಿ

## ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್ ಮಾರ್ಗಸೂಚಿಗಳು

### ಸಲ್ಲಿಸುವ ಮೊದಲು

1. **ನಿಮ್ಮ ಬದಲಾವಣೆಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ:**
   - ಪ್ರಭಾವಿತ ನೋಟ್ಬುಕ್ಸ್‌ಗಳನ್ನು ಸಂಪೂರ್ಣವಾಗಿ ಚಾಲನೆ ಮಾಡಿ
   - ಎಲ್ಲಾ ಸೆಲ್‌ಗಳು ದೋಷರಹಿತರಾಗಿ ಕಾರ್ಯಗತವಾಗುತ್ತವೆ ಎಂದು ಖಚಿತಪಡಿಸಿ
   - ಔಟ್‌ಪುಟ್‌ಗಳು ಸೂಕ್ತವಾಗಿವೆ ಎಂದು ಪರಿಶೀಲಿಸಿ

2. **ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ನವೀಕರಣ:**
   - ಹೊಸ ಕಾನ್ಸೆಪ್ಟ್ಗಳನ್ನು ಸೇರಿಸಿದರೆ README.md ನವೀಕರಿಸಿ
   - ಸಂಕೀರ್ಣ ಕೋಡ್‌ಗೆ ನೋಟ್ಬುಕ್‌ಗಳಲ್ಲಿ ಟಿಪ್ಪಣಿಗಳನ್ನು ಸೇರಿಸಿ
   - ಮಾರ್ಕ್‌ಡೌನ್ ಸೆಲ್‌ಗಳು ಉದ್ದೇಶವನ್ನು ವಿವರಿಸುತ್ತವೆ ಎಂದು ಖಾತ್ರಿಪಡಿಸಿ

3. **ಫೈಲ್ ಬದಲಾವಣೆಗಳು:**
   - `.env` ಫೈಲ್‌ಗಳನ್ನು ಕೂಡುವುದನ್ನು ತಪ್ಪಿಸಿ (`.env.example` ಬಳಸಿ)
   - `venv/` ಅಥವಾ `__pycache__/` ಡೈರೆಕ್ಟರಿಗಳನ್ನು ಸೇರಿಸಬೇಡಿ
   - ನೋಟ್ಬುಕ್ ಔಟ್‌ಪುಟ್‌ಗಳನ್ನು ತತ್ವಗಳ ಪ್ರದರ್ಶನಕ್ಕೆ ಉಳಿಸಿ
   - ತಾತ್ಕಾಲಿಕ ಫೈಲ್‌ಗಳು ಮತ್ತು ಬ್ಯಾಕಪ್ ನೋಟ್ಬುಕ್‌ಗಳು (`*-backup.ipynb`) ತೆಗೆದುಹಾಕಿ

### ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್ ಶೀರ್ಷಿಕೆಯ ಸ್ವರೂಪ

- `[Fix] ಪಾಠ-XX READMEನಲ್ಲಿ ತಿದ್ದುಪಡಿ ಸರಿಪಡಿಸಿ`
- `[Update] ಪಾಠ-XX ರಲ್ಲಿ ಕೋಡ್ ಮಾದರಿಯನ್ನು ಸುಧಾರಿಸಿ`
- `[Docs] ಸೆಟಪ್ ಸೂಚನೆಗಳನ್ನು ನವೀಕರಿಸಿ`

- ನೋಟ್ಬುಕ್‌ಗಳು ದೋಷರಹಿತವಾಗಿ ಕಾರ್ಯಗತವಾಗಬೇಕು
- README ಫೈಲ್‌ಗಳು ಸ್ಪಷ್ಟವಾಗಿದ್ದು ನಿಖರವಾಗಿರಬೇಕು
- ರೆಪೊದಲ್ಲಿನ ಇತರ ಕೋಡ್ ಮಾದರಿಗಳನ್ನು ಅನುಸರಿಸಿ
- ಇತರ ಪಾಠಗಳೊಂದಿಗೆ ಪ್ರಾಮಾಣಿಕತೆಯನ್ನು ಉಳಿಸಿ

## ಹೆಚ್ಚುವರಿ ಟಿಪ್ಪಣಿಗಳು

### ಸಾಮಾನ್ಯ ತಪ್ಪಾಟಿಕೆಗಳು

1. **ಪೈಥಾನ್ ಆವೃತ್ತಿ ವ್ಯತ್ಯಾಸ:**
   - ಪೈಥಾನ್ 3.12+ ಬಳಸದ್ಧಂತೆ ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
   - ಕೆಲವು ಪ್ಯಾಕೇಜ್‌ಗಳು ಹಳೆಯ ಆವೃತ್ತಿಗಳೊಂದಿಗೆ ಕೆಲಸ ಮಾಡದಿರಬಹುದು
   - ಪೈಥಾನ್ ಆವೃತ್ತಿಯನ್ನು ಸ್ಪಷ್ಟಗೊಳಿಸಲು `python3 -m venv` ಬಳಸಿ

2. **ಪರಿಸರ变量ಗಳು:**
   - ಹಂಚಿಕೆ `.env.example` ನಿಂದ `.env` ಸೃಷ್ಠಿಸಿ ಎಂದೆಂದಿಗೂ
   - `.env` ಫೈಲ್ ಅನ್ನು ಸೇರಿಸಬೇಡಿ (ಇದು `.gitignore`ನಲ್ಲಿ ಇದೆ)
   - ಕೀಲಿಕಾರರಹಿತ Entra ID ಪ್ರಮಾಣೀಕರಣಕ್ಕಾಗಿ `az login` ಮೂಲಕ ಸाइनಿನ್ ಮಾಡಿ

3. **ಪ್ಯಾಕೇಜ್ ಸಂಘರ್ಷಗಳು:**
   - ಹೊಸ ವರ್ಚುಯಲ್ ಪರಿಸರ ಬಳಸಿ
   - ವೈಯಕ್ತಿಕ ಪ್ಯಾಕೇಜ್‌ಗಳಿಗಿಂತ `requirements.txt` ನಿಂದ ಸ್ಥಾಪನೆ ಮಾಡಿ
   - ಕೆಲವು ನೋಟ್ಬುಕ್ಸ್‌ಗಳಿಗೆ ಅವುಗಳ ಮಾರ್ಕ್‌ಡೌನ್ ಸೆಲ್‌ನಲ್ಲಿ ಹೇಳಿದ ಹೆಚ್ಚುವರಿ ಪ್ಯಾಕೇಜ್‌ಗಳು ಬೇಕಾಗಬಹುದು

4. **ಅಜ್ಯೂರ್ ಸೇವೆಗಳು:**
   - ಅಜ್ಯೂರ್ AI ಸೇವೆಗಳಿಗೆ ಸಕ್ರಿಯ ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್ ಅಗತ್ಯ
   - ಕೆಲವು ವೈಶಿಷ್ಟ್ಯಗಳು ಪ್ರಾದೇಶಿಕವಾಗಿವೆ
   - ನಿಮ್ಮ ಅಜ್ಯೂರ್ OpenAI ಮಾದರಿ ನಿಯೋಜನೆ Responses API ಅನ್ನು ಬೆಂಬಲಿಸಬೇಕು ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
