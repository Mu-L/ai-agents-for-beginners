# 🔄 ಅಜೂರ್ ಓಪನ್‌ಎಐ (ರಿಸ್ಪಾನ್ಸಸ್ API) (.NET) ಬಳಸಿ ಮೂಲ ಏಜಂಟ್ ವರ್ಕ್‌ಫ್ಲೋಗಳು

## 📋 ವರ್ಕ್‌ಫ್ಲೋ ಆಯೋಜನೆ ಪಾಠ

ಈ ನೋಟ್‌ಬುಕ್‌ Microsoft Agent Framework for .NET ಮತ್ತು ಅಜೂರ್ ಓಪನ್‌ಎಐ (ರಿಸ್ಪಾನ್ಸಸ್ API) ಬಳಸಿ sofisticate ಮಾಡಿದ **ಏಜಂಟ್ ವರ್ಕ್‌ಫ್ಲೋಗಳನ್ನು** ನಿರ್ಮಿಸುವ ವಿಧಾನವನ್ನು ತೋರಿಸುತ್ತದೆ. ನೀವು ಸಂಘಟಿತ ಆಯೋಜನೆ ಮಾದರಿಗಳ ಮೂಲಕ ಸಂಕೀರ್ಣ ಕಾರ್ಯಗಳನ್ನು ಸಾಧಿಸಲು AI ಏಜೆಂಟ್‌ಗಳು ಸಹಕರಿಸುವ ಬಹುಹಂತ ವ್ಯವಹಾರ ಪ್ರಕ್ರಿಯೆಗಳನ್ನು ರಚಿಸುವುದನ್ನು ಕಲಿಯುತ್ತೀರಿ.

## 🎯 ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

### 🏗️ **ವರ್ಕ್‌ಫ್ಲೋ معماري ಮೂಲಭೂತಗಳು**
- **Workflow Builder**: ಸಂಕೀರ್ಣ ಬಹುಹಂತ AI ಪ್ರಕ್ರಿಯೆಗಳ ವಿನ್ಯಾಸ ಮತ್ತು ಆಯೋಜನೆ
- **ಏಜಂಟ್ ಸಹಕಾರ**: ವರ್ಕ್‌ಫ್ಲೋಗಳಲ್ಲಿ specialization ಮಾಡಿದ ಹಲವಾರು ಏಜೆಂಟ್‌ಗಳನ್ನು ಸಂಯೋಜನೆ
- **ಅಜೂರ್ ಓಪನ್‌ಎಐ (ರಿಸ್ಪಾನ್ಸಸ್ API)**: ವರ್ಕ್‌ಫ್ಲೋಗಳಲ್ಲಿ ಅಜೂರ್ ಓಪನ್‌ಎಐ ರಿಸ್ಪಾನ್ಸಸ್ API ಬಳಸಿ
- **ದೃಶ್ಯ ವರ್ಕ್‌ಫ್ಲೋ ವಿನ್ಯಾಸ**: ಉತ್ತಮ ಅರ್ಥಮಾಡಿಕೋಳಲು ವರ್ಕ್‌ಫ್ಲೋ ರಚನೆಗಳನ್ನು ಸೃಷ್ಟಿ ಮತ್ತು ದೃಶ್ಯೀಕರಣ

### 🔄 **ಪ್ರಕ್ರಿಯೆ ಆಯೋಜನೆ ಮಾದರಿಗಳು**
- **ಕ್ರಮಬದ್ಧ ಪ್ರೋಸೆಸಿಂಗ್**: ಹಲವಾರು ಏಜೆಂಟ್ ಚಟುವಟಿಕೆಗಳನ್ನು ತರ್ಕಬದ್ಧ ಕ್ರಮದಲ್ಲಿ ಶೃಂಗರಿಸುವುದು
- **ಸ್ಥಿತಿ ನಿರ್ವಹಣೆ**: ವರ್ಕ್‌ಫ್ಲೋ ಹಂತಗಳ ನಡುವಣ ಸಾಂದರ್ಭ ಮತ್ತು ಡೇಟಾ ಹರಿವು ಕಾಪಾಡಿಕೊಳ್ಳುವುದು
- **ದೋಷ ನಿರ್ವಹಣೆ**: ಕಟ್ಟುನಿಟ್ಟಾದ ದೋಷ ಮರುಸ್ಥಾಪನೆ ಮತ್ತು ವರ್ಕ್‌ಫ್ಲೋ ದಿರ್ಘಕಾಲಿಕತೆ ಅನುಷ್ಠಾನ
- **ಕಾರ್ಯಕ್ಷಮತೆ ಆಶ್ರಯ**: ಉದ್ಯಮ-ಮಟ್ಟದ ಕಾರ್ಯಾಚರಣೆಗೆ ಪರಿಣಾಮಕಾರಿ ವರ್ಕ್‌ಫ್ಲೋ ವಿನ್ಯಾಸ

### 🏢 **ಉದ್ಯಮ ವರ್ಕ್‌ಫ್ಲೋ ಅನ್ವಯಣೆಗಳು**
- **ವ್ಯವಹಾರ ಪ್ರಕ್ರಿಯಾ ಸ್ವಯಂಕರಣ**: ಸಂಕೀರ್ಣ ಸಂಸ್ಥೆಯ workflows ಸ್ವಯಂಚಾಲಿತಗೊಳಿಸುವುದು
- **ವಿಷಯ ಉತ್ಪಾದನಾ ಪೈಪ್ಲೈನ್**: ವಿಮರ್ಶೆ ಮತ್ತು ಅಂಗಳದಿಂದ ಸರ್ಕಾರಿ ಹಂತಗಳಿರುವ ಸಂಪಾದನೆ workflows
- **ಗ್ರಾಹಕ ಸೇವೆ ಸ್ವಯಂಕರಣ**: ಬಹು ಹಂತ ಗ್ರಾಹಕ ವಿಚಾರಣೆ ಪರಿಹಾರ
- **ಡೇಟಾ ಪ್ರಕ್ರಿಯೆ workflows**: AI-ಸಕ್ತಿ ಪರಿವರ್ತನೆಯೊಂದಿಗೆ ETL ವರ್ಕ್‌ಫ್ಲೋಗಳು

## ⚙️ ಮುಂಚೂಣಿಯ ಅಗತ್ಯತೆಗಳು ಮತ್ತು ಸೆಟಪ್

### 📦 **ಅಗತ್ಯ NuGet ಪ್ಯಾಕೇಜುಗಳು**

ಈ ವರ್ಕ್‌ಫ್ಲೋ ತೋರಿಸುವಿಕೆ ಪ್ರಮುಖ .NET ಪ್ಯಾಕೇಜುಗಳನ್ನು ಬಳಸುತ್ತದೆ:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **ಅಜೂರ್ ಓಪನ್‌ಎಐ ಸಂರಚನೆ**

**ಪರಿಸರ ಸೆಟಪ್ (.env ಫೈಲ್):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**ಅಜೂರ್ ಓಪನ್‌ಎಐ ಪ್ರವೇಶ:**
1. ಅಜೂರ್ ಪೋರ್ಟಲ್‌ನಲ್ಲಿ ಅಜೂರ್ ಓಪನ್‌ಎಐ ಸಂಪನ್ಮೂಲವನ್ನು ರಚಿಸಿ
2. ಮಾದರಿಯನ್ನು ನಿಯೋಜಿಸಿ (ಉದಾಹರಣೆಗೆ, `gpt-5-mini`) ಮತ್ತು ನಿಯೋಜನೆಯ ಹೆಸರನ್ನು નોંધಿಸಿ
3. `az login` ಬಳಸಿ ಸೈನ್ ಇನ್ ಮಾಡಿ ಮತ್ತು ಮೇಲ್ಕಂಡಂತೆ ಪರಿಸರ ಚರಗಳನ್ನು ಸಂರಚಿಸಿ

### 🏗️ **ವರ್ಕ್‌ಫ್ಲೋ معماري ಅವಲೋಕನ**

```mermaid
graph TD
    A[ಕಾರ್ಯಪ್ರವಾಹ ನಿರ್ಮಾಪಕ] --> B[ಏಜೆಂಟ್ ಪರಿಶೀಲೆಕ]
    B --> C[ಕಾರ್ಯಪ್ರವಾಹ ಕಾರ್ಯಗತಗೊಳಿಸುವ ಇಂಜಿನ್]
```

**ಪ್ರಮುಖ ಘಟಕಗಳು:**
- **WorkflowBuilder**: workflows ವಿನ್ಯಾಸಕ್ಕಾಗಿ ಮುಖ್ಯ ಆಯೋಜನಾ ಎಂಜಿನ್
- **AIAgent**: ವಿಶಿಷ್ಟ ಹೊಣೆಗಾರಿಕೆಗಳಿರುವ ವೈಯಕ್ತಿಕ specialization ಮಾಡಿದ ಏಜೆಂಟ್‌ಗಳು
- **ಅಜೂರ್ ಓಪನ್‌ಎಐ ಕ್ಲೈಯಂಟ್**: ಅಜೂರ್ ಓಪನ್‌ಎಐ ರಿಸ್ಪಾನ್ಸಸ್ API ಏಕೀಕರಣ
- **ಕಾರ್ಯನಿರ್ವಹಣಾ ಸನ್ನಿವೇಶ**: workflows ಹಂತಗಳ ಮಧ್ಯೆ ಸ್ಥಿತಿ ಮತ್ತು ಡೇಟಾ ಹರಿವು ನಿರ್ವಹಣೆ

## 🎨 **ಉದ್ಯಮ ವರ್ಕ್‌ಫ್ಲೋ ವಿನ್ಯಾಸ ಮಾದರಿಗಳು**

## 🏢 **ಉದ್ಯಮ ಉಪಕಾರಣೆಗಳು**

### 🎯 **ನಂಬಿಕಸ್ಥತೆ ಮತ್ತು ಉತ್ತೇಜನಶೀಲತೆ**
- **ನಿಶ್ಚಿತಾತ್ಮಕ ಕಾರ್ಯಾಚರಣೆ**: ನಿರಂತರ, ಪುನರಾವರ್ತಿತ workflows ಫಲಿತಾಂಶಗಳು
- **ದೋಷ ಪುನರ್ವಸತಿ**: ಯಾವುದೇ workflow ಹಂತದಲ್ಲಿ ವೈಫಲ್ಯಗಳ ಪ್ರಾಯೋಗಿಕ ನಿರ್ವಹಣೆ
- **ಕಾರ್ಯಕ್ಷಮತೆ ನಿಗಾದರ್ಶನ**: ಕಾರ್ಯಾಚರಣೆ ಮೆಟ್ರಿಕ್ಸ್ ಮತ್ತು ಸುಧಾರಣೆ ಅವಕಾಸಗಳ ಪತ್ತೆ
- **ಸಂಪನ್ಮೂಲ ನಿರ್ವಹಣೆ**: AI ಮಾದರಿ ಸಂಪನ್ಮೂಲಗಳ ಪರಿಣಾಮಕಾರಿ ಹಂಚಿಕೆ ಮತ್ತು ಬಳಕೆ

### 🔒 **ಭದ್ರತೆ ಮತ್ತು ಅನುಕೂಲತೆಗಳು**
- **ಭದ್ರವಾದ ಆದಾಯರಣೆ**: `az login` ಮೂಲಕ Microsoft Entra ID ಪ್ರಾಮಾಣೀಕರಣ (AzureCliCredential)
- **ವಿಚಾರಣಾ ದಾಖಲೆಗಳು**: workflow ಕಾರ್ಯಾಚರಣೆ ಮತ್ತು ನಿರ್ಣಯ ಬಿಂದುವಿನ ಸಂಪೂರ್ಣ ಲಾಗಿಂಗ್
- **ಪ್ರವೇಶ ನಿಯಂತ್ರಣ**: workflow ಕಾರ್ಯಾಚರಣೆ ಮತ್ತು ನಿಗಾದರ್ಶನಕ್ಕೆ ಸೂಕ್ಷ್ಮ ಅನುಮತಿಗಳು
- **ಡೇಟಾ ಗೌಪ್ಯತೆ**: workflowsಾದ್ಯಂತ ಸಂವೇದನಶೀಲ ಮಾಹಿತಿಯ ಭದ್ರ ನಿರ್ವಹಣೆ

### 📊 **ನಿರೀಕ್ಷಣೆ ಮತ್ತು ನಿರ್ವಹಣೆ**
- **ದೃಶ್ಯ workflow ವಿನ್ಯಾಸ**: ಪ್ರಕ್ರಿಯೆ ಹರಿವು ಮತ್ತು ಅವಲಂಬನೆಗಳ ಸ್ಪಷ್ಟ ಪ್ರದರ್ಶನ
- **ಕಾರ್ಯಾಚರಣೆ ನಿಗಾದರ್ಶನ**: workflow ಪ್ರಗತಿ ಹಾಗೂ ಕಾರ್ಯಕ್ಷಮತೆಯ ರಿಯಲ್-ಟೈಮ್ ಟ್ರ್ಯಾಕಿಂಗ್
- **ದೋಷ ವರದಿ**: ವಿಶದ ದೋಷ ವಿಶ್ಲೇಷಣೆ ಮತ್ತು ಡಿ-ಬಗಿಂಗ್ ಸಾಮರ್ಥ್ಯಗಳು
- **ಕಾರ್ಯಕ್ಷಮತೆ ವಿಶ್ಲೇಷಣೆ**: ಸುಧಾರಣೆ ಮತ್ತು ಕಾರ್ಯನಿಲುವಿನ ಯೋಜನೆಗಾಗಿ ಮೆಟ್ರಿಕ್ಸ್

## 💻 ಕೋಡ್ ಕಾರ್ಯಾಚರಣೆ

ಪೂರ್ಣ ಅನುಷ್ಠಾನ `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` ನಲ್ಲಿ ಲಭ್ಯವಿದೆ. ಈ ಫೈಲ್ ತೋರಿಸುತ್ತದೆ:

1. **ಪರಿಸರ ಸಂರಚನೆ** - `.env` ಫೈಲ್‌ನಿಂದ ಅಜೂರ್ ಓಪನ್‌ಎಐ ಸಂರಚನೆ ಲೋಡ್ ಮಾಡುವುದು
2. **ಅಜೂರ್ ಓಪನ್‌ಎಐ ಕ್ಲೈಯಂಟ್ ಸೆಟಪ್** - ಅಜೂರ್ ಓಪನ್‌ಎಐ ರಿಸ್ಪಾನ್ಸಸ್ API ಅನ್ನು ಬಳಸಲು ಕ್ಲೈಯಂಟ್ ಅನ್ನು ಸಂರಚಿಸುವುದು
3. **ಏಜಂಟ್ ಸೃಷ್ಟಿ** - specialization ಮಾಡಿದ ಏಜೆಂಟ್‌ಗಳನ್ನು (ಮೇಲುಕಟ್ಟಿನ ಡೆಸ್ಕ್ ಮತ್ತು ಕಾನ್ಸಿಯರ್ಜ್) ವ್ಯಾಖ್ಯಾನಿಸುವುದು
4. **ವರ್ಕ್‌ಫ್ಲೋ ಬಿಲ್ಡರ್** - ಕ್ರಮಬದ್ಧ ಪ್ರೋಸೆಸಿಂಗ್ ಜೊತೆಯೊಂದಿಗೆ ಬಹು-ಏಜಂಟ್ ವರ್ಕ್‌ಫ್ಲೋ ರಚನೆ
5. **ವರ್ಕ್‌ಫ್ಲೋ ಕಾರ್ಯಗತಗೊಳಿಸುವಿಕೆ** - ಸ್ರವಣ ಫಲಿತಾಂಶಗಳೊಂದಿಗೆ ವರ್ಕ್‌ಫ್ಲೋ ಚಾಲನೆ
