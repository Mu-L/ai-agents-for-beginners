# 🔄 ಅಜೂರ್ ಓಪನ್‌ಎಐ (_Responses API_) ಸಹಿತ ಮೂಲಭೂತ ಏಜೆಂಟ್ ಕಾರ್\u200cಯಪ್ರವಾಹಗಳು (.NET)

## 📋 ವರ୍କ\u200cಫ್ಲೋ ಒರ್ಕೆಸ್ಟ್ರೇಶನ್ ಟ್ಯೂಟೋರಿಯಲ್

ಈ ನೋಟ್ಬುಕ್‌ Мicrosoft Agent Framework for .NET ಮತ್ತು ಅಜೂರ್ ಓಪನ್‌ಎಐ (_Responses API_) ಬಳಸಿ ಸುಧಾರಿತ **ಏಜೆಂಟ್ ಕಾರ್\u200cಯಪ್ರವಾಹಗಳನ್ನು** ನಿರ್ಮಿಸುವುದನ್ನು ತೋರಿಸುತ್ತದೆ. ನೀವು ಬಹುಪದ ವಹಿವಾಟುಗಳಲ್ಲಿ ಎಐ ಏಜೆಂಟ್‌ಗಳು ಸಂಯೋಜನೆಯ ಮೂಲಕ ಸಂಕೀರ್ಣ ಕಾರ್ಯಗಳನ್ನು ಸಾಧಿಸಲು ಸಹಕರಿಸುವ ಮೂಲಕ ವಿನ್ಯಾಸ ಮತ್ತು ಒರ್ಕೆಸ್ಟ್ರೇಶನ್ ಮಾದರಿಗಳನ್ನು ಕಲಿಯುತ್ತೀರಿ.

## 🎯 ಕಲಿಕಾ ಉದ್ದೇಶಗಳು

### 🏗️ **ಕಾರ್\u200cಯಪ್ರವಾಹ ವಾಸ್ತುಶಿಲ್ಪ ಮೂಲತತ್ವಗಳು**
- **Workflow Builder**: ಸಂಕೀರ್ಣ ಬಹುಪದ AI ಪ್ರಕ್ರಿಯೆಗಳನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸಿ ಮತ್ತು ಒರ್ಕೆಸ್ಟ್ರೇಟ್ ಮಾಡುವುದು
- **ಏಜೆಂಟ್ ಸಂಯೋಜನೆ**: ಕಾರ್\u200cಯಪ್ರವಾಹಗಳಲ್ಲಿ ಅನೇಕ ವಿಶೇಷ ಏಜೆಂಟ್‌ಗಳನ್ನು ಸಂಯೋಜಿಸುವುದು
- **ಅಜೂರ್ ಓಪನ್‌ಎಐ (_Responses API_)**: ಕಾರ್\u200cಯಪ್ರವಾಹಗಳಲ್ಲಿ ಅಜೂರ್ ಓಪನ್‌ಎಐ ಪ್ರತಿಕ್ರಿಯೆಗಳ API ಬಳಕೆ
- **ದೃಶ್ಯ ವರಕ\u200cಫ್ಲೋ ವಿನ್ಯಾಸ**: ಉತ್ತಮ ಅರ್ಥಮಾಡಿಕೊಳ್\u200cಲು ವರಕ\u200cಫ್ಲೋ ರಚನೆಗಳನ್ನು ಸೃಷ್ಟಿಸಿ ಮತ್ತು ದೃಶ್ಯೀಕರಣ ಮಾಡಿಕೊಳ್ಳಿ

### 🔄 **ಪ್ರಕ್ರಿಯಾ ಒರ್ಕೆಸ್ಟ್ರೇಶನ್ ಮಾದರಿಗಳು**
- **ಕ್ರಮಕ್ರಮದ ಪ್ರಕ್ರಿಯೆ**: ಲಾಜಿಕಲ್ ಕ್ರಮದಲ್ಲಿ ಅನೇಕ ಏಜೆಂಟ್ ಕಾರ್ಯಗಳನ್ನು ಸರಣಿಯಲ್ಲಿ ಜೋಡಿಸು
- **ಸ್ಥಿತಿ ನಿರ್ವಹಣೆ**: ಕಾರ್\u200cಯಪ್ರವಾಹ ಹಂತಗಳಲ್ಲಿಈ ಸಂಧರ್ಭ ಮತ್ತು ದತ್ತಾಂಶ ಹರಿವನ್ನು ಕಾಪಾಡುವುದು
- **ದೋಷ ನಿಭಂಧನೆ**: ಬಲವಾದ ದೋಷ ಪರಿಹಾರ ಮತ್ತು ಕಾರ್\u200cಯಪ್ರವಾಹ ಸ್ಥಿರತೆ ಅನುಷ್ಠಾನ
- **ಕಾರ್ಯಕ್ಷಮತಾ ಸುಧಾರಣೆ**: ಉದ್ಯಮ-ಮಟ್ಟದ ಕಾರ್ಯಾಚರಣೆಗೆ ಪರಿಣಾಮಕಾರಿ ಕಾರ್\u200cಯಪ್ರವಾಹ ವಿನ್ಯಾಸ

### 🏢 **ಉದ್ಯಮ ಕಾರ್\u200cಯಪ್ರವಾಹ ಅನ್ವಯಿಕೆಗಳು**
- **ವ್ಯವಸಾಯ ಪ್ರಕ್ರಿಯಾ ಸ್ವಯಂಚಾಲನ**: ಸಂಕೀರ್ಣ ಸಂಘಟನಾತ್ಮಕ ಕಾರ್\u200cಯಪ್ರವಾಹಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತಗೊಳಿಸುವುದು
- **ವಿಷಯ ಉತ್ಪಾದನಾ ಪೈಪ್ಲೈನ್**: ವಿಮರ್ಶೆ மற்றும் ಅನುಮೋದನಾ ಹಂತವಿರುವ ಸಂಪಾದಕೀಯ ವರಕ\u200cಫ್ಲೋಗಳು
- **ಗ್ರಾಹಕ ಸೇವಾ ಸ್ವಯಂಚಾಲನ**: ಬಹುಪದ ಗ್ರಾಹಕ ವಿಚಾರಣೆ ಪರಿಹಾರ
- **ದತ್ತಾಂಶ ಪ್ರಕ್ರಿಯೆ ವರक\u200cಫ್ಲೋಗಳು**: AI-ಪ್ರೇರಿತ ಪರಿವರ್ತನೆಯೊಂದಿಗೆ ETL ವರಕ\u200cಫ್ಲೋಗಳು

## ⚙️ ಪೂರ್ವಾಪೇಕ್ಷಿತಗಳು ಮತ್ತು ಸೆಟ್ ಅಪ್

### 📦 **ಅಗತ್ಯ NuGet ಪ್ಯಾಕೇಜುಗಳು**

ಈ ವರಕ\u200cಫ್ಲೋ ಪ್ರದರ್ಶನವು ಕೆಲವು ಪ್ರಮುಖ .NET ಪ್ಯಾಕೇಜುಗಳನ್ನು ಬಳಸುತ್ತದೆ:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **ಅಜೂರ್ ಓಪನ್‌ಎಐ ಸಂರಚನೆ**

**ಪರಿಸರ ಸೆಟ್ ಅಪ್ (.env ಫೈಲ್):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**ಅಜೂರ್ ಓಪನ್‌ಎಐ ಪ್ರವೇಶ:**
1. ಅಜೂರ್ ಪೋರ್ಟಲ್‌ನಲ್ಲಿ ಅಜೂರ್ ಓಪನ್‌ಎಐ ಸಂಪನ್ಮೂಲವನ್ನು ರಚಿಸಿ
2. ಒಂದು ಮಾದರಿಯನ್ನು (ಉದಾಹರಣೆಗೆ, `gpt-4.1-mini`) ನಿಯೋಜಿಸಿ ಮತ್ತು ನಿಯೋಜನೆಯ ಹೆಸರನ್ನು ಗಮನಿಸಿ
3. `az login` ಮೂಲಕ ಸೈನ್ ಇನ್ ಮಾಡಿ ಮತ್ತು ಮೇಲ್ಕಾಣಿಸಿದಂತೆ ಪರಿಸರ ವ್ಯತ್ಯಯಗಳನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ

### 🏗️ **ವರಕ\u200cಫ್ಲೋ ವಾಸ್ತುಶಿಲ್ಪ ಅವಲೋಕನ**

```mermaid
graph TD
    A[ಕಾರ್ಯಪ್ರವಾಹ ನಿರ್ಮಾಪಕರ] --> B[ಏಜೆಂಟ್ рэгಿಸ್ಟ್ರಿ]
    B --> C[ಕಾರ್ಯಪ್ರವಾಹ ನಿರ್ವಹಣಾ ಎಂಜಿನ್]
```

**ಪ್ರಮುಖ ಅಂಶಗಳು:**

- **AIAgent**: ನಿಖರ ಸಾಮರ್ಥ್ಯಗಳಿರುವ ವೈಯಕ್ತಿಕ ವಿಶೇಷ ಏಜೆಂಟ್‌ಗಳು
- **ಅಜೂರ್ ಓಪನ್‌ಎಐ ಕ್ಲೈಂಟ್**: ಅಜೂರ್ ಓಪನ್‌ಎಐ ಪ್ರತಿಕ್ರಿಯೆಗಳ API ಅಳವಡಿಕೆ
- **ನಿರ್ವಹಣಾ ಸಂಧರ್ಭ**: ವರಕ\u200cಫ್ಲೋ ಹಂತಗಳ ನಡುವೆ ಸ್ಥಿತಿ ಮತ್ತು ದತ್ತಾಂಶ ಹರಿವನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ
