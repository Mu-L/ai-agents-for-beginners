## 💻 ಕೋಡ್ ಚಲಿಸಿ

ಪೂರ್ಣ ಜಾರಿಗೆ `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` ಲಭ್ಯವಿದೆ. ಈ ಫೈಲ್ ಪ್ರವಾಸ ಯೋಜನೆಗೆ **Fan-Out/Fan-In samtid ಕಾರ್ಯಪ್ರವಾಹ** ಪ್ರದರ್ಶಿಸುತ್ತದೆ:

### 🏗️ **ಕಾರ್ಯಪ್ರವಾಹ معماري**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**ಪ್ರಮುಖ ಘಟಕಗಳು:**

1. **ConcurrentStartExecutor**: ಬಳಕೆದಾರ ವಿನಂತಿಯನ್ನು ಎಲ್ಲಾ ಏಜೆಂಟ್‌ಗಳಿಗೆ ಒಂದೇ ಸಮಯದಲ್ಲಿ ಪ್ರಸಾರ ಮಾಡುತ್ತದೆ
2. **Researcher Agent**: ಗುರಿ ಸ್ಥಳ ಮತ್ತು ಆಕರ್ಷಣೆಗಳನ್ನು samtid ವಿಶ್ಲೇಷಣೆ ಮಾಡುವುದು
3. **Planner Agent**: ಸಂಯೋಜಿತ ಪ್ರವಾಸ ಯೋಜನೆಗಳನ್ನು samtid ಸೃಷ್ಟಿಸುತ್ತದೆ
4. **ConcurrentAggregationExecutor**: ಇಬ್ಬರು ಏಜೆಂಟ್‌ಗಳ ಫಲಿತಾಂಶಗಳನ್ನು ಸಂಗ್ರಹಿಸಿ ವಿಲೀಕರಿಸುತ್ತದೆ

### 🎯 **Fan-Out/Fan-In ಮಾದರಿ**

ಈ ಕಾರ್ಯಪ್ರವಾಹವು ಕ್ಲಾಸಿಕ್ **Fan-Out/Fan-In** ಮಾದರಿಯನ್ನು ಪ್ರದರ್ಶಿಸುತ್ತದೆ:
- **Fan-Out**: ಒಂದು ಇನ್ಪುಟ್ ಸಂದೇಶ ಹಲವು ಏಜೆಂಟ್‌ಗಳಿಗೆ ಒಂದೇ ಸಮಯದಲ್ಲಿ ಪ್ರಸಾರವಾಗುತ್ತದೆ
- **ಸಮಂತರ ಪ್ರೋಸೆಸಿಂಗ್**: ವಿವಿಧ ಏಜೆಂಟ್‌ಗಳು ಒಂದೇ ಕಾರ್ಯವನ್ನು ಪಾರಲಲಾಗಿ ಕಾರ್ಯಗತಗೊಳಿಸುತ್ತವೆ
- **Fan-In**: ಎಲ್ಲ ಏಜೆಂಟ್‌ಗಳ ಫಲಿತಾಂಶಗಳನ್ನು ಸಂಗ್ರಹಿಸಿ ಒಟ್ಟಾರೆ ಔಟ್‌ಪುಟ್‌ನಾಗಿ ವಿಲೀಕರಿಸಲಾಗುತ್ತದೆ
