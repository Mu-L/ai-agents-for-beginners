# ⏩ ಅಜೂರ್ ಓಪನ್‌ಎಐ (ಪ್ರತಿಕ್ರಿಯೆಗಳು API) (.NET) ಜೊತೆ ಕ್ರಮಬದ್ಧ ಏಜೆಂಟ್ ವರ್ಕ್‌ಫ್ಲೋಗಳು

## 📋 ಸುಧಾರಿತ ಕ್ರಮಬದ್ಧ ಪ್ರಕ್ರಿಯೆ ಪಾಠಶಾಲೆ

ಈ ನೋಟ್‌ಬುಕ್ .NET ಮತ್ತು ಅಜೂರ್ ಓಪನ್‌ಎಐ (ಪ್ರತಿಕ್ರಿಯೆಗಳು API)ಗಾಗಿ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಏಜೆಂಟ್ ಫ್ರೇಮ್‌ವರ್ಕ್ ಬಳಸಿ **ಕ್ರಮಬದ್ಧ ವರ್ಕ್‌ಫ್ಲೋ ಮಾದರಿಗಳನ್ನು** ತೋರಿಸುತ್ತದೆ. ಏಜೆಂಟ್‌ಗಳು ನಿರ್ದಿಷ್ಟ ಕ್ರಮದಲ್ಲಿ ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಗಂಭೀರ, ಹಂತ ಹಂತವಾಗಿ ಪ್ರಕ್ರಿಯೆ ಪೈಪ್‌ಲೈನ್ಗಳನ್ನು ನೀವು ನಿರ್ಮಿಸುವ ವಿಧಾನವನ್ನು ತಿಳಿಯೋರು, ಮತ್ತು ಪ್ರತಿ ಹಂತದ ಫಲಿತಾಂಶ ಹಿಂದಿನ ಹಂತದ ಫಲಿತಾಂಶದ ಮೇಲೆ ಸುದೃಢವಾಗಿದೆ.

## 🎯 ಅಧ್ಯಯನ ಗುರಿಗಳು

### 🔄 **ಕ್ರಮಬದ್ಧ ಪ್ರಕ್ರಿಯೆ ವಾಸ್ತುಶಿಲ್ಪ**
- **ಸರಳ ವರ್ಕ್‌ಫ್ಲೋ ವಿನ್ಯಾಸ**: ಸ್ಪಷ್ಟ ಅವಲಂಬನೆಗಳೊಂದಿಗೆ ಹಂತ ಹಂತದ ಪ್ರಕ್ರಿಯೆ ಪೈಪ್‌ಲೈನ್ಗಳನ್ನು ರಚಿಸಿ
- **ಸ್ಥಿತಿ ನಿರ್ವಹಣೆ**: ಕ್ರಮಬದ್ಧ ವರ್ಕ್‌ಫ್ಲೋ ಹಂತಗಳಾದ್ಯಾಂತ ಸಂದರ್ಭ ಮತ್ತು ಡೇಟಾ ಹರಿವು ಕಾಪಾಡಿ
- **ಅಜೂರ್ ಓಪನ್‌ಎಐ (ಪ್ರತಿಕ್ರಿಯೆಗಳು API)**: ಬಹು ಹಂತ .NET ವರ್ಕ್‌ಫ್ಲೋಗಳಲ್ಲಿ ಅಜೂರ್ ಓಪನ್‌ಎಐ ಮಾದರಿಗಳನ್ನು ಬಳಸಿಕೊಳ್ಳಿ
- **ಉದ್ಯಮ ಪೈಪ್‌ಲೈನ್ ಮಾದರಿಗಳು**: ಉತ್ಪಾದನೆಗೆ ಸಿದ್ಧ ಕ್ರಮಬದ್ಧ ಪ್ರಕ್ರಿಯೆ ವ್ಯವಸ್ಥೆಗಳನ್ನು ನಿರ್ಮಿಸಿ

### 🏗️ **ಸುಧಾರಿತ ಕ್ರಮಬದ್ಧ ಮಾದರಿಗಳು**
- **ಹಂತ-ಗೇಟು ಪ್ರಕ್ರಿಯೆ**: ವರ್ಕ್‌ಫ್ಲೋ ಹಂತಗಳ ನಡುವೆ ಮಾನ್ಯತೆ ಪರೀಕ್ಷೆಗಳನ್ನು ಜಾರಿಗೆಂದ್ರಿ
- **ಸಂದರ್ಭ ಕಾಪಡೆ**: ಎಲ್ಲಾ ಹಂತಗಳಲ್ಲಿ ಸ್ಥಿತಿ ಮತ್ತು ಸಂಗ್ರಹಿತ ಜ್ಞಾನ ಕಾಪಾಡಿ
- **ತಪ್ಪು ವಿಸ್ತರಣೆ**: ಕ್ರಮಬದ್ಧ ಪ್ರಕ್ರಿಯೆ ಸರಂಡಿಗಳಲ್ಲಿ ವೈಫಲ್ಯಗಳನ್ನು ಸೌಮ್ಯವಾಗಿ ನಿರ್ವಹಿಸಿ
- **ಕೆಲಸದ ಶಕ್ತಿ ಉತ್ತಮವೃದ್ದಿ**: ಕನಿಷ್ಠ ಓವರ್‌ಹೆಡ್‌ನೊಂದಿಗೆ ಪರಿಣಾಮಕಾರಿ ಕ್ರಮಬದ್ಧ ಕಾರ್ಯಾಚರಣೆ

### 🏢 **ಉದ್ಯಮ ಕ್ರಮಬದ್ಧ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು**
- **ದಾಖಲೆ ಪ್ರಕ್ರಿಯೆ ಪೈಪ್‌ಲೈನ್**: ಬಹು ಹಂತದ ದಾಖಲೆ ವಿಶ್ಲೇಷಣೆ, ಪರಿವರ್ತನೆ ಮತ್ತು ಮಾನ್ಯತೆ
- **ಗುಣಮಟ್ಟ ಭರವಸೆ ವರ್ಕ್‌ಫ್ಲೋಗಳು**: ಕ್ರಮಬದ್ಧ ವಿಮರ್ಶೆ, ಮಾನ್ಯತೆ ಮತ್ತು ಅನುಮೋದನಾ ಪ್ರಕ್ರಿಯೆಗಳು
- **ವಿಷಯ ಉತ್ಪಾದನೆ ಪೈಪ್‌ಲೈನ್**: ಸಂಶೋಧನೆ → ಬರವಣಿಗೆ → ಸಂಪಾದನೆ → ವಿಮರ್ಶೆ → ಪ್ರಕಟಣೆ
- **ವ್ಯವಹಾರ ಪ್ರಕ್ರಿಯೆ ಸ್ವಯಂಚಾಲನೆ**: ಸ್ಪಷ್ಟ ಹಂತ ಅವಲಂಬನೆಗಳೊಂದಿಗೆ ಬಹು ಹಂತ ವ್ಯವಹಾರ ವರ್ಕ್‌ಫ್ಲೋಗಳು

## ⚙️ ಪೂರ್ವಾಪೇಕ್ಷೆಗಳು ಮತ್ತು ಸಂರಚನೆ

### 📦 **ಅವಶ್ಯಕ NuGet ಪ್ಯಾಕೇಜುಗಳು**

.NET ಕ್ರಮಬದ್ಧ ವರ್ಕ್‌ಫ್ಲೋಗಳಿಗೆ ಅವಶ್ಯಕ ಪ್ಯಾಕೇಜುಗಳು:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **ಅಜೂರ್ ಓಪನ್‌ಎಐ ಸಂರಚನೆ**

**ಪರಿಸರ ಸಂರಚನೆ (.env ಫೈಲ್):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**ಕಾನ್ಫಿಗರೇಷನ್ ನಿರ್ವಹಣೆ:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🎯 ಪ್ರಾಯೋಗಿಕ ಅನ್ವಯಿಕೆಗಳು

ಈ ಕ್ರಮಬದ್ಧ ಮಾದರಿ ಸೂಕ್ತವಾಗಿದೆ:
- **ಇ-ಕಾಮರ್ಸ್**: ಉತ್ಪನ್ನ ವಿಶ್ಲೇಷಣೆ → ಬೆಲೆ ನಿಗಧಿ → ಉಲ್ಲೇಖ ರಚನೆ
- **ರಿಯಲ್ ಎಸ್ಟೇಟ್**: ಆಸ್ತಿಯ ವಿಶ್ಲೇಷಣೆ → ಮೌಲ್ಯಮಾಪನ → ಜಾಹೀರಿ ಸೃಷ್ಟಿ
- **ಬీమಾ**: ದಾವೆ ವಿಶ್ಲೇಷಣೆ → ಅಂಕಿಅಂಶ → ಉಲ್ಲೇಖ ರಚನೆ
- **ವಿಷಯ ಸೃಷ್ಟಿ**: ಸಂಶೋಧನೆ → ಬರವಣಿಗೆ → ಸಂಪಾದನೆ → ಪ್ರಕಟಣೆ

### 🔍 ಸ್ಥಿತಿ ಹರಿವು ಅರಿತುಕೊಳ್ಳುವುದು

ಕ್ರಮದಲ್ಲಿ ಪ್ರತಿ ಏಜೆಂಟ್ ಸ್ವೀಕರಿಸುವದು:
- **ಮೂಲ ಇನ್ಪುಟ್**: ಪ್ರಾಥಮಿಕ ಬಳಕೆದಾರ ಸಂದೇಶ (ಚಿತ್ರ + ಪಠ್ಯ)
- **ಹಿಂದಿನ ಏಜೆಂಟ್ ಔಟ್‌ಪುಟ್‌ಗಳು**: ಸಂಭಾಷಣಾ ಇತಿಹಾಸದಲ್ಲಿನ ಎಲ್ಲ ಹಿಂದಿನ ಏಜೆಂಟ್ ಪ್ರತಿಕ್ರಿಯೆಗಳು
