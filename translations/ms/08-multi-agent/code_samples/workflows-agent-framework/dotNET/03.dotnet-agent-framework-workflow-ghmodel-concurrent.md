## 💻 Menjalankan Kod

Pelaksanaan lengkap tersedia dalam `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Fail ini menunjukkan **aliran kerja serentak Fan-Out/Fan-In** untuk perancangan perjalanan:

### 🏗️ **Seni Bina Aliran Kerja**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Komponen Utama:**

1. **ConcurrentStartExecutor**: Menghantar siaran permintaan pengguna kepada semua ejen serentak
2. **Ejen Penyelidik**: Menganalisis destinasi dan tarikan secara serentak
3. **Ejen Perancang**: Membuat rancangan perjalanan terperinci secara serentak
4. **ConcurrentAggregationExecutor**: Mengumpul dan menggabungkan keputusan dari kedua-dua ejen

### 🎯 **Corak Fan-Out/Fan-In**

Aliran kerja ini menunjukkan corak klasik **Fan-Out/Fan-In**:
- **Fan-Out**: Satu mesej input disiarkan kepada pelbagai ejen serentak
- **Pemprosesan Serentak**: Pelbagai ejen bekerja secara selari pada tugasan yang sama
- **Fan-In**: Keputusan dari semua ejen dikumpul dan digabungkan menjadi satu output

### 🔍 Memahami Executor Tersuai

**ConcurrentStartExecutor:**
- Melaksanakan `IMessageHandler<string>` untuk menerima input rentetan
- Menyiarkan mesej kepada semua ejen yang bersambung
- Menghantar `TurnToken` untuk mencetuskan pemprosesan serentak

**ConcurrentAggregationExecutor:**
- Melaksanakan `IMessageHandler<ChatMessage>` untuk menerima respons ejen
- Mengumpul mesej dengan cara selamat berbenang
- Menggabungkan apabila semua respons yang dijangkakan tiba
- Menghasilkan output akhir menggunakan `context.YieldOutputAsync()`

### ⚡ Manfaat Prestasi

**Serentak vs Bersiri:**

- Berurutan: Agen1 (30s) → Agen2 (30s) = **60 saat jumlah keseluruhan**
- Serentak: Agen1 (30s) || Agen2 (30s) = **30 saat jumlah keseluruhan**

**Peningkatan hasil keluaran**: Sehingga N× lebih pantas untuk N agen serentak (bergantung pada beban kerja dan sumber)

### 🛡️ Pengendalian Ralat

Aliran kerja mengendalikan kegagalan agen individu dengan baik:
- Jika satu agen gagal, agen lain terus memproses
- Agregator boleh melaksanakan logik tamat masa
- Keputusan separa boleh dikembalikan jika perlu
