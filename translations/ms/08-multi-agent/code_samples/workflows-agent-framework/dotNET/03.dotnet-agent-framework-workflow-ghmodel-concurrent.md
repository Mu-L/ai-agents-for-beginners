# ⚡ Aliran Kerja Ejen Serentak dengan Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial Pemprosesan Selari Berprestasi Tinggi

Nota ini menunjukkan **corak aliran kerja serentak** menggunakan Rangka Kerja Ejen Microsoft untuk .NET dan Azure OpenAI (Responses API). Anda akan belajar bagaimana membina aliran kerja pemprosesan selari berprestasi tinggi yang memaksimumkan keluaran dengan menjalankan pelbagai ejen AI serentak sambil mengekalkan penyelarasan dan konsistensi data.

## 🎯 Objektif Pembelajaran

### 🚀 **Asas Pemprosesan Serentak**
- **Pelaksanaan Ejen Selari**: Jalankan pelbagai ejen AI secara serentak untuk prestasi maksimum
- **Corak Async/Await**: Manfaatkan model pengaturcaraan async .NET untuk kecekapan serentak
- **Azure OpenAI (Responses API)**: Selaraskan pelbagai panggilan serentak ke Azure OpenAI Responses API
- **Pengurusan Sumber**: Urus sumber model AI dengan cekap merentasi operasi serentak

### 🏗️ **Arkitektur Serentak Lanjutan**
- **Pemparalelan Berdasarkan Tugas**: Gunakan Perpustakaan Paralel Tugas .NET untuk pelaksanaan serentak terbaik
- **Corak Penyelarasan**: Selaraskan ejen serentak sambil mengelakkan keadaan perlumbaan
- **Pengimbangan Beban**: Agihkan kerja dengan cekap merentasi kapasiti pemprosesan serentak yang tersedia
- **Toleransi Kesalahan**: Tangani kegagalan ejen individu tanpa menghentikan keseluruhan aliran kerja

### 🏢 **Aplikasi Serentak Perusahaan**
- **Pemprosesan Dokumen Berjumlah Tinggi**: Proses pelbagai dokumen serentak
- **Analisis Kandungan Masa Nyata**: Analisis serentak aliran data masuk
- **Pengoptimuman Pemprosesan Kumpulan**: Maksimumkan keluaran untuk operasi pemprosesan data berskala besar
- **Analisis Multi-Modal**: Pemprosesan selari jenis dan format kandungan berbeza

## 💻 Menjalankan Kod

Pelaksanaan lengkap tersedia dalam `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Fail ini menunjukkan **aliran kerja serentak Fan-Out/Fan-In** untuk perancangan perjalanan:

### 🏗️ **Arkitektur Aliran Kerja**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Komponen Utama:**

1. **ConcurrentStartExecutor**: Menyiarkan permintaan pengguna kepada semua ejen serentak
2. **Ejen Penyelidik**: Menganalisis destinasi dan tarikan serentak
3. **Ejen Perancang**: Membina rancangan perjalanan terperinci secara serentak
4. **ConcurrentAggregationExecutor**: Mengumpul dan menggabungkan keputusan daripada kedua-dua ejen

### 🎯 **Corak Fan-Out/Fan-In**

Aliran kerja ini menunjukkan corak klasik **Fan-Out/Fan-In**:
- **Fan-Out**: Satu mesej input disiarkan ke pelbagai ejen secara serentak
- **Pemprosesan Serentak**: Pelbagai ejen bekerja selari pada tugas yang sama
- **Fan-In**: Keputusan daripada semua ejen dikumpulkan dan digabungkan menjadi output tunggal

### 🔍 Memahami Executor Tersuai

**ConcurrentStartExecutor:**
- Melaksanakan `IMessageHandler<string>` untuk menerima input string
- Menyiarkan mesej kepada semua ejen yang bersambung
- Menghantar `TurnToken` untuk mencetuskan pemprosesan serentak

**ConcurrentAggregationExecutor:**
- Melaksanakan `IMessageHandler<ChatMessage>` untuk menerima respons ejen
- Mengumpul mesej dengan kaedah selamat benang
- Menggabungkan apabila semua respons yang dijangka diterima
- Menghasilkan output akhir menggunakan `context.YieldOutputAsync()`

### ⚡ Manfaat Prestasi

**Serentak vs Berturutan:**
- Berturutan: Ejen1 (30s) → Ejen2 (30s) = **60 saat jumlah**
- Serentak: Ejen1 (30s) || Ejen2 (30s) = **30 saat jumlah**

**Peningkatan keluaran**: Sehingga N× lebih pantas untuk N ejen serentak (bergantung pada beban kerja dan sumber)

### 🛡️ Pengendalian Ralat

Aliran kerja mengendalikan kegagalan ejen individu dengan cermat:
- Jika satu ejen gagal, yang lain terus memproses
- Penggabung boleh melaksanakan logik tamat masa
- Keputusan sebahagian boleh dipulangkan jika perlu
