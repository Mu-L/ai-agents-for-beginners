# ⏩ Aliran Kerja Agen Berturutan dengan Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial Pemprosesan Berturutan Lanjutan

Notebook ini menunjukkan **corak aliran kerja berturutan** menggunakan Microsoft Agent Framework untuk .NET dan Azure OpenAI (Responses API). Anda akan belajar cara membina saluran pemprosesan langkah demi langkah yang canggih di mana agen melaksanakan dalam urutan tertentu, dengan setiap peringkat membina atas hasil peringkat sebelumnya.

## 🎯 Objektif Pembelajaran

### 🔄 **Senibina Pemprosesan Berturutan**
- **Reka Bentuk Aliran Kerja Linear**: Cipta saluran pemprosesan langkah demi langkah dengan pergantungan yang jelas
- **Pengurusan Negeri**: Mengekalkan konteks dan aliran data merentasi peringkat aliran kerja berturutan
- **Azure OpenAI (Responses API)**: Manfaatkan model Azure OpenAI dalam aliran kerja .NET berbilang peringkat
- **Corak Saluran Perniagaan**: Bina sistem pemprosesan berturutan yang sedia untuk pengeluaran

### 🏗️ **Corak Berturutan Lanjutan**
- **Pemprosesan Pintu Peringkat**: Laksanakan pemeriksaan pengesahan antara peringkat aliran kerja
- **Pemeliharaan Konteks**: Mengekalkan keadaan dan pengetahuan terkumpul merentasi semua peringkat
- **Penghantaran Ralat**: Mengendalikan kegagalan dengan teratur dalam rantai pemprosesan berturutan
- **Pengoptimuman Prestasi**: Pelaksanaan berturutan yang cekap dengan overhead minimal

### 🏢 **Aplikasi Berturutan Perusahaan**
- **Saluran Pemprosesan Dokumen**: Analisis dokumen berbilang peringkat, transformasi, dan pengesahan
- **Aliran Kerja Jaminan Kualiti**: Semakan berturutan, pengesahan, dan proses kelulusan
- **Saluran Pengeluaran Kandungan**: Penyelidikan → Penulisan → Penyuntingan → Semakan → Penerbitan
- **Automasi Proses Perniagaan**: Aliran kerja perniagaan berbilang langkah dengan pergantungan peringkat yang jelas

### 🔑 **Konfigurasi Azure OpenAI**

**Persediaan Persekitaran (fail .env):**
```env

AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Pengurusan Konfigurasi:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
```

### 🏗️ **Senibina Aliran Kerja Berturutan**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Ejen Analisis]
    B --> C[Titik Semak 1]
    C --> D[Stage 2: Ejen Pemprosesan]
    D --> E[Titik Semak 2]
    E --> F[Stage 3: Ejen Pengesahan]
    F --> G[Output Akhir]
```

- **Titik Pemeriksaan**: Titik pengesahan antara peringkat untuk memastikan kualiti dan konsistensi
- **Klien Azure OpenAI**: Akses model AI yang konsisten merentasi semua peringkat aliran kerja

## 💻 Menjalankan Kod

Pelaksanaan lengkap tersedia dalam `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Fail ini menunjukkan **aliran kerja analisis perabot tiga peringkat**:

1. **Peringkat 1 - Agen Jualan**: Menganalisis imej perabot dan menyediakan cadangan pembelian
2. **Peringkat 2 - Agen Harga**: Memberikan pecahan harga terperinci dan pilihan bajet
3. **Peringkat 3 - Agen Sebut Harga**: Menghasilkan dokumen sebut harga profesional dalam format Markdown

### 🏗️ **Senibina Aliran Kerja**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Setiap agen:
- Menerima output dari peringkat sebelumnya sebagai konteks
- Membina atas analisis sebelumnya dengan kepakaran khusus
- Mengekalkan kesinambungan aliran kerja melalui pengurusan negeri

### 🚀 Menjalankan Contoh

**Prasyarat:**
- Letakkan imej perabot di `../imgs/home.png` (atau kemas kini pembolehubah `imgPath`)
- Konfigurasikan fail `.env` anda dengan titik akhir dan penyebaran Azure OpenAI, kemudian log masuk dengan `az login`

```bash
# Jadikan skrip boleh dilaksanakan (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Jalankan aliran kerja berurutan
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Atau pada Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Output Dijangka

Aliran kerja akan:
1. **Agen Jualan**: Kenal pasti barangan perabot dari imej dan berikan cadangan
2. **Agen Harga**: Tambah analisis harga terperinci dengan tingkat bajet dan cadangan pembelian
3. **Agen Sebut Harga**: Hasilkan dokumen sebut harga berformat dengan semua maklumat disusun

Output akhir akan menjadi sebut harga perabot yang komprehensif dan profesional berdasarkan analisis imej.

### 🎯 Aplikasi Dunia Sebenar

Corak berturutan ini sesuai untuk:
- **E-dagang**: Analisis produk → Pemasaran harga → Penjanaan sebut harga
- **Hartanah**: Analisis harta → Penilaian → Penciptaan senarai

### 🔍 Memahami Aliran Negeri

Setiap agen dalam urutan menerima:
- **Input Asal**: Mesej pengguna awal (imej + teks)
- **Output Agen Terdahulu**: Semua respons agen sebelumnya dalam sejarah perbualan
- **Konteks Terkumpul**: Negeri lengkap yang dikekalkan sepanjang aliran kerja

Ini membolehkan pemprosesan berbilang peringkat yang canggih di mana setiap agen membina atas konteks menyeluruh dari semua peringkat sebelumnya.
