# 🔄 Aliran Kerja Ejen Asas dengan Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial Orkestrasi Aliran Kerja

Buku nota ini menunjukkan cara membina **aliran kerja ejen** yang canggih menggunakan Microsoft Agent Framework untuk .NET dan Azure OpenAI (Responses API). Anda akan belajar untuk membuat proses perniagaan berbilang langkah di mana ejen AI bekerjasama untuk menyelesaikan tugas kompleks melalui corak orkestrasi berstruktur.

## 🎯 Objektif Pembelajaran

### 🏗️ **Asas Seni Bina Aliran Kerja**
- **Pembina Aliran Kerja**: Reka dan orkestrasi proses AI berbilang langkah yang kompleks
- **Penyelarasan Ejen**: Menyelaras beberapa ejen khusus dalam aliran kerja
- **Azure OpenAI (Responses API)**: Menggunakan Azure OpenAI Responses API dalam aliran kerja
- **Reka Bentuk Aliran Kerja Visual**: Membuat dan memvisualisasikan struktur aliran kerja untuk pemahaman yang lebih baik

### 🔄 **Corak Orkestrasi Proses**
- **Pemprosesan Bersiri**: Rantai beberapa tugas ejen dalam urutan logik
- **Pengurusan Keadaan**: Menjaga konteks dan aliran data merentasi tahap aliran kerja
- **Pengendalian Ralat**: Melaksanakan pemulihan ralat yang kukuh dan ketahanan aliran kerja
- **Pengoptimuman Prestasi**: Mereka bentuk aliran kerja yang cekap untuk operasi skala perusahaan

### 🏢 **Aplikasi Aliran Kerja Enterprise**
- **Automasi Proses Perniagaan**: Automasi aliran kerja organisasi yang kompleks
- **Saluran Pengeluaran Kandungan**: Aliran kerja editorial dengan peringkat semakan dan kelulusan
- **Automasi Perkhidmatan Pelanggan**: Penyelesaian pertanyaan pelanggan berbilang langkah
- **Aliran Kerja Pemprosesan Data**: Aliran kerja ETL dengan transformasi dipacu AI

## ⚙️ Keperluan & Persediaan

### 📦 **Pakej NuGet Diperlukan**

Demonstrasi aliran kerja ini menggunakan beberapa pakej utama .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Konfigurasi Azure OpenAI**

**Persediaan Persekitaran (fail .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Akses Azure OpenAI:**
1. Cipta sumber Azure OpenAI dalam portal Azure
2. Lancarkan model (contohnya, `gpt-4.1-mini`) dan nota nama pelancaran
3. Log masuk dengan `az login` dan konfigurasikan pembolehubah persekitaran seperti yang ditunjukkan di atas

### 🏗️ **Gambaran Keseluruhan Seni Bina Aliran Kerja**

```mermaid
graph TD
    A[Pembina Aliran Kerja] --> B[Daftar Ejen]
    B --> C[Enjin Pelaksanaan Aliran Kerja]
    C --> D[Agent 1: Penjana Kandungan]
    C --> E[Agent 2: Pemeriksa Kandungan] 
    D --> F[Keputusan Aliran Kerja]
    E --> F
    G[Azure OpenAI (API Respons)] --> D
    G --> E
```

**Komponen Utama:**
- **WorkflowBuilder**: Enjin orkestrasi utama untuk mereka bentuk aliran kerja
- **AIAgent**: Ejen khusus individu dengan kebolehan tertentu
- **Azure OpenAI Client**: Integrasi Azure OpenAI Responses API
- **Execution Context**: Mengurus keadaan dan aliran data antara tahap aliran kerja

## 💻 Menjalankan Kod

Pelaksanaan lengkap tersedia dalam `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Fail ini menunjukkan:

1. **Konfigurasi Persekitaran** - Memuat konfigurasi Azure OpenAI dari fail `.env`
2. **Persediaan Klien Azure OpenAI** - Mengkonfigurasi klien untuk menggunakan Azure OpenAI Responses API
3. **Penciptaan Ejen** - Mendefinisikan ejen khusus (Kaunter Hadapan dan Concierge)
4. **Pembina Aliran Kerja** - Membuat aliran kerja berbilang ejen dengan pemprosesan bersiri
5. **Pelaksanaan Aliran Kerja** - Menjalankan aliran kerja dengan hasil streaming

### 🚀 Menjalankan Contoh

```bash
# Jadikan skrip boleh dilaksanakan (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Jalankan aliran kerja
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Atau di Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Output Dijangka

Aliran kerja akan:
1. Menerima permintaan destinasi perjalanan anda ("Saya ingin pergi ke Paris")
2. Ejen Kaunter Hadapan memberikan cadangan awal
3. Ejen Concierge mengkaji dan menyempurnakan cadangan
4. Output akhir memaparkan aliran perbualan lengkap
