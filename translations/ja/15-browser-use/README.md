# コンピュータ利用エージェント (CUA) の構築

コンピュータ利用エージェントは、人間がブラウザを開いてページを検査し、見た内容から次に行うべき最適な行動を取るのと同じように、ウェブサイトと対話できます。このレッスンでは、Airbnb を検索し、構造化されたリスティングデータを抽出し、ストックホルムで最も安い宿泊先を特定するブラウザ自動化エージェントを構築します。

このレッスンでは、AI駆動のナビゲーションに Browser-Use、ブラウザ制御に Playwright と Chrome DevTools Protocol (CDP)、ビジョン対応の推論に Azure OpenAI、そして構造化抽出に Pydantic を組み合わせています。

## はじめに

このレッスンで学ぶ内容：

- APIのみの自動化よりもコンピュータ利用エージェントが適している場合の理解
- 信頼性の高いブラウザライフサイクル管理のために Browser-Use と Playwright、CDP を組み合わせる方法
- Azure OpenAI のビジョンと構造化 Pydantic 出力を活用して動的ウェブページからリスティングデータを抽出する方法
- エージェントファースト、アクターファースト、ハイブリッドのブラウザ自動化ワークフローを使い分ける判断基準

## 学習目標

このレッスンを終えた頃には、あなたは以下のことができるようになります：

- Azure OpenAI と Playwright を使って Browser-Use を設定する
- 実際のウェブサイトをナビゲートし、動的なUI要素を扱うブラウザ自動化ワークフローを構築する
- 表示されているページ内容から型付きの結果を抽出し、下流のビジネスロジックに変換する
- ブラウザタスクの予測可能性に基づいてエージェントパターンとアクターパターンを選択する

## アーキテクチャ概要

ノートブックはハイブリッドなブラウザ自動化ワークフローを示しています：

1. ChromeをCDP有効で起動し、PlaywrightとBrowser-Useの両方が同じブラウザセッションを共有できるようにします。
2. Browser-UseエージェントがAirbnbのオープンナビゲーションタスク（ポップアップを閉じる、ストックホルムを検索するなど）を処理します。
3. 構造化されたPydanticスキーマを使ってアクティブページを検査し、リスティングのタイトル、1泊あたりの価格、評価、URLを抽出します。
4. Pythonのロジックで抽出したリスティングを比較し、最も安い結果を強調表示します。

このアプローチは、Browser-Useの得意な柔軟でビジョンベースの推論を維持しながら、必要な時に決定論的なブラウザ制御を提供します。

## 重要なポイントとベストプラクティス

### エージェントとアクターを使い分けるタイミング

| シナリオ | エージェントを使う | アクターを使う |
|----------|-----------|-----------|
| 動的レイアウト | はい、AIはページ変更に適応可能 | いいえ、セレクターが壊れやすい |
| 構造が既知 | いいえ、エージェントは直接制御より遅い | はい、高速かつ正確 |
| 要素の発見 | はい、自然言語が有効 | いいえ、正確なセレクターが必要 |
| タイミング制御 | いいえ、予測不可能 | はい、待機やリトライを完全に制御可能 |
| 複雑なワークフロー | はい、予期しないUI状態に対応 | いいえ、明示的な分岐が必要 |

### Browser-Useのベストプラクティス

1. 探索や動的ナビゲーションにはまずエージェントを使う。
2. インタラクションが予測可能になったら直接ページ制御に切り替える。
3. 抽出データは構造化された出力モデルを使って検証し、型安全にする。
4. 目に見えるUI変更が起こるアクションの後には戦略的に遅延を入れる。
5. 反復中はスクリーンショットを取得し、障害のデバッグを容易にする。
6. ウェブサイトは変化すると想定し、ポップアップやレイアウト変更に対するフォールバック戦略を設計する。
7. 柔軟性と正確性の両方を得るためにエージェントとアクターのパターンを組み合わせる。

### ブラウザエージェントの安全ガードレール

ブラウザエージェントはライブウェブサイト上で動作するため、既知のAPIを呼び出すだけのスクリプトよりも厳格な制限が必要です。ノートブックのデモから実際のワークフローに移行する前に、エージェントが何を見てクリックし、送信できるかの範囲を明確に定義してください。

1. **ブラウズ環境のスコーピング。** エージェントは専用のブラウザプロファイルやサンドボックスで実行し、タスクに必要なドメインに制限します。
2. **観察と行動を分離。** まずエージェントに検索、読み取り、データ抽出をさせ、フォーム送信、メッセージ送信、旅行予約、購入、レコード削除、アカウント設定変更などは明示的な承認ステップを必須にします。
3. **機密情報をプロンプトやトレースに含めない。** パスワード、支払い情報、セッションCookie、未加工の個人データはモデルコンテキストに入れず、認証はユーザーに任せ、ログからは機密フィールドを省きます。
4. **ページ内容は信用できない入力として扱う。** ウェブサイトにはエージェント向けの指示（目標変更、データ開示、保護解除、サイト訪問指示）が含まれることがあり、エージェントはそれらを無視すべきです。
5. **リスキーなステップ前後には決定論的なチェックを。** 最終ステップをユーザーに承認してもらう前に、現在のURL、ページタイトル、選択項目、価格、受取人、操作概要をコードで検証します。
6. **予算と停止条件の設定。** エージェントが使うアクション数、リトライ数、タブ数、時間を制限し、ページ状態が曖昧な場合はクリックを続けず停止します。
7. **有用な証拠のみ記録。** アクション概要、タイムスタンプ、URL、選択要素の説明、スクリーンショット参照を保存し、不要な機密ページ内容は保存しません。

Airbnbサンプルでは、安全なデフォルトとしてリスティングの検索と価格抽出のみを行います。ログイン、ホストへの連絡、予約完了はユーザー承認が必要な別操作とします。

## 実世界の例：Microsoft Project Opal

このレッスンで構築するエージェントは、小規模なローカル版の<strong>コンピュータ利用エージェント (CUA)</strong> であり、人間のようにブラウザを操作するプログラムです。Microsoft はこの考え方を企業向けに展開しており、Microsoft 365 Copilot の機能である<strong>[Project Opal (Frontier)](https://support.microsoft.com/en-us/microsoft-365-copilot/get-started-with-project-opal-frontier)</strong>として提供しています。

Project Opal では、タスクを記述するとエージェントがあなたに代わって<strong>Windows 365 Cloud PC上で安全にコンピューター操作</strong>を行い、組織のブラウザベースのアプリケーションやサイト、データを横断して作業します。作業は<strong>非同期でバックグラウンド実行</strong>され、いつでも作業の指示や制御を行えます。例としては：

- セキュリティグループのメンバーシップ要求管理
- コンプライアンスレビュー向けの監査証跡収集・検証
- ITインシデントのトリアージ（チケットステータス更新、担当者割り当て、重複終了）
- Excelデータをまとめて財務決算資料を作成

Opal は<strong>実運用対応で信頼性の高い</strong>コンピュータ利用エージェントの良い例であり、これまでのレッスンのコンセプトを強化しています：

| このコースのコンセプト | Project Opal の適用例 |
|------------------------|-----------------------------|
| **人間の介入 (Lesson 06)** | Opalはログイン認証情報や機密情報、曖昧な指示を待機し、明示的な確認なしにパスワード入力やフォーム送信を一切行いません。作業中に<em>制御を引き継ぐ</em>と<em>戻す</em>ことが可能です。 |
| **信頼性と安全性 (Lessons 06 & 18)** | 専用のWindows 365 Cloud PCで隔離実行され、初期状態はブラウザのみ（他のコンピュータアクセスはIntune管理でブロック）で、<em>あなたの</em>IDを使って認可された範囲のみにアクセスし、すべてのアクションが監査ログに記録されます。 |
| **計画とメタ認知 (Lessons 07 & 09)** | Opalは最初に作業計画を生成し、各ステップごとに自己推論を監督し、疑わしい動作を検知すると一時停止します。 |
| **再利用可能な機能・ツール (Lesson 04)** | <strong>スキル</strong>機能を使うと、繰り返しの仕事の指示を記述し（`.md`ファイルでインポートまたはOpalで編集）、会話間で再利用できます。 |

> **利用状況:** Project Opal は現在 [Frontier早期アクセスプログラム](https://adoption.microsoft.com/copilot/frontier-program/) に参加しているMicrosoft 365 Copilotのユーザーが利用可能であり、管理者による設定が必要です。実験的な Frontier 機能のため、機能は随時変更される可能性があります。
