## 💻 コードの実行

実装全体は `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` にあります。このファイルは旅行計画向けの<strong>ファンアウト/ファンイン同時ワークフロー</strong>を示しています：

### 🏗️ <strong>ワークフローアーキテクチャ</strong>

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**主な構成要素：**

1. **ConcurrentStartExecutor**：ユーザーのリクエストを全エージェントに同時送信
2. **Researcher Agent**：旅先や観光地を同時に分析
3. **Planner Agent**：詳細な旅行計画を同時に作成
4. **ConcurrentAggregationExecutor**：両エージェントの結果を収集・統合

### 🎯 **ファンアウト/ファンインパターン**

このワークフローは古典的な<strong>ファンアウト/ファンイン</strong>パターンを示しています：
- <strong>ファンアウト</strong>：1つの入力メッセージを複数エージェントに同時送信
- <strong>同時処理</strong>：複数のエージェントが同じタスクを並列処理
- <strong>ファンイン</strong>：すべてのエージェントの結果を集めて単一の出力に統合

### 🔍 カスタムエグゼキュータの理解

**ConcurrentStartExecutor:**
- `IMessageHandler<string>`を実装し文字列入力を受け入れる
- 接続されているすべてのエージェントにメッセージをブロードキャスト
- 同時処理を開始するため`TurnToken`を送信

**ConcurrentAggregationExecutor:**
- `IMessageHandler<ChatMessage>`を実装しエージェント応答を受信
- スレッドセーフな方法でメッセージを収集
- すべての応答が揃うと集約
- `context.YieldOutputAsync()`で最終出力を返す

### ⚡ パフォーマンスの利点

**同時実行 vs 逐次実行：**
- 逐次：Agent1（30秒）→ Agent2（30秒）= **合計60秒**
- 同時：Agent1（30秒）|| Agent2（30秒）= **合計30秒**

<strong>スループット向上</strong>：N個の同時エージェントで最大N倍高速化（負荷とリソースによる）

### 🛡️ エラーハンドリング

ワークフローは個別エージェントの障害を優雅に処理：
- 1つのエージェントが失敗しても他は処理を継続
- 集約器はタイムアウトロジックを実装可能
- 必要に応じて部分結果を返却可能
