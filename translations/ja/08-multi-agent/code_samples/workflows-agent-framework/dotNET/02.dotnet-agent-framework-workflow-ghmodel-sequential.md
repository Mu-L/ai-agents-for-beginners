# ⏩ Azure OpenAI（Responses API）を使った連続エージェントワークフロー（.NET）

## 📋 高度な連続処理チュートリアル

## 💻 コードの実行

完全な実装は `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs` にあります。このファイルは<strong>3ステージの家具分析ワークフロー</strong>を示しています：

1. **ステージ1 - 販売エージェント**：家具画像を分析し購入提案を行う
2. **ステージ2 - 価格エージェント**：詳細な価格内訳と予算オプションを提供
3. **ステージ3 - 見積りエージェント**：Markdown形式の専門的な見積り文書を生成

### 🏗️ <strong>ワークフローアーキテクチャ</strong>

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

各エージェントは：
- 前のステージの出力をコンテキストとして受け取る
- 専門知識を生かして前の分析に基づき処理を構築する
- 状態管理によってワークフローの継続性を保つ

### 🔍 状態フローの理解

シーケンス内の各エージェントは以下を受け取ります：
- <strong>元の入力</strong>：初期ユーザーメッセージ（画像＋テキスト）
- <strong>前エージェントの出力</strong>：会話履歴中のすべての前エージェントの応答
- <strong>蓄積されたコンテキスト</strong>：ワークフロー全体を通じて維持される完全な状態

これにより、各エージェントがこれまでのすべてのステージの包括的なコンテキストに基づき高度な多段階処理を行えます。
