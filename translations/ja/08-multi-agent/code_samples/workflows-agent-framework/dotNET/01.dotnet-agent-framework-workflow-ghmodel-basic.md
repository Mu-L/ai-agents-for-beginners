# 🔄 Azure OpenAI（Responses API）を用いた基本的なエージェントワークフロー（.NET）

## 📋 ワークフローオーケストレーションチュートリアル

このノートブックでは、Microsoft Agent Framework for .NET と Azure OpenAI（Responses API）を使用して高度な<strong>エージェントワークフロー</strong>を構築する方法を示します。AIエージェントが協力して複雑なタスクを達成する多段階のビジネスプロセスを、構造化されたオーケストレーションパターンで作成する方法を学びます。

## 🎯 学習目標

### 🏗️ <strong>ワークフローアーキテクチャの基礎</strong>
- **Workflow Builder**: 複雑な複数ステップのAIプロセスを設計・オーケストレーションする
- **Agent Coordination**: ワークフロー内で複数の専門エージェントを調整する
- **Azure OpenAI（Responses API）**: ワークフローでAzure OpenAI Responses APIを活用する
- **Visual Workflow Design**: ワークフロー構造を作成・可視化し理解を深める

### 🔄 <strong>プロセスオーケストレーションパターン</strong>
- **Sequential Processing**: 複数のエージェントタスクを論理的な順序で連鎖させる
- **State Management**: ワークフロー段階間でコンテキストとデータフローを維持する
- **Error Handling**: 堅牢なエラー回復とワークフローレジリエンスを実装する
- **Performance Optimization**: 企業規模の運用向けに効率的なワークフローを設計する

### 🏢 <strong>エンタープライズワークフローアプリケーション</strong>
- **Business Process Automation**: 複雑な組織のワークフローを自動化する
- **Content Production Pipeline**: レビューと承認段階を含む編集ワークフロー
- **Customer Service Automation**: 複数手順の顧客問い合わせ対応を自動化する
- **Data Processing Workflows**: AI駆動の変換を含むETLワークフロー

## ⚙️ 前提条件とセットアップ

### 📦 **必須NuGetパッケージ**

このワークフロードemonstrationは以下の主要な.NETパッケージを使用します：

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI設定**

**環境設定（.envファイル）：**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAIアクセス方法：**
1. AzureポータルでAzure OpenAIリソースを作成する
2. モデル（例：`gpt-4.1-mini`）をデプロイし、デプロイ名を確認する
3. `az login`でサインインし、上記のように環境変数を設定する

### 🏗️ <strong>ワークフローアーキテクチャ概要</strong>

```mermaid
graph TD
    A[ワークフロービルダー] --> B[エージェントレジストリ]
    B --> C[ワークフロー実行エンジン]
    C --> D[Agent 1: コンテンツ生成者
    C --> E[Agent 2: コンテンツレビュアー 
    D --> F[ワークフロー結果]
    E --> F
    G[Azure OpenAI（レスポンスAPI）] --> D
    G --> E
```

**主なコンポーネント：**
- **WorkflowBuilder**: ワークフロー設計のためのメインオーケストレーションエンジン
- **AIAgent**: 特定の能力を持つ専門エージェント
- **Azure OpenAI クライアント**: Azure OpenAI Responses APIとの統合
- **Execution Context**: ワークフロー段階間の状態とデータフローを管理

## 💻 コードの実行

完全な実装は `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` にあります。このファイルでは以下を示しています：

1. <strong>環境設定</strong> - `.env`ファイルからAzure OpenAI構成を読み込む
2. **Azure OpenAIクライアントセットアップ** - Azure OpenAI Responses APIを使うクライアントを構成する
3. <strong>エージェント作成</strong> - 専門エージェント（フロントデスクとコンシェルジュ）を定義する
4. <strong>ワークフロービルダー</strong> - 複数エージェントによる逐次処理ワークフローを作成する
5. <strong>ワークフロー実行</strong> - ストリーミング結果付きでワークフローを実行する

### 🚀 実例の実行

```bash
# スクリプトを実行可能にする（Unix/Linux/macOS）
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# ワークフローを実行する
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Windowsの場合：
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 期待される出力

ワークフローは以下を行います：
1. あなたの旅行先リクエスト（「パリに行きたい」）を受け付ける
2. フロントデスクエージェントが初期推奨を提供する
3. コンシェルジュエージェントが推奨内容を見直し、改良する
4. 完全な会話ストリームが最終出力として表示される

### 🔧 カスタマイズ

ワークフローは以下のようにカスタマイズできます：
- エージェントの指示を変更して動作を変える
- 複数のエージェントを追加して複雑な多段階ワークフローを作成する
- ユーザーメッセージを変更して異なるシナリオをテストする
- ワークフローのエッジを調整して異なる実行パターンを作る
