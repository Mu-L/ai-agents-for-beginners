# 初心者のためのAIエージェント - 学習ガイド

このガイドは、コースを進める際の実用的な参考書として使ってください。これは
レッスンの代わりになるものではありません。どこから始めるか、各レッスンで何を
頼りにするか、そしてアイデアをつなげて小さな動作するエージェントデモにする
方法を考える助けになります。

ここが初めてなら、シンプルに始めましょう：

1. [Course Setup](./00-course-setup/README.md) を読みます。
2. レッスン01～06を順番に完了します。
3. 学びながら、一つの小さなデモアイデアを心に留めておきます。
4. 各レッスンの後で「これまでできなかったことが、今エージェントは何ができる
   ようになったか？」と自問します。

## 覚えておくと良いシンプルなデモ

エージェントを学ぶ良い方法は、コースを通じて一つのデモアイデアを追うことです。

例のデモ：<strong>コースヘルパーエージェント</strong>。

ユーザーの依頼：

> 「エージェントがツールを使う方法を学びたい。適切なレッスンを見つけて、
> > まず読むべきものを要約し、簡単な練習課題を与えてほしい。」

通常のチャットボットは既知の情報から答えられます。エージェントはさらに


1. <strong>コースファイルを読み取ったり検索して</strong> 適切なレッスンを探します。
2. <strong>ツールを使って</strong> レッスンリンク、例、または補助資料を取得します。
3. <strong>計画を立てて</strong> 長い答えではなく短い学習経路を示します。
4. <strong>現在の会話文脈を使って</strong> 学習者の目標に集中し続けます。
5. <strong>メモリを使って</strong> 必要なら便利な好みを覚えます。
6. **トレース、引用、ログを表示して** ユーザーが何が起きたか理解できるようにします。
7. <strong>ガードレールを適用して</strong> リスクのある行動や機微なデータを扱う前に対策をします。

各レッスンを学んだら、このデモに戻り、このレッスンはどんな新しい機能を
追加するか考えてみてください。

## 目指すもの

コースの終わりには、以下の要素を組み合わせたエージェントシステムを説明し
作成できるようになるはずです：

| 部品 | 一般的な意味 | デモでの役割 |
|------|------------------------|-------------|
| モデル | ユーザーの要求を解釈する推論エンジン | 学習者がツール利用のレッスンを求めていることを理解する |
| ツール | エージェントが使える関数、API、ファイル、ブラウザやサービス | リポジトリを検索したりレッスン内容を取得する |
| 知識 | 答えの根拠となる文書やデータ | コースのREADMEファイルやレッスン資料 |
| コンテキスト | 次のモデル呼び出しに含まれる情報 | ユーザーの目標やツールの結果 |
| メモリ | 後で使うために保存される情報 | 学習者がハンズオンのPython例を好むこと |
| 計画 | 大きな目標を小さなステップに分けること | レッスンを見つけ要約し練習を提案する |
| オーケストレーション | ツール、ステップ、エージェント間の作業分配 | プランナーが検索ツールを呼び出し、その後要約者を呼ぶ |
| 信頼 | 安全性、セキュリティ、評価、可観測性 | ツール呼び出しをログに記録し重要な行動は確認する |

## モデルとプロバイダー

コースのコード例は<strong>Microsoft Agent Framework (MAF)</strong>を使い、推奨APIの<strong>Azure OpenAI Responses API</strong>を対象にしています。このAPIはチャット補完、ツール呼び出し、マルチモーダル入力、状態を持つ会話を1つのAPIで実現します。接続は<strong>Microsoft Foundry</strong>プロジェクト（`FoundryChatClient`使用）からか、Azure OpenAIへ直接（`OpenAIChatClient`使用）行います。

レッスンを進める中で、いくつかのプロバイダーオプションがあります：

- **Microsoft Foundry / Azure OpenAI (Responses API)** — レッスン全体で使われる主要ルート。`az login`でキー不要のEntra ID認証が可能です。
- **Foundry Local** — OpenAI互換APIを通じてデバイス上で完全にモデルを実行（クラウド不要、APIキー不要）。オフラインやコストなし実験に最適。[Course Setup](./00-course-setup/README.md)を参照。
- **MiniMax** — 大きなコンテキストモデルが使えるOpenAI互換プロバイダーで、代替として利用可能。

> **注意：** GitHub Modelsは廃止予定（2026年7月引退）でResponses APIをサポートしていません。サンプルはAzure OpenAI / Microsoft Foundryへ更新されています。

## 初心者にやさしい重要なアイデア

### ツール

ツールとは、エージェントがモデルの外で作業を実行するために呼び出せるものです。良いツールは
明確な名前、狭い役割、型付き入力、予測可能な出力、安全な失敗方法を備えています。


コースヘルパーデモでのツール例は次のようなものです：

- `search_lessons(query)`
- `read_lesson(path)`
- `create_practice_task(topic)`

### RAGと知識

RAGは、推測するのではなくソース資料からエージェントが回答するのを助けます。この
コースでは、そのソース資料としてレッスンのREADME、コードサンプル、またはレッスンからリンクされた外部
リソースが使えます。

回答を文書、データ、または現在の
プロジェクトファイルに基づかせたいときにRAGを使用します。

### 計画

計画は、リクエストに複数のステップがある場合に有用です。計画は短く保ち、
開発者やユーザーが検査できる程度に見えるようにします。

デモの計画例は次のようなものです：

1. ツール使用に関連するレッスンを見つける。
2. 最も関連性の高いレッスンを要約する。
3. 一つの練習タスクを推奨する。

### コンテキスト

コンテキストとはモデルが現在見ている情報です。コンテキストが少なすぎると、
エージェントが重要な詳細を見逃すことがあります。多すぎると、
エージェントが遅くなったりコストが増えたり、


良いコンテキスト設計とは、次に呼ぶモデルに対して適切な情報を選ぶことです。

### メモリ

メモリとは後で使うために保存した情報です。すべて保存してはいけません。有用で安全かつ更新や削除が
簡単な情報のみを保存します。

例えば、「学習者はPythonの例を好む」という記憶は有用かもしれません。
個人の機微な情報は通常保存しません。

### 評価と可観測性

評価は：エージェントは正しいことをしたか？と問います。

可観測性は：どのように起きたかを見ることができるか？と問います。

本番エージェントでは、モデル呼び出し、ツール呼び出し、取得したコンテキスト、
レイテンシー、コスト、失敗、ユーザーフィードバックを追跡します。

### 信頼性とセキュリティ

信頼できるエージェントは有用なプロンプトだけでは不十分です。最小権限のツールを使い、
影響の大きい行動は人間の承認を得て、必要に応じてデータの編集を行い、
監査が必要な行動にはログやレシートを残します。

## 15分間の復習ルーティン

各レッスンの後でこのルーティンを使います：

1. **レッスンを一文で要約する。**
2. **新しいエージェント機能を名付ける。** 例えば、ツール使用、検索、
   計画、メモリ、可観測性、またはセキュリティ。
3. **コースヘルパーデモに追加する。** デモは今どう変わるか？
4. **リスクを探す。** この機能が誤用されたら何が起こるか？
5. **テスト問題を1つ作る。** エージェントが正しく振る舞っているかどうかどうチェックする？

## クイックセルフチェック

先に進む前に、これらの質問に答えてみてください：

1. エージェントは普通のチャットボット自身ができない何をできるか？
2. 最初にどのツールがエージェントに必要ですか？なぜですか？
3. エージェントの回答を支えるべき知識源は何ですか？
4. 次のモデル呼び出しに含めるべきコンテキストは何ですか？

5. エージェントは何を覚えておくべきで、何を保存しないようにすべきですか？
6. エージェントはいつ人間の承認を求めるべきですか？
7. 後でエージェントをデバッグまたは監査するのに役立つログ、トレース、またはレシートは何ですか？


## 推奨キャップストーン演習

コースの最後に、学習者がこの
リポジトリをナビゲートするのを助ける小さなエージェントを作成します。

最低限のバージョン:

- ユーザーからトピックを受け取る。
- 最も関連性の高いレッスンを見つける。
- 最初に読むべき内容を要約する。
- ひとつの実践課題を提案する。
- 使用されたレッスンファイルやリンクを表示する。

拡張バージョン:

- 学習者の好みのプログラミング言語を記憶する。
- 回答前に簡単な計画を立てる。
- 最終回答の前にセルフチェックのステップを追加する。
- ツール呼び出しと取得した情報源を記録する。
- ブラウザー起動やUI自動化タスクの前に確認を求める。

これにより、ツール、RAG、計画、
コンテキスト、メモリ、可観測性、信頼を一つのプロジェクトで小規模ながら実践的に練習できます。
