# ⚡ Azure OpenAI (Responses API) संग समवर्ती Agent Workflowहरू (.NET)

## 📋 उच्च-प्रदर्शन समानान्तर प्रशोधन ट्युटोरियल

यो नोटबुकले Microsoft Agent Framework for .NET र Azure OpenAI (Responses API) को प्रयोग गरेर **समवर्ती workflow ढाँचा** प्रदर्शन गर्दछ। तपाईंले उच्च-प्रदर्शन, समानान्तर प्रशोधन workflowहरू कसरी निर्माण गर्ने भन्ने सिक्नुहुनेछ जुन अधिकतम थ्रूपुट सुनिश्चित गर्न बहु AI एजेन्टहरू एकसाथ चलाउँछन् जबकि समन्वय र डाटा स्थिरता कायम राख्छन्।

## 🎯 सिकाइ उद्देश्यहरू

### 🚀 **समवर्ती प्रशोधन आधारभूत कुरा**
- **समानान्तर Agent कार्यान्वयन**: अधिकतम प्रदर्शनका लागि बहु AI एजेन्टहरू एकैचोटी चलाउनुहोस्
- **Async/Await ढाँचाहरू**: कुशल समवर्तीका लागि .NET को async प्रोग्रामिङ मोडेलको उपयोग गर्नुहोस्
- **Azure OpenAI (Responses API)**: Azure OpenAI Responses API लाई बहु समवर्ती कलहरू समन्वय गर्नुहोस्
- **स्रोत व्यवस्थापन**: समवर्ती अपरेसनहरूमा AI मोडेल स्रोतहरू कुशलता साथ व्यवस्थापन गर्नुहोस्

### 🏗️ **उन्नत समवर्ती वास्तुकला**
- **कार्य आधारित समानान्तरता**: उत्तम समवर्ती कार्यान्वयनका लागि .NET Task Parallel Library प्रयोग गर्नुहोस्
- **समन्वय ढाँचाहरू**: दुर्घटना अवस्थाबाट बच्दै समवर्ती एजेन्टहरू समन्वय गर्नुहोस्
- **लोड सन्तुलन**: उपलब्ध समवर्ती प्रशोधन क्षमता अनुसार कामलाई कुशलता पूर्वक बाँड्नुहोस्
- **गल्ती सहनशक्ति**: सम्पूर्ण workflow रोक्न नदिई व्यक्तिगत एजेन्ट विफलताहरू व्यवस्थापन गर्नुहोस्

### 🏢 **उद्यम समवर्ती अनुप्रयोगहरू**
- **उच्च मात्रा कागजात प्रशोधन**: बहु कागजातहरू एकैचोटी प्रशोधन गर्नुहोस्
- **रियल-टाइम समग्री विश्लेषण**: आउँदै गरेको डाटा स्ट्रिमहरूमा समवर्ती विश्लेषण
- **ब्याच प्रशोधन अनुकूलन**: ठूलो परिमाण डाटा प्रशोधन कार्यहरूको लागि थ्रूपुट अधिकतम गर्नुहोस्
- **बहु-मोडल विश्लेषण**: विभिन्न समग्री प्रकार र ढाँचाहरूको समानान्तर प्रशोधन

```

```

```

```

```
Single Input → Multiple Concurrent Processors → Result Aggregation → Single Output
```

## 💻 कोड चलाउने तरिका

पूर्ण कार्यान्वयन `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` मा उपलब्ध छ। यस फाइलले यात्रा योजना बनाउने **Fan-Out/Fan-In समवर्ती workflow** प्रदर्शन गर्दछ:

### 🏗️ **Workflow वास्तुकला**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**मुख्य कम्पोनेंटहरू:**

1. **ConcurrentStartExecutor**: प्रयोगकर्ताको अनुरोध सबै एजेन्टहरूमा एकैचोटी प्रसारण गर्दछ
2. **Researcher Agent**: गन्तव्यहरू र आकर्षणहरू समवर्ती विश्लेषण गर्दछ
3. **Planner Agent**: विस्तृत यात्रा योजनाहरू समवर्ती बनाएर कार्य गर्दछ
4. **ConcurrentAggregationExecutor**: दुबै एजेन्टबाट परिणाम सङ्कलन र मर्ज गर्दछ

### 🎯 **Fan-Out/Fan-In ढाँचा**

यो workflow ले क्लासिक **Fan-Out/Fan-In** ढाँचालाई प्रदर्शन गर्दछ:
- **Fan-Out**: एउटा इनपुट सन्देशलाई बहु एजेन्टहरूलाई एकैचोटी प्रसारण
- **समवर्ती प्रशोधन**: बहु एजेन्टहरू समान काममा समानान्तर कार्य गर्दछ
- **Fan-In**: सबै एजेन्टहरूबाट प्राप्त परिणामहरू सङ्कलन गरी एकल आउटपुटमा समेटिन्छ

### 🔧 अनुकूलन विकल्पहरू

**अझ धेरै समवर्ती एजेन्टहरू थप्नुहोस्:**
```csharp

    .AddFanInEdge(aggregationExecutor, sources: [researcherAgent, plannerAgent, budgetAgent])
    .WithOutputFrom(aggregationExecutor)
    .Build();

```

### 🎯 वास्तविक-विश्व अनुप्रयोगहरू

यो समवर्ती ढाँचा उपयुक्त छ:
- **सामग्री सिर्जना**: बहु लेखकहरूले विभिन्न सेरो समानान्तर लेखन गर्दैछन्
- **कोड समीक्षा**: विभिन्न दृष्टिकोणबाट बहु समीक्षकहरूले कोड विश्लेषण गर्दै
- **बजार अनुसन्धान**: फरक बजार खण्डहरूको समानान्तर विश्लेषण
- **कागजात प्रशोधन**: समवर्ती रूपमा डाटा निकासी, विश्लेषण, र प्रमाणीकरण
- **बहु-दृष्टिकोण विश्लेषण**: एउटै इनपुटमा विविध दृष्टिकोणहरू प्राप्त गर्नुहोस्

### 🔍 अनुकूलन Executer हरू बुझ्नुहोस्

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` कार्यान्वयन गर्द्छ स्ट्रिंग इनपुट स्वीकार गर्न
- सबै जडित एजेन्टहरूलाई सन्देश प्रसारण गर्दछ
- समवर्ती प्रशोधन सुरु गर्न `TurnToken` पठाउँछ

**ConcurrentAggregationExecutor:**
- `IMessageHandler<ChatMessage>` कार्यान्वयन गर्दछ एजेन्ट प्रतिक्रियाहरू प्राप्त गर्न
- थ्रेड-सेफ तरिकाले सन्देशहरू सङ्कलन गर्दछ
- सबै अपेक्षित प्रतिक्रियाहरू आएसँगै सङ्कलन गर्दछ
- `context.YieldOutputAsync()` प्रयोग गरी अन्तिम आउटपुट दिन्छ

### ⚡ प्रदर्शन लाभहरू

**समवर्ती र अनुक्रमानुसार:**
- अनुक्रमानुसार: Agent1 (३० सेकेन्ड) → Agent2 (३० सेकेन्ड) = **कुल ६० सेकेन्ड**
- समवर्ती: Agent1 (३० सेकेन्ड) || Agent2 (३० सेकेन्ड) = **कुल ३० सेकेन्ड**

**थ्रूपुट सुधार**: N समवर्ती एजेन्टहरूका लागि N गुणा छिटो (कार्यभार र स्रोतहरूमा निर्भर)
