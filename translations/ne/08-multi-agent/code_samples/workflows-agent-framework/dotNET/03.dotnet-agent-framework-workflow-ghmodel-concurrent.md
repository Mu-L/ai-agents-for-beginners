# ⚡ Azure OpenAI (Responses API) सँग समकक्ष एजेन्ट वर्कफ्लोहरू (.NET)

## 📋 उच्च-प्रदर्शन समानान्तर प्रशोधन ट्यूटोरियल

यो नोटबुकले Microsoft Agent Framework for .NET र Azure OpenAI (Responses API) प्रयोग गरेर **समकक्ष कार्यप्रवाह ढाँचाहरू** देखाउँछ। तपाईं कसरी धेरै AI एजेन्टहरूलाई एकैसाथ चलाएर अधिकतम प्रसारणदर प्राप्त गर्ने, समन्वय र डाटा स्थिरता कायम राखेर उच्च-प्रदर्शन, समानान्तर प्रशोधन कार्यप्रवाहहरू निर्माण गर्ने सिक्नुहुनेछ।

## 🎯 सिकाइ उद्देश्यहरू

### 🚀 **समकक्ष प्रशोधन आधारहरू**
- **समानान्तर एजेन्ट कार्यान्वयन**: अधिकतम प्रदर्शनका लागि धेरै AI एजेन्टहरू एकसाथ चलाउनुहोस्
- **Async/Await ढाँचाहरू**: दक्ष समकक्षताका लागि .NET को async प्रोग्रामिङ मोडेलको उपयोग गर्नुहोस्
- **Azure OpenAI (Responses API)**: Azure OpenAI Responses API का धेरै समकक्ष कलहरू समन्वय गर्नुहोस्
- **स्रोत व्यवस्थापन**: समकक्ष कार्यहरुमा AI मोडल स्रोतहरू कुशलतापूर्वक व्यवस्थापन गर्नुहोस्

### 🏗️ **उन्नत समकक्षता वास्तुकला**
- **टास्क-आधारित समानान्तरता**: उत्तम समकक्ष कार्यान्वयनका लागि .NET Task Parallel Library प्रयोग गर्नुहोस्
- **समन्वयन ढाँचाहरू**: प्रतिस्पर्धा अवस्थाहरूबाट बच्दै समकक्ष एजेन्टहरू समन्वय गर्नुहोस्
- **लोड सन्तुलन**: उपलब्ध समकक्ष प्रशोधन क्षमता भरि कार्य कुशलतापूर्वक वितरण गर्नुहोस्
- **फेल्ट सहिष्णुता**: सम्पूर्ण कार्यप्रवाह नहटाई व्यक्तिगत एजेन्ट विफलताहरू व्यवस्थापन गर्नुहोस्

### 🏢 **उद्यम समकक्ष अनुप्रयोगहरू**
- **उच्च-पात्र दस्तावेज प्रशोधन**: धेरै दस्तावेजहरू एकसाथ प्रशोधन गर्नुहोस्
- **रियल-टाइम सामग्री विश्लेषण**: आउने डाटा स्ट्रिमहरूको समकक्ष विश्लेषण गर्नुहोस्
- **ब्याच प्रशोधन अनुकूलन**: ठूलो-स्तर डाटा प्रशोधन कार्यहरूमा अधिकतम throughput हासिल गर्नुहोस्
- **मल्टि-मोडल विश्लेषण**: फरक सामग्री प्रकार र ढाँचाहरूको समानान्तर प्रशोधन

- **स्रोत उपयोगीकरण**: उपलब्ध AI मोडल क्षमताको अधिकतम दक्षता

- **इलास्टिक स्केलिङ**: कार्यभारको आधारमा समकछ एजेन्ट संख्या गतिशील रूपमा समायोजन गर्नुहोस्

- **त्रुटि पृथकीकरण**: व्यक्तिगत एजेन्ट विफलताहरूले अन्य समकछ कार्यहरू असर गर्दैनन्

- **त्रुटि पुनःप्राप्ति**: असफल समकछ कार्यहरूको लागि स्वतः पुन: प्रयास यन्त्रहरू
- **लोड वितरण**: उपलब्ध एजेन्टहरूमा काम समान रूपमा वितरण गर्नुहोस्

### 📊 **प्रदर्शन अनुगमन**
- **समकछ कार्यान्वयन मेट्रिक्स**: सबै समानान्तर कार्यहरूको प्रदर्शन ट्र्याक गर्नुहोस्
- **स्रोत उपयोग विश्लेषण**: CPU, मेमोरी, र नेटवर्क उपयोग अनुगमन गर्नुहोस्
- **थ्रूपुट विश्लेषण**: समकछ प्रशोधनबाट दक्षता प्राप्ति मापन गर्नुहोस्
- **बोटलनेक पहिचान**: प्रदर्शन सीमाबद्धताहरू पहिचान र समाधान गर्नुहोस्

## 💻 कोड चलाउँदै

पूर्ण कार्यान्वयन `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` मा उपलब्ध छ। यो फाइलले यात्रा योजना बनाउनका लागि **Fan-Out/Fan-In समकछ कार्यप्रवाह** देखाउँछ:

### 🏗️ **वर्कफ्लो वास्तुकला**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**प्रमुख कम्पोनेन्टहरू:**

1. **ConcurrentStartExecutor**: सबै एजेन्टलाई एकैचोटी प्रयोगकर्ता अनुरोध प्रसारण गर्छ
2. **Researcher Agent**: गन्तव्यहरू र आकर्षणहरूको समकछ विश्लेषण गर्छ
3. **Planner Agent**: समकछ रूपमा विस्तृत यात्रा योजना बनाउँछ
4. **ConcurrentAggregationExecutor**: दुबै एजेन्टहरूबाट परिणामहरू सङ्कलन र मर्ज गर्छ

### 🎯 **Fan-Out/Fan-In ढाँचा**

यो वर्कफ्लोले क्लासिक **Fan-Out/Fan-In** ढाँचा देखाउँछ:
- **Fan-Out**: एउटै इनपुट सन्देश धेरै एजेन्टहरूलाई एकैचोटि प्रसारण गरिन्छ
- **समकछ प्रशोधन**: धेरै एजेन्टहरू एउटै कार्यमा समानान्तर रूपमा काम गर्छन्
- **Fan-In**: सबै एजेन्टका परिणामहरू सङ्कलन गरी एउटै आउटपुटमा एकत्रित गरिन्छ

### 🔧 अनुकूलन विकल्पहरू

**थप समकछ एजेन्टहरू थप्नुहोस्:**
```csharp
// Create additional specialized agents
AIAgent budgetAgent = azureClient.GetOpenAIResponseClient(deployment).CreateAIAgent(
    name: "Budget-Agent", instructions: "Calculate travel costs...");

// Add to fan-out

    .AddFanOutEdge(startExecutor, targets: [researcherAgent, plannerAgent, budgetAgent])
    .AddFanInEdge(aggregationExecutor, sources: [researcherAgent, plannerAgent, budgetAgent])
    .WithOutputFrom(aggregationExecutor)
    .Build();

// Update aggregation count

```

### 🔍 अनुकूलित एक्जिक्युटरहरू बुझ्दै

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` कार्यान्वयन गरेर स्ट्रिङ इनपुट स्वीकार्छ
- जडित सबै एजेन्टहरूमा सन्देश प्रसारण गर्छ
- समकछ प्रशोधन शुरू गर्न `TurnToken` पठाउँछ

**ConcurrentAggregationExecutor:**
- `IMessageHandler<ChatMessage>` कार्यान्वयन गरेर एजेन्ट प्रतिक्रिया प्राप्त गर्छ
- सन्देशहरू थ्रेड-सेफ तरिकाले सङ्कलन गर्छ
- सबै अपेक्षित प्रतिक्रियाहरू आइपुग्दा एकत्रित गर्छ
- `context.YieldOutputAsync()` प्रयोग गरी अन्तिम आउटपुट दिन्छ

### ⚡ प्रदर्शन लाभहरू

**समकछ र अनुक्रमिक तुलना:**
