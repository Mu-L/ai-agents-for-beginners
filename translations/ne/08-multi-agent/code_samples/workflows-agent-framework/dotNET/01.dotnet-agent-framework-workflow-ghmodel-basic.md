# 🔄 Azure OpenAI (Responses API) सँग आधारभूत एजेन्ट कार्यप्रवाहहरू (.NET)

## 📋 कार्यप्रवाह समन्वयन ट्यूटोरियल

यो नोटबुकले कसरी Microsoft Agent Framework for .NET र Azure OpenAI (Responses API) प्रयोग गरेर परिष्कृत **एजेन्ट कार्यप्रवाहहरू** बनाउन सकिन्छ भन्ने देखाउँछ। तपाईंले बहु-चरणिय व्यावसायिक प्रक्रियाहरू सिर्जना गर्न सिक्नुहुनेछ जहाँ AI एजेन्टहरूले संरचित समन्वयन ढाँचाहरू मार्फत जटिल कार्यहरू पूरा गर्न मिलेर काम गर्छन्।

## 🎯 सिकाईका उद्देश्यहरू

### 🏗️ **कार्यप्रवाह वास्तुकला आधारभूत कुरा**
- **Workflow Builder**: जटिल बहु-चरण AI प्रक्रियाहरू डिजाइन र समन्वय गर्नुहोस्
- **Agent Coordination**: कार्यप्रवाह भित्र धेरै विशेष एजेन्टहरू समन्वय गर्नुहोस्
- **Azure OpenAI (Responses API)**: कार्यप्रवाहहरूमा Azure OpenAI Responses API प्रयोग गर्नुहोस्
- **Visual Workflow Design**: राम्रो बुझाइका लागि कार्यप्रवाह संरचनाहरू सिर्जना र देखाउनुहोस्

### 🔄 **प्रक्रिया समन्वयन ढाँचाहरू**
- **Sequential Processing**: धेरै एजेन्ट कार्यहरू तार्किक क्रममा श्रृंखला गर्नुहोस्
- **State Management**: कार्यप्रवाह चरणहरूभर सन्दर्भ र डाटा प्रवाह कायम गर्नुहोस्
- **Error Handling**: बलियो त्रुटि सुधार र कार्यप्रवाह लचकता लागू गर्नुहोस्
- **Performance Optimization**: उद्यम-स्तर अपरेशनहरूका लागि प्रभावकारी कार्यप्रवाह डिजाइन गर्नुहोस्

### 🏢 **उद्यम कार्यप्रवाह अनुप्रयोगहरू**
- **Business Process Automation**: जटिल संगठनात्मक कार्यप्रवाहहरू स्वचालित गर्नुहोस्
- **Content Production Pipeline**: समीक्षा र अनुमोदन चरणहरू सहित सम्पादकीय कार्यप्रवाहहरू
- **Customer Service Automation**: बहु-चरण ग्राहक प्रश्न समाधान
- **Data Processing Workflows**: AI-संचालित रूपान्तरण सहित ETL कार्यप्रवाहहरू

## ⚙️ पूर्वशर्तहरू र सेटअप

### 📦 **आवश्यक NuGet प्याकेजहरू**

यस कार्यप्रवाह प्रदर्शनले केही प्रमुख .NET प्याकेजहरू प्रयोग गर्दछ:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI कन्फिगरेसन**

**परिबेश सेटअप (.env फाइल):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI पहुँच:**
1. Azure पोर्टलमा Azure OpenAI स्रोत सिर्जना गर्नुहोस्
2. मोडल तैनाथ गर्नुहोस् (उदाहरणका लागि, `gpt-4.1-mini`) र तैनाथी नाम नोट गर्नुहोस्
3. `az login` गरेर लगइन गर्नुहोस् र माथि देखाइएको अनुसार वातावरण चरहरू कन्फिगर गर्नुहोस्

### 🏗️ **कार्यप्रवाह वास्तुकला अवलोकन**

```mermaid
graph TD
    A[कार्यप्रवाह निर्माता] --> B[एजेन्ट दर्ता]
    B --> C[कार्यप्रवाह कार्यान्वयन इन्जिन]
    C --> D[Agent 1: सामग्री उत्पन्नकर्ता]
    C --> E[Agent 2: सामग्री समीक्षक] 
    D --> F[कार्यप्रवाह नतिजाहरू]
    E --> F
    G[Azure OpenAI (प्रतिक्रिया API)] --> D
    G --> E
```

**प्रमुख अंशहरू:**
- **WorkflowBuilder**: कार्यप्रवाहहरू डिजाइन गर्ने मुख्य समन्वयन इन्जिन
- **AIAgent**: विशिष्ट क्षमता भएका व्यक्तिगत विशेषज्ञ एजेन्टहरू
- **Azure OpenAI Client**: Azure OpenAI Responses API एकीकरण
- **Execution Context**: कार्यप्रवाह चरणहरू बीच अवस्था र डाटा प्रवाह व्यवस्थापन गर्नुहोस्

## 🎨 **उद्यम कार्यप्रवाह डिजाइन ढाँचाहरू**

### 📝 **सामग्री उत्पादन कार्यप्रवाह**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **कागजात प्रशोधन पाइपलाइन**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **व्यापार बुद्धिमत्ता कार्यप्रवाह**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **ग्राहक सेवा स्वचालन**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **उद्यम लाभहरू**

### 🎯 **विश्वसनीयता र स्केलबिलिटी**
- **Deterministic Execution**: सुसंगत, पुनरावृत्त कार्यप्रवाह परिणामहरू
- **Error Recovery**: कार्यप्रवाहको कुनै पनि चरणमा असफलता को सहज नियन्त्रण
- **Performance Monitoring**: कार्यसम्पादन मेट्रिक्स र अनुकूलन अवसरहरूको ट्र्याकिंग
- **Resource Management**: AI मोडल स्रोतहरूको प्रभावकारी बाँडफाँट र प्रयोग

### 🔒 **सुरक्षा र अनुपालन**
- **Secure Authentication**: Microsoft Entra ID प्रमाणीकरण `az login` (AzureCliCredential) मार्फत
- **Audit Trails**: कार्यप्रवाह कार्यान्वयन र निर्णय बिन्दुहरूको पूर्ण लगिङ
- **Access Control**: कार्यप्रवाह कार्यान्वयन र अनुगमनका लागि सूक्ष्म अनुमतिहरू
- **Data Privacy**: कार्यप्रवाहहरू भरि संवेदनशील सूचनाहरूको सुरक्षित ह्यान्डलिंग

### 📊 **परीक्षण र व्यवस्थापन**
- **Visual Workflow Design**: प्रक्रिया प्रवाह र निर्भरताहरूको स्पष्ट प्रतिनिधित्व
- **Execution Monitoring**: कार्यप्रवाह प्रगति र प्रदर्शनको वास्तविक-समय ट्र्याकिंग
- **Error Reporting**: विस्तृत त्रुटि विश्लेषण र डिबगिंग क्षमता
- **Performance Analytics**: अनुकूलन र क्षमताका योजना बनाउन मेट्रिक्सहरू

आउनुहोस्, तपाईंको पहिलो उद्यम-तय AI कार्यप्रवाह बनाऔं! 🚀

## 💻 कोड चलाउने तरिका

पूर्ण कार्यान्वयन `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` मा उपलब्ध छ। यस फाइलले देखाउँछ:

1. **Environment Configuration** - `.env` फाइलबाट Azure OpenAI कन्फिगरेसन लोड गर्दै
2. **Azure OpenAI Client Setup** - Azure OpenAI Responses API प्रयोग गर्न क्लाइन्ट कन्फिगर गर्दै
3. **Agent Creation** - विशेष एजेन्टहरू (Front Desk र Concierge) परिभाषित गर्दै
4. **Workflow Builder** - क्रमागत प्रोसेसिङसँग बहु-एजेन्ट कार्यप्रवाह निर्माण गर्दै
5. **Workflow Execution** - स्ट्रिमिङ परिणामहरूसँग कार्यप्रवाह चलाउँदै
