# 🔄 Azure OpenAI (Responses API) संग आधारभूत एजेन्ट कार्यप्रवाहहरू (.NET)

## 📋 कार्यप्रवाह संयोजन तालिम

यो नोटबुकले माइक्रोसफ्ट एजेन्ट फ्रेमवर्क फॉर .NET र Azure OpenAI (Responses API) प्रयोग गरी जटिल **एजेन्ट कार्यप्रवाहहरू** कसरी बनाउने देखाउँछ। तपाईंले बहु-चरण व्यावसायिक प्रक्रियाहरू सिर्जना गर्ने सिक्नुहुनेछ जहाँ AI एजेन्टहरूले जटिल कार्यहरू संरचित संयोजन ढाँचाहरू मार्फत मिलेर सम्पन्न गर्छन्।

## 🎯 सिकाइका उद्देश्यहरू

### 🏗️ **कार्यप्रवाह वास्तुकला आधारहरू**
- **Workflow Builder**: जटिल बहु-चरण AI प्रक्रियाहरू डिजाइन गर्ने र संयोजन गर्ने
- **Agent Coordination**: कार्यप्रवाह भित्र धेरै विशिष्ट एजेन्टहरू समन्वय गर्ने
- **Azure OpenAI (Responses API)**: कार्यप्रवाहहरूमा Azure OpenAI Responses API को उपयोग गर्ने
- **Visual Workflow Design**: राम्रो बुझ्नका लागि कार्यप्रवाह संरचनाहरू सिर्जना र दृश्यात्मक बनाउने

### 🔄 **प्रक्रिया संयोजन ढाँचाहरू**
- **Sequential Processing**: बहु एजेन्ट कार्यहरू तार्किक क्रममा श्रृंखला बनाउने
- **State Management**: कार्यप्रवाह चरणहरू बीच सन्दर्भ र डेटा प्रवाह कायम राख्ने
- **Error Handling**: मजबुत त्रुटि पुनर्प्राप्ति र कार्यप्रवाह सहनशीलता कार्यान्वयन गर्ने
- **Performance Optimization**: उद्यम स्तरका अपरेशनहरूका लागि कुशल कार्यप्रवाहहरू डिजाइन गर्ने

### 🏢 **उद्यम कार्यप्रवाह अनुप्रयोगहरू**
- **Business Process Automation**: जटिल संगठनात्मक कार्यप्रवाहहरू स्वचालित गर्ने
- **Content Production Pipeline**: समीक्षा र अनुमोदन चरणहरूले सम्पादकीय कार्यप्रवाहहरू
- **Customer Service Automation**: बहु-चरण ग्राहक सोधपुछ समाधान
- **Data Processing Workflows**: AI-शक्ति संचालित रूपान्तरणसहित ETL कार्यप्रवाहहरू

## ⚙️ पूर्वआवश्यकता र सेटअप

### 📦 **आवश्यक NuGet प्याकेजहरू**

यो कार्यप्रवाह प्रदर्शनमा केही प्रमुख .NET प्याकेजहरू प्रयोग गरिएको छ:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI सेटअप**

**Environment Setup (.env फाइल):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Azure OpenAI पहुँच:**
1. Azure पोर्टलमा Azure OpenAI स्रोत सिर्जना गर्नुहोस्
2. एउटा मोडेल परिनियोजन गर्नुहोस् (जस्तै, `gpt-5-mini`) र परिनियोजन नाम नोट गर्नुहोस्
3. `az login` गरि साइन इन गर्नुहोस् र माथि देखाइएको जस्तो वातावरण चरहरू कन्फिगर गर्नुहोस्

### 🏗️ **कार्यप्रवाह वास्तुकला अवलोकन**

```mermaid
graph TD
    A[कार्यप्रवाह निर्माता] --> B[एजेन्ट रजिस्ट्री]
    B --> C[कार्यप्रवाह कार्यान्वयन इन्जिन]
    C --> D[Agent 1: सामग्री निर्माता]
    C --> E[Agent 2: सामग्री समीक्षक] 
    D --> F[कार्यप्रवाह परिणामहरू]
    E --> F
    G[Azure OpenAI (प्रतिक्रिया API)] --> D
    G --> E
```

**प्रमुख कम्पोनेन्टहरू:**
- **WorkflowBuilder**: कार्यप्रवाह डिज़ाइनको लागि मुख्य संयोजन इन्जिन
- **AIAgent**: विशेष क्षमताहरू भएका व्यक्तिगत एजेन्टहरू
- **Azure OpenAI Client**: Azure OpenAI Responses API एकीकरण
- **Execution Context**: कार्यप्रवाह चरणहरू बीच स्थिति र डेटा प्रवाह व्यवस्थापन

## 🎨 **उद्यम कार्यप्रवाह डिजाइन ढाँचाहरू**

### 📝 **सामग्री उत्पादन कार्यप्रवाह**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **कागजात प्रक्रिया पाइपलाइन**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **व्यवसाय बुद्धिमत्ता कार्यप्रवाह**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **ग्राहक सेवा स्वचालन**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **उद्यम लाभहरू**

### 🎯 **विश्वसनीयता र स्केलेबिलिटी**
- **Deterministic Execution**: सुसंगत, दोहोरिने कार्यप्रवाह नतिजाहरू
- **Error Recovery**: कार्यप्रवाहको कुनै पनि चरणमा असफलताका लागि सजिलो व्यवस्थापन
- **Performance Monitoring**: कार्यान्वयन मेट्रिक्स र सुधार अवसरहरू ट्र्याक गर्ने
- **Resource Management**: AI मोडेल स्रोतहरूको कुशल आवंटन र उपयोग

### 🔒 **सुरक्षा र अनुपालन**
- **Secure Authentication**: `az login` (AzureCliCredential) मार्फत Microsoft Entra ID प्रमाणीकरण
- **Audit Trails**: कार्यप्रवाह कार्यान्वयन र निर्णय बिन्दुहरूको पूर्ण लगिङ
- **Access Control**: कार्यप्रवाह कार्यान्वयन र निगरानीका लागि व्यावहारिक अनुमति
- **Data Privacy**: कार्यप्रवाहभर संवेदनशील जानकारीको सुरक्षित व्यवस्थापन

### 📊 **अवलोकन र व्यवस्थापन**
- **Visual Workflow Design**: प्रक्रिया प्रवाह र निर्भरता स्पष्ट प्रतिनिधित्व
- **Execution Monitoring**: कार्यप्रवाह प्रगति र प्रदर्शनको वास्तविक-समय ट्र्याकिंग
- **Error Reporting**: विस्तृत त्रुटि विश्लेषण र डिबगिङ क्षमताहरू
- **Performance Analytics**: सुधार र क्षमता योजना को मेट्रिक्सहरू

तपाईंको पहिलो उद्यम-तयार AI कार्यप्रवाह बनाऔं! 🚀

## 💻 कोड चलाउने तरिका

पूर्ण कार्यान्वयन `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` मा उपलब्ध छ। यस फाइलले देखाउँछ:

1. **Environment Configuration** - `.env` फाइलबाट Azure OpenAI कन्फिगुरेसन लोड गर्ने
2. **Azure OpenAI Client Setup** - Azure OpenAI Responses API प्रयोगका लागि क्लाइन्ट सेटअप गर्ने
3. **Agent Creation** - विशिष्ट एजेन्टहरू परिभाषित गर्ने (Front Desk र Concierge)
4. **Workflow Builder** - क्रमशः प्रक्रिया गर्ने बहु-एजेन्ट कार्यप्रवाह सिर्जना गर्ने
5. **Workflow Execution** - स्ट्रिमिङ परिणामहरूसहित कार्यप्रवाह चलाउने
