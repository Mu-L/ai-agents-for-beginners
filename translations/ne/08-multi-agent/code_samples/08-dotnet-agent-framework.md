# 🤝 उद्यम बहु-एजेन्ट कार्यप्रवाह प्रणालीहरू (.NET)

## 📋 सिकाइ उद्देश्यहरू

यो नोटबुकले कसरी माइक्रोसफ्ट एजेन्ट फ्रेमवर्कलाई प्रयोग गरेर .NET मा Azure OpenAI (Responses API) सँग बहु-विशेषज्ञता एजेन्ट प्रणालीहरू निर्माण गर्ने देखाउँछ। तपाईंले संरचित कार्यप्रवाहहरू मार्फत सँगै काम गर्ने धेरै विशेषज्ञ एजेन्टहरूलाई सुरक्षित अनुरूप बनाउने तरिका सिक्नुहुनेछ, उत्पादन-तयार समाधानहरूको लागि .NET का उद्यम सुविधाहरूको प्रयोग गरेर।

**तपाईंले निर्माण गर्ने उद्यम बहु-एजेन्ट क्षमता:**
- 👥 **एजेन्ट सहकार्य**: टाइप-सुरक्षित एजेन्ट समन्वय कम्पाइल-समय प्रमाणीकरणसहित
- 🔄 **कार्यप्रवाह संयोजन**: .NET को asynchronous ढाँचाहरूको साथ घोषणात्मक कार्यप्रवाह परिभाषा
- 🎭 **भूमिका विशेषज्ञता**: बलियो रूपमा टाइप गरिएका एजेन्ट व्यक्तित्व र विशेषज्ञता क्षेत्रहरू
- 🏢 **उद्यम एकीकरण**: अनुगमन र त्रुटि व्यवस्थापनसहित उत्पादन-तयार ढाँचाहरू


## यस नमुनाले के देखाउँछ

यस बहु-एजेन्ट कार्यप्रवाह प्रणालीले दुई विशेषज्ञ एजेन्टहरूसहित होटल यात्रा सिफारिस सेवा सिर्जना गर्दछ:

1. **FrontDesk Agent**: यात्रा एजेन्ट जसले क्रियाकलाप र स्थान सिफारिसहरू प्रदान गर्दछ
2. **Concierge Agent**: सिफारिसहरूको समीक्षा गरेर प्रामाणिक, गैर-पर्यटक अनुभव सुनिश्चित गर्दछ

एजेन्टहरू एक कार्यप्रवाहमा सँगै काम गर्छन् जहाँ:
- FrontDesk एजेन्टले प्रारम्भिक यात्रा अनुरोध प्राप्त गर्छ
- Concierge एजेन्टले सिफारिसहरूको समीक्षा र सुधार गर्दछ
- कार्यप्रवाहले वास्तविक-समयमा प्रतिक्रियाहरू स्ट्रिम गर्छ

## मुख्य अवधारणाहरू

### एजेन्ट समन्वय
नमुनाले माइक्रोसफ्ट एजेन्ट फ्रेमवर्क प्रयोग गरेर टाइप-सुरक्षित एजेन्ट समन्वय कम्पाइल-समय प्रमाणीकरणसहित देखाउँछ।

### कार्यप्रवाह संयोजन
.NET को asynchronous ढाँचाहरूसँग घोषणात्मक कार्यप्रवाह परिभाषा प्रयोग गरी धेरै एजेन्टहरूलाई एक पाइपलाइनमा जडान गर्दछ।

### स्ट्रिमिङ प्रतिक्रियाहरू
asynchronous enumerables र event-driven वास्तुकलाको प्रयोग गरी एजेन्ट प्रतिक्रियाहरूको वास्तविक-समय स्ट्रिमिङ लागू गर्दछ।

### उद्यम एकीकरण
उत्पादन-तयार ढाँचाहरू देखाउँछ जसमा समावेश छन्:
- वातावरण चर कन्फिगरेसन
- सुरक्षित प्रमाणपत्र व्यवस्थापन
- त्रुटि व्यवस्थापन
- असिंक्रोनस इभेन्ट प्रक्रिया
