# 🔄 Základní pracovní postupy agentů s Azure OpenAI (Responses API) (.NET)

## 📋 Návod na orchestraci pracovních postupů

Tento notebook ukazuje, jak vytvořit sofistikované **pracovní postupy agentů** pomocí Microsoft Agent Framework pro .NET a Azure OpenAI (Responses API). Naučíte se vytvářet vícestupňové obchodní procesy, kde AI agenti spolupracují, aby dosáhli složitých úkolů prostřednictvím strukturovaných vzorců orchestrací.

- **Azure OpenAI (Responses API)**: Využití Azure OpenAI Responses API v pracovních postupech

- **Zacházení s chybami**: Implementace robustního zotavení z chyb a odolnosti pracovních postupů

### 🔑 **Konfigurace Azure OpenAI**

**Nastavení prostředí (.env soubor):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Přístup k Azure OpenAI:**
1. Vytvořte zdroj Azure OpenAI v portálu Azure
2. Nasadte model (například `gpt-5-mini`) a poznamenejte si název nasazení
3. Přihlaste se pomocí `az login` a nastavte proměnné prostředí podle výše uvedeného

## 💻 Spuštění kódu

Kompletní implementace je k dispozici v `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Tento soubor ukazuje:

1. **Konfigurace prostředí** - Načtení konfigurace Azure OpenAI ze souboru `.env`
2. **Nastavení Azure OpenAI klienta** - Konfigurace klienta pro použití Azure OpenAI Responses API
3. **Vytvoření agentů** - Definování specializovaných agentů (Front Desk a Concierge)
4. **Tvůrce pracovního postupu** - Vytvoření vícero agentového pracovního postupu s sekvenčním zpracováním
5. **Spuštění pracovního postupu** - Spuštění pracovního postupu se streamováním výsledků

### 🚀 Spuštění příkladu

```bash
# Nastavte skript jako spustitelný (Unix/Linux/macOS)

# Spusťte pracovní postup
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

```powershell

```

### 📝 Očekávaný výstup

Pracovní postup:
1. Přijme váš požadavek na cestovní destinaci ("Chtěl bych jet do Paříže")
2. Agent Front Desk poskytne počáteční doporučení
3. Agent Concierge doporučení přezkoumá a upřesní
4. Konečný výstup zobrazí kompletní konverzační tok
