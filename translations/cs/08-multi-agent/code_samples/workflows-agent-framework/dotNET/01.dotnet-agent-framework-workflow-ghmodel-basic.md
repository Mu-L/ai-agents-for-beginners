# 🔄 Základní pracovní postupy agentů s Azure OpenAI (Responses API) (.NET)

## 📋 Tutoriál orchestrací pracovních postupů

Tento poznámkový blok ukazuje, jak vytvářet sofistikované **pracovní postupy agentů** pomocí Microsoft Agent Framework pro .NET a Azure OpenAI (Responses API). Naučíte se vytvářet vícefázové obchodní procesy, kde AI agenti spolupracují na dosažení složitých úkolů prostřednictvím strukturovaných orchestrací.

## 🎯 Výukové cíle

### 🏗️ **Základy architektury pracovních postupů**
- **Workflow Builder**: Navrhování a orchestraci složitých vícefázových AI procesů
- **Koordinace agentů**: Koordinace několika specializovaných agentů v pracovních postupech
- **Azure OpenAI (Responses API)**: Využití Azure OpenAI Responses API v pracovních postupech
- **Visual Workflow Design**: Vytváření a vizualizace struktury pracovních postupů pro lepší pochopení

### 🔄 **Vzory orchestrací procesů**
- **Sekvenční zpracování**: Řetězení více úkolů agentů v logickém pořadí
- **Správa stavu**: Udržování kontextu a toku dat mezi kroky pracovního postupu
- **Zpracování chyb**: Implementace robustního zotavení z chyb a odolnosti pracovního postupu
- **Optimalizace výkonu**: Návrh efektivních pracovních postupů pro podnikové operace

### 🏢 **Podnikové aplikace pracovních postupů**

### 🔑 **Konfigurace Azure OpenAI**

**Nastavení prostředí (.env soubor):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Přístup k Azure OpenAI:**
1. Vytvořte zdroj Azure OpenAI v Azure portálu
2. Nasadíte model (například `gpt-4.1-mini`) a zaznamenejte si název nasazení
3. Přihlaste se pomocí `az login` a nastavte proměnné prostředí podle výše uvedeného návodu

- **Azure OpenAI klient**: Integrace Azure OpenAI Responses API
- **Execution Context**: Spravuje stav a tok dat mezi fázemi pracovního postupu

## 💻 Spuštění kódu

Kompletní implementace je k dispozici v souboru `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Tento soubor demonstruje:

1. **Konfigurace prostředí** - Načítání nastavení Azure OpenAI ze souboru `.env`
2. **Nastavení Azure OpenAI klienta** - Konfigurace klienta pro použití Azure OpenAI Responses API
3. **Vytvoření agentů** - Definice specializovaných agentů (Front Desk a Concierge)
4. **Workflow Builder** - Vytvoření multi-agentního pracovního postupu se sekvenčním zpracováním
5. **Spuštění pracovního postupu** - Spuštění pracovního postupu se streamováním výsledků

### 🚀 Spuštění příkladu

```bash
# Umožněte skriptu spustitelnost (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Spusťte pracovní postup
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Nebo na Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Očekávaný výstup

Pracovní postup:
1. Přijme váš požadavek na cestovní destinaci („Chtěl bych jet do Paříže“)
2. Agent Front Desk poskytne počáteční doporučení
3. Agent Concierge převezme doporučení k revizi a vylepšení
4. Výsledkem je kompletní stream konverzace zobrazený uživateli

### 🔧 Přizpůsobení

Můžete upravit pracovní postup:
- Úpravou instrukcí agentů ke změně jejich chování

- Změnou zprávy od uživatele pro testování různých scénářů
- Úpravou hran pracovního postupu pro vytvoření odlišných vzorů spuštění
