# ⏩ Sekvenční pracovní postupy agentů s Azure OpenAI (Responses API) (.NET)

## 📋 Pokročilý tutoriál sekvenčního zpracování

Tento zápisník demonstruje **vzor sekvenčních pracovních postupů** pomocí Microsoft Agent Framework pro .NET a Azure OpenAI (Responses API). Naučíte se, jak vytvořit složité, krok za krokem zpracovatelské pipeline, kde agenti vykonávají úkoly v konkrétním pořadí, přičemž každá fáze staví na výsledcích předchozí fáze.

## 🎯 Cíle učení

### 🔄 **Architektura sekvenčního zpracování**
- **Lineární návrh workflow**: Vytváření krokových zpracovatelských pipeline s jasnými závislostmi
- **Správa stavu**: Uchovávání kontextu a datového toku napříč sekvenčními fázemi workflow
- **Azure OpenAI (Responses API)**: Využití Azure OpenAI modelů v vícefázových .NET pracovních postupech
- **Podnikoví vzory pipeline**: Stavba produkčně připravených sekvenčních zpracovatelských systémů

### 🏗️ **Pokročilé sekvenční vzory**
- **Stage-Gate zpracování**: Implementace validačních kontrolních bodů mezi fázemi workflow
- **Zachování kontextu**: Uchovávání stavu a akumulovaných znalostí napříč všemi fázemi
- **Propagace chyb**: Elegantní zpracování selhání v sekvenčních řetězcích zpracování
- **Optimalizace výkonu**: Efektivní sekvenční vykonávání s minimální režii

### 🏢 **Podnikové sekvenční aplikace**
- **Pipeline zpracování dokumentů**: Vícefázová analýza, transformace a validace dokumentů
- **Workflow kontroly kvality**: Sekvenční revize, validace a schvalovací procesy
- **Pipeline produkce obsahu**: Výzkum → Psaní → Editace → Revize → Publikace
- **Automatizace obchodních procesů**: Vícekrokové obchodní workflow s jasnými závislostmi fází

### 🏗️ **Architektura sekvenčního workflow**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Agent pro analýzu]
    B --> C[Kontrolní bod 1]
    C --> D[Stage 2: Agent pro zpracování]
    D --> E[Kontrolní bod 2]
    E --> F[Stage 3: Agent pro ověřování]
    F --> G[Konečný výstup]
    
    H[Kontext stavu] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Responses API)] --> B
    I --> D
    I --> F
```

**Klíčové komponenty:**
- **Sekvenční agenti**: Specializovaní agenti pro každou fázi zpracování
- **Kontext stavu**: Uchovává akumulovaná data a rozhodnutí napříč fázemi
- **Kontrolní body**: Validační body mezi fázemi pro zajištění kvality a konzistence
- **Azure OpenAI klient**: Konzistentní přístup k AI modelu napříč všemi fázemi workflow

## 💻 Spuštění kódu

Kompletní implementace je k dispozici v souboru `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Tento soubor demonstruje **třífázový workflow analýzy nábytku**:

1. **Fáze 1 - Prodejní agent**: Analyzuje obrázky nábytku a poskytuje návrhy k nákupu
2. **Fáze 2 - Cenový agent**: Poskytuje podrobné rozklady cen a možnosti rozpočtu
3. **Fáze 3 - Agent pro nabídku**: Generuje profesionální nabídku v Markdown formátu

### 🏗️ **Architektura workflow**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Každý agent:
- Přijímá výstup z předchozí fáze jako kontext
- Staví na předchozí analýze s specializovanou expertízou
- Udržuje kontinuitu workflow pomocí správy stavu

### 🚀 Spuštění příkladu

**Požadavky:**
- Umístěte obrázek nábytku na `../imgs/home.png` (nebo aktualizujte proměnnou `imgPath`)
- Nakonfigurujte svůj `.env` soubor s koncovým bodem Azure OpenAI a nasazením, poté se přihlaste pomocí `az login`

```bash
# Udělejte skript spustitelným (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Spusťte sekvenční pracovní postup
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Nebo na Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Očekávaný výstup

Workflow provede:
1. **Prodejní agent**: Identifikuje položky nábytku na obrázku a poskytne doporučení
2. **Cenový agent**: Přidá podrobnou cenovou analýzu s rozpočtovými úrovněmi a doporučeními nákupu
3. **Agent pro nabídku**: Vygeneruje formátovaný dokument nabídky se všemi informacemi sloučenými dohromady

Konečným výstupem bude komplexní, profesionální nabídka na nábytek založená na analýze obrázku.
