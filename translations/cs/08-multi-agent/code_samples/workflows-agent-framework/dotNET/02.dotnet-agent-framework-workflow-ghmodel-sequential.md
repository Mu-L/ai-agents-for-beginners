# ⏩ Sekvenční pracovní postupy agentů s Azure OpenAI (Responses API) (.NET)

## 📋 Pokročilý tutoriál sekvenčního zpracování

Tento notebook demonstruje **vzory sekvenčních pracovních postupů** s využitím Microsoft Agent Framework pro .NET a Azure OpenAI (Responses API). Naučíte se, jak vytvářet sofistikované postupné zpracovatelské pipeline, ve kterých agenti vykonávají jednotlivé kroky v konkrétním pořadí, přičemž každá fáze navazuje na výsledky té předchozí.

- **Azure OpenAI (Responses API)**: Využití modelů Azure OpenAI v vícefázových .NET pracovních postupech
- **Vzorové pipeline pro podniky**: Vytváření produkčně připravených systémů sekvenčního zpracování

### 🏗️ **Pokročilé sekvenční vzory**
- **Stage-Gate zpracování**: Implementace validačních kontrolních bodů mezi fázemi pracovního postupu

- **Propagace chyb**: Elegantní zpracování selhání v řetězcích sekvenčního zpracování
- **Optimalizace výkonu**: Efektivní sekvenční spouštění s minimální režijní zátěží

### 🏗️ **Architektura sekvenčního pracovního postupu**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Analytický agent]
    B --> C[Kontrolní bod 1]
    C --> D[Stage 2: Zpracovatelský agent]
    D --> E[Kontrolní bod 2]
    E --> F[Stage 3: Validující agent]
    F --> G[Konečný výstup]
    
    H[Kontext stavu] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Responses API)] --> B
    I --> D
    I --> F
```

**Klíčové komponenty:**
- **Sekvenční agenti**: Specializovaní agenti pro každou fázi zpracování
- **Kontext stavu**: Udržuje kumulovaná data a rozhodnutí napříč fázemi
- **Kontrolní body**: Validační body mezi fázemi pro zajištění kvality a konzistence
- **Azure OpenAI klient**: Konzistentní přístup k AI modelům ve všech fázích pracovního postupu

## 💻 Spuštění kódu

Kompletní implementace je dostupná v souboru `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Tento soubor demonstruje **třístupňový pracovní postup analýzy nábytku**:

1. **Fáze 1 - Prodejní agent**: Analyzuje obrázky nábytku a poskytuje doporučení ke koupi
2. **Fáze 2 - Cenový agent**: Poskytuje podrobné rozklady cen a možnosti rozpočtu
3. **Fáze 3 - Agent pro nabídku**: Generuje profesionální nabídkový dokument v Markdown formátu

### 🏗️ **Architektura pracovního postupu**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Každý agent:
- Přijímá výstup z předchozí fáze jako kontext
- Navazuje na předchozí analýzy specializovanou expertízou
- Udržuje kontinuitu pracovního postupu správou stavu

### 🚀 Spuštění příkladu

**Požadavky:**
- Umístěte obrázek nábytku na `../imgs/home.png` (nebo upravte proměnnou `imgPath`)
- Nakonfigurujte svůj `.env` soubor s endpointem a nasazením Azure OpenAI, poté se přihlaste pomocí `az login`

```bash
# Nastavte skript jako spustitelný (Unix/Linux/macOS)

./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Očekávaný výstup

Pracovní postup:
1. **Prodejní agent**: Identifikuje položky nábytku na obrázku a poskytne doporučení
2. **Cenový agent**: Přidá detailní cenovou analýzu s rozdělením rozpočtu a doporučením nákupu
3. **Agent pro nabídku**: Vygeneruje formátovaný nabídkový dokument se syntetizovanými informacemi

Konečný výstup bude komplexní profesionální nabídka na nábytek založená na analýze obrázku.
