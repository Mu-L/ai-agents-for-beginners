# ⏩ Alur Kerja Agen Berurutan dengan Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial Pemrosesan Berurutan Lanjutan

Buku catatan ini menunjukkan **pola alur kerja berurutan** menggunakan Microsoft Agent Framework untuk .NET dan Azure OpenAI (Responses API). Anda akan belajar cara membangun jalur pemrosesan langkah demi langkah yang canggih di mana agen menjalankan tugas dalam urutan tertentu, dengan setiap tahap membangun hasil dari tahap sebelumnya.

## 🎯 Tujuan Pembelajaran

### 🔄 **Arsitektur Pemrosesan Berurutan**
- **Desain Alur Kerja Linear**: Membuat jalur pemrosesan langkah demi langkah dengan ketergantungan yang jelas
- **Manajemen Status**: Mempertahankan konteks dan aliran data di seluruh tahap alur kerja berurutan
- **Azure OpenAI (Responses API)**: Memanfaatkan model Azure OpenAI dalam alur kerja .NET multi-tahap
- **Pola Jalur Perusahaan**: Membangun sistem pemrosesan berurutan siap produksi

### 🏗️ **Pola Berurutan Lanjutan**
- **Pemrosesan Stage-Gate**: Menerapkan titik pemeriksaan validasi antar tahap alur kerja
- **Preservasi Konteks**: Mempertahankan status dan pengetahuan yang terakumulasi di seluruh tahap
- **Propagasi Kesalahan**: Menangani kegagalan dengan baik dalam rantai pemrosesan berurutan
- **Optimasi Performa**: Eksekusi berurutan yang efisien dengan overhead minimal

### 🏢 **Aplikasi Berurutan Perusahaan**
- **Jalur Pemrosesan Dokumen**: Analisis, transformasi, dan validasi dokumen multi-tahap
- **Alur Kerja Penjaminan Mutu**: Proses tinjauan, validasi, dan persetujuan berurutan
- **Jalur Produksi Konten**: Riset → Penulisan → Pengeditan → Tinjauan → Penerbitan
- **Otomasi Proses Bisnis**: Alur kerja bisnis multi-langkah dengan ketergantungan tahap yang jelas

```env

AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Manajemen Konfigurasi:**

## 💻 Menjalankan Kode

Implementasi lengkap tersedia dalam `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. File ini menunjukkan **alur kerja analisis furnitur tiga tahap**:

1. **Tahap 1 - Agen Penjualan**: Menganalisis gambar furnitur dan memberikan saran pembelian
2. **Tahap 2 - Agen Harga**: Menyediakan rincian harga dan opsi anggaran
3. **Tahap 3 - Agen Penawaran**: Membuat dokumen penawaran profesional dalam format Markdown

### 🏗️ **Arsitektur Alur Kerja**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Setiap agen:
- Menerima output dari tahap sebelumnya sebagai konteks
- Membangun analisis sebelumnya dengan keahlian khusus
- Mempertahankan kontinuitas alur kerja melalui manajemen status

### 🚀 Menjalankan Contoh

**Prasyarat:**
- Tempatkan gambar furnitur di `../imgs/home.png` (atau perbarui variabel `imgPath`)
- Konfigurasikan file `.env` Anda dengan endpoint dan deployment Azure OpenAI, lalu masuk dengan `az login`

```bash
# Buat skrip dapat dieksekusi (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Jalankan alur kerja berurutan
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Atau di Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Output yang Diharapkan

Alur kerja akan:
1. **Agen Penjualan**: Mengidentifikasi barang furnitur dari gambar dan memberikan rekomendasi
2. **Agen Harga**: Menambahkan analisis harga detail dengan tingkatan anggaran dan rekomendasi berbelanja
3. **Agen Penawaran**: Membuat dokumen penawaran yang terformat dengan semua informasi yang diolah

Output akhir akan menjadi penawaran furnitur profesional yang komprehensif berdasarkan analisis gambar.

### 🔍 Memahami Aliran Status

Setiap agen dalam urutan menerima:
- **Input Asli**: Pesan pengguna awal (gambar + teks)
- **Output Agen Sebelumnya**: Semua respons agen sebelumnya dalam riwayat percakapan
- **Konteks Terakumulasi**: Status lengkap yang dipertahankan sepanjang alur kerja

Ini memungkinkan pemrosesan multi-tahap yang canggih di mana setiap agen membangun berdasarkan konteks komprehensif dari semua tahap sebelumnya.
