# ⚡ Alur Kerja Agen Konkuren dengan Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial Pemrosesan Paralel Berkinerja Tinggi

Notebook ini menunjukkan **pola alur kerja konkuren** menggunakan Microsoft Agent Framework untuk .NET dan Azure OpenAI (Responses API). Anda akan belajar cara membangun alur kerja pemrosesan paralel berkinerja tinggi yang memaksimalkan throughput dengan menjalankan beberapa agen AI secara bersamaan sambil menjaga koordinasi dan konsistensi data.

## 🎯 Tujuan Pembelajaran

### 🚀 **Dasar-Dasar Pemrosesan Konkuren**
- **Eksekusi Agen Paralel**: Jalankan beberapa agen AI secara bersamaan untuk kinerja maksimal
- **Pola Async/Await**: Manfaatkan model pemrograman async .NET untuk konkuren yang efisien
- **Azure OpenAI (Responses API)**: Koordinasikan banyak panggilan konkuren ke Azure OpenAI Responses API
- **Manajemen Sumber Daya**: Kelola sumber daya model AI secara efisien pada operasi konkuren

### 🏗️ **Arsitektur Konkurensi Lanjutan**
- **Task-Based Parallelism**: Gunakan Task Parallel Library .NET untuk eksekusi konkuren optimal
- **Pola Sinkronisasi**: Koordinasikan agen konkuren sambil menghindari kondisi balapan
- **Load Balancing**: Distribusikan pekerjaan secara efisien di kapasitas pemrosesan konkuren yang tersedia
- **Fault Tolerance**: Tangani kegagalan agen individu tanpa menghentikan seluruh alur kerja

### 🏢 **Aplikasi Konkuren di Perusahaan**
- **Pemrosesan Dokumen Volume Tinggi**: Proses banyak dokumen secara bersamaan
- **Analisis Konten Waktu Nyata**: Analisis data masuk secara konkuren
- **Optimasi Pemrosesan Batch**: Maksimalkan throughput untuk operasi pemrosesan data skala besar
- **Analisis Multi-Modal**: Pemrosesan paralel berbagai tipe dan format konten

## 💻 Menjalankan Kode

Implementasi lengkap tersedia di `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Berkas ini menunjukkan **alur kerja konkuren Fan-Out/Fan-In** untuk perencanaan perjalanan:

### 🏗️ **Arsitektur Alur Kerja**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Komponen Kunci:**

1. **ConcurrentStartExecutor**: Menyiarkan permintaan pengguna ke semua agen secara bersamaan
2. **Researcher Agent**: Menganalisis destinasi dan atraksi secara konkuren
3. **Planner Agent**: Membuat rencana perjalanan detail secara konkuren
4. **ConcurrentAggregationExecutor**: Mengumpulkan dan menggabungkan hasil dari kedua agen

### 🎯 **Pola Fan-Out/Fan-In**

Alur kerja ini menunjukkan pola klasik **Fan-Out/Fan-In**:
- **Fan-Out**: Satu pesan input disiarkan ke banyak agen sekaligus
- **Pemrosesan Konkuren**: Banyak agen bekerja paralel pada tugas yang sama
- **Fan-In**: Hasil dari semua agen dikumpulkan dan digabung menjadi output tunggal

Alur kerja akan:
1. **Broadcast Permintaan**: Kirim "Rencanakan perjalanan ke Seattle pada bulan Desember" ke kedua agen

### 🔍 Memahami Custom Executors

**ConcurrentStartExecutor:**
- Mengimplementasikan `IMessageHandler<string>` untuk menerima input string
- Menyiarkan pesan ke semua agen yang terhubung
- Mengirim `TurnToken` untuk memicu pemrosesan konkuren

**ConcurrentAggregationExecutor:**
- Mengimplementasikan `IMessageHandler<ChatMessage>` untuk menerima respons agen
- Mengumpulkan pesan dengan cara thread-safe
- Mengagregasi saat semua respons yang diharapkan tiba
- Menghasilkan output akhir menggunakan `context.YieldOutputAsync()`

### ⚡ Manfaat Kinerja

**Konkuren vs Berurutan:**
- Berurutan: Agent1 (30s) → Agent2 (30s) = **60 detik total**
- Konkuren: Agent1 (30s) || Agent2 (30s) = **30 detik total**

**Peningkatan Throughput**: Sampai N× lebih cepat untuk N agen konkuren (tergantung beban kerja dan sumber daya)

### 🛡️ Penanganan Kesalahan

Alur kerja menangani kegagalan agen individu dengan elegan:
- Jika satu agen gagal, yang lain tetap melanjutkan pemrosesan
- Aggregator dapat menerapkan logika timeout
- Hasil parsial dapat dikembalikan jika diperlukan
