# ⚡ Alur Kerja Agen Konkuren dengan Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial Pemrosesan Paralel Berkinerja Tinggi

Notebook ini menunjukkan **pola alur kerja konkuren** menggunakan Microsoft Agent Framework untuk .NET dan Azure OpenAI (Responses API). Anda akan belajar cara membangun alur kerja pemrosesan paralel berkinerja tinggi yang memaksimalkan throughput dengan menjalankan beberapa agen AI secara bersamaan sambil menjaga koordinasi dan konsistensi data.

## 🎯 Tujuan Pembelajaran

### 🚀 **Dasar-dasar Pemrosesan Konkuren**
- **Eksekusi Agen Paralel**: Jalankan beberapa agen AI secara bersamaan untuk kinerja maksimal
- **Pola Async/Await**: Manfaatkan model pemrograman async .NET untuk konkuren yang efisien
- **Azure OpenAI (Responses API)**: Koordinasi beberapa panggilan konkuren ke Azure OpenAI Responses API
- **Manajemen Sumber Daya**: Kelola sumber daya model AI secara efisien pada operasi konkuren

### 🏗️ **Arsitektur Konkuren Tingkat Lanjut**
- **Paralelisme Berbasis Task**: Gunakan .NET Task Parallel Library untuk eksekusi konkuren optimal
- **Pola Sinkronisasi**: Koordinasi agen konkuren sambil menghindari kondisi balapan
- **Load Balancing**: Distribusikan pekerjaan secara efisien pada kapasitas pemrosesan konkuren tersedia
- **Ketahanan Kesalahan**: Tangani kegagalan agen individu tanpa menghentikan seluruh alur kerja

### 🏢 **Aplikasi Konkuren Perusahaan**
- **Pemrosesan Dokumen Volume Tinggi**: Proses banyak dokumen secara bersamaan
- **Analisis Konten Waktu Nyata**: Analisis konkuren aliran data masuk
- **Optimasi Pemrosesan Batch**: Maksimalkan throughput untuk operasi pemrosesan data skala besar
- **Analisis Multi-Modal**: Pemrosesan paralel berbagai jenis dan format konten

## 💻 Menjalankan Kode

Implementasi lengkap tersedia di `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. File ini memperlihatkan **alur kerja Fan-Out/Fan-In konkuren** untuk perencanaan perjalanan:

### 🏗️ **Arsitektur Alur Kerja**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Komponen Kunci:**

1. **ConcurrentStartExecutor**: Menyiarkan permintaan pengguna ke semua agen secara bersamaan
2. **Agen Peneliti**: Menganalisis destinasi dan atraksi secara konkuren
3. **Agen Perencana**: Membuat rencana perjalanan rinci secara konkuren
4. **ConcurrentAggregationExecutor**: Mengumpulkan dan menggabungkan hasil dari kedua agen

### 🎯 **Pola Fan-Out/Fan-In**

Alur kerja ini memperlihatkan pola klasik **Fan-Out/Fan-In**:
- **Fan-Out**: Satu pesan input disiarkan ke banyak agen secara bersamaan
- **Pemrosesan Konkuren**: Beberapa agen bekerja paralel pada tugas yang sama
- **Fan-In**: Hasil dari semua agen dikumpulkan dan digabungkan menjadi output tunggal

### 🔍 Memahami Executor Kustom

**ConcurrentStartExecutor:**
- Mengimplementasikan `IMessageHandler<string>` untuk menerima input string
- Menyiarkan pesan ke semua agen terhubung
- Mengirim `TurnToken` untuk memicu pemrosesan konkuren

**ConcurrentAggregationExecutor:**
- Mengimplementasikan `IMessageHandler<ChatMessage>` untuk menerima respons agen
- Mengumpulkan pesan dengan cara thread-safe
- Menggabungkan ketika semua respons yang diharapkan tiba
- Menghasilkan output akhir menggunakan `context.YieldOutputAsync()`

### ⚡ Manfaat Kinerja

**Konkuren vs Sequential:**
- Sequential: Agen1 (30s) → Agen2 (30s) = **60 detik total**
- Konkuren: Agen1 (30s) || Agen2 (30s) = **30 detik total**

**Peningkatan throughput**: Hingga N× lebih cepat untuk N agen konkuren (tergantung beban kerja dan sumber daya)

### 🛡️ Penanganan Kesalahan

Alur kerja menangani kegagalan agen individu dengan mulus:
- Jika satu agen gagal, agen lain terus memproses
- Aggregator dapat menerapkan logika timeout
- Hasil parsial dapat dikembalikan jika perlu
