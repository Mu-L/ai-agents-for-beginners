# 🔍 بازیابی تقویت‌شده سازمانی (RAG) با Microsoft Foundry (.NET)

## 📋 اهداف یادگیری

این دفترچه یادداشت نحوه ساخت سیستم‌های بازیابی تقویت‌شده کیفیت‌سنجی شده (RAG) در سطح سازمانی با استفاده از چارچوب عامل مایکروسافت در .NET با Microsoft Foundry را نشان می‌دهد. شما یاد خواهید گرفت که چگونه عامل‌های آماده برای تولید بسازید که بتوانند در اسناد جستجو کنند و پاسخ‌های دقیق و آگاه از زمینه با امنیت و مقیاس‌پذیری سازمانی ارائه دهند.

**قابلیت‌های RAG سازمانی که خواهید ساخت:**
- 📚 **هوش سندی**: پردازش پیشرفته اسناد با خدمات هوش مصنوعی Azure
- 🔍 **جستجوی معنایی**: جستجوی برداری با عملکرد بالا و ویژگی‌های سازمانی
- 🛡️ **ادغام امنیتی**: دسترسی مبتنی بر نقش و الگوهای حفاظت داده‌ها
- 🏢 **معماری مقیاس‌پذیر**: سیستم‌های RAG آماده تولید با نظارت

## 🎯 معماری RAG سازمانی

### اجزای اصلی سازمانی
- **Microsoft Foundry**: پلتفرم AI سازمانی مدیریت‌شده با امنیت و تطابق
- **عامل‌های پایدار**: عامل‌های حالت‌دار با سابقه مکالمه و مدیریت زمینه
- **مدیریت فروشگاه برداری**: فهرست‌بندی و بازیابی اسناد در سطح سازمانی
- **ادغام هویت**: احراز هویت Azure AD و کنترل دسترسی مبتنی بر نقش

### مزایای سازمانی .NET
- **ایمنی نوع**: اعتبارسنجی زمان کامپایل برای عملیات RAG و ساختارهای داده‌ای
- **عملکرد ناهمگام**: پردازش اسناد و جستجوی غیرمسدودکننده
- **مدیریت حافظه**: استفاده کارآمد از منابع برای مجموعه‌های بزرگ اسناد
- **الگوهای ادغام**: ادغام بومی سرویس Azure با تزریق وابستگی

## 🏗️ معماری فنی

### خط لوله RAG سازمانی
```
Document Upload → Security Validation → Vector Processing → Index Creation
                      ↓                    ↓                  ↓
User Query → Authentication → Semantic Search → Context Ranking → AI Response
```

### اجزای اصلی .NET
- **Azure.AI.Agents.Persistent**: مدیریت عامل سازمانی با حفظ حالت
- **Azure.Identity**: احراز هویت یکپارچه برای دسترسی امن به سرویس Azure
- **Microsoft.Agents.AI.AzureAI**: پیاده‌سازی چارچوب عامل بهینه‌شده برای Azure
- **System.Linq.Async**: عملیات LINQ ناهمگام با عملکرد بالا

## ⚙️ پیش‌نیازها و راه‌اندازی

**محیط توسعه:**
- SDK .NET 9.0 یا بالاتر
- ویژوال استودیو 2022 یا VS Code با افزونه C#
- اشتراک Azure با دسترسی Microsoft Foundry

**بسته‌های NuGet مورد نیاز:**
```xml
<PackageReference Include="Microsoft.Extensions.AI" Version="9.9.0" />
<PackageReference Include="Azure.AI.Agents.Persistent" Version="1.2.0-beta.5" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

**راه‌اندازی احراز هویت Azure:**
```bash
# نصب Azure CLI و احراز هویت
az login
az account set --subscription "your-subscription-id"
```

**پیکربندی محیط:**
* پیکربندی Microsoft Foundry (به‌طور خودکار از طریق Azure CLI مدیریت می‌شود)
* اطمینان از احراز هویت به اشتراک Azure صحیح
