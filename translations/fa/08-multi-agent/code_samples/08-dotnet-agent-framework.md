# 🤝 سیستم‌های کاری چندعاملی سازمانی (.NET)

## 📋 اهداف یادگیری

این دفترچه یادداشت نشان می‌دهد چگونه می‌توان سیستم‌های چندعاملی پیشرفته با درجه سازمانی را با استفاده از چارچوب Agent مایکروسافت در .NET به همراه Azure OpenAI (API پاسخ‌ها) ساخت. شما یاد خواهید گرفت که چندین عامل تخصصی را از طریق جریان‌های کاری ساختاریافته با هم هماهنگ کنید و از قابلیت‌های سازمانی .NET برای راه‌حل‌های آمادهٔ تولید بهره ببرید.

**قابلیت‌های چندعاملی سازمانی که خواهید ساخت:**
- 👥 **همکاری عامل‌ها**: هماهنگی ایمن از نظر نوع با اعتبارسنجی در زمان کامپایل
- 🔄 **هماهنگی جریان کاری**: تعریف جریان کاری بیانی با الگوهای async در .NET
- 🎭 **تخصص نقش**: شخصیت‌ها و حوزه‌های تخصصی عامل‌ها با انواع قوی
- 🏢 **ادغام سازمانی**: الگوهای آماده تولید همراه با پایش و مدیریت خطا

## این نمونه چه چیزی را نشان می‌دهد

این سیستم جریان کاری چندعاملی یک سرویس پیشنهادی سفر هتل ایجاد می‌کند که شامل دو عامل تخصصی است:

1. **عامل پذیرش**: یک عامل سفر که پیشنهادات فعالیت و مکان را ارائه می‌دهد
2. **عامل کنسیرج**: پیشنهادات را بررسی می‌کند تا اطمینان حاصل شود که تجربه‌ها اصیل و غیرتوریستی هستند

عامل‌ها در جریان کاری این‌گونه با هم کار می‌کنند:
- عامل پذیرش درخواست اولیه سفر را دریافت می‌کند
- عامل کنسیرج پیشنهاد را مرور و اصلاح می‌کند
- جریان کاری پاسخ‌ها را به صورت هم‌زمان منتقل می‌کند

## مفاهیم کلیدی

### هماهنگی عامل‌ها
نمونه هماهنگی ایمن از نظر نوع عامل‌ها را با استفاده از چارچوب Agent مایکروسافت همراه با اعتبارسنجی زمان کامپایل نشان می‌دهد.

### هماهنگی جریان کاری
از تعریف بیانی جریان کاری با الگوهای async در .NET برای اتصال چند عامل در یک خط لوله استفاده می‌کند.

### پخش پاسخ‌ها
پاسخ‌های عامل را در زمان واقعی با استفاده از async enumerables و معماری رویداد محور پخش می‌کند.

### ادغام سازمانی
الگوهای آماده تولید شامل موارد زیر را نشان می‌دهد:
- پیکربندی متغیرهای محیطی
- مدیریت امن اعتبارنامه‌ها
- مدیریت خطا
- پردازش رویدادهای ناهمزمان
