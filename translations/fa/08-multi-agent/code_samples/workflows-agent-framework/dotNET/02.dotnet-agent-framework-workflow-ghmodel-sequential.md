# ⏩ گردش‌های کاری متوالی عوامل با Azure OpenAI (API پاسخ‌ها) (.NET)

## 📋 آموزش پیشرفته پردازش متوالی

این دفترچه نشان می‌دهد که چگونه می‌توان از **الگوهای گردش کاری متوالی** با استفاده از چارچوب عامل مایکروسافت برای .NET و Azure OpenAI (API پاسخ‌ها) بهره‌برداری کرد. شما یاد خواهید گرفت که چگونه خط‌لوله‌های پردازش پیچیده و گام‌به‌گام بسازید که در آن عوامل به ترتیب خاصی اجرا می‌شوند، به طوری که هر مرحله بر نتایج مرحله قبلی بنا می‌شود.

## 🎯 اهداف یادگیری

### 🔄 **معماری پردازش متوالی**
- **طراحی گردش کاری خطی**: ایجاد خط‌لوله‌های پردازش گام‌به‌گام با وابستگی‌های واضح  
- **مدیریت وضعیت**: نگهداری زمینه و جریان داده‌ها در مراحل متوالی گردش کاری  
- **Azure OpenAI (API پاسخ‌ها)**: بهره‌گیری از مدل‌های Azure OpenAI در گردش‌های کاری چندمرحله‌ای .NET  
- **الگوهای خط‌لوله سازمانی**: ساخت سیستم‌های پردازش متوالی آماده تولید  

### 🏗️ **الگوهای پیشرفته متوالی**
- **پردازش مرحله-دروازه**: پیاده‌سازی نقاط اعتبارسنجی بین مراحل گردش کاری  
- **حفظ زمینه**: نگهداری وضعیت و دانش انباشته شده در تمام مراحل  
- **انتشار خطا**: مدیریت شکست‌ها به طور نرم و قابل کنترل در زنجیره‌های پردازش متوالی  
- **بهینه‌سازی عملکرد**: اجرای متوالی کارآمد با حداقل سربار  

### 🏢 **برنامه‌های سازمانی متوالی**
- **خط‌لوله پردازش اسناد**: تحلیل، تبدیل و اعتبارسنجی چندمرحله‌ای اسناد  
- **گردش‌های کاری تضمین کیفیت**: فرآیندهای بازبینی، اعتبارسنجی و تصویب متوالی

- **اتوماسیون فرآیندهای کسب‌وکار**: گردش‌های کاری کسب‌وکار چندمرحله‌ای با وابستگی‌های مرحله‌ای مشخص  

## ⚙️ پیش‌نیازها و راه‌اندازی  

### 📦 **بسته‌های NuGet مورد نیاز**  

بسته‌های ضروری برای گردش‌های کاری متوالی .NET:  

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **پیکربندی Azure OpenAI**

**تنظیم محیط (.env فایل):**  
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**مدیریت پیکربندی:**  
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

## 💻 اجرای کد

پیاده‌سازی کامل در فایل `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs` موجود است. این فایل گردش کاری **سه مرحله‌ای تحلیل مبلمان** را نشان می‌دهد:  

1. **مرحله 1 - عامل فروش**: تحلیل تصاویر مبلمان و ارائه پیشنهادهای خرید  
2. **مرحله 2 - عامل قیمت**: ارائه تجزیه و تحلیل دقیق قیمت و گزینه‌های بودجه  
3. **مرحله 3 - عامل نقل‌قول**: تولید سند نقل‌قول حرفه‌ای در قالب Markdown  

### 🏗️ **معماری گردش کاری**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

هر عامل:
- خروجی مرحله قبلی را به عنوان زمینه دریافت می‌کند  
- بر اساس تحلیل قبلی با تخصص ویژه توسعه می‌یابد  
- با مدیریت وضعیت، تداوم گردش کاری را حفظ می‌کند

**پیش‌نیازها:**

- فایل `.env` خود را با نقطه پایان و استقرار Azure OpenAI پیکربندی کنید، سپس با `az login` وارد شوید

### 📝 خروجی مورد انتظار

گردش کاری:  
1. **عامل فروش**: شناسایی اقلام مبلمان از تصویر و ارائه توصیه‌ها  
2. **عامل قیمت**: افزودن تجزیه و تحلیل قیمت با سطوح بودجه و توصیه‌های خرید  
3. **عامل نقل‌قول**: تولید سند نقل‌قول قالب‌بندی شده با تمام اطلاعات تجمیع‌شده  

خروجی نهایی یک نقل‌قول جامع و حرفه‌ای مبلمان بر اساس تحلیل تصویر خواهد بود.
