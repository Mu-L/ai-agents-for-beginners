# 🔄 گردش‌های کاری پایه کارگزار با Azure OpenAI (Responses API) (.NET)

## 📋 آموزش هماهنگ‌سازی گردش کار

این دفترچه یادداشت نشان می‌دهد چگونه گردش‌های کاری پیچیده **کارگزار** را با استفاده از چارچوب کارگزار مایکروسافت برای .NET و Azure OpenAI (Responses API) بسازید. شما خواهید آموخت که چگونه فرایندهای تجاری چند مرحله‌ای ایجاد کنید که در آن کارگزاران هوش مصنوعی برای انجام وظایف پیچیده از طریق الگوهای هماهنگ‌سازی ساختاریافته با هم همکاری می‌کنند.

## 🎯 اهداف آموزشی

### 🏗️ **اصول معماری گردش کار**
- **سازنده گردش کار**: طراحی و هماهنگی فرایندهای پیچیده چند مرحله‌ای هوش مصنوعی
- **هماهنگی کارگزار**: هماهنگی چند کارگزار تخصصی در گردش‌های کاری
- **Azure OpenAI (Responses API)**: استفاده از API پاسخ‌های Azure OpenAI در گردش‌های کاری
- **طراحی بصری گردش کار**: ایجاد و تجسم ساختارهای گردش کار برای درک بهتر

### 🔄 **الگوهای هماهنگ‌سازی فرآیند**
- **پردازش ترتیبی**: زنجیره کردن چند کارگزار وظیفه‌ای به ترتیب منطقی
- **مدیریت وضعیت**: حفظ زمینه و جریان داده در مراحل گردش کار
- **مدیریت خطا**: پیاده‌سازی بازیابی خطای مقاوم و پایداری گردش کار
- **بهینه‌سازی عملکرد**: طراحی گردش‌های کاری کارآمد برای عملیات در مقیاس سازمانی

### 🏢 **کاربردهای گردش کار سازمانی**
- **اتوماسیون فرآیندهای کسب‌وکار**: خودکارسازی گردش‌های کاری پیچیده سازمانی
- **خط تولید محتوا**: گردش‌های کاری ویرایشی با مراحل بازبینی و تأیید
- **اتوماسیون خدمات مشتری**: حل درخواست‌های چند مرحله‌ای مشتری
- **گردش‌های کاری پردازش داده**: گردش‌های کاری ETL با تبدیل مبتنی بر هوش مصنوعی

## ⚙️ پیش‌نیازها و راه‌اندازی

### 📦 **بسته‌های NuGet مورد نیاز**

این نمایش گردش کار چند بسته کلیدی .NET را استفاده می‌کند:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **پیکربندی Azure OpenAI**

**راه‌اندازی محیط (.env فایل):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**دسترسی به Azure OpenAI:**
1. یک منبع Azure OpenAI در پورتال Azure ایجاد کنید
2. یک مدل مستقر کنید (مثلاً `gpt-5-mini`) و نام استقرار را یادداشت کنید
3. با `az login` وارد شوید و متغیرهای محیطی را مطابق بالا تنظیم کنید

### 🏗️ **نمای کلی معماری گردش کار**

```mermaid
graph TD
    A[سازنده جریان کاری] --> B[ثبت‌نام نماینده]
    B --> C[موتور اجرای جریان کاری]
    C --> D[Agent 1: تولیدکننده محتوا]
    C --> E[Agent 2: بازبین محتوا] 
    D --> F[نتایج جریان کاری]
    E --> F
    G[آزور اوپن‌ای‌آی (رابط پاسخ‌ها)] --> D
    G --> E
```

**اجزای کلیدی:**
- **WorkflowBuilder**: موتور اصلی هماهنگ‌سازی برای طراحی گردش‌های کاری
- **AIAgent**: کارگزاران تخصصی فردی با قابلیت‌های خاص
- **کلاینت Azure OpenAI**: یکپارچه‌سازی API پاسخ‌های Azure OpenAI
- **محیط اجرایی**: مدیریت وضعیت و جریان داده بین مراحل گردش کار
