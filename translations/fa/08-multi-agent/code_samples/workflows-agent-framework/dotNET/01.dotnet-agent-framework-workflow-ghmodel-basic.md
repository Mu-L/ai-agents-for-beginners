# 🔄 گردش‌کارهای پایه‌ای نماینده با Azure OpenAI (API پاسخ‌ها) (.NET)

## 📋 آموزش هماهنگ‌سازی گردش‌کار

این دفترچه یادداشت نشان می‌دهد چگونه گردش‌کارهای پیچیده‌ی **نماینده** را با استفاده از چارچوب نماینده مایکروسافت برای .NET و Azure OpenAI (API پاسخ‌ها) بسازید. شما یاد می‌گیرید چگونه فرایندهای چندمرحله‌ای کسب‌وکار ایجاد کنید که در آن نمایندگان هوش مصنوعی برای انجام وظایف پیچیده از طریق الگوهای هماهنگ‌سازی ساختارمند همکاری می‌کنند.

## 🎯 اهداف یادگیری

### 🏗️ **مبانی معماری گردش‌کار**
- **سازنده گردش‌کار**: طراحی و هماهنگی فرایندهای پیچیده چندمرحله‌ای هوش مصنوعی
- **هماهنگی نمایندگان**: هماهنگی چند نماینده تخصصی در گردش‌کارها

### 🔄 **الگوهای هماهنگی فرایند**
- **پردازش متوالی**: زنجیره کردن وظایف متعدد نماینده به ترتیب منطقی
- **مدیریت وضعیت**: حفظ زمینه و جریان داده در سراسر مراحل گردش‌کار
- **مدیریت خطا**: پیاده‌سازی بازیابی مقاوم و استحکام گردش‌کار
- **بهینه‌سازی عملکرد**: طراحی گردش‌کارهای کارآمد برای عملیات در مقیاس سازمانی

- **خط تولید محتوا**: گردش‌کارهای ویرایشی با مراحل بازبینی و تأیید

- **گردش‌کارهای پردازش داده**: گردش‌کارهای ETL با تبدیل‌های مبتنی بر هوش مصنوعی

## ⚙️ پیش‌نیازها و راه‌اندازی

### 📦 **بسته‌های NuGet مورد نیاز**

این نمونه گردش‌کار از چند بسته کلیدی .NET استفاده می‌کند:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **پیکربندی Azure OpenAI**

**تنظیم محیط (.env file):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**دسترسی به Azure OpenAI:**
1. ایجاد منبع Azure OpenAI در پرتال Azure
2. استقرار یک مدل (برای مثال، `gpt-4.1-mini`) و یادداشت نام استقرار
3. ورود با `az login` و پیکربندی متغیرهای محیطی همان‌طور که بالاتر نشان داده شده است

### 🏗️ **بررسی اجمالی معماری گردش‌کار**

```mermaid
graph TD
    A[سازنده جریان کاری] --> B[ثبت‌نام عامل]
    B --> C[موتور اجرای جریان کاری]
    C --> D[Agent 1: تولیدکننده محتوا]
    C --> E[Agent 2: بازبینی‌کننده محتوا] 
    D --> F[نتایج جریان کاری]
    E --> F
    G[آزور OpenAI (رابط برنامه‌نویسی پاسخ‌ها)] --> D
    G --> E
```

**اجزای کلیدی:**
- **WorkflowBuilder**: موتور اصلی هماهنگی برای طراحی گردش‌کارها
- **AIAgent**: نمایندگان تخصصی منفرد با قابلیت‌های خاص
- **Azure OpenAI Client**: یکپارچه‌سازی API پاسخ‌های Azure OpenAI
- **Execution Context**: مدیریت وضعیت و جریان داده بین مراحل گردش‌کار
