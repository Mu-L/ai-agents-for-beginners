# ⚡ گردش‌کارهای هم‌زمان عامل با Azure OpenAI (Responses API) (.NET)

## 📋 آموزش پردازش موازی با عملکرد بالا

این نوت‌بوک **الگوهای گردش‌کار هم‌زمان** را با استفاده از Microsoft Agent Framework برای .NET و Azure OpenAI (Responses API) نشان می‌دهد. شما یاد می‌گیرید چگونه گردش‌کارهای پردازش موازی با عملکرد بالا بسازید که با اجرای هم‌زمان چند عامل هوش مصنوعی حداکثر توان عملیاتی را فراهم کرده و هماهنگی و سازگاری داده‌ها را حفظ می‌کنند.

## 💻 اجرای کد

پیاده‌سازی کامل در فایل `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` موجود است. این فایل یک **گردش‌کار Fan-Out/Fan-In هم‌زمان** برای برنامه‌ریزی سفر را نشان می‌دهد:

### 🏗️ **معماری گردش‌کار**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**اجزای کلیدی:**

1. **ConcurrentStartExecutor**: درخواست کاربر را هم‌زمان به همه عامل‌ها پخش می‌کند
2. **عامل پژوهشگر**: به صورت هم‌زمان مقاصد و جاذبه‌ها را تحلیل می‌کند
3. **عامل برنامه‌ریز**: برنامه‌های سفر دقیق را به‌صورت هم‌زمان می‌سازد
4. **ConcurrentAggregationExecutor**: نتایج هر دو عامل را جمع‌آوری و ادغام می‌کند

### 🎯 **الگوی Fan-Out/Fan-In**

این گردش‌کار الگوی کلاسیک **Fan-Out/Fan-In** را نشان می‌دهد:
- **Fan-Out**: یک پیام ورودی به طور هم‌زمان به چند عامل ارسال می‌شود
- **پردازش هم‌زمان**: چند عامل به‌صورت موازی روی همان کار فعالیت می‌کنند
- **Fan-In**: نتایج تمام عامل‌ها جمع‌آوری و در یک خروجی واحد ترکیب می‌شود

### 🔍 درک اجزای سفارشی Executor‌ها

**ConcurrentStartExecutor:**
- پیاده‌سازی `IMessageHandler<string>` برای دریافت ورودی رشته‌ای
- پیام‌ها را به همه عامل‌های متصل پخش می‌کند
- ارسال `TurnToken` برای فعال‌سازی پردازش هم‌زمان

**ConcurrentAggregationExecutor:**
- پیاده‌سازی `IMessageHandler<ChatMessage>` برای دریافت پاسخ عامل‌ها
- جمع‌آوری پیام‌ها به صورت ایمن در برابر نخ‌ها
- ادغام پس از دریافت تمام پاسخ‌های مورد انتظار
- خروجی نهایی را با استفاده از `context.YieldOutputAsync()` ارائه می‌دهد

### ⚡ مزایای عملکرد

**هم‌زمان در مقابل متوالی:**
- متوالی: عامل1 (30ثانیه) → عامل2 (30ثانیه) = **در مجموع 60 ثانیه**
- هم‌زمان: عامل1 (30ثانیه) || عامل2 (30ثانیه) = **در مجموع 30 ثانیه**

**افزایش توان عملیاتی**: تا N برابر سریع‌تر برای N عامل هم‌زمان (بستگی به حجم کار و منابع دارد)
