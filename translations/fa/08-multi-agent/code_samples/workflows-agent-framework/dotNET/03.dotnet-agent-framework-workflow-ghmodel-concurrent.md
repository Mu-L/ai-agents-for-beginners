# ⚡ گردش‌کارهای همزمان عامل با Azure OpenAI (Responses API) (.NET)

## 📋 آموزش پردازش موازی با عملکرد بالا

این دفترچه الکترونیکی **الگوهای گردش‌کار همزمان** را با استفاده از Microsoft Agent Framework برای .NET و Azure OpenAI (Responses API) نشان می‌دهد. شما خواهید آموخت چگونه گردش‌کارهای پردازش موازی با عملکرد بالا بسازید که توان عملیاتی را با اجرای همزمان چندین عامل هوش مصنوعی به حداکثر می‌رساند، در حالی که هماهنگی و ثبات داده‌ها را حفظ می‌کند.

## 💻 اجرای کد

پیاده‌سازی کامل در فایل `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` موجود است. این فایل گردش‌کار همزمان **Fan-Out/Fan-In** برای برنامه‌ریزی سفر را نشان می‌دهد:

### 🏗️ **معماری گردش‌کار**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**اجزای کلیدی:**

1. **ConcurrentStartExecutor**: درخواست کاربر را به تمام عوامل به طور همزمان ارسال می‌کند
2. **Researcher Agent**: به طور همزمان مقاصد و جاذبه‌ها را تحلیل می‌کند
3. **Planner Agent**: به طور همزمان برنامه‌های سفر دقیق ایجاد می‌کند
4. **ConcurrentAggregationExecutor**: نتایج هر دو عامل را جمع‌آوری و ادغام می‌کند

### 🎯 **الگوی Fan-Out/Fan-In**

این گردش‌کار الگوی کلاسیک **Fan-Out/Fan-In** را نشان می‌دهد:
- **Fan-Out**: یک پیام ورودی به چندین عامل به طور همزمان ارسال می‌شود
- **پردازش همزمان**: چندین عامل به صورت موازی روی همان کار کار می‌کنند
- **Fan-In**: نتایج همه عوامل جمع‌آوری و در یک خروجی ادغام می‌شوند

### 📝 خروجی مورد انتظار

گردش‌کار موارد زیر را انجام می‌دهد:
1. **ارسال درخواست**: فرستادن «برای ماه دسامبر برنامه سفر به سیاتل بساز» به هر دو عامل
2. **پردازش همزمان**: هر دو عامل به طور همزمان کار می‌کنند:
   - محقق جاذبه‌ها و جزئیات را شناسایی می‌کند
   - برنامه‌ریز برنامه سفر و لجستیک را ایجاد می‌کند
3. **تجمیع**: پاسخ‌های هر دو عامل را در یک خروجی جامع ترکیب می‌کند
4. **نمایش نتایج**: برنامه سفر ادغام‌شده با تمام اطلاعات را نمایش می‌دهد

### 🔍 آشنایی با Executorهای سفارشی

**ConcurrentStartExecutor:**
- پیاده‌سازی `IMessageHandler<string>` برای دریافت ورودی رشته‌ای
- پیام‌ها را به همه عوامل متصل شده ارسال می‌کند
- ارسال `TurnToken` برای آغاز پردازش همزمان

**ConcurrentAggregationExecutor:**
- پیاده‌سازی `IMessageHandler<ChatMessage>` برای دریافت پاسخ عوامل
- جمع‌آوری پیام‌ها به صورت ایمن برای رشته‌ها
- زمانی که همه پاسخ‌های مورد انتظار رسید، تجمیع می‌کند
- خروجی نهایی را با استفاده از `context.YieldOutputAsync()` می‌دهد

### ⚡ مزایای عملکرد

**همزمان در مقابل متوالی:**
- متوالی: Agent1 (30 ثانیه) → Agent2 (30 ثانیه) = **مجموعاً 60 ثانیه**
- همزمان: Agent1 (30 ثانیه) || Agent2 (30 ثانیه) = **مجموعاً 30 ثانیه**

**افزایش توان عملیاتی**: تا N برابر سریع‌تر برای N عامل همزمان (بسته به بار کاری و منابع)

### 🛡️ مدیریت خطا

گردش‌کار شکست‌های عوامل فردی را با نرمی مدیریت می‌کند:
- اگر یک عامل شکست بخورد، عوامل دیگر به کار ادامه می‌دهند
- تجمیع‌کننده می‌تواند منطق تایم‌اوت داشته باشد
- در صورت نیاز نتایج جزئی برگردانده می‌شود
