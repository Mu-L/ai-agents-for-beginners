# پاسخ‌های آزمون دانش

## سؤال ۱

**پاسخ: A2.** وجود چند عامل زمانی مفید است که یک وظیفه نقش‌های تخصصی متمایز، ابزارها یا مجوزهای مجزایی داشته باشد و انتقال‌های روشنی که نیاز به هماهنگی دارد. یک ربات پشتیبانی ساده معمولاً می‌تواند به عنوان یک عامل واحد باقی بماند. حجم بالای درخواست‌های یکسان عمدتاً یک نگرانی مقیاس‌پذیری است و خودبه‌خود طراحی چندعاملی لازم ندارد.

## سؤال ۲

**پاسخ: A1.** یک عامل واحد معمولاً زمانی که یک مجموعه دستورالعمل و ابزار می‌تواند وظیفه را انجام دهد، آسان‌تر برای آزمایش، مشاهده و اجرا است. دسترسی به چندین ابزار به طور خودکار نیاز به چند عامل ندارد. نقش‌های جدایی که مجوزها و نیازهای ممیزی متفاوتی دارند ممکن است طراحی چندعاملی را توجیه کنند.
