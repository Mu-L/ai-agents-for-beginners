# ⚡ Azure OpenAI (Responses API) (.NET) के साथ समानांतर एजेंट वर्कफ़्लो

## 📋 उच्च-प्रदर्शन समानांतर प्रसंस्करण ट्यूटोरियल

यह नोटबुक Microsoft Agent Framework for .NET और Azure OpenAI (Responses API) का उपयोग करके **समकालिक वर्कफ़्लो पैटर्न** प्रदर्शित करता है। आप सीखेंगे कि कैसे उच्च-प्रदर्शन, समानांतर प्रसंस्करण वर्कफ़्लो बनाएँ जो कई AI एजेंटों को एक साथ कार्यान्वित करके थ्रूपुट को अधिकतम करते हैं जबकि समन्वय और डेटा सुसंगतता बनाए रखते हैं।

## 🎯 सीखने के उद्देश्य

### 🚀 **समानांतर प्रसंस्करण के मूल तत्व**
- **समानांतर एजेंट निष्पादन**: अधिकतम प्रदर्शन के लिए एक साथ कई AI एजेंट चलाएँ
- **Async/Await पैटर्न**: कुशल समकालिकता के लिए .NET के async प्रोग्रामिंग मॉडल का लाभ उठाएँ
- **Azure OpenAI (Responses API)**: Azure OpenAI Responses API के लिए कई समकालिक कॉल्स को समन्वयित करें
- **संसाधन प्रबंधन**: समकालिक संचालन के दौरान AI मॉडल संसाधनों का कुशल प्रबंधन

### 🏗️ **उन्नत समकालिक वास्तुकला**
- **टास्क-आधारित समानांतरता**: सर्वोत्तम समकालिक निष्पादन के लिए .NET Task Parallel Library का उपयोग करें
- **सिंक स्ट्रॉनाइज़ेशन पैटर्न**: रेस कंडीशंस से बचते हुए समकालिक एजेंटों का समन्वय करें
- **लोड संतुलन**: उपलब्ध समकालिक प्रसंस्करण क्षमता के बीच कार्य को कुशलतापूर्वक वितरित करें
- **फॉल्ट सहिष्णुता**: पूरे वर्कफ़्लो को रोके बिना व्यक्तिगत एजेंट विफलताओं को संभालें

### 🏢 **एंटरप्राइज समकालिक एप्लिकेशन**
- **उच्च-मात्रा दस्तावेज़ प्रसंस्करण**: कई दस्तावेज़ों को एक साथ संसाधित करें
- **रियल-टाइम सामग्री विश्लेषण**: आने वाले डेटा स्ट्रीम का समकालिक विश्लेषण
- **बैच प्रोसेसिंग अनुकूलन**: बड़े पैमाने पर डेटा प्रसंस्करण संचालन के लिए थ्रूपुट अधिकतम करें
- **मल्टी-मॉडल विश्लेषण**: विभिन्न सामग्री प्रकारों और प्रारूपों की समानांतर प्रसंस्करण

## 💻 कोड चलाना

पूर्ण कार्यान्वयन `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` में उपलब्ध है। यह फ़ाइल यात्रा योजना के लिए एक **फैन-आउट/फैन-इन समकालिक वर्कफ़्लो** प्रदर्शित करती है:

### 🏗️ **वर्कफ़्लो वास्तुकला**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**मुख्य घटक:**

1. **ConcurrentStartExecutor**: उपयोगकर्ता अनुरोध को सभी एजेंटों को एक साथ प्रसारित करता है
2. **Researcher Agent**: गंतव्यों और आकर्षणों का समानांतर विश्लेषण करता है
3. **Planner Agent**: विस्तार से यात्रा योजनाएँ समानांतर बनाता है
4. **ConcurrentAggregationExecutor**: दोनों एजेंटों से परिणाम एकत्र करता है और विलय करता है

### 🎯 **फैन-आउट/फैन-इन पैटर्न**

यह वर्कफ़्लो क्लासिक **फैन-आउट/फैन-इन** पैटर्न दिखाता है:
- **फैन-आउट**: एक इनपुट संदेश कई एजेंटों को एक साथ प्रसारित किया जाता है
- **समकालिक प्रसंस्करण**: कई एजेंट एक ही कार्य पर समानांतर काम करते हैं
- **फैन-इन**: सभी एजेंटों के परिणाम एकत्रित और समेकित होकर एक आउटपुट बनाते हैं

### 📝 अपेक्षित आउटपुट

वर्कफ़्लो निम्न कार्य करेगा:
1. **अनुरोध प्रसारण**: "डिसंबर में सिएटल की यात्रा की योजना बनाएं" संदेश दोनों एजेंटों को भेजेगा
2. **समकालिक प्रसंस्करण**: दोनों एजेंट एक साथ कार्य करेंगे:
   - शोधकर्ता आकर्षणों और विवरणों की पहचान करता है
   - योजनाकार यात्रा कार्यक्रम और लॉजिस्टिक्स बनाता है
3. **समेकन**: दोनों प्रतिक्रियाओं को व्यापक आउटपुट में संयोजित करें
4. **परिणाम प्रदर्शित करें**: सभी जानकारी के साथ संयुक्त यात्रा योजना दिखाएं

### 🔧 कस्टमाइज़ेशन विकल्प

**और समकालिक एजेंट जोड़ें:**
```csharp
// Create additional specialized agents
AIAgent budgetAgent = azureClient.GetOpenAIResponseClient(deployment).CreateAIAgent(
    name: "Budget-Agent", instructions: "Calculate travel costs...");

// Add to fan-out
var workflow = new WorkflowBuilder(startExecutor)
    .AddFanOutEdge(startExecutor, targets: [researcherAgent, plannerAgent, budgetAgent])
    .AddFanInEdge(aggregationExecutor, sources: [researcherAgent, plannerAgent, budgetAgent])
    .WithOutputFrom(aggregationExecutor)
    .Build();

// Update aggregation count
if (this._messages.Count == 3) { ... }
```

**एजेंट निर्देश संशोधित करें:**

यह समकालिक पैटर्न आदर्श है:

- **कोड समीक्षा**: कई समीक्षक विभिन्न दृष्टिकोणों से कोड का विश्लेषण करते हैं
- **बाज़ार अनुसंधान**: विभिन्न बाज़ार खंडों का समानांतर विश्लेषण
- **दस्तावेज़ प्रसंस्करण**: समकालिक निकर्षण, विश्लेषण और सत्यापन
- **बहु-दृष्टिकोण विश्लेषण**: एक ही इनपुट पर विभिन्न दृष्टिकोण प्राप्त करना

### 🔍 कस्टम कार्यान्वयन को समझना

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` को लागू करता है ताकि स्ट्रिंग इनपुट स्वीकार कर सके
- सभी जुड़े एजेंटों को संदेश प्रसारित करता है
- समकालिक प्रसंस्करण को ट्रिगर करने के लिए `TurnToken` भेजता है

**ConcurrentAggregationExecutor:**
- एजेंट प्रतिक्रियाओं को प्राप्त करने के लिए `IMessageHandler<ChatMessage>` लागू करता है
- संदेशों को थ्रेड-सेफ तरीके से इकट्ठा करता है
- सभी अपेक्षित प्रतिक्रियाएं आने पर समेकित करता है
- `context.YieldOutputAsync()` का उपयोग करके अंतिम आउटपुट देता है

### ⚡ प्रदर्शन लाभ

**समकालिक बनाम अनुक्रमिक:**
- अनुक्रमिक: Agent1 (30s) → Agent2 (30s) = **कुल 60 सेकंड**
- समकालिक: Agent1 (30s) || Agent2 (30s) = **कुल 30 सेकंड**

**थ्रूपुट में सुधार**: N समकालिक एजेंटों के लिए N गुना तेज (कार्यभार और संसाधनों पर निर्भर करता है)
