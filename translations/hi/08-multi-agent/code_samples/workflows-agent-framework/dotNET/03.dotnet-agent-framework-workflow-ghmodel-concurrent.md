# ⚡ Azure OpenAI (Responses API) के साथ समानांतर एजेंट वर्कफ़्लो (.NET)

## 📋 उच्च-दक्षता समानांतर प्रसंस्करण ट्यूटोरियल

यह नोटबुक Microsoft एजेंट फ्रेमवर्क फॉर .NET और Azure OpenAI (Responses API) का उपयोग करके **समानांतर वर्कफ़्लो पैटर्न** प्रदर्शित करता है। आप सीखेंगे कि कैसे उच्च-दक्षता, समानांतर प्रसंस्करण वर्कफ़्लो बनाएँ जो कई AI एजेंटों को एक साथ निष्पादित करके थ्रूपुट को अधिकतम करते हैं जबकि समन्वय और डेटा उचितता बनाए रखते हैं।

## 🎯 सीखने के उद्देश्य

### 🚀 **समानांतर प्रसंस्करण के मूल तत्व**
- **समानांतर एजेंट निष्पादन**: अधिकतम प्रदर्शन के लिए कई AI एजेंटों को एक साथ चलाना
- **Async/Await पैटर्न**: कुशल समवर्ती के लिए .NET के async प्रोग्रामिंग मॉडल का उपयोग
- **Azure OpenAI (Responses API)**: Azure OpenAI Responses API को कई समानांतर कॉलों के माध्यम से समन्वित करना
- **रिसोर्स प्रबंधन**: समानांतर ऑपरेशनों में AI मॉडल संसाधनों का कुशल प्रबंधन

### 🏗️ **उन्नत समानांतर वास्तुकला**
- **टास्क-आधारित समानांतरता**: सर्वश्रेष्ठ समानांतर निष्पादन के लिए .NET टास्क पैरेलल लाइब्रेरी का उपयोग
- **सिंक पैटर्न**: रेस कंडीशन से बचते हुए समानांतर एजेंटों का समन्वय
- **लोड बैलेंसिंग**: उपलब्ध समानांतर प्रसंस्करण क्षमता में कार्य का कुशल वितरण
- **फॉल्ट टॉलरेंस**: पूरे वर्कफ़्लो को रोकने के बिना व्यक्तिगत एजेंट विफलताओं को संभालना

### 🏢 **एंटरप्राइज समानांतर अनुप्रयोग**
- **उच्च मात्रा दस्तावेज़ प्रसंस्करण**: एक साथ कई दस्तावेज़ों को प्रक्रिया करना
- **रीयल-टाइम कंटेंट एनालिसिस**: आने वाले डेटा स्ट्रीम्स का समानांतर विश्लेषण
- **बैच प्रसंस्करण अनुकूलन**: बड़े पैमाने के डेटा प्रोसेसिंग ऑपरेशनों के लिए थ्रूपुट अधिकतम करना
- **मल्टी-मोडल एनालिसिस**: विभिन्न सामग्री प्रकारों और फ़ॉर्मैटों का समानांतर प्रसंस्करण

## 💻 कोड चलाना

पूर्ण कार्यान्वयन `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` में उपलब्ध है। यह फ़ाइल यात्रा योजना के लिए एक **फैन-आउट/फैन-इन समानांतर वर्कफ़्लो** प्रदर्शित करती है:

### 🏗️ **वर्कफ़्लो वास्तुकला**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**मुख्य घटक:**

1. **ConcurrentStartExecutor**: उपयोगकर्ता अनुरोध को सभी एजेंटों पर एक साथ प्रसारित करता है
2. **Researcher Agent**: गंतव्यों और आकर्षणों का समानांतर विश्लेषण करता है
3. **Planner Agent**: समानांतर में विस्तृत यात्रा योजनाएं बनाता है
4. **ConcurrentAggregationExecutor**: दोनों एजेंटों से परिणाम एकत्रित करता है और विलय करता है

### 🎯 **फैन-आउट/फैन-इन पैटर्न**

यह वर्कफ़्लो क्लासिक **फैन-आउट/फैन-इन** पैटर्न को प्रदर्शित करता है:
- **फैन-आउट**: एक इनपुट संदेश को एक साथ कई एजेंटों को प्रसारित करना
- **समानांतर प्रसंस्करण**: एक ही कार्य पर कई एजेंट समानांतर में काम करते हैं
- **फैन-इन**: सभी एजेंटों के परिणाम एकत्रित किए जाते हैं और एकल आउटपुट में विलय किए जाते हैं

### 📝 अपेक्षित आउटपुट

वर्कफ़्लो ये करेगा:
1. **अनुरोध प्रसारित करें**: "डिसंबर में सिएटल की यात्रा योजना बनाएं" दोनों एजेंटों को भेजें
2. **समानांतर प्रसंस्करण**: दोनों एजेंट एक साथ काम करेंगे:
   - शोधकर्ता आकर्षणों और विवरणों की पहचान करता है
   - योजनाकार यात्रा कार्यक्रम और लॉजिस्टिक्स बनाता है
3. **संकलन**: दोनों प्रतिक्रियाओं को साझा आउटपुट में संयोजित करें
4. **परिणाम प्रदर्शित करें**: सभी जानकारी के साथ संयुक्त यात्रा योजना दिखाएं

### 🔧 अनुकूलन विकल्प

**अधिक समानांतर एजेंट जोड़ें:**
```csharp
// Create additional specialized agents
AIAgent budgetAgent = azureClient.GetOpenAIResponseClient(deployment).CreateAIAgent(
    name: "Budget-Agent", instructions: "Calculate travel costs...");

// Add to fan-out
var workflow = new WorkflowBuilder(startExecutor)
    .AddFanOutEdge(startExecutor, targets: [researcherAgent, plannerAgent, budgetAgent])
    .AddFanInEdge(aggregationExecutor, sources: [researcherAgent, plannerAgent, budgetAgent])
    .WithOutputFrom(aggregationExecutor)
    .Build();

// Update aggregation count
if (this._messages.Count == 3) { ... }
```

यह समानांतर पैटर्न आदर्श है:

- **कोड समीक्षा**: कई समीक्षक विभिन्न दृष्टिकोणों से कोड का विश्लेषण करते हैं
- **बाजार अनुसंधान**: विभिन्न बाजार क्षेत्रों का समानांतर विश्लेषण
- **दस्तावेज़ प्रसंस्करण**: समानांतर निष्कर्षण, विश्लेषण और सत्यापन
- **मल्टी-पर्सपेक्टिव एनालिसिस**: एक ही इनपुट पर विविध दृष्टिकोण प्राप्त करना

### 🔍 कस्टम एक्सेक्यूटर्स को समझना

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` लागू करता है ताकि स्ट्रिंग इनपुट स्वीकारा जा सके
- संदेशों को सभी जुड़े एजेंटों को प्रसारित करता है
- समानांतर प्रसंस्करण शुरू करने के लिए `TurnToken` भेजता है

**ConcurrentAggregationExecutor:**
- एजेंट प्रतिक्रियाओं को प्राप्त करने के लिए `IMessageHandler<ChatMessage>` लागू करता है
- थ्रेड-सुरक्षित तरीके से संदेश एकत्र करता है
- सभी अपेक्षित प्रतिक्रियाएं आने पर समेकित करता है
- `context.YieldOutputAsync()` का उपयोग करके अंतिम आउटपुट देता है

### ⚡ प्रदर्शन लाभ

**समानांतर बनाम अनुक्रमिक:**
- अनुक्रमिक: एजेंट1 (30s) → एजेंट2 (30s) = **कुल 60 सेकंड**
- समानांतर: एजेंट1 (30s) || एजेंट2 (30s) = **कुल 30 सेकंड**

**थ्रूपुट सुधार**: N समानांतर एजेंटों के लिए N× तक तेज (वर्कलोड और संसाधनों पर निर्भर)
