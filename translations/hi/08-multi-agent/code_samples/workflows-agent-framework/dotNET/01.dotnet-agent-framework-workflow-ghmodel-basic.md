# 🔄 Azure OpenAI (Responses API) के साथ बुनियादी एजेंट वर्कफ़्लो (.NET)

## 📋 वर्कफ़्लो ऑर्केस्ट्रेशन ट्यूटोरियल

यह नोटबुक दिखाता है कि Microsoft Agent Framework for .NET और Azure OpenAI (Responses API) का उपयोग करके कैसे परिष्कृत **एजेंट वर्कफ़्लो** बनाए जाएं। आप सीखेंगे कि कैसे बहु-चरणीय व्यापार प्रक्रियाएं बनाईं जाएं जहाँ AI एजेंट संरचित ऑर्केस्ट्रेशन पैटर्न के माध्यम से जटिल कार्यों को पूरा करने के लिए सहयोग करते हैं।

## 🎯 सीखने के उद्देश्य

### 🏗️ **वर्कफ़्लो आर्किटेक्चर के मूलभूत सिद्धांत**
- **वर्कफ़्लो बिल्डर**: जटिल बहु-चरणीय AI प्रक्रियाओं को डिजाइन और ऑर्केस्ट्रेट करना
- **एजेंट समन्वय**: वर्कफ़्लो में कई विशेषज्ञ एजेंटों का समन्वय करना
- **Azure OpenAI (Responses API)**: वर्कफ़्लो में Azure OpenAI Responses API का उपयोग करना
- **दृश्य वर्कफ़्लो डिज़ाइन**: बेहतर समझ के लिए वर्कफ़्लो संरचनाओं का सृजन और दृश्य बनाना

### 🔄 **प्रोसेस ऑर्केस्ट्रेशन पैटर्न**
- **अनुक्रमिक प्रसंस्करण**: कई एजेंट कार्यों को तार्किक क्रम में जोड़ना
- **स्थिति प्रबंधन**: वर्कफ़्लो चरणों के बीच संदर्भ और डेटा प्रवाह बनाए रखना
- **त्रुटि हैंडलिंग**: मजबूत त्रुटि पुनर्प्राप्ति और वर्कफ़्लो स्थिरता लागू करना
- **प्रदर्शन अनुकूलन**: एंटरप्राइज-स्तर संचालन के लिए कुशल वर्कफ़्लो डिजाइन करना

### 🏢 **एंटरप्राइज वर्कफ़्लो अनुप्रयोग**
- **व्यवसाय प्रक्रिया स्वचालन**: जटिल संगठनात्मक वर्कफ़्लो स्वचालित करना
- **सामग्री उत्पादन पाइपलाइन**: समीक्षा और अनुमोदन चरणों के साथ संपादकीय वर्कफ़्लो
- **ग्राहक सेवा स्वचालन**: बहु-चरण ग्राहक पूछताछ समाधान
- **डेटा प्रसंस्करण वर्कफ़्लो**: AI-संकायित परिवर्तन के साथ ETL वर्कफ़्लो

## ⚙️ आवश्यकताएँ और सेटअप

```xml

```

### 🔑 **Azure OpenAI कॉन्फ़िगरेशन**

**पर्यावरण सेटअप (.env फ़ाइल):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Azure OpenAI एक्सेस:**
1. Azure पोर्टल में एक Azure OpenAI संसाधन बनाएँ
2. एक मॉडल (उदाहरण के लिए, `gpt-5-mini`) तैनात करें और तैनाती नाम नोट करें
3. `az login` के साथ साइन इन करें और ऊपर दिखाए गए अनुसार पर्यावरण चर कॉन्फ़िगर करें

### 🏗️ **वर्कफ़्लो आर्किटेक्चर अवलोकन**

```mermaid
graph TD
    A[वर्कफ़्लो बिल्डर] --> B[एजेंट रजिस्ट्री]
    B --> C[वर्कफ़्लो निष्पादन इंजन]
    C --> D[Agent 1: सामग्री जनरेटर]
    C --> E[Agent 2: सामग्री समीक्षक] 
    D --> F[वर्कफ़्लो परिणाम]
    E --> F
    G[Azure OpenAI (प्रतिक्रिया API)] --> D
    G --> E
```

**प्रमुख घटक:**
- **WorkflowBuilder**: वर्कफ़्लो डिजाइन के लिए मुख्य ऑर्केस्ट्रेशन इंजन
- **AIAgent**: विशिष्ट क्षमताओं वाले व्यक्तिगत विशेषज्ञ एजेंट
- **Azure OpenAI Client**: Azure OpenAI Responses API एकीकरण
- **Execution Context**: वर्कफ़्लो चरणों के बीच स्थिति और डेटा प्रवाह प्रबंधित करता है

## 🎨 **एंटरप्राइज वर्कफ़्लो डिज़ाइन पैटर्न**

### 📝 **सामग्री उत्पादन वर्कफ़्लो**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **डॉक्यूमेंट प्रोसेसिंग पाइपलाइन**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **व्यवसाय बुद्धिमत्ता वर्कफ़्लो**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **ग्राहक सेवा स्वचालन**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **एंटरप्राइज लाभ**

### 🎯 **विश्वसनीयता और विस्तारशीलता**
- **निर्धारित निष्पादन**: लगातार, दोहराने योग्य वर्कफ़्लो परिणाम
- **त्रुटि पुनर्प्राप्ति**: वर्कफ़्लो के किसी भी चरण में विफलताओं का सुन्दर प्रबंधन
- **प्रदर्शन निगरानी**: निष्पादन मेट्रिक्स और अनुकूलन अवसरों का ट्रैकिंग
- **संसाधन प्रबंधन**: AI मॉडल संसाधनों का कुशल आवंटन और उपयोग

### 🔒 **सुरक्षा और अनुपालन**
- **सुरक्षित प्रमाणीकरण**: `az login` के माध्यम से Microsoft Entra ID प्रमाणीकरण (AzureCliCredential)
- **ऑडिट ट्रेल्स**: वर्कफ़्लो निष्पादन और निर्णय बिंदुओं का पूर्ण लॉगिंग
- **एक्सेस नियंत्रण**: वर्कफ़्लो निष्पादन और निगरानी के लिए सूक्ष्म अनुमतियाँ
- **डेटा गोपनीयता**: वर्कफ़्लो के दौरान संवेदनशील जानकारी का सुरक्षित प्रबंधन

### 📊 **निगरानी और प्रबंधन**
- **दृश्य वर्कफ़्लो डिज़ाइन**: प्रक्रिया प्रवाह और निर्भरताओं का स्पष्ट प्रतिनिधित्व
- **निष्पादन निगरानी**: वर्कफ़्लो प्रगति और प्रदर्शन का वास्तविक समय ट्रैकिंग
- **त्रुटि रिपोर्टिंग**: विस्तृत त्रुटि विश्लेषण और डिबगिंग क्षमताएँ
- **प्रदर्शन विश्लेषिकी**: अनुकूलन और क्षमता नियोजन के लिए मेट्रिक्स

आइए आपका पहला एंटरप्राइज-तैयार AI वर्कफ़्लो बनाएं! 🚀

## 💻 कोड चलाना

पूर्ण कार्यान्वयन `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` में उपलब्ध है। यह फ़ाइल प्रदर्शित करती है:

1. **पर्यावरण कॉन्फ़िगरेशन** - `.env` फ़ाइल से Azure OpenAI कॉन्फ़िगरेशन लोड करना
2. **Azure OpenAI क्लाइंट सेटअप** - Azure OpenAI Responses API का उपयोग करने के लिए क्लाइंट को कॉन्फ़िगर करना
3. **एजेंट निर्माण** - विशेषज्ञ एजेंटों (फ़्रंट डेस्क और कंसीयर्ज) को परिभाषित करना
4. **वर्कफ़्लो बिल्डर** - अनुक्रमिक प्रसंस्करण के साथ एक मल्टी-एजेंट वर्कफ़्लो बनाना
5. **वर्कफ़्लो निष्पादन** - स्ट्रीमिंग परिणामों के साथ वर्कफ़्लो चलाना

### 🚀 उदाहरण चलाना

```bash
# स्क्रिप्ट को निष्पादन योग्य बनाएं (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# वर्कफ़्लो चलाएं
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

या Windows पर:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 अपेक्षित आउटपुट

वर्कफ़्लो निम्न करेगा:
1. आपकी यात्रा गंतव्य अनुरोध स्वीकार करेगा ("मैं पेरिस जाना चाहता हूँ")
2. फ्रंट डेस्क एजेंट प्रारंभिक सिफारिश प्रदान करेगा
3. कंसीयर्ज एजेंट सिफारिश की समीक्षा और सुधार करेगा
4. अंतिम आउटपुट पूरी बातचीत स्ट्रीम को प्रदर्शित करेगा

### 🔧 अनुकूलन

आप वर्कफ़्लो को अनुकूलित कर सकते हैं:
- एजेंट निर्देशों को संशोधित करके उनके व्यवहार को बदलना
- अधिक एजेंट जोड़कर जटिल बहु-चरणीय वर्कफ़्लो बनाना
- विभिन्न परिदृश्यों का परीक्षण करने के लिए उपयोगकर्ता संदेश बदलना
- विभिन्न निष्पादन पैटर्न बनाने के लिए वर्कफ़्लो किनारों को समायोजित करना
