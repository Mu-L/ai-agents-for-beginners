# 🔄 Azure OpenAI (Responses API) के साथ बुनियादी एजेंट वर्कफ़्लो (.NET)

## 📋 वर्कफ़्लो ऑर्केस्ट्रेशन ट्यूटोरियल

यह नोटबुक Microsoft एजेंट फ्रेमवर्क फॉर .NET और Azure OpenAI (Responses API) का उपयोग करके परिष्कृत **एजेंट वर्कफ़्लो** बनाने का तरीका दिखाती है। आप सीखेंगे कि कैसे मल्टी-स्टेप व्यापार प्रक्रियाएं बनाएं जहां AI एजेंट जटिल कार्यों को पूरा करने के लिए संरचित ऑर्केस्ट्रेशन पैटर्न के माध्यम से सहयोग करते हैं।

## 🎯 सीखने के उद्देश्य

### 🏗️ **वर्कफ़्लो आर्किटेक्चर के मूल सिद्धांत**
- **वर्कफ़्लो बिल्डर**: जटिल मल्टी-स्टेप AI प्रक्रियाओं को डिजाइन और ऑर्केस्ट्रेट करें
- **एजेंट समन्वय**: वर्कफ़्लो में कई विशिष्ट एजेंटों का समन्वय करें
- **Azure OpenAI (Responses API)**: वर्कफ़्लो में Azure OpenAI Responses API का उपयोग करें
- **दृश्य वर्कफ़्लो डिज़ाइन**: बेहतर समझ के लिए वर्कफ़्लो संरचनाओं को बनाएं और विज़ुअलाइज़ करें

### 🔄 **प्रक्रिया ऑर्केस्ट्रेशन पैटर्न**
- **अनुक्रमिक प्रक्रिया**: कई एजेंट कार्यों को तार्किक क्रम में जोड़ें
- **स्थिति प्रबंधन**: वर्कफ़्लो चरणों के बीच संदर्भ और डेटा प्रवाह बनाए रखें
- **त्रुटि प्रबंधन**: मजबूत त्रुटि पुनर्प्राप्ति और वर्कफ़्लो स्थिरता लागू करें
- **प्रदर्शन अनुकूलन**: उद्यम-स्तरीय संचालन के लिए कुशल वर्कफ़्लो डिज़ाइन करें

### 🏢 **उद्यम वर्कफ़्लो अनुप्रयोग**
- **व्यवसाय प्रक्रिया स्वचालन**: जटिल संगठनात्मक वर्कफ़्लो को स्वचालित करें
- **सामग्री उत्पादन पाइपलाइन**: समीक्षा और अनुमोदन चरणों के साथ संपादकीय वर्कफ़्लो
- **ग्राहक सेवा स्वचालन**: मल्टी-स्टेप ग्राहक प्रश्न समाधान
- **डेटा प्रसंस्करण वर्कफ़्लोज़**: AI-संचालित रूपांतरण के साथ ETL वर्कफ़्लोज़

## ⚙️ आवश्यकताएँ और सेटअप

```xml

```

### 🔑 **Azure OpenAI कॉन्फ़िगरेशन**

**पर्यावरण सेटअप (.env फ़ाइल):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI पहुँच:**
1. Azure पोर्टल में एक Azure OpenAI संसाधन बनाएं
2. एक मॉडल तैनात करें (उदाहरण के लिए, `gpt-4.1-mini`) और तैनाती नाम नोट करें
3. `az login` से साइन इन करें और उपरोक्त दिखाए अनुसार पर्यावरण चर कॉन्फ़िगर करें

### 🏗️ **वर्कफ़्लो आर्किटेक्चर अवलोकन**

```mermaid
graph TD
    A[वर्कफ़्लो बिल्डर] --> B[एजेंट रजिस्ट्री]
    B --> C[वर्कफ़्लो निष्पादन इंजन]
    C --> D[Agent 1: सामग्री जनरेटर]
    C --> E[Agent 2: सामग्री समीक्षक] 
    D --> F[वर्कफ़्लो परिणाम]
    E --> F
    G[एज़्योर OpenAI (प्रतिक्रिया API)] --> D
    G --> E
```

**मुख्य घटक:**
- **WorkflowBuilder**: वर्कफ़्लो डिज़ाइन करने के लिए मुख्य ऑर्केस्ट्रेशन इंजन
- **AIAgent**: विशिष्ट क्षमताओं वाले व्यक्तिगत विशिष्ट एजेंट
- **Azure OpenAI Client**: Azure OpenAI Responses API एकीकरण
- **Execution Context**: वर्कफ़्लो चरणों के बीच स्थिति और डेटा प्रवाह का प्रबंधन करता है

## 🎨 **उद्यम वर्कफ़्लो डिज़ाइन पैटर्न**

### 📝 **सामग्री उत्पादन वर्कफ़्लो**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **दस्तावेज़ प्रसंस्करण पाइपलाइन**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **व्यवसाय बुद्धिमत्ता वर्कफ़्लो**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **ग्राहक सेवा स्वचालन**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **उद्यम लाभ**

### 🎯 **विश्वसनीयता और मापनीयता**
- **नियतकालिक निष्पादन**: स्थिर, पुनरावृत्त वर्कफ़्लो परिणाम
- **त्रुटि पुनर्प्राप्ति**: किसी भी वर्कफ़्लो चरण में विफलताओं को सौम्यता से प्रबंधित करना
- **प्रदर्शन निगरानी**: निष्पादन मेट्रिक्स और अनुकूलन अवसरों को ट्रैक करना
- **संसाधन प्रबंधन**: AI मॉडल संसाधनों का कुशल आवंटन और उपयोग

### 🔒 **सुरक्षा और अनुपालन**
- **सुरक्षित प्रमाणीकरण**: `az login` के माध्यम से Microsoft Entra ID प्रमाणीकरण (AzureCliCredential)
- **ऑडिट ट्रेल्स**: वर्कफ़्लो निष्पादन और निर्णय बिंदुओं का पूर्ण लॉगिंग
- **प्रवेश नियंत्रण**: वर्कफ़्लो निष्पादन और निगरानी के लिए सूक्ष्म अनुमति
- **डेटा गोपनीयता**: वर्कफ़्लोज़ में संवेदनशील जानकारी का सुरक्षित प्रबंधन

### 📊 **प्रेक्षण योग्यता और प्रबंधन**
- **दृश्य वर्कफ़्लो डिज़ाइन**: प्रक्रिया प्रवाह और निर्भरताओं का स्पष्ट प्रतिनिधित्व
- **निष्पादन निगरानी**: वर्कफ़्लो प्रगति और प्रदर्शन का रियल-टाइम ट्रैकिंग
- **त्रुटि रिपोर्टिंग**: विस्तृत त्रुटि विश्लेषण और डिबगिंग क्षमताएं
- **प्रदर्शन विश्लेषिकी**: अनुकूलन और क्षमता नियोजन के लिए मेट्रिक्स

चलिए आपका पहला उद्यम-तैयार AI वर्कफ़्लो बनाते हैं! 🚀

## 💻 कोड चलाना

पूरी कार्यान्वयन `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` में उपलब्ध है। यह फ़ाइल दिखाती है:

1. **पर्यावरण कॉन्फ़िगरेशन** - `.env` फ़ाइल से Azure OpenAI कॉन्फ़िगरेशन लोड करना
2. **Azure OpenAI क्लाइंट सेटअप** - Azure OpenAI Responses API का उपयोग करने के लिए क्लाइंट को कॉन्फ़िगर करना
3. **एजेंट निर्माण** - विशिष्ट एजेंट परिभाषित करना (फ्रंट डेस्क और कॉन्सर्ज)
4. **वर्कफ़्लो बिल्डर** - अनुक्रमिक प्रक्रिया के साथ एक मल्टी-एजेंट वर्कफ़्लो बनाना
5. **वर्कफ़्लो निष्पादन** - स्ट्रीमिंग परिणामों के साथ वर्कफ़्लो चलाना

### 🚀 उदाहरण चलाना

```bash
# स्क्रिप्ट को निष्पादन योग्य बनाएं (यूनिक्स/लिनक्स/मैकओएस)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# वर्कफ़्लो चलाएँ
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

या विंडोज़ पर:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 अपेक्षित आउटपुट

वर्कफ़्लो:
1. आपकी यात्रा गंतव्य अनुरोध स्वीकार करेगा ("मैं पेरिस जाना चाहूंगा")
2. फ्रंट डेस्क एजेंट एक प्रारंभिक सिफारिश प्रदान करेगा
3. कॉन्सर्ज एजेंट सिफारिश की समीक्षा और संशोधन करेगा
4. अंतिम आउटपुट पूरी बातचीत स्ट्रीम दिखाएगा

### 🔧 अनुकूलन

आप वर्कफ़्लो को अनुकूलित कर सकते हैं:
- एजेंट निर्देशों को संशोधित करके उनके व्यवहार को बदलना
- जटिल मल्टी-स्टेप वर्कफ़्लोज़ बनाने के लिए अधिक एजेंट जोड़ना
- विभिन्न परिदृश्यों का परीक्षण करने के लिए उपयोगकर्ता संदेश बदलना
- विभिन्न निष्पादन पैटर्न बनाने के लिए वर्कफ़्लो एज जोड़ना या समायोजित करना
