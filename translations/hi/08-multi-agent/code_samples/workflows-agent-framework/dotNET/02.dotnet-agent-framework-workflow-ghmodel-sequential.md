# ⏩ Azure OpenAI (Responses API) (.NET) के साथ अनुक्रमिक एजेंट वर्कफ़्लो

## 📋 उन्नत अनुक्रमिक प्रक्रिया ट्यूटोरियल

यह नोटबुक Microsoft Agent Framework for .NET और Azure OpenAI (Responses API) का उपयोग करके **अनुक्रमिक वर्कफ़्लो पैटर्न** प्रदर्शित करता है। आप देखेंगे कि कैसे जटिल, चरण-दर-चरण प्रोसेसिंग पाइपलाइनों का निर्माण किया जाता है जहां एजेंट विशिष्ट क्रम में निष्पादित होते हैं, प्रत्येक चरण पिछली चरण के परिणामों के आधार पर बनता है।

## 🎯 सीखने के उद्देश्य

### 🔄 **अनुक्रमिक प्रोसेसिंग आर्किटेक्चर**
- **रैखिक वर्कफ़्लो डिजाइन**: स्पष्ट निर्भरताओं के साथ चरण-दर-चरण प्रोसेसिंग पाइपलाइनों का निर्माण करें
- **स्थिति प्रबंधन**: अनुक्रमिक वर्कफ़्लो चरणों के माध्यम से संदर्भ और डेटा प्रवाह बनाए रखें
- **Azure OpenAI (Responses API)**: मल्टी-स्टेज .NET वर्कफ़्लोज़ में Azure OpenAI मॉडल्स का लाभ उठाएं
- **उद्यम पाइपलाइन पैटर्न**: उत्पादन-तैयार अनुक्रमिक प्रोसेसिंग सिस्टम बनाएँ

### 🏗️ **उन्नत अनुक्रमिक पैटर्न**
- **स्टेज-गेट प्रोसेसिंग**: वर्कफ़्लो चरणों के बीच सत्यापन चेकपॉइंट्स लागू करें
- **संदर्भ संरक्षण**: सभी चरणों में स्थिति और संचित ज्ञान बनाए रखें
- **त्रुटि संचरण**: अनुक्रमिक प्रोसेसिंग श्रृंखलाओं में विफलताओं को संभवतः संभालें
- **प्रदर्शन अनुकूलन**: न्यूनतम ओवरहेड के साथ कुशल अनुक्रमिक निष्पादन

### 🏢 **उद्यम अनुक्रमिक अनुप्रयोग**
- **दस्तावेज़ प्रोसेसिंग पाइपलाइन**: बहु-चरण दस्तावेज़ विश्लेषण, परिवर्तन और सत्यापन
- **गुणवत्ता आश्वासन वर्कफ़्लोज़**: अनुक्रमिक समीक्षा, सत्यापन, और अनुमोदन प्रक्रियाएं
- **सामग्री उत्पादन पाइपलाइन**: अनुसंधान → लेखन → संपादन → समीक्षा → प्रकाशन
- **व्यावसायिक प्रक्रिया स्वचालन**: स्पष्ट चरण निर्भरताओं वाले बहु-चरण व्यावसायिक वर्कफ़्लोज़

## ⚙️ आवश्यकताएँ और सेटअप

### 📦 **आवश्यक NuGet पैकेज**

.NET अनुक्रमिक वर्कफ़्लोज़ के लिए आवश्यक पैकेज:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

```env

AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**कॉन्फ़िगरेशन प्रबंधन:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");

```

### 🏗️ **अनुक्रमिक वर्कफ़्लो आर्किटेक्चर**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: विश्लेषण एजेंट]
    B --> C[चेकपॉइंट 1]
    C --> D[Stage 2: प्रोसेसिंग एजेंट]
    D --> E[चेकपॉइंट 2]
    E --> F[Stage 3: सत्यापन एजेंट]
    F --> G[अंतिम आउटपुट]
    
    H[स्थिति संदर्भ] --> B
    H --> D
    H --> F

    I --> D
    I --> F
```

**प्रमुख घटक:**
- **अनुक्रमिक एजेंट**: प्रत्येक प्रोसेसिंग चरण के लिए विशेषीकृत एजेंट
- **स्थिति संदर्भ**: चरणों के बीच संचित डेटा और निर्णय बनाए रखता है
- **चेकपॉइंट्स**: गुणवत्ता और निरंतरता सुनिश्चित करने के लिए चरणों के बीच सत्यापन बिंदु
- **Azure OpenAI क्लाइंट**: सभी वर्कफ़्लो चरणों में सुसंगत AI मॉडल एक्सेस

## 🎨 **अनुक्रमिक वर्कफ़्लो डिजाइन पैटर्न**

### 📝 **दस्तावेज़ प्रोसेसिंग पाइपलाइन**
```
Raw Document → Content Extraction → Analysis → Validation → Structured Output
```

### 🎯 **सामग्री निर्माण वर्कफ़्लो**
```
Brief/Requirements → Research → Content Creation → Review → Final Polish
```

### 🔍 **गुणवत्ता आश्वासन पाइपलाइन**
```
Initial Review → Technical Validation → Compliance Check → Final Approval
```

### 💼 **व्यावसायिक खुफिया वर्कफ़्लो**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

## 🏢 **उद्यम अनुक्रमिक लाभ**

### 🎯 **विश्वसनीयता और गुणवत्ता**
- **निर्धारित प्रोसेसिंग**: संरचित चरणों के माध्यम से सुसंगत, पुनरावृत्त परिणाम
- **गुणवत्ता गेट्स**: प्रत्येक चरण पर गुणवत्ता सुनिश्चित करने वाले सत्यापन बिंदु
- **त्रुटि पृथक्करण**: एक चरण की समस्याएं बाद के चरणों तक संचरित नहीं होती हैं
- **जांच ट्रेल्स**: प्रत्येक चरण में निर्णय और परिवर्तनों का संपूर्ण ट्रैकिंग

### 📈 **स्केलेबिलिटी और प्रदर्शन**
- **मॉड्यूलर डिजाइन**: प्रत्येक चरण को स्वतंत्र रूप से अनुकूलित किया जा सकता है
- **संसाधन प्रबंधन**: चरणों के बीच AI मॉडल संसाधनों का कुशल आवंटन
- **स्थिति अनुकूलन**: अनुकूलतम प्रदर्शन के लिए चरणों के बीच न्यूनतम स्थिति स्थानांतरण
- **समानांतर स्टेज समूह**: कई अनुक्रमिक वर्कफ़्लोज़ समानांतर में चल सकते हैं

### 🔒 **सुरक्षा और अनुपालन**
- **चरण-स्तर सुरक्षा**: विभिन्न प्रोसेसिंग चरणों के लिए भिन्न सुरक्षा नीतियां
- **डेटा सत्यापन**: प्रत्येक चेकपॉइंट पर डेटा अखंडता और अनुपालन सुनिश्चित करें
- **प्रवेश नियंत्रण**: विभिन्न वर्कफ़्लो चरणों के लिए सूक्ष्म अनुमति
- **नियामक अनुपालन**: संरचित प्रोसेसिंग के माध्यम से नियामक आवश्यकताओं को पूरा करें

### 📊 **मॉनिटरिंग और विश्लेषण**
- **चरण-स्तर मेट्रिक्स**: प्रत्येक वर्कफ़्लो चरण के लिए प्रदर्शन निगरानी
- **बॉटलनेक पहचान**: धीमे चरणों की पहचान और अनुकूलन
- **गुणवत्ता मेट्रिक्स**: प्रत्येक चरण पर गुणवत्ता और सफलता दर ट्रैक करें
- **प्रक्रिया अनुकूलन**: चरण-स्तर विश्लेषण के आधार पर निरंतर सुधार

चलिए मजबूत अनुक्रमिक AI प्रोसेसिंग पाइपलाइनों का निर्माण करें! 🚀

### 📝 अपेक्षित आउटपुट

वर्कफ़्लो करेगा:
1. **बिक्री एजेंट**: छवि से फर्नीचर आइटम पहचानना और सुझाव प्रदान करना
2. **मूल्य एजेंट**: बजट स्तरों और खरीद सुझावों के साथ विस्तृत मूल्य निर्धारण विश्लेषण जोड़ना
3. **कोट एजेंट**: सभी जानकारी को समेटते हुए एक स्वरूपित कोट दस्तावेज़ उत्पन्न करना

अंतिम आउटपुट एक व्यापक, व्यावसायिक फर्नीचर कोट होगा जो छवि विश्लेषण पर आधारित होगा।

### 🎯 वास्तविक विश्व अनुप्रयोग

यह अनुक्रमिक पैटर्न आदर्श है:
- **ई-कॉमर्स**: उत्पाद विश्लेषण → मूल्य निर्धारण → कोट निर्माण
- **रियल एस्टेट**: संपत्ति विश्लेषण → मूल्यांकन → लिस्टिंग निर्माण
- **बीमा**: दावा विश्लेषण → मूल्यांकन → कोट निर्माण
- **सामग्री निर्माण**: अनुसंधान → लेखन → संपादन → प्रकाशन

### 🔍 स्थिति प्रवाह को समझना

अनुक्रम के प्रत्येक एजेंट को मिलता है:
- **मूल इनपुट**: प्रारंभिक उपयोगकर्ता संदेश (छवि + पाठ)
- **पहले एजेंट आउटपुट**: वार्तालाप इतिहास में पहले सभी एजेंट प्रतिक्रियाएं
- **संचित संदर्भ**: पूरे वर्कफ़्लो में बनाए रखा गया पूर्ण स्थिति

यह परिष्कृत बहु-चरण प्रोसेसिंग को सक्षम बनाता है जहां प्रत्येक एजेंट सभी पिछले चरणों के व्यापक संदर्भ पर आधारित होता है।
