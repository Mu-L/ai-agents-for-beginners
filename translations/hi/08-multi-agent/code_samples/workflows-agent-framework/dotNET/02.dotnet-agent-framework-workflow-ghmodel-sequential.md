# ⏩ Azure OpenAI (Responses API) के साथ क्रमिक एजेंट वर्कफ़्लो (.NET)

## 📋 उन्नत क्रमिक प्रसंस्करण ट्यूटोरियल

यह नोटबुक Microsoft एजेंट फ्रेमवर्क फॉर .NET और Azure OpenAI (Responses API) का उपयोग करके **क्रमिक वर्कफ़्लो पैटर्न** प्रदर्शित करता है। आप सीखेंगे कि कैसे जटिल, चरण-दर-चरण प्रसंस्करण पाइपलाइनों का निर्माण किया जाता है जहाँ एजेंट एक विशिष्ट क्रम में निष्पादित होते हैं, प्रत्येक चरण पिछले चरण के परिणामों पर आधारित होता है।

## 🎯 सीखने के उद्देश्य

### 🔄 **क्रमिक प्रसंस्करण वास्तुकला**
- **रैखिक वर्कफ़्लो डिज़ाइन**: स्पष्ट निर्भरताओं के साथ चरण-दर-चरण प्रसंस्करण पाइपलाइनों का निर्माण
- **स्थिति प्रबंधन**: क्रमिक वर्कफ़्लो चरणों के बीच संदर्भ और डेटा प्रवाह बनाए रखना
- **Azure OpenAI (Responses API)**: .NET के बहु-चरण वर्कफ़्लो में Azure OpenAI मॉडल का उपयोग
- **एंटरप्राइज़ पाइपलाइन पैटर्न**: उत्पादन-तैयार क्रमिक प्रसंस्करण प्रणालियाँ बनाना

### 🏗️ **उन्नत क्रमिक पैटर्न**
- **स्टेज-गेट प्रोसेसिंग**: वर्कफ़्लो चरणों के बीच सत्यापन जांच बिंदु लागू करना
- **संदर्भ संरक्षण**: सभी चरणों में स्थिति और संचित ज्ञान बनाए रखना
- **त्रुटि प्रसार**: क्रमिक प्रसंस्करण श्रृंखलाओं में विफलताओं को सहजता से संभालना
- **प्रदर्शन अनुकूलन**: न्यूनतम ओवरहेड के साथ कुशल क्रमिक निष्पादन

### 🏢 **एंटरप्राइज़ क्रमिक अनुप्रयोग**
- **दस्तावेज़ प्रसंस्करण पाइपलाइन**: मल्टी-चरण दस्तावेज़ विश्लेषण, रूपांतरण, और सत्यापन
- **गुणवत्ता आश्वासन वर्कफ़्लो**: क्रमिक समीक्षा, सत्यापन, और अनुमोदन प्रक्रियाएँ
- **सामग्री उत्पादन पाइपलाइन**: अनुसंधान → लेखन → संपादन → समीक्षा → प्रकाशन
- **व्यवसाय प्रक्रिया स्वचालन**: स्पष्ट चरण निर्भरताओं के साथ बहु-चरण व्यावसायिक वर्कफ़्लो

## ⚙️ पूर्व आवश्यकताएँ और सेटअप

### 📦 **आवश्यक NuGet पैकेज**

.NET क्रमिक वर्कफ़्लो के लिए आवश्यक पैकेज:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Azure OpenAI कॉन्फ़िगरेशन**

**पर्यावरण सेटअप (.env फ़ाइल):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**कॉन्फ़िगरेशन प्रबंधन:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

**मुख्य घटक:**
- **क्रमिक एजेंट**: प्रत्येक प्रसंस्करण चरण के लिए विशेष एजेंट
- **स्थिति संदर्भ**: चरणों के बीच संचित डेटा और निर्णय बनाए रखता है
- **चेकपॉइंट्स**: गुणवत्ता और स्थिरता सुनिश्चित करने के लिए चरणों के बीच सत्यापन बिंदु
- **Azure OpenAI क्लाइंट**: सभी वर्कफ़्लो चरणों में लगातार AI मॉडल पहुंच

## 🎨 **क्रमिक वर्कफ़्लो डिज़ाइन पैटर्न**

```
Raw Document → Content Extraction → Analysis → Validation → Structured Output
```

### 🎯 **सामग्री निर्माण वर्कफ़्लो**
```
Brief/Requirements → Research → Content Creation → Review → Final Polish
```

### 🔍 **गुणवत्ता आश्वासन पाइपलाइन**
```
Initial Review → Technical Validation → Compliance Check → Final Approval
```

### 💼 **व्यवसाय खुफिया वर्कफ़्लो**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

## 🏢 **एंटरप्राइज़ क्रमिक लाभ**

### 🎯 **भरोसेमंदी और गुणवत्ता**
- **निर्धारित प्रसंस्करण**: संरचित चरणों के माध्यम से लगातार, दोहराने योग्य परिणाम
- **गुणवत्ता गेट्स**: प्रत्येक चरण पर गुणवत्ता सुनिश्चित करने वाले सत्यापन बिंदु
- **त्रुटि पृथक्करण**: एक चरण में समस्याएँ अगले चरणों में प्रसारित नहीं होतीं
- **ऑडिट ट्रेल्स**: प्रत्येक चरण में निर्णय और रूपांतरणों का पूर्ण ट्रैकिंग

### 📈 **विपुलता और प्रदर्शन**
- **मॉड्यूलर डिज़ाइन**: प्रत्येक चरण को स्वतंत्र रूप से अनुकूलित किया जा सकता है
- **संसाधन प्रबंधन**: चरणों के बीच AI मॉडल संसाधनों का कुशल आवंटन
- **स्थिति अनुकूलन**: इष्टतम प्रदर्शन के लिए चरणों के बीच न्यूनतम स्थिति स्थानांतरण
- **समानांतर चरण समूह**: कई क्रमिक वर्कफ़्लो समानांतर में चल सकते हैं

### 🔒 **सुरक्षा और अनुपालन**
- **चरण-स्तरीय सुरक्षा**: विभिन्न प्रसंस्करण चरणों के लिए विभिन्न सुरक्षा नीतियाँ
- **डेटा मान्यता**: प्रत्येक चेकपॉइंट पर डेटा अखंडता और अनुपालन सुनिश्चित करें
- **अभिगम नियंत्रण**: विभिन्न वर्कफ़्लो चरणों के लिए सूक्ष्म अनुमतियाँ
- **नियामक अनुपालन**: संरचित प्रसंस्करण के माध्यम से नियामक आवश्यकताओं को पूरा करें

### 📊 **मॉनिटरिंग और विश्लेषण**
- **चरण-स्तरीय मेट्रिक्स**: प्रत्येक वर्कफ़्लो चरण के लिए प्रदर्शन निगरानी
- **बॉटलनेक पहचान**: धीमे चरणों की पहचान और अनुकूलन
- **गुणवत्ता मेट्रिक्स**: प्रत्येक चरण पर गुणवत्ता और सफलता दर ट्रैक करें
- **प्रक्रिया अनुकूलन**: चरण-स्तरीय विश्लेषण पर आधारित सतत सुधार

आइए मजबूत क्रमिक AI प्रसंस्करण पाइपलाइनों का निर्माण करें! 🚀

### 📝 अपेक्षित आउटपुट

वर्कफ़्लो:
1. **बिक्री एजेंट**: छवि से फर्नीचर आइटम पहचानता है और सिफारिशें प्रदान करता है
2. **मूल्य एजेंट**: बजट स्तरों और खरीदारी सिफारिशों के साथ विस्तृत मूल्य विश्लेषण जोड़ता है
3. **कोट एजेंट**: सभी जानकारी को संकलित करते हुए एक स्वरूपित कोट दस्तावेज़ बनाता है

अंतिम आउटपुट छवि विश्लेषण के आधार पर एक व्यापक, पेशेवर फर्नीचर कोट होगा।

### 🎯 वास्तविक दुनिया के अनुप्रयोग

यह क्रमिक पैटर्न आदर्श है:
- **ई-कॉमर्स**: उत्पाद विश्लेषण → मूल्य निर्धारण → कोट निर्माण
- **रियल एस्टेट**: संपत्ति विश्लेषण → मूल्यांकन → सूची निर्माण
- **बीमा**: दावा विश्लेषण → आकलन → कोट निर्माण
- **सामग्री निर्माण**: अनुसंधान → लेखन → संपादन → प्रकाशन

### 🔍 स्थिति प्रवाह की समझ

श्रृंखला में प्रत्येक एजेंट प्राप्त करता है:
- **मूल इनपुट**: प्रारंभिक उपयोगकर्ता संदेश (छवि + पाठ)
- **पिछले एजेंट आउटपुट्स**: वार्तालाप इतिहास में सभी पिछले एजेंट प्रतिक्रियाएँ
- **संचित संदर्भ**: पूरे वर्कफ़्लो में बनाए रखा गया पूर्ण स्थिति

यह उन्नत बहु-चरण प्रसंस्करण सक्षम करता है जहाँ प्रत्येक एजेंट सभी पिछले चरणों के व्यापक संदर्भ पर आधारित होता है।
