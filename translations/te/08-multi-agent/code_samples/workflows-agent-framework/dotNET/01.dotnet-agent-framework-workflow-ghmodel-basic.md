# 🔄 Azure OpenAI (Responses API) (.NET) తో ప్రాథమిక ఏజెంట్ వర్క్‌ఫ్లోలు

## 📋 వర్క్‌ఫ్లో ఆర్కెస్ట్రేషన్ ట్యుటోరియల్

ఈ నోట్‌బుక్ Microsoft Agent Framework for .NET మరియు Azure OpenAI (Responses API) ఉపయోగించి సాఫిస్టికేటెడ్ **ఏజెంట్ వర్క్‌ఫ్లోలు** ఎలా నిర్మించాలో ప్రదర్శిస్తుంది. మీరు బహుళదశ వ్యాపార ప్రక్రియలను సృష్టించడం నేర్చుకుంటారు, ఇక్కడ AI ఏజెంట్లు సంక్లిష్ట పనులను పూర్తిచేసేందుకు క్రమబద్ధీకరించిన ఆర్కెస్ట్రేషన్ నమూనాల ద్వారా కలసికట్టుగా పనిచేస్తారు.

## 🎯 అభ్యాస లక్ష్యాలు

### 🏗️ **వర్క్‌ఫ్లో ఆర్కిటెక్చర్ మౌలికాలు**
- **వర్క్‌ఫ్లో బిల్డర్**: క్లిష్ట బహుళదశ AI ప్రక్రియలను డిజైన్ చేయడం మరియు ఆర్కెస్ట్రేట్ చేయడం
- **ఏజెంట్ సమన్వయము**: వర్క్‌ఫ్లోలలో బహుళ విశిష్ట ఏజెంట్లను సమన్వయం చేయడం
- **Azure OpenAI (Responses API)**: వర్క్‌ఫ్లోల్లో Azure OpenAI Responses API ను ఉపయోగించడం
- **విజువల్ వర్క్‌ఫ్లో డిజైన్**: మెరుగైన అవగాహన కోసం వర్క్‌ఫ్లో గమ్యరూపాలను సృష్టించడం మరియు విజువలైజ్ చేయడం

### 🔄 **ప్రక్రియ ఆర్కెస్ట్రేషన్ నమూనాలు**
- **క్రమబద్ధ ప్రాసెసింగ్**: స్థానిక క్రమంలో బహుళ ఏజెంట్ పనులను వరుసగా కలపడం
- **స్థితి నిర్వహణ**: వర్క్‌ఫ్లో దశల మధ్య సందర్భం మరియు డేటా ప్రవాహాన్ని నిర్వహించడం
- **లోపాల నిర్వహణ**: బలమైన లోప పునరుద్ధరణ మరియు వర్క్‌ఫ్లో ప్రతిఘటనను అమలు చేయడం
- **దక్షత మెరుగుదల**: సంస్థ స్థాయి కార్యకలాపాల కోసం సమర్థవంతమైన వర్క్‌ఫ్లోలను డిజైన్ చేయడం

### 🏢 **సంస్థ వర్క్‌ఫ్లో అప్లికేషన్లు**
- **వ్యాపార ప్రక్రియ ఆటోమేషన్**: క్లిష్ట సాంస్థిక వర్క్‌ఫ్లోలను ఆటోమేట్ చేయడం
- **కంటెంట్ ఉత్పత్తి పైప్‌లైన్**: సమీక్ష మరియు ఆమోద దశలతో సవరణ వర్క్‌ఫ్లోలు
- **కస్టమర్ సేవ ఆటోమేషన్**: బహుళదశ కస్టమర్ విచారణ పరిష్కారం
- **డేటా ప్రాసెసింగ్ వర్క్‌ఫ్లోలు**: AI నడిపే మార్పిడి సహిత ETL వర్క్‌ఫ్లోలు

### 🔑 **Azure OpenAI కాన్ఫిగరేషన్**

**పరిసరాల అమరిక (.env ఫైల్):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI యాక్సెస్:**
1. Azure పోర్టల్‌లో Azure OpenAI వనరును సృష్టించండి
2. ఒక మోడల్‌ను డిప్లాయ్ చేయండి (ఉదాహరణకు, `gpt-4.1-mini`) మరియు డిప్లయ్‌మెంట్ పేరును గమనించండి
3. `az login` తో సైన్-ఇన్ అయి పైగా చూపించినట్లుగా పరిసరాల వేరియబుల్స్‌ని కాన్ఫిగర్ చేయండి

### 🏗️ **వర్క్‌ఫ్లో ఆర్కిటెక్చర్ అవలోకనం**

```mermaid
graph TD
    A[వర్క్‌ఫ్లో బిల్డర్] --> B[ఏజెంట్ రిజిస్ట్రీ]
    B --> C[వర్క్‌ఫ్లో ఎగ్జిక్యూషన్ ఇంజిన్]
    C --> D[Agent 1: కంటెంట్ జనరేటర్]
    C --> E[Agent 2: కంటెంట్ రివ్యూ] 
    D --> F[వర్క్‌ఫ్లో ఫలితాలు]
    E --> F
    G[అజ్యూర్ ఓపెన్‌ఏఐ (ప్రతిస్పందనల API)] --> D
    G --> E
```

**ప్రధాన భాగాలు:**
- **WorkflowBuilder**: వర్క్‌ఫ్లోల డిజైన్ కోసం ప్రధాన ఆర్కెస్ట్రేషన్ ఇంజిన్
- **AIAgent**: ప్రత్యేక సామర్థ్యాలతో వ్యక్తిగత ఏజెంట్లు
- **Azure OpenAI Client**: Azure OpenAI Responses API ఇంటిగ్రేషన్
- **Execution Context**: వర్క్‌ఫ్లో దశల మధ్య స్థితి మరియు డేటా ప్రవాహాన్ని నిర్వహిస్తుంది

## 🎨 **సంస్థ వర్క్‌ఫ్లో డిజైన్ నమూనాలు**

```
User Request → Content Generation → Quality Review → Final Output
```

```
Document Input → Analysis → Extraction → Validation → Structured Output
```

```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **కస్టమర్ సేవ ఆటోమేషన్**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **సంస్థ ప్రయోజనాలు**

### 🎯 **నమ్మకం & స్కేలబిలిటీ**
- **నిర్ణాయక అమలు**: స్థిరమైన, పునరావృత వర్క్‌ఫ్లో ఫలితాలు
- **లోపాల పునరుద్ధరణ**: ఎప్పటి వర్క్‌ఫ్లో దశ లోపాలను సాహజంగా పరిష్కరించడం
- **పనితీరు పరిశీలన**: అమలు ప్రమాణాలు మరియు మెరుగుదల అవకాశాల ట్రాకింగ్
- **వనరుల నిర్వహణ**: AI మోడల్ వనరుల సమర్థవంతమైన కేటాయింపు మరియు వినియోగం

### 🔒 **సెక్యూటి & అనుపాలన**
- **భద్రతా ఆథెంటికేషన్**: `az login` ద్వారా Microsoft Entra ID ఆథెంటికేషన్ (AzureCliCredential)
- **ఆడిట్ ట్రైల్స్**: వర్క్‌ఫ్లో అమలు మరియు నిర్ణయ బिंदువుల పూర్తి లాగింగ్
- **యాక్సెస్ నియంత్రణ**: వర్క్‌ఫ్లో అమలు మరియు పరిశీలనకు కచ్చితమైన అనుమతులు
- **డేటా ప్రైవసీ**: వర్క్‌ఫ్లోలలో సున్నితమైన సమాచారాన్ని భద్రంగా నిర్వహించడం

### 📊 **అవగాహన & నిర్వహణ**
- **విజువల్ వర్క్‌ఫ్లో డిజైన్**: ప్రక్రియ ప్రవాహాలు మరియు ఆధారిత సంబంధాల స్పష్ఠమైన ప్రాతినిధ్యం
- **అమలు పరిశీలన**: వర్క్‌ఫ్లో పురోగతిని మరియు పనితీరును ప్రత్యక్ష సమయ ట్రాకింగ్
- **లోప నివేదిక**: వివరమైన లోప విశ్లేషణ మరియు డీబగ్గింగ్ సామర్థ్యాలు
- **పనితీరు విశ్లేషణలు**: మెరుగుదల మరియు సామర్థ్య ప్రణాళిక కోసం ప్రమాణాలు

మీ మొదటి సంస్థ-ఇన్నది AI వర్క్‌ఫ్లోని నిర్మిద్దాం! 🚀

## 💻 కోడ్ నడిపించడం

పూర్తి అమలు `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` లో లభ్యం. ఈ ఫైలు ఇవి ప్రదర్శిస్తుంది:

1. **పరిసరాల కాన్ఫిగరేషన్** - `.env` ఫైల్ నుండి Azure OpenAI కాన్ఫిగరేషన్ లోడ్ చేయడం
2. **Azure OpenAI క్లయింట్ సెటప్** - Azure OpenAI Responses API ఉపయోగించడానికి క్లయింట్‌ను కాన్ఫిగర్ చేయడం
3. **ఏజెంట్ సృష్టి** - ప్రత్యేక ఏజెంట్లను నిర్వచించడం (Front Desk మరియు Concierge)
4. **వర్క్‌ఫ్లో బిల్డర్** - వరుసప్రక్రియతో బహుళ ఏజెంట్ వర్క్‌ఫ్లో సృష్టించడం
5. **వర్క్‌ఫ్లో అమలును** - స్ట్రిమింగ్ ఫలితాలతో వర్క్‌ఫ్లో నడిపించడం

### 🚀 ఉదాహరణ నడిపించడం

```bash
# స్క్రిప్ట్‌ను అమలు చేయదగినదిగా మార్చండి (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# వర్క్‌ఫ్లోను నడిపించండి
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

లేదా విండోస్‌లో:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 ఆశించిన అవుట్‌పుట్

వర్క్‌ఫ్లో:
1. మీ ప్రయాణ గమ్యం అభ్యర్థన అందుకుంటుంది ("నేను పారిస్‌కు వెళ్లాలనుకుంటున్నాను")
2. Front Desk ఏజెంట్ ప్రారంభ సిఫార్సును అందిస్తుంది
3. Concierge ఏజెంట్ సిఫార్సును సమీక్షించి మెరుగుపరుస్తుంది
4. చివరి అవుట్‌పుట్ పూర్తి సంభాషణ స్ట్రీమ్‌ను చూపిస్తుంది

### 🔧 కస్టమైజేషన్

మీరు వర్క్‌ఫ్లోను కస్టమైజ్ చేయవచ్చు:
- ఏజెంట్ సూచనలను మార్చి వారి ప్రవర్తనను మార్చడం
- క్లిష్ట బహుళదశ వర్క్‌ఫ్లోలు సృష్టించడానికి మరిన్ని ఏజెంట్లను జోడించడం
- వేర్వేరు పరిస్థితులను పరీక్షించడానికి వినియోగదారు సందేశాన్ని మార్చడం
- వేర్వేరు అమలు నమూనాలను సృష్టించడానికి వర్క్‌ఫ్లో ఎడ్జ్‌లను సవరించడం
