# 🔄 Azure OpenAI (Responses API) తో బేసిక్ ఏజెంట్ వర్క్‌ఫ్లోస్ (.NET)

## 📋 వర్క్‌ఫ్లో ఆర్కెస్ట్రేషన్ ట్యుటోరియల్

ఈ నోట్‌బుక్ Microsoft Agent Framework for .NET మరియు Azure OpenAI (Responses API) ఉపయోగించి సాంకీర్తిక **ఏజెంట్ వర్క్‌ఫ్లోస్** ఎలా నిర్మించాలో చూపిస్తుంది. మీరు బహుళదశ వ్యాపార ప్రక్రియలను సృష్టించడం నేర్చుకుంటారు, వాటిలో AI ఏజెంట్లు కాంకీర్గత ఆర్కెస్ట్రేషన్ నమూనాల ద్వారా క్లిష్టమైన పనులను సాధించేందుకు కలిసి పని చేస్తారు.

## 🎯 నేర్చుకునే లక్ష్యాలు

### 🏗️ **వర్క్‌ఫ్లో నిర్మాణం ప్రాథమికాలు**
- **వర్క్‌ఫ్లో బిల్డర్**: క్లిష్టమైన బహుళదశ AI ప్రక్రియలను రూపకల్పన చేసి ఆర్కెస్ట్రేట్ చేయడం
- **ఏజెంట్ సమన్వయం**: వర్క్‌ఫ్లోల్లో బహుళ నిపుణ ఏజెంట్లను సమన్వయపరిచడం
- **Azure OpenAI (Responses API)**: వర్క్‌ఫ్లోల్లో Azure OpenAI Responses API ఉపయోగించడం
- **దృశ్య వర్క్‌ఫ్లో డిజైన్**: మెరుగైన అర్థం కోసం వర్క్‌ఫ్లో కట్టడాలను సృష్టించడం మరియు వీక్షించటం

### 🔄 **ప్రక్రియ ఆర్కెస్ట్రేషన్ నమూనాలు**
- **క్రమబద్ధమైన ప్రాసెసింగ్**: బహుళ ఏజెంట్ పనులను తార్కిక క్రమంలో గొలుపు చేయడం
- **స్థితి నిర్వహణ**: వర్క్‌ఫ్లో దశల మధ్య సందర్భం మరియు డేటా ప్రవాహం నిర్వహణ
- **లోపాల నిర్వహణ**: బలమైన లోప రికవరీ మరియు వర్క్‌ఫ్లో స్థితిస్థాపకత అమలు
- **ప్రदర్బం ప్రతిపాదన**: సంస్థ పరిమాణ కార్యకలాపాల కోసం సమర్ధవంతమైన వర్క్‌ఫ్లో రూపకల్పన

### 🏢 **సంస్థ వర్క్‌ఫ్లో అనువర్తనాలు**
- **వ్యవసాయ ప్రక్రియ ఆటోమేషన్**: క్లుష్టమైన సంస్థ వర్క్‌ఫ్లోలను ఆటోమేట్ చేయడం
- **కంటెంట్ ఉత్పత్తి పైప్‌లైన్**: సమీక్ష మరియు ఆమోద దశలతో సంపాదనా వర్క్‌ఫ్లోలు
- **కస్టమర్ సేవ ఆటోమేషన్**: బహుళ దశ కస్టమర్ విచారణ పరిష్కారం
- **డేటా ప్రాసెసింగ్ వర్క్‌ఫ్లోలు**: AI-శక్తిమంతమైన రూపాంతరం కలిగిన ETL వర్క్‌ఫ్లోలు

## ⚙️ అవసరమైన పదార్థాలు & సెటప్

### 📦 **అవసరమైన NuGet ప్యాకేజీలు**

ఈ వర్క్‌ఫ్లో ప్రదర్శనలో కొన్ని ముఖ్య .NET ప్యాకేజీలు ఉపయోగించబడ్డాయి:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI కాన్ఫిగరేషన్**

**పరిసర సెటప్ (.env ఫైల్):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Azure OpenAI యాక్సెస్:**
1. Azure పోర్టల్‌లో Azure OpenAI వనరు సృష్టించండి
2. మోడల్ (ఉదాహరణకి, `gpt-5-mini`)‌ని మోపండి మరియు మోపింపు పేరును గమనించండి
3. `az login` తో సైన్ ఇన్ అయి పైన చూపినట్లుగా పరిసర వేరియబుల్స్ సెటప్ చేయండి

### 🏗️ **వర్క్‌ఫ్లో నిర్మాణ అవలోకనం**

```mermaid
graph TD
    A[వర్క్‌ఫ్లో బిల్డర్] --> B[ఏజెంట్ రిజిస్ట్రి]
    B --> C[వర్క్‌ఫ్లో ఎగ్జిక్యూషన్ ఇంజిన్]
    C --> D[Agent 1: కంటెంట్ జనరేటర్]
    C --> E[Agent 2: కంటెంట్ రివ్యూయర్] 
    D --> F[వర్క్‌ఫ్లో ఫలితాలు]
    E --> F
    G[అజ్యూర్ ఓపెన్‌ఏఐ (ప్రతిస్పందనలు API)] --> D
    G --> E
```

**ముఖ్య భాగాలు:**
- **WorkflowBuilder**: వర్క్‌ఫ్లో రూపకల్పన కోసం ప్రధాన ఆర్కెస్ట్రేషన్ ఇంజిన్
- **AIAgent**: నిర్దిష్ట సామర్థ్యాలు ఉన్న ప్రత్యేక ఏజెంట్లు
- **Azure OpenAI Client**: Azure OpenAI Responses API ఇంటిగ్రేషన్
- **Execution Context**: వర్క్‌ఫ్లో దశల మధ్య స్థితి మరియు డేటా ప్రవాహాన్ని నిర్వహిస్తుంది

- **పర్ఫార్మెన్స్ మానిటరింగ్**: అమలు మీట్రిక్స్ మరియు మెరుగుదల అవకాశాలు ట్రాక్ చేయడం
- **వనరు నిర్వహణ**: AI మోడల్ వనరుల సమర్ధవంతమైన కేటాయింపు మరియు వినియోగం

- **ప్రాప్తి నియంత్రణ**: వర్క్‌ఫ్లో అమలు మరియు మానిటరింగ్ కోసం సవివర అనుమతులు
- **డేటా గోప్యత**: వర్క్‌ఫ్లోలలో సున్నితమైన సమాచారాన్ని భద్రంగా నిర్వహణ

### 📊 **పరిశీలన & నిర్వహణ**
- **దృశ్య వర్క్‌ఫ్లో డిజైన్**: ప్రక్రియ ధారాలు మరియు ఆధారిత వ్యవస్థల స్పష్టమైన ప్రదర్శన
- **అమలు నిర్వహణ**: వర్క్‌ఫ్లో పురోగతి మరియు ప్రతిఫలాన్ని రియల్-టైమ్ ట్రాకింగ్
- **లోప నివేదిక**: వివరమైన లోప విశ్లేషణ మరియు డీబగ్గింగ్ సామర్థ్యాలు
- **పర్ఫార్మెన్స్ విశ్లేషణ**: మెరుగుదల మరియు సామర్థ్య ప్రణాళిక మీట్రిక్స్

మీరు మీ మొదటి సంస్థ-తోగల AI వర్క్‌ఫ్లోని నిర్మించుకుందాం! 🚀

## 💻 కోడ్ నిర్వహణ

పూర్తి అమలు `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` లో లభిస్తుంది. ఈ ఫైల్ సూచిస్తుంది:

1. **పరిసర సెటప్** - `.env` ఫైల్ నుండి Azure OpenAI కాన్ఫిగరేషన్ లోడ్ చేయడం
2. **Azure OpenAI క్లయింట్ సెటప్** - Azure OpenAI Responses API ఉపయోగించేందుకు క్లయింట్ కాన్ఫిగర్ చేయడం
3. **ఏజెంట్ సృష్టి** - ప్రత్యేక ఏజెంట్లు (ఫ్రంట్ డెస్క్ మరియు కాన్సియర్జ్) నిర్వచనం
4. **వర్క్‌ఫ్లో బిల్డర్** - క్రమబద్ధమైన ప్రాసెసింగ్ తో బహుళ ఏజెంట్ వర్క్‌ఫ్లో సృష్టించడం
5. **వర్క్‌ఫ్లో అమలు** - స్ట్రీమింగ్ ఫలితాలతో వర్క్‌ఫ్లో నడపడం

### 🚀 ఉదాహరణను నడిపించడం

```bash
# స్క్రిప్టును ఎక్సిక్యూటబుల్‌గా మారుస్తుంది (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# వర్క్‌ఫ్లో ను నడపండి
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

లేదా Windowsలో:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 ఎదురుచూసే అవుట్‌పుట్

వర్క్‌ఫ్లో చేస్తుంది:
1. మీ ప్రయాణ గమ్యానికి అభ్యర్థనను స్వీకరిస్తుంది ("నేను ప్యారిస్‌కు వెళ్లాలనుకుంటున్నాను")
2. ఫ్రంట్ డెస్క్ ఏజెంట్ ప్రాథమిక సిఫారసును అందిస్తుంది
3. కాన్సియర్జ్ ఏజెంట్ సిఫారసును సమీక్షించి మెరుగుపరుస్తుంది
4. చివరి అవుట్‌పుట్ పూర్తి సంభాషణ స్ట్రీమ్‌ను ప్రదర్శిస్తుంది

### 🔧 అనుకూలీకరణ

మీరు వర్క్‌ఫ్లోను ఇలా అనుకూలీకరించవచ్చు:
- ఏజెంట్ సూచనలను మార్చి వారి ప్రవర్తనను మార్చడం
