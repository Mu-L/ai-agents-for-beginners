# ⚡ ఆజూర్ ఓపెన్ ఏఐ (రెస్పాన్స్ API) తో సమకాలీన ఏజెంట్ వర్క్‌ఫ్లోలను (.NET) ఉపయోగించుకుని నిర్వహించడం

## 📋 అధిక పనితీరు సమాంతర ప్రాసెసింగ్ పాఠశాల

ఈ నొట్బుక్ Microsoft Agent Framework for .NET మరియు Azure OpenAI (Responses API) ఉపయోగించి **సమకాలీన వర్క్‌ఫ్లో నమూనాలను** చూపిస్తుంది. మీరు అనేక AI ఏజెంట్లను సమకాలీనంగా అమలు చేయడంతో గరిష్ట పార్గా throughput కలిగించే అధిక పనితీరు సమాంతర ప్రాసెసింగ్ వర్క్‌ఫ్లోలను ఎలా రూపొందించాలో నేర్చుకుంటారు, అంతేకాక సమన్వయం మరియు డేటా సంతులనం ఎలా నిర్వహించాలో కూడా.

## 🎯 అభ్యాస లక్ష్యాలు

### 🚀 **సమకాలీన ప్రాసెసింగ్ మౌలికాలు**
- **సమాంతర ఏజెంట్ అమలు**: గరిష్ట పనితీరు కోసం అనేక AI ఏజెంట్లను ఒకేసారి నడపడం
- **Async/Await నమూనాలు**: సమర్థవంతమైన concurrency కోసం .NET యొక్క async ప్రోగ్రామింగ్ మోడల్ ఉపయోగించడం
- **Azure OpenAI (Responses API)**: Azure OpenAI Responses API కు అనేక సమకాలీన కాల్స్ సమన్వయం
- **వనరుల నిర్వహణ**: సమకాలీన ఆపరేషన్లలో AI మోడల్ వనరులను సమర్థవంతంగా నిర్వహించడం

### 🏗️ **ముందునుండి concurrency معماری**
- **టాస్క్ ఆధారిత సమాంతరత**: ఉత్తమ concurrency అమలుకు .NET టాస్క్ ప్యారల్ లైబ్రరీ ఉపయోగించడం
- **సింక్రోనైజేషన్ నమూనాలు**: రేస్ కండిషన్లను నివారిస్తూ concurrency ఏజెంట్లను సమన్వయం చేయడం
- **లోడ్బాలెన్సింగ్**: అందుబాటులో ఉన్న concurrency ప్రాసెసింగ్ సామర్థ్యంపై పనిని సమర్థవంతంగా పంపిణీ చేయడం
- **ఫాల్ట్ టాలరెన్స్**: మొత్తం వర్క్‌ఫ్లోను నిలిపిపోకుండా వ్యక్తిగత ఏజెంట్ విఫలాలను నిర్వహించడం

### 🏢 **ఎంటర్ప్రైజ్ సమకాలీన అనువర్తనాలు**
- **హై వాల్యూమ్ డాక్యుమెంట్ ప్రాసెసింగ్**: అనేక డాక్యుమెంట్లను సమకాలీనంగా ప్రాసెస్ చేయడం
- **రియల్ టైమ్ కంటెంట్ విశ్లేషణ**: వచ్చే డేటా స్ట్రీమ్స్ యొక్క concurrency విశ్లేషణ
- **బ్యాచ్ ప్రాసెసింగ్ ఆప్టిమైజేషన్**: పెద్ద స్థాయి డేటా ప్రాసెసింగ్ ఆపరేషన్లకు throughput గరిష్టం చేయడం
- **మల్టీ మోడల్ విశ్లేషణ**: వివిధ కంటెంట్ రకాల సమాంతర ప్రాసెసింగ్

## 🏢 **ఎంటర్ప్రైజ్ పనితీరు ప్రయోజనాలు**

- **లీనియర్ పనితీరు స్కేలింగ్**: throughput పెంచడానికి ఇంకా ఏజెంట్లను జోడించడం
- **వనరు వినియోగం**: అందుబాటులో ఉన్న AI మోడల్ సామర్థ్యం గరిష్ట సమర్థవంతత
- **ప్రాసెసింగ్ సమయం తగ్గింపు**: సమాంతర అమలుతో గణనీయమైన సమయం తగ్గింపు
- **ఇలాస్టిక్ స్కేలింగ్**: పనిభారం ఆధారంగా concurrency ఏజెంట్ సంఖ్యను గుణాత్మకంగా సర్దుబాటు చేయడం

### 🛡️ **నమ్మకం & మన్నింపు**
- **ఫాల్ట్ ఐసోలేషన్**: వ్యక్తిగత ఏజెంట్ విఫలాలు ఇతర concurrency ఆపరేషన్లపై ప్రభావం చూపవు
- **గ్రేస్ఫుల్ డిగ్రడేషన్**: ఏజెంట్ సామర్థ్యం తగ్గినా సిస్టమ్ పనిచేస్తూనే ఉంటుంది
- **లోపం పునఃప్రయత్నం**: విఫలమైన concurrency ఆపరేషన్లకు ఆటోమేటిక్ రీప్రయత్న యంత్రాలు
- **లోడ్ పంపిణీ**: అందుబాటులో ఉన్న ఏజెంట్లకు పనిని సమానంగా పంపిణీ చేయడం

### 📊 **పనితీరు మానిటరింగ్**
- **సమాంతర అమలు ప్రమాణాలు**: అన్ని సమాంతర ఆపరేషన్ల పనితీరు ట్రాక్ చేయడం
- **వనరు వినియోగ విశ్లేషణ**: CPU, మెమరీ మరియు నెట్‌వర్క్ వినియోగం పర్యవేక్షణ

## 💻 కోడ్ నిర్వహణ

పూర్తి అమలు `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` లో ఉంది. ఈ ఫైల్ ట్రావెల్ ప్లానింగ్ కోసం **ఫ్యాన్-అవుట్/ఫ్యాన్-ఇన్ concurrency వర్క్‌ఫ్లో** చూపిస్తుంది:

### 🏗️ **వర్క్‌ఫ్లో రూపకల్పన**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**ప్రధాన భాగాలు:**

1. **ConcurrentStartExecutor**: యూజర్ అభ్యర్థనని సమాంతరంగా అన్ని ఏజెంట్లకు ప్రసారం చేస్తుంది
2. **రీసర్చర్ ఏజెంట్**: గమ్యస్థానాలు మరియు ఆకర్షణలను concurrency లో విశ్లేషిస్తుంది
3. **ప్లానర్ ఏజెంట్**: సమాంతరంగా సమగ్రమైన ప్రయాణ ప్లాన్లను సృష్టిస్తుంది
4. **ConcurrentAggregationExecutor**: రెండవ ఏజెంట్ల నుండి ఫలితాలను సేకరించి విలీనం చేస్తుంది

### 🎯 **ఫ్యాన్-అవుట్/ఫ్యాన్-ఇన్ నమూనా**

ఈ వర్క్‌ఫ్లో క్లాసిక్ **ఫ్యాన్-అవుట్/ఫ్యాన్-ఇన్** నమూనాను ప్రదర్శిస్తుంది:
- **ఫ్యాన్-అవుట్**: ఒక ఇన్పుట్ సందేశం అనేక ఏజెంట్లకు సమాంతరంగా ప్రసారం చేయబడుతుంది
- **సమాంతర ప్రాసెసింగ్**: అనేక ఏజెంట్లు అదే పనిపై సమాంతరంగా పని చేయడం
- **ఫ్యాన్-ఇన్**: అన్ని ఏజెంట్ల ఫలితాలు సేకరించి ఒకే అవుట్‌పుట్‌లో విలీనం చేయబడతాయి

4. **ఫలితాలు ప్రదర్శన**: అన్ని సమాచారంతో విలీనం చేసిన ప్రయాణ ప్రణాళిక చూపించడం

### 🔧 అనుకూలీకరణ ఎంపికలు

**ఇంకా concurrency ఏజెంట్లను జోడించండి:**
```csharp
// Create additional specialized agents
AIAgent budgetAgent = azureClient.GetOpenAIResponseClient(deployment).CreateAIAgent(
    name: "Budget-Agent", instructions: "Calculate travel costs...");

// Add to fan-out
var workflow = new WorkflowBuilder(startExecutor)
    .AddFanOutEdge(startExecutor, targets: [researcherAgent, plannerAgent, budgetAgent])
    .AddFanInEdge(aggregationExecutor, sources: [researcherAgent, plannerAgent, budgetAgent])
    .WithOutputFrom(aggregationExecutor)
    .Build();

// Update aggregation count
if (this._messages.Count == 3) { ... }
```

**ఏజెంట్ సూచనలను మార్చండి:**
```csharp
const string ResearcherAgentInstructions = "Your custom instructions for research...";
const string PlanAgentInstructions = "Your custom instructions for planning...";
```

**పనిని మార్చండి:**
```csharp
StreamingRun run = await InProcessExecution.StreamAsync(
    workflow, 
    "Plan a European vacation for 2 weeks in summer"
);
```

### 🎯 వాస్తవ ప్రపంచ అనువర్తనాలు

ఈ concurrency నమూనా సరైనది:
- **కంటెంట్ సృష్టి**: అనేక రచయితలు వేర్వేరు విభాగాలు ఒకేసారి సృష్టించడం
- **కోడ్ సమీక్ష**: అనేక సమీక్షకులు వేర్వేరు కోడ్ కోణాలను విశ్లేషించడం
- **మార్కెట్ పరిశోధన**: వేర్వేరు మార్కెట్ విభాగాల సమాంతర విశ్లేషణ
- **డాక్యుమెంట్ ప్రాసెసింగ్**: concurrency లో డాక్యుమెంట్ల నుండి డేటా సేకరణ, విశ్లేషణ మరియు ధ్రువీకరణ
- **బహు-ప్రజ్ఞాపరిశీలన**: ఒకే ఇన్పుట్ పై వేర్వేరు దృక్కోణాలు పొందడం

### 🔍 అనుకూల ఎక్సిక్యూటర్ల అవగాహన

**ConcurrentStartExecutor:**
- `IMessageHandler<string>`ను అమలు చేసి స్ట్రింగ్ ఇన్పుట్ స్వీకరించడం
- అన్ని కనెక్ట్ అయిన ఏజెంట్లకు సందేశాలను ప్రసారం చేస్తుంది
- concurrency ప్రాసెసింగ్ మార్గదర్శకంగా `TurnToken` పంపుతుంది

**ConcurrentAggregationExecutor:**
- ఏజెంట్ ప్రతిస్పందనలు స్వీకరించడానికి `IMessageHandler<ChatMessage>`ను అమలు చేస్తుంది
- థ్రెడ్-సురక్షిత విధానంలో సందేశాలను సేకరిస్తుంది
- అన్ని ఆశించిన ప్రతిస్పందనలు వచ్చినప్పుడు విలీనం చేస్తుంది
- `context.YieldOutputAsync()` ఉపయోగించి తుది ఫలితాన్ని ఇస్తుంది

### ⚡ పనితీరు లాభాలు

**సమాంతర vs అనుక్రమ:**
- అనుక్రమ: Agent1 (30 సెకన్లు) → Agent2 (30 సెకన్లు) = **మొత్తం 60 సెకన్లు**
- సమాంతర: Agent1 (30 సెకన్లు) || Agent2 (30 సెకన్లు) = **మొత్తం 30 సెకన్లు**

**తరగతి పెరుగుదల**: N concurrencyAgentల కోసం N రెట్లు వేగవంతం (పనిభారం, వనరుల ఆధారంగా)

### 🛡️ లోప నిర్వహణ

వర్క్‌ఫ్లో వ్యక్తిగత ఏజెంట్ విఫలాలను సక్సెస్‌గా నిర్వహిస్తుంది:
- ఒక ఏజెంట్ విఫలమైతే, ఇతరులు ప్రాసెసింగ్‌ను కొనసాగిస్తారు
- సమాహారకుడు టైమౌట్ లాజిక్ అమలు చేయవచ్చు
- అవసరమైతే భాగస్వామ్య ఫలితాలు ఇవ్వవచ్చు

### 📊 ఇతర ముఖ్యమైన అంశాలు
