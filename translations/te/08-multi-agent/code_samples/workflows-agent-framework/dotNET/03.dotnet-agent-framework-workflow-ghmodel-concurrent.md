# ⚡ Azure OpenAI (Responses API) తో సమకాలీన ఏజెంట్ వర్క్‌ఫ్లోస్ (.NET)

## 📋 అధిక పనితీరు సమాంతర ప్రాసెసింగ్ ట్యుటోరియల్

ఈ నోట్‌బుక్ Microsoft Agent Framework ఫర్ .NET మరియు Azure OpenAI (Responses API) ఉపయోగించి **సమకాలీన వర్క్‌ఫ్లో నమూనాలు**ను ప్రదర్శిస్తుంది. మీరు అధిక పనితీరు, సమాంతర ప్రాసెసింగ్ వర్క్‌ఫ్లోస్‌ను ఎలా నిర్మించాలో నేర్చుకుంటారు, ఇవి సమాంతరంగా అనేక AI ఏజెంట్లను అమలు చేస్తూ గరిష్ట ఉత్పాదకతను సాధించడం అలాగే సమన్వయం మరియు డేటా సారూప్యతను కొనసాగిస్తుంది.

## 🎯 నేర్చుకునే లక్ష్యాలు

### 🚀 **సమకాలీన ప్రాసెసింగ్ ప్రాథమికాలు**
- **సమాంతర ఏజెంట్ అమలు**: గరిష్ట పనితీర్పుకోసం అనేక AI ఏజెంట్లను సమకాళీనంగా అమలు చేయండి
- **Async/Await నమూనాలు**: సమర్థవంతమైన సమకాలీనానికి .NET async ప్రోగ్రామింగ్ మోడల్‌ను ఉపయోగించండి
- **Azure OpenAI (Responses API)**: Azure OpenAI Responses API కు బహుళ సమకాలీన కాల్స్‌ను సమన్వయించండి
- **వనరుల నిర్వహణ**: సమకాలీన ఆపరేషన్లపై AI మోడల్ వనరులను సమర్ధవంతంగా నిర్వహించండి

- **టాస్క్ ఆధారిత సమాంతర ప్రాసెసింగ్**: ఆప్టిమల్ సమకాలీన అమలుకు .NET Task Parallel Library ఉపయోగించండి
- **సమన్వయ నమూనాలు**: రేస్ కాండిషన్లను నివారిస్తూ సమకాలీన ఏజెంట్లను సమన్వయపరచండి
- **లోడ్ బ్యాలెన్సింగ్**: అందుబాటులో ఉన్న సమాంతర ప్రాసెసింగ్ సామర్థ్యంపై పనిని సమర్థవంతంగా పంపిణీ చేయండి
- **ఫాల్ట్ సహనశక్తి**: మొత్తం వర్క్‌ఫ్లోను ఆపకుండా వ్యక్తిగత ఏజెంట్ వైఫల్యాలను నిర్వహించండి

## 💻 కోడ్ అమలు

పూర్తి అమలు `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` లో అందుబాటులో ఉంది. ఈ ఫైలు ప్రయాణ ప్రణాళిక కోసం **Fan-Out/Fan-In సమకాలీన వర్క్‌ఫ్లో**ను ప్రదర్శిస్తుంది:

### 🏗️ **వర్క్‌ఫ్లో ఆర్కిటెక్చర్**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**ముఖ్య భాగాలు:**

1. **ConcurrentStartExecutor**: వినియోగదారుడి అభ్యర్థనను అన్ని ఏజెంట్లకు ఒకేసారి ప్రసారం చేస్తుంది
2. **Researcher Agent**: గమ్యస్థానాలు మరియు ఆకట్టుకునే ప్రదేశాలను సమాంతరంగా విశ్లేషిస్తుంది
3. **Planner Agent**: సమాంతరంగా వివరిస్తున్న ప్రయాణ ప్రణాళికలను సృష్టిస్తుంది
4. **ConcurrentAggregationExecutor**: రెండు ఏజెంట్ల నుండి ఫలితాలను సేకరించి విలీనం చేస్తుంది

### 🎯 **Fan-Out/Fan-In నమూనా**

ఈ వర్క్‌ఫ్లో క్లాసిక్ **Fan-Out/Fan-In** నమూనాను ప్రదర్శిస్తుంది:
- **Fan-Out**: ఒక ఇన్‌పుట్ సందేశాన్ని అనేక ఏజెంట్లకు సమకాలీనంగా ప్రసారం చేయడం
- **సమకాలీన ప్రాసెసింగ్**: అనేక ఏజెంట్లు ఒకే పని మీద సమాంతరంగా పనిచేస్తాయి
- **Fan-In**: అన్ని ఏజెంట్ల ఫలితాలను సేకరించి ఒక్క output గా విలీనం చేయడం

   - రీసర్చర్ ఆకర్షణలు మరియు వివరాలను గుర్తిస్తుంది
   - ప్లానర్ యాత్రా ప్రణాళిక మరియు లాజిస్టిక్స్ రూపొందిస్తుంది
3. **సమాహారం**: రెండు ప్రతిస్పందనలను సమగ్ర output గా కలపడం
4. **ఫలితాలు ప్రదర్శన**: అన్ని సమాచారం ఉన్న విలీనం చేసిన ప్రయాణ ప్రణాళికను చూపించడం

### 🔧 అనుకూలీకరణ ఎంపికలు

**మరింత సమకాలీన ఏజెంట్లను జోడించండి:**
```csharp
// Create additional specialized agents
AIAgent budgetAgent = azureClient.GetOpenAIResponseClient(deployment).CreateAIAgent(

    .AddFanInEdge(aggregationExecutor, sources: [researcherAgent, plannerAgent, budgetAgent])
    .WithOutputFrom(aggregationExecutor)
    .Build();

// Update aggregation count
if (this._messages.Count == 3) { ... }
```

**ఏజెంట్ సూచనలను మార్చండి:**

### 🎯 వాస్తవ ప్రపంచ అనువర్తనలు

ఈ సమకాలీన నమూనా అనువైనది:
- **కంటెంట్ సృష్టి**: అనేక రచయితలు విభిన్న భాగాలను సమాంతరంగా తయారుచేస్తున్నారు
- **కోడ్ సమీక్ష**: అనేక సమీక్షకులు విభిన్న కోణాల నుండి కోడ్‌ను విశ్లేషిస్తున్నారు
- **మార్కెట్ పరిశోధన**: వేర్వేరు మార్కెట్ విభాగాల సమాంతర విశ్లేషణ
- **డాక్యుమెంట్ ప్రాసెసింగ్**: సమకాలీన ఎక్స్‌ట్రాక్షన్, విశ్లేషణ, మరియు ధృవీకరణ
- **బహుళ-పరిశీలన విశ్లేషణ**: ఒకే ఇన్‌పుట్ పై విభిన్న దృక్కోణాలు పొందడం

### 🔍 అనుకూల ఎగ్జిక్యూటర్ల అవగాహన

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` ను అమలు చేస్తుంది, స్ట్రింగ్ ఇన్‌పుట్ ను స్వీకరించడానికి
- అన్ని కనెక్ట్ అయిన ఏజెంట్లకు సందేశాలు ప్రసారం చేస్తుంది
- సమకాలీన ప్రాసెసింగ్ ప్రారంభించడానికి `TurnToken` పంపుతుంది

**ConcurrentAggregationExecutor:**
- `IMessageHandler<ChatMessage>` ను అమలు చేస్తుంది, ఏజెంట్ ప్రతిస్పందనలను స్వీకరించడానికి
- అజ్ఞాతం రహితం గల విధంగా సందేశాలను సేకరిస్తుంది
- అన్ని ఊహించిన ప్రతిస్పందనలు రాగానే సమాహారం చేస్తుంది
- `context.YieldOutputAsync()` ఉపయోగించి తుది output ను ఇస్తుంది

### ⚡ పనితీరు లాభాలు

**సమకాలీన vs క్రమబద్ధం:**
- క్రమబద్ధం: Agent1 (30s) → Agent2 (30s) = **మొత్తం 60 సెకన్లు**
- సమకాలీన: Agent1 (30s) || Agent2 (30s) = **మొత్తం 30 సెకన్లు**

**థ్రూపుట్ అభివృద్ధి**: N సమకాలీన ఏజెంట్ల కోసం N× వేగవంతమైనది (పనిభారం మరియు వనరులపై ఆధారపడి)
