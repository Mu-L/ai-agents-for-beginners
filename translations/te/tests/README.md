# ఏజెంట్ స్మోక్ టెస్టులు

ఈ ఫోల్డర్ కోర్సులో మీరు నిర్మించే ఏజెంట్ల కోసం **స్మోక్-టెస్టు కాటలాగ్‌లను** కలిగి ఉంది.
స్మోక్ టెస్ట్ అంటే ఒక తక్కువ ఖర్చుతో, వేగంగా జరిగే తనిఖీ, ఇది **డిప్లాయ్ చేయబడిన Microsoft Foundry హోస్టెడ్
ఏజెంట్** అందుబాటులో ఉందా, స్పందిస్తున్నదా, మరియు దాని అత్యంత ప్రాథమిక ప్రాంప్ట్
ఆశించిన అనుసరిస్తున్నదా అనే విషయాన్ని చెక్ చేస్తుంది. ఇది మొదటి గేట్ — మీరు [పాఠం 10](../10-ai-agents-production/README.md) మరియు
[పాఠం 16](../16-deploying-scalable-agents/README.md) లో నేర్పబడే పూర్తయిన మూల్యాంకన
పైప్‌లైన్‌కి ప్రతిపాదన కాదు.

ఈ కాటలాగ్‌లను [AI Smoke Test](https://github.com/marketplace/actions/ai-smoke-test)
GitHub యాక్షన్ ద్వారా [`.github/workflows/smoke-test.yml`](../../../.github/workflows/smoke-test.yml)
వర్క్‌ఫ్లో ద్వారా వినియోగిస్తారు.

## ఎలా నడపాలి

1. Microsoft Foundryలో పాఠ్యపు ఏజెంట్‌ను హోస్టెడ్ ఏజెంట్‌గా డిప్లాయ్ చేయండి (డిప్లాయ్‌మెంట్ వర్క్‌ఫ్లో కోసం
   పాఠం 16 చూడండి). **ఏజెంట్ పేరు** మరియు మీ
   **Foundry ప్రాజెక్ట్ ఎండ్‌పాయింట్** గుర్తుంచుకోండి.
2. ఈ రిపాజిటరీ సీక్రెట్లను చేర్చండి (సెట్టింగ్స్ → సీక్రెట్స్ మరియు వేరియబుల్స్ → యాక్షన్స్):
   `AZURE_CLIENT_ID`, `AZURE_TENANT_ID`, `AZURE_SUBSCRIPTION_ID`. ఫెడరేటెడ్
   ఐడెంటిటీకి **Azure AI User** పాత్ర **Foundry ప్రాజెక్ట్ స్కోప్‌లో** అవసరం.
3. **యాక్షన్స్** ట్యాబ్ నుండి, **Smoke-test hosted agents** ని నడిపి పాఠానికి అనుగుణమైన
   `tests_file` ఎంచుకొని, తరువాత సరిపోయే `agent_name` మరియు
   `project_endpoint` అందించండి.

## ఏ పాఠాలకు స్మోక్ టెస్టులు ఉన్నాయి?

స్మోక్ టెస్టులు మీరు **ఏజెంట్‌ను డిప్లాయ్ చేసే** పాఠాలకు వర్తిస్తాయి, దాని టెక్స్ట్ ప్రతిస్పందనను
తెలిసిన కంటెంట్‌తో సరిపోల్చి నిర్ధారించగలిగే. సిద్ధాంతపరమైన, కేవలం స్థానికంగా నడిచే,
లేదా క్రియేటివ్, అస్పష్ట అవుట్‌పుట్ ఉత్పత్తిచేయు పాఠాలు ఉద్దేశపూర్వకంగా తప్పించబడ్డాయి:

- **పాఠం 17 (స్థానిక AI ఏజెంట్లు సృష్టించడం)** పూర్తిగా మీ వర్క్‌స్టేషన్‌పై Foundry Localతో నడుస్తుంది మరియు
  Foundry Responses ఎండ్‌పాయింట్‌ను **ప్రదర్శించదు**, కాబట్టి ఈ
  యాక్షన్ వర్తించదు. స్థానికంగా నోట్‌బుక్ నడిపి ధృవీకరించండి.
- డిజైన్-ప్యాట్రన్ మరియు సిద్ధాంత పాఠాలు (02, 03, 06, 07, 09, 12) ఏజెంట్‌ను
  ఒకటిగా కూడా స్మోక్-టెస్ట్‌కు పంపించవు.

## కాటలాగ్ స్కీమా (త్వరిత సూచన)

ప్రతి కాటలాగ్ ఒక JSON డాక్యుమెంట్, ఇది టాప్-లెవెల్ `tests` అర్రే కలిగి ఉంటుంది. ప్రతి ఎంట్రీ ఒక ప్రాంప్ట్‌ని పోస్ట్ చేసి
ప్రతిస్పందనపై నిర్ధారణ చేస్తుంది:

| ఫీల్డ్ | అర్థం |
|-------|---------|
| `id` | లాగ్‌లో ముద్రించబడే ప్రత్యేక మెట్లు గుర్తింపు. |
| `description` | మానవ పాఠకులకు అర్థమయ్యే ఉద్దేశ్యం. |
| `prompt` | ఏజెంట్‌కు పంపబడే సందేశం. |
| `assertions.status` | ఆశించిన HTTP స్థితి (డిఫాల్ట్ 200). |
| `assertions.contains_any` | ప్రతిస్పందనలో వీటిలో ఏదైనా ఉపసరగతలు ఉంటే పాస్. |
| `assertions.contains_all` | ప్రతిస్పందనలో ప్రతీ ఉపసరగత కూడా ఉంటే పాస్. |
| `assertions.contains_none` | ప్రతిస్పందనలో వీటిలో ఏదీ లేకపోతే పాస్. |
| `save_response_id_as` | తర్వాతి మల్టీ-టర్న్ మెటలో ఉపయోగించడానికి ప్రతిస్పందన idని నిల్వ చేయండి. |
| `use_previous_response_id` | ఈ టర్న్‌ను భద్రపరుచిన ప్రతిస్పందన idతో జతచేయండి. |

నిర్ధారణలు కేసు-అస్పష్ట ఉపసరగత తనిఖీలు. పూర్తి స్కీమా సహా,
[యాక్షన్ డాక్యుమెంటేషన్](https://github.com/marketplace/actions/ai-smoke-test) చూడండి,
దీని లోపల Foundry-నియంత్రిత సంభాషణ వనరులు ఉన్నాయి.
