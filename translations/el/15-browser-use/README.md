# Δημιουργία Πρακτόρων Χρήσης Υπολογιστή (CUA)

Οι πράκτορες χρήσης υπολογιστή μπορούν να αλληλεπιδρούν με ιστοσελίδες με τον ίδιο τρόπο που θα το έκανε ένα άτομο: ανοίγοντας έναν περιηγητή, εξετάζοντας τη σελίδα και λαμβάνοντας την επόμενη καλύτερη ενέργεια με βάση όσα βλέπουν. Σε αυτό το μάθημα, θα δημιουργήσετε έναν πράκτορα αυτοματοποίησης περιηγητή που αναζητά στο Airbnb, εξάγει δομημένα δεδομένα καταχωρίσεων και εντοπίζει τη φθηνότερη διαμονή στη Στοκχόλμη.

Το μάθημα συνδυάζει το Browser-Use για πλοήγηση με AI-driven, το Playwright και το Chrome DevTools Protocol (CDP) για έλεγχο περιηγητή, το Azure OpenAI για οπτικά ενεργοποιημένη συλλογιστική και το Pydantic για δομημένη εξαγωγή.

## Εισαγωγή

Αυτό το μάθημα θα καλύψει:

- Κατανόηση πότε οι πράκτορες χρήσης υπολογιστή είναι καλύτερη επιλογή από αυτοματοποίηση μόνο με API
- Συνδυασμός Browser-Use με Playwright και CDP για αξιόπιστη διαχείριση κύκλου ζωής περιηγητή
- Χρήση Azure OpenAI vision και δομημένης εξόδου Pydantic για εξαγωγή δεδομένων καταχωρίσεων από δυναμικές ιστοσελίδες
- Απόφαση πότε να χρησιμοποιηθεί ροή εργασίας αυτοματοποίησης περιηγητή με πρώτο πράκτορα, πρώτο ηθοποιό ή υβριδική

## Στόχοι Εκμάθησης

Μετά την ολοκλήρωση αυτού του μαθήματος, θα ξέρετε πώς να:

- Ρυθμίσετε το Browser-Use με Azure OpenAI και Playwright
- Δημιουργήσετε μια ροή εργασίας αυτοματοποίησης περιηγητή που πλοηγείται σε πραγματική ιστοσελίδα και χειρίζεται δυναμικά στοιχεία UI
- Εξάγετε τυποποιημένα αποτελέσματα από ορατό περιεχόμενο σελίδας και να τα μετατρέψετε σε επιχειρηματική λογική
- Επιλέξετε μεταξύ προτύπων πράκτορα και ηθοποιού με βάση το πόσο προβλέψιμο είναι το έργο του περιηγητή

## Παράδειγμα Κώδικα

Αυτό το μάθημα περιλαμβάνει ένα σημειωματάριο διδασκαλίας:

- [15-browser-user.ipynb](./15-browser-user.ipynb): Εκκινεί μια συνεδρία Chrome μέσω CDP, αναζητά καταχωρίσεις Airbnb για τη Στοκχόλμη, εξάγει τιμές με όραση Browser-Use, και επιστρέφει την φθηνότερη επιλογή ως δομημένα δεδομένα.

## Προαπαιτούμενα

- Python 3.12+
- Διαμόρφωση ανάπτυξης Azure OpenAI στο περιβάλλον σας
- Chrome ή Chromium εγκατεστημένο τοπικά
- Εγκατάσταση εξαρτημάτων Playwright
- Βασική εξοικείωση με async Python

## Ρύθμιση

Εγκαταστήστε τα πακέτα που χρησιμοποιούνται στο σημειωματάριο:

```bash
pip install browser_use playwright python-dotenv
playwright install chromium
```

Ορίστε τις μεταβλητές περιβάλλοντος Azure OpenAI που χρησιμοποιεί το σημειωματάριο:

```bash
AZURE_OPENAI_ENDPOINT=...
AZURE_OPENAI_API_KEY=...
AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=...
# Προαιρετικό: χρησιμοποιεί την πιο πρόσφατη έκδοση API όταν παραλείπεται
AZURE_OPENAI_API_VERSION=...
```

## Επισκόπηση Αρχιτεκτονικής

Το σημειωματάριο παρουσιάζει μια υβριδική ροή εργασίας αυτοματοποίησης περιηγητή:

1. Ο Chrome ξεκινά με ενεργοποιημένο CDP ώστε και το Playwright και το Browser-Use να μοιράζονται την ίδια συνεδρία περιηγητή.
2. Ένας πράκτορας Browser-Use χειρίζεται ανοιχτού τύπου εργασίες πλοήγησης όπως το άνοιγμα του Airbnb, το κλείσιμο αναδυόμενων παραθύρων και την αναζήτηση για Στοκχόλμη.
3. Η τρέχουσα σελίδα εξετάζεται με ένα δομημένο σχήμα Pydantic για να εξαχθούν τίτλοι καταχωρίσεων, νυχτερινές τιμές, αξιολογήσεις και διευθύνσεις URL.
4. Η λογική Python συγκρίνει τις εξαγόμενες καταχωρίσεις και επισημαίνει το φθηνότερο αποτέλεσμα.

Αυτή η προσέγγιση διατηρεί την ευέλικτη, βασισμένη στην όραση συλλογιστική που έχει το Browser-Use ενώ παράλληλα παρέχει ντετερμινιστικό έλεγχο περιηγητή όταν χρειάζεται.

## Βασικά Συμπεράσματα και Καλές Πρακτικές

### Πότε να χρησιμοποιήσετε Πράκτορα έναντι Ηθοποιού

| Σενάριο | Χρησιμοποιήστε Πράκτορα | Χρησιμοποιήστε Ηθοποιό |
|----------|-----------|-----------|
| Δυναμικές διατάξεις | Ναι, το AI προσαρμόζεται καλά σε αλλαγές σελίδας | Όχι, εύθραυστοι επιλογείς μπορούν να σπάσουν |
| Γνωστή δομή | Όχι, ένας πράκτορας είναι πιο αργός από τον άμεσο έλεγχο | Ναι, γρήγορος και ακριβής |
| Εύρεση στοιχείων | Ναι, η φυσική γλώσσα λειτουργεί καλά | Όχι, απαιτούνται ακριβείς επιλογείς |
| Έλεγχος χρονοδιαγράμματος | Όχι, λιγότερο προβλέψιμο | Ναι, πλήρης έλεγχος σε αναμονές και επαναλήψεις |
| Πολύπλοκες ροές εργασίας | Ναι, χειρίζεται απροσδόκητες καταστάσεις UI | Όχι, απαιτεί ρητή διακλάδωση |

### Καλές Πρακτικές Browser-Use

1. Ξεκινήστε με έναν πράκτορα για εξερεύνηση και δυναμική πλοήγηση.
2. Μεταβείτε σε άμεσο έλεγχο σελίδας όταν η αλληλεπίδραση γίνει προβλέψιμη.
3. Χρησιμοποιήστε δομημένα πρότυπα εξόδου ώστε τα εξαγόμενα δεδομένα να είναι επικυρωμένα και ασφαλή ως προς τους τύπους.
4. Προσθέστε καθυστερήσεις στρατηγικά μετά από ενέργειες που προκαλούν ορατές αλλαγές UI.
5. Καταγράψτε στιγμιότυπα οθόνης κατά την επανάληψη ώστε να είναι ευκολότερη η αποσφαλμάτωση αποτυχιών.
6. Αναμένετε αλλαγές στις ιστοσελίδες και σχεδιάστε στρατηγικές εκπτώσεων για αναδυόμενα παράθυρα και αλλαγές διάταξης.
7. Συνδυάστε τα πρότυπα πράκτορα και ηθοποιού για να έχετε τόσο ευελιξία όσο και ακρίβεια.

### Ασφαλιστικές Ράγες για Πράκτορες Περιηγητών

Οι πράκτορες περιηγητή λειτουργούν σε ζωντανές ιστοσελίδες, γι' αυτό έχουν αυστηρότερα όρια από ένα σενάριο που καλεί μόνο ένα γνωστό API. Πριν περάσετε από επίδειξη σημειωματαρίου σε πραγματική ροή εργασίας, ορίστε τους ελέγχους γύρω από το τι μπορεί να δει, να κάνει κλικ και να υποβάλει ο πράκτορας.

1. **Περιορίστε το περιβάλλον περιήγησης.** Τρέξτε τον πράκτορα σε ειδικό προφίλ περιηγητή ή sandbox και περιορίστε τον μόνο στους τομείς που απαιτούνται για την εργασία.
2. **Διαχωρίστε την παρατήρηση από την ενέργεια.** Αφήστε τον πράκτορα να αναζητά, να διαβάζει και να εξάγει δεδομένα πρώτα. Απαιτήστε ρητό βήμα έγκρισης πριν υποβάλει φόρμες, στείλει μηνύματα, κλείσει ταξίδια, κάνει αγορές, διαγράψει εγγραφές ή αλλάξει ρυθμίσεις λογαριασμού.
3. **Κρατήστε μυστικά έξω από προτροπές και ίχνη.** Μην τοποθετείτε κωδικούς, στοιχεία πληρωμών, cookies συνεδρίας ή ακατέργαστα προσωπικά δεδομένα στο πλαίσιο του μοντέλου. Αφήστε τον χρήστη να αναλάβει το συνδρομές και να αποκρύψει ευαίσθητα πεδία από τα αρχεία καταγραφής.
4. **Θεωρήστε το περιεχόμενο της σελίδας ως μη αξιόπιστη είσοδο.** Μια ιστοσελίδα μπορεί να περιέχει οδηγίες που απευθύνονται στον πράκτορα, όχι στον χρήστη. Ο πράκτορας πρέπει να αγνοεί κείμενο που του ζητά να αλλάξει τον στόχο του, να αποκαλύψει δεδομένα, να απενεργοποιήσει προστασίες ή να επισκεφτεί άσχετες σελίδες.
5. **Χρησιμοποιήστε ντετερμινιστικούς ελέγχους γύρω από ριψοκίνδυνα βήματα.** Επαληθεύστε το τρέχον URL, τον τίτλο της σελίδας, το επιλεγμένο στοιχείο, την τιμή, τον παραλήπτη και την περίληψη ενέργειας με κώδικα πριν ζητήσετε από τον χρήστη να εγκρίνει το τελικό βήμα.
6. **Ορίστε προϋπολογισμούς και συνθήκες διακοπής.** Περιορίστε τον αριθμό ενεργειών, επαναλήψεων, καρτελών και λεπτών που μπορεί να χρησιμοποιήσει ο πράκτορας. Σταματήστε όταν η κατάσταση της σελίδας είναι ασαφής αντί να συνεχίσετε να κάνετε κλικ.
7. **Καταγράψτε χρήσιμες αποδείξεις, όχι τα πάντα.** Κρατήστε περιλήψεις ενεργειών, χρονικές σημάνσεις, URLs, περιγραφές επιλεγμένων στοιχείων και αναφορές στιγμιοτύπων οθόνης ώστε οι αποτυχίες να μπορούν να εξεταστούν χωρίς να αποθηκεύεται περιττό ευαίσθητο περιεχόμενο σελίδας.

Στο παράδειγμα του Airbnb, η ασφαλής προεπιλογή είναι να αναζητούνται καταχωρίσεις και να εξάγονται οι τιμές. Η σύνδεση, η επικοινωνία με οικοδεσπότη ή η ολοκλήρωση κράτησης πρέπει να είναι ξεχωριστές ενέργειες με έγκριση χρήστη.

### Εφαρμογές στον Πραγματικό Κόσμο

- Κράτηση ταξιδιού και παρακολούθηση τιμών
- Σύγκριση τιμών και έλεγχοι διαθεσιμότητας σε ηλεκτρονικό εμπόριο
- Δομημένη εξαγωγή από δυναμικές ιστοσελίδες
- Οπτικά ευαίσθητη δοκιμή και επαλήθευση UI
- Παρακολούθηση και ειδοποίηση ιστοσελίδας
- Έξυπνη συμπλήρωση φορμών σε ροές πολλαπλών βημάτων

## Παράδειγμα σε Πραγματικό Κόσμο: Microsoft Project Opal

Ο πράκτορας που δημιουργείτε σε αυτό το μάθημα είναι μια μικρή, τοπική έκδοση ενός **πράκτορα χρήσης υπολογιστή (CUA)** — ένα πρόγραμμα που χειρίζεται έναν περιηγητή όπως ένας άνθρωπος. Η Microsoft φέρνει αυτή την ίδια ιδέα στην επιχείρηση με το **[Project Opal (Frontier)](https://support.microsoft.com/en-us/microsoft-365-copilot/get-started-with-project-opal-frontier)**, μια δυνατότητα στο Microsoft 365 Copilot.

Με το Project Opal, περιγράφετε μια εργασία και ο πράκτορας εργάζεται εκ μέρους σας χρησιμοποιώντας **χρήση υπολογιστή σε ασφαλές Windows 365 Cloud PC**, λειτουργώντας σε όλες τις εφαρμογές, τις τοποθεσίες και τα δεδομένα της οργάνωσής σας που βασίζονται σε περιηγητή. Λειτουργεί **ασύγχρονα στο παρασκήνιο**, και μπορείτε να καθοδηγήσετε τη δουλειά ή να αναλάβετε τον έλεγχο οποιαδήποτε στιγμή. Παραδείγματα εργασιών περιλαμβάνουν:

- Διαχείριση αιτημάτων συμμετοχής σε ομάδες ασφαλείας
- Συλλογή και επικύρωση αποδεικτικών στοιχείων ελέγχου για συμμόρφωση
- Διαχείριση συμβάντων IT (ενημέρωση κατάστασης εισιτηρίου, ανάθεση υπευθύνων, κλείσιμο διπλότυπων)
- Σύνταξη δεδομένων Excel σε οικονομική παρουσίαση κλεισίματος

Το Opal είναι μια χρήσιμη αναφορά για το πώς μοιάζει ένας **πρακτικός, αξιόπιστος πράκτορας χρήσης υπολογιστή** — και ενισχύει έννοιες από προηγούμενα μαθήματα:

| Έννοια σε αυτό το μάθημα | Πώς το Project Opal το εφαρμόζει |
|------------------------|-----------------------------|
| **Άνθρωπος στη βρόχο** (Μάθημα 06) | Το Opal σταματά για διαπιστευτήρια σύνδεσης, ευαίσθητα δεδομένα ή ασαφείς οδηγίες, και ποτέ δεν εισάγει κωδικούς ή δεν υποβάλλει φόρμες χωρίς ρητή επιβεβαίωση. Μπορείτε να *αναλάβετε τον έλεγχο* και να *επιστρέψετε τον έλεγχο* στη μέση μιας εργασίας. |
| **Αξιόπιστοι & ασφαλείς πράκτορες** (Μαθήματα 06 και 18) | Τρέχει σε απομονωμένο Windows 365 Cloud PC, είναι μόνο περιηγητής από προεπιλογή (απαγόρευση άλλης πρόσβασης σε υπολογιστή μέσω Intune), χρησιμοποιεί *τη δική σας* ταυτότητα ώστε να έχει πρόσβαση μόνο σε αυτά που είστε εξουσιοδοτημένοι, και καταγράφει κάθε ενέργεια για έλεγχο. |
| **Σχεδιασμός & μεταγνωστική λειτουργία** (Μαθήματα 07 και 09) | Το Opal δημιουργεί αρχικά ένα πλάνο για την εργασία, στη συνέχεια επιβλέπει τη συλλογιστική του σε κάθε βήμα και σταματά αν ανιχνεύσει ύποπτη δραστηριότητα. |
| **Επαναχρησιμοποιήσιμες δυνατότητες / εργαλεία** (Μάθημα 04) | Οι **Δεξιότητες** σας επιτρέπουν να γράψετε οδηγίες για επαναλαμβανόμενες εργασίες (εισαγόμενες από αρχείο `.md` ή δημιουργημένες με το Opal) και να τις επαναχρησιμοποιήσετε σε συζητήσεις. |

> **Διαθεσιμότητα:** Το Project Opal είναι προς το παρόν διαθέσιμο σε χρήστες στο [πρόγραμμα πρώιμης πρόσβασης Frontier](https://adoption.microsoft.com/copilot/frontier-program/) με συνδρομή Microsoft 365 Copilot, και ο διαχειριστής σας πρέπει να ολοκληρώσει τη ρύθμιση. Επειδή είναι πειραματική δυνατότητα Frontier, οι λειτουργίες μπορεί να αλλάξουν με την πάροδο του χρόνου.
