# Δοκιμές Καπνίσματος Πρακτόρων

Αυτός ο φάκελος περιέχει **καταλόγους δοκιμών καπνίσματος** για τους πράκτορες που δημιουργείτε στο μάθημα.
Μια δοκιμή καπνίσματος είναι ένας φθηνός, γρήγορος έλεγχος ότι ένας **αναπτυγμένος πράκτορας φιλοξενούμενος στο Microsoft Foundry**
είναι προσβάσιμος, ανταποκρίνεται και ακολουθεί τις πιο βασικές προσδοκίες προτροπής του.
Είναι η πρώτη πύλη — όχι υποκατάστατο για ολόκληρη την αξιολόγηση
τη ροή εργασίας που μαθαίνετε στο [Μάθημα 10](../10-ai-agents-production/README.md) και
[Μάθημα 16](../16-deploying-scalable-agents/README.md).

Οι κατάλογοι καταναλώνονται από το [AI Smoke Test](https://github.com/marketplace/actions/ai-smoke-test)
GitHub Action μέσω της ροής εργασίας [`.github/workflows/smoke-test.yml`](../../../.github/workflows/smoke-test.yml).


## Πώς να εκτελέσετε

1. **Αναπτύξτε τον πράκτορα του μαθήματος** στο Microsoft Foundry ως φιλοξενούμενο πράκτορα (δείτε
   το Μάθημα 16 για τη ροή εργασιών ανάπτυξης). Σημειώστε το **όνομα πράκτορα** και το
   **τελικό σημείο έργου Foundry**.
2. Προσθέστε αυτά τα μυστικά αποθετηρίων (Ρυθμίσεις → Μυστικά και μεταβλητές → Ενέργειες):
   `AZURE_CLIENT_ID`, `AZURE_TENANT_ID`, `AZURE_SUBSCRIPTION_ID`. Η ομοσπονδιακή
   ταυτότητα χρειάζεται τον ρόλο **Azure AI User** στο **πεδίο έργου Foundry**.
3. Από την καρτέλα **Ενέργειες**, εκτελέστε **Δοκιμές καπνίσματος φιλοξενούμενων πρακτόρων** και επιλέξτε το
   `tests_file` για το μάθημα, στη συνέχεια δώστε το αντίστοιχο `agent_name` και
   `project_endpoint`.

## Ποια μαθήματα έχουν δοκιμές καπνίσματος;

Οι δοκιμές καπνίσματος εφαρμόζονται σε μαθήματα όπου **αναπτύσσετε έναν πράκτορα** του οποίου οι απαντήσεις κειμένου μπορούν
να θεωρηθούν έναντι γνωστού περιεχομένου. Τα μαθήματα που είναι εννοιολογικά, εκτελούνται μόνο τοπικά,
ή παράγουν μη ντετερμινιστική δημιουργική έξοδο εξαιρούνται σκόπιμα:

- Το **Μάθημα 17 (Δημιουργία Τοπικών Πρακτόρων Τεχνητής Νοημοσύνης)** εκτελείται ολόκληρο στο σταθμό εργασίας σας με
  Foundry Local και **δεν** εκθέτει τελικό σημείο Απαντήσεων Foundry, οπότε αυτή
  η ενέργεια δεν εφαρμόζεται. Επιβεβαιώστε το εκτελώντας το σημειωματάριο τοπικά.
- Τα μαθήματα σχεδίασης προτύπων και θεωρίας (02, 03, 06, 07, 09, 12) δεν περιλαμβάνουν κανέναν
  αναπτυσσόμενο πράκτορα για δοκιμή καπνίσματος.

## Σχήμα καταλόγου (γρήγορη αναφορά)

Κάθε κατάλογος είναι ένα έγγραφο JSON με έναν πίνακα `tests` κορυφαίου επιπέδου. Κάθε εγγραφή στέλνει
μια προτροπή και κάνει έλεγχο στην απάντηση:

| Πεδίο | Σημασία |
|-------|---------|
| `id` | Μοναδικός αναγνωριστικός βήματος που εκτυπώνεται στο αρχείο καταγραφής. |
| `description` | Ανθρώπινα αναγνώσιμος σκοπός. |
| `prompt` | Το μήνυμα που αποστέλλεται στον πράκτορα. |
| `assertions.status` | Αναμενόμενη κατάσταση HTTP (προεπιλογή 200). |
| `assertions.contains_any` | Επιτυχία αν η απάντηση περιέχει οποιαδήποτε από αυτές τις υποσυμβολοσειρές. |
| `assertions.contains_all` | Επιτυχία αν η απάντηση περιέχει κάθε υποσυμβολοσειρά. |
| `assertions.contains_none` | Επιτυχία αν η απάντηση δεν περιέχει καμία από αυτές τις υποσυμβολοσειρές. |
| `save_response_id_as` | Αποθήκευση του αναγνωριστικού απάντησης για αργότερο βήμα πολλαπλών γύρων. |
| `use_previous_response_id` | Αποστολή αυτής της στροφής σε αλυσίδα με αποθηκευμένο αναγνωριστικό απάντησης. |

Οι διαπιστεύσεις είναι έλεγχοι υποσυμβολοσειράς ανεξάρτητοι πεζών-κεφαλαίων. Δείτε την
[τεκμηρίωση της ενέργειας](https://github.com/marketplace/actions/ai-smoke-test) για
το πλήρες σχήμα, συμπεριλαμβανομένων πόρων συνομιλίας που διαχειρίζεται το Foundry.
