# ⚡ Συγχρονισμένες Ροές Εργασίας Πρακτόρων με Azure OpenAI (Responses API) (.NET)

## 📋 Εκμάθηση Υψηλής Απόδοσης Παράλληλης Επεξεργασίας

Αυτό το σημειωματάριο δείχνει **πρότυπα συγχρονισμένης ροής εργασίας** χρησιμοποιώντας το Microsoft Agent Framework για .NET και το Azure OpenAI (Responses API). Θα μάθετε πώς να δημιουργήσετε ροές εργασίας υψηλής απόδοσης, παράλληλης επεξεργασίας που μεγιστοποιούν τη διεκπεραίωση εκτελώντας πολλούς πρακτόρες AI ταυτόχρονα ενώ διατηρείται ο συντονισμός και η συνοχή των δεδομένων.

## 💻 Εκτέλεση Κώδικα

Η πλήρης υλοποίηση είναι διαθέσιμη στο `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Αυτό το αρχείο παρουσιάζει μια **Παράλληλη ροή εργασίας Fan-Out/Fan-In** για σχεδιασμό ταξιδιών:

### 🏗️ **Αρχιτεκτονική Ροής Εργασίας**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Κύρια Συστατικά:**

1. **ConcurrentStartExecutor**: Εκπέμπει το αίτημα χρήστη ταυτόχρονα σε όλους τους πράκτορες
2. **Researcher Agent**: Αναλύει προορισμούς και αξιοθέατα παράλληλα
3. **Planner Agent**: Δημιουργεί λεπτομερείς ταξιδιωτικά σχέδια παράλληλα
4. **ConcurrentAggregationExecutor**: Συλλέγει και συγχωνεύει αποτελέσματα από τους δύο πράκτορες

### 🎯 **Πρότυπο Fan-Out/Fan-In**

Αυτή η ροή εργασίας παρουσιάζει το κλασικό πρότυπο **Fan-Out/Fan-In**:
- **Fan-Out**: Ένα εισερχόμενο μήνυμα εκπέμπεται ταυτόχρονα σε πολλούς πράκτορες
- **Παράλληλη Επεξεργασία**: Πολλοί πράκτορες εργάζονται παράλληλα στην ίδια εργασία
- **Fan-In**: Τα αποτελέσματα από όλους τους πράκτορες συλλέγονται και συγχωνεύονται σε ένα ενιαίο αποτέλεσμα

### 🔍 Κατανόηση Προσαρμοσμένων Εκτελεστών

**ConcurrentStartExecutor:**
- Υλοποιεί το `IMessageHandler<string>` για αποδοχή εισόδου κειμένου
- Εκπέμπει μηνύματα σε όλους τους συνδεδεμένους πράκτορες
- Στέλνει `TurnToken` για να ενεργοποιήσει την παράλληλη επεξεργασία

**ConcurrentAggregationExecutor:**
- Υλοποιεί το `IMessageHandler<ChatMessage>` για λήψη απαντήσεων πρακτόρων
- Συλλέγει μηνύματα με ασφάλεια νήματος (thread-safe)
- Καταγράφει τα αποτελέσματα όταν λάβει όλες τις αναμενόμενες απαντήσεις
- Επιστρέφει το τελικό αποτέλεσμα χρησιμοποιώντας `context.YieldOutputAsync()`

### ⚡ Οφέλη Απόδοσης

**Παράλληλη έναντι Αλληλουχίας:**
- Αλληλουχία: Agent1 (30s) → Agent2 (30s) = **60 δευτερόλεπτα συνολικά**
- Παράλληλη: Agent1 (30s) || Agent2 (30s) = **30 δευτερόλεπτα συνολικά**

**Βελτίωση διεκπεραίωσης**: Έως N φορές πιο γρήγορα για N παράλληλους πράκτορες (ανάλογα με το φορτίο εργασίας και τους πόρους)
