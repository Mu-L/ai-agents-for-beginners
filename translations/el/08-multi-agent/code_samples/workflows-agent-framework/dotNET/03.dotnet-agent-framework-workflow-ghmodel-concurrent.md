# ⚡ Συγχρονισμένες Ροές Εργασίας Πράκτορα με Azure OpenAI (Responses API) (.NET)

## 📋 Εκπαιδευτικό Υλικό για Υψηλής απόδοσης Παράλληλη Επεξεργασία

Αυτό το σημειωματάριο δείχνει **παραδείγματα συγχρονισμένων ροών εργασίας** χρησιμοποιώντας το Microsoft Agent Framework για .NET και το Azure OpenAI (Responses API). Θα μάθετε πώς να δημιουργείτε ροές εργασίας υψηλής απόδοσης και παράλληλης επεξεργασίας που μεγιστοποιούν τη ροή εργασιών εκτελώντας πολλαπλούς πράκτορες AI ταυτόχρονα, διατηρώντας παράλληλα τον συντονισμό και τη συνοχή των δεδομένων.

## 🎯 Στόχοι Μάθησης

### 🚀 **Βασικές Αρχές Παράλληλης Επεξεργασίας**
- **Παράλληλη Εκτέλεση Πρακτόρων**: Εκτέλεση πολλών πρακτόρων AI ταυτόχρονα για μέγιστη απόδοση
- **Πρότυπα Async/Await**: Αξιοποίηση του προγραμματιστικού μοντέλου async του .NET για αποδοτική σύγχρονη λειτουργία

- **Επεξεργασία Μεγάλου Όγκου Εγγράφων**: Επεξεργασία πολλών εγγράφων ταυτόχρονα

## 💻 Εκτέλεση του Κώδικα

Η πλήρης υλοποίηση είναι διαθέσιμη στο αρχείο `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Αυτό το αρχείο παρουσιάζει μια **συγχρονισμένη ροή εργασίας Fan-Out/Fan-In** για προγραμματισμό ταξιδιών:

### 🏗️ **Αρχιτεκτονική Ροής Εργασίας**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Κύρια Συστατικά:**

1. **ConcurrentStartExecutor**: Μεταδίδει το αίτημα χρήστη σε όλους τους πράκτορες ταυτόχρονα
2. **Researcher Agent**: Αναλύει προορισμούς και αξιοθέατα παράλληλα
3. **Planner Agent**: Δημιουργεί λεπτομερή σχέδια ταξιδίου παράλληλα
4. **ConcurrentAggregationExecutor**: Συλλέγει και συγχωνεύει αποτελέσματα και από τους δύο πράκτορες

### 🎯 **Πρότυπο Fan-Out/Fan-In**

Αυτή η ροή εργασίας δείχνει το κλασικό πρότυπο **Fan-Out/Fan-In**:
- **Fan-Out**: Ένα μήνυμα εισόδου μεταδίδεται σε πολλούς πράκτορες ταυτόχρονα
- **Παράλληλη Επεξεργασία**: Πολλοί πράκτορες εργάζονται παράλληλα πάνω στην ίδια εργασία
- **Fan-In**: Τα αποτελέσματα από όλους τους πράκτορες συλλέγονται και συγχωνεύονται σε μία έξοδο

### 🔍 Κατανόηση Προσαρμοσμένων Εκτελεστών

**ConcurrentStartExecutor:**
- Υλοποιεί `IMessageHandler<string>` για αποδοχή εισόδου συμβολοσειράς
- Μεταδίδει μηνύματα σε όλους τους συνδεδεμένους πράκτορες
- Στέλνει `TurnToken` για ενεργοποίηση παράλληλης επεξεργασίας

**ConcurrentAggregationExecutor:**
- Υλοποιεί `IMessageHandler<ChatMessage>` για λήψη απαντήσεων πράκτορα
- Συλλέγει μηνύματα με ασφάλεια νημάτων
- Συγκεντρώνει όταν φτάσουν όλες οι αναμενόμενες απαντήσεις
- Παραδίδει το τελικό αποτέλεσμα με `context.YieldOutputAsync()`

### ⚡ Οφέλη Απόδοσης

**Παράλληλο vs Διαδοχικό:**
- Διαδοχικό: Πράκτορας1 (30s) → Πράκτορας2 (30s) = **60 δευτερόλεπτα συνολικά**
- Παράλληλο: Πράκτορας1 (30s) || Πράκτορας2 (30s) = **30 δευτερόλεπτα συνολικά**

**Βελτίωση ροής εργασίας**: Έως και N× πιο γρήγορα για N παράλληλους πράκτορες (ανάλογα με το φόρτο εργασίας και τους πόρους)

### 🛡️ Διαχείριση Σφαλμάτων

Η ροή εργασίας διαχειρίζεται αποτυχίες μεμονωμένων πρακτόρων με ομαλό τρόπο:
- Εάν ένας πράκτορας αποτύχει, οι άλλοι συνεχίζουν την επεξεργασία
- Ο συλλέκτης μπορεί να εφαρμόσει λογική χρονικού ορίου
- Μερικά αποτελέσματα μπορούν να επιστραφούν αν χρειάζεται
