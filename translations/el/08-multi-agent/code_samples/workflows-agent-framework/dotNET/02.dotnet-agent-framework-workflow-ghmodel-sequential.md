# ⏩ Ακολουθιακά Ροή Εργασίας Πρακτόρων με Azure OpenAI (API Απαντήσεων) (.NET)

## 📋 Εκπαιδευτικό Σεμινάριο Προηγμένης Ακολουθιακής Επεξεργασίας

Αυτό το σημειωματάριο παρουσιάζει **μοτίβα ακολουθιακών ροών εργασίας** χρησιμοποιώντας το Microsoft Agent Framework για .NET και Azure OpenAI (API Απαντήσεων). Θα μάθετε πώς να δημιουργείτε εξελιγμένες, βήμα προς βήμα ροές επεξεργασίας όπου οι πράκτορες εκτελούνται με συγκεκριμένη σειρά, με κάθε στάδιο να χτίζει πάνω στα αποτελέσματα του προηγούμενου σταδίου.

## 🎯 Στόχοι Μάθησης

### 🔄 **Αρχιτεκτονική Ακολουθιακής Επεξεργασίας**
- **Σχεδιασμός Γραμμικής Ροής Εργασίας**: Δημιουργία βήμα προς βήμα σωληνώσεων με σαφείς εξαρτήσεις
- **Διαχείριση Κατάστασης**: Διατήρηση του πλαισίου και ροής δεδομένων ανάμεσα στα στάδια της ακολουθιακής ροής εργασίας
- **Azure OpenAI (API Απαντήσεων)**: Αξιοποίηση των μοντέλων Azure OpenAI σε πολυεπίπεδες ροές εργασίας .NET
- **Μοτίβα Επιχειρησιακής Σωληνοειδούς Επεξεργασίας**: Δημιουργία συστημάτων ακολουθιακής επεξεργασίας έτοιμων για παραγωγή

### 🏗️ **Προηγμένα Ακολουθιακά Μοτίβα**
- **Επεξεργασία με Στάδια-Πύλες**: Εφαρμογή σημείων επικύρωσης ανάμεσα στα στάδια της ροής εργασίας
- **Διατήρηση Πλαισίου**: Διατήρηση κατάστασης και συσσωρευμένης γνώσης σε όλα τα στάδια
- **Διάδοση Σφαλμάτων**: Διαχείριση αποτυχιών με ομαλό τρόπο στις ακολουθιακές αλυσίδες επεξεργασίας
- **Βελτιστοποίηση Απόδοσης**: Αποδοτική ακολουθιακή εκτέλεση με ελάχιστο φόρτο

### 🏢 **Επιχειρησιακές Ακολουθιακές Εφαρμογές**
- **Σωλήνας Επεξεργασίας Εγγράφων**: Πολυεπίπεδη ανάλυση, μετασχηματισμός και επικύρωση εγγράφων
- **Ροές Εργασίας Ποιοτικού Ελέγχου**: Ακολουθιακή αναθεώρηση, επικύρωση και διαδικασίες έγκρισης
- **Σωλήνας Παραγωγής Περιεχομένου**: Έρευνα → Γραφή → Επεξεργασία → Αναθεώρηση → Δημοσίευση
- **Αυτοματοποίηση Επιχειρησιακών Διαδικασιών**: Πολυβηματικές επιχειρησιακές ροές εργασίας με σαφείς εξαρτήσεις σταδίων

## ⚙️ Απαιτήσεις & Ρύθμιση

### 📦 **Απαιτούμενα Πακέτα NuGet**

Βασικά πακέτα για ακολουθιακές ροές εργασίας .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Διαμόρφωση Azure OpenAI**

**Ρύθμιση Περιβάλλοντος (.env αρχείο):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Διαχείριση Διαμόρφωσης:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

- **Σημεία Ελέγχου**: Σημεία επικύρωσης ανάμεσα στα στάδια για διασφάλιση ποιότητας και συνέπειας
- **Πελάτης Azure OpenAI**: Σταθερή πρόσβαση σε μοντέλα AI σε όλα τα στάδια ροής εργασίας

## 💻 Εκτέλεση Κώδικα

Η πλήρης υλοποίηση είναι διαθέσιμη στο `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Αυτό το αρχείο παρουσιάζει μια **τριών σταδίων ροή ανάλυσης επίπλων**:

1. **Στάδιο 1 - Πράκτορας Πωλήσεων**: Αναλύει εικόνες επίπλων και παρέχει προτάσεις αγοράς
2. **Στάδιο 2 - Πράκτορας Τιμών**: Παρέχει λεπτομερείς αναλύσεις τιμών και επιλογές προϋπολογισμού
3. **Στάδιο 3 - Πράκτορας Προσφορών**: Δημιουργεί επαγγελματικό έγγραφο προσφοράς σε μορφή Markdown

### 🏗️ **Αρχιτεκτονική Ροής Εργασίας**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Κάθε πράκτορας:
- Λαμβάνει την έξοδο από το προηγούμενο στάδιο ως πλαίσιο
- Βασίζεται στην προηγούμενη ανάλυση με εξειδικευμένη τεχνογνωσία
- Διατηρεί τη συνέχεια της ροής εργασίας μέσω διαχείρισης κατάστασης

### 🚀 Εκτέλεση Παραδείγματος

**Απαιτήσεις:**
- Τοποθετήστε μια εικόνα επίπλου στο `../imgs/home.png` (ή ενημερώστε τη μεταβλητή `imgPath`)
- Ρυθμίστε το αρχείο `.env` με το endpoint και την υλοποίηση Azure OpenAI, και κάντε σύνδεση με `az login`

```bash
# Κάντε το σενάριο εκτελέσιμο (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Εκτελέστε τη διαδοχική ροή εργασίας
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Ή σε Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Αναμενόμενη Έξοδος

Η ροή εργασίας θα:
1. **Πράκτορας Πωλήσεων**: Αναγνωρίσει αντικείμενα επίπλων από την εικόνα και παρέχει συστάσεις
2. **Πράκτορας Τιμών**: Προσθέσει λεπτομερή ανάλυση τιμών με επίπεδα προϋπολογισμού και προτάσεις αγορών
3. **Πράκτορας Προσφορών**: Δημιουργήσει ένα μορφοποιημένο έγγραφο προσφοράς με συνθεμένες όλες τις πληροφορίες

Το τελικό αποτέλεσμα θα είναι μια ολοκληρωμένη, επαγγελματική προσφορά επίπλων βασισμένη στην ανάλυση εικόνας.
