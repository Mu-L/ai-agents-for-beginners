# ⚡ Egyidejű Ügynök Munkafolyamatok Azure OpenAI-val (Válaszok API) (.NET)

## 📋 Nagy Teljesítményű Párhuzamos Feldolgozás Bemutató

Ez a jegyzetfüzet bemutatja az **egyidejű munkafolyamat mintákat** a Microsoft Agent Framework .NET-hez és az Azure OpenAI (Válaszok API) segítségével. Megtanulod, hogyan építs magas teljesítményű párhuzamos feldolgozó munkafolyamatokat, amelyek több AI ügynök egyidejű futtatásával maximalizálják az áteresztőképességet, miközben fenntartják az együttműködést és az adatkonszisztenciát.

## 💻 Kód Futattása

A teljes megvalósítás elérhető a `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` fájlban. Ez a fájl bemutat egy **Fan-Out/Fan-In egyidejű munkafolyamatot** utazástervezéshez:

### 🏗️ **Munkafolyamat Architektúra**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Kulcskomponensek:**

1. **ConcurrentStartExecutor**: Egyidejűleg továbbítja a felhasználói kérést az összes ügynöknek
2. **Researcher Agent**: Egyidejűleg elemzi az úticélokat és látványosságokat
3. **Planner Agent**: Egyidejűleg készít részletes utazási terveket
4. **ConcurrentAggregationExecutor**: Összegyűjti és egyesíti mindkét ügynök eredményeit

### 🎯 **Fan-Out/Fan-In Minta**

Ez a munkafolyamat a klasszikus **Fan-Out/Fan-In** mintát mutatja be:
- **Fan-Out**: Egy bemeneti üzenet egyszerre több ügynökhöz kerül továbbításra
- **Egyidejű Feldolgozás**: Több ügynök párhuzamosan dolgozik ugyanazon a feladaton
- **Fan-In**: Az összes ügynöktől érkező eredmény összegyűjtése és egyetlen kimenetté egyesítése

### 🔍 Egyedi Végrehajtók Megértése

**ConcurrentStartExecutor:**
- Megvalósítja az `IMessageHandler<string>` interfészt string bemenet kezeléséhez
- Üzeneteket sugároz minden csatlakoztatott ügynöknek
- Elküldi a `TurnToken`-t az egyidejű feldolgozás elindításához

**ConcurrentAggregationExecutor:**
- Megvalósítja az `IMessageHandler<ChatMessage>` interfészt az ügynök válaszok fogadásához
- Szálbiztosan gyűjti össze az üzeneteket
- Összegzi az összes várt válasz megérkezésekor
- Visszaadja a végső kimenetet `context.YieldOutputAsync()` segítségével

### ⚡ Teljesítmény Előnyök

**Egyidejű vs Szekvenciális:**

- Szekvenciális: Agent1 (30 mp) → Agent2 (30 mp) = **összesen 60 másodperc**
- Párhuzamos: Agent1 (30 mp) || Agent2 (30 mp) = **összesen 30 másodperc**

**Átviteli sebesség javítása**: Akár N× gyorsabb N párhuzamos ügynök esetén (függ a munkaterheléstől és az erőforrásoktól)
