## 📋 Nagyteljesítményű párhuzamos feldolgozási bemutató

Ez a jegyzetfüzet **egyszerre futó munkafolyamat-mintákat** mutat be a Microsoft Agent Framework .NET-hez és az Azure OpenAI (Responses API) szolgáltatásával. Megtanulhatod, hogyan építs fel nagyteljesítményű, párhuzamos feldolgozási munkafolyamatokat, amelyek maximalizálják az áteresztőképességet több AI ügynök egyidejű futtatásával, miközben megőrzik az együttműködést és az adatkonszisztenciát.

## 💻 A kód futtatása

A teljes megvalósítás elérhető a `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` fájlban. Ez a fájl egy **Fan-Out/Fan-In párhuzamos munkafolyamatot** mutat be utazástervezéshez:

### 🏗️ **Munkafolyamat architektúra**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Kulcsfontosságú komponensek:**

1. **ConcurrentStartExecutor**: A felhasználói kérést egyszerre sugározza az összes ügynöknek
2. **Researcher Agent**: Egyidejűleg elemzi az úti célokat és látnivalókat
3. **Planner Agent**: Részletes utazási terveket készít párhuzamosan
4. **ConcurrentAggregationExecutor**: Összegyűjti és összevonja az eredményeket mindkét ügynöktől

### 🎯 **Fan-Out/Fan-In minta**

Ez a munkafolyamat a klasszikus **Fan-Out/Fan-In** mintát mutatja be:
- **Fan-Out**: Egy bemenő üzenet egyszerre több ügynökhöz kerül sugárzásra
- **Párhuzamos feldolgozás**: Több ügynök dolgozik párhuzamosan ugyanazon a feladaton
- **Fan-In**: Az összes ügynöktől származó eredményt összegyűjtik és egyetlen kimenetté egyesítik

### 🔍 Egyedi végrehajtók megértése

**ConcurrentStartExecutor:**
- Megvalósítja az `IMessageHandler<string>` felületet string bemenet fogadására
- Üzenetek sugárzása az összes csatlakoztatott ügynöknek
- `TurnToken` küldése az egyidejű feldolgozás elindításához

**ConcurrentAggregationExecutor:**
- Megvalósítja az `IMessageHandler<ChatMessage>` felületet ügynök válaszok fogadására
- Üzenetek gyűjtése szálbiztos módon
- Összesítés minden várható válasz megérkezésekor
- Végső output kiadása a `context.YieldOutputAsync()` használatával

### ⚡ Teljesítmény előnyök

**Párhuzamos vs. szekvenciális:**
- Szekvenciális: Ügynök1 (30s) → Ügynök2 (30s) = **összesen 60 másodperc**
- Párhuzamos: Ügynök1 (30s) || Ügynök2 (30s) = **összesen 30 másodperc**

**Áteresztőképesség javulás**: Akár N-szer gyorsabb N párhuzamos ügynöknél (munkaterheléstől és erőforrásoktól függően)
