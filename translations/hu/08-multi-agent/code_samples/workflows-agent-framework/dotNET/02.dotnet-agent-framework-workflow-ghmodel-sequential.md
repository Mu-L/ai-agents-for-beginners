# ⏩ Szekvenciális ügynök munkafolyamatok az Azure OpenAI-val (Responses API) (.NET)

## 📋 Haladó szekvenciális feldolgozási útmutató

Ez a jegyzetfüzet bemutatja a **szekvenciális munkafolyamat mintákat** a Microsoft Agent Framework .NET-hez és az Azure OpenAI-hoz (Responses API). Megtanulhatja, hogyan építsen kifinomult, lépésről lépésre haladó feldolgozási csővezetékeket, ahol az ügynökök meghatározott sorrendben hajtják végre a feladatokat, az egyes szakaszok pedig az előző szakasz eredményeire épülnek.

## 🎯 Tanulási célok

### 🔄 **Szekvenciális feldolgozási architektúra**
- **Lineáris munkafolyamat tervezés**: Lépésről lépésre haladó feldolgozási csővezetékek létrehozása világos függőségekkel
- **Állapotkezelés**: Kontextus és adatáramlás fenntartása a szekvenciális munkafolyamat szakaszai között
- **Azure OpenAI (Responses API)**: Azure OpenAI modellek alkalmazása több szakaszos .NET munkafolyamatokban
- **Vállalati csővezeték minták**: Gyártásra kész szekvenciális feldolgozó rendszerek építése

### 🏗️ **Haladó szekvenciális minták**
- **Stage-Gate feldolgozás**: Érvényesítési ellenőrzőpontok bevezetése a munkafolyamat szakaszai között
- **Kontextus megőrzése**: Állapot és felhalmozott tudás megtartása minden szakasz között
- **Hibaterjedés kezelése**: Hibák szelíd kezelése a szekvenciális feldolgozó láncokban
- **Teljesítmény optimalizálás**: Hatékony szekvenciális végrehajtás minimális költséggel

### 🏢 **Vállalati szekvenciális alkalmazások**
- **Dokumentumfeldolgozó csővezeték**: Többszakaszos dokumentumelemzés, átalakítás és érvényesítés
- **Minőségellenőrzési munkafolyamatok**: Szekvenciális áttekintés, érvényesítés és jóváhagyási folyamatok
- **Tartalomkészítő csővezeték**: Kutatás → Írás → Szerkesztés → Áttekintés → Kiadás
- **Üzleti folyamat automatizálás**: Többlépéses üzleti munkafolyamatok világos szakaszfuggőségekkel


**Konfigurációkezelés:**

```mermaid

    A[Initial Input] --> B[Stage 1: Elemző Ügynök]
```

## 💻 A kód futtatása

A teljes megvalósítás elérhető a `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs` fájlban. Ez a fájl egy **háromszakaszos bútor elemzési munkafolyamatot** mutat be:

1. **1. szakasz - Értékesítési ügynök**: Bútor képeket elemez és vásárlási javaslatokat ad
2. **2. szakasz - Ár ügynök**: Részletes árösszetevőket és költségvetési opciókat kínál
3. **3. szakasz - Árajánlat ügynök**: Professzionális ajánlat dokumentumot generál Markdown formátumban

### 🏗️ **Munkafolyamat architektúra**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Minden ügynök:
- Az előző szakasz eredményét kontextusként kapja meg
- Az előző elemzésre épít speciális szakértelemmel
- Állapotkezeléssel fenntartja a munkafolyamat folytonosságát

### 🚀 A példa futtatása

**Előfeltételek:**
- Helyezzen el egy bútor képet a `../imgs/home.png` helyen (vagy frissítse az `imgPath` változót)
- Állítsa be a `.env` fájlját az Azure OpenAI végpontjával és telepítésével, majd jelentkezzen be az `az login` segítségével

```bash
# Tedd futtathatóvá a szkriptet (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Futtasd le a szekvenciális munkafolyamatot
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Vagy Windows-on:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Várt kimenet

A munkafolyamat:
1. **Értékesítési ügynök**: Felismeri a bútor tárgyakat a képen és ajánlásokat ad
2. **Ár ügynök**: Részletes ár elemzést ad költségvetési szintekkel és vásárlási javaslatokkal
3. **Árajánlat ügynök**: Formázott ajánlat dokumentumot készít az összes információ szintetizálásával

A végső kimenet egy átfogó, professzionális bútor ajánlat lesz a kép elemzése alapján.
