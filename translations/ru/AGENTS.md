# AGENTS.md

## Обзор проекта

Этот репозиторий содержит "AI Agents for Beginners" - комплексный образовательный курс, обучающий всему необходимому для создания ИИ агентов. Курс состоит из 18 уроков (пронумерованных 00-18), охватывающих основы, шаблоны проектирования, фреймворки, развертывание в производстве, локальных/устройственных агентов и безопасность ИИ агентов.

**Ключевые технологии:**
- Python 3.12+
- Jupyter ноутбуки для интерактивного обучения
- Фреймворки ИИ: Microsoft Agent Framework (MAF)
- Azure AI сервисы: Microsoft Foundry, Microsoft Foundry Agent Service V2

**Архитектура:**
- Структура на основе уроков (папки 00-15+)
- Каждый урок содержит: документацию README, примеры кода (Jupyter ноутбуки) и изображения
- Мультиязычная поддержка через автоматизированную систему перевода
- Один ноутбук Python на урок с использованием Microsoft Agent Framework

## Команды для настройки

### Требования
- Python версии 3.12 или выше
- Подписка Azure (для Microsoft Foundry)
- Установленная и аутентифицированная Azure CLI (`az login`)

### Начальная настройка

1. **Клонируйте или форкните репозиторий:**
   ```bash
   gh repo fork microsoft/ai-agents-for-beginners --clone
   # ИЛИ
   git clone https://github.com/microsoft/ai-agents-for-beginners.git
   cd ai-agents-for-beginners
   ```

2. **Создайте и активируйте виртуальное окружение Python:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # В Windows: venv\Scripts\activate
   ```

3. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Настройте переменные окружения:**
   ```bash
   cp .env.example .env
   # Отредактируйте .env, указав свои API-ключи и конечные точки
   ```

### Обязательные переменные окружения

Для **Microsoft Foundry** (обязательно):
- `AZURE_AI_PROJECT_ENDPOINT` - конечная точка проекта Microsoft Foundry
- `AZURE_AI_MODEL_DEPLOYMENT_NAME` - имя развертывания модели (например, gpt-4.1-mini)

Для **Azure AI Search** (Урок 05 - RAG):
- `AZURE_SEARCH_SERVICE_ENDPOINT` - конечная точка Azure AI Search
- `AZURE_SEARCH_API_KEY` - ключ API Azure AI Search

Аутентификация: Выполните `az login` перед запуском ноутбуков (использует `AzureCliCredential`).

## Рабочий процесс разработки

### Запуск Jupyter ноутбуков

Каждый урок содержит несколько Jupyter ноутбуков для разных фреймворков:

1. **Запустите Jupyter:**
   ```bash
   jupyter notebook
   ```

2. **Перейдите в папку урока** (например, `01-intro-to-ai-agents/code_samples/`)

3. **Откройте и запустите ноутбуки:**
   - `*-python-agent-framework.ipynb` - Использование Microsoft Agent Framework (Python)
   - `*-dotnet-agent-framework.ipynb` - Использование Microsoft Agent Framework (.NET)

### Работа с Microsoft Agent Framework

**Microsoft Agent Framework + Microsoft Foundry:**
- Требуется подписка Azure
- Использует `FoundryChatClient` для Agent Service V2 (агенты видны в портале Foundry)
- Готово для продакшена с встроенной наблюдаемостью
- Шаблон имени файла: `*-python-agent-framework.ipynb`

Для уроков, где агент развернут как агент Microsoft Foundry (01, 04, 05, 16), репозиторий содержит каталоги смоук-тестирования в папке `tests/`, которые запускаются workflow `.github/workflows/smoke-test.yml` через action [AI Smoke Test](https://github.com/marketplace/actions/ai-smoke-test). Это легкий пост-разверточный контроль (доступен ли агент и соответствует ли базовым ожиданиям по подсказкам?), дополняющий пайплайн оценки в Уроках 10 и 16. Урок 17 запускается локально с Foundry Local и не имеет размещенной конечной точки, поэтому проверяется запуском ноутбука напрямую.

## Дополнительные заметки

### Частые ошибки

1. **Несоответствие версии Python:**
   - Используйте Python 3.12+
   - Некоторые пакеты не работают на старых версиях
   - Используйте `python3 -m venv` для явного указания версии Python

2. **Переменные окружения:**
   - Всегда создавайте `.env` из `.env.example`
   - Не коммитьте файл `.env` (он в `.gitignore`)
   - Войдите через `az login` для аутентификации Entra ID без ключа

3. **Конфликты пакетов:**
   - Используйте свежее виртуальное окружение
   - Устанавливайте через `requirements.txt`, а не отдельные пакеты
   - Некоторые ноутбуки могут требовать дополнительные пакеты, указанные в markdown ячейках

4. **Сервисы Azure:**
   - Сервисы Azure AI требуют активной подписки
   - Некоторые функции доступны только в отдельных регионах
   - Убедитесь, что развертывание вашей модели Azure OpenAI поддерживает Responses API

## Специфика проекта

### Мультиязычная поддержка

В репозитории используется автоматизированная система перевода:
- Поддержка более 50 языков
- Переводы в каталогах `/translations/<lang-code>/`
- Workflow GitHub Actions обрабатывает обновления переводов
- Исходные файлы на английском в корне репозитория

### Структура урока

Каждый урок следует последовательному шаблону:
1. Видео-превью с ссылкой
2. Письменный контент урока (README.md)
3. Примеры кода в нескольких фреймворках
4. Цели обучения и требования
5. Дополнительные учебные ресурсы со ссылками

### Именование примеров кода

Формат: `<номер-урока>-python-agent-framework.ipynb`
- `01-python-agent-framework.ipynb` - Урок 1, MAF Python
- `14-sequential.ipynb` - Урок 14, продвинутые шаблоны MAF
- `16-python-agent-framework.ipynb` - Урок 16, продакшен агент поддержки клиентов
- `17-local-agent-foundry-local.ipynb` - Урок 17, локальный агент с Foundry Local + Qwen

### Особые каталоги

- `translated_images/` - Локализованные изображения для переводов
- `images/` - Оригинальные изображения для английского контента
- `.devcontainer/` - Конфигурация контейнера для разработки VS Code
- `.github/` - Workflow и шаблоны GitHub Actions

### Зависимости

Основные пакеты из `requirements.txt`:
- `agent-framework` - Microsoft Agent Framework
- `a2a-sdk` - Поддержка протокола Agent-to-Agent
- `azure-ai-inference`, `azure-ai-projects` - Сервисы Azure AI
- `azure-identity` - Аутентификация Azure (AzureCliCredential)
- `azure-search-documents` - Интеграция с Azure AI Search
- `mcp[cli]` - Поддержка Model Context Protocol
