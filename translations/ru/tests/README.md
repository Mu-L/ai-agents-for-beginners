# Smoke-тесты агентов

Эта папка содержит **каталоги smoke-тестов** для агентов, которых вы создаёте в курсе.
Smoke-тест — это недорогая, быстрая проверка того, что **развернутый хостингом Microsoft Foundry
агент** доступен, отвечает и следует своим базовым ожиданиям подсказок.
Это первый барьер — не замена полноценного конвейера оценки, который вы изучаете в
[Уроке 10](../10-ai-agents-production/README.md) и
[Уроке 16](../16-deploying-scalable-agents/README.md).

Каталоги используются GitHub Action [AI Smoke Test](https://github.com/marketplace/actions/ai-smoke-test)
через workflow [`.github/workflows/smoke-test.yml`](../../../.github/workflows/smoke-test.yml).


## Как запустить

1. **Разверните агента урока** в Microsoft Foundry как хостинг-агента (см.
   Урок 16 для процесса развертывания). Запомните **имя агента** и конечную точку
   вашего **Foundry проекта**.
2. Добавьте эти секреты репозитория (Настройки → Секреты и переменные → Actions):
   `AZURE_CLIENT_ID`, `AZURE_TENANT_ID`, `AZURE_SUBSCRIPTION_ID`. Федерированная
   учётная запись должна иметь роль **Пользователь Azure AI** с областью действия **Foundry проекта**.
3. На вкладке **Actions** запустите **Smoke-test hosted agents**, выберите
   `tests_file` для урока, затем укажите соответствующие `agent_name` и
   `project_endpoint`.

## В каких уроках есть smoke-тесты?

Smoke-тесты применимы к урокам, где вы **развёртываете агента**, чьи текстовые ответы
можно проверить на соответствие известному содержимому. Уроки, которые концептуальны,
работают только локально или генерируют недетерминированный творческий контент, преднамеренно исключены:

- **Урок 17 (Создание локальных AI-агентов)** полностью выполняется на вашем рабочем месте с
  помощью Foundry Local и **не** предоставляет конечную точку Foundry Responses, поэтому
  это действие к нему не применяется. Проверьте его, запустив блокнот локально.
- Уроки по шаблонам проектирования и теории (02, 03, 06, 07, 09, 12) не содержат ни одного
  развертываемого агента для smoke-тестирования.

## Схема каталога (быстрая справка)

Каждый каталог — это JSON-документ с верхнеуровневым массивом `tests`. Каждая запись POST-ит
один запрос и проверяет ответ:

| Поле | Значение |
|-------|---------|
| `id` | Уникальный идентификатор шага, выводимый в журнал. |
| `description` | Читаемое человеку описание цели. |
| `prompt` | Сообщение, отправленное агенту. |
| `assertions.status` | Ожидаемый HTTP статус (по умолчанию 200). |
| `assertions.contains_any` | Тест пройден, если ответ содержит любой из этих подстрок. |
| `assertions.contains_all` | Тест пройден, если ответ содержит каждую подстроку. |
| `assertions.contains_none` | Тест пройден, если ответ не содержит ни одной из этих подстрок. |
| `save_response_id_as` | Сохраняет ID ответа для последующего шага с несколькими ходами. |
| `use_previous_response_id` | Отправляет этот ход связным с сохранённым ID ответа. |

Проверки — это нечувствительные к регистру поиски подстрок. Смотрите
[документацию действия](https://github.com/marketplace/actions/ai-smoke-test) для
полной схемы, включая ресурсы разговоров, управляемые Foundry.
