# ⏩ Последовательные рабочие процессы агентов с Azure OpenAI (Responses API) (.NET)

## 📋 Расширенный учебник по последовательной обработке

В этой записной книжке демонстрируются **паттерны последовательного рабочего процесса** с использованием Microsoft Agent Framework для .NET и Azure OpenAI (Responses API). Вы научитесь создавать сложные, пошаговые конвейеры обработки, где агенты выполняются в определённом порядке, и каждый этап основывается на результатах предыдущего.

## 🎯 Цели обучения

### 🔄 **Архитектура последовательной обработки**

- **Распространение ошибок**: Обрабатывайте сбои аккуратно в последовательных цепочках обработки

```env
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Управление конфигурацией:**

### 🏗️ **Архитектура последовательного рабочего процесса**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Аналитический агент]
    B --> C[Контрольная точка 1]
    C --> D[Stage 2: Агент обработки]
    D --> E[Контрольная точка 2]
    E --> F[Stage 3: Агент валидации]
    F --> G[Финальный вывод]
    
    H[Контекст состояния] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Responses API)] --> B
    I --> D
    I --> F
```

**Ключевые компоненты:**
- **Последовательные агенты**: Специализированные агенты для каждого этапа обработки
- **Контекст состояния**: Поддерживает накопленные данные и решения на всех этапах
- **Контрольные точки**: Точки валидации между этапами для обеспечения качества и согласованности
- **Клиент Azure OpenAI**: Единый доступ к AI-моделям на всех этапах рабочего процесса

## 💻 Запуск кода

Полная реализация доступна в файле `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Этот файл демонстрирует **трёхэтапный рабочий процесс анализа мебели**:

1. **Этап 1 - Агент продаж**: Анализирует изображения мебели и предоставляет рекомендации по покупке
2. **Этап 2 - Агент цены**: Предоставляет подробные разбивки по ценам и варианты бюджета
3. **Этап 3 - Агент коммерческого предложения**: Генерирует профессиональный документ-коммерческое предложение в формате Markdown

### 🏗️ **Архитектура рабочего процесса**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Каждый агент:
- Получает вывод предыдущего этапа в качестве контекста
- Строит анализ на основе предыдущих данных с специализированной экспертизой
- Поддерживает непрерывность рабочего процесса через управление состоянием

### 🚀 Запуск примера

**Требования:**
- Поместите изображение мебели по адресу `../imgs/home.png` (или обновите переменную `imgPath`)
- Настройте файл `.env` с вашим эндпоинтом Azure OpenAI и развертыванием, затем выполните вход с помощью `az login`

```bash
# Сделайте скрипт исполняемым (Unix/Linux/macOS)

./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Ожидаемый результат

Рабочий процесс выполнит:
1. **Агент продаж**: Идентифицирует предметы мебели на изображении и даст рекомендации
2. **Агент цены**: Добавит подробный анализ цен с уровнями бюджета и рекомендациями по покупкам
3. **Агент коммерческого предложения**: Сгенерирует форматированный документ предложения со всей собранной информацией

Итоговый результат будет комплексным, профессиональным коммерческим предложением мебели на основе анализа изображения.

### 🔍 Понимание потока состояния

Каждый агент в последовательности получает:
- **Исходный ввод**: Начальное сообщение пользователя (изображение + текст)
- **Выводы предыдущих агентов**: Все предыдущие ответы агентов в истории разговора
- **Накопленный контекст**: Полное состояние, поддерживаемое на протяжении рабочего процесса

Это позволяет выполнять сложную многоэтапную обработку, где каждый агент строит анализ на основе полного контекста всех предыдущих этапов.
