# ⏩ Последовательные рабочие процессы агентов с Azure OpenAI (Responses API) (.NET)

## 📋 Руководство по расширенной последовательной обработке

В этом блокноте демонстрируются **образцы последовательных рабочих процессов** с использованием Microsoft Agent Framework для .NET и Azure OpenAI (Responses API). Вы узнаете, как создавать сложные поэтапные конвейеры обработки, где агенты выполняются в определённом порядке, и каждый этап строится на результатах предыдущего.

**Настройка окружения (.env файл):**
```env

AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Управление конфигурацией:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");

var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **Архитектура последовательного рабочего процесса**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Агент анализа]
    B --> C[Контрольная точка 1]
    C --> D[Stage 2: Агент обработки]
    D --> E[Контрольная точка 2]
    E --> F[Stage 3: Агент проверки]
    F --> G[Итоговый результат]
    
    H[Контекст состояния] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API ответов)] --> B
    I --> D
    I --> F
```

**Ключевые компоненты:**
- **Последовательные агенты**: Специализированные агенты для каждого этапа обработки
- **Контекст состояния**: Поддерживает накопленные данные и решения на этапах
- **Контрольные точки**: Точки валидации между этапами для обеспечения качества и согласованности
- **Azure OpenAI клиент**: Обеспечивает единый доступ к AI-моделям на всех этапах рабочего процесса

## 💻 Запуск кода

Полная реализация доступна в файле `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Этот файл демонстрирует **трёхэтапный рабочий процесс анализа мебели**:

1. **Этап 1 - Торговый агент**: Анализирует изображения мебели и даёт рекомендации по покупке
2. **Этап 2 - Ценовой агент**: Предоставляет подробные ценовые раскладки и варианты бюджета
3. **Этап 3 - Агент котировок**: Генерирует профессиональный документ с коммерческим предложением в формате Markdown

### 🏗️ **Архитектура рабочего процесса**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Каждый агент:
- Получает результат с предыдущего этапа в качестве контекста
- Развивает предыдущий анализ, опираясь на специализированные знания
- Поддерживает непрерывность рабочего процесса через управление состоянием

### 📝 Ожидаемый результат

Рабочий процесс выполнит:
1. **Торговый агент**: Определит мебель на изображении и даст рекомендации
2. **Ценовой агент**: Добавит детальный анализ цен с бюджетными вариантами и рекомендациями по покупке
3. **Агент котировок**: Сформирует отформатированный документ с коммерческим предложением, синтезируя всю информацию

Итоговый результат — полноценное профессиональное коммерческое предложение по мебели на основе анализа изображений.

### 🔍 Понимание потока состояния

Каждый агент в последовательности получает:
- **Исходный ввод**: Первоначальное сообщение пользователя (изображение + текст)
- **Результаты предыдущих агентов**: Все предыдущие ответы агентов в истории разговора
- **Накопленный контекст**: Полное состояние, поддерживаемое на протяжении всего рабочего процесса

Это позволяет выполнять сложную многоэтапную обработку, где каждый агент строит вывод на основе всестороннего контекста всех предыдущих этапов.
