# 🔄 Основные рабочие процессы агентов с Azure OpenAI (API Responses) (.NET)

## 📋 Учебное пособие по оркестровке рабочих процессов

В этом блокноте показано, как создавать сложные **рабочие процессы агентов** с использованием Microsoft Agent Framework для .NET и Azure OpenAI (API Responses). Вы научитесь создавать многоэтапные бизнес-процессы, в которых агенты ИИ сотрудничают для выполнения сложных задач посредством структурированных паттернов оркестровки.

## 🎯 Цели обучения

### 🏗️ **Основы архитектуры рабочих процессов**
- **Конструктор рабочих процессов**: Разработка и оркестрация сложных многоэтапных процессов ИИ
- **Координация агентов**: Координация нескольких специализированных агентов в рабочих процессах
- **Azure OpenAI (API Responses)**: Использование Azure OpenAI Responses API в рабочих процессах
- **Визуальный дизайн рабочих процессов**: Создание и визуализация структуры процессов для лучшего понимания

### 🔄 **Паттерны оркестровки процессов**
- **Последовательная обработка**: Цепочка нескольких задач агентов в логическом порядке
- **Управление состоянием**: Поддержка контекста и потока данных между этапами процесса
- **Обработка ошибок**: Реализация надежного восстановления после сбоев и устойчивости процессов
- **Оптимизация производительности**: Проектирование эффективных процессов для корпоративного масштаба

### 🏢 **Применение в корпоративных рабочих процессах**
- **Автоматизация бизнес-процессов**: Автоматизация сложных организационных рабочих процессов

## ⚙️ Требования и настройка

### 📦 **Необходимые пакеты NuGet**

Для демонстрации рабочего процесса используются несколько ключевых пакетов .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Конфигурация Azure OpenAI**

**Настройка окружения (файл .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Доступ к Azure OpenAI:**
1. Создайте ресурс Azure OpenAI в портале Azure
2. Разверните модель (например, `gpt-5-mini`) и запомните имя развертывания
3. Войдите с помощью `az login` и настройте переменные окружения, как показано выше

### 🏗️ **Обзор архитектуры рабочих процессов**

```mermaid
graph TD
    A[Конструктор рабочих процессов] --> B[Реестр агентов]
    B --> C[Движок выполнения рабочих процессов]
    C --> D[Agent 1: Генератор контента]
    C --> E[Agent 2: Рецензент контента] 
    D --> F[Результаты рабочих процессов]
    E --> F
    G[Azure OpenAI (Responses API)] --> D
    G --> E
```

**Ключевые компоненты:**
- **WorkflowBuilder**: Основной движок оркестровки для проектирования рабочих процессов
- **AIAgent**: Отдельные специализированные агенты с конкретными возможностями
- **Azure OpenAI Client**: Интеграция с Azure OpenAI Responses API
- **Execution Context**: Управляет состоянием и потоком данных между этапами рабочего процесса

## 💻 Запуск кода

Полная реализация доступна в `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Этот файл демонстрирует:

1. **Настройка окружения** – загрузка конфигурации Azure OpenAI из файла `.env`
2. **Настройка клиента Azure OpenAI** – конфигурирование клиента для использования Azure OpenAI Responses API
3. **Создание агентов** – определение специализированных агентов (Front Desk и Concierge)
4. **Конструктор рабочих процессов** – создание многоагентского процесса с последовательной обработкой
5. **Запуск рабочего процесса** – выполнение рабочего процесса с потоковой передачей результатов

### 🚀 Запуск примера

```bash
# Сделайте скрипт исполняемым (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Запустите рабочий процесс
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Или на Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Ожидаемый результат

Рабочий процесс будет:
1. Принимать ваш запрос о путешествии («Я бы хотел поехать в Париж»)
2. Агент Front Desk предоставит первоначальную рекомендацию
3. Агент Concierge проверит и уточнит рекомендацию
4. Финальный вывод отобразит полный поток беседы

- Добавления дополнительных агентов для создания сложных многоэтапных процессов
- Изменения сообщения пользователя для тестирования различных сценариев
