# 🔄 Основные рабочие процессы агентов с Azure OpenAI (Responses API) (.NET)

## 📋 Учебник по оркестрации рабочих процессов

В этой тетрадке показано, как создавать сложные **рабочие процессы агентов** с использованием Microsoft Agent Framework для .NET и Azure OpenAI (Responses API). Вы научитесь создавать многошаговые бизнес-процессы, в которых агенты ИИ сотрудничают для выполнения сложных задач через структурированные паттерны оркестрации.

## 🎯 Цели обучения

### 🏗️ **Основы архитектуры рабочих процессов**
- **Конструктор рабочих процессов**: Проектирование и оркестрация сложных многошаговых процессов ИИ
- **Координация агентов**: Координация нескольких специализированных агентов в рабочих процессах
- **Azure OpenAI (Responses API)**: Использование Azure OpenAI Responses API в рабочих процессах
- **Визуальное проектирование рабочих процессов**: Создание и визуализация структур рабочих процессов для лучшего понимания

### 🔄 **Паттерны оркестрации процессов**
- **Последовательная обработка**: Связывание нескольких задач агентов в логическом порядке
- **Управление состоянием**: Поддержание контекста и потока данных между этапами рабочего процесса
- **Обработка ошибок**: Реализация надежного восстановления после ошибок и устойчивости рабочего процесса
- **Оптимизация производительности**: Проектирование эффективных рабочих процессов для корпоративных операций

### 🏢 **Корпоративные приложения рабочих процессов**
- **Автоматизация бизнес-процессов**: Автоматизация сложных организационных рабочих процессов
- **Конвейер производства контента**: Редакционные рабочие процессы с этапами проверки и утверждения
- **Автоматизация обслуживания клиентов**: Многошаговое разрешение запросов клиентов
- **Рабочие процессы обработки данных**: ETL рабочие процессы с AI-управляемой трансформацией

## ⚙️ Требования и настройка

### 📦 **Требуемые пакеты NuGet**

В этой демонстрации рабочего процесса используются несколько ключевых пакетов .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Настройка Azure OpenAI**

**Настройка окружения (файл .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Доступ к Azure OpenAI:**
1. Создайте ресурс Azure OpenAI в портале Azure
2. Разверните модель (например, `gpt-4.1-mini`) и запомните имя развертывания
3. Выполните вход с помощью `az login` и настройте переменные окружения, как показано выше

### 🏗️ **Обзор архитектуры рабочего процесса**

```mermaid
graph TD
    A[Конструктор рабочих процессов] --> B[Реестр агентов]
    B --> C[Движок выполнения рабочих процессов]
    C --> D[Agent 1: Генератор контента]
    C --> E[Agent 2: Рецензент контента] 
    D --> F[Результаты рабочих процессов]
    E --> F
    G[Azure OpenAI (API ответов)] --> D
    G --> E
```

**Ключевые компоненты:**
- **WorkflowBuilder**: Основной движок оркестрации для проектирования рабочих процессов
- **AIAgent**: Отдельные специализированные агенты с конкретными возможностями
- **Клиент Azure OpenAI**: Интеграция с Azure OpenAI Responses API
- **Execution Context**: Управляет состоянием и потоком данных между этапами рабочего процесса

## 💻 Запуск кода

Полная реализация доступна в `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Этот файл демонстрирует:

1. **Конфигурация окружения** - загрузка конфигурации Azure OpenAI из файла `.env`
2. **Настройка клиента Azure OpenAI** - конфигурирование клиента для использования Azure OpenAI Responses API
3. **Создание агентов** - определение специализированных агентов (ресепшн и консьерж)
4. **Конструктор рабочих процессов** - создание многоагентного рабочего процесса с последовательной обработкой
5. **Выполнение рабочего процесса** - запуск процесса с потоковой выдачей результатов

### 🚀 Запуск примера

```bash
# Сделайте скрипт исполняемым (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Запустите рабочий процесс
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Или на Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Ожидаемый результат

Рабочий процесс будет:
1. Принимать ваш запрос о месте путешествия ("Я хотел бы поехать в Париж")
2. Адвокат на ресепшн дает первоначальную рекомендацию
3. Агент консьержа рассматривает и уточняет рекомендацию
4. Итоговый вывод отображает полный поток разговора

- Инструкции агентам для изменения их поведения
- Добавляя больше агентов для создания сложных многоэтапных рабочих процессов
- Изменяя сообщение пользователя для тестирования разных сценариев
- Настраивая связи рабочего процесса для создания различных паттернов выполнения
