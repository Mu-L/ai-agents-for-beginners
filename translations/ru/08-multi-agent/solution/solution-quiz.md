# Ответы для проверки знаний

## Вопрос 1

**Ответ: A2.** Несколько агентов полезны, когда задача требует различных специализированных ролей, отдельных инструментов или разрешений, а также четкой координации передачи задач. Простой бот поддержки обычно может оставаться одним агентом. Большой объем одинаковых запросов — это скорее вопрос масштабируемости и сам по себе не требует конструкции с несколькими агентами.

## Вопрос 2

**Ответ: A1.** Один агент обычно проще тестировать, наблюдать и управлять, когда один набор инструкций и инструментов может справиться с задачей. Доступ к нескольким инструментам не требует автоматически нескольких агентов. Отдельные роли с разными разрешениями и требованиями к аудиту могут оправдывать дизайн с несколькими агентами.
