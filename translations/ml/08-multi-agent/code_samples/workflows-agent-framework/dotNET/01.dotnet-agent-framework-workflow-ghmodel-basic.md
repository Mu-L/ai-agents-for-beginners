# 🔄 Azure OpenAI (Responses API) ഉപയോഗിച്ച് അടിസ്ഥാന ഏജന്റ് വർക്‌ഫ്ലോകൾ (.NET)

## 📋 വർക്‌ഫ്ലോ ഒർക്കസ്ത്രേഷൻ ട്യൂട്ടോറിയൽ

ഈ നോട്ട്‌ബുക്ക് Microsoft Agent Framework for .NETയും Azure OpenAI (Responses API)യും ഉപയോഗിച്ച് സാങ്കീർണ്ണമായ **ഏജന്റ് വർക്‌ഫ്ലോകൾ** എങ്ങനെ നിർമ്മിക്കാമെന്ന് കാണിക്കുന്നു. നിങ്ങൾ ബഹു-നടപടികാല ബിസിനസ് പ്രക്രിയകൾ സൃഷ്ടിക്കാൻ പഠിക്കും, അവിടെ AI ഏജന്റുമാർ സുവ്യവസ്ഥപ്പെടുത്തിയ ഒർക്കസ്ത്രേഷൻ മാതൃകകളിലൂടെ ചേർന്ന് സങ്കീർണ്ണപ്പെട്ട പ്രവർത്തനങ്ങൾ വിജയകരമായി പൂർത്തിയാക്കും.

## 🎯 പഠന ലക്ഷ്യങ്ങൾ

### 🏗️ **വർക്‌ഫ്ലോ ഘടനയുടെ അടിസ്ഥാനങ്ങൾ**
- **Workflow Builder**: സങ്കീർണ്ണ ബഹു-നടപടി AI പ്രക്രിയകൾ രൂപകൽപ്പന ചെയ്ത് ഒർക്കത്രീറ്റ് ചെയ്യുക
- **Agent Coordination**: വർക്‌ഫ്ലോകളിൽ സ്വതന്ത്രമായ പ്രത്യേക ഏജന്റുമാരെ ഏകോപിപ്പിക്കുക
- **Azure OpenAI (Responses API)**: വർക്‌ഫ്ലോകളിൽ Azure OpenAI Responses API പ്രയോജനംപ്പെടുത്തുക
- **Visual Workflow Design**: വർക്‌ഫ്ലോ ഘടനകൾ സൃഷ്ടിച്ച് മികച്ച മനസിലാക്കലിനായി ദർശന സഹായം നൽകുക

### 🔄 **പ്രക്രിയ ഒർക്കസ്ഥ്രേഷൻ മാതൃകകൾ**
- **Sequential Processing**: പല ഏജന്റ് പ്രവൃത്തികളും ഉപരി കലാപമില്ലാതെ പരമ്പരാഗതമായി ബന്ധിപ്പിക്കൽ
- **State Management**: വർക്‌ഫ്ലോ ഘട്ടങ്ങളിലുടനീളം സന്ദർഭവും ഡാറ്റാ പ്രവാഹവും നിലനിർത്തൽ
- **Error Handling**: ശക്തമായ പിശക് കൊണ്ടുവന്ന പുനരുദ്ധാരണവും വർക്‌ഫ്ലോ സുസ്ഥിരതയും നടപ്പാക്കൽ
- **Performance Optimization**: സംരംഭപരമായ വ്യാപക പ്രവർത്തനങ്ങൾക്ക് തെളിവുറ്റ വർക്‌ഫ്ലോകൾ രൂപകൽപ്പന ചെയ്യൽ

### 🏢 **സംരംഭ വർക്‌ഫ്ലോ അപ്ലിക്കേഷനുകൾ**
- **Business Process Automation**: സങ്കീർണ്ണമായ സംഘടനാ വർക്‌ഫ്ലോകൾ സ്വയം പ്രവർത്തിപ്പിക്കൽ
- **Content Production Pipeline**: അവലോകനവും അംഗീകാര ഘട്ടങ്ങളും ഉള്ള എഡിറ്റോറിയൽ വർക്‌ഫ്ലോകൾ
- **Customer Service Automation**: ബഹു-നടപടി ഉപഭോക്തൃ ചോദ്യ പരിഹാരം
- **Data Processing Workflows**: AI സംയോജിതമായ പുനരൂപീകരണത്തോടെ ETL വർക്‌ഫ്ലോകൾ

## ⚙️ മുൻ‌വിധികളും ക്രമീകരണവും

### 📦 **ആവശ്യമായ NuGet പാക്കേജുകൾ**

ഈ വർക്‌ഫ്ലോ ഡെമോയിൽ പല βασിക .NET പാക്കേജുകൾ ഉപയോഗിക്കുന്നു:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI കോൺഫിഗറേഷൻ**

**പരിസ്ഥിതി ക്രമീകരണം (.env ഫയൽ):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI ആക്‌സസ്:**
1. Azure പോർട്ടലിൽ Azure OpenAI റിസോഴ്‌സ് സൃഷ്ടിക്കൂ
2. ഒരു മോഡൽ (ഉദാഹരണം: `gpt-4.1-mini`) വിന്യസിച്ച് വിന്യാസത്തിന്റെ പേര് ശ്രദ്ധിക്കുക
3. `az login` ഉപയോഗിച്ച് സൈൻ ഇൻ ചെയ്‌തു പരിസ്ഥിതി ചാരങ്ങൾ മുകളിൽ കാണിച്ച പോലെ ക്രമീകരിക്കുക

### 🏗️ **Workflow Architecture അവലോകനം**

```mermaid
graph TD
    A[പ്രവർത്തനപ്രവാഹ നിർമ്മാതാവ്] --> B[ഏജന്റ് രജിസ്ട്രി]
    B --> C[പ്രവർത്തനപ്രവാഹ നിർവഹണ എഞ്ചിൻ]
    C --> D[Agent 1: ഉള്ളടക്ക നിർമ്മാതാവ്
    C --> E[Agent 2: ഉള്ളടക്ക समीക്ഷകൻ 
    D --> F[പ്രവർത്തനഫലം]
    E --> F
    G[അസ്യൂർ ഒപ്പൺഎഐ (പ്രതിസന്ധികൾ API)] --> D
    G --> E
```

**പ്രധാന ഘടകങ്ങൾ:**
- **WorkflowBuilder**: വർക്‌ഫ്ലോ രൂപകൽപ്പനയ്ക്കും ഒർക്കസ്ഥ്രേഷനും പ്രധാന എഞ്ചിൻ
- **AIAgent**: പ്രത്യേക കഴിവുകൾ ഉള്ള വ്യക്തിഗത ഏജന്റുമാർ
- **Azure OpenAI Client**: Azure OpenAI Responses API ഇന്റഗ്രേഷൻ
- **Execution Context**: വർക്‌ഫ്ലോ ഘട്ടങ്ങളിലെ നിലയും ഡാറ്റാ പ്രവാഹവും നിയന്ത്രിക്കുന്നു

## 🏢 **സംരംഭത്തിന് ലഭിക്കുന്ന ഗുണങ്ങൾ**

### 🎯 **നിരന്തരതയും സ്‌കെയിലബിലിടിയും**
- **നിശ്ചിത നിർവഹണം**: സ്ഥിരതയുള്ള, ആവർത്തനയോഗ്യമായ വർക്‌ഫ്ലോ ഫലങ്ങൾ
- **പിശക് പുനരുദ്ധരണം**: പ്രവൃത്തിതടസ്സങ്ങളിലെ പരാജയങ്ങളെ ക്രൂരമായി കൈകാര്യം ചെയ്യൽ
- **പ്രകടന നിരീക്ഷണം**: നിർവഹണ മെട്രിക്സും മെച്ചപ്പെടുത്തൽ അവസരങ്ങളും പിഞ്ചുക
- **സ്‌റോതസ്സ് മാനേജ്മെന്റ്**: AI മോഡൽ റിസോഴ്‌സുകളുടെ കാര്യക്ഷമമായ വിന്യാസവും ഉപയോഗവും

### 🔒 **സുരക്ഷയും അനുസരണയും**
- **സുരക്ഷിത അതെന്റിഫിക്കേഷൻ**: `az login` (AzureCliCredential) വഴി Microsoft Entra ID അതെന്റിഫിക്കേഷൻ
- **ഓഡിറ്റ് ട്രെയിലുകൾ**: Workflow നിർവഹണവും തീരുമാനം പോയിന്റുകളും പൂർണ്ണമായ ലോഗ് ചെയ്യൽ
- **പ്രവേശന നിയന്ത്രണം**: Workflow നിർവഹണത്തിന്റെയും നിരീക്ഷണത്തിന്റെയും സുഗ്രഹ വിഭാഗീയ അനുവാദങ്ങൾ
- **ഡാറ്റാ സ്വകാര്യത**: Workflowകളിൽ സമഗ്രമായ സംവേദനാത്മക വിവരങ്ങളുടെ സുരക്ഷിത കൈകാര്യം

### 📊 **ദേശീയതയും മാനേജ്മെന്റും**
- **ദൃശ്യ വർക്‌ഫ്ലോ ഡിസൈൻ**: പ്രക്രിയ പ്രവാഹങ്ങളും ആശ്രിതതകളും വ്യക്തമായി പ്രതിനിധീകരണം
- **നിർവഹണ നിരീക്ഷണം**: Workflow പുരോഗതിയും പ്രവർത്തനക്ഷമതയും സജീവമായി നിരീക്ഷിക്കൽ
- **പിശക് റിപ്പോർട്ടിംഗ്**: വിശദമായ പിശക് വിശകലനവും ഡീബഗ്ഗിംഗും
- **പ്രകടന വിശകലനങ്ങൾ**: മെച്ചപ്പെടുത്തലിനും ശേഷി നിർണയത്തിനും മെട്രികുകൾ

## 💻 കോഡ് നടപ്പിലാക്കൽ

`01.dotnet-agent-framework-workflow-ghmodel-basic.cs` ഫയലിൽ മുഴുവൻ നടപ്പിലാക്കൽ ലഭ്യമാണ്. ഈ ഫയൽ കാണിക്കുന്നു:

1. **പരിസ്ഥിതി ക്രമീകരണം** - `.env` ഫയലിൽ നിന്നുള്ള Azure OpenAI കോൺഫിഗറേഷൻ ലോഡുചെയ്യൽ
2. **Azure OpenAI Client ക്രമീകരണം** - Azure OpenAI Responses API ഉപയോഗിക്കാൻ ക്ലയന്റ് ക്രമീകരിക്കൽ
3. **ഏജന്റ് സൃഷ്ടി** - പ്രത്യേക കഴിവുകൾ ഉള്ള ഏജന്റുമാരെ നിർവചിക്കൽ (ഫ്രണ്ട് ഡെസ്ക്, കോൺസിയർജ്)
4. **Workflow Builder** - പരമ്പരാഗത പ്രക്രിയയോടെ ബഹു ഏജന്റ് Workflow നിർമ്മിക്കൽ
5. **Workflow നിർവഹണം** - ഫലങ്ങൾ സ്ട്രീമിംഗോടെ Workflow നടത്തൽ

### 📝 പ്രതീക്ഷിച്ച ഫലം
