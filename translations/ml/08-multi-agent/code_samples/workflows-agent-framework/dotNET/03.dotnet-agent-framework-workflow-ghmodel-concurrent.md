# ⚡ Azure OpenAI (Responses API) ഉപയോഗിച്ച് സമകാലിക ഏജന്റ് വർക്ക്‌ഫ്ലോകൾ (.NET)

## 📋 ഉയർന്ന പ്രകടനമുള്ള പരലൽ പ്രോസസ്സിംഗ് ട്യൂട്ടോറിയൽ

ഈ നോട്ട്‌ബുക് Microsoft Agent Framework for .NET ഉം Azure OpenAI (Responses API) ഉം ഉപയോഗിച്ച് **സമകാലിക വർക്ക്‌ഫ്ലോ പാറ്റേണുകൾ** കാണിക്കുന്നു. ഒരേസമയം നിരവധി AI ഏജന്റുകളെ പ്രവർത്തിപ്പിച്ചുകൊണ്ട് കോർഡിനേഷൻയും ഡാറ്റാ സാമ്യസ്ഥിതിയുമുണ്ടാക്കി throughput ഏറ്റവും ഉയർത്തുന്ന ഉയർന്ന പ്രകടനമുള്ള സമകാലിക പ്രോസസ്സിംഗ് വർക്ക്‌ഫ്ലോകൾ എങ്ങനെ നിർമ്മിക്കാമെന്ന് നിങ്ങൾ പഠിക്കും.

- **Async/Await പാറ്റേണുകൾ**: ഫലപ്രദമായ സമകാലികതയ്ക്ക് .NET എന്ന അസിങ്ക്രണസ് പ്രോഗ്രാമിങ് മോഡൽ ഉപയോഗിക്കുക
- **Azure OpenAI (Responses API)**: Azure OpenAI Responses API-യിൽ ഒന്നിച്ചുകൊണ്ടുള്ള സമകാലിക കോൾകൾ ഏകീകരിക്കുക

- **Synchronization Points**: സമകാലിക പ്രവർത്തനങ്ങളിൽ ഡാറ്റാ സാമ്യം ഉറപ്പാക്കുന്നു

## 💻 കോഡ് എങ്ങനെ ഓടിക്കുക

പൂർണ്ണമായ നിർവഹണം `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` എന്ന ഫയലിൽ ലഭ്യമാണ്. ഈ ഫയൽ യാത്രാ പദ്ധതിക്കായി **Fan-Out/Fan-In സമകാലിക വർക്ക്‌ഫ്ലോ** കാണിക്കുന്നു:

### 🏗️ **വർക്ക്‌ഫ്ലോ ആർക്കിടെക്ചർ**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**പ്രധാന ഘടകങ്ങൾ:**

1. **ConcurrentStartExecutor**: ഉപയോക്തൃ അഭ്യർത്ഥന എല്ലാ ഏജന്റുകൾക്കും ഒരേ സമയം സംപ്രേഷണം ചെയ്യുന്നു
2. **Researcher Agent**: ഗമ്യസ്ഥലങ്ങളും ആകർഷണങ്ങളും സമകാലികമായി വിശകലനം ചെയ്യുന്നു
3. **Planner Agent**: വിശദമായ യാത്രാ പദ്ധതികൾ സമകാലികമായി ഒരുക്കുന്നു
4. **ConcurrentAggregationExecutor**: ഇരുവരും ലഭിച്ച ഫലങ്ങൾ ശേഖരിച്ചു മിശ്രണം ചെയ്യുന്നു

### 🎯 **Fan-Out/Fan-In പാറ്റേൺ**

ഈ വർക്‌ഫ്ലോ ക്ലാസിക് **Fan-Out/Fan-In** പാറ്റേൺ കാണിക്കുന്നു:
- **Fan-Out**: ഒരേ ഇൻപുട്ട് സന്ദേശം ഒരേസമയം പല ഏജന്റുകള്ക്കും പ്രക്ഷേപണം ചെയ്യുന്നു
- **സമകാലിക പ്രോസസ്സിംഗ്**: ഒരേ ടാസ്‌കിൽ ഒരേസമയം നിരവധി ഏജന്റുകൾ പ്രവർത്തിക്കുന്നു
- **Fan-In**: എല്ലാ ഏജന്റുകളുടെ ഫലങ്ങൾ ശേഖരിച്ച് ഏക output-ലേക്ക് മിച്ച്ഛിച്ചുചേർത്തെടുക്കുന്നു

### 📝 പ്രതീക്ഷിക്കുന്ന ഔട്ട്‌പുട്ട്

വർക്‌ഫ്ലോ ഈ ರೀತിയാകും:
1. **Broadcast Request**: "Plan a trip to Seattle in December" രണ്ട് ഏജന്റുകൾക്കും അയയ്ക്കുക
2. **സമകാലിക പ്രോസസ്സിംഗ്**: രണ്ട് ഏജന്റുകളും ഒരേസമയം പ്രവർത്തിക്കുന്നു:
   - റിസർചർ ആകർഷണങ്ങളും വിശദാംശങ്ങളും തിരിച്ചറിയുന്നു
   - പ്ലാനർ യാത്രാപദ്ധതിയും യാത്രാസൂത്രണവും രൂപപ്പെടുത്തുന്നു
3. **Aggregation**: രണ്ട് പ്രതികരണങ്ങളും സംയോജിപ്പിച്ച് സമ്പൂർണ്ണ output നൽകുന്നു
4. **Display Results**: എല്ലാ വിവരങ്ങളുമുള്ള സംയോജിത യാത്രാ പദ്ധതി പ്രദർശിപ്പിക്കുന്നു

### 🔧 കസ്റ്റമൈസേഷൻ ഓപ്ഷനുകൾ

**കൂടുതൽ സമകാലിക ഏജന്റുകൾ ചേർക്കുക:**
```csharp
// Create additional specialized agents
AIAgent budgetAgent = azureClient.GetOpenAIResponseClient(deployment).CreateAIAgent(
    name: "Budget-Agent", instructions: "Calculate travel costs...");

// Add to fan-out
var workflow = new WorkflowBuilder(startExecutor)
    .AddFanOutEdge(startExecutor, targets: [researcherAgent, plannerAgent, budgetAgent])
    .AddFanInEdge(aggregationExecutor, sources: [researcherAgent, plannerAgent, budgetAgent])
    .WithOutputFrom(aggregationExecutor)
    .Build();

// Update aggregation count
if (this._messages.Count == 3) { ... }
```

**എജന്റ് നിർദ്ദേശങ്ങൾ تعديلിക്കുക:**
```csharp
const string ResearcherAgentInstructions = "Your custom instructions for research...";
const string PlanAgentInstructions = "Your custom instructions for planning...";
```

**ടാസ്‌ക് മാറ്റുക:**
```csharp
StreamingRun run = await InProcessExecution.StreamAsync(
    workflow, 
    "Plan a European vacation for 2 weeks in summer"
);
```

### 🎯 യാഥാർത്ഥ്യ ആപ്ലിക്കേഷനുകൾ

ഈ സമകാലിക പാറ്റേൺ അനുയോജ്യം:
- **ഉള്ളടക്കം സൃഷ്ടി**: ഒരേസമയം വിവിധ വിഭാഗങ്ങൾ എഴുതുന്ന നിരവധി എഴുത്തുകാരെ
- **കോഡ് റിവ്യൂ**: വ്യത്യസ്ത കാഴ്ചപ്പാടുകളിൽ നിന്ന് കോഡ് പരിശോധിക്കാനായി നിരവധി റിവ്യൂവേഴ്‌സ്
- **മാർക്കറ്റ് റിസർച്ച്**: വ്യത്യസ്ത വിപണി വിഭാഗങ്ങളുടെ പരലൽ വിശകലനം
- **ഡോക്യുമെന്റ് പ്രോസസ്സിംഗ്**: ഒരേസമയം വീതക്കുമാറ്റം, വിശകലനം, സ്ഥിരീകരണം
- **മൾട്ടി-പേഴ്സ്പെക്ടീവ് അനാലിസിസ്**: ഒരേ ഇൻപുട്ടിൽ വ്യത്യസ്ത കാഴ്ചപ്പാടുകൾ നേടുന്നത്

### 🔍 കസ്റ്റം എക്സിക്യൂട്ടറുകൾ മനസിലാക്കുക

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` നടപ്പിലാക്കി സ്റ്റ്രിംഗ് ഇൻപുട്ട് സ്വീകരിക്കുന്നു
- ബന്ധിപ്പിച്ച എല്ലാ ഏജന്റുകൾക്കും സന്ദേശങ്ങൾ പ്രക്ഷേപണം ചെയ്യുന്നു
- `TurnToken` അയച്ച് സമകാലിക പ്രോസസ്സിംഗ് തുടങ്ങുന്നു

**ConcurrentAggregationExecutor:**
- `IMessageHandler<ChatMessage>` നടപ്പിലാക്കി ഏജന്റ് പ്രതികരണങ്ങൾ സ്വീകരിക്കുന്നു
- ത്രെഡ്-സുരക്ഷിതമായി സന്ദേശങ്ങൾ ശേഖരിക്കുന്നു
- പ്രതീക്ഷിക്കപ്പെട്ട എല്ലാ പ്രതികരണങ്ങളും എത്തിയാൽ ഏകീകരിക്കുന്നു
- `context.YieldOutputAsync()` ഉപയോഗിച്ച് അന്തിമ output നൽകുന്നു

### ⚡ പ്രകടന ലാഭങ്ങൾ

**സമകാലികവും ശ്രേണിയിലുമായും:**
- ശ്രേണി: Agent1 (30s) → Agent2 (30s) = **മൊത്തം 60 സെക്കൻഡ്**
- സമകാലികം: Agent1 (30s) || Agent2 (30s) = **മൊത്തം 30 സെക്കൻഡ്**

**ത്രുപുട്ട് മെച്ചപ്പെടുത്തൽ**: N സമകാലിക ഏജന്റുകൾക്ക് N തവണ വേഗത്തിൽ (ജോലി ഭാരവും റിസോഴ്‌സുകൾ ആശ്രയിച്ചുകൂടി)

### 🛡️ പിശക് കൈകാര്യം

സമകാലിക പ്രവർത്തനങ്ങളിൽ വ്യക്തിഗത ഏജന്റ് പരാജയങ്ങൾ മനോഹരമായ രീതിയിൽ കൈകാര്യം ചെയ്യും:
- ഒരാളെ പരാജയപ്പെടുമ്പോഴും മറ്റു ഏജന്റുകൾ പ്രവർത്തനം തുടരുന്നു
- Aggregator ടൈംഔട്ട് ലൊജിക് നടപ്പിലാക്കാം
- ഭാഗിക ഫലങ്ങൾ ആവശ്യമെങ്കിൽ നൽകാം
