# മാറ്റങ്ങൾ രേഖപ്പെടുത്തിയത്

**ആരംഭകർക്കുള്ള AI ഏജന്റ്‍സുകൾ** കോഴ്സിലെ എല്ലാ ശ്രദ്ധേയമായ മാറ്റങ്ങളും ഈ ഫയലിൽ രേഖപ്പെടുത്തപ്പെട്ടിരിക്കുന്നു.

## [പറിയാത്തവ] — 2026-07-13

ഈ റിലീസ് രണ്ട് പുതിയ പാഠങ്ങൾ ചേർക്കുന്നു, ഡിപ്ലോയ്മെന്റ് ആർക്കിന് സമാപനം ചെയ്യുന്നതിന് — മൈക്രോസോഫ്‌റ്റ് ഫൗണ്ട്രീക്ക് ഏജന്റ്സിനെ സ്കെയിൽ ചെയ്യൽ മുതൽ ഒരു ഒറ്റ വര്‍ക്കുംക്ഷനിലേക്കുള്ള കുറച്ചുകൂടി പിന്തുടരൽ — കൂടാതെ ഒരു സ്മോക്ക്-ടെസ്റ്റ് പൈപ്പ്‌ലൈൻ, പുതുക്കിയ കോഴ്സ് നാവിഗേഷൻ, പുതിയ ലേണർ കഴിവുകൾ, അപ്ഡേറ്റ് ചെയ്ത ബ്രാൻഡിംഗ് എന്നിവയും.

- **പാഠം 16 — മൈക്രോസോഫ്‌റ്റ് ഫൗണ്ടറിയുമായി സ്കാലബിൾ ഏജന്റ്സിന്റെ ഡിപ്ലോയ്മെന്റ്.** പുതിയ പാഠം [16-deploying-scalable-agents/README.md](./16-deploying-scalable-agents/README.md) കൂടാതെ റണബിൾ നോട്ട്ബുക്ക് [16-python-agent-framework.ipynb](./16-deploying-scalable-agents/code_samples/16-python-agent-framework.ipynb) പ്രൊഡക്ഷൻ കസ്റ്റമർ-സപ്പോർട്ട് ഏജന്റ് നിർമ്മിക്കുന്നു (ടൂളുകൾ, RAG, മെമ്മറി, മോഡൽ റൂട്ടിംഗ്, പ്രതികരണ കാഷിങ്, മനുഷ്യ അംഗീകാരം, മൂല്യനിർണയ ഗേറ്റ്, OpenTelemetry ട്രേസിംഗ്), വികസനം/ഡിപ്ലോയ്മെന്റ്/റൺടൈം മറമെയ്ഡ് ഡയഗ്രാമുകൾ, ഒരു നോളഡ്ജ് ചെക്ക്, അസൈൻമെന്റ്, ചലഞ്ച് എന്നിവയോടുകൂടി.
- **പാഠം 17 — ഫൗണ്ടറി ലോക്കലും ക്വെൻ എന്നിവ ഉപയോഗിച്ച് ലോക്കൽ AI ഏജന്റ്സുകളുടെ സൃഷ്ടി.** പുതിയ പാഠം [17-creating-local-ai-agents/README.md](./17-creating-local-ai-agents/README.md) കൂടാതെ നോട്ട്ബുക്ക് [17-local-agent-foundry-local.ipynb](./17-creating-local-ai-agents/code_samples/17-local-agent-foundry-local.ipynb) പൂര്‍ണമായും ഉപകരണത്തിൽ പ്രവർത്തിക്കുന്ന എഞ്ചിനീയറിംഗ് അസിസ്റ്റന്റ്റ് നിർമ്മിക്കുന്നു (ക്വെൻ ഫംഗ്ഷൻ കോൾ ഫൗണ്ടറി ലോക്കലിലൂടെ, സാൻഡ്‌ബോക്സ്ഡ് ഫയൽ ടൂളുകൾ, ലോക്കൽ RAG ക്രോമയുമായി, ഐഷ്യൽ ലോക്കൽ MCP), ലോക്കൽ-ഓൺലി / ലോക്കൽ-RAG / ടൂൾ-കോൾ ചെയ്യുന്ന ഡയഗ്രാമുകൾ, നോളഡ്ജ് ചെക്ക്, അസൈൻമെന്റ്, ചലഞ്ച് എന്നിവയോടുകൂടി.
- **സ്മോക്ക്-ടെസ്റ്റ് പൈപ്പ്‌ലൈൻ.** പുതിയ [AI Smoke Test](https://github.com/marketplace/actions/ai-smoke-test) വർക്ക്‌ഫ്ലോ [.github/workflows/smoke-test.yml](../../.github/workflows/smoke-test.yml) കൂടാതെ ഓരോ പാഠത്തിനും കീഴിൽ [tests/](./tests/README.md) പട്ടികകൾ ഉണ്ട്, പാഠങ്ങൾ 01, 04, 05, 16ലെ ഡിപ്ലോയബിള്‍ ഏജന്റ്സിനുള്ള, ഓരോ പട്ടികയെയും പാഠത്തിനും ഹോസ്റ്റ് ചെയ്ത ഏജന്റ് നാമത്തിനും മാറ്റി നൽകുന്ന ഇൻഡക്സ് README ഉൾപ്പെടെ. പാഠം 16-ന് "ഡിപ്ലോയ്ഡ് ഏജന്റ് സ്മോക്ക് ടെസ്റ്റുകളുമായി പരിശോദിക്കൽ" വിഭാഗം ലഭിച്ചു; പാഠങ്ങൾ 01/04/05-ന് ഐഷ്യൽ സ്മോക്ക് ടെസ്റ്റ് പൊയിന്റർ.
- **ലേണർ കഴിവുകൾ.** പുതിയ ഏജന്റ് സ്കിൽസ് `.agents/skills/` എന്നിടത്ത്: [deploying-scalable-agents](./.agents/skills/deploying-scalable-agents/SKILL.md), [local-ai-agents](./.agents/skills/local-ai-agents/SKILL.md) (പാഠം 16, 17 മാർഗനിർദേശങ്ങൾ പാക്കേജിംഗിൽ), [testing-course-samples](./.agents/skills/testing-course-samples/SKILL.md) (മൈക്രോസോഫ്‌ട് ഫൗണ്ടറി / അസ്യൂർ ഓപ്പൺ AI സെറ്റപ്പിനെ എങ്ങനെ ലൈവ് പരിശോധിക്കാമെന്നുള്ളത്).
- **നോട്ട്ബുക്ക് വാലിഡേഷൻ റണ്ണർ.** എല്ലാ പൈതൺ നോട്ട്‌ബുക്കുകളും ഹെഡ്ലെസ് ആയി `nbconvert` ഉപയോഗിച്ച് പ്രവർത്തിപ്പിച്ച് PASS/FAIL മാട്രിക്‌സ് (മറ്റും `results.json` സഹിതം) പ്രിന്റ് ചെയ്യുന്നത് പുതിയ [scripts/validate-notebooks.ps1](../../scripts/validate-notebooks.ps1). ഇത് റിപോസ്റ്റെറി റൂട്ട് ഉം പൈതൺ ഉം സ്വയം കണ്ടു പിടിക്കുന്നു, കോഴ്സ് അല്ലാത്ത നോട്ട്ബുക്കുകൾ (`.venv`, `site-packages`, `translations`, സ്കിൽ ടെംപ്ലേറ്റ് ആസ്തികൾ) ഒപ്പം `.NET` നോട്ട്ബുക്കുകൾ ഡീഫോൾട്ട് ഒഴിവാക്കി, `-Filter`, `-Timeout`, `-List`, `-IncludeDotnet`, `-Python` എന്നിവ പിന്തുണയുള്ളത്.
- **കോഴ്സ് നാവിഗേഷൻ.** പാഠങ്ങൾ 11–15-ല്‍ മുന്‍/അടുത്ത പാഠ ലിങ്കുകൾ ചേർത്തു (മുമ്പ് കാണാതെ), അതിനാൽ മുഴുവൻ കോഴ്സ് 00 → 18 എന്ന കുട്ടിയിൽ ചുമർക്കുന്നു.
- **പുതിയ തംബ്‌നെയിലുകൾ.** പാഠങ്ങൾ 16, 17-ന് തംബ്‌നെയിലുകൾ, പുതുക്കിയ റിപ്പോസിറ്ററി സോഷ്യൽ ഇമേജ് [images/repo-thumbnailv3.png](./images/repo-thumbnailv3.png) (ഇപ്പോൾ രണ്ട് പുതിയ പാഠങ്ങളും `aka.ms/ai-agents-beginners` URL-നെ പരസ്യപ്പെടുത്തുന്നത്).

- **ഒഴുക്ക് പോയ മോഡലുകളിലല്ലാത്ത മോഡൽ റഫറൻസുകൾ ഏകീകരിച്ചു.** മുഴുവൻ കോഴ്സിലും `gpt-4o` / `gpt-4o-mini` എന്ന റഫറൻസുകൾ `gpt-4.1-mini` ആയും മാറ്റി (മുഴുവൻ പതിപ്പുകളും 2026-ൽ ഒഴിവാക്കപ്പെടുന്നു). പാഠം 16-ന്റെ മോഡൽ റൂട്ടിംഗ് ഉദാഹരണത്തിൽ ചെറിയ/വലിയത് താരതമ്യം `gpt-4.1-mini` (ചെറിയത്) ഉം `gpt-4.1` (വലിയത്) ഉം ഉപയോഗിക്കുന്നു. പൈതൺ നോട്ട്ബുക്കുകൾ ഹൈഡ്കോഡ് ചെയ്ത മോഡൽ പേരിന്റെ പകരം അന്തരീക്ഷ വ്യത്യാസങ്ങളിലൂടെ (`AZURE_AI_MODEL_DEPLOYMENT_NAME` / `AZURE_OPENAI_DEPLOYMENT`) മോഡൽ തിരഞ്ഞെടുക്കുന്നു.

### കുറിപ്പുകളും അറിയാവുന്ന പരിമിതികളും

- **ലൈവ് അസ്യൂറിനെതിരെ പ്രവർത്തിച്ചിട്ടില്ല.** പുതിയ പാഠങ്ങളുടെ նോട്ട്ബുക്കുകൾ വിദ്യാഭ്യാസ സാമ്പിളുകളാണ്; നിങ്ങളുടെ സ്വന്തം മൈക്രോസോഫ്റ്റ് ഫൗണ്ടറി / ഫൗണ്ടറി ലോക്കൽ സെറ്റപ്പിൽ അത് പ്രവർത്തിപ്പിച്ച് പരിശോധിക്കുക. സ്മോക്ക്-ടെസ്റ്റ് വർക്ക്‌ഫ്ലോക്കിന് പാഠത്തിന്റെ ഏജന്റ് ഡിപ്ലോയ്ചെയ്യുകയും അസ്യൂർ OIDC രഹസ്യങ്ങൾ (`AZURE_CLIENT_ID`, `AZURE_TENANT_ID`, `AZURE_SUBSCRIPTION_ID`) **അസ്യൂർ AI യൂസർ** റോളിൽ ഫൗണ്ടറി പ്രോജക്ട് സ്‌കോപ്പിൽ കോൻഫിഗർ ചെയ്യേണ്ടതാണ്.
- **പാഠം 17 ലോക്കൽ-ഓൺലിയാണ്.** അതിനോട് ഒരു ഫൗണ്ടറി റസ്പോൺസ് എന്റ്പോയിന്റ് ഇല്ല, അതിനാൽ സ്മോക്ക് ടെസ്റ്റ് ആക്ഷൻ അനുയോജ്യം അല്ല; നിങ്ങളുടെ വർക്സ്‌റ്റേഷനിൽ നോട്ട്ബുക്ക് ഓടിച്ച് പരിശോധിക്കുക.

## [പറിയാത്തവ] — 2026-07-06

ഈ റിലീസ് കോഴ്സിനെ **അസ്യൂർ ഓപ്പൺAI റസ്പോൺസസ് API**ക്ക് മാറ്റുന്നു, ഉൽപ്പന്ന നാമങ്ങൾ **മൈക്രോസോഫ്‌റ്റ് ഫൗണ്ടറി**ക്കും **മൈക്രോസോഫ്‌റ്റ് ഏജന്റ് ഫ്രെയിംവർക്ക് (MAF)**ക്കും മാനദണ്ഡമാക്കുന്നു, ഗിറ്റ്ഹബ് മോഡലുകൾ ഒഴിവാക്കുന്നു, SDK പതിപ്പുകൾ അപ്ഡേറ്റ് ചെയ്യുന്നു, ലോക്കൽ മോഡലുകളും ഫൗണ്ടറിയിൽ മറ്റ് ഫ്രെയിംവർക്ക് ഹോസ്റ്റിംഗും സംബന്ധിച്ച പുതിയ ഉള്ളടക്കം ചേർക്കുന്നു.

### കുറിപ്പുകളും അറിയാവുന്ന പരിധികളുമ്

- **പ്രവർത്തിപ്പിച്ചോ/കോമ്പൈൽ ചെയ്തോ ചെയ്തിട്ടില്ല.** API/നാമകരണം ശരിയാക്കലിനായി പുതുക്കിയ പഠന മാതൃകകളാണ്; ഇവ സജീവ Azure വിഭവങ്ങളെതിരെ പ്രവർത്തിപ്പിച്ചിട്ടില്ല, .NET മാതൃകകൾ ഈ പരിസ്ഥിതിയിൽ കോമ്പൈൽ ചെയ്തിട്ടില്ല. നിങ്ങളുടെ സ്വന്തം Microsoft Foundry / Azure OpenAI വിന്യാസത്തിലൂടെ സ്ഥിരീകരിക്കുക.
- **മോഡൽ വിന്യാസം Responses API പിന്തുണയ്ക്കണം.** `gpt-4.1-mini`, `gpt-4.1` അല്ലെങ്കിൽ `gpt-5.x` മോഡൽ പോലുള്ള വിന്യാസം ഉപയോഗിക്കുക. പഴയ മോഡലുകൾ Responses മുഖ്യ പ്രവർത്തനം പിന്തുണയ്ക്കുന്നു, എന്നാൽ എല്ലാ ഫീച്ചറുകളും അല്ല.
- **എജന്റ്-ഫ്രെയിംവർക് പതിപ്പ്.** മാതൃകകൾ ഏറ്റവും പുതിയ MAF (`>=1.10.0`) ലക്ഷ്യമിടുന്നു. ശുദ്ധമായ എജന്റ്-സൃഷ്ടി വിളി `client.as_agent(...)`; APIകൾ ഫ്രെയിംവർക് പ്രസിദ്ധീകരിച്ച ഡോക്‌സ്, ഇൻസ്റ്റാൾ ചെയ്ത ബിൽഡ് എന്നിവയെതിരെ ശരിയാക്കിയപ്പോൾ പരിശോധിച്ചു. വ്യത്യസ്ത പതിപ്പ് നിഗമിപ്പിച്ചാൽ, മേധോഡിന്റെ ലഭ്യത ( `as_agent` vs `create_agent`) സ്ഥിരീകരിക്കേണ്ടത് ആവശ്യമാണ്.
- **ലെസ്സൺ 08 വർക്ക്ഫ്ലോ നോട്ട്ബുക്ക് 04** ഇച്ഛയാൽ `AzureAIAgentClient` (`agent-framework-azure-ai` നിന്നുള്ള) നിലനിർത്തുന്നു, കാരണം ഇത് Microsoft Foundry Agent Service സന്ദർശനം ഉപയോഗിക്കുന്നു (Bing ഗ്രൗണ്ടിംഗ്, കോഡ് ഇൻറർപ്രിറ്റർ); ഇത് ഇതിനകം Responses അടിസ്ഥാനമാക്കിയതാണ്.
- **.NET ഡീഫോൾട്ട് വിന്യാസം.** രണ്ടു ലെസ്സൺ 08 ഡോട്ട്.NET വർക്ക്ഫ്ലോ മാതൃകകൾ മുമ്പ് പ്രത്യേക മോഡൽ ഹാർഡ്‌ക്കോഡ് ചെയ്തിരുന്നു; ഇപ്പോൾ അവർ `AZURE_OPENAI_DEPLOYMENT` (`gpt-4.1-mini`) ആയി ഡീഫോൾട്ടാണ്. ഒരു മാതൃക മൾട്ടിമോഡൽ/വിഷൻ ഇൻപുട്ടിൽ ആശ്രയിച്ചാൽ, `AZURE_OPENAI_DEPLOYMENT` അനുയോജ്യമായ മോഡൽ ആയി സജ്ജമാക്കുക.
- **Foundry Local** OpenAI-സംബന്ധിച്ച **Chat Completions** എൻഡ്‌പോയിന്റ് പ്രദർശിപ്പിക്കുന്നു, ഇത് പ്രാദേശിക വികസനത്തിനാണ്; പൂർണ്ണ Responses API ഫീച്ചർ സജ്ജീകരണത്തിനായി Azure OpenAI / Microsoft Foundry ഉപയോഗിക്കുക.
