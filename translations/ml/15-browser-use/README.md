# കമ്പ്യൂട്ടർ ഉപയോഗ ഏജന്റ്സ് (CUA) നിർമ്മിക്കുന്നത്

കമ്പ്യൂട്ടർ ഉപയോഗ ഏജന്റുകൾ一个 വ്യക്തി പോലെയും വെബ്സൈറ്റുകളുമായി ഇടപഴകുകയും ബ്രൗസർ തുറക്കുകയും പേജാണ് പരിശോധിക്കുകയും കണ്ടതിൽ നിന്ന് അടുത്ത മികച്ച നടപടി എടുക്കുകയും ചെയ്യും. ഈ പാഠത്തിൽ, നിങ്ങൾ ഒരു ബ്രൗസർ ഓട്ടോമേഷൻ ഏജന്റിനെ നിർമിക്കും, അത് Airbnb യിൽ തിരയുകയും ഘടനാപരമായ ലിസ്റ്റിംഗ് ഡാറ്റ സ്രോതസ്സാക്കുകയും സ്റ്റോക്ക്ഹോംയിൽ ഏറ്റവും കുറഞ്ഞ താമസം കണ്ടെത്തുകയും ചെയ്യും.

ഈ പാഠം AI-നൽകിയ നാവിഗേഷനിനായി Browser-Use, ബ്രൗസർ നിയന്ത്രണത്തിനായി Playwright, Chrome DevTools Protocol (CDP), ദൃശ്യ-सാധിതമായ കാരണം ദോഷണത്തിനായി Azure OpenAI, ഘടനാപരമായ സ്രോതസ്സിനായി Pydantic എന്നിവയുടെ കൂട്ട സംയോജനമാണ്.

## പരിചയം

ഈ പാഠം ഉൾക്കൊള്ളുന്നത്:

- API-മാത്രമായ ഓട്ടോമേഷൻക്കാൾ കമ്പ്യൂട്ടർ ഉപയോഗ ഏജന്റുകൾ എപ്പോൾ നല്ലതാണ് എന്ന് മനസ്സിലാക്കൽ
- വിശ്വാസযোগ্য ബ്രൗസർ ലൈഫ്‌സൈക്കിൾ മാനേജ്മെന്റിനായി Browser-Use ന് Playwright, CDP എന്നിവകൂടെ ചേർക്കൽ
- ഡൈനാമിക് വെബ് പേജുകളിൽ നിന്ന് ലിസ്റ്റിംഗ് ഡാറ്റ നേടാൻ Azure OpenAI ദൃശ്യമേഖലയും ഘടനാപരമായ Pydantic ഔട്ട്പുട്ടും ഉപയോഗിക്കൽ
- ഏജന്റ്-ഫസ്റ്റ്, ആക്ടർ-ഫസ്റ്റ്, ഹൈബ്രിഡ് ബ്രൗസർ ഓട്ടോമേഷൻ പ്രവൃത്തി പ്രക്രിയകളിൽ എപ്പോൾ ഉപയോഗിക്കണം എന്ന് തീരുമാനിക്കൽ

## പഠനലക്ഷ്യങ്ങൾ

ഈ പാഠം പൂർത്തിയാക്കിയശേഷം, നിങ്ങൾക്ക് അറിയാം:

- Azure OpenAI, Playwright എന്നിവയുമായി Browser-Use കോൺഫിഗർ ചെയ്യുന്നത്
- ഒരു യഥാർത്ഥ വെബ്സൈറ്റ് നാവിഗേറ്റ് ചെയ്യുകയും ഡൈനാമിക് UI ഘടകങ്ങൾ കൈകാര്യം ചെയ്യുകയും ചെയ്യുന്ന ബ്രൗസർ ഓട്ടോമേഷൻ പ്രവൃത്തി പ്രവണതി നിർമ്മിക്കുക
- ദൃശ്യമാകുന്ന പേജ് ഉള്ളടക്കത്തിൽ നിന്നുള്ള ടൈപ്പ് ചെയ്ത ഫലം സ്രോതസ്സാക്കിയും അതിനെ ബിസിനസ്സ് ലജിക്കിലേക്ക് മാറ്റുകയും ചെയ്യുക
- ബ്രൗസർ ടാസ്‌ക് എത്രമാത്രം പ്രവചനശേഷിയുള്ളതാണെന്ന് അടിസ്ഥാനപ്പെടുത്തി ഏജന്റ്, ആക്ടർ മാതൃകകൾ തിരഞ്ഞെടുക്കുക

## കോഡ് സാമ്പിൾ

ഈ പാഠത്തിൽ ഒരു നോട്ട്‌ബുക്ക് ട്യൂട്ടോറിയൽ ഉണ്ട്:

- [15-browser-user.ipynb](./15-browser-user.ipynb): CDP മേൽ ക്രോം സെഷൻ ആരംഭിക്കുകയും Airbnb യിൽ സ്റ്റോക്ക്ഹോം ലിസ്റ്റിംഗുകൾ തിരയുകയും Browser-Use ദൃശ്യമേഖല ഉപയോഗിച്ച് വിലകൾ സ്രോതസ്സാക്കുകയും ഏറ്റവും കുറഞ്ഞ ഓപ്ഷൻ ഘടനാപരമായ ഡാറ്റയായി തിരികെ നൽകുകയും ചെയ്യുന്നു.

## പ്രാരംഭ ആവശ്യങ്ങൾ

- Python 3.12+
- നിങ്ങളുടെ പരിതസ്ഥിതിയിൽ കോൺഫിഗർ ചെയ്ത Azure OpenAI ഡെപ്ലോയ്മെന്റ്
- ലോക്കലി ഇൻസ്റ്റാൾ ചെയ്ത Chrome അല്ലെങ്കിൽ Chromium
- പ്ലേവ്റൈറ്റിന് ആവശ്യമായ ഡിപെൻഡൻസികൾ ഇൻസ്റ്റാൾ ചെയ്തിട്ടുണ്ടാകണം
- അസിൻക്രൺ പൈത്തൺ അടിസ്ഥാന പരിചയവും

## ക്രമീകരണം

നോട്ട്‌ബുക്കിൽ ഉപയോഗിച്ച പാക്കേജുകൾ ഇൻസ്റ്റാൾ ചെയ്യുക:

```bash
pip install browser_use playwright python-dotenv
playwright install chromium
```

നോട്ട്‌ബുക്കിൽ ഉപയോഗിക്കുന്ന Azure OpenAI പരിസ്ഥിതി ചോദ്യങ്ങൾ സജ്ജമാക്കുക:

```bash
AZURE_OPENAI_ENDPOINT=...
AZURE_OPENAI_API_KEY=...
AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=...
# ഓപ്ഷണൽ: ഒഴിവാക്കിയാൽ最新 API പതിപ്പിലേക്ക് ഡിഫോൾട്ട് ആണ്
AZURE_OPENAI_API_VERSION=...
```

## ആർക്കിടെക്ചർ അവലോകനം

നോട്ട്‌ബുക്ക് സമ്പൂർണ്ണ ഹൈബ്രിഡ് ബ്രൗസർ ഓട്ടോമേഷൻ പ്രവൃത്തി പരമ്പരയെ കാണിക്കുന്നു:

1. Playwright, Browser-Use രണ്ടും ഒരേ ബ്രൗസർ സെഷനിൽ പങ്കിടാൻ CDP പ്രവർത്തനസജ്ജമാക്കിയ ക്രോം ആരംഭിക്കുന്നു.
2. Browser-Use ഏജന്റ് Airbnb തുറക്കൽ, പോപ്-അപ് തള്ളൽ, സ്റ്റോക്ക്ഹോം തിരയൽ പോലുള്ള ഓപ്പൺ-എൻഡഡ് നാവിഗേഷൻ ടാസ്കുകൾ കൈകാര്യം ചെയ്യുന്നു.
3. ആക്ടീവ് പേജ് ഘടനാപരമായ Pydantic സ്കീമ ഉപയോഗിച്ച് അസ്പഷ്ടം, രാത്രികാല വിലകൾ, റേറ്റിംഗുകൾ, URL ഇല استخراج ചെയ്തു.
4. പൈത്തൺ ലജിക്ക് സ്രോതസ്സാക്കിയ ലിസ്റ്റിംഗുകൾ താരതമ്യം ചെയ്തു ഏറ്റവും കുറഞ്ഞ ഫലം ഹൈലൈറ്റ് ചെയ്യുന്നു.

ഈ സമീപനം Browser-Use നയുടെ സൗകര്യമായ ദൃശ്യ അധിഷ്ഠിത കാരണം ചിട്ടപ്പെടുത്തപ്പെട്ട ബ്രൗസർ നിയന്ത്രണത്തോടൊപ്പം നൽകുന്നു.

## പ്രധാന ഗോൾകളും മികച്ച പ്രയോഗങ്ങളും

### ഏജന്റ് vs ആക്ടർ എപ്പോൾ ഉപയോഗിക്കണം

| സാഹചര്യം | ഏജന്റ് ഉപയോഗിക്കുക | ആക്ടർ ഉപയോഗിക്കുക |
|----------|-----------|-----------|
| ഡൈനാമിക് ലെയൗട്ടുകൾ | ശരി, AI പേജ് മാറ്റങ്ങൾക്കൊപ്പം ചേരാം | കാണാതെ പോകാൻ സാധ്യതയുള്ള സെലക്ടറുകൾ ഉപയോഗിക്കരുത് |
| അറിയപ്പെടുന്ന ഘടന | വേണ്ട, ഏജന്റ് നേരിട്ട് നിയന്ത്രപ്പെടുത്തുന്നതിനേക്കാൾ മന്ദഗതിയുള്ളവൻ | ശരി, വേഗവും കൃത്യവുമാണ് |
| ഘടകങ്ങൾ കണ്ടെത്തൽ | ശരി, സ്വാഭാവിക ഭാഷ നല്ല ഫലമേർപ്പെടുത്തുന്നു | വേണ്ട, കൃത്യമായ സെലക്ടറുകൾ ആവശ്യമാണ് |
| സമയ നിയന്ത്രണം | വേണ്ട, കുറവായി പ്രവചിക്കാവുന്നതാണ് | ശരി, കാത്തിരിപ്പിലും പുനരുദ്ധരണിലും പൂർണ്ണം നിയന്ത്രണം |
| സങ്കീർണ്ണ പ്രവൃത്തി പ്രവണതകൾ | ശരി, അപ്രതീക്ഷിത UI നിലകൾ കൈകാര്യം ചെയ്യുന്നു | വേണ്ട, വ്യക്തമായ ശാഖകൾ ആവശ്യമുണ്ട് |

### Browser-Use മികച്ച പ്രയോഗങ്ങൾ

1. ഗവേഷണത്തിനും ഡൈനാമിക് നാവിഗേഷനും ഏജന്റ് ഉപയോഗിച്ച് ആരംഭിക്കുക.
2. ഇടപെടൽ പ്രവചനശേഷിയുള്ളതിനാൽ നേരിട്ടുള്ള പേജ് നിയന്ത്രണത്തിലേക്ക് മാറുക.
3. ഘടനാപരമായ ഔട്ട്പുട്ട് മോഡലുകൾ ഉപയോഗിക്കുക, സ്രോതസ്സാക്കിയ ഡാറ്റ സാധൂകരിക്കുകയും ടൈപ്പ്-സെഫായിരിക്കണം.
4. കാഴ്ചക്ക് തീരുന്ന UI മാറ്റങ്ങൾക്ക് ശേഷം തന്ത്രപരമായി ദെലൈ ചേർക്കുക.
5. തകരാറുകൾ എളുപ്പത്തിൽ ഡിബഗ് ചെയ്യാൻ സ്‌ക്രീൻഷോട്ടുകൾ എടുക്കുക.
6. വെബ്സൈറ്റുകൾ മാറുന്നതിന്‍റെ സാധ്യത പ്രതീക്ഷിച്ച് പോപ്-അപ്പുകൾക്കും ലെയൗട്ട് മാറ്റങ്ങൾക്കുമായി ഫാൾബാക്ക് തന്ത്രങ്ങൾ രൂപപ്പെടുത്തിയിരിക്കുക.
7. ഉപയോഗിക്കാൻ സൗകര്യവാനും കൃത്യവുമായ നൈപുണ്യം കൈവരിക്കാൻ ഏജന്റ്, ആക്ടർ മാതൃകകൾ ചേർത്ത് ഉപയോഗിക്കുക.

### ബ്രൗസർ ഏജന്റ്സിനുള്ള സുരക്ഷാ മാർഗ്ഗരേഖകൾ

ബ്രൗസർ ഏജന്റുകൾ സജീവ വെബ്സൈറ്റുകളിൽ പ്രവർത്തിക്കുന്നതിനാൽ, അവർക്ക് ഒരു അറിയപ്പെട്ട API മാത്രം വിളിക്കുന്ന സ്‌ക്രിപ്റ്റുകളേക്കാൾ ഇടപെടലുകൾ കുറഞ്ഞ നിയന്ത്രണങ്ങൾ ആവശ്യമാണ്. ഒരു നോട്ട്‌ബുക്കിൽ നിന്നു യഥാർത്ഥ പ്രവൃത്തി പ്രവണതയിലേക്ക് മടങ്ങുമ്പോൾ ഏജന്റ് കാണാൻ, ക്ലിക്ക് ചെയ്യാൻ, സമർപ്പിക്കാൻ കഴിയുന്ന കാര്യങ്ങൾ നിയന്ത്രിക്കുക.

1. **ബ്രൗസിംഗ് പരിസ്ഥിതി പരിധി വരையുക.** ഏജന്റിനെ പ്രത്യേക ബ്രൗസർ പ്രൊഫൈലിൽ അല്ലെങ്കിൽ സാൻഡ്‌ബോക്സിൽ പ്രവർത്തിപ്പിക്കുക, ടാസ്‌കിനായി ആവശ്യമായ ഡൊമൈനുകളിൽ മാത്രമേ അനുവദിക്കരുത്.
2. **കാഴ്‌വയെയും പ്രവർത്തനത്തെയും വേർതിരിക്കുക.** ഏജന്റ് ആദ്യം തിരയുകയും വായിക്കുകയും ഡാറ്റ എടുത്തുകൊടുക്കുകയും ചെയ്യട്ടെ; ഫോമുകൾ സമർപ്പിക്കൽ, സന്ദേശങ്ങൾ അയയ്ക്കൽ, യാത്ര ബുക്കിംഗ്, വാങ്ങലുകൾ, റെക്കോർഡുകൾ ഇല്ലാതാക്കൽ, അക്കൗണ്ട് ക്രമീകരണങ്ങൾ മാറ്റൽ തുടങ്ങിയ കാര്യങ്ങൾക്ക് മുമ്പായി വ്യക്തമായ അംഗീകാരം ആവശ്യപ്പെടുക.
3. **രഹസ്യങ്ങൾ പ്രോംപ്റ്റുകളിലും ട്രേസുകളിലും നിന്നു അകറ്റുക.** പാസ്വേഡുകൾ, പണമടക്കൽ വിശദാംശങ്ങൾ, സെഷൻ കുക്കികൾ, അല്ലെങ്കിൽ കച്ചവട വ്യക്തിഗത ഡാറ്റ മോഡൽ സാഹചര്യത്തിൽ ഇടാതെ ഉപയോക്താവ് തിരിച്ചറിയലിനായി പ്രവർത്തിക്കണം, ലോഗുകളിൽ സൂക്ഷ്മാംശങ്ങൾ റിഡാക്ട് ചെയ്‌തിരിക്കണം.
4. **പേജ് ഉള്ളടക്കം അവിശ്വസനീയമായ ഇൻപുട്ടായി കാണുക.** ഒരു വെബ്സൈറ്റ് ഏജന്റിന് വേണ്ടി നിർദ്ദേശങ്ങൾ ഉൾക്കൊള്ളാമെങ്കിലും ഉപയോക്താവിന് അല്ല. ഏജന്റ് ലക്ഷ്യം മാറ്റുക, ഡാറ്റ വെളിപ്പെടുത്തുക, സുരക്ഷ പദ്ധതികൾ നിർത്തുക, അല്ലെങ്കിൽ അനുബന്ധവല്ലാത്ത സൈറ്റുകൾ സന്ദർശിക്കുവാൻ പേജിലെ ടെക്സ്റ്റ് നിർദ്ദേശങ്ങൾ പരിഗണിക്കരുത്.
5. **വിപത്തുകളുള്ള ഘട്ടങ്ങൾക്കു deterministic പരിശോധനകൾ നടത്തുക.** നിലവിലുള്ള URL, പേജ് ശീർഷകം, തിരഞ്ഞെടുക്കപ്പെട്ട ഇനം, വില, ലഭിക്കുന്നവ, പ്രവർത്തന സംഗ്രഹം കോഡിലൂടെ ഉറപ്പുവരുത്തി ഉപയോഗക്കാരന്റെ അംഗീകാരം തേടുക.
6. **ബജറ്റുകളും നിർത്തൽ നിബന്ധനകളും സജ്ജീകരിക്കുക.** ഏജന്റ് ഉപയോഗിക്കാവുന്ന പ്രവർത്തനങ്ങൾ, പുന:മുന്നേറ്റങ്ങൾ, ടാബുകൾ, മിനിറ്റുകൾ എന്നിവയുടെ എണ്ണം പരിധിതമാക്കുക. പേജ് സ്ഥിതി വ്യക്തതരഹിതമാകുമ്പോൾ ക്ലിക്കിംഗ് തുടരാതെ നിർത്തുക.
7. **അവശ്യമായ തെളിവുകൾ മാത്രമേ റെക്കോർഡ് ചെയ്യരുത്.** പ്രവർത്തന സംഗ്രഹങ്ങൾ, ടൈംസ്റ്റാമ്പുകൾ, URL-കൾ, തിരഞ്ഞെടുക്കപ്പെട്ട ഘടക വിവരണങ്ങൾ, സ്‌ക്രീൻഷോട്ട് സൂചനകൾ സൂക്ഷിക്കുക, അനാവശ്യ സ്മരണകളും സ്വകാര്യ പേജും ഉള്ളടക്കങ്ങളും സൂക്ഷിക്കാതിരിക്കുക.

Airbnb സാമ്പിളിൽ സുഖ സുരക്ഷിതമായ ഡീഫോൾട്ടാണ് ലിസ്റ്റിംഗുകൾ തിരയുകയും വിലകൾ സ്രോതസ്സാക്കുകയും ചെയ്യുന്നത്. സൈൻ ഇൻ ചെയ്യൽ, ഹോസ്റ്റിൽ ബന്ധപ്പെടൽ അല്ലെങ്കിൽ ബുക്കിംഗ് പൂർത്തീകരിക്കൽ ഉപയോക്താവ് അംഗീകരിക്കുന്ന വേർതിരിച്ച പ്രവര്‍ത്തനങ്ങൾ ആയിരിക്കണം.

### യഥാർത്ഥ ലോക പ്രയോഗങ്ങൾ

- യാത്രാ ബുക്കിംഗ്, വില നിരീക്ഷണം
- ഇ-കൊമേഴ്‌സ് വില താരതമ്യം, ലഭ്യത പരിശോധിക്കൽ
- ഡൈനാമിക് വെബ്സൈറ്റുകളിൽ നിന്നുള്ള ഘടനാപരമായ സ്രോതസ്സാക്കൽ
- ദൃശ്യജ്ഞാനം അടിസ്ഥാനമാക്കിയുള്ള UI പരിശോധന, സ്ഥിരീകരണം
- വെബ്സൈറ്റ് നിരീക്ഷണം, അലർട്ട്
- ബഹുവിധ പ്രക്രിയകളിൽ ബുദ്ധിമുട്ടില്ലാത്ത ഫാം നിറച്ച് നൽകൽ

## യഥാർത്ഥ ലോക ഉദാഹരണം: Microsoft Project Opal

ഈ പാഠത്തിൽ നിങ്ങൾ നിർമ്മിക്കുന്ന ഏജന്റ് ഒരു ചെറുയിട, പ്രാദേശിക **കമ്പ്യൂട്ടർ ഉപയോഗ ഏജന്റ് (CUA)** ആണ് — ഒരു ആളുപോലെ ബ്രൗസർ നിയന്ത്രിക്കുന്ന പ്രോഗ്രാം. Microsoft ഈ ആശയം സംരംഭ ലോകത്ത് കൊണ്ട് വരുന്നത് **[Project Opal (Frontier)](https://support.microsoft.com/en-us/microsoft-365-copilot/get-started-with-project-opal-frontier)** മൈക്രോസോഫ്റ്റ് 365 കോപൈലറ്റിലെ ഒരു കഴിവായി.

Project Opal-ൽ നിങ്ങൾ പ്രവർത്തി വിവരിച്ച് ഏജന്റ് നിങ്ങളുടെ പക്ഷത്തേയ്ക്ക് പ്രവർത്തിക്കുന്നു **സുരക്ഷിത Windows 365 Cloud PC-യിലുള്ള കമ്പ്യൂട്ടർ ഉപയോഗം** ഉപയോഗിച്ച്, നിങ്ങളുടെ സംഘടനയിലെ ബ്രൗസർ അധിഷ്ഠിത ആപ്ലിക്കേഷനുകൾ, സൈറ്റുകൾ, ഡാറ്റകൾ എന്നിവയ്ക്കിടയിലായി പ്രവർത്തിക്കുന്നു. ഇത് **പശ്ചാത്തലത്തിൽ അസിൻക്രണസായി പ്രവർത്തിക്കുന്നു**, നിങ്ങൾക്ക് എപ്പോഴെങ്കിലും പ്രവർത്തി നയിക്കാനോ നിയന്ത്രണം എടുക്കാനോ കഴിയും. ഉദാഹരണ ജോലികൾ:

- സുരക്ഷാ ഗൃഹാംഗ അംഗത്വ അപേക്ഷകൾ കൈകാര്യംചെയ്യൽ
- അന്വയം റിവ്യൂവുകൾക്കായി ഓഡിറ്റ് തെളിവുകൾ ശേഖരിക്കുകയും ശരിയാക്കുകയും ചെയ്യുക
- ഐടി സംഭവങ്ങൾ ത്രിയാജ് ചെയ്യുക (ടിക്കറ്റ് നില പുതുക്കൽ, ഉടമകൾ നിയോഗിക്കൽ, പകർപെട്ട ചേർക്കലുകൾ അടയ്ക്കൽ)
- എക്സെൽ ഡാറ്റ സാമ്പത്തിക ക്ലോസ് ഡെക്കിലേക്ക് സംയോജിപ്പിക്കൽ

Opal ഒരു **ഉത്പാദന നിലവാരമുള്ള, വിശ്വസനീയമായ** കമ്പ്യൂട്ടർ ഉപയോഗ ഏജന്റിന്റെ ഉദാഹരണമാണ് — മുൻപുള്ള പാഠങ്ങളിൽ നിന്നുള്ള ആശയങ്ങൾ ശക്തിപ്പെടുത്തുന്നു:

| ഈ കോഴ്സിലെ ആശയം | Project Opal എങ്ങനെ പ്രയോഗിക്കുന്നു |
|------------------------|-----------------------------|
| **മനുഷ്യൻ-ഇൻ-ദ-ലൂപ്പ്** (പാഠം 06) | ലോഗിൻ ക്രെഡൻഷ്യലുകൾക്കായി, gevo-sensitive ഡാറ്റയ്‌ക്കായി, അനിശ്ചിത നിർദ്ദേശങ്ങൾക്കായി Opal കഴിയുമ്പോൾ തൽക്കാലം നിർത്തുന്നു, പാസ്വേഡുകൾ നൽകുകയോ ഫോമുകൾ സമർപ്പിക്കുകയോ ഒരാത്മീയ സ്ഥിരീകരണമില്ലാതെ ചെയ്യാറില്ല. നിങ്ങൾക്ക് പ്രവർത്തി മധ്യേ *Take Control* ചെയ്യാനും *Return Control* ചെയ്യാനുമാകും. |
| **വിശ്വസനീയവും സുരക്ഷിതവും ആയ ഏജന്റുകൾ** (പാഠങ്ങൾ 06 & 18) | ഒറ്റപ്പെട്ട Windows 365 Cloud PC-യിൽ പ്രവർത്തിക്കുന്നു, മൊത്തം ബ്രൗസർ മാത്രം ഉപയോഗിക്കുന്നു (മറ്റ് കമ്പ്യൂട്ടർ പ്രവേശനം തടയുന്നു, Intune ഉപയോഗിച്ച് നടക്കുന്നു), നിങ്ങളുടെ ഐഡന്റിറ്റി ഉപയോഗിച്ച് നിങ്ങള്ക്ക് അനുമതി ഉണ്ടായ സ്ഥലങ്ങൾ മാത്രമേ പ്രവേശിക്കുകയുള്ളൂ, ഓരോ നടപടി ഓഡിറ്റിന് ലോഗ് ചെയ്യുന്നു. |
| **പരിചയപ്പെട്ടതും മെറ്റാകോഗ്നിഷനും** (പാഠങ്ങൾ 07 & 09) | Opal ആദ്യം ജോലി പ്ലാൻ സൃഷ്ടിക്കുന്നു, എന്നാൽ ഓരോ ഘട്ടത്തിലും തന്റെ കാരണം സ്വയം ഉദ്ദേശിച്ച് സംശയാസ്പദ പ്രവർത്തനം കണ്ടെത്തിയാൽ നിർത്തുന്നു. |
| **പുനരുപയോഗയോഗ്യമായ കഴിവുകൾ / ഉപകരണങ്ങൾ** (പാഠം 04) | **Skills** നിങ്ങളെ ആവർത്തനത്താഴങ്ങളിൽ നിർദ്ദേശങ്ങൾ എഴുതാനും (ഒരു `.md` ഫയലിൽ നിന്നും ഇറക്കുമതി ചെയ്യപ്പെട്ടതോ Opal-ൽ സ്വയം എഴുതി സംഭരിച്ചതോ) സംഭാവന നൽകാനും ഇടവരുത്തുന്നു. |

> **ലഭ്യത:** Project Opal നിലവിലെത്തിയത് Microsoft 365 Copilot സബ്സ്ക്രിപ്ഷനോടുള്ള [Frontier തുടങ്ങൽ പ്രോഗ്രാമിലുള്ള](https://adoption.microsoft.com/copilot/frontier-program/) ഉപയോക്താക്കൾക്കായി, നിങ്ങളുടെ അഡ്മിൻ ക്രമീകരണം പൂർത്തിയാക്കണം. ഇത് പരീക്ഷണാത്മക Frontier ഫീച്ചറായതിനാൽ കഴിവുകൾ സമയാനുസരിച്ച് മാറ്റമായേക്കാം.
