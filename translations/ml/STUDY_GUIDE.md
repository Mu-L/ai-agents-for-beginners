# ആരംഭിക്കാനാകുന്നവർക്കുള്ള AI ഏജൻറുകൾ - പഠന ഗൈഡ്

കോഴ്സ് മുൻപേ ചെല്ലുമ്പോൾ ഈ ഗൈഡ് പ്രായോഗിക കൂട്ടായി ഉപയോഗിക്കുക. ഇത് പാഠങ്ങൾക്ക് പകരം മാത്രം അല്ല. ഇത് നിങ്ങൾക്ക് എവിടെ തുടങ്ങണമെന്നും, ഓരോ പാഠത്തിൽ എന്ത് കാണണമെന്നും, ആശയങ്ങൾ ചെറിയ പ്രവർത്തിക്കുന്ന ഏജൻറ് ഡെമോയിലേക്ക് എങ്ങനെ ബന്ധിപ്പിക്കാമെന്നും തീരുമാനിക്കാൻ സഹായിക്കും.

ഇത് പാഠങ്ങൾ മാറ്റിവെക്കാനല്ല. ഇത് നിങ്ങൾക്ക് തുടങ്ങേണ്ടിടം, ഓരോ പാഠത്തിൽ തിരയേണ്ടതും, ആശയങ്ങളെ ചെറിയ പ്രവർത്തന ഏജൻറ് ഡെമോയിലേക്ക് ബന്ധിപ്പിക്കുന്ന വിധം സഹായിക്കുന്നു.


ഇത് നിങ്ങളുടെ ആദ്യമായിട്ടാണ് ഇതിൽ വന്നത് എങ്കിൽ, ലളിതമായി ആരംഭിക്കുക:

1. [Course Setup](./00-course-setup/README.md) വായിക്കുക.
2. ലേസൺ 01-06 ക്രമത്തിൽ പൂർത്തിയാക്കുക.
3. പഠിക്കുമ്പോൾ ഒരു ചെറിയ ഡെമോ ആശയം മനസ്സിൽ വയ്ക്കുക.
4. ഓരോ പാഠത്തിനുശേഷവും ചോദിക്കുക: “എന്റെ ഏജൻറ് ഇനി എന്ത് ചെയ്യാൻ കഴിയും, മുമ്പ് ചെയ്യാൻ കഴിയാതിരുന്നത് എന്താണ്?”


## മനസ്സിൽ വയ്ക്കാനുള്ള ലളിതമായ ഒരു ഡെമോ

ഏജൻറുകളെ പഠിക്കാൻ ഒരൊറ്റ ഡെമോ ആശയം കോഴ്സിന്റെ മുഴുവൻ ഭാഗം പിന്തുടരുക നല്ല മാർഗമാണ്.

ഉദാഹരണ ഡെമോ: **ഒരു കോഴ്സ് സഹായക ഏജൻറ്**.

ഉപഭോക്താവ് ചോദിക്കുന്നു:

> "എഞ്ചിനീയറിംഗ് എങ്ങനെ ടൂൾസ് ഉപയോഗിക്കുന്നു എന്നതിൽ പഠിക്കാൻ ഞാൻ ആഗ്രഹിക്കുന്നു. ശരിയായ പാഠങ്ങൾ കണ്ടെത്തുക, ഞാൻ ആദ്യം എന്തു വായിക്കണം എന്ന് സംഗ്രഹിക്കുക, എനിക്ക് ചെറിയ പരിശീലന പ്രവർത്തി നൽകുക."


ഒരു സാദാരണ ചാറ്റ്‌ബോട്ട് ഇതിനകം അറിയുന്നതിൽ നിന്ന് മറുപടി നൽകാം. എന്നാൽ ഒരു ഏജൻറ് കൂടുതൽ ചെയ്യാൻ കഴിയും:

1. **കോഴ്‌സ് ഫയലുകൾ വായിക്കുക അല്ലെങ്കിൽ തിരയുക** ശരിയായ പാഠങ്ങൾ കണ്ടെത്താൻ.
2. **ടൂളുകൾ ഉപയോഗിക്കുക** പാഠങ്ങളുടെ ലിങ്കുകൾ, ഉദാഹരണങ്ങൾ, അല്ലെങ്കിൽ പിന്തുണ വിദ്യാർത്ഥി സാമഗ്രി ലഭിക്കാൻ.
3. **പ്ലാൻ ചെയ്യുക** ഒരു ദൈർഘ്യമേറിയ ഉത്തരത്തിന് പകരം ചെറിയ പഠന പാത.
4. **സന്ദർഭം ഉപയോഗിക്കുക** ഇപ്പോഴത്തെ സംഭാഷണത്തിൽ നിന്ന്, പഠനാർത്ഥിയുടെ ലക്ഷ്യത്തിൽ ശ്രദ്ധ കേന്ദ്രീകരിക്കാൻ.

5. **ഉപയോഗപ്രദമായ മുൻഗാമികളുള്ള ഓർമ്മകൾ സൂക്ഷിക്കുക** അപ്ലിക്കേഷൻ മെമ്മറി പിന്തുണയ്ക്കുമ്പോൾ.
6. **ട്രേസ്, ഉദ്ധരണികൾ, അല്ലെങ്കിൽ ലോഗുകൾ കാണിക്കുക** ഉപയോക്താവിന് എന്താണെന്ന് മനസ്സിലാക്കാൻ.
7. **ഗാർഡ്‌റെയിൽസ് ഉപയോഗിക്കുക** അപകടകരമായ പ്രവർത്തനങ്ങൾ ചെയ്യുന്നതിന് മുൻപ് അല്ലെങ്കിൽ സാന്ദ്രമായ ഡാറ്റ ഉപയോഗിക്കുന്നതിനുമുമ്പ്.

ഓരോ പാഠവും പഠിക്കുമ്പോളും, ഈ ഡെമോയിലേക്ക് മടങ്ങി ചോദിക്കുക: ഈ പാഠം ഏത് പുതിയ കഴിവുകൾ ചേർക്കും?


## നിങ്ങൾ നിർമ്മിക്കാൻ പോവുന്നത്

## മോഡലുകളും പ്രൊവൈഡറുകളും

കോഴ്സ് കോഡ് ഉദാഹരണങ്ങൾ **Microsoft Agent Framework (MAF)** ഉപയോഗിക്കുകയും ലക്ഷ്യമിടുന്നത് **Azure OpenAI Responses API** ആണ് — ഇതാണ് ഇനി മുന്നോട്ട് വെക്കപ്പെടുന്ന API, ഇത് ചാറ്റ് പൂർത്തീകരണങ്ങൾ, ടൂൾ കോല്ലിങ്, മൾട്ടിമോഡൽ ഇൻപുട്ട്, സ്റ്റേറ്റ്ഫുൾ സംഭാഷണങ്ങൾ ഒരേ API സ്ര്ഫേസിൽ ചേർക്കുന്നു. നിങ്ങൾ either **Microsoft Foundry** പ്രൊജക്ടിലൂടെ (`FoundryChatClient`) അല്ലെങ്കിൽ Azure OpenAI നേരിട്ട് (`OpenAIChatClient`) വഴി കണക്ട് ചെയ്യാം.


പാഠങ്ങൾ നടക്കുന്നതിനിടെ, നിങ്ങൾക്ക് ചില പ്രൊവൈഡർ ഓപ്ഷനുകൾ ഉണ്ട്:

- **Microsoft Foundry / Azure OpenAI (Responses API)** — പാഠങ്ങൾ മുഴുവൻ ഉപയോഗിക്കുന്ന പ്രധാന വഴിയാണ്. കീ ഇല്ലാതെ Entra ID പ്രമാണീകരണത്തിന് `az login` ഉപയോഗിച്ച് സൈൻ ഇൻ ചെയ്യുക.
- **Foundry Local** — OpenAI- അനുസൃത API വഴിയാണ് മോഡലുകൾ সম্পൂർണ്ണമായി ഡിവൈസിൽ പ്രവർത്തിക്കുന്നത് (ക്ളൗഡ് ഇല്ല, API കീകൾ ഇല്ല). ഓഫ്ലൈൻ അല്ലെങ്കിൽ ചെലവിലേറാത്ത പരീക്ഷണങ്ങൾക്ക് അനുയോജ്യം. കാണുക [Course Setup](./00-course-setup/README.md).
- **MiniMax** — OpenAI- അനുസൃത വലിയ കോൺടെക്സ്റ്റ് മോഡലുകൾ ഉപയോഗിക്കുന്ന പ്രൊവൈഡർ, ഒരു ഡായ്പ്-ഇൻ പകരമായി ഉപയോഗിക്കാൻ കഴിയുന്ന.

> **കുറിപ്പ്:** GitHub മോഡലുകൾ അവസാനഘട്ടത്തിലാണ് (ജൂലൈ 2026-ന് വിരമിക്കും) കൂടാതെ Responses API പിന്തുണയില്ല. സാമ്പിൾകൾ Azure OpenAI / Microsoft Foundry ഉപയോഗിച്ച് അപ്ഡേറ്റ് ചെയ്യപ്പെട്ടിരിക്കുന്നു.

പാഠത്തിൽ, ഉറവിട സാമഗ്രികൾ പാഠ READMEs, കോഡ് ഉദാഹരണങ്ങൾ, അല്ലെങ്കിൽ പാഠങ്ങളിൽ നിന്നും ബന്ധപ്പെട്ട
ബാഹ്യ ശ്രോതസ്സുകൾ ആകാം.

ഉത്തരം രേഖകൾ, ഡാറ്റ, അല്ലെങ്കിൽ നിലവിലുള്ള പ്രോജക്ട് ഫയലുകളിൽ അടിസ്ഥാനമാക്കിയിരിക്കേണ്ടപ്പോൾ RAG ഉപയോഗിക്കുക.


### പദ്ധതികൽ‌പന

കാര്യങ്ങളുടെ ആവശ്യം ഒന്നിലധികം ഘട്ടങ്ങളുള്ളപ്പോൾ പദ്ധതികൽ‌പന പ്രയോജനകരമാണ്. പദ്ധതികൾ ചുരുങ്ങിയതും
ഡെവലപ്പർ അല്ലെങ്കിൽ ഉപയോക്താവ് പരിശോധിക്കാൻ ആകാവുന്നത്ര വ്യക്തവുമാകണം.

ഡെമോയ്ക്ക്, ഒരു പദ്ധതി ഇതുപോലെയാകാം:

1. ഉപകരണം ഉപയോഗിക്കുന്നതുമായി ബന്ധപ്പെട്ട പാഠങ്ങൾ കണ്ടെത്തുക.
2. ഏറ്റവും പ്രസക്തമായ പാഠങ്ങൾ സംഗ്രഹിക്കുക.
3. ഒരു പരിശീലന പ്രവർത്തി ശുപാർശ ചെയ്യുക.

### സാഹചര്യങ്ങൾ

സാഹചര്യമെങ്കിലും മോഡൽ ഇപ്പോൾ കാണുന്നതാണ്. വളരെ കുറവ് സാഹചര്യത്തെ തുടർന്ന് ഏജന്റ്
പ്രധാനപ്പെട്ട വിശദാംശങ്ങൾ ശ്രദ്ധിക്കാൻ പറ്റാതെ പോകാം. ഏറെ സസ്പംഘമുണ്ടെങ്കിൽ ഏജന്റ് മന്ദഗതിയിലും,
ചെലവിൽ, അല്ലെങ്കിൽ പാഴ്‌പെടുത്തലിൽ കൂടുതൽ വിപ്ലവം ഉണ്ടാക്കാം.

നല്ല സാഹചര്യ എഞ്ചിനീയറിങ് എന്നത് അടുത്ത മോഡൽ കോൾക്കായി ശരിയായ വിവരങ്ങൾ തിരഞ്ഞെടുക്കുന്നതാണ്.


### ഓർമ്മ

ഓർമ്മ ശേഷി പിന്നീട് ഉപയോഗിക്കാനായി സൂക്ഷിച്ചിരിക്കുന്ന വിവരങ്ങളാണ്. എല്ലാം സൂക്ഷിക്കരുത്.
ആ വിവരം ഉപയോഗപ്രദവും, സുരക്ഷിതവുമായിരിക്കുമ്പോൾ മാത്രമേ സൂക്ഷിക്കാവൂ; അപ്ഡേറ്റ് ചെയ്യാനും ഇല്ലാതാക്കാനും എളുപ്പമുള്ളവയാവണം.

ഉദാഹരണത്തിന്, "അഭിപ്രായകാരൻ പൈതൺ ഉദാഹരണങ്ങൾ ഇഷ്ടപ്പെടുന്നു" ഓർമ്മിക്കുക പ്രയോജനകരമായിരിക്കാം.
വ്യക്തിഗത സെൻസിറ്റീവ് ഡാറ്റ ഓർമ്മിക്കുക സാധാരണയായി വേണ്ടതാണ്.

### മൂല്യനിർണ്ണയം, കാണാനാവുക

മൂല്യനിർണ്ണയം ചോദിക്കുന്നു: ഏജന്റ് ശരിയായ കാര്യങ്ങൾ ചെയ്തു എന്നോ?

കാണാനാവുക ചോദിക്കുന്നു: അത് എങ്ങനെയായി നടന്നു എന്ന് നാം കാണാനാവുന്നോ?

പ്രൊഡക്ഷൻ ഏജന്റുകൾക്കായി മോഡൽ കോൾസ്, ഉപകരണ കോൾസ്, ലഭിച്ച സാഹചര്യങ്ങൾ,
വൈകി, ചെലവ്, പരാജയങ്ങൾ, ഉപയോക്തൃ പ്രതികരണം എന്നിവ നിരീക്ഷിക്കുക.

### വിശ്വാസ്യതയും സുരക്ഷയും

വിശ്വസനീയമായ ഏജന്റുകൾക്ക് സഹായകരമായ പ്രോംപ്റ്റിനക്കമല്ല. കുറഞ്ഞ പരിമിതിയുള്ള ഉപകരണങ്ങൾ,
ഉയർന്ന സ്വാധീനമുള്ള പ്രവർത്തനങ്ങൾക്കായി മനുഷ്യനോടുള്ള അംഗീകാരം, ആവശ്യമായ സ്ഥലങ്ങളിൽ ഡാറ്റ റീഡക്ഷൻ,
നിർണ്ണായക നടപടികൾക്ക് ലോഗുകളും രേഖകളും ഉപയോഗിക്കുക.

## 15 മിനിറ്റ് റിവ്യൂ റൂട്ടീൻ

ഓരോ പാഠത്തിനും ശേഷം ഈ റൂട്ടീൻ ഉപയോഗിക്കുക:

1. **ഒരു വാക്യത്തിൽ പാഠം സംഗ്രഹിക്കുക.**
2. **പുതിയ ഏജന്റ് കഴിവ് നാമകരണം ചെയ്യുക.** ഉദാഹരണം: ഉപകരണം ഉപയോഗം, തിരയൽ,
   പദ്ധതികൽ‌പന, ഓർമ്മ, കാണാനാവുക, അല്ലെങ്കിൽ സുരക്ഷ.
3. **അത് കോഴ്‌സ് സഹായി ഡെമോയിലേക്ക് ചേർക്കുക.** ഡെമോയിൽ ഇപ്പോൾ എന്ത് മാറ്റം സംഭവിച്ചു?
4. **ആപത്ത് കണ്ടെത്തുക.** ഈ കഴിവ് തെറ്റായ ഉപയോഗം കാരണം എന്ത് തെറ്റ് സംഭവിക്കാം?
5. **ഒരു പരിശോധന ചോദ്യം എഴുതുക.** ഏജന്റ് നല്ല പെരുമാറ്റം കാണിക്കുന്നുണ്ടെന്ന് നിങ്ങൾ എങ്ങനെ ഉറപ്പുവരുത്തും?

## ത്വരിത സ്വയം പരിശോധന

മുന്നോട്ട് പോകുന്നതിന് മുമ്പ്, ഈ ചോദ്യങ്ങൾക്ക് ഉത്തരം നൽകാൻ ശ്രമിക്കുക:

1. ഒരു ഏജന്റ് സാധാരണ ചാറ്റ്ബോട്ട് സ്വയം ചെയ്യാനാകാത്ത എന്തെല്ലാം ചെയ്യാം?
2. നിങ്ങളുടെ ഏജന്റിന് ആദ്യം ഏത് ഉപകരണമാകും ആവശ്യം, എന്തുകൊണ്ട്?
3. ഏജന്റിന്റെ ഉത്തരം ഏതൊരിടത്തെയാണ് നോക്കി ഉറപ്പു വരുത്തേണ്ടത്?
4. അടുത്ത മോഡൽ കോൾൽ ഏത് സാഹചര്യമാണു അടങ്ങേണ്ടത്?

5. ഏജന്റ് എന്ത് ഓർമ്മിക്കണം, എന്ത് സൂക്ഷിക്കുന്നത് ഒഴിവാക്കണം?
6. ഏജന്റ് മനുഷ്യയുടെ അംഗീകാരം 언제 ചോദിക്കണം?
7. ഏജന്റ് പിന്നീട് ഡിബഗ് ചെയ്യാനോ ഓഡിറ്റ് ചെയ്യാനോ സഹായിക്കുന്ന ലോഗുകൾ, ട്രേസുകൾ, അല്ലെങ്കിൽ രസീത് എന്തെല്ലാം ആണ്?


## നിർദ്ദേശിച്ച ക്യാപ്സ്ടോൺ വ്യായാമം

കോഴ്‌സ് അവസാനത്തിൽ, ഒരു ചെറിയ ഏജന്റ് നിർമ്മിക്കുക, അത് പഠനാർത്ഥിയ്ക്ക് ഈ
റിപോസിറ്ററി നാവിഗേറ്റ് ചെയ്യാൻ സഹായിക്കും.

കുറഞ്ഞ പതിപ്പ്:

- ഉപയോക്തൃo കുറഞ്ഞ വിഷയം സ്വീകരിക്കുക.
- ഏറ്റവും പ്രാസക്തമായ പാഠങ്ങൾ കണ്ടെത്തുക.
- ആദ്യം എന്ത് വായിക്കണമെന്ന് സംക്ഷേപിക്കുക.
- ഒരു പ്രായോഗിക പരിശീലന പ്രവര്‍ത്തനം sujjest ചെയ്യുക.
- ഉപയോഗിച്ച പാഠ ഫയലുകളും ലിങ്കുകളുമെല്ലാം കാണിക്കുക.

വികസിത പതിപ്പ്:

- പഠനാർത്ഥിയുടെ ഇഷ്ടപ്പെട്ട പ്രോഗ്രാമിംഗ് ഭാഷ ഓർത്തുപിടിക്കുക.
- മറുപടി നൽകുന്നതിനു മുമ്പ് ലളിതമായ ഒരു പദ്ധതിയൊരുക്കുക.
- അന്തിമ പ്രതികരണം നൽകുന്നതിനു മുൻപ് സ്വയം പരിശോധനാ ഘട്ടം ചേർക്കുക.
- ഉപകരണം വിളികളും ലഭിച്ച ഉറവിടങ്ങളും ലോഗ് ചെയ്യുക.
- ബ്രൗസർ അല്ലെങ്കിൽ UI ഓട്ടോമേഷന്‍ പ്രവർത്തനങ്ങൾ തുടങ്ങുന്നതിന് മുമ്പ് സ്ഥിരീകരണം ചോദിക്കുക.

ഇത് ഒരു ചെറിയ പക്ഷേ യാഥാർത്ഥ്യമായ രീതിയിൽ ഉപകരണങ്ങൾ, RAG, പദ്ധതിയിടൽ,
സാന്ദർഭ്യം, ഓർമ്മ, വിദഗ്ദ്ധം കാണൽ, വിശ്വാസം എന്നിവ ഒരே പ്രൊജക്ടിൽ അഭ്യാസം ചെയ്യാൻ ആണ്.
