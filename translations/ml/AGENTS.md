## സജ്ജീകരണ കമാൻഡുകൾ

### ആവശ്യകതകൾ
- പൈത്തൺ 3.12 അല്ലെങ്കിൽ അതിനുമുകളിൽ
- Azure സബ്‌സ്ക്രിപ്ഷൻ (Microsoft Foundry-ക്കായി)
- Azure CLI ഇൻസ്റ്റാൾ ചെയ്ത് ഓർത്തентиക്കേറ്റ് ചെയ്തിരിക്കണം (`az login`)

### ആദ്യ സജ്ജീകരണം

1. **റിപോസിറ്ററി ക്ലോൺ ചെയ്യുക അല്ലെങ്കിൽ ഫോർക്ക് ചെയ്യുക:**
   ```bash
   gh repo fork microsoft/ai-agents-for-beginners --clone
   # അല്ലെങ്കിലും
   git clone https://github.com/microsoft/ai-agents-for-beginners.git
   cd ai-agents-for-beginners
   ```

2. **Python വിൻചുറ്ററ നടത്തുക:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # Windows-ൽ: venv\Scripts\activate
   ```

3. **അവശ്യമായ പാക്കേജുകൾ ഇൻസ്റ്റാൾ ചെയ്യുക:**
   ```bash
   pip install -r requirements.txt
   ```

4. **പരിസ്ഥിതി ചാരങ്ങൾ സജ്ജമാക്കുക:**
   ```bash
   cp .env.example .env
   # നിങ്ങളുടെ API കീകളും എന്‍ഡ്പോയിന്റുകളും ചേര്‍ത്തുപയോഗിക്കാന്‍ .env ഫയല്‍ എഡിറ്റ് ചെയ്യുക
   ```

### ആവശ്യമായ പരിസ്ഥിതി ചാരങ്ങൾ

**Microsoft Foundry** (ആവശ്യമാണ്):
- `AZURE_AI_PROJECT_ENDPOINT` - Microsoft Foundry പ്രോജക്ട് എന്റ്പോയിന്റ്
- `AZURE_AI_MODEL_DEPLOYMENT_NAME` - മോഡൽ ഡിപ്ലോയ്മെന്റ് നാമം (ഉദാ: gpt-5-mini)

**Azure AI Search** (പാഠം 05 - RAG) വേണ്ടി:
- `AZURE_SEARCH_SERVICE_ENDPOINT` - Azure AI Search എന്റ്പോയിന്റ്
- `AZURE_SEARCH_API_KEY` - Azure AI Search API കീ

ഓർത്തентиക്കേഷൻ: നോട്ട്‌ബുക്കുകൾ പ്രവർത്തിപ്പിക്കുന്നതിന് മുമ്പ് `az login` ഓടിക്കുക (ഉപയോഗിക്കുന്നു `AzureCliCredential`).

## ഡെവലപ്പ്മെന്റ് പ്രവൃത്തിപ്രവാഹം

### Jupyter നോട്ട്‌ബുക്കുകൾ പ്രവർത്തിപ്പിക്കുക

ഓരോ പാഠത്തിനും വ്യത്യസ്ത ഫ്രെയിംവർക്കുകൾക്കായി നിരവധി Jupyter നോട്ട്‌ബുക്കുകൾ ഉണ്ട്:

1. **Jupyter ആരംഭിക്കുക:**
   ```bash
   jupyter notebook
   ```

2. **ഒരു പാഠ ഡയറക്ടറിയിലേയ്ക്ക് പോവുക** (ഉദാ: `01-intro-to-ai-agents/code_samples/`)

3. **നോട്ട്‌ബുക്കുകൾ തുറന്ന് പ്രവർത്തിപ്പിക്കുക:**
   - `*-python-agent-framework.ipynb` - Microsoft Agent Framework ഉപയോഗിച്ച് (Python)
   - `*-dotnet-agent-framework.ipynb` - Microsoft Agent Framework (.NET) ഉപയോഗിച്ച്

### Microsoft Agent Framework ഉപയോഗിച്ച് പ്രവർത്തിക്കുന്നത്

**Microsoft Agent Framework + Microsoft Foundry:**
- Azure സബ്‌സ്ക്രിപ്ഷൻ ആവശ്യം
- Agent Service V2-യ്ക്ക് `FoundryChatClient` ഉപയോഗിക്കുന്നു (Foundry പോർട്ടലിൽ ഏജന്റുകൾ കാണാം)
- ഇൻബിൽറ്റ് ഓബ്‌സർവബിലിറ്റിയുമായി പ്രൊഡക്ഷൻ റെഡി
- ഫയൽ പേട്ടേൺ: `*-python-agent-framework.ipynb`

## പരീക്ഷണ നിർദ്ദേശങ്ങൾ

ഇത് ഒരു വിദ്യാഭ്യാസ റിപോസിറ്ററിയാണ്, സ്വയംപരീക്ഷണങ്ങളുള്ള പ്രൊഡക്ഷൻ കോഡ് അല്ല. നിങ്ങളുടെ സജ്ജീകരണവും മാറ്റങ്ങളും പരിശോധിക്കാനായി:

### മാനുവൽ ടെസ്റ്റിംഗ്

1. **Python പരിസ്ഥിതി പരിശോധന:**
   ```bash
   python --version  # 3.12+ ആകണം
   pip list | grep -E "(agent-framework|azure-ai|azure-identity)"
   ```

2. **നോട്ട്‌ബുക്ക് നിർവഹണ പരിശോധന:**
   ```bash
   # നോട്ട് ബുക്ക് സ്ക്രിപ്റ്റായി മാറ്റി ഓടിക്കുക (ടെസ്റ്റുകൾ ഇറക്കുമതി ചെയ്യുന്നു)

   ```

### വ്യക്തിഗത നോട്ട്‌ബുക്കുകൾ പ്രവർത്തിപ്പിക്കൽ

Jupyter-ൽ നോട്ട്‌ബുക്കുകൾ തുറന്ന് ക്രമത്തിലായി സെല്ലുകൾ പ്രവർത്തിപ്പിക്കുക. ഓരോ നോട്ട്‌ബുക്കും സ്വയം സമ്പൂർണമാണ്, കൂടാതെ:
- ഇമ്പോർട്ട് സ്റ്റേറ്റ്മെന്റുകൾ
- കോൺഫിഗറേഷൻ ലോഡിംഗ്
- ഉദാഹരണം ഏജന്റ് നടപ്പാക്കലുകൾ
- മാർക്ക്ഡൗൺ സെല്ലുകളിൽ പ്രതീക്ഷിക്കുന്ന ഔട്ട്പുട്ടുകൾ

### നിർമ്മിത ഏജന്റുകളുടെ സ്മോക്ക്-ടെസ്റ്റിംഗ്

Microsoft Foundry ഹോസ്റ്റുചെയ്യുന്ന ഏജന്റായി ഡിപ്ലോയ്മെന്റ് ഉള്ള പാഠങ്ങളിൽ (01, 04, 05, 16), റിപോ `tests/` എന്ന ഡയറക്ടറിയിൽ സ്മോക്ക്-ടെസ്റ്റ് കാറ്റലോഗുകൾ ഉൾക്കൊള്ളുന്നു, അവ `.github/workflows/smoke-test.yml` പ്രവർത്തിതന്ത്രത്തിൽ [AI Smoke Test](https://github.com/marketplace/actions/ai-smoke-test) ആക്ഷൻ വഴി ഓടുന്നു. ഇവ ലഘുലേഖ പ്രൊഡോയ്ക്കുക ശേഷം ഗേറ്റ് പോലെയാണ് (ഏജന്റ് എത്തിക്കുന്നുണ്ടോ, അടിസ്ഥാന പ്രോംപ്റ്റ് പ്രതീക്ഷകൾ പിന്തുടരുന്നുണ്ടോ?), പാഠം 10, 16 ലെ മൂല്യനിർണയ പൈപ്പ്ലൈനുമായി കൂട്ടായി. കാറ്റലോഗ്-ടു-പാഠം-ടു-ഏജന്റ് മാപ്പിംഗ് കാണാൻ [tests/README.md](./tests/README.md) കാണുക. പാഠം 17 ഫൗണ്ടറി ലൊക്കലുമായി പ്രാദേശികമായി പ്രവർത്തിക്കുന്നു, ഹോസ്റ്റുചെയ്യുന്ന എന്റ്പോയിന്റ് ഇല്ല, അതിനാൽ അതിന്റെ നോട്ട്‌ബുക്ക് നേരിട്ട് പ്രവർത്തിപ്പിച്ച് കേസ്തുതലാക്കുന്നു.

### പൊതുവായ കുടുക്കുകൾ

1. **Python പതിപ്പ് പൊരുത്തക്കേടുകൾ:**
   - Python 3.12+ ഉപയോഗിക്കുകയാണെന്ന് ഉറപ്പാക്കുക
   - പഴയ പതിപ്പുകളിൽ ചില പാക്കേജുകൾ പ്രവർത്തിക്കാറില്ല
   - Python പതിപ്പ് വ്യക്തമാക്കാൻ `python3 -m venv` ഉപയോഗിക്കുക

2. **പരിസ്ഥിതി ചാരങ്ങൾ:**
   - എല്ലായ്പ്പോഴും `.env.example` നിന്ന് `.env` സൃഷ്ടിക്കുക
   - `.env` ഫയൽ കമ്മിറ്റ് ചെയ്യാൻ കഴിയില്ല (`.gitignore`-ൽ ഉൾപ്പെടുത്തിയിരിക്കുന്നു)
   - കീൾസ്സ് Entra ഐഡി ഓർത്തентиക്കേഷനായി `az login` ഉപയോഗിക്കുക

   - പുതിയ വിൻചുറ്റർ ഉപയോഗിക്കുക
   - ഇനങ്ങൾക്ക് പകരം `requirements.txt` വഴിയാണ് ഇൻസ്റ്റാൾ ചെയ്യുന്നത്
   - ചില നോട്ട്‌ബുക്കുകളിൽ മാർക്ക്ഡൗൺ സെല്ലുകളിൽ പറയുന്ന അധിക പാക്കേജുകൾ ആവശ്യമാകാം
