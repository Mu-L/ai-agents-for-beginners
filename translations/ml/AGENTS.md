## പ്രോജക്ട് അവലോകനം

ഈ റിപ്പോസിറ്ററി "ആർട്ടിഫിഷ്യൽ ഇന്റലിജൻസ് ഏജന്റുകൾ ആരംഭിക്കുന്നവർക്കായി" എന്ന സമഗ്ര വിദ്യാഭ്യാസ കോഴ്‌സിനെ ഉൾക്കൊള്ളുന്നു - AI ഏജന്റുകൾ നിർമ്മിക്കാൻ ആവശ്യമായ എല്ലാ കാര്യങ്ങളും പഠിപ്പിക്കുന്ന കോഴ്‌സ്. കോഴ്‌സ് അടിസ്ഥാനങ്ങൾ, ഡിസൈൻ പാടേണുകൾ, ഫ്രെയിംവർക്കുകൾ, പ്രൊഡക്ഷൻ ഡിപ്ലോയ്‌മെന്റ്, ലൊക്കൽ/ഓൺ-ഡിവൈസ് ഏജന്റുകൾ, AI ഏജന്റുകളുടെ സുരക്ഷ എന്നിവ 18 പാഠങ്ങളായി (00-18 നമ്പർ ചെയ്ത) ഉൾക്കൊള്ളുന്നു.

**പ്രധാന സാങ്കേതികവിദ്യകൾ:**
- Python 3.12+
- ഇടപെടൽ പഠനത്തിനായി Jupyter നോട്ട്‌ബുക്കുകൾ
- AI ഫ്രീംവർക്കുകൾ: Microsoft Agent Framework (MAF)
- Azure AI സേവനങ്ങൾ: Microsoft Foundry, Microsoft Foundry Agent Service V2

**ആധാരഘടന:**
- പാഠം അടിസ്ഥാന മായ ഘടന (00-15+ ഡയറക്ടറികൾ)
- ഓരോ പാഠത്തിലും: README രേഖപ്പെടുത്തൽ, കോഡ് സാമ്പിളുകൾ (Jupyter നോട്ട്‌ബുക്കുകൾ), ചിത്രങ്ങൾ
- ഓട്ടോമേറ്റഡ് ഭാഷാപരിവർത്തന സംവിധാനത്തിലൂടെ ബഹുഭാഷാ പിന്തുണ
- ഓരോ പാഠത്തിനും മൈക്രോസോഫ്റ്റ് ഏജന്റ് ഫ്രെയിംവര്‍ക്കുപയോഗിച്ച് ഒരു പൈതൺ നോട്ട്‌ബുക്ക്

## ക്രമീകരണ കമാൻഡുകൾ

### ആവശ്യമായ മുൻകൂർ വ്യവസ്ഥകൾ
- Python 3.12 അല്ലെങ്കിൽ അതിനേക്കാൾ പുതിയത്
- Azure സബ്‌സ്‌ക്രിപ്ഷൻ (Microsoft Foundryക്കായി)
- Azure CLI ഇൻസ്റ്റാൾ ചെയ്യുകയും അംഗീകരിക്കുകയും ചെയ്തു (`az login`)

### പ്രാഥമിക ക്രമീകരണം

1. **റിപ്പോസിറ്ററി ക്ലോൺ ചെയ്യുക അല്ലെങ്കിൽ ഫോർക്കുചെയ്യുക:**
   ```bash
   gh repo fork microsoft/ai-agents-for-beginners --clone
   # അല്ലെങ്കില്‍
   git clone https://github.com/microsoft/ai-agents-for-beginners.git
   cd ai-agents-for-beginners
   ```

2. **പൈതൺ വിർച്വൽ എൻവയോൺമെന്റ് സൃഷ്ടിക്കുകയും സജീവമാക്കുകയും ചെയ്യുക:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # വിൻഡോസ് പ്ലാറ്റ്ഫോമിൽ: venv\Scripts\activate
   ```

3. **ഡിപ്പൻഡൻസികൾ ഇൻസ്റ്റാൾ ചെയ്യുക:**
   ```bash
   pip install -r requirements.txt
   ```

4. **പരിസരമാറ്റങ്ങൾ ക്രമീകരിക്കുക:**
   ```bash
   cp .env.example .env
   # നിങ്ങളുടെ API കീകളും എൻഡ്പോയിന്റുകളും ഉൾപ്പെടുത്തി .env എഡിറ്റ് ചെയ്യുക
   ```

### ആവശ്യമായ പരിസരമാറ്റങ്ങൾ

**Microsoft Foundry** (ആവശ്യമാണ്):
- `AZURE_AI_PROJECT_ENDPOINT` - Microsoft Foundry പ്രോജക്ട് എന്റ്പോയിന്റ്
- `AZURE_AI_MODEL_DEPLOYMENT_NAME` - മോഡൽ ഡിപ്ലോയ്‌മെന്റ് നാമം (ഉദാഹരണം: gpt-4.1-mini)

**Azure AI Search** (പാഠം 05 - RAG):
- `AZURE_SEARCH_SERVICE_ENDPOINT` - Azure AI Search എന്റ്പോയിന്റ്
- `AZURE_SEARCH_API_KEY` - Azure AI Search API കീ

അംഗീകാരം: നോട്ട്‌ബുക്ക്‌ ഓടിക്കുന്നതിന് മുമ്പ് `az login` ഓടിക്കുക (ഇത് `AzureCliCredential` ഉപയോഗിക്കുന്നു).

## വികസന പ്രവാഹം

### Jupyter നോട്ട്‌ബുക്കുകൾ പ്രവർത്തിപ്പിക്കല്‍

ഓരോ പാഠത്തിലും വിവിധ ഫ്രെയിംവർക്കുകളെ 위한 പല Jupyter നോട്ട്‌ബുക്കുകളുണ്ട്:

1. **Jupyter തുടങ്ങിയ്ക്കുക:**
   ```bash
   jupyter notebook
   ```

2. **ഒരു പാഠ ഡയറക്ടറി കാണുക** (ഉദാഹൃണം: `01-intro-to-ai-agents/code_samples/`)

3. **നോട്ട്‌ബുക്കുകൾ തുറക്കുക, ഓടിക്കുക:**
   - `*-python-agent-framework.ipynb` - Microsoft Agent Framework ഉപയോഗിച്ച് (Python)
   - `*-dotnet-agent-framework.ipynb` - Microsoft Agent Framework ഉപയോഗിച്ച് (.NET)

### Microsoft Agent Framework ഉപയോഗിച്ച് ജോലി ചെയ്യൽ

**Microsoft Agent Framework + Microsoft Foundry:**
- Azure സബ്‌സ്‌ക്രിപ്ഷൻ ആവശ്യമാണ്
- Agent Service V2-ക്കായി `FoundryChatClient` ഉപയോഗിക്കുന്നു (Foundry പോർട്ടലിൽ ഏജന്റുകൾ കാണാം)
- സജ്ജീകരണത്തോട് കൂടിയ പ്രൊഡക്ഷൻ റെഡിയായി
- ഫയൽ പാടേൺ: `*-python-agent-framework.ipynb`

## പരിശോധനാ നിർദേശങ്ങൾ

ഇത് സാംപിള്‍ കോഡുള്ള പഠന റിപ്പോസിറ്ററിയാണ്; പ്രൊഡക്ഷന്‍ കോഡോ ഓട്ടോമേറ്റഡ് ടെസ്റ്റുകളോ ഇല്ല. നിങ്ങളുടെ ക്രമീകരണവും മാറ്റങ്ങളും സ്ഥിരീകരിക്കാൻ:

### മാനുവൽ പരിശോധന

1. **Python പരിസ്ഥിതി പരിശോധിക്കുക:**
   ```bash
   python --version  # 3.12+ ആയിരിക്കണം
   pip list | grep -E "(agent-framework|azure-ai|azure-identity)"
   ```

2. **നോട്ട്‌ബുക്ക് നിർവ്വഹണം പരിശോധിക്കുക:**
   ```bash
   # നോട്ട്‌ബുക്ക് സ്‌ക്രിപ്റ്റ് ആയി മാറ്റി റൺ ചെയ്യുക (ടെസ്റ്റ് ഇംപോർട്ടുകൾ)
   jupyter nbconvert --to script <lesson-folder>/code_samples/<notebook>.ipynb --stdout | python
   ```

3. **പരിസരമാറ്റങ്ങൾ സ്ഥിരീകരിക്കുക:**
   ```bash
   python -c "import os; from dotenv import load_dotenv; load_dotenv(); print('✓ AZURE_AI_PROJECT_ENDPOINT' if os.getenv('AZURE_AI_PROJECT_ENDPOINT') else '✗ AZURE_AI_PROJECT_ENDPOINT missing')"
   ```

### വ്യത്യസ്ത നോട്ബുക്കുകൾ ഓടിക്കൽ

Jupyter-ൽ നോട്ട്‌ബുക്കുകൾ തുറന്ന് സെല്ലുകൾ നിരവധി ക്രമത്തിൽ പ്രവർത്തിപ്പിക്കുക. ഓരോ നോട്ട്‌ബുക്കിലും:
- ഇറക്കുമതി സ്റ്റേറ്റ്മെന്റുകൾ
- കോൺഫിഗറേഷൻ ലോഡ് ചെയ്യൽ
- ഉദാഹരണ ഏജന്റ് നടപ്പിലാക്കലുകൾ
- മാർക്ക്ഡൗൺ സെല്ലുകളിൽ പ്രതീക്ഷിക്കുന്ന output-കൾ

### ഡിപ്ലോയ് ചെയ്യുന്ന ഏജന്റുകളുടെ സ്മോക്ക്-ടെസ്റ്റ്

മൈക്രോസോഫ്റ്റ് ഫൗണ്ട്രി ഹോസ്റ്റ് ചെയ്ത ഏജന്റായി ഡിപ്ലോയ് ചെയ്യുന്ന പാഠങ്ങൾക്കായി (01, 04, 05, 16), ഈ റിപ്പോസിറ്ററിയിൽ `tests/` മന്ത്രി സ്മോക്ക്-ടെസ്റ്റ് കാറ്റലോഗുകൾ ഉണ്ട്, ഇവ `.github/workflows/smoke-test.yml` പ്രവാഹത്തിലൂടെ [AI Smoke Test](https://github.com/marketplace/actions/ai-smoke-test) ആക്ഷൻ ഉപയോഗിച്ച് ഓടിക്കുന്നു. ഇവ ഒരു ലഘുവായ പോസ്റ്റ്-ഡിപ്ലോയ് ഗെയ്റ്റാണ് (ഏജന്റ് ലഭ്യതയും അടിസ്ഥാന പ്രോമ്പ്റ്റ് പ്രതീക്ഷകളും പരിശോധിക്കുന്നു), పാഠങ്ങൾ 10, 16ലെ വിലയിരുത്തൽ പൈപ്പ്ലൈനിന് സമ്പൂർണമാണ്. കാറ്റലോഗ്-പാഠം-ഏജന്റ് മാപ്പിംഗ് `[tests/README.md](./tests/README.md)`-ൽ കാണുക. പാഠം 17 Foundry Local-ഉം Qwen-ഉം ഉപയോഗിച്ച് ലൊക്കലായി ഓടുകയും ഹോസ്റ്റ് ചെയ്ത എന്റ്പോയിന്റ് ഇല്ലാതെ പ്രവർത്തിക്കുകയും ചെയ്യുന്നു; അതിനാൽ ആ നോട്ട്‌ബുക്ക് നേരിട്ട് ഓടിച്ചുപരിശോധിക്കുന്നു.

### പൊതുവായ ഗെറ്റ്ചാസ്

   - Python 3.12+ ഉപയോഗിക്കുന്നതായി ഉറപ്പാക്കുക
   - പഴയ പതിപ്പുകളിൽ ചില പാക്കേജുകൾ പ്രവർത്തിക്കണമെന്നില്ല

   - എല്ലായ്പ്പോഴും `.env.example` ഉപയോഗിച്ച് `.env` നിർമിക്കുക
