# ⏩ 使用 Azure OpenAI（Responses API）實現.NET 的序列式代理工作流程

## 📋 進階序列處理教學

本筆記展示如何使用 Microsoft Agent Framework for .NET 與 Azure OpenAI（Responses API）實現<strong>序列工作流程模式</strong>。您將學會如何建立複雜且分步的處理管線，其中代理會依特定順序執行，每個階段都基於前一階段的結果進行。

### 🏗️ <strong>序列工作流程架構</strong>

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: 分析代理
    B --> C[檢查點 1]
    C --> D[Stage 2: 處理代理
```

- **Azure OpenAI 用戶端**：跨所有工作流程階段保持一致的 AI 模型存取

## 💻 執行程式碼

完整實作見於 `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`，此檔展示一個<strong>三階段家具分析工作流程</strong>：

1. **階段 1 - 銷售代理**：分析家具圖像並提供購買建議
2. **階段 2 - 價格代理**：提供詳細價格拆解與預算選項
3. **階段 3 - 報價代理**：生成專業 Markdown 格式報價文件

### 🏗️ <strong>工作流程架構</strong>

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

每個代理：
- 接收前一階段輸出作為上下文
- 透過專業知識在前次分析基礎上進行建構
- 透過狀態管理維持工作流程連貫性

**前置條件：**

- <strong>前代理輸出</strong>：所有先前代理在對話歷史中的回應
