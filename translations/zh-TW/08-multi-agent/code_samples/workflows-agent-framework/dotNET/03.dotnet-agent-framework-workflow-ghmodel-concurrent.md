### 🏗️ <strong>工作流程架構</strong>

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**主要元件：**

1. **ConcurrentStartExecutor**：同時將使用者請求廣播給所有代理
2. **Researcher 代理**：平行分析目的地與景點
3. **Planner 代理**：平行建立詳細旅行計劃
4. **ConcurrentAggregationExecutor**：收集並合併兩個代理的結果

### 🎯 **扇出/扇入模式**

此工作流程示範經典的<strong>扇出/扇入</strong>模式：
- <strong>扇出</strong>：一則輸入訊息同時廣播給多個代理
- <strong>並行處理</strong>：多個代理平行處理相同任務
- <strong>扇入</strong>：收集並合併所有代理產生的結果為單一輸出

### 🔍 了解自訂執行器

**ConcurrentStartExecutor：**
- 實作 `IMessageHandler<string>` 以接受字串輸入
- 將訊息廣播給所有連接代理
- 傳送 `TurnToken` 觸發並行處理

**ConcurrentAggregationExecutor：**
- 實作 `IMessageHandler<ChatMessage>` 以接收代理回應
- 以執行緒安全方式收集訊息
- 在所有預期回應抵達後聚合結果
- 使用 `context.YieldOutputAsync()` 輸出最終結果

### ⚡ 效能優勢

**並行 vs 序列：**
- 序列：代理1 (30秒) → 代理2 (30秒) = **共 60 秒**
- 並行：代理1 (30秒) || 代理2 (30秒) = **共 30 秒**

<strong>吞吐量提升</strong>：N 個並行代理可加快至 N 倍（取決於工作負載與資源）

### 🛡️ 錯誤處理

工作流程優雅處理單一代理的失敗：
- 若一個代理失敗，其他仍繼續處理
- 聚合器可實作逾時邏輯
- 必要時可回傳部分結果
