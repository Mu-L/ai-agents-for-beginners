# AI 代理設計原則

## 介紹

建構 AI 代理系統有許多思考方式。由於模糊性是生成式 AI 設計的特點而非錯誤，工程師有時難以確定從何開始。我們建立了一套以人為中心的 UX 設計原則，讓開發者能構建以客戶為中心的代理系統，來解決其商業需求。這些設計原則不是規範性的架構，而是為定義與建立代理體驗的團隊提供的起點。

一般來說，代理人應該：

- 擴展並放大人類能力（如頭腦風暴、解決問題、自動化等）
- 補足知識缺口（快速讓我掌握知識領域、翻譯等）
- 促進和支持符合個人偏好的協作方式
- 使我們成為更好的自己（例如：人生教練/任務管理員，幫助我們學習情緒調節與正念技巧，建立韌性等）

## 代理設計原則

![代理設計原則](../../../translated_images/zh-TW/agentic-design-principles.1cfdf8b6d3cc73c2.webp)

### 代理環境 (Space)

這是代理人操作的環境。這些原則指引我們如何設計代理，以參與物理及數位世界。

- **連結，而非合併** — 幫助連結人與人、事件和可行動的知識，促進協作與連結。
- 代理幫助連結事件、知識和人。
- 代理拉近人際距離。它們並非被設計用來取代或貶低人類。
- <strong>易於存取但偶爾隱形</strong> — 代理大多在背景運作，僅在相關且適當時提醒我們。
  - 代理對授權用戶而言於任意裝置或平台皆易於發現與存取。
  - 代理支援多模式輸入與輸出（聲音、語音、文字等）。
  - 代理能無縫切換前景與背景；主動與被動，依據對用戶需求的感知。
  - 代理可能以隱形形式運作，但其背景流程與與其他代理合作對用戶而言透明且可控。

### 代理時間 (Time)

這是代理人隨時間的運作方式。這些原則指引我們如何設計代理互動於過去、現在與未來。

- <strong>過去</strong>：反思包含狀態與情境的歷史。
  - 代理根據豐富歷史資料分析，而非僅事件、人或狀態，提供更相關結果。
  - 代理從過去事件建立連結，積極反思記憶以應對當前情況。
- <strong>現在</strong>：提醒多於通知。
  - 代理體現全面與人互動的方法。事件發生時，代理超越靜態通知或其他形式。代理可簡化流程或動態產生提示，在適當時機引導用戶注意。
  - 代理根據情境環境、社會與文化變化，並依用戶意圖調整訊息。
  - 代理互動可逐步進行，隨時間演變/成長，讓用戶逐漸被賦能。
- <strong>未來</strong>：適應與演化。
  - 代理適應各種裝置、平台與模式。
  - 代理適應用戶行為、無障礙需求，並可自由客製化。
  - 代理藉由持續用戶互動塑形並演進。

### 代理核心 (Core)

這是代理設計核心的關鍵元素。

- <strong>接受不確定性但建立信任</strong>。
  - 預期一定程度的代理不確定性。不確定性是代理設計的關鍵元素。
  - 信任與透明是代理設計的基礎層。
  - 人類可控制代理開啟/關閉，代理狀態始終清晰可見。

## 實作這些原則的指南

使用上述設計原則時，請遵循以下指南：

1. <strong>透明性</strong>：告知用戶 AI 參與，如何運作（包括過去動作），如何提供反饋與修改系統。
2. <strong>控制</strong>：允許用戶自訂、指定偏好與個人化，並掌控系統及其屬性（包含遺忘功能）。
3. <strong>一致性</strong>：致力於跨裝置與端點提供一致的多模式體驗。盡可能使用熟悉的 UI/UX 元素（例如語音互動用麥克風圖示），減輕客戶認知負擔（例如簡潔回應、視覺輔助與「深入了解」內容）。

## 如何利用這些原則和指南設計旅遊代理

假設您正在設計一個旅遊代理，可思考如何運用設計原則與指南如下：

1. <strong>透明性</strong> — 告知用戶旅遊代理是 AI 代理。提供基本入門說明（如「你好」訊息、範例提示）。產品頁面清楚記錄這些。顯示用戶曾詢問提示清單。清楚說明如何提供反饋（點讚/倒讚、發送反饋按鈕等）。明確說明代理是否有限制使用或主題。
2. <strong>控制</strong> — 明確告知用戶完成建立後如何修改代理（例如系統提示）。讓用戶選擇代理回覆詳盡程度、寫作風格與禁談事項。允許用戶查看並刪除相關檔案、資料、提示與過往對話。
3. <strong>一致性</strong> — 確保分享提示、添加檔案或照片及標記對象的圖示符合標準易辨識。用迴紋針圖示表示檔案上傳/與代理共享，圖片圖示表示圖形上傳。
