# संगणक वापर एजंट्स (CUA) तयार करणे

संगणक वापर एजंट्स वेबसाइट्सशी त्याच पद्धतीने संवाद साधू शकतात जसे एखादा व्यक्ती करतो: ब्राउजर उघडणे, पृष्ठ तपासणे, आणि पाहिलेल्या गोष्टींमध्ये सर्वोत्तम पुढील क्रिया घेणे. या धड्यात, तुम्ही एक ब्राउजर ऑटोमेशन एजंट तयार कराल जो Airbnb द्वारे शोध करतो, संरचित लिस्टिंग डेटा काढतो, आणि स्टॉफहोल्ममध्ये स्वस्तीतला निवास ओळखतो.

हा धडा AI-ने संचालित नेव्हिगेशनसाठी Browser-Use, ब्राउजर नियंत्रणासाठी Playwright आणि Chrome DevTools Protocol (CDP), दृष्टीसक्षम तर्कासाठी Azure OpenAI, आणि संरचित काढणीसाठी Pydantic यांचा एकत्र वापर करतो.

## परिचय

हा धडा खालील गोष्टींचा समावेश करेल:

- API-फक्त ऑटोमेशनपेक्षा संगणक वापर एजंट्स कधी चांगले असतात हे समजून घेणे
- खात्रीलायक ब्राउझर जीवनचक्र व्यवस्थापनासाठी Browser-Use सह Playwright आणि CDP एकत्र करणे
- डायनॅमिक वेब पृष्ठांवरून लिस्टिंग डेटा काढण्यासाठी Azure OpenAI दृष्टी आणि संरचित Pydantic आउटपुट वापरणे
- एजंट-प्रथम, अभिनेता-प्रथम किंवा मिश्र ब्राउजर ऑटोमेशन वर्कफ्लोसाठी निर्णय घेणे

## शिकण्याच्या उद्दिष्टे

हा धडा पूर्ण केल्यावर, तुम्हाला कसे करावे याची माहिती असेल:

- Azure OpenAI आणि Playwright सह Browser-Use कॉन्फिगर करा
- वास्तविक वेबसाइटवर नेव्हिगेट करणारे आणि डायनॅमिक UI घटक हाताळणारे ब्राउजर ऑटोमेशन वर्कफ्लो तयार करा
- दृश्यमान पृष्ठ सामग्रीतून टाइप केलेले निकाल काढा आणि त्यांना डाउनस्ट्रीम व्यवसाय लॉजिकमध्ये रूपांतर करा
- ब्राउजर टास्क कितपत भाकीत केला जाऊ शकतो यावर आधारित एजंट आणि अभिनेता पॅटर्नमधून निवड करा

## कोड नमुना

या धड्यामध्ये एक नोटबुक ट्युटोरियल समाविष्ट आहे:

- [15-browser-user.ipynb](./15-browser-user.ipynb): CDP वर क्रोम सत्र सुरू करते, स्टॉफहोल्म लिस्टिंगसाठी Airbnb शोधते, Browser-Use दृष्टीसह किंमती काढते, आणि सर्वात स्वस्त पर्याय संरचित डेटा म्हणून परत करते.

## पूर्वअट

- Python 3.12+
- वातावरणात Azure OpenAI सुरू केलेले आहे
- स्थानिकरित्या Chrome किंवा Chromium स्थापित आहे
- Playwright अवलंबित्वे स्थापित आहेत
- अ‍ॅसिंक्रोनस Python ची मूलभूत ओळख आहे

## सेटअप

नोटबुकमध्ये वापरलेले पॅकेजेस स्थापित करा:

```bash

playwright install chromium
```

नोटबुकने वापरलेले Azure OpenAI पर्यावरण चल सेट करा:

```bash
AZURE_OPENAI_ENDPOINT=...

AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=...
# ऐच्छिक: वगळल्यास तो नवीनतम API आवृत्तीवर सेट होतो
AZURE_OPENAI_API_VERSION=...
```

## आर्किटेक्चर आढावा

नोटबुक एक मिश्र ब्राउजर ऑटोमेशन वर्कफ्लो दाखवते:

1. CDP सक्षम केलेले Chrome सुरु होते जेणेकरून Playwright आणि Browser-Use एकाच ब्राउजर सत्र सामायिक करू शकतात.
2. Browser-Use एजंट ओपन-एंडेड नेव्हिगेशन कार्य हाताळतो जसे Airbnb उघडणे, पॉप-अप्स बंद करणे, आणि स्टॉफहोल्मसाठी शोधणे.
3. सक्रिय पृष्ठ संरचित Pydantic स्कीमासह तपासले जाते ज्यामध्ये लिस्टिंग शीर्षके, दररोजची किंमत, रेटिंग्स, आणि URL काढले जातात.
4. Python लॉजिक काढलेल्या लिस्टिंगची तुलना करते आणि सर्वात स्वस्त निकाल हायलाइट करते.

या दृष्टिकोनामुळे Browser-Use च्या लवचिक, दृष्टी-आधारित तर्कक्षमतेला चालना मिळते आणि आवश्यक तेव्हा तुम्हाला निश्चित ब्राउजर नियंत्रण देखील मिळते.

## मुख्य मुद्दे आणि सर्वोत्तम मार्गदर्शक तत्त्वे

### एजंट आणि अभिनेता कधी वापरायचा

| परिस्थिती | एजंट वापरा | अभिनेता वापरा |
|----------|--------------|--------------|
| डायनॅमिक लेआउट्स | होय, AI पृष्ठ बदलांना अनुकूल होऊ शकतो | नाही, भंगुर सिलेक्टर तोडू शकतात |
| ज्ञात रचना | नाही, एजंट थेट नियंत्रणापेक्षा मंदगती आहे | होय, वेगवान आणि अचूक |
| घटक शोधणे | होय, नैसर्गिक भाषा चांगली काम करते | नाही, अचूक सिलेक्टर आवश्यक आहेत |
| वेळ नियंत्रण | नाही, कमी भाकीत करता येणारे | होय, वेट्स आणि पुनःप्रयत्नांवर पूर्ण नियंत्रण |
| गुंतागुंतीची वर्कफ्लो | होय, अपेक्षित नसलेल्या UI स्थिती हाताळतो | नाही, स्पष्ट शाखांकन आवश्यक आहे |

### Browser-Use सर्वोत्तम मार्गदर्शक तत्त्वे

1. अन्वेषण आणि डायनॅमिक नेव्हिगेशनसाठी एजंट पासून प्रारंभ करा.
2. परस्परसंवाद भाकीत करता येण्याजोगा झाल्यावर थेट पृष्ठ नियंत्रणात बदला.
3. काढलेल्या डेटाचे प्रमाणीकरण व प्रकार-सुरक्षिततेसाठी संरचित आउटपुट मॉडेल वापरा.
4. दृश्यमान UI बदल घडवून आणणाऱ्या कृतीनंतर धोरणात्मक विलंब जोडा.
5. अयशस्वीतेचे डिबगिंग सुलभ होईल म्हणून पुनरावृत्ती दरम्यान स्क्रीनशॉट कॅप्चर करा.
6. वेबसाइट बदल होण्याची अपेक्षा ठेवा आणि पॉप-अप्स व लेआउट बदलांसाठी फॉलबॅक धोरणे तयार करा.
7. दोन्ही लवचिकता आणि अचूकता मिळविण्यासाठी एजंट आणि अभिनेता पॅटर्न मिसळा.

### ब्राउजर एजंटसाठी सुरक्षितता मार्गदर्शक तत्त्वे

ब्राउजर एजंट्स थेट वेबसाइट्सवर कार्य करतात, त्यामुळे त्यांना फक्त ज्ञात API कॉल करणाऱ्या स्क्रिप्टच्या तुलनेत घट्ट मर्यादा आवश्यक आहेत. नोटबुक डेमोमधून प्रत्यक्ष वर्कफ्लो कडे जाताना, एजंट काय पाहू शकतो, क्लिक करू शकतो आणि सादर करू शकतो याभोवतीचे नियंत्रण निश्चित करा.

1. **ब्राउझिंग वातावरणाचा अभ्यास करा.** एजंटला समर्पित ब्राउजर प्रोफाइल किंवा सॅंडबॉक्समध्ये चालवा, आणि कार्यासाठी आवश्यक असलेल्या डोमेनपुरतेच मर्यादित ठेवा.
2. **पर्यवेक्षण आणि क्रिया विभाजित करा.** एजंटला प्रथम शोध, वाचन, आणि डेटा काढण्याची परवानगी द्या; फॉर्म सबमिट करणे, संदेश पाठवणे, प्रवास बुक करणे, खरेदी करणे, रेकॉर्ड्स हटवणे किंवा अकाउंट सेटिंग्ज बदलण्यापूर्वी स्पष्ट मंजुरीची आवश्यकता ठेवा.
3. **प्रॉम्प्ट्स आणि ट्रेसेसमधून गुपिते वगळा.** मॉडेल कंटेक्स्टमध्ये पासवर्ड, पेमेंट तपशील, सेशन कुकीज, किंवा कच्च्या वैयक्तिक डेटा ठेऊ नका. प्रमाणीकरणासाठी वापरकर्त्याकडे सोपवा आणि संवेदनशील क्षेत्रे लॉगमधून बाजूला करा.
4. **पृष्ठ सामग्री अविश्वसनीय इनपुट म्हणून वागवा.** वेबसाइटवर अशी सूचना असू शकते जी एजंटसाठी आहे, वापरकर्ता नाही. एजंटने अशा पृष्ठ लेखनाला दुर्लक्ष करावे जे त्याचा उद्देश बदलण्यास, डेटा उघड करण्यास, सुरक्षितता बंद करण्यास, किंवा असंबंधित साइट्स भेट देण्याचा आग्रह करते.
5. **धोकादायक टप्प्याभोवती निश्चित तपासणी वापरा.** अंतिम टप्प्याची मंजुरी मागण्यापूर्वी सध्याचा URL, पृष्ठ शीर्षक, निवडलेला घटक, किंमत, प्राप्तकर्ता, आणि क्रिया सारांश कोडद्वारे तपासा.
6. **बजेटे आणि थांबण्याच्या अटी ठेवा.** एजंट वापरू शकणाऱ्या क्रिया, पुनःप्रयत्न, टॅब्स, आणि मिनिटांची संख्या मर्यादित करा. पृष्ठ स्थिती अस्पष्ट असेल तर क्लिक करणे थांबवा.
7. **सगळं काही नोंदवू नका, उपयुक्त पुरावे नोंदवा.** क्रिया सारांश, टाईमस्टॅम्प, URL, निवडलेल्या घटकांचे वर्णन, आणि स्क्रीनशॉट संदर्भ ठेवा जेणेकरून अयशस्वीता पुनरावलोकनासाठी उपलब्ध राहतील पण आवश्यक नसलेली संवेदनशील माहिती संग्रहित होणार नाही.

Airbnb उदाहरणात सुरक्षित डीफॉल्ट म्हणजे लिस्टिंग शोधणे आणि किंमती काढणे होय. साइन इन करणे, होस्टशी संपर्क करणे, किंवा बुकिंग पूर्ण करणे वेगळे वापरकर्ता मंजूर केलेले कार्य असावे.

### वास्तविक जगातील अनुप्रयोग

- प्रवासाचे बुकिंग व किंमतीचे निरीक्षण
- ई-कॉमर्स किंमत तुलना आणि उपलब्धता तपासणी
- डायनॅमिक वेबसाइट्सवरून संरचित काढणी
- दृष्टी अवगत UI चाचणी आणि पडताळणी
- वेबसाइट निरीक्षण आणि अलर्टिंग
- बहु-टप्प्यांत आधारित प्रवाहांमध्ये बुद्धिमान फॉर्म भरणे

## वास्तविक उदाहरण: Microsoft प्रोजेक्ट Opal

या धड्यात तयार केलेला एजंट हा एक लहान, स्थानिक आवृत्ती आहे **संगणक वापर एजंट (CUA)** ची — एक प्रोग्राम जो व्यक्ती कशी ब्राउजर चालवतो तसे चालवतो. Microsoft हीच कल्पना एंटरप्राइझसाठी **[Project Opal (Frontier)](https://support.microsoft.com/en-us/microsoft-365-copilot/get-started-with-project-opal-frontier)** या Microsoft 365 Copilot मधील क्षमतेसह आणत आहे.

Project Opal सह, तुम्ही एक कार्य वर्णन करता आणि एजंट तुमच्या वतीने **सुरक्षित विंडोज 365 क्लाऊड पीसीवर संगणक वापर करून** तुमच्या संस्थेच्या ब्राउजर-आधारित अनुप्रयोग, साइट्स, आणि डेटावर ऑपरेट करतो. हे **बॅकग्राऊंडमध्ये असिंक्रोनस पद्धतीने** कार्य करते, आणि तुम्ही कधीही काम मार्गदर्शन करू शकता किंवा नियंत्रण घेऊ शकता. उदाहरणे म्हणजे:

- सुरक्षा गट सदस्यत्व विनंत्या व्यवस्थापन
- अनुपालन पुनरावलोकनांसाठी ऑडिट पुरावा संकलन आणि प्रमाणीकरण
- IT घटनांची वर्गवारी (तिकीट स्थिती अपडेट करणे, मालक नियुक्त करणे, डुप्लिकेट बंद करणे)
- अर्थसंकल्पीय क्लोज डेकमध्ये Excel डेटा संकलित करणे

Opal हा **उत्पादन-स्तरीय, विश्वासार्ह** संगणक वापर एजंट कसा दिसतो याचा उपयुक्त संदर्भ आहे — आणि तो आधीच्या धड्यांतील संकल्पना मजबूत करतो:
