# ⚡ Azure OpenAI (Responses API) सह समवर्ती एजंट कार्यप्रवाह (.NET)

## 📋 उच्च-कार्यक्षमता समानांतर प्रक्रिया ट्यूटोरियल

हा नोटबुक Microsoft Agent Framework for .NET आणि Azure OpenAI (Responses API) वापरून **समवर्ती कार्यप्रवाह नमुने** दर्शवितो. तुम्हाला उच्च-कार्यक्षमता, समांतर प्रक्रिया कार्यप्रवाह कसे तयार करायचे हे शिकायला मिळेल जे एकाच वेळी अनेक AI एजंट एकत्र चालवून थ्रूपुट कमाल करतात आणि समन्वय व डेटा सुसंगतता राखतात.

## 🎯 शिकण्याच्या उद्दिष्टे

### 🚀 **समवर्ती प्रक्रियेची मूलतत्त्वे**
- **समांतर एजंट कार्यान्वयन**: जास्तीत जास्त कार्यक्षमता साठी अनेक AI एजंट एकाच वेळी चालवा
- **Async/Await नमुने**: कार्यक्षम समवर्तीतेसाठी .NET चे async प्रोग्रामिंग मॉडेल वापरा
- **Azure OpenAI (Responses API)**: Azure OpenAI Responses API ला अनेक समवर्ती कॉल समन्वयित करा
- **संसाधन व्यवस्थापन**: समवर्ती ऑपरेशन्समध्ये AI मॉडेल संसाधने कार्यक्षमतेने व्यवस्थापित करा

### 🏗️ **प्रगत समवर्ती आर्किटेक्चर**
- **कार्य-आधारित समानांतरता**: उत्कृष्ट समवर्ती कार्यान्वयनासाठी .NET Task Parallel Library वापरा
- **समन्वय नमुने**: स्पर्धात्मक परिस्थितींना टाळत समवर्ती एजंट समन्वयित करा
- **लोड समतोलाना**: उपलब्ध समवर्ती प्रक्रिया क्षमतेत कार्य कार्यक्षमतेने वितरित करा
- **तुटलेले सहनशक्ती**: संपूर्ण कार्यप्रवाह थांबविण्याशिवाय वेगवेगळ्या एजंटच्या अयशस्वीपणाची हाताळणी करा

### 🏢 **एंटरप्राइझ समवर्ती अनुप्रयोग**
- **उच्च-खंड दस्तऐवज प्रक्रिया**: अनेक दस्तऐवज एकाच वेळी प्रक्रिया करा
- **रिअल-टाइम सामग्री विश्लेषण**: येणाऱ्या डेटास्ट्रीमचे समवर्ती विश्लेषण
- **बॅच प्रक्रिया अनुकूलन**: मोठ्या प्रमाणात डेटाप्रक्रिया ऑपरेशन्ससाठी थ्रूपुट कमाल करा
- **मल्टी-मोडल विश्लेषण**: वेगवेगळ्या सामग्री प्रकार आणि स्वरूपांचे समांतर प्रक्रिया

## 💻 कोड चालविणे

पूर्ण अंमलबजावणी `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` मध्ये उपलब्ध आहे. ही फाइल प्रवास नियोजनासाठी **फॅन-आउट/फॅन-इन समवर्ती कार्यप्रवाह** दर्शवते:

### 🏗️ **कार्यप्रवाह आर्किटेक्चर**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**मुख्य घटक:**

1. **ConcurrentStartExecutor**: वापरकर्त्याचा विनंती सर्व एजंटना एकाच वेळी प्रसारित करतो
2. **Researcher Agent**: गंतव्ये आणि आकर्षणे समवर्तीपणे विश्लेषण करतो
3. **Planner Agent**: तपशीलवार प्रवास योजना समवर्तीपणे तयार करतो
4. **ConcurrentAggregationExecutor**: दोन्ही एजंटकडून परिणाम गोळा करतो आणि विलीन करतो

### 🎯 **फॅन-आउट/फॅन-इन नमुना**

हा कार्यप्रवाह क्लासिक **फॅन-आउट/फॅन-इन** नमुना दर्शवितो:
- **फॅन-आउट**: एक इनपुट संदेश अनेक एजंटना एकाच वेळी प्रसारित होतो
- **समवर्ती प्रक्रिया**: अनेक एजंट एकाच कार्यावर समांतर काम करतात
- **फॅन-इन**: सर्व एजंटकडून निष्पत्ती गोळा करुन एकत्रित आउटपुट मध्ये रूपांतरित होतात

### 🔍 सानुकूल कार्यान्वयक समजून घेणे

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` अंमलात आणतो जेणेकरून स्ट्रिंग इनपुट स्वीकारता येतो
- सर्व कनेक्टेड एजंटना संदेश प्रसारित करतो
- समवर्ती प्रक्रिया सुरु करण्यासाठी `TurnToken` पाठवतो

**ConcurrentAggregationExecutor:**
- एजंट प्रतिसाद स्वीकारण्यासाठी `IMessageHandler<ChatMessage>` अंमलात आणतो
- संदेश थ्रेड-सुरक्षित पद्धतीने गोळा करतो
- अपेक्षित सर्व प्रतिसाद आल्यावर एकत्रीकरण करतो
- `context.YieldOutputAsync()` वापरुन अंतिम आउटपुट प्रदान करतो

**समवर्ती विरुद्ध अनुक्रमिक:**
- अनुक्रमिक: Agent1 (30s) → Agent2 (30s) = **एकूण 60 सेकंद**
