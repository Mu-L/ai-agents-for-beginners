# ⚡ Azure OpenAI (Responses API) सह समकालीन एजंट वर्कफ्लोज (.NET)

## 📋 उच्च-कार्यक्षमता समानांतर प्रक्रिया ट्युटोरियल

हा नोटबुक Microsoft Agent Framework for .NET आणि Azure OpenAI (Responses API) वापरून **समकालीन वर्कफ्लो नमुने** कसे करायचे हे दर्शवितो. तुम्ही एकाच वेळी एकाधिक AI एजंट्स चालवून आणि समन्वय व डेटा सुसंगतता राखून उच्च-कार्यक्षमता, समानांतर प्रक्रिया वर्कफ्लोज कसे तयार करायचे ते शिकाल.

## 🎯 शिकण्याच्या उद्दिष्टे

### 🚀 **समकालीन प्रक्रिया मूलभूत बाबी**
- **समानांतर एजंट कार्यवाही**: उच्च कार्यक्षमतेसाठी अनेक AI एजंट्स एकाच वेळी चालवा
- **Async/Await नमुने**: कार्यक्षम समकालीनतेसाठी .NET चे async प्रोग्रामिंग मॉडेल वापरा
- **Azure OpenAI (Responses API)**: Azure OpenAI Responses API ला एकाहून अधिक समकालीन कॉल समन्वयित करा
- **संसाधन व्यवस्थापन**: समकालीन ऑपरेशन्समध्ये AI मॉडेल संसाधने कार्यक्षमतेने व्यवस्थापित करा

### 🏗️ **प्रगत समकालीन आर्किटेक्चर**
- **टास्क-आधारित समानांतरता**: सर्वोत्तम समकालीन कार्यवाहीसाठी .NET Task Parallel Library वापरा
- **सिंक्रोनायझेशन नमुने**: रेस कंडीशन्स टाळत समकालीन एजंट्सचे समन्वय करा
- **लोड बॅलन्सिंग**: उपलब्ध समकालीन प्रक्रिया क्षमतेवर काम कार्यक्षमतेने वितरित करा
- **दोष सहिष्णुता**: संपूर्ण वर्कफ्लो थांबविण्याशिवाय व्यक्तिगत एजंट चुका हाताळा

### 🏢 **उद्योग समकालीन अनुप्रयोग**
- **उच्च-व्हॉल्यूम दस्तऐवज प्रक्रिया**: एकाच वेळी अनेक दस्तऐवज प्रक्रिया करा
- **रिअल-टाइम कंटेंट विश्लेषण**: येणाऱ्या डेटा प्रवाहांचे समकालीन विश्लेषण
- **बॅच प्रक्रिया ऑप्टिमायझेशन**: मोठ्या प्रमाणावर डेटा प्रक्रिया ऑपरेशन्ससाठी जास्तीत जास्त थ्रूपुट
- **मल्टी-मोडल विश्लेषण**: वेगवेगळ्या कंटेंट प्रकार व फॉरमॅट्सची समानांतर प्रक्रिया


## 💻 कोड चालविणे

संपूर्ण अंमलबजावणी `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` मध्ये उपलब्ध आहे. ही फाइल प्रवास नियोजनासाठी **Fan-Out/Fan-In समकालीन वर्कफ्लो** दर्शविते:

### 🏗️ **वर्कफ्लो आर्किटेक्चर**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**महत्वाचे घटक:**

1. **ConcurrentStartExecutor**: सर्व एजंट्सना वापरकर्त्याचा विनंत्यास एकाच वेळी प्रसारित करतो
2. **Researcher Agent**: एकाच वेळी स्थळे आणि आकर्षणे विश्लेषित करतो
3. **Planner Agent**: एकाच वेळी सविस्तर प्रवास योजना तयार करतो
4. **ConcurrentAggregationExecutor**: दोन्ही एजंट्सकडून परिणाम एकत्र करतो

### 🎯 **Fan-Out/Fan-In नमुना**

हा वर्कफ्लो पारंपरिक **Fan-Out/Fan-In** नमुना दर्शवितो:
- **Fan-Out**: एक इनपुट संदेश एकाच वेळी अनेक एजंट्सना प्रसारित होतो
- **समकालीन प्रक्रिया**: अनेक एजंट्स एकाच कामावर समानांतर कार्य करतात
- **Fan-In**: सर्व एजंट्सचे परिणाम गोळा करून एकत्र आउटपुट तयार करतो

```csharp

    .AddFanOutEdge(startExecutor, targets: [researcherAgent, plannerAgent, budgetAgent])
    .AddFanInEdge(aggregationExecutor, sources: [researcherAgent, plannerAgent, budgetAgent])
    .WithOutputFrom(aggregationExecutor)
    .Build();
```

- **कोड पुनरावलोकन**: अनेक पुनरावलोकक वेगवेगळ्या दृष्टिकोनातून कोड तपासतात
- **मार्केट संशोधन**: वेगवेगळ्या बाजार विभागांची समानांतर विश्लेषण
- **दस्तऐवज प्रक्रिया**: समकालीन विलीनीकरण, विश्लेषण आणि पडताळणी
- **बहु-दृष्टिकोन विश्लेषण**: एकाच इनपुटवर विविध दृष्टीकोन मिळवणे

### 🔍 सानुकूल Executor समजून घेणे

**ConcurrentStartExecutor:**
- स्ट्रिंग इनपुट स्वीकारण्यासाठी `IMessageHandler<string>` राबवितो
- सर्व कनेक्टेड एजंट्सना संदेश प्रसारित करतो
- समकालीन प्रक्रिया सुरू करण्यासाठी `TurnToken` पाठवितो

**ConcurrentAggregationExecutor:**
- एजंट प्रतिसाद मिळवण्यासाठी `IMessageHandler<ChatMessage>` राबवितो
- संदेश थ्रेड-सेफ पद्धतीने गोळा करतो
- अपेक्षित सर्व प्रतिसाद आल्यानंतर एकत्रित करतो
- `context.YieldOutputAsync()` वापरून अंतिम आउटपुट प्रदान करतो

### ⚡ कार्यक्षमता फायदे

**समकालीन विरुद्ध अनुक्रमिक:**
- अनुक्रमिक: एजंट1 (30s) → एजंट2 (30s) = **एकूण 60 सेकंद**
- समकालीन: एजंट1 (30s) || एजंट2 (30s) = **एकूण 30 सेकंद**
