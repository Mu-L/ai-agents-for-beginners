# ⏩ Azure OpenAI (Responses API) बरोबर अनुक्रमिक एजंट वर्कफ्लो (.NET)

## 📋 प्रगत अनुक्रमिक प्रक्रिया ट्युटोरियल

हे नोटबुक Microsoft Agent Framework for .NET आणि Azure OpenAI (Responses API) वापरून **अनुक्रमिक वर्कफ्लो पॅटर्न्स** दर्शविते. तुम्हाला कसे विकसित करायचे हे शिकवले जाईल ज्यामध्ये एजंट विशिष्ट क्रमाने कार्य करतात, प्रत्येक टप्पा मागील टप्प्याच्या परिणामांवर आधारित असतो.

## 🎯 शिकण्याचे उद्दिष्टे

### 🔄 **अनुक्रमिक प्रक्रिया आर्किटेक्चर**
- **रेखीय वर्कफ्लो डिझाइन**: स्पष्ट अवलंबित्वांसह टप्प्याटप्प्याने प्रक्रिया पाइपलाइन तयार करा
- **स्थिती व्यवस्थापन**: अनुक्रमिक वर्कफ्लो टप्प्यांमध्ये संदर्भ व डेटा प्रवाह राखा
- **Azure OpenAI (Responses API)**: मल्टि-टप्पा .NET वर्कफ्लो मध्ये Azure OpenAI मॉडेल्सचा उपयोग करा
- **एंटरप्राइझ पाइपलाइन पॅटर्न्स**: उत्पादनासाठी तयार अनुक्रमिक प्रक्रिया सिस्टम तयार करा

### 🏗️ **प्रगत अनुक्रमिक पॅटर्न्स**
- **स्टेज-गेट प्रक्रिया**: वर्कफ्लो टप्प्यांमधील प्रमाणीकरण तपासणी लागू करा
- **संदर्भ संरक्षण**: सर्व टप्प्यांमध्ये स्थिती आणि संकलित ज्ञान राखा
- **त्रुटी प्रसार**: अनुक्रमिक प्रक्रिया साखळीत अपयश योग्य प्रकारे हाताळा
- **कार्यक्षमता अनुकूलन**: किमान ओव्हरहेडसह प्रभावी अनुक्रमिक अंमलबजावणी

### 🏢 **एंटरप्राइझ अनुक्रमिक अनुप्रयोग**
- **दस्तऐवज प्रक्रिया पाइपलाइन**: मल्टि-टप्पा दस्तऐवज विश्लेषण, रूपांतरण, आणि प्रमाणीकरण
- **गुणवत्ता आश्वासन वर्कफ्लो**: अनुक्रमिक पुनरावलोकन, प्रमाणीकरण, आणि मंजुरी प्रक्रिया
- **सामग्री उत्पादन पाइपलाइन**: संशोधन → लेखन → संपादन → पुनरावलोकन → प्रकाशन
- **व्यावसायिक प्रक्रिया ऑटोमेशन**: स्पष्ट टप्प्यांच्या अवलंबित्वांसह मल्टि-स्टेप व्यवसाय वर्कफ्लो

## ⚙️ पूर्वअट आणि सेटअप

### 📦 **आवश्यक NuGet पॅकेजेस**

.NET अनुक्रमिक वर्कफ्लो साठी आवश्यक पॅकेजेस:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

**पर्यावरण सेटअप (.env फाइल):**
```env

AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**कॉन्फिगरेशन व्यवस्थापन:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
```


**मुख्य घटक:**

- **स्थिती संदर्भ**: टप्प्यांदरम्यान संकलित डेटा आणि निर्णयांची देखभाल करतो
- **चेकपॉइंट्स**: गुणवत्तेसाठी टप्प्यांमधील प्रमाणीकरण बिंदू
- **Azure OpenAI क्लायंट**: सर्व वर्कफ्लो टप्प्यांमध्ये सुसंगत AI मॉडेल प्रवेश

## 🎨 **अनुक्रमिक वर्कफ्लो डिझाइन पॅटर्न्स**

### 📝 **दस्तऐवज प्रक्रिया पाइपलाइन**
```
Raw Document → Content Extraction → Analysis → Validation → Structured Output
```

### 🎯 **सामग्री निर्मिती वर्कफ्लो**
```
Brief/Requirements → Research → Content Creation → Review → Final Polish
```

### 🔍 **गुणवत्ता आश्वासन पाइपलाइन**
```
Initial Review → Technical Validation → Compliance Check → Final Approval
```

### 💼 **व्यावसायिक बुद्धिमत्ता वर्कफ्लो**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

## 🏢 **एंटरप्राइझ अनुक्रमिक फायदे**

### 🎯 **विश्वसनीयता & गुणवत्ता**
- **नियत प्रक्रिया**: संरचित टप्प्यांमधून सातत्यपूर्ण, पुनरावृत्ती शक्य परिणाम
- **गुणवत्ता दाले**: प्रत्येक टप्प्यावर प्रमाणीकरण बिंदू गुणवत्तेची खात्री करतात
- **त्रुटी पृथक्करण**: एका टप्प्यातील समस्या पुढील टप्प्यांपर्यंत पोहोचत नाहीत
- **ऑडिट ट्रेल्स**: प्रत्येक टप्प्यावर निर्णय आणि रूपांतरणांचे संपूर्ण ट्रॅकिंग

### 📈 **स्केलेबिलिटी & कार्यक्षमता**
- **मॉड्युलर डिझाइन**: प्रत्येक टप्पा स्वतंत्रपणे अनुकूलित केला जाऊ शकतो
- **संसाधन व्यवस्थापन**: टप्प्यांमधील AI मॉडेल संसाधनांचे कार्यक्षम वाटप
- **स्थिती अनुकूलन**: उत्कृष्ट कार्यक्षमतेसाठी टप्प्यांमधील किमान स्थिती हस्तांतरण
- **समानांतर टप्पा गट**: एकाच वेळी अनेक अनुक्रमिक वर्कफ्लो चालवू शकतात

### 🔒 **सुरक्षा & अनुपालन**
- **टप्पा-स्तरीय सुरक्षा**: वेगवेगळ्या प्रक्रिया टप्प्यांसाठी वेगळ्या सुरक्षा धोरणे
- **डेटा प्रमाणीकरण**: प्रत्येक चेकपॉइंटवर डेटा अखंडता आणि अनुपालन सुनिश्चित करा
- **प्रवेश नियंत्रण**: वेगवेगळ्या वर्कफ्लो टप्प्यांसाठी तपशीलवार परवानग्या
- **नियामक अनुपालन**: संरचित प्रक्रियेने नियामक आवश्यकता पूर्ण करा

### 📊 **नियंत्रण आणि विश्लेषण**
- **टप्पा-स्तरीय मेट्रिक्स**: प्रत्येक वर्कफ्लो टप्प्याचे कार्यक्षमता नियंत्रण
- **बॉटलनेक ओळख**: मंद टप्प्यांची ओळख करून त्यांचे अनुकूलन करा
- **गुणवत्ता मेट्रिक्स**: प्रत्येक टप्प्यावर गुणवत्ता आणि यशाच्या दरांचा मागोवा
- **प्रक्रिया अनुकूलन**: टप्पा-स्तरीय विश्लेषणांवर आधारित सतत सुधारणा

चला मजबूत अनुक्रमिक AI प्रक्रिया पाइपलाइन तयार करूया! 🚀

1. **विक्री एजंट**: प्रतिमेतून फर्निचर आयटम ओळखतो आणि शिफारसी देतो

### 🎯 प्रत्यक्ष जगातील अनुप्रयोग

हा अनुक्रमिक पॅटर्न योग्य आहे:
- **ई-कॉमर्स**: उत्पादन विश्लेषण → किंमत ठरवणे → कोट तयार करणे
- **रिअल इस्टेट**: मालमत्ता विश्लेषण → मूल्यांकन → लिस्टिंग तयार करणे
- **इन्शुरन्स**: दावे विश्लेषण → मूल्यमापन → कोट तयार करणे
- **सामग्री निर्मिती**: संशोधन → लेखन → संपादन → प्रकाशन

### 🔍 स्थिती प्रवाह समजून घेणे

अनुक्रमातील प्रत्येक एजंट प्राप्त करतो:
- **मूळ इनपुट**: प्रारंभिक वापरकर्ता संदेश (प्रतिमा + मजकूर)
- **मागील एजंट आउटपुट्स**: संभाषण इतिहासातील सर्व मागील एजंट प्रतिसाद
- **संकलित संदर्भ**: संपूर्ण वर्कफ्लोमध्ये राखलेली स्थिती

हे प्रगत मल्टी-टप्पा प्रक्रिया सक्षम करते जिथे प्रत्येक एजंट सर्व मागील टप्प्यांमधील व्यापक संदर्भावर आधार घेऊन काम करतो.
