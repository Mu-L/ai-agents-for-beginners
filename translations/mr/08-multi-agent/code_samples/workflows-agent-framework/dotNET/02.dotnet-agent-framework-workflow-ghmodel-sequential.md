# ⏩ Azure OpenAI (Responses API) सह अनुक्रमिक एजंट वर्कफ्लो (.NET)

## 📋 प्रगत अनुक्रमिक प्रक्रिया ट्युटोरियल

हा नोटबुक Microsoft Agent Framework for .NET आणि Azure OpenAI (Responses API) वापरून **अनुक्रमिक वर्कफ्लो नमुने** दर्शवितो. येथे तुम्हाला अशी सुसारी, टप्प्याटप्प्याने प्रक्रिया पाइपलाइन कशी तयार करायची हे शिकवले जाईल जिथे एजंट विशिष्ट क्रमाने कार्य करतात, प्रत्येक टप्पा मागील टप्प्याच्या परिणामांवर आधारित असतो.

## 🎯 शिकण्याच्या उद्दिष्टे

### 🔄 **अनुक्रमिक प्रक्रिया आर्किटेक्चर**
- **रेषीय वर्कफ्लो डिझाईन**: स्पष्ट अवलंबित्वांसह टप्प्याटप्प्याने प्रक्रिया पाइपलाइन तयार करा
- **स्थिती व्यवस्थापन**: अनुक्रमिक वर्कफ्लो टप्प्यांमध्ये संदर्भ आणि डेटा प्रवाह जपणे
- **Azure OpenAI (Responses API)**: बहु-टप्प्याच्या .NET वर्कफ्लोमध्ये Azure OpenAI मॉडेलचा उपयोग करा
- **उद्योग पाइपलाइन नमुने**: उत्पादन-तयार अनुक्रमिक प्रक्रिया प्रणाली तयार करा

### 🏗️ **प्रगत अनुक्रमिक नमुने**
- **स्टेज-गेट प्रक्रिया**: वर्कफ्लो टप्प्यांमध्ये वैधता तपासणी बिंदू अंमलात आणा
- **संदर्भ संरक्षण**: सर्व टप्प्यांमध्ये स्थिती आणि संकलित ज्ञान जतन करा
- **त्रुटी प्रसार थांबविणे**: अनुक्रमिक प्रक्रिया साखळीत त्वरीत अपयश हाताळा
- **कार्यक्षमता ऑप्टिमायझेशन**: किमान ओव्हरहेडसह कार्यक्षम अनुक्रमिक अंमलबजावणी

### 🏢 **उद्योग अनुक्रमिक अनुप्रयोग**
- **डॉक्युमेंट प्रक्रिया पाइपलाइन**: बहु-टप्प्याचा दस्तऐवज विश्लेषण, रूपांतरण व वैधता
- **गुणवत्ता आश्वासन वर्कफ्लो**: अनुक्रमिक पुनरावलोकन, वैधता आणि मान्यता प्रक्रिया
- **सामग्री उत्पादन पाइपलाइन**: संशोधन → लेखन → संपादन → पुनरावलोकन → प्रकाशन
- **व्यवसाय प्रक्रिया ऑटोमेशन**: स्पष्ट स्टेज अवलंबित्वांसह बहु-टप्प्याच्या व्यवसाय वर्कफ्लो

## ⚙️ पूर्वतयारी आणि सेटअप

### 📦 **आवश्यक NuGet पॅकेजेस**

.NET अनुक्रमिक वर्कफ्लो साठी आवश्यक पॅकेजेस:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Azure OpenAI कॉन्फिगरेशन**

**पर्यावरण सेटअप (.env फाइल):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**कॉन्फिगरेशन व्यवस्थापन:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **अनुक्रमिक वर्कफ्लो आर्किटेक्चर**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: विश्लेषण एजंट]
    B --> C[तपासणीबिंदू 1]

    D --> E[तपासणीबिंदू 2]
```

**महत्वाचे घटक:**
- **अनुक्रमिक एजंट्स**: प्रत्येक प्रक्रिया टप्प्यासाठी विशेष एजंट्स
- **स्थिती संदर्भ**: टप्प्यांमधील संकलित डेटा आणि निर्णय टिकवतो
- **चेकपॉइंट्स**: दर्जा आणि सातत्य सुनिश्चित करण्यासाठी टप्प्यांमधील वैधता बिंदू
- **Azure OpenAI क्लायंट**: सर्व वर्कफ्लो टप्प्यांमध्ये सुसंगत AI मॉडेल प्रवेश

## 🎨 **अनुक्रमिक वर्कफ्लो डिझाईन नमुने**

### 📝 **डॉक्युमेंट प्रक्रिया पाइपलाइन**
```
Raw Document → Content Extraction → Analysis → Validation → Structured Output
```

### 🎯 **सामग्री निर्मिती वर्कफ्लो**
```
Brief/Requirements → Research → Content Creation → Review → Final Polish
```

### 🔍 **गुणवत्ता आश्वासन पाइपलाइन**
```
Initial Review → Technical Validation → Compliance Check → Final Approval
```

### 💼 **व्यवसाय बुद्धिमत्ता वर्कफ्लो**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

## 🏢 **उद्योग अनुक्रमिक फायदे**

### 🎯 **विश्वसनीयता आणि गुणवत्ता**
- **निर्धारित प्रक्रिया**: रचनाबद्ध टप्प्यांमधून सुसंगत, पुनरावृत्तीयोग्य निष्पत्ती
- **गुणवत्ता गेट्स**: प्रत्येक टप्प्यावर गुणवत्ता सुनिश्चित करणारे वैधता तपासणी बिंदू
- **त्रुटी विभाजन**: एका टप्प्यातील समस्या पुढील टप्प्यांमध्ये पसरणार नाहीत
- **ऑडिट ट्रेल्स**: प्रत्येक टप्प्यातील निर्णय आणि रूपांतरणे संपूर्णपणे ट्रॅक करा

### 📈 **स्केलेबिलिटी आणि कामगिरी**
- **मॉड्यूलर डिझाईन**: प्रत्येक टप्पा स्वतंत्रपणे ऑप्टिमाइझ केला जाऊ शकतो
- **संसाधन व्यवस्थापन**: टप्प्यांमध्ये AI मॉडेल संसाधनांचे कार्यक्षम वाटप
- **स्थिती ऑप्टिमायझेशन**: टप्प्यांमध्ये स्थिती हस्तांतरित करणे कमी करून उत्कृष्ठ कामगिरी
- **सामांतर टप्पा गट**: एकाच वेळी एकाधिक अनुक्रमिक वर्कफ्लो चालवता येतात

### 🔒 **सुरक्षा आणि अनुपालन**
- **टप्पा-स्तरीय सुरक्षा**: वेगवेगळ्या प्रक्रिया टप्प्यांसाठी वेगवेगळ्या सुरक्षा धोरणे
- **डेटा वैधता**: प्रत्येक तपासणी बिंदूमध्ये डेटा अखंडता आणि अनुपालन सुनिश्चित करा
- **प्रवेश नियंत्रण**: वेगवेगळ्या वर्कफ्लो टप्प्यांसाठी सूक्ष्म परवानग्या
- **नियमित अनुपालन**: रचनाबद्ध प्रक्रियेद्वारे नियमांचे पालन करा

### 📊 **निगराणी आणि विश्लेषण**
- **टप्पा-स्तरीय मेट्रिक्स**: प्रत्येक वर्कफ्लो टप्प्याची कामगिरी निरीक्षण
- **बॉटलनेक ओळखणे**: हळू टप्पे ओळखून ऑप्टिमाइझ करा
- **गुणवत्ता मेट्रिक्स**: प्रत्येक टप्प्यावर गुणवत्ता आणि यशाची दर नोंदवा
- **प्रक्रिया ऑप्टिमायझेशन**: टप्पा-स्तरीय विश्लेषणावर आधारित सातत्यपूर्ण सुधारणा

चला मजबूत अनुक्रमिक AI प्रक्रिया पाइपलाइन तयार करूया! 🚀

1. **विक्री एजंट**: प्रतिमेमधून फर्निचर आयटम ओळखतो आणि शिफारसी देतो

### 🎯 वास्तविक जगातील अनुप्रयोग

हा अनुक्रमिक नमुना आदर्श आहे:
- **ई-कॉमर्स**: उत्पादन विश्लेषण → किंमत निर्धारण → कोट जनरेशन
- **रिअल एस्टेट**: मालमत्ता विश्लेषण → मूल्यांकन → लिस्टिंग तयार करणे
- **विमा**: दावे विश्लेषण → मूल्यांकन → कोट जनरेशन
- **सामग्री निर्मिती**: संशोधन → लेखन → संपादन → प्रकाशन

### 🔍 स्थिती प्रवाह समजून घेणे

अनुक्रमातील प्रत्येक एजंट प्राप्त करतो:
- **मूळ इनपुट**: प्रारंभिक वापरकर्ता संदेश (प्रतिमा + मजकूर)
- **मागील एजंट आउटपुट्स**: संभाषण इतिहासातील सर्व मागील एजंट प्रतिसाद
- **संकलित संदर्भ**: कार्यप्रवाहादरम्यान संपूर्ण स्थिती टिकवलेली

हे सखोल बहु-टप्प्याच्या प्रक्रियेची परवानगी देते जिथे प्रत्येक एजंट सर्व मागील टप्प्यांमधील संपूर्ण संदर्भावर आधारित कार्य करतो.
