# 🔄 Azure OpenAI (Responses API) सह मूलभूत एजंट वर्कफ्लोज (.NET)

## 📋 वर्कफ्लो ऑर्केस्ट्रेशन ट्युटोरियल

हे नोटबुक Microsoft Agent Framework for .NET आणि Azure OpenAI (Responses API) वापरून कसे सुसंस्कृत **एजंट वर्कफ्लोज** तयार करायचे ते दाखवते. तुम्ही बहु-स्टेप व्यवसाय प्रक्रिया तयार करण्यास शिकाल जिथे AI एजंट एकत्र येऊन संरचित ऑर्केस्ट्रेशन पॅटर्न्सद्वारे जटिल कार्य पूर्ण करतात.

## 🎯 शिकण्याची उद्दिष्टे

### 🏗️ **वर्कफ्लो आर्किटेक्चर मूलतत्त्वे**
- **वर्कफ्लो बिल्डर**: जटिल बहु-स्टेप AI प्रक्रिया डिझाइन आणि ऑर्केस्ट्रेट करा
- **एजंट समन्वय**: वर्कफ्लोजमध्ये अनेक तज्ञ एजंट समन्वयित करा
- **Azure OpenAI (Responses API)**: वर्कफ्लोजमध्ये Azure OpenAI Responses API चा वापर करा
- **व्हिज्युअल वर्कफ्लो डिझाइन**: वर्कफ्लो संरचना तयार करा आणि कल्पना स्पष्ट करा

### 🔄 **प्रक्रिया ऑर्केस्ट्रेशन पॅटर्न्स**
- **क्रमवारीने प्रक्रिया**: अनेक एजंट कार्यांना तर्कसंगत क्रमाने जोडणे
- **स्टेट मॅनेजमेंट**: वर्कफ्लो स्टेजेसमध्ये संदर्भ आणि डेटा प्रवाह राखणे
- **त्रुटी हाताळणी**: मजबूत त्रुटी पुनर्प्राप्ती आणि वर्कफ्लो लवचिकता अमलात आणणे
- **कार्यक्षमता अनुकूलन**: एंटरप्राइज स्तरावर कार्यक्षम वर्कफ्लोज डिझाइन करणे

### 🏢 **एंटरप्राइज वर्कफ्लो अनुप्रयोग**
- **व्यवसाय प्रक्रिया स्वयंचलिती**: जटिल संघटनात्मक वर्कफ्लोज स्वयंचलित करा
- **सामग्री उत्पादन पाइपलाइन**: पुनरावलोकन आणि मंजुरी स्टेजसह संपादकीय वर्कफ्लोज
- **ग्राहक सेवा स्वयंचलिती**: बहु-स्टेप ग्राहक चौकशी निराकरण
- **डेटा प्रक्रिया वर्कफ्लोज**: AI-शक्ती संपन्न रूपांतरणासह ETL वर्कफ्लोज

## ⚙️ आवश्यकता आणि सेटअप

### 📦 **आवश्यक NuGet पॅकेजेस**

हा वर्कफ्लो डेमोंस्ट्रेशन काही प्रमुख .NET पॅकेजेस वापरतो:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI कॉन्फिगरेशन**

**पर्यावरण सेटअप (.env फाइल):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Azure OpenAI प्रवेश:**
1. Azure पोर्टलमध्ये Azure OpenAI संसाधन तयार करा
2. एक मॉडेल तैनात करा (उदाहरणार्थ, `gpt-5-mini`) आणि तैनात नाव नोंदवा
3. `az login` वापरून साइन इन करा आणि वर दाखविल्यासारखे पर्यावरण चल सेट करा

### 🏗️ **वर्कफ्लो आर्किटेक्चर अवलोकन**

```mermaid
graph TD
    A[वर्कफ्लो बिल्डर] --> B[एजंट नोंदणी]
    B --> C[वर्कफ्लो अंमलबजावणी इंजिन]
    C --> D[Agent 1: सामग्री जनक]
    C --> E[Agent 2: सामग्री मूल्यांकनकर्ता] 
    D --> F[वर्कफ्लो परिणाम]
    E --> F
    G[Azure OpenAI (उत्तर API)] --> D
    G --> E
```

**प्रमुख घटक:**
- **WorkflowBuilder**: वर्कफ्लोज डिझाइन करण्यासाठी मुख्य ऑर्केस्ट्रेशन इंजिन
- **AIAgent**: विशिष्ट क्षमता असलेले वैयक्तिक तज्ञ एजंट
- **Azure OpenAI क्लायंट**: Azure OpenAI Responses API एकीकरण
- **Execution Context**: वर्कफ्लो स्टेजेस दरम्यान स्थिती आणि डेटा प्रवाह व्यवस्थापन

## 🎨 **एंटरप्राइज वर्कफ्लो डिझाइन पॅटर्न्स**

### 📝 **सामग्री उत्पादन वर्कफ्लो**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **डॉक्युमेंट प्रोसेसिंग पाइपलाइन**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **बिझनेस इंटेलिजन्स वर्कफ्लो**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **ग्राहक सेवा स्वयंचलिती**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **एंटरप्राइज फायदे**

### 🎯 **विश्वसनीयता आणि स्केलेबिलिटी**
- **निर्धारित अमलबजावणी**: सातत्यपूर्ण, पुनरावृत्तीक्षम वर्कफ्लो निकाल
- **त्रुटी पुनर्प्राप्ती**: कोणत्याही वर्कफ्लो स्टेजवर अपयशांचे सौम्य हाताळणी
- **कार्यक्षमता निरीक्षण**: अमलबजावणी मेट्रिक्स आणि ऑप्टिमायझेशन संधींचे ट्रॅकिंग
- **संसाधन व्यवस्थापन**: AI मॉडेल संसाधनांचा प्रभावी वाटप आणि वापर

### 🔒 **सुरक्षा आणि अनुपालन**
- **सुरक्षित प्रमाणीकरण**: `az login` (AzureCliCredential) द्वारे Microsoft Entra ID प्रमाणीकरण
- **ऑडिट ट्रेल्स**: वर्कफ्लो अमलबजावणी आणि निर्णय बिंदूंचे संपूर्ण लॉगिंग
- **प्रवेश नियंत्रण**: वर्कफ्लो अमलबजावणी आणि निरीक्षणासाठी सूक्ष्म परवानग्या
- **डेटा गोपनीयता**: वर्कफ्लोजमध्ये संवेदनशील माहितीचे सुरक्षित हाताळणी

### 📊 **निरिक्षण आणि व्यवस्थापन**
- **व्हिज्युअल वर्कफ्लो डिझाइन**: प्रक्रिया प्रवाह आणि अवलंबित्वांची स्पष्ट सादरीकरण
- **अमलबजावणी निरीक्षण**: वर्कफ्लो प्रगती आणि कार्यक्षमता यांचे वास्तविक वेळेतील ट्रॅकिंग
- **त्रुटी अहवाल**: सविस्तर त्रुटी विश्लेषण आणि डिबगिंग क्षमता
- **कार्यक्षमता विश्लेषण**: ऑप्टिमायझेशन आणि क्षमता नियोजनासाठी मेट्रिक्स

चला तुमचा पहिला एंटरप्राइज-तयार AI वर्कफ्लो तयार करूया! 🚀

## 💻 कोड चालवणे

संपूर्ण अंमलबजावणी `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` मध्ये उपलब्ध आहे. ही फाइल दर्शवते:

1. **पर्यावरण कॉन्फिगरेशन** - `.env` फाइलमधून Azure OpenAI कॉन्फिगरेशन लोड करणे
2. **Azure OpenAI क्लायंट सेटअप** - Azure OpenAI Responses API वापरण्यासाठी क्लायंट कॉन्फिगर करणे
3. **एजंट निर्मिती** - विशिष्ट एजंट्स (Front Desk आणि Concierge) परिभाषित करणे
4. **वर्कफ्लो बिल्डर** - क्रमवार प्रक्रिया असलेले बहु-एजंट वर्कफ्लो तयार करणे
5. **वर्कफ्लो अमलबजावणी** - स्ट्रीमिंग परिणामांसह वर्कफ्लो चालवणे

### 📝 अपेक्षित आउटपुट

वर्कफ्लो खालीलप्रमाणे कार्य करेल:
1. आपली प्रवास स्थळ विनंती स्वीकारेल ("मला पॅरिसला जावे आहे")
2. Front Desk एजंट प्राथमिक शिफारस प्रदान करेल
3. Concierge एजंट शिफारस पुनरावलोकन आणि सुधारित करेल
4. अंतिम आउटपुट पूर्ण संभाषण प्रवाह दर्शवेल

### 🔧 सानुकूलन

तुम्ही वर्कफ्लो सानुकूलित करू शकता:
- एजंट सूचनांमध्ये बदल करून त्यांचे वर्तन बदलणे
- अधिक एजंट्स जोडून जटिल बहु-स्टेप वर्कफ्लोज तयार करणे
- वेगवेगळ्या परिस्थितींची चाचणी करण्यासाठी वापरकर्ता संदेश बदलणे
- वेगवेगळे अमलबजावणी पॅटर्न तयार करण्यासाठी वर्कफ्लो एजेस समायोजित करणे
