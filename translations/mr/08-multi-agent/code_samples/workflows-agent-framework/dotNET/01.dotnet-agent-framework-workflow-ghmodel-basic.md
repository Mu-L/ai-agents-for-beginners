# 🔄 Azure OpenAI (Responses API) सह मूलभूत एजंट कार्यप्रवाह (.NET)

## 📋 कार्यप्रवाह आयोजन ट्युटोरियल

हा नोटबुक Microsoft Agent Framework for .NET आणि Azure OpenAI (Responses API) वापरून प्रगत **एजंट कार्यप्रवाह** कसे तयार करायचे ते दाखवते. तुम्हाला बहु-टप्प्यांचे व्यावसायिक प्रक्रियेत AI एजंट्स एकत्र काम करून रचना केलेल्या आयोजन नमुन्यांद्वारे क्लिष्ट कार्ये कशी पार पाडतात हे शिकायला मिळेल.

## 🎯 शिक्षण उद्दिष्टे

### 🏗️ **कार्यप्रवाह स्थापत्य मूलसिद्धांत**
- **वर्कफ्लो बिल्डर**: क्लिष्ट बहु-टप्प्यांच्या AI प्रक्रियांना डिझाईन आणि आयोजित करा
- **एजंट समन्वय**: कार्यप्रवाहांत अनेक तज्ञ एजंट्सचे समन्वय करा
- **Azure OpenAI (Responses API)**: कार्यप्रवाहांमध्ये Azure OpenAI Responses API चा उपयोग करा
- **दृश्य कार्यप्रवाह डिझाईन**: चांगल्या समजुतीसाठी कार्यप्रवाह संरचनांची निर्मिती आणि दृश्यमानता करा

### 🔄 **प्रक्रिया आयोजन नमुने**
- **अनुक्रमिक प्रक्रिया**: अनेक एजंट कार्ये तार्किक क्रमाने साखळीबद्ध करा
- **स्थिती व्यवस्थापन**: कार्यप्रवाह टप्प्यांमधील संदर्भ आणि डेटा प्रवाह राखा
- **त्रुटी हाताळणी**: मजबूत त्रुटी पुनर्प्राप्ती आणि कार्यप्रवाह स्थिरता लागू करा
- **कामगिरी ऑप्टिमायझेशन**: एंटरप्राइझ-स्तरीय ऑपरेशन्स साठी कार्यक्षम कार्यप्रवाह डिझाईन करा

### 🏢 **एंटरप्राइझ कार्यप्रवाह अनुप्रयोग**
- **व्यवसाय प्रक्रिया स्वयंचलन**: क्लिष्ट संस्थात्मक कार्यप्रवाह स्वयंचलित करा
- **सामग्री उत्पादन पाइपलाइन**: पुनरावलोकन आणि मंजुरी टप्प्यांसह संपादकीय कार्यप्रवाह
- **ग्राहक सेवा स्वयंचलन**: बहु-टप्प्यांचे ग्राहक चौकशी निराकरण
- **डेटा प्रक्रिया कार्यप्रवाह**: AI-सक्षम परिवर्तनांसह ETL कार्यप्रवाह

## ⚙️ पूर्वअट व सेटअप

### 📦 **आवश्यक NuGet पॅकेजेस**

हा कार्यप्रवाह प्रदर्शन काही मुख्य .NET पॅकेजेस वापरतो:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI संरचना**

**पर्यावरण सेटअप (.env फाइल):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI प्रवेश:**
1. Azure पोर्टलमध्ये Azure OpenAI संसाधन तयार करा
2. एखादा मॉडेल (उदा., `gpt-4.1-mini`) तैनात करा आणि तैनातीचे नाव नोंद करा
3. `az login` वापरून लॉगिन करा आणि वरीलप्रमाणे पर्यावरण व्हेरिएबल कॉन्फिगर करा

### 🏗️ **कार्यप्रवाह स्थापत्य अवलोकन**

```mermaid
graph TD
    A[वर्कफ्लो बिल्डर] --> B[एजंट नोंदणी]
    B --> C[वर्कफ्लो अंमलबजावणी इंजिन]
    C --> D[Agent 1: सामग्री तयार करणारा
    C --> E[Agent 2: सामग्री पुनरावलोकक 
    D --> F[वर्कफ्लो परिणाम]
    E --> F
    G[Azure OpenAI (प्रतिक्रिया API)] --> D
    G --> E
```

**मुख्य घटक:**
- **WorkflowBuilder**: कार्यप्रवाह डिझाइनसाठी मुख्य आयोजन इंजिन
- **AIAgent**: विशिष्ट क्षमता असलेले स्वतंत्र तज्ञ एजंट्स
- **Azure OpenAI Client**: Azure OpenAI Responses API समाकलन
- **Execution Context**: कार्यप्रवाह टप्प्यांमधील स्थिती आणि डेटा प्रवाह व्यवस्थापित करतो

## 🎨 **एंटरप्राइझ कार्यप्रवाह डिझाईन नमुने**

### 📝 **सामग्री उत्पादन कार्यप्रवाह**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **दस्तऐवज प्रक्रिया पाइपलाइन**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **व्यवसाय बुद्धिमत्ता कार्यप्रवाह**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **ग्राहक सेवा स्वयंचलन**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **एंटरप्राइझ फायदे**

### 🎯 **विश्वसनीयता व प्रमाणापयोग्यता**
- **निर्धारीत अंमलबजावणी**: सातत्यपूर्ण, पुनरावृत्ती होणारे कार्यप्रवाह परिणाम
- **त्रुटी पुनर्प्राप्ती**: कार्यप्रवाह टप्प्यामधील अपयशाचे सुसंवारित हस्तक्षेप
- **कामगिरी देखरेख**: कार्यप्रवाह अंमलबजावणी मेट्रिक्स आणि ऑप्टिमायझेशन संधींचे ट्रॅकिंग
- **संसाधन व्यवस्थापन**: AI मॉडेल संसाधनांचे कार्यक्षम वाटप आणि वापर

### 🔒 **सुरक्षा व अनुपालन**
- **सुरक्षित प्रमाणीकरण**: `az login` (AzureCliCredential) द्वारे Microsoft Entra ID प्रमाणीकरण
- **ऑडिट ट्रेल्स**: कार्यप्रवाह अंमलबजावणी आणि निर्णय बिंदूंचे पूर्ण लॉगिंग
- **प्रवेश नियंत्रण**: कार्यप्रवाह अंमलबजावणी व देखरेखीच्या सूक्ष्म परवानग्या
- **डेटा गोपनीयता**: संपूर्ण कार्यप्रवाहांमध्ये संवेदनशील माहितीचे सुरक्षित हाताळणी

### 📊 **अवलोकनक्षमता व व्यवस्थापन**
- **दृश्य कार्यप्रवाह डिझाईन**: प्रक्रिया प्रवाह व अवलंबित्वांचे स्पष्ट दर्शन
- **अंमलबजावणी देखरेख**: कार्यप्रवाह प्रगती आणि कामगिरीचे रिअल-टाइम ट्रॅकिंग
- **त्रुटी अहवाल**: सविस्तर त्रुटी विश्लेषण आणि डीबगिंग क्षमता
- **कामगिरी विश्लेषण**: ऑप्टिमायझेशन व क्षमता नियोजनासाठी मेट्रिक्स

चला तुमचा पहिला एंटरप्राइझ-तयार AI कार्यप्रवाह तयार करूया! 🚀

## 💻 कोड चालविणे

संपूर्ण अंमलबजावणी `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` मध्ये उपलब्ध आहे. हा फाइल खालील गोष्टी दाखवतो:

1. **पर्यावरण संरचना** - `.env` फाइलमधून Azure OpenAI संरचना लोड करणे
2. **Azure OpenAI क्लायंट सेटअप** - Azure OpenAI Responses API वापरण्यासाठी क्लायंट कॉन्फिगर करणे
3. **एजंट निर्मिती** - तज्ञ एजंट्स (फ्रंट डेस्क आणि कॉन्सिएर्ज) परिभाषित करणे
4. **वर्कफ्लो बिल्डर** - अनुक्रमिक प्रक्रियेसह बहु-एजंट कार्यप्रवाह तयार करणे
5. **वर्कफ्लो अंमलबजावणी** - स्ट्रीमिंग परिणामांसह कार्यप्रवाह चालविणे

### 🚀 उदाहरण चालविणे

```bash
# स्क्रिप्ट एक्सिक्युटेबल करा (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# वर्कफ्लो चालवा
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Windows वर:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 अपेक्षित आउटपुट

कार्यप्रवाह:
1. तुमच्या प्रवास स्थळाची विनंती स्वीकारेल ("मला पॅरिसला जायचे आहे")
2. फ्रंट डेस्क एजंट पहिल्या शिफारशी देतो
3. कॉन्सिएर्ज एजंट शिफारशीचा पुनरावलोकन करतो आणि सुधारणा करतो
4. अंतिम आउटपुट संपूर्ण संभाषण प्रवाह दर्शवतो

### 🔧 सानुकूलन

तुम्ही कार्यप्रवाह खालीलप्रमाणे सानुकूलित करू शकता:
- एजंट सूचनांमध्ये बदल करुन त्यांची वर्तणूक बदलणे
- क्लिष्ट बहु-टप्प्यांचे कार्यप्रवाह तयार करण्यासाठी अधिक एजंट्स जोडणे
- विविध परिस्थिती तपासण्यासाठी वापरकर्ता संदेश बदलणे
- विविध अंमलबजावणी नमुने तयार करण्यासाठी कार्यप्रवाह कड्यांमध्ये समायोजन करणे
