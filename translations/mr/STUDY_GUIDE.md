# नवशिक्यांसाठी एआय एजंट्स - अध्ययन मार्गदर्शक

हा मार्गदर्शक कोर्समधून जाताना व्यावहारिक साथीदार म्हणून वापरा. तो धडे बदलण्याचा हेतू नाही.
तो तुम्हाला ठरवायला मदत करतो की कुठून सुरुवात करावी, प्रत्येक धड्यामध्ये काय पाहावे,
आणि कल्पना कशा जोडायच्या एक लहान काम करणारा एजंट डेमोत.


जर हे तुमचं पहिलं परीक्षण असेल, तर सोपं सुरुवात करा:

1. [Course Setup](./00-course-setup/README.md) वाचा.
2. क्रमाने धडे ०१-०६ पूर्ण करा.
३. शिकत असताना मनात एक लहान डेमो कल्पना ठेवा.
४. प्रत्येक धड्यानंतर विचार करा: "आता माझा एजंट काय करू शकतो ज्याला तो आधी करू शकत नव्हता?"


## लक्षात ठेवण्यासाठी एक साधा डेमो

एजंट शिकण्याचा एक चांगला मार्ग म्हणजे कोर्समध्ये एका डेमो कल्पनेचा पाठपुरावा करणं.

उदाहरण डेमो: **एक कोर्स हेल्पर एजंट**.

वापरकर्ता विचारतो:

> "मला शिकायचं आहे की एजंट कसे टूल्स वापरतात. योग्य धडे शोधा, जे मला आधी वाचायला हवे त्याचा सारांश द्या,
> आणि मला एक छोटा सराव कार्य द्या."

एक नियमित चैटबॉट ज्या माहितीतून तो आधीपासून आहे त्यातून उत्तर देऊ शकतो. पण एजंट आणखी करू शकतो:

1. **कोर्स फाईल्स वाचा किंवा शोधा** आणि योग्य धडे शोधा.
2. **टूल्स वापरा** धडा लिंक, उदाहरणे, किंवा सहाय्यक सामग्री मिळवण्यासाठी.
3. **कोणत्याही एका लांब उत्तराऐवजी योजनाबद्ध शिका** छोट्या टप्प्यांत.
4. चालू संभाषणाचा **संदर्भ वापरा** आणि शिष्याच्या ध्येयावर लक्ष ठेवा.
5. **महत्त्वाच्या प्राधान्यांची आठवण ठेवा** जर अॅप्लिकेशनला मेमरी सपोर्ट असेल तर.
6. **ट्रेस, उद्धरणे, किंवा लॉग्स दाखवा** ज्यामुळे वापरकर्ता काय घडलं ते समजू शकेल.
7. जोखमीच्या क्रिया किंवा संवेदनशील माहिती वापरण्यापूर्वी **गार्डरेल्स लागू करा**.


प्रत्येक धडा शिकताना, या डेमोवर परत येऊन विचारा: या धड्यानं कोणती नवीन क्षमता वाढली आहे?


## तुम्ही जे बांधत आहात

कोर्सच्या शेवटी, तुम्हाला अशी एजंट प्रणाली समजावून सांगता आणि तयार करता यायला हवी जी या भागांना एकत्र करते:


| भाग | सोप्या भाषेत अर्थ | डेमोत |
|------|------------------------|-------------|
| मॉडेल | वापरकर्त्याच्या विनंतीचे भाष्य करणारा यंत्र | समजतो की शिकणाऱ्याला टूल वापराविषयी धडे हवेत |
| टूल्स | फंक्शन्स, APIs, फाईल्स, ब्राउझर्स, किंवा एजंट जे वापरू शकतो | रेपोमध्ये शोधतो किंवा धडा सामग्री मिळवतो |
| ज्ञान | उत्तरा आधार देण्यासाठी वापरलेली कागदपत्रे किंवा डेटा | कोर्स README फाईल्स आणि धडा साहित्य |
| संदर्भ | पुढील मॉडेल कॉलमध्ये समाविष्ट माहिती | वापरकर्त्याचा उद्देश आणि टूल निकाल |
| मेमरी | नंतर वापरासाठी जतन केलेली माहिती | शिकणाऱ्याला हाताळण्याचे Python उदाहरण आवडतात |
| नियोजन | मोठ्या उद्देशाला लहान टप्प्यांत विभागणे | धडे शोधा, सारांश करा, सराव सूचवा |
| संयोजन | टूल्स, टप्पे, किंवा एजंट्स यांच्यामध्ये काम वाटप | एक योजनाकार शोध टूल कॉल करतो, नंतर सारांशकार |
| विश्वास | सुरक्षितता, सुरक्षा, मूल्यांकन, आणि निरीक्षण | टूल कॉल्स लॉग करतो आणि मोठ्या परिणामांच्या क्रियांपूर्वी विचारतो |

## मॉडेल्स आणि प्रदाते

कोर्स कोड सॅम्पलमध्ये **Microsoft Agent Framework (MAF)** वापरले आहे आणि **Azure OpenAI Responses API** चालवते — पुढे जाण्यासाठी शिफारस केलेली API, जी चैट पूर्णता, टूल कॉलिंग, मल्टीमॉडाल इनपुट, आणि स्थितीवार संभाषणे एका API पृष्ठभागात जोडते. तुम्ही किंवा तर **Microsoft Foundry** प्रोजेक्ट (with `FoundryChatClient`) किंवा थेट Azure OpenAI (with `OpenAIChatClient`) कनेक्ट करता.


धडे पूर्ण करत असताना, तुमच्याकडे काही प्रदाता पर्याय आहेत:

- **Microsoft Foundry / Azure OpenAI (Responses API)** — धड्यांत सर्वाधिक वापरल्या जाणाऱ्या मुख्य मार्गावरून. कीलेस Entra ID प्रमाणीकरणासाठी `az login` वापरा.
- **Foundry Local** — ऑन-डिव्हाइस पूर्णपणे OpenAI-समर्पक API द्वारे मॉडेल चालवा (कोणतीही क्लाऊड नाही, कोणतीही API की नाही). ऑफलाइन किंवा खर्चशून्य प्रयोगांसाठी आदर्श. पहा [Course Setup](./00-course-setup/README.md).
- **MiniMax** — मोठ्या-कॉन्टेक्स्ट मॉडेल्ससह OpenAI-समर्पक प्रदाता, एक ड्रॉप-इन पर्याय म्हणून वापरता येतो.

> **टीप:** GitHub मॉडेल्स रद्द होत आहेत (जुलै 2026 मध्ये बंद होणार) आणि Responses API ला समर्थन करत नाहीत. नमुने Azure OpenAI / Microsoft Foundry वापरण्यासाठी अद्ययावत केले गेले आहेत.

## सुरुवातीच्या सोप्या भाषेत मुख्य कल्पना

### उपकरणे

उपकरण म्हणजे एजंट मॉडेलच्या बाहेर काम करण्यासाठी कॉल करू शकतो असा काहीतरी. चांगल्या उपकरणाचे
स्पष्ट नाव, मर्यादित काम, टाइप केलेले इनपुट, अपेक्षित आउटपुट, आणि अपयश होण्याचा सुरक्षित मार्ग असतो.


कोर्स हेल्पर डेमो साठी, एक उपकरण असू शकते:

- `search_lessons(query)`
- `read_lesson(path)`
- `create_practice_task(topic)`

### RAG आणि ज्ञान

RAG एजंटला अंदाज न लावता स्रोत सामग्रीमधून उत्तर देण्यास मदत करते. या
कोर्समध्ये, त्या स्रोत सामग्रीमध्ये धडा READMEs, कोड नमुने, किंवा धड्यांमधून लिंक केलेले बाह्य
संसाधने असू शकतात.

जेव्हा उत्तर कागदपत्रे, डेटा, किंवा चालू
प्रोजेक्ट फाइल्सवर आधारित असावे तेव्हा RAG वापरा.

### नियोजन

विनंतीमध्ये दोन किंवा अधिक टप्पे असतील तेव्हा नियोजन उपयुक्त असते. नियोजन लहान
आणि विकासक किंवा वापरकर्त्याला तपासण्यास पर्याप्त दिसणारे ठेवा.

डेमो साठी, एक योजना असू शकते:

1. उपकरण वापराशी संबंधित धडे शोधा.
2. सर्वात संबंधित धडे सारांश करा.
3. एक सराव काम सुचवा.

### संदर्भ

संदर्भ हा मॉडेल सध्या काय पाहतो ते आहे. फार कमी संदर्भामुळे एजंट महत्त्वपूर्ण तपशील
चुकवू शकतो. खूप जास्त संदर्भामुळे एजंट स्लो, अधिक खर्चिक,
किंवा गोंधळण्यास सोपा होऊ शकतो.

चांगले संदर्भ अभियांत्रण म्हणजे पुढील मॉडेल कॉलसाठी योग्य माहिती निवडणे.


### मेमरी

मेमरी म्हणजे नंतर वापरण्यासाठी साठवलेली माहिती. सर्व काही जपू नका. माहिती फक्त तेव्हा जपा
जेव्हा ती उपयुक्त, सुरक्षित आणि अपडेट किंवा डिलीट करणे सोपे असेल.

उदाहरणार्थ, "शिकणारा Python उदाहरणे पसंत करतो" हे लक्षात ठेवणे उपयुक्त असू शकते.
संवेदनशील वैयक्तिक डेटा लक्षात ठेवणे सहसा नाही.

### मूल्यांकन आणि निरीक्षणक्षमता

मूल्यांकन विचारते: एजंटने योग्य काम केले का?

निरीक्षणक्षमता विचारते: आपण हे कसे घडले ते पाहू शकतो का?

उत्पादन एजंटसाठी, मॉडेल कॉल्स, उपकरण कॉल्स, प्राप्त संदर्भ,
विलंब, खर्च, अपयश, आणि वापरकर्ता अभिप्राय यांचे निवारण ठेवा.

### विश्वास आणि सुरक्षा

विश्वासार्ह एजंटसाठी फक्त सहायक प्रॉम्प्ट पुरेसा नाही. कमी-प्राधान्य उपकरणे वापरा,
उच्च परिणामकारक क्रियांसाठी मानवी मान्यता घ्या, आवश्यकतेनुसार डेटा छुपवा, आणि
ऑडिट करण्यासाठी क्रियांचा लॉग किंवा पावती ठेवा.

## १५-मिनिटांचे पुनरावलोकन दिनचर्या

प्रत्येक धड्यानंतर ही दिनचर्या वापरा:

1. **धड्याचा एक वाक्यात सारांश करा.**
2. **नवीन एजंट क्षमता नाव द्या.** उदाहरणार्थ: उपकरण वापर, पुनर्प्राप्ती,
   नियोजन, मेमरी, निरीक्षणक्षमता, किंवा सुरक्षा.
3. **ते कोर्स हेल्पर डेमोत जोडा.** डेमोत आत्ता काय बदल झाले आहे?
4. **जोखीम शोधा.** ही क्षमता चुकीच्या प्रकारे वापरली गेल्यास काय चुकू शकते?
5. **एक चाचणी प्रश्न लिहा.** एजंट नीट वागत आहे की नाही हे कसे तपासाल?

## झटपट स्व-तपासणी

पुढे जाण्यापूर्वी, हे प्रश्न उत्तर देण्याचा प्रयत्न करा:

1. कोणते काम एजंट स्वतःहून नियमित चाटबॉट करू शकत नाही?
2. तुमच्या एजंटला प्रथम कोणते उपकरण आवश्यक आहे आणि का?
3. एजंटच्या उत्तरासाठी कोणता ज्ञान स्रोत आधार असावा?
4. पुढील मॉडेल कॉलमध्ये कोणता संदर्भ असावा?

5. एजंटने काय लक्षात ठेवले पाहिजे, आणि काय साठविणे टाळले पाहिजे?
6. एजंटने मानव मंजुरी कधी विचारावी?
7. एजंटची नंतर डिबग किंवा ऑडिट करण्यात मदत करणारे कोणते लॉग, ट्रेस किंवा पावती असाव्यात?


## सूचवलेले कॅपस्टोन व्यायाम

कोर्सच्या शेवटी, एक लहान एजंट तयार करा जो शिकणाऱ्यांना या
रेपॉझिटरीमध्ये नेव्हिगेट करण्यात मदत करेल.

कमीतकमी आवृत्ती:

- वापरकर्त्याकडून एक विषय स्वीकारा.
- सर्वात संबंधित धडे शोधा.
- प्रथम काय वाचायचे ते सारांश द्या.
- एक प्रत्यक्ष प्रॅक्टिस कार्य सुचवा.
- कोणती धडा फायली किंवा लिंक वापरल्या गेल्या ते दाखवा.

विस्तार आवृत्ती:

- शिकणाऱ्याची पसंतीची प्रोग्रामिंग भाषा लक्षात ठेवा.
- उत्तर देण्यापूर्वी सोपा योजना वापरा.
- अंतिम उत्तराच्या आधी एक स्व-तपासणी टप्पा जोडा.
- साधन कॉल्स आणि मिळवलेल्या स्रोतांची नोंद ठेवा.
- ब्राउझर किंवा UI स्वयंचलित कार्ये सुरू करण्यापूर्वी पुष्टी मागा.

यामुळे तुम्हाला एक लहान पण वास्तववादी मार्ग मिळतो जेथे तुम्ही साधने, RAG, योजना,
संदर्भ, स्मृती, निरीक्षणक्षमता, आणि विश्वास यांचा एक प्रकल्पात सराव करू शकता.
