# 🔄 使用 Azure OpenAI (Responses API) 的基本代理工作流程 (.NET)

## 📋 工作流程編排教程

本筆記本示範如何使用 Microsoft Agent Framework for .NET 及 Azure OpenAI (Responses API) 建立複雜的 <strong>代理工作流程</strong>。您將學習如何創建多步驟的商業流程，在該流程中 AI 代理通過結構化的編排模式協同完成複雜任務。

```xml
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI 設定**

**環境設定 (.env 檔案)：**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Azure OpenAI 存取：**
1. 在 Azure 入口網站建立 Azure OpenAI 資源
2. 部署一個模型（例如 `gpt-5-mini`），並記下部署名稱
3. 使用 `az login` 登入並依上述方式設定環境變數

## 💻 執行程式碼

完整實作位於 `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`。此檔展示：

1. <strong>環境配置</strong> - 從 `.env` 檔案載入 Azure OpenAI 配置
2. **Azure OpenAI Client 設定** - 配置客戶端使用 Azure OpenAI Responses API
3. <strong>代理建立</strong> - 定義專業代理（前台和禮賓）
4. **Workflow Builder** - 創建多代理的逐步處理工作流程
5. <strong>工作流程執行</strong> - 執行帶有串流結果的工作流程

### 🚀 執行範例

```bash
# 令此腳本可執行（Unix/Linux/macOS）

# 執行工作流程
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```
