# ⏩ ایزور اوپن اے آئی (Responses API) کے ساتھ تسلسل ایجنٹ ورک فلو (.NET)

## 📋 ایڈوانس تسلسل پراسیسنگ ٹیوٹوریل

یہ نوٹ بک مائیکروسافٹ ایجنٹ فریم ورک برائے .NET اور ایزور اوپن اے آئی (Responses API) کا استعمال کرتے ہوئے **تسلسلی ورک فلو پیٹرنز** کی نمائندگی کرتی ہے۔ آپ سیکھیں گے کہ کس طرح پیچیدہ، مرحلہ وار پراسیسنگ پائپ لائنز بنائی جاتی ہیں جہاں ایجنٹس مخصوص ترتیب میں عمل انجام دیتے ہیں، ہر مرحلہ پچھلے مرحلے کے نتائج پر مبنی ہوتا ہے۔

## 🎯 سیکھنے کے مقاصد

### 🔄 **تسلسل پراسیسنگ فن تعمیر**
- **لینیئر ورک فلو ڈیزائن**: واضح انحصار کے ساتھ مرحلہ وار پراسیسنگ پائپ لائنز بنائیں
- **اسٹیٹ مینجمنٹ**: تسلسل ورک فلو مراحل میں کنٹیکسٹ اور ڈیٹا فلو برقرار رکھیں
- **ایزور اوپن اے آئی (Responses API)**: .NET ملٹی-اسٹیج ورک فلو میں ایزور اوپن اے آئی ماڈلز کا فائدہ اٹھائیں
- **انٹرپرائز پائپ لائن پیٹرنز**: پروڈکشن ریڈی تسلسل پراسیسنگ سسٹمز بنائیں

### 🏗️ **ایڈوانس تسلسل پیٹرنز**
- **اسٹیج-گیٹ پراسیسنگ**: ورک فلو مراحل کے درمیان ویلیڈیشن چیک پوائنٹس نافذ کریں
- **کنٹیکسٹ پریزرویشن**: تمام مراحل میں اسٹیٹ اور جمع شدہ علم برقرار رکھیں
- **خرابی کی منتقلی**: تسلسلی پراسیسنگ چینز میں ناکامیوں کو بخوبی سنبھالیں
- **کارکردگی کی اصلاح**: کم از کم اضافی بوجھ کے ساتھ موثر تسلسلی عمل

### 🏢 **انٹرپرائز تسلسل ایپلیکیشنز**
- **دستاویز پراسیسنگ پائپ لائن**: کثیر مرحلہ دستاویز تجزیہ، تبدیلی، اور ویلیڈیشن
- **کوالیٹی ایشورنس ورک فلو**: تسلسلی جائزہ، ویلیڈیشن، اور منظوری کے عمل
- **مواد پیداوار پائپ لائن**: تحقیق → تحریر → تدوین → جائزہ → اشاعت
- **کاروباری پراسیس آٹومیشن**: واضح مرحلہ انحصار کے ساتھ کثیر مرحلہ کاروباری ورک فلو

## ⚙️ ضروریات اور سیٹ اپ

### 📦 **ضروری NuGet پیکیجز**

.NET تسلسلی ورک فلو کے لیے ضروری پیکیجز:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **ایزور اوپن اے آئی کنفیگریشن**

**ماحول کا سیٹ اپ (.env فائل):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**کنفیگریشن مینجمنٹ:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **تسلسلی ورک فلو فن تعمیر**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: تجزیہ ایجنٹ]
    B --> C[چیک پوائنٹ 1]
    C --> D[Stage 2: پروسیسنگ ایجنٹ]
    D --> E[چیک پوائنٹ 2]
    E --> F[Stage 3: توثیقی ایجنٹ]
    F --> G[حتمی نتیجہ]
    
    H[ریاست کا سیاق و سباق] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (جوابات API)] --> B
    I --> D
    I --> F
```

**اہم اجزاء:**
- **تسلسلی ایجنٹس**: ہر پراسیسنگ مرحلے کے لیے مخصوص ایجنٹس
- **اسٹیٹ کنٹیکسٹ**: مراحل کے دوران جمع شدہ ڈیٹا اور فیصلے برقرار رکھتا ہے
- **چیک پوائنٹس**: مراحل کے درمیان معیار اور مستقل مزاجی کی تصدیقی نقاط
- **ایزور اوپن اے آئی کلائنٹ**: تمام ورک فلو مراحل میں مستقل AI ماڈل تک رسائی

## 🎨 **تسلسل ورک فلو ڈیزائن پیٹرنز**

### 📝 **دستاویز پراسیسنگ پائپ لائن**
```
Raw Document → Content Extraction → Analysis → Validation → Structured Output
```

### 🎯 **مواد تخلیق ورک فلو**
```
Brief/Requirements → Research → Content Creation → Review → Final Polish
```

### 🔍 **کوالیٹی ایشورنس پائپ لائن**
```
Initial Review → Technical Validation → Compliance Check → Final Approval
```

### 💼 **کاروباری انٹیلیجنس ورک فلو**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

## 🏢 **انٹرپرائز تسلسل فوائد**

### 🎯 **قابلیت اور معیار**
- **متعین شدہ پراسیسنگ**: منظم مراحل کے ذریعہ مستقل، دہرانے والے نتائج
- **کوالیٹی گیٹس**: ہر مرحلے پر معیار کی تصدیقی چیک پوائنٹس
- **خرابی علیحدگی**: ایک مرحلے کی پریشانیاں اگلے مراحل تک منتقل نہیں ہوتیں
- **آڈٹ ٹریلز**: ہر مرحلے میں فیصلوں اور تبدیلیوں کا مکمل ریکارڈ

### 📈 **اسکیل ایبلٹی اور کارکردگی**
- **ماڈیولر ڈیزائن**: ہر مرحلہ آزادانہ طور پر بہتر کیا جا سکتا ہے
- **وسائل مینجمنٹ**: مراحل میں AI ماڈل وسائل کی موثر تخصیص
- **اسٹیٹ آپٹیمائزیشن**: انتہائی کارکردگی کے لئے مراحل کے درمیان کم سے کم اسٹیٹ ٹرانسفر
- **متوازی مرحلہ گروپس**: متعدد تسلسلی ورک فلو متوازی طور پر چل سکتے ہیں

### 🔒 **سلامتی اور تعمیل**
- **مرحلہ سطح کی سلامتی**: مختلف پراسیسنگ مراحل کے لیے مختلف سلامتی پالیسیاں
- **ڈیٹا ویلیڈیشن**: ہر چیک پوائنٹ پر ڈیٹا کی سالمیت اور تعمیل یقینی بنائیں
- **رسائی کنٹرول**: مختلف ورک فلو مراحل کے لیے باریک بین اجازتیں
- **ریگولیٹری تعمیل**: منظم پراسیسنگ کے ذریعہ ریگولیٹری تقاضے پورے کریں

### 📊 **مانیٹرنگ اور تجزیات**
- **مرحلہ سطح میٹرکس**: ہر ورک فلو مرحلے کی کارکردگی کی نگرانی
- **بہتراکی نشاندہی**: سست مراحل کی شناخت اور اصلاح
- **معیار کے میٹرکس**: ہر مرحلے پر معیار اور کامیابی کی شرح کی نگرانی
- **عمل کی بہتری**: مرحلہ سطح کے تجزیات کی بنیاد پر مستقل بہتری

آئیں مضبوط تسلسلی AI پراسیسنگ پائپ لائنز بنائیں! 🚀

## 💻 کوڈ چلانا

مکمل عمل درآمد `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs` میں دستیاب ہے۔ یہ فائل **تین مرحلہ فرنیچر تجزیہ ورک فلو** دکھاتی ہے:

1. **مرحلہ 1 - سیلز ایجنٹ**: فرنیچر کی تصاویر کا تجزیہ کرتا ہے اور خریداری کی تجاویز دیتا ہے
2. **مرحلہ 2 - پرائس ایجنٹ**: تفصیلی قیمتوں کا تجزیہ اور بجٹ آپشنز فراہم کرتا ہے
3. **مرحلہ 3 - کوٹ ایجنٹ**: مارکی ڈاؤن فارمیٹ میں پیشہ ورانہ کوٹ دستاویز تیار کرتا ہے

### 🏗️ **ورک فلو فن تعمیر**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

ہر ایجنٹ:
- پچھلے مرحلے سے آؤٹ پٹ بطور کنٹیکسٹ وصول کرتا ہے
- خاص مہارت کے ساتھ پچھلے تجزیے کی بنیاد پر تعمیر کرتا ہے
- اسٹیٹ مینجمنٹ کے ذریعہ ورک فلو تسلسل برقرار رکھتا ہے

### 📝 متوقع آؤٹ پٹ

ورک فلو یہ کرے گا:
1. **سیلز ایجنٹ**: تصویر سے فرنیچر کی اشیاء شناخت کریں اور سفارشات فراہم کریں
2. **پرائس ایجنٹ**: بجٹ کی سطحوں اور خریداری کی تجاویز کے ساتھ تفصیلی قیمتوں کا تجزیہ شامل کریں
3. **کوٹ ایجنٹ**: تمام معلومات کو یکجا کرتے ہوئے منظم کوٹ دستاویز تیار کریں
