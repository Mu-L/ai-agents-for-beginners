# 🔄 ایزور اوپن اے آئی کے ساتھ بنیادی ایجنٹ ورک فلو (Responses API) (.NET)

## 📋 ورک فلو آرکیسٹریشن ٹیوٹوریل

یہ نوٹ بک دکھاتی ہے کہ مائیکروسافٹ ایجنٹ فریم ورک برائے .NET اور ایزور اوپن اے آئی (Responses API) کا استعمال کرتے ہوئے پیچیدہ **ایجنٹ ورک فلو** کیسے بنائیں۔ آپ سیکھیں گے کہ کس طرح کثیر مرحلہ کاروباری عمل بنائیں جہاں AI ایجنٹ منظم آرکیسٹریشن پیٹرنز کے ذریعے پیچیدہ کاموں کو پورا کرنے کے لیے تعاون کرتے ہیں۔

## 🎯 سیکھنے کے مقاصد

### 🏗️ **ورک فلو کی فن تعمیر کے بنیادی اصول**
- **ورک فلو بلڈر**: پیچیدہ کثیر مرحلہ AI عمل کی ڈیزائننگ اور آرکیسٹریشن
- **ایجنٹ کوآرڈینیشن**: ورک فلو کے اندر متعدد ماہر ایجنٹس کو ہم آہنگ کرنا
- **ایزور اوپن اے آئی (Responses API)**: ورک فلو میں ایزور اوپن اے آئی ریسپانسز API کا استعمال
- **بصری ورک فلو ڈیزائن**: بہتر فہم کے لیے ورک فلو ڈھانچے بنائیں اور دیکھیں

### 🔄 **عمل آرکیسٹریشن پیٹرنز**
- **تسلسل پر مبنی عمل کاری**: متعدد ایجنٹ کاموں کو منطقی ترتیب میں جوڑیں
- **حالت کا انتظام**: ورک فلو کے مراحل کے درمیان سیاق و سباق اور ڈیٹا کے بہاؤ کو برقرار رکھیں
- **خرابی سنبھالنا**: مضبوط خرابی کی بحالی اور ورک فلو کی مضبوطی کو نافذ کریں
- **کارکردگی کی اصلاح**: کاروباری سطح کے آپریشنز کے لیے مؤثر ورک فلو ڈیزائن کریں

### 🏢 **کاروباری ورک فلو ایپلیکیشنز**
- **کاروباری عمل کی خودکار کاری**: پیچیدہ تنظیمی ورک فلو کو خودکار بنائیں
- **مواد کی پیداوار کی پائپ لائن**: جائزہ اور منظوری کے مراحل کے ساتھ ایڈیٹوریل ورک فلو
- **کسٹمر سروس کی خودکار کاری**: کثیر مرحلہ صارف کی استفسار کا حل
- **ڈیٹا پروسیسنگ ورک فلو**: AI سے چلنے والی تبدیلی کے ساتھ ETL ورک فلو

## ⚙️ ضروریات اور ترتیب

### 📦 **ضروری NuGet پیکجز**

اس ورک فلو ڈیمو میں کئی اہم .NET پیکجز استعمال ہوتے ہیں:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **ایزور اوپن اے آئی کی ترتیب**

**ماحول کی ترتیب (.env فائل):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**ایزور اوپن اے آئی تک رسائی:**
1. ایزور پورٹل میں ایک ایزور اوپن اے آئی ریسورس بنائیں
2. ایک ماڈل (مثلاً `gpt-5-mini`) تعینات کریں اور تعیناتی کا نام نوٹ کریں
3. `az login` سے سائن ان کریں اور ماحول متغیرات کو اوپر دکھائے گئے مطابق ترتیب دیں

### 🏗️ **ورک فلو فن تعمیر کا جائزہ**

```mermaid
graph TD
    A[ورک فلو بلڈر] --> B[ایجنٹ رجسٹری]
    B --> C[ورک فلو ایگزیکیوشن انجن]
    C --> D[Agent 1: مواد بنانے والا
    C --> E[Agent 2: مواد کا جائزہ لینے والا 
    D --> F[ورک فلو نتائج]
    E --> F
    G[ایزور اوپن اے آئی (رسپانسز API)] --> D
    G --> E
```

**اہم اجزاء:**
- **WorkflowBuilder**: ورک فلو ڈیزائن کرنے کے لیے مرکزی آرکیسٹریشن انجن
- **AIAgent**: مخصوص صلاحیتوں کے حامل انفرادی ماہر ایجنٹس
- **ایزور اوپن اے آئی کلائنٹ**: ایزور اوپن اے آئی Responses API کی انٹیگریشن
- **ایگزیکیوشن کانٹیکسٹ**: ورک فلو مراحل کے درمیان حالت اور ڈیٹا کے بہاؤ کا انتظام کرتا ہے

## 🎨 **کاروباری ورک فلو ڈیزائن پیٹرنز**

### 📝 **مواد کی پیداوار کا ورک فلو**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **دستاویزات پروسیسنگ پائپ لائن**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **کاروباری انٹیلی جنس ورک فلو**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **کسٹمر سروس کی خودکار کاری**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **کاروباری فوائد**

### 🎯 **قابل اعتماد اور پیمانہ پذیری**
- **متعین کردہ ایگزیکیوشن**: مستقل، دہرائے جانے والے ورک فلو نتائج
- **خرابی کی بحالی**: ورک فلو کے کسی بھی مرحلے پر خرابیوں کا شائستہ سنبھالنا
- **کارکردگی کی نگرانی**: ایگزیکیوشن میٹرکس اور اصلاح کے مواقع کو مانیٹر کرنا
- **وسائل کا انتظام**: AI ماڈل وسائل کی مؤثر تقسیم اور استعمال

### 🔒 **سیکیورٹی اور تعمیل**
- **محفوظ توثیق**: `az login` (AzureCliCredential) کے ذریعے Microsoft Entra ID توثیق
- **آڈٹ ٹریلز**: ورک فلو ایگزیکیوشن اور فیصلے کے پوائنٹس کا مکمل لاگنگ
- **رسائی کنٹرول**: ورک فلو ایگزیکیوشن اور نگرانی کے لیے مخصوص اجازتیں
- **ڈیٹا کی رازداری**: ورک فلو کے دوران حساس معلومات کا محفوظ انتظام

### 📊 **مشاہدہ اور انتظام**
- **بصری ورک فلو ڈیزائن**: عمل کے بہاؤ اور انحصار کی واضح نمائندگی
- **ایگزیکیوشن کی نگرانی**: ورک فلو کی پیش رفت اور کارکردگی کی حقیقی وقت میں نگرانی
- **خرابی کی رپورٹنگ**: تفصیلی خرابی تجزیہ اور ڈیبگنگ صلاحیتیں
- **کارکردگی کا تجزیہ**: اصلاح اور صلاحیت کی منصوبہ بندی کے لیے میٹرکس

آئیں آپ کا پہلا کاروباری تیار AI ورک فلو بنائیں! 🚀

## 💻 کوڈ چلانا

مکمل عمل درآمد `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` میں دستیاب ہے۔ یہ فائل دکھاتی ہے:

1. **ماحولیاتی ترتیب** - `.env` فائل سے ایزور اوپن اے آئی کی ترتیب لوڈ کرنا
2. **ایزور اوپن اے آئی کلائنٹ کی ترتیب** - کلائنٹ کو ایزور اوپن اے آئی Responses API کے استعمال کے لیے ترتیب دینا
3. **ایجنٹ کی تخلیق** - مخصوص ایجنٹس (فرنٹ ڈیسک اور کانسیئرژ) کی تعریف
4. **ورک فلو بلڈر** - تسلسل پر مبنی پروسیسنگ کے ساتھ کثیر ایجنٹ ورک فلو بنانا
5. **ورک فلو ایگزیکیوشن** - اسٹریمنگ نتائج کے ساتھ ورک فلو چلانا

### 🚀 مثال چلانا

```bash
# اسکرپٹ کو قابل عمل بنائیں (یونکس/لینکس/میک او ایس)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# ورک فلو چلائیں
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

یا ونڈوز پر:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

آپ ورک فلو کو تخصیص کر سکتے ہیں:
- ایجنٹ کی ہدایات میں ترمیم کرکے ان کے رویے کو تبدیل کرنا
