# 🔄 بنیادی ایجنٹ ورک فلو Azure OpenAI (Responses API) کے ساتھ (.NET)

## 📋 ورک فلو رچنا کا سبق

یہ نوٹ بک دکھاتی ہے کہ کیسے مائیکروسافٹ ایجنٹ فریم ورک برائے .NET اور Azure OpenAI (Responses API) استعمال کرتے ہوئے پیچیدہ **ایجنٹ ورک فلو** تعمیر کیے جائیں۔ آپ سیکھیں گے کہ کس طرح کثیر مرحلہ کاروباری عمل بنائیں جہاں AI ایجنٹ مل کر منظم رچنا کے نمونوں کے ذریعے پیچیدہ کام انجام دیتے ہیں۔

## 🎯 سیکھنے کے مقاصد

### 🏗️ **ورک فلو آرکیٹیکچر کے بنیادی اصول**
- **ورک فلو بلڈر**: پیچیدہ کثیر مرحلہ AI عمل کی ڈیزائننگ اور رچنا
- **ایجنٹ کوآرڈینیشن**: ورک فلو میں متعدد ماہر ایجنٹوں کا کوآرڈینیٹ کرنا
- **Azure OpenAI (Responses API)**: ورک فلو میں Azure OpenAI Responses API کا استعمال
- **بصری ورک فلو ڈیزائن**: بہتر تفہیم کے لیے ورک فلو کے ڈھانچے کی تخلیق اور بصری نمائندگی

### 🔄 **عمل رچنا کے نمونے**
- **تسلسل میں عمل کاری**: متعدد ایجنٹ کاموں کو منطقی ترتیب میں جوڑنا
- **حالت کا انتظام**: ورک فلو کے مراحل میں سیاق و سباق اور ڈیٹا کا بہاؤ برقرار رکھنا
- **خرابی کا انتظام**: مضبوط خرابی بازیابی اور ورک فلو کی لچک کا نفاذ
- **کارکردگی کی بہتری**: انٹرپرائز سکیل آپریشنز کے لیے مؤثر ورک فلو ڈیزائن کرنا

### 🏢 **انٹرپرائز ورک فلو ایپلیکیشنز**
- **کاروباری عمل کی خودکار کاری**: پیچیدہ تنظیمی ورک فلو کی خودکار کاری
- **مواد کی پیداوار کی لائن**: جانچ اور منظوری کے مراحل کے ساتھ اداریہ ورک فلو
- **کسٹمر سروس کی خودکار کاری**: کثیر مرحلہ صارف استفسار کا حل
- **ڈیٹا پراسیسنگ ورک فلو**: AI سے چلنے والے ETL ورک فلو

## ⚙️ پیشگی ضروریات اور سیٹ اپ

### 📦 **ضروری NuGet پیکجز**

یہ ورک فلو ڈیمو کئی اہم .NET پیکجز استعمال کرتا ہے:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI کی ترتیب**

**ماحول کی ترتیب (.env فائل):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI تک رسائی:**
1. Azure پورٹل میں Azure OpenAI ریسورس بنائیں
2. ایک ماڈل ڈیپلائے کریں (مثلاً، `gpt-4.1-mini`) اور ڈیپلائیمنٹ کا نام نوٹ کریں
3. `az login` کے ساتھ سائن ان کریں اور اوپر دکھائے گئے طریقے سے ماحول کے متغیرات ترتیب دیں

### 🏗️ **ورک فلو آرکیٹیکچر کا جائزہ**

```mermaid
graph TD
    A[ورک فلو بلڈر] --> B[ایجنٹ رجسٹری]
    B --> C[ورک فلو ایگزیکیوشن انجن]
    C --> D[Agent 1: مواد جنریٹر]
    C --> E[Agent 2: مواد جائزہ کار] 
    D --> F[ورک فلو کے نتائج]
    E --> F
    G[Azure OpenAI (ردعمل API)] --> D
    G --> E
```

**اہم اجزاء:**
- **WorkflowBuilder**: ورک فلو ڈیزائن کرنے کے لیے مرکزی رچنا انجن
- **AIAgent**: مخصوص صلاحیتوں والے انفرادی ماہر ایجنٹ
- **Azure OpenAI Client**: Azure OpenAI Responses API انٹیگریشن
- **Execution Context**: ورک فلو کے مراحل کے درمیان حالت اور ڈیٹا کے بہاؤ کا انتظام

## 🎨 **انٹرپرائز ورک فلو ڈیزائن نمونے**

### 📝 **مواد کی پیداوار کا ورک فلو**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **دستاویز پراسیسنگ لائن**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **کاروباری انٹیلی جنس ورک فلو**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **کسٹمر سروس کی خودکار کاری**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **انٹرپرائز کے فوائد**

### 🎯 **بھروسہ مندی اور توسیع پذیری**
- **متعین عمل کاری**: مستقل، دہرائے جانے والے ورک فلو نتائج
- **خرابی سے بازیابی**: کسی بھی ورک فلو مرحلے میں خرابیاں ہنس کر سنبھالنا
- **کارکردگی کی نگرانی**: عمل کاری کے میٹرکس اور بہتری کے مواقع کی نگرانی
- **وسائل کا انتظام**: AI ماڈل وسائل کی مؤثر تقسیم اور استعمال

### 🔒 **سلامتی اور تعمیل**
- **مضبوط توثیق**: `az login` (AzureCliCredential) کے ذریعے Microsoft Entra ID کی توثیق
- **آڈٹ ٹریل**: ورک فلو عمل اور فیصلہ سازی کے مکمل لاگز
- **رسائی کا کنٹرول**: ورک فلو عمل اور نگرانی کے لیے مفصل اجازت نامے
- **ڈیٹا کی پرائیویسی**: ورک فلو کے دوران حساس معلومات کی محفوظ ہینڈلنگ

### 📊 **مشاہدہ اور انتظام**
- **بصری ورک فلو ڈیزائن**: عمل کے بہاؤ اور انحصار کی واضح نمائندگی
- **عمل کاری کی نگرانی**: ورک فلو کے پیش رفت اور کارکردگی کی حقیقی وقت میں ٹریکنگ
- **خرابی کی رپورٹنگ**: تفصیلی خرابی کے تجزیے اور ڈیبگ کرنے کی صلاحیتیں
- **کارکردگی کا تجزیہ**: بہتری اور گنجائش کی منصوبہ بندی کے میٹرکس

## 💻 کوڈ چلانا

مکمل عمل درآمد `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` میں دستیاب ہے۔ یہ فائل درج ذیل دکھاتی ہے:

1. **ماحول کی ترتیب** - `.env` فائل سے Azure OpenAI کی ترتیب لوڈ کرنا
2. **Azure OpenAI کلائنٹ سیٹ اپ** - Azure OpenAI Responses API استعمال کرنے کے لیے کلائنٹ کی ترتیب
3. **ایجنٹ کی تخلیق** - ماہر ایجنٹس کی تعریف (فرنٹ ڈیسک اور کنسیئرج)
4. **ورک فلو بلڈر** - کثیر ایجنٹ ورک فلو بنانا جس میں متسلسل عمل کاری ہو
5. **ورک فلو کا نفاذ** - اسٹریمینگ نتائج کے ساتھ ورک فلو چلانا

### 🚀 مثال چلانا

```bash
# اسکرپٹ کو قابل اجرا بنائیں (یونکس/لینکس/میک او ایس)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# ورک فلو چلائیں
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

یا ونڈوز پر:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```
