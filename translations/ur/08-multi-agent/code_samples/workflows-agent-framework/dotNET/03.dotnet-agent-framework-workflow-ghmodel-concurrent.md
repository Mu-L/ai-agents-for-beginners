# ⚡ Azure OpenAI (Responses API) کے ساتھ متوازی ایجنٹ ورک فلو (.NET)

## 📋 اعلیٰ کارکردگی والی متوازی پراسیسنگ ٹیوٹوریل

یہ نوٹ بک Microsoft Agent Framework for .NET اور Azure OpenAI (Responses API) استعمال کرتے ہوئے **متوازی ورک فلو پیٹرنز** کی مثال پیش کرتی ہے۔ آپ سیکھیں گے کہ کس طرح اعلیٰ کارکردگی، متوازی پراسیسنگ ورک فلو بنائیں جو متعدد AI ایجنٹس کو بیک وقت چلانے کے ذریعے throughput کو زیادہ سے زیادہ کرتے ہیں، جبکہ کوآرڈینیشن اور ڈیٹا کی مطابقت کو برقرار رکھتے ہیں۔

## 🎯 سیکھنے کے مقاصد

### 🚀 **متوازی پراسیسنگ کی بنیادیں**
- **متوازی ایجنٹ کی انجام دہی**: زیادہ سے زیادہ کارکردگی کے لیے متعدد AI ایجنٹس کو بیک وقت چلائیں
- **Async/Await پیٹرنز**: مؤثر متوازی پراسیسنگ کے لیے .NET کے async پروگرامنگ ماڈل کا استعمال کریں
- **Azure OpenAI (Responses API)**: Azure OpenAI Responses API کو متعدد متوازی کالز سے ہم آہنگ کریں
- **وسائل کا انتظام**: متوازی آپریشنز کے دوران AI ماڈل کے وسائل کو مؤثر طریقے سے سنبھالیں

### 🏗️ **جدید متوازی فن تعمیر**
- **ٹاسک پر مبنی متوازی عمل**: بہترین متوازی انجام دہی کے لیے .NET Task Parallel Library کا استعمال کریں
- **ہم آہنگی پیٹرنز**: ریس کنڈیشنز سے بچتے ہوئے متعدد ایجنٹس کو کوآرڈینیٹ کریں
- **لوڈ بیلنسنگ**: دستیاب متوازی پراسیسنگ کی صلاحیت کے درمیان کام کو مؤثر طریقے سے تقسیم کریں
- **فالٹ ٹولرنس**: پوری ورک فلو کو روکے بغیر انفرادی ایجنٹ کی ناکامیوں کو سنبھالیں

### 🏢 **انٹرپرائز متوازی ایپلیکیشنز**
- **اعلیٰ حجم دستاویز پراسیسنگ**: ایک ساتھ متعدد دستاویزات کو پراسیس کریں
- **ریئل ٹائم مواد کا تجزیہ**: آنے والے ڈیٹا اسٹریمز کا متوازی تجزیہ
- **بیچ پراسیسنگ کی اصلاح**: بڑے پیمانے پر ڈیٹا پراسیسنگ آپریشنز کے لیے throughput کو زیادہ سے زیادہ کریں
- **کئی ماڈلز کا تجزیہ**: مختلف مواد کی اقسام اور فارمیٹس کی متوازی پراسیسنگ

## 💻 کوڈ چلانا

مکمل عملدرآمد فائل `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` میں دستیاب ہے۔ یہ فائل سفر کی منصوبہ بندی کے لیے **Fan-Out/Fan-In متوازی ورک فلو** کی مثال دیتی ہے:

### 🏗️ **ورک فلو فن تعمیر**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**اہم اجزاء:**

1. **ConcurrentStartExecutor**: صارف کی درخواست کو تمام ایجنٹس کو بیک وقت نشر کرتا ہے
2. **Researcher Agent**: متوازی طور پر مقامات اور سیاحتی مقامات کا تجزیہ کرتا ہے
3. **Planner Agent**: متوازی طور پر تفصیلی سفر کے منصوبے بناتا ہے
4. **ConcurrentAggregationExecutor**: دونوں ایجنٹس کے نتائج کو جمع اور مرج کرتا ہے

### 🎯 **Fan-Out/Fan-In پیٹرن**

یہ ورک فلو کلاسک **Fan-Out/Fan-In** پیٹرن کی مثال ہے:
- **Fan-Out**: ایک ان پٹ پیغام متعدد ایجنٹس کو بیک وقت بھیجا جاتا ہے
- **متوازی پراسیسنگ**: متعدد ایجنٹس ایک ہی کام پر بیک وقت کام کرتے ہیں
- **Fan-In**: تمام ایجنٹس کے نتائج کو جمع کر کے ایک مشترکہ آؤٹ پٹ تیار کیا جاتا ہے

### 📝 متوقع نتیجہ

ورک فلو یہ کرے گا:
1. **درخواست نشر کرنا**: "ڈیسمبر میں سیئٹل کا سفر منصوبہ بنائیں" کا پیغام دونوں ایجنٹس کو بھیجیں
2. **متوازی پراسیسنگ**: دونوں ایجنٹس بیک وقت کام کرتے ہیں:
   - محقق سیاحتی مقامات اور تفصیلات شناخت کرتا ہے
   - منصوبہ ساز سفر کا روٹ اور لوجسٹکس تیار کرتا ہے
3. **نتائج کا اشتراک**: دونوں جوابات کو جامع آؤٹ پٹ میں جوڑیں
4. **نتائج کی نمائش**: تمام معلومات کے ساتھ یکجا شدہ سفر کا منصوبہ دکھائیں

### 🔍 کسٹم ایگزیکیوٹرز کو سمجھنا

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` کو نافذ کرتا ہے تاکہ سٹرنگ ان پٹ قبول کی جا سکے
- تمام متصل ایجنٹس کو پیغامات نشر کرتا ہے
- `TurnToken` بھیجتا ہے تاکہ متوازی پراسیسنگ شروع ہو

**ConcurrentAggregationExecutor:**
- `IMessageHandler<ChatMessage>` کو نافذ کرتا ہے تاکہ ایجنٹ کے جوابات وصول کرے
- تھریڈ سیف طریقے سے پیغامات جمع کرتا ہے
- جب تمام متوقع جوابات آ جائیں تو نتائج کو اشتراک کرتا ہے
- `context.YieldOutputAsync()` کا استعمال کرتے ہوئے حتمی آؤٹ پٹ دیتا ہے
