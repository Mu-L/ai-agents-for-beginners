# ⚡ ایژور اوپن اے آئی (Responses API) کے ساتھ متوازی اجنٹ ورک فلو (.NET)

## 📋 ہائی پرفارمنس متوازی پراسیسنگ ٹیوٹوریل

یہ نوٹ بک مائیکروسافٹ ایجنٹ فریم ورک برائے .NET اور ایژور اوپن اے آئی (Responses API) استعمال کرتے ہوئے **متوازی ورک فلو پیٹرنز** کو ظاہر کرتی ہے۔ آپ سیکھیں گے کہ کیسے ہائی پرفارمنس، متوازی پراسیسنگ ورک فلو بنایا جائے جو کئی AI ایجنٹس کو بیک وقت چلانے کے ذریعے زیادہ سے زیادہ تھروپٹ کو یقینی بناتا ہے، جبکہ کوآرڈینیشن اور ڈیٹا کی مطابقت بھی برقرار رکھتا ہے۔

## 🎯 سیکھنے کے مقاصد

### 🚀 **متوازی پراسیسنگ کی بنیادی باتیں**
- **متوازی ایجنٹ اجرا**: زیادہ سے زیادہ کارکردگی کے لیے کئی AI ایجنٹس بیک وقت چلائیں
- **Async/Await پیٹرنز**: مؤثر متوازی کے لیے .NET کے async پروگرامنگ ماڈل کا فائدہ اٹھائیں
- **ایژور اوپن اے آئی (Responses API)**: ایژور اوپن اے آئی Responses API کو متوازی کالز میں منسق کریں
- **وسائل کا انتظام**: متوازی آپریشنز میں AI ماڈل کے وسائل کو مؤثر طریقے سے سنبھالیں

### 🏗️ **اعلی درجے کی متوازی فن تعمیر**
- **ٹاسک بیسڈ متوازی پراسیسنگ**: بہترین متوازی اجرا کے لیے .NET ٹاسک پیرلل لائبریری کا استعمال کریں
- **ہم آہنگی کے پیٹرنز**: مسابقہ کی حالتوں سے بچتے ہوئے متوازی ایجنٹس کو منسق کریں
- **لوڈ بیلنسنگ**: دستیاب متوازی پراسیسنگ کی صلاحیت کے حساب سے کام کی مؤثر تقسیم
- **فالٹ ٹولیرنس**: پورا ورک فلو روکے بغیر فرد واحد ایجنٹ کی ناکامیوں کو سنبھالیں

### 🏢 **انٹرپرائز متوازی اطلاقات**
- **اعلی حجم دستاویز پراسیسنگ**: کئی دستاویزات بیک وقت پراسیس کریں
- **ریئل ٹائم مواد کا تجزیہ**: آنے والے ڈیٹا سٹریمز کا متوازی تجزیہ
- **بیچ پراسیسنگ کی اصلاح**: بڑی مقدار میں ڈیٹا پراسیسنگ کے آپریشنز کے لیے تھروپٹ کو زیادہ سے زیادہ کریں
- **کثیر وضعی تجزیہ**: مختلف مواد کی اقسام اور فارمیٹس کی متوازی پراسیسنگ

## 💻 کوڈ چلانا

مکمل نفاذ `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` میں دستیاب ہے۔ یہ فائل سیاحت کی منصوبہ بندی کے لیے **فین آؤٹ/فین ان متوازی ورک فلو** کو ظاہر کرتی ہے:

### 🏗️ **ورک فلو فن تعمیر**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**اہم اجزاء:**

1. **ConcurrentStartExecutor**: یوزر کی درخواست کو تمام ایجنٹس کو بیک وقت نشر کرتا ہے
2. **Researcher Agent**: مقامات اور سیاحتی مقامات کا متوازی تجزیہ کرتا ہے
3. **Planner Agent**: تفصیلی سفری منصوبے متوازی طور پر بناتا ہے
4. **ConcurrentAggregationExecutor**: دونوں ایجنٹس کے نتائج کو جمع اور یکجا کرتا ہے

### 🎯 **فین آؤٹ/فین ان پیٹرن**

یہ ورک فلو کلاسیکی **فین آؤٹ/فین ان** پیٹرن کو ظاہر کرتا ہے:
- **فین آؤٹ**: ایک ان پٹ پیغام بیک وقت کئی ایجنٹس کو نشر کیا جاتا ہے
- **متوازی پراسیسنگ**: کئی ایجنٹس ایک ہی کام پر متوازی کام کرتے ہیں
- **فین ان**: تمام ایجنٹس کے نتائج جمع کر کے ایک واحد آؤٹ پٹ میں یکجا کیے جاتے ہیں

### 🔍 کاسٹم ایکزیکیورز کو سمجھنا

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` کو نافذ کرتا ہے تاکہ اسٹرنگ ان پٹ قبول کرے
- تمام جڑے ہوئے ایجنٹس کو پیغامات نشر کرتا ہے
- `TurnToken` بھیج کر متوازی پراسیسنگ کو شروع کرتا ہے

**ConcurrentAggregationExecutor:**
- `IMessageHandler<ChatMessage>` کو نافذ کرتا ہے تاکہ ایجنٹس کے جوابات وصول کرے
- تھریڈ محفوظ طریقے سے پیغامات جمع کرتا ہے
- جب تمام متوقع جوابات آ جائیں تو نتائج جمع کرتا ہے
- `context.YieldOutputAsync()` کے ذریعے حتمی آؤٹ پٹ فراہم کرتا ہے

### ⚡ کارکردگی کے فوائد

**متوازی بمقابلہ تسلسل:**
- تسلسل: Agent1 (30s) → Agent2 (30s) = **کل 60 سیکنڈ**
