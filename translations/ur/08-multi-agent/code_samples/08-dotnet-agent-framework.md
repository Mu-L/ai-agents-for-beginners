# 🤝 انٹرپرائز ملٹی ایجنٹ ورک فلو سسٹمز (.NET)

## 📋 تعلیمی مقاصد

یہ نوٹ بک دکھاتی ہے کہ مائیکروسافٹ ایجنٹ فریم ورک کو .NET میں Azure OpenAI (Responses API) کے ساتھ استعمال کرتے ہوئے پیچیدہ انٹرپرائز گریڈ ملٹی ایجنٹ سسٹمز کیسے بنائیں۔ آپ سیکھیں گے کہ کس طرح مخصوص مہارت رکھنے والے ایجنٹس کو منظم ورک فلو کے ذریعے ایک ساتھ کام کرنے کے لیے ربط دیا جائے، اور .NET کی انٹرپرائز خصوصیات کو پیداواری حل کے لیے استعمال کیا جائے۔

**وہ انٹرپرائز ملٹی ایجنٹ صلاحیتیں جو آپ بنائیں گے:**
- 👥 **ایجنٹ تعاون**: کمپائل ٹائم کی توثیق کے ساتھ ٹائپ-سیف ایجنٹ کوآرڈینیشن
- 🔄 **ورک فلو آرکسٹریشن**: .NET کے اسینک پیٹرن کے ساتھ بیانیہ ورک فلو کی تعریف
- 🎭 **رول کی تخصیص**: مضبوط قسم کے ایجنٹ شخصیت اور مہارت کے شعبے
- 🏢 **انٹرپرائز انٹیگریشن**: مانیٹرنگ اور ایرر ہینڈلنگ کے ساتھ پیداواری پیٹرنز

## کوڈ کا نمونہ

اس سبق کے لیے مکمل کام کرنے والا کوڈ متعلقہ C# فائل میں دستیاب ہے: [`08-dotnet-agent-framework.cs`](../../../../08-multi-agent/code_samples/08-dotnet-agent-framework.cs)

نمونہ چلانے کے لیے:

```bash
# فائل کو قابلِ اجرا بنائیں (لینکس/میک او ایس)
chmod +x 08-dotnet-agent-framework.cs

# نمونہ چلائیں
./08-dotnet-agent-framework.cs
```

یا .NET CLI کا استعمال کرتے ہوئے:

```bash
dotnet run 08-dotnet-agent-framework.cs
```

## یہ نمونہ کیا ظاہر کرتا ہے

یہ ملٹی ایجنٹ ورک فلو سسٹم ایک ہوٹل ٹریول ریکمنڈیشن سروس بناتا ہے جس میں دو مخصوص ایجنٹس ہوتے ہیں:

1. **فرنٹ ڈیسک ایجنٹ**: ایک سفر ایجنٹ جو سرگرمی اور مقام کی سفارشات دیتا ہے
2. **کنسیئرج ایجنٹ**: سفارشات کا جائزہ لیتا ہے تاکہ مستند، غیر سیاحتی تجربات کو یقینی بنایا جا سکے

ایجنٹس ایک ورک فلو میں مل کر کام کرتے ہیں جہاں:
- فرنٹ ڈیسک ایجنٹ ابتدائی سفری درخواست وصول کرتا ہے
- کنسیئرج ایجنٹ سفارش کا جائزہ لے کر اسے بہتر بناتا ہے
- ورک فلو جوابات کو حقیقی وقت میں اسٹریم کرتا ہے

## کلیدی تصورات

### ایجنٹ کوآرڈینیشن
یہ نمونہ مائیکروسافٹ ایجنٹ فریم ورک کے ذریعے کمپائل ٹائم کی توثیق کے ساتھ ٹائپ-سیف ایجنٹ کوآرڈینیشن دکھاتا ہے۔

### ورک فلو آرکسٹریشن
.NET کے اسینک پیٹرنز کے ساتھ بیانیہ ورک فلو تعریف کا استعمال کرتا ہے تاکہ متعدد ایجنٹس کو پائپ لائن میں جوڑا جا سکے۔

### اسٹریمنگ جوابات
ایجنٹ کے جوابات کو حقیقی وقت میں اسٹریم کرنے کے لیے اسینک اینیومیریبلز اور واقعہ پر مبنی آرکیٹیکچر نافذ کرتا ہے۔

### انٹرپرائز انٹیگریشن
پیداواری پیٹرنز دکھاتا ہے جن میں شامل ہیں:
- ماحول کی متغیرات کی ترتیب
- محفوظ اسناد کا انتظام
- ایرر ہینڈلنگ
- غیر متزامن واقعہ پروسیسنگ
