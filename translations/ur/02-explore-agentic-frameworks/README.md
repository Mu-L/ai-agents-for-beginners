# ای آئی ایجنٹ فریم ورکس کی تلاش

ای آئی ایجنٹ فریم ورکس ایسے سافٹ ویئر پلیٹ فارمز ہیں جو ای آئی ایجنٹ بنانے، تعینات کرنے اور ان کا انتظام آسان بنانے کے لیے ڈیزائن کیے گئے ہیں۔ یہ فریم ورکس ڈویلپرز کو پہلے سے تیار شدہ اجزاء، انتزاعیات، اور ٹولز فراہم کرتے ہیں جو پیچیدہ ای آئی سسٹمز کی ترقی کو آسان بناتے ہیں۔

یہ فریم ورکس ڈویلپرز کو ان کی درخواستوں کے منفرد پہلوؤں پر توجہ مرکوز کرنے میں مدد دیتے ہیں کیونکہ یہ عام چیلنجز کے معیار شدہ حل فراہم کرتے ہیں جو ای آئی ایجنٹس کی ترقی میں پیش آتے ہیں۔ یہ ای آئی سسٹمز کی تعمیر میں توسیع پذیری، رسائی اور کارکردگی کو بہتر بناتے ہیں۔

## تعارف

اس سبق میں شامل ہوں گے:

- ای آئی ایجنٹ فریم ورکس کیا ہیں اور یہ ڈویلپرز کو کیا حاصل کرنے کے قابل بناتے ہیں؟
- ٹیمیں ان کا استعمال کیسے کر سکتی ہیں تاکہ جلد پروٹوٹائپ بنائیں، بار بار ترمیم کریں اور اپنے ایجنٹ کی صلاحیتوں کو بہتر بنائیں؟
- مائیکروسافٹ کے بنائے گئے فریم ورکس اور ٹولز کے درمیان کیا فرق ہے (<a href="https://aka.ms/ai-agents-beginners/ai-agent-service" target="_blank">Microsoft Foundry Agent Service</a> اور <a href="https://learn.microsoft.com/azure/ai-services/openai/how-to/responses" target="_blank">Microsoft Agent Framework</a>)؟
- کیا میں اپنے موجودہ ایزور ماحولیاتی نظام کے ٹولز کو براہِ راست شامل کر سکتا ہوں، یا مجھے الگ سے حل درکار ہیں؟
- مائیکروسافٹ فاؤنڈری ایجنٹ سروس کیا ہے اور یہ مجھے کیسے مدد دے رہی ہے؟

## سیکھنے کے مقاصد

اس سبق کے مقاصد آپ کی مدد کرنا ہیں کہ آپ سمجھ سکیں:

- ای آئی ایجنٹ فریم ورکس کا ای آئی ترقی میں کردار۔
- ذہین ایجنٹس بنانے کے لیے ای آئی ایجنٹ فریم ورکس کا استعمال کیسے کریں۔
- ای آئی ایجنٹ فریم ورکس کی اہم صلاحیتیں۔
- مائیکروسافٹ ایجنٹ فریم ورک اور مائیکروسافٹ فاؤنڈری ایجنٹ سروس کے مابین فرق۔

## ای آئی ایجنٹ فریم ورکس کیا ہیں اور یہ ڈویلپرز کو کیا کام کرنے دیتے ہیں؟

روایتی ای آئی فریم ورکس آپ کی ایپس میں ای آئی کو شامل کرنے اور ان ایپس کو بہتر بنانے کے درج ذیل طریقے فراہم کرتے ہیں:

- **ذاتی نوعیت**: ای آئی صارف کے رویے اور ترجیحات کا تجزیہ کر کے ذاتی سفارشات، مواد، اور تجربات فراہم کر سکتی ہے۔
مثال: نیٹ فلکس جیسے اسٹریمنگ سروسز ای آئی کا استعمال کرتی ہیں تاکہ دیکھنے کی تاریخ کی بنیاد پر فلمیں اور شوز تجویز کریں، جس سے صارف کی شرکت اور اطمینان میں اضافہ ہوتا ہے۔
- **خودکار اور موثر کام**: ای آئی دہرانے والے کام خودکار بنا سکتی ہے، ورک فلو کو آسان کرتی ہے اور عملی کارکردگی کو بہتر بناتی ہے۔
مثال: کسٹمر سروس ایپس میں ای آئی سے چلنے والے چیٹ بوٹس عام سوالات سنبھالتے ہیں، جواب کے وقت کو کم کرتے ہیں اور پیچیدہ مسائل کے لیے انسانی ایجنٹس کو فارغ کرتے ہیں۔
- **بہتر صارف تجربہ**: ای آئی ذہین خصوصیات جیسے وائس ریکگنیشن، قدرتی زبان کا پراسیسنگ، اور پیش گوئی متن فراہم کر کے مجموعی صارف تجربہ کو بہتر بنا سکتی ہے۔
مثال: ورچوئل اسسٹنٹس جیسے سری اور گوگل اسسٹنٹ وائس کمانڈز کو سمجھ کر جواب دیتے ہیں، جس سے صارفین کے لیے اپنے آلات کے ساتھ تعامل آسان ہو جاتا ہے۔

### یہ سب بہت اچھا ہے، لیکن پھر ہمیں ای آئی ایجنٹ فریم ورک کی کیوں ضرورت ہے؟

ای آئی ایجنٹ فریم ورکس صرف ای آئی فریم ورکس سے کچھ زیادہ نمائندگی کرتے ہیں۔ انہیں ذہین ایجنٹس بنانے کے لیے ڈیزائن کیا گیا ہے جو صارفین، دوسرے ایجنٹس، اور ماحول کے ساتھ بات چیت کر سکتے ہیں تاکہ مخصوص اہداف حاصل کریں۔ یہ ایجنٹس خود مختار رویہ دکھا سکتے ہیں، فیصلے کر سکتے ہیں، اور بدلتے حالات کے مطابق ڈھل سکتے ہیں۔ آئیے ای آئی ایجنٹ فریم ورکس کی کچھ اہم صلاحیتوں پر نظر ڈالیں:

- **ایجنٹ تعاون اور ہم آہنگی**: متعدد ایجنٹس بنانے کی اجازت دیتے ہیں جو ساتھ کام کر سکتے ہیں، بات چیت کر سکتے ہیں، اور پیچیدہ کام حل کرنے کے لیے ہم آہنگی کر سکتے ہیں۔
- **کام کا خودکار اور انتظام**: کئی مرحلوں والے ورک فلو، کام کی تفویض، اور ایجنٹس کے درمیان ڈائنامک کام کے انتظام کے طریقے فراہم کرتے ہیں۔
- **سیاق و سباق کی سمجھ بوجھ اور مطابقت**: ایجنٹس کو سیاق و سباق سمجھنے، بدلتے ماحول کے مطابق ڈھلنے، اور حقیقی وقت کی معلومات کی بنیاد پر فیصلے کرنے کی صلاحیت فراہم کرتے ہیں۔

خلاصہ یہ ہے کہ، ایجنٹس آپ کو زیادہ کرنے دیتے ہیں، خود کاری کو اگلے درجے تک لے جاتے ہیں، اور زیادہ ذہین سسٹمز بناتے ہیں جو اپنے ماحول سے سیکھ اور ڈھل سکتے ہیں۔

## ایجنٹ کی صلاحیتوں کو جلد پروٹوٹائپ، ترمیم، اور بہتر بنانے کا طریقہ؟

یہ ایک تیزی سے بدلنے والا شعبہ ہے، لیکن کچھ چیزیں زیادہ تر ای آئی ایجنٹ فریم ورکز میں مشترک ہیں جو آپ کو جلد پروٹوٹائپ اور بار بار ترمیم میں مدد دیتی ہیں، جیسے ماڈیولر اجزاء، تعاون کے اوزار، اور حقیقی وقت میں سیکھنا۔ آئیے ان پر تفصیل سے بات کریں:

- **ماڈیولر اجزاء کا استعمال کریں**: ای آئی SDKs پہلے سے تیار اجزاء فراہم کرتے ہیں جیسے ای آئی اور میموری کنیکٹرز، قدرتی زبان یا کوڈ پلگ ان کے ذریعے فنکشن کالنگ، پرامپٹ ٹیمپلیٹس، اور مزید۔
- **تعاون کے اوزار کا فائدہ اٹھائیں**: مخصوص کرداروں اور کاموں کے ساتھ ایجنٹس ڈیزائن کریں، تاکہ وہ تعاون کے ورک فلو کا تجربہ کریں اور انہیں بہتر بنائیں۔
- **حقیقی وقت میں سیکھیں**: فیڈبیک لوپس نافذ کریں جہاں ایجنٹس بات چیت سے سیکھ کر اپنے رویے کو متحرک طور پر ایڈجسٹ کریں۔

### ماڈیولر اجزاء کا استعمال کریں

مائیکروسافٹ ایجنٹ فریم ورک جیسے SDKs پہلے سے تیار اجزاء مہیا کرتے ہیں جیسے ای آئی کنیکٹرز، ٹول تعریفیں، اور ایجنٹ انتظام۔

**ٹیمیں انہیں کیسے استعمال کر سکتی ہیں**: ٹیمیں ان اجزاء کو جلدی سے جوڑ کر ایک فنکشنل پروٹوٹائپ بنا سکتی ہیں بغیر نیا سے شروع کیے، جس سے تیز تجربہ اور ترمیم ممکن ہوتی ہے۔

**عملی طور پر یہ کیسے کام کرتا ہے**: آپ یوزر ان پٹ سے معلومات نکالنے کے لیے پہلے سے بنی پارسر، ڈیٹا محفوظ اور بازیافت کرنے کے لیے میموری ماڈیول، اور یوزرز کے ساتھ بات چیت کے لیے پرامپٹ جنریٹر استعمال کر سکتے ہیں، بغیر خود سے ان اجزاء کو بنانے کے۔

**مثال کوڈ**. آئیے مائیکروسافٹ ایجنٹ فریم ورک کے `FoundryChatClient` کے ساتھ ایک مثال دیکھیں تاکہ ماڈل یوزر ان پٹ کو ٹول کالنگ کے ساتھ جواب دے سکے:

``` python
# مائیکروسافٹ ایجنٹ فریم ورک پائتھن کی مثال

import asyncio
import os

from agent_framework import tool
from agent_framework.foundry import FoundryChatClient
from azure.identity import AzureCliCredential


# سفر بک کرنے کے لیے ایک نمونہ ٹول فنکشن کی تعریف کریں
@tool(approval_mode="never_require")
def book_flight(date: str, location: str) -> str:
    """Book travel given location and date."""
    return f"Travel was booked to {location} on {date}"


async def main():
    provider = FoundryChatClient(
        project_endpoint=os.environ["AZURE_AI_PROJECT_ENDPOINT"],
        model=os.environ["AZURE_AI_MODEL_DEPLOYMENT_NAME"],
        credential=AzureCliCredential(),
    )
    agent = provider.as_agent(
        name="travel_agent",
        instructions="Help the user book travel. Use the book_flight tool when ready.",
        tools=[book_flight],
    )

    response = await agent.run("I'd like to go to New York on January 1, 2025")
    print(response)
    # مثال آؤٹ پٹ: آپ کی 1 جنوری 2025 کو نیو یارک کی پرواز کامیابی سے بک ہو چکی ہے۔ محفوظ سفر! ✈️🗽


if __name__ == "__main__":
    asyncio.run(main())
```

اس مثال سے آپ دیکھ سکتے ہیں کہ کس طرح آپ پہلے سے بنی پارسر کا فائدہ اٹھا کر یوزر ان پٹ سے کلیدی معلومات نکال سکتے ہیں، مثلاً فلائٹ بکنگ درخواست کی ابتدا، منزل، اور تاریخ۔ یہ ماڈیولر طریقہ آپ کو اعلی سطحی منطق پر توجہ مرکوز کرنے دیتا ہے۔

### تعاون کے اوزار کا فائدہ اٹھائیں

مائیکروسافٹ ایجنٹ فریم ورک جیسے فریم ورکس متعدد ایجنٹس بنانے میں آسانی فراہم کرتے ہیں جو ایک ساتھ کام کر سکتے ہیں۔

**ٹیمیں انہیں کیسے استعمال کر سکتی ہیں**: ٹیمیں مخصوص کرداروں اور کاموں کے ساتھ ایجنٹس ڈیزائن کر کے تعاون کے ورک فلو کی آزمائش اور بہتری کر سکتی ہیں اور مجموعی نظام کی کارکردگی کو بڑھا سکتی ہیں۔

**عملی طور پر یہ کیسے کام کرتا ہے**: آپ بہت سے ایجنٹس کی ٹیم بنا سکتے ہیں جہاں ہر ایجنٹ کا خاص فنکشن ہو، جیسے ڈیٹا حاصل کرنا، تجزیہ یا فیصلہ سازی۔ یہ ایجنٹس بات چیت کر کے اور معلومات کا تبادلہ کر کے ایک مشترکہ مقصد حاصل کر سکتے ہیں، جیسے صارف کے سوال کا جواب دینا یا کوئی کام مکمل کرنا۔

**مثال کوڈ (مائیکروسافٹ ایجنٹ فریم ورک)**:

```python
# مائیکروسافٹ ایجنٹ فریم ورک کا استعمال کرتے ہوئے مل کر کام کرنے والے متعدد ایجنٹس تخلیق کرنا

import os
from agent_framework.foundry import FoundryChatClient
from azure.identity import AzureCliCredential

provider = FoundryChatClient(
    project_endpoint=os.environ["AZURE_AI_PROJECT_ENDPOINT"],
    model=os.environ["AZURE_AI_MODEL_DEPLOYMENT_NAME"],
    credential=AzureCliCredential(),
)

# ڈیٹا بازیافت ایجنٹ
agent_retrieve = provider.as_agent(
    name="dataretrieval",
    instructions="Retrieve relevant data using available tools.",
    tools=[retrieve_tool],
)

# ڈیٹا تجزیہ ایجنٹ
agent_analyze = provider.as_agent(
    name="dataanalysis",
    instructions="Analyze the retrieved data and provide insights.",
    tools=[analyze_tool],
)

# ایک کام پر ایجنٹس کو ترتیب وار چلانا
retrieval_result = await agent_retrieve.run("Retrieve sales data for Q4")
analysis_result = await agent_analyze.run(f"Analyze this data: {retrieval_result}")
print(analysis_result)
```

پچھلے کوڈ میں آپ دیکھ سکتے ہیں کہ ایک ایسا کام بنایا جا رہا ہے جس میں متعدد ایجنٹس کام کرتے ہوئے ڈیٹا کا تجزیہ کرتے ہیں۔ ہر ایجنٹ مخصوص فنکشن انجام دیتا ہے، اور کام ایجنٹس کو ہم آہنگ کر کے مطلوبہ نتیجہ حاصل کرنے کے لیے چلایا جاتا ہے۔ مخصوص کرداروں والے ایجنٹس بنانے سے کام کی کامیابی اور کارکردگی میں بہتری آتی ہے۔

### حقیقی وقت میں سیکھیں

جدید فریم ورکس حقیقی وقت میں سیاق و سباق کو سمجھنے اور مطابقت فراہم کرنے کی صلاحیتیں رکھتے ہیں۔

**ٹیمیں انہیں کیسے استعمال کر سکتی ہیں**: ٹیمیں فیڈبیک لوپس نافذ کر سکتی ہیں جہاں ایجنٹس بات چیت سے سیکھ کر اپنے رویوں کو متحرک طور پر ایڈجسٹ کرتے ہیں، جس سے مسلسل بہتری اور صلاحیتوں کی نفاست ہوتی ہے۔

**عملی طور پر یہ کیسے کام کرتا ہے**: ایجنٹس صارف کے فیڈبیک، ماحولیاتی ڈیٹا، اور کام کے نتائج کا تجزیہ کر کے اپنے علم کی بنیاد اپ ڈیٹ کرتے ہیں، فیصلہ سازی کے الگوردمز کو ایڈجسٹ کرتے ہیں، اور وقت کے ساتھ کارکردگی بہتر بناتے ہیں۔ یہ مرحلہ وار سیکھنے کا عمل ایجنٹس کو بدلتے حالات اور صارف کی ترجیحات کے مطابق ڈھالنے میں مدد دیتا ہے، جس سے مجموعی نظام کی تاثیر بڑھتی ہے۔

## مائیکروسافٹ ایجنٹ فریم ورک اور مائیکروسافٹ فاؤنڈری ایجنٹ سروس میں کیا فرق ہے؟

ان طریقوں کا موازنہ کئی طرح سے کیا جاسکتا ہے، لیکن آئیے ان کے ڈیزائن، صلاحیتوں اور ہدف کے استعمال کے لحاظ سے کچھ اہم فرق دیکھتے ہیں:

## مائیکروسافٹ ایجنٹ فریم ورک (MAF)

مائیکروسافٹ ایجنٹ فریم ورک `FoundryChatClient` کا استعمال کرتے ہوئے ای آئی ایجنٹس بنانے کے لیے ایک ہموار SDK فراہم کرتا ہے۔ یہ ڈویلپرز کو اجازت دیتا ہے کہ وہ ایجنٹس بنائیں جو ایزور اوپن ای آئی ماڈلز کو ٹول کالنگ، بات چیت کے انتظام، اور ایزور آئیڈینٹی کے ذریعے انٹرپرائز سیکورٹی کے ساتھ استعمال کریں۔

**استعمال کے معاملات**: ٹول کے استعمال، متعدد مرحلوں والے ورک فلو، اور انٹرپرائز انضمام کے منظرناموں کے ساتھ پروڈکشن کے قابل ایجنٹس بنانا۔

مائیکروسافٹ ایجنٹ فریم ورک کے چند اہم بنیادی تصورات درج ذیل ہیں:

- **ایجنٹس**۔ ایک ایجنٹ `FoundryChatClient` کے ذریعے بنایا جاتا ہے اور نام، ہدایات، اور ٹولز کے ساتھ ترتیب دیا جاتا ہے۔ ایجنٹ یہ کام کرسکتا ہے:
  - **صارف کے پیغامات کی پروسیسنگ** اور Azure OpenAI ماڈلز کے ذریعے جوابات تیار کرنا۔
  - **ٹولز کو خودکار طور پر کال کرنا** بات چیت کے سیاق و سباق کی بنیاد پر۔
  - **بات چیت کی حالت برقرار رکھنا** متعدد تعاملات کے درمیان۔

  یہاں ایک کوڈ کا ٹکڑا ہے جو ایجنٹ بنانے کا طریقہ دکھاتا ہے:

    ```python
    import os
    from agent_framework.foundry import FoundryChatClient
    from azure.identity import AzureCliCredential

    provider = FoundryChatClient(
        project_endpoint=os.environ["AZURE_AI_PROJECT_ENDPOINT"],
        model=os.environ["AZURE_AI_MODEL_DEPLOYMENT_NAME"],
        credential=AzureCliCredential(),
    )
    agent = provider.as_agent(
        name="my_agent",
        instructions="You are a helpful assistant.",
    )

    response = await agent.run("Hello, World!")
    print(response)
    ```

- **ٹولز**۔ فریم ورک ٹولز کو Python فنکشنز کے طور پر تعریف کرنے کی حمایت کرتا ہے جنہیں ایجنٹ خودکار طور پر کال کر سکتا ہے۔ جب ایجنٹ بنایا جاتا ہے تو ٹولز رجسٹر کیے جاتے ہیں:

    ```python
    def get_weather(location: str) -> str:
        """Get the current weather for a location."""
        return f"The weather in {location} is sunny, 72\u00b0F."

    agent = provider.as_agent(
        name="weather_agent",
        instructions="Help users check the weather.",
        tools=[get_weather],
    )
    ```

- **کئی ایجنٹس کا ہم آہنگی**۔ آپ مختلف تخصصات کے ساتھ متعدد ایجنٹس بنا سکتے ہیں اور ان کے کام کو ہم آہنگ کر سکتے ہیں:

    ```python
    planner = provider.as_agent(
        name="planner",
        instructions="Break down complex tasks into steps.",
    )

    executor = provider.as_agent(
        name="executor",
        instructions="Execute the planned steps using available tools.",
        tools=[execute_tool],
    )

    plan = await planner.run("Plan a trip to Paris")
    result = await executor.run(f"Execute this plan: {plan}")
    ```

- **ایزور آئیڈینٹی انضمام**۔ فریم ورک محفوظ، بغیر کلید کی تصدیق کے لیے `AzureCliCredential` (یا `DefaultAzureCredential`) کا استعمال کرتا ہے، جس سے API کیز کا انتظام کرنے کی ضرورت نہیں ہوتی۔

## مائیکروسافٹ فاؤنڈری ایجنٹ سروس

مائیکروسافٹ فاؤنڈری ایجنٹ سروس ایک حالیہ اضافہ ہے، جو مائیکروسافٹ اگنائٹ 2024 میں متعارف کرایا گیا۔ یہ زیادہ لچکدار ماڈلز کے ساتھ ای آئی ایجنٹس کی ترقی اور تعیناتی کی اجازت دیتا ہے، جیسے کہ براہ راست اوپن سورس LLMs جیسے لاما 3، میسٹرل، اور کوہیر کو کال کرنا۔

مائیکروسافٹ فاؤنڈری ایجنٹ سروس مضبوط انٹرپرائز سیکورٹی کے میکانزم اور ڈیٹا اسٹوریج کے طریقے فراہم کرتا ہے، جو اسے انٹرپرائز ایپلیکیشنز کے لیے موزوں بناتا ہے۔

یہ مائیکروسافٹ ایجنٹ فریم ورک کے ساتھ بغیر کسی رکاوٹ کے کام کرتا ہے تاکہ ایجنٹس تیار اور تعینات کیے جا سکیں۔

یہ سروس اس وقت پبلک پریویو میں ہے اور ایجنٹس بنانے کے لیے Python اور C# کی حمایت کرتی ہے۔

مائیکروسافٹ فاؤنڈری ایجنٹ سروس Python SDK استعمال کرتے ہوئے، ہم صارف کی طرف سے تعریف کردہ ٹول کے ساتھ ایجنٹ بنا سکتے ہیں:

```python
import asyncio
from azure.identity import DefaultAzureCredential
from azure.ai.projects import AIProjectClient

# ٹول فنکشنز کی تعریف کریں
def get_specials() -> str:
    """Provides a list of specials from the menu."""
    return """
    Special Soup: Clam Chowder
    Special Salad: Cobb Salad
    Special Drink: Chai Tea
    """

def get_item_price(menu_item: str) -> str:
    """Provides the price of the requested menu item."""
    return "$9.99"


async def main() -> None:
    credential = DefaultAzureCredential()
    project_client = AIProjectClient.from_connection_string(
        credential=credential,
        conn_str="your-connection-string",
    )

    agent = project_client.agents.create_agent(
        model="gpt-4.1-mini",
        name="Host",
        instructions="Answer questions about the menu.",
        tools=[get_specials, get_item_price],
    )

    thread = project_client.agents.create_thread()

    user_inputs = [
        "Hello",
        "What is the special soup?",
        "How much does that cost?",
        "Thank you",
    ]

    for user_input in user_inputs:
        print(f"# User: '{user_input}'")
        message = project_client.agents.create_message(
            thread_id=thread.id,
            role="user",
            content=user_input,
        )
        run = project_client.agents.create_and_process_run(
            thread_id=thread.id, agent_id=agent.id
        )
        messages = project_client.agents.list_messages(thread_id=thread.id)
        print(f"# Agent: {messages.data[0].content[0].text.value}")


if __name__ == "__main__":
    asyncio.run(main())
```

### بنیادی تصورات

مائیکروسافٹ فاؤنڈری ایجنٹ سروس کے درج ذیل بنیادی تصورات ہیں:

- **ایجنٹ**۔ مائیکروسافٹ فاؤنڈری ایجنٹ سروس مائیکروسافٹ فاؤنڈری کے ساتھ ضم ہوتا ہے۔ مائیکروسافٹ فاؤنڈری میں، ایک ای آئی ایجنٹ ایک "سمارٹ" مائکرو سروس کے طور پر کام کرتا ہے جو سوالات کے جواب (RAG)، کام انجام دینے، یا مکمل ورک فلو خود کار بنانے کے لیے استعمال کیا جا سکتا ہے۔ یہ تخلیقی ای آئی ماڈلز کی طاقت کو ایسے ٹولز کے ساتھ جوڑ کر حاصل کرتا ہے جو اسے حقیقی دنیا کے ڈیٹا ذرائع تک رسائی اور تعامل کی اجازت دیتے ہیں۔ یہاں ایک ایجنٹ کی مثال ہے:

    ```python
    agent = project_client.agents.create_agent(
        model="gpt-4.1-mini",
        name="my-agent",
        instructions="You are helpful agent",
        tools=code_interpreter.definitions,
        tool_resources=code_interpreter.resources,
    )
    ```

    اس مثال میں، ایک ایجنٹ `gpt-4.1-mini` ماڈل، نام `my-agent`، اور ہدایات `You are helpful agent` کے ساتھ بنایا گیا ہے۔ ایجنٹ کو کوڈ کی تعبیر کرنے کے کام انجام دینے کے لیے ٹولز اور وسائل سے لیس کیا گیا ہے۔

- **تھریڈ اور پیغامات**۔ تھریڈ ایک اور اہم تصور ہے۔ یہ ایک ایجنٹ اور صارف کے درمیان گفتگو یا تعامل کی نمائندگی کرتا ہے۔ تھریڈز کو گفتگو کی پیش رفت کو ٹریک کرنے، سیاق و سباق کی معلومات ذخیرہ کرنے، اور تعامل کی حالت کو منظم کرنے کے لیے استعمال کیا جا سکتا ہے۔ یہاں ایک تھریڈ کی مثال ہے:

    ```python
    thread = project_client.agents.create_thread()
    message = project_client.agents.create_message(
        thread_id=thread.id,
        role="user",
        content="Could you please create a bar chart for the operating profit using the following data and provide the file to me? Company A: $1.2 million, Company B: $2.5 million, Company C: $3.0 million, Company D: $1.8 million",
    )
    
    # ایجنٹ سے کہیں کہ وہ تھریڈ پر کام کرے
    run = project_client.agents.create_and_process_run(thread_id=thread.id, agent_id=agent.id)
    
    # تمام پیغامات لائیں اور لاگ کریں تاکہ ایجنٹ کے جواب کو دیکھا جا سکے
    messages = project_client.agents.list_messages(thread_id=thread.id)
    print(f"Messages: {messages}")
    ```

    پچھلے کوڈ میں، ایک تھریڈ بنایا گیا ہے۔ اس کے بعد، تھریڈ کو پیغام بھیجا گیا ہے۔ `create_and_process_run` کو کال کر کے، ایجنٹ سے تھریڈ پر کام کرنے کو کہا جاتا ہے۔ آخر میں، پیغامات حاصل کرکے لاگ کیے گئے ہیں تاکہ ایجنٹ کے جواب کو دیکھا جا سکے۔ پیغامات صارف اور ایجنٹ کے درمیان گفتگو کی پیش رفت کی نشاندہی کرتے ہیں۔ یہ بھی سمجھنا ضروری ہے کہ پیغامات مختلف قسم کے ہو سکتے ہیں جیسے متن، تصویر، یا فائل، یعنی ایجنٹس کے کام کا نتیجہ جیسے تصویر یا متن کا جواب۔ ایک ڈویلپر کے طور پر، آپ اس معلومات کو جواب کو مزید پراسیس کرنے یا صارف کو پیش کرنے کے لیے استعمال کر سکتے ہیں۔

- **مائیکروسافٹ ایجنٹ فریم ورک کے ساتھ انضمام**۔ مائیکروسافٹ فاؤنڈری ایجنٹ سروس، مائیکروسافٹ ایجنٹ فریم ورک کے ساتھ بغیر کسی رکاوٹ کام کرتا ہے، جس کا مطلب ہے کہ آپ `FoundryChatClient` کا استعمال کر کے ایجنٹس بنا سکتے ہیں اور انہیں ایجنٹ سروس کے ذریعے پروڈکشن منظرناموں میں تعینات کر سکتے ہیں۔

**استعمال کے معاملات**: مائیکروسافٹ فاؤنڈری ایجنٹ سروس انٹرپرائز ایپلیکیشنز کے لیے ڈیزائن کی گئی ہے جو محفوظ، قابل پیمائش، اور لچکدار ای آئی ایجنٹ تعیناتی کی ضرورت ہوتی ہے۔

## ان طریقوں کے درمیان کیا فرق ہے؟
 
یہ واقعی کچھ حد تک اوورلیپڈ معلوم ہوتے ہیں، لیکن ان کے ڈیزائن، صلاحیتوں، اور ہدف کے استعمال کے لحاظ سے چند اہم فرق ہیں:
 
- **مائیکروسافٹ ایجنٹ فریم ورک (MAF)**: ای آئی ایجنٹس بنانے کے لیے پروڈکشن-ریڈی SDK ہے۔ یہ ٹول کالنگ، بات چیت کے انتظام، اور ایزور شناختی انضمام کے لیے ہموار API فراہم کرتا ہے۔
- **مائیکروسافٹ فاؤنڈری ایجنٹ سروس**: مائیکروسافٹ فاؤنڈری میں ایجنٹس کے لیے ایک پلیٹ فارم اور تعیناتی سروس ہے۔ یہ Azure OpenAI، Azure AI سرچ، Bing سرچ، اور کوڈ کے نفاذ جیسے خدمات کے لیے اندرونی کنیکٹوٹی فراہم کرتا ہے۔
 
ابھی بھی فیصلہ نہیں کیا کہ کون سا منتخب کریں؟

### استعمال کے معاملات
 
آئیے دیکھیں کہ ہم آپ کی مدد کر سکتے ہیں عام استعمال کے چند کیسز کے ذریعے:
 
> سوال: میں پروڈکشن کے قابل ای آئی ایجنٹ ایپلیکیشنز بنا رہا ہوں اور جلد شروع کرنا چاہتا ہوں۔
>

> جواب: مائیکروسافٹ ایجنٹ فریم ورک بہترین انتخاب ہے۔ یہ `FoundryChatClient` کے ذریعے آسان، پائیتھونک API فراہم کرتا ہے جو آپ کو چند لائنوں کوڈ میں ایجنٹس کو ٹولز اور ہدایات کے ساتھ تعریف کرنے دیتا ہے۔

> سوال: مجھے انٹرپرائز گریڈ تعیناتی چاہیے جیسے Azure انضمام، سرچ، اور کوڈ نفاذ۔
>
> جواب: مائیکروسافٹ فاؤنڈری ایجنٹ سروس بہترین انتخاب ہے۔ یہ ایک پلیٹ فارم سروس ہے جو مختلف ماڈلز، Azure AI سرچ، Bing سرچ، اور Azure فنکشنز کے لیے تیار شدہ صلاحیتیں فراہم کرتی ہے۔ یہ Foundry پورٹل میں ایجنٹس بنانے اور بڑے پیمانے پر تعینات کرنے کو آسان بناتا ہے۔
 
> سوال: میں اب بھی الجھن میں ہوں، بس مجھے ایک اختیار دے دیں۔
>
> جواب: مائیکروسافٹ ایجنٹ فریم ورک سے شروع کریں تاکہ اپنے ایجنٹس بنائیں، اور جہاں ضرورت ہو پروڈکشن میں تعیناتی اور پیمائش کے لیے مائیکروسافٹ فاؤنڈری ایجنٹ سروس استعمال کریں۔ یہ طریقہ آپ کو اپنے ایجنٹ لوجک پر جلدی ترمیم کرنے دیتا ہے جبکہ انٹرپرائز تعیناتی کے لیے واضح راستہ فراہم کرتا ہے۔

## کیا میں اپنے موجودہ ایزور ماحولیاتی نظام کے ٹولز کو براہِ راست شامل کر سکتا ہوں، یا مجھے الگ سے حل درکار ہیں؟


جواب ہاں ہے، آپ اپنے موجودہ Azure ایکو سسٹم ٹولز کو براہ راست Microsoft Foundry Agent Service کے ساتھ ضم کر سکتے ہیں خاص طور پر، کیونکہ اسے دوسرے Azure سروسز کے ساتھ بغیر رکاوٹ کام کرنے کے لیے بنایا گیا ہے۔ آپ مثال کے طور پر Bing، Azure AI Search، اور Azure Functions کو ضم کر سکتے ہیں۔ Microsoft Foundry کے ساتھ بھی گہری انضمام موجود ہے۔
