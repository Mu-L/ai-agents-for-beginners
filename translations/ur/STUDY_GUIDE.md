# ابتدائی افراد کے لیے AI ایجنٹس - مطالعہ کا رہنما

اس رہنما کو کورس کے دوران ایک عملی ساتھی کے طور پر استعمال کریں۔ یہ
اسباق کی جگہ لینے کے لیے نہیں ہے۔ یہ آپ کی مدد کرتا ہے کہ کہاں سے شروع کریں، 
ہر سبق میں کیا دیکھنا ہے، اور خیالات کو ایک چھوٹے کام کرنے والے ایجنٹ
ڈیمو میں کیسے مربوط کیا جائے۔

اگر یہ آپ کا پہلی بار یہاں آنا ہے، تو سادہ شروع کریں:

1. [Course Setup](./00-course-setup/README.md) پڑھیں۔
2. اسباق 01-06 کو ترتیب سے مکمل کریں۔
3. سیکھتے ہوئے ایک چھوٹا ڈیمو آئیڈیا ذہن میں رکھیں۔
4. ہر سبق کے بعد پوچھیں: "میرا ایجنٹ اب کیا کر سکتا ہے جو پہلے نہیں کر سکتا تھا؟"


## ذہن میں رکھنے کے لیے ایک سادہ ڈیمو

ایجنٹس سیکھنے کا ایک اچھا طریقہ ہے کہ پورے کورس میں ایک ڈیمو آئیڈیا کو فالو کریں۔

مثال کے طور پر ڈیمو: **ایک کورس مددگار ایجنٹ**۔

صارف سوال کرتا ہے:

> "میں سیکھنا چاہتا ہوں کہ ایجنٹس ٹولز کیسے استعمال کرتے ہیں۔ صحیح اسباق تلاش کریں، 
> جو مجھے پہلے پڑھنا چاہیے اس کا خلاصہ دیں، اور مجھے ایک چھوٹا عملي کام دیں۔"

ایک عام چیٹ بوٹ جو کچھ جانتا ہے اس کا جواب دے سکتا ہے۔ ایک ایجنٹ زیادہ کر سکتا ہے:

1. **کورس کی فائلیں پڑھیں یا تلاش کریں** تاکہ صحیح اسباق مل سکیں۔
2. **ٹولز استعمال کریں** تاکہ سبق کے لنکس، مثالیں، یا مددگار مواد حاصل کیا جا سکے۔
3. **منصوبہ بندی کریں** ایک مختصر سیکھنے کا راستہ، ایک لمبا جواب دینے کے بجائے۔
4. **موجودہ گفتگو کا تناظر استعمال کریں** تاکہ سیکھنے والے کے مقصد پر توجہ برقرار رکھی جا سکے۔

5. **اگر ایپ میموری کی حمایت کرتی ہے تو مفید ترجیحات یاد رکھیں۔**
6. **نقشے، حوالہ جات، یا لاگز دکھائیں** تاکہ صارف سمجھ سکے کہ کیا ہوا۔
7. **خطرناک کارروائیاں کرنے سے پہلے یا حساس ڈیٹا استعمال کرنے سے پہلے حفاظتی اقدامات کریں۔**

جب آپ ہر سبق پڑھیں، تو اس ڈیمو پر واپس آئیں اور پوچھیں: اس سبق سے کونسی نئی صلاحیت آئے گی؟


## جس کی طرف آپ تعمیر کر رہے ہیں

کورس کے اختتام تک، آپ کو ایجنٹ سسٹمز کی وضاحت اور تعمیر کرنے کے قابل ہونا چاہیے
جو ان حصوں کو یکجا کرتے ہیں:

| حصہ | عام زبان میں مطلب | ڈیمو میں |
|------|------------------------|-------------|
| ماڈل | استدلال کا انجن جو صارف کی درخواست کی تشریح کرتا ہے | سمجھتا ہے کہ سیکھنے والا ٹول کے استعمال کے بارے میں درس چاہتا ہے |
| ٹولز | افعال، APIs، فائلیں، براؤزرز، یا خدمات جو ایجنٹ استعمال کر سکتا ہے | ریپو تلاش کرتا ہے یا سبق کا مواد لاتا ہے |
| علم | دستاویزات یا ڈیٹا جو جواب کی بنیاد بناتے ہیں | کورس README فائلیں اور سبق کا مواد |
| تناظر | اگلے ماڈل کال میں شامل معلومات | صارف کا مقصد اور ٹول کے نتائج |
| میموری | بعد میں استعمال کے لیے محفوظ کی گئی معلومات | سیکھنے والا ہاتھ سے Python کی مثالیں پسند کرتا ہے |
| منصوبہ بندی | بڑے مقصد کو چھوٹے مراحل میں تقسیم کرنا | اسباق تلاش کرنا، ان کا خلاصہ، عملی کام تجویز کرنا |
| تنظیم | کام کو ٹولز، مراحل، یا ایجنٹس میں تقسیم کرنا | ایک پلانر سرچ ٹول کو کال کرتا ہے، پھر ایک سمریزر کو |
| اعتماد | حفاظتی، سلامتی، جائزہ، اور قابل مشاہدہ ہونا | ٹول کالز کے لاگز رکھتا ہے اور پرُ اثر اقدام سے پہلے پوچھتا ہے |

## ماڈلز اور فراہم کنندگان

کورس کے کوڈ کے نمونے **Microsoft Agent Framework (MAF)** استعمال کرتے ہیں اور **Azure OpenAI Responses API** کو ہدف بناتے ہیں — آگے کا تجویز کردہ API جو چیٹ مکملات، ٹول کال کرنا، ملٹی موڈل ان پٹ، اور ریاستی گفتگو کو ایک ہی API سطح پر یکجا کرتا ہے۔ آپ یا تو **Microsoft Foundry** پروجیکٹ ( `FoundryChatClient` کے ساتھ) یا Azure OpenAI سے براہ راست ( `OpenAIChatClient` کے ساتھ) جڑتے ہیں۔


جیسے جیسے آپ سبقوں پر کام کرتے ہیں، آپ کے پاس چند فراہم کنندہ کے اختیارات ہیں:

- **Microsoft Foundry / Azure OpenAI (Responses API)** — سبقوں میں استعمال ہونے والا بنیادی راستہ۔ Entra ID کی بغیر کلید کی تصدیق کے لئے `az login` کے ساتھ سائن ان کریں۔
- **Foundry Local** — مکمل طور پر آلے پر ماڈلز چلائیں جو OpenAI-مطابق API کے ذریعے ہیں (کوئی کلاؤڈ نہیں، کوئی API کیز نہیں)۔ آف لائن یا مفت تجربات کے لیے مثالی۔ دیکھیں [Course Setup](./00-course-setup/README.md)۔
- **MiniMax** — ایک OpenAI-مطابق فراہم کنندہ جو بڑے کانٹیکسٹ والے ماڈلز کے ساتھ، بطور متبادل استعمال کیا جا سکتا ہے۔

> **نوٹ:** GitHub Models اب استعمال میں نہیں ہے (جولائی 2026 میں بند ہو رہا ہے) اور Responses API کی حمایت نہیں کرتا۔ نمونے اب Azure OpenAI / Microsoft Foundry استعمال کرنے کے لیے اپ ڈیٹ کر دیے گئے ہیں۔

## ابتدائی دوست اصطلاحات میں کلیدی خیالات

### اوزار

ایک آلہ وہ چیز ہے جسے ایجنٹ ماڈل کے باہر کام کرنے کے لیے کال کر سکتا ہے۔ ایک اچھا آلہ
ایک واضح نام، ایک محدود کام، ٹائپڈ ان پٹس، متوقع آوٹ پٹ، اور ناکامی کا محفوظ طریقہ رکھتا ہے۔


کورس ہیلپر ڈیمو کے لیے، ایک آلہ ہو سکتا ہے:

- `search_lessons(query)`
- `read_lesson(path)`
- `create_practice_task(topic)`

### RAG اور علم

RAG ایجنٹ کو اندازہ لگانے کے بجائے ماخذ مواد سے جواب دینے میں مدد دیتا ہے۔ اس
کورس میں، ماخذ مواد سبق کے README فائلیں، کوڈ نمونے، یا بیرونی
وسائل ہو سکتے ہیں جو سبق سے منسلک ہیں۔

جب جواب دستاویزات، ڈیٹا، یا موجودہ پروجیکٹ فائلوں پر مبنی ہونا چاہیے تو RAG استعمال کریں۔


### منصوبہ بندی

جب درخواست میں ایک سے زیادہ مراحل ہوں تو منصوبہ بندی مفید ہوتی ہے۔ منصوبے مختصر اور
اس حد تک قابلِ نظر رکھیں کہ کوئی ڈویلپر یا صارف ان کا جائزہ لے سکے۔

ڈیمو کے لیے، ایک منصوبہ ہو سکتا ہے:

1. ٹول کے استعمال سے متعلق سبق تلاش کریں۔
2. سب سے متعلقہ سبق کا خلاصہ کریں۔
3. ایک عملی کام تجویز کریں۔

### سیاق و سباق

سیاق و سباق وہ ہوتا ہے جو ماڈل ابھی دیکھ رہا ہے۔ بہت کم سیاق و سباق ایجنٹ کو
اہم تفصیلات کو کھو دینے کا باعث بن سکتا ہے۔ بہت زیادہ سیاق و سباق ایجنٹ کو سست،
زیادہ مہنگا، یا زیادہ آسانی سے الجھنے والا بنا سکتا ہے۔

اچھا سیاق و سباق کا انجنئیرنگ مطلب ہے اگلی ماڈل کال کے لیے صحیح معلومات کا انتخاب کرنا۔


### یادداشت

یادداشت وہ معلومات ہے جو بعد میں محفوظ کی جاتی ہے۔ سب کچھ محفوظ نہ کریں۔ معلومات فقط اس وقت محفوظ کریں جب وہ مفید، محفوظ، اور آسانی سے اپ ڈیٹ یا حذف کی جا سکے۔


مثلا، یاد رکھنا "سیکھنے والا پائتھن مثالوں کو ترجیح دیتا ہے" مفید ہو سکتا ہے۔
حساس ذاتی ڈیٹا کو یاد رکھنا عموماً مناسب نہیں ہوتا۔

### تشخیص اور معائنہ

تشخیص پوچھتی ہے: کیا ایجنٹ نے صحیح کام کیا؟

معائنہ پوچھتا ہے: کیا ہم دیکھ سکتے ہیں یہ کیسے ہوا؟

پیداوار کے ایجنٹس کے لیے، ماڈل کالز، ٹول کالز، حاصل کردہ سیاق و سباق،
تاخیر، لاگت، ناکامیاں، اور صارف کی رائے کا ریکارڈ رکھیں۔

### اعتماد اور سیکیورٹی

قابل اعتماد ایجنٹس کو صرف مددگار پرامپٹ سے زیادہ چیز کی ضرورت ہوتی ہے۔ کم از کم مراعات والے اوزار استعمال کریں،
زیادہ اثر والی کارروائیوں کے لیے انسانی منظوری لیں، جہاں ضرورت ہو ڈیٹا کو چھپائیں، اور ایسی کارروائیوں کے لیے لاگز یا
رسیدیں رکھیں جن کا آڈٹ کرنا ضروری ہو۔

## 15 منٹ کی جائزہ کی روٹین

ہر سبق کے بعد اس روٹین کو استعمال کریں:

1. **سبق کا خلاصہ ایک جملے میں کریں۔**
2. **نئی ایجنٹ صلاحیت کا نام دیں۔** مثال کے طور پر: ٹول کا استعمال، بازیافت،
   منصوبہ بندی، یادداشت، معائنہ، یا سیکیورٹی۔
3. **اسے کورس ہیلپر ڈیمو میں شامل کریں۔** اب ڈیمو میں کیا تبدیلی آئی ہے؟
4. **خطرہ معلوم کریں۔** اگر یہ صلاحیت غلط استعمال ہو تو کیا غلط ہو سکتا ہے؟
5. **ایک ٹیسٹ سوال لکھیں۔** آپ کیسے چیک کریں گے کہ ایجنٹ اچھا برتاؤ کرتا ہے؟

## فوری خود جانچ

آگے بڑھنے سے پہلے، ان سوالات کے جواب دینے کی کوشش کریں:

1. ایجنٹ وہ کیا کر سکتا ہے جو ایک عام چیٹ بوٹ خود نہیں کر سکتا؟
2. آپ کے ایجنٹ کو سب سے پہلے کس آلے کی ضرورت ہوگی، اور کیوں؟
3. ایجنٹ کے جواب کی بنیاد کس ماخذ علم کو دینی چاہیے؟
4. اگلی ماڈل کال میں کون سا سیاق و سباق شامل ہونا چاہیے؟

5. ایجنٹ کو کیا یاد رکھنا چاہیے، اور اسے کیا ذخیرہ کرنے سے گریز کرنا چاہیے؟
6. ایجنٹ کو کب انسانی منظوری طلب کرنی چاہیے؟
7. کون سے لاگز، ٹریسز، یا رسیدیں آپ کو بعد میں ایجنٹ کی جانچ یا آڈٹ کرنے میں مدد دیں گی؟


## تجویز کردہ کیپ اسٹون مشق

کورس کے اختتام پر، ایک چھوٹا ایجنٹ بنائیں جو سیکھنے والے کو اس
ریپوزیٹری میں نیویگیٹ کرنے میں مدد دے۔

کم از کم ورژن:

- صارف سے ایک موضوع قبول کریں۔
- سب سے متعلقہ اسباق تلاش کریں۔
- پہلے کیا پڑھنا ہے کا خلاصہ فراہم کریں۔
- ایک عملی مشق کا کام تجویز کریں۔
- دکھائیں کہ کون سی سبق کی فائلیں یا روابط استعمال ہوئے۔

بڑھا ہوا ورژن:

- سیکھنے والے کی ترجیحی پروگرامنگ زبان یاد رکھیں۔
- جواب دینے سے پہلے ایک سادہ منصوبہ استعمال کریں۔
- آخری جواب سے پہلے ایک خود چیک کا مرحلہ شامل کریں۔
- ٹول کالز اور حاصل کردہ ذرائع کو لاگ کریں۔
- براؤزر یا UI خودکار کام کھولنے سے پہلے تصدیق طلب کریں۔

یہ آپ کو ایک چھوٹے مگر حقیقت پسندانہ طریقے سے ٹولز، RAG، منصوبہ بندی،
سیاق و سباق، یادداشت، مشاہدہ، اور اعتماد کو ایک منصوبے میں مشق کرنے کا موقع دیتا ہے۔
