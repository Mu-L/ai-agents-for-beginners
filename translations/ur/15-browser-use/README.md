# کمپیوٹر یوز ایجنٹس (CUA) کی تعمیر

کمپیوٹر یوز ایجنٹس ویب سائٹس کے ساتھ اسی طرح بات چیت کر سکتے ہیں جیسے ایک انسان کرتا ہے: براؤزر کھول کر، صفحہ کا جائزہ لے کر، اور جو وہ دیکھتے ہیں اس میں سے اگلا بہترین عمل انجام دے کر۔ اس سبق میں، آپ ایک براؤزر آٹومیشن ایجنٹ بنائیں گے جو Airbnb پر تلاش کرتا ہے، ساختہ لسٹنگ ڈیٹا نکالتا ہے، اور اسٹاک ہوم میں سب سے سستی رہائش کی نشاندہی کرتا ہے۔

یہ سبق AI سے چلنے والی نیویگیشن کے لیے Browser-Use، براؤزر کنٹرول کے لیے Playwright اور Chrome DevTools Protocol (CDP)، وژن سے لیس منطق کے لیے Azure OpenAI، اور ساختہ استخراج کے لیے Pydantic کو یکجا کرتا ہے۔

## تعارف

یہ سبق درج ذیل موضوعات کو شامل کرے گا:

- سمجھنا کہ کب کمپیوٹر یوز ایجنٹس API-صرف آٹومیشن سے بہتر ہیں
- قابل اعتماد براؤزر لائف سائیکل مینجمنٹ کے لیے Browser-Use کو Playwright اور CDP کے ساتھ یکجا کرنا
- متحرک ویب صفحات سے لسٹنگ ڈیٹا نکالنے کے لیے Azure OpenAI وژن اور ساختی Pydantic آؤٹ پٹ کا استعمال
- یہ فیصلہ کرنا کہ کب ایجنٹ-فرسٹ، ایکٹر-فرسٹ، یا ہائبرڈ براؤزر آٹومیشن ورک فلو استعمال کرنا چاہیے

## تعلیمی مقاصد

اس سبق کو مکمل کرنے کے بعد، آپ جانیں گے کہ کیسے:

- Azure OpenAI اور Playwright کے ساتھ Browser-Use کو ترتیب دینا
- ایک براؤزر آٹومیشن ورک فلو بنانا جو حقیقی ویب سائٹ پر نیویگیٹ کرے اور متحرک UI عناصر کو سنبھالے
- مرئی صفحے کے مواد سے ٹائپڈ نتائج نکال کر انہیں کاروباری منطق میں تبدیل کرنا
- براؤزر کے کام کی پیش گوئی کے مطابق ایجنٹ اور ایکٹر کے پیٹرنز میں انتخاب کرنا

## کوڈ مثال

اس سبق میں ایک نوٹ بک ٹیوٹوریل شامل ہے:

- [15-browser-user.ipynb](./15-browser-user.ipynb): CDP پر کروم سیشن شروع کرتا ہے، اسٹاک ہوم لسٹنگز کے لیے Airbnb پر تلاش کرتا ہے، Browser-Use وژن کے ساتھ قیمتیں نکالتا ہے، اور سب سے سستا انتخاب ساختہ ڈیٹا کے طور پر واپس کرتا ہے۔

## ضروریات

- پائتھون 3.12+  
- آپ کے ماحول میں Azure OpenAI کی تعیناتی کی گئی ہو
- لوکل طور پر کروم یا کرومیم انسٹال ہو
- Playwright کی Dependencies انسٹال ہوں
- async Python کی بنیادی واقفیت

## ترتیبات

نوٹ بک میں استعمال ہونے والے پیکجز انسٹال کریں:

```bash
pip install browser_use playwright python-dotenv
playwright install chromium
```

نوٹ بک میں استعمال ہونے والے Azure OpenAI ماحول کے متغیرات سیٹ کریں:

```bash
AZURE_OPENAI_ENDPOINT=...
AZURE_OPENAI_API_KEY=...
AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=...
# اختیاری: جب شامل نہ کیا جائے تو یہ جدید ترین API ورژن پر ڈیفالٹ ہوتا ہے
AZURE_OPENAI_API_VERSION=...
```

## فن تعمیر کا جائزہ

نوٹ بک ایک ہائبرڈ براؤزر آٹومیشن ورک فلو دکھاتی ہے:

1. کروم CDP کے ساتھ شروع ہوتا ہے تاکہ Playwright اور Browser-Use دونوں اسی براؤزر سیشن کو استعمال کر سکیں۔
2. ایک Browser-Use ایجنٹ کھلے نیویگیشن کے کام سنبھالتا ہے جیسے Airbnb کھولنا، پاپ اپس کو بند کرنا، اور اسٹاک ہوم میں تلاش کرنا۔
3. فعال صفحے کا جائزہ ایک ساختہ Pydantic اسکیمہ کے ساتھ لیا جاتا ہے تاکہ لسٹنگ کے عنوانات، راتانہ قیمتیں، ریٹنگز، اور URLs نکالے جا سکیں۔
4. پائتھون منطق نکالی گئی لسٹنگز کا موازنہ کرتی ہے اور سب سے سستی نتیجہ اجاگر کرتی ہے۔

یہ طریقہ Browser-Use کی لچکدار، وژن پر مبنی منطق کو برقرار رکھتا ہے جبکہ آپ کو جب ضرورت ہو تو براؤزر کنٹرول یقینی بناتا ہے۔

## کلیدی نکات اور بہترین طریقے

### کب ایجنٹ بمقابلہ ایکٹر استعمال کریں

| منظر نامہ | ایجنٹ استعمال کریں | ایکٹر استعمال کریں |
|----------|-----------------|---------------|
| متحرک لے آؤٹس | ہاں، AI صفحے کی تبدیلیوں کے مطابق ڈھل سکتا ہے | نہیں، نازک سلیکٹرز ٹوٹ سکتے ہیں |
| معلوم ساخت | نہیں، ایجنٹ براہ راست کنٹرول سے سست ہوتا ہے | ہاں، تیز اور دقیق |
| عناصر تلاش کرنا | ہاں، قدرتی زبان اچھا کام کرتی ہے | نہیں، درست سلیکٹرز ضروری ہیں |
| ٹائمنگ کنٹرول | نہیں، کم متوقع | ہاں، انتظار اور دوبارہ کوششوں پر مکمل کنٹرول |
| پیچیدہ ورک فلو | ہاں، غیر متوقع UI حالتوں کو سنبھالتا ہے | نہیں، واضح شاخ بندی کی ضرورت ہے |

### Browser-Use کے بہترین طریقے

1. جستجو اور متحرک نیویگیشن کے لئے ایجنٹ کے ساتھ شروع کریں۔
2. جب تعامل قابل پیش گوئی ہو جائے تو براہ راست صفحے کا کنٹرول سنبھالیں۔
3. ساختہ آؤٹ پٹ ماڈلز استعمال کریں تاکہ نکالا گیا ڈیٹا تصدیق شدہ اور ٹائپ محفوظ ہو۔
4. ایسے اعمال کے بعد معقول وقفے شامل کریں جو مرئی UI تبدیلیاں لاتے ہیں۔
5. دوران تکرار اسکرین شاٹس لیں تاکہ ناکامیاں آسانی سے ڈیبگ کی جا سکیں۔
6. توقع رکھیں کہ ویب سائٹس بدلیں گی اور پاپ اپس اور لے آؤٹ شفٹ کے لیے متبادل حکمت عملی تیار کریں۔
7. لچک اور درستگی دونوں کے لیے ایجنٹ اور ایکٹر پیٹرنز کو ملائیں۔

### براؤزر ایجنٹس کے لیے حفاظتی حدود

براؤزر ایجنٹس لائیو ویب سائٹس پر کام کرتے ہیں، لہٰذا انہیں اسکرپٹ سے سخت حدود کی ضرورت ہوتی ہے جو صرف معلوم API کو کال کرتا ہے۔ نوٹ بک ڈیمو سے حقیقی ورک فلو پر جانے سے قبل، اس بات کا تعین کریں کہ ایجنٹ کیا دیکھ سکتا ہے، کلک کر سکتا ہے، اور جمع کر سکتا ہے۔

1. **براؤزنگ ماحول کو محدود کریں۔** ایجنٹ کو مخصوص براؤزر پروفائل یا سینڈباکس میں چلائیں، اور اسے اس مخصوص ڈومین تک محدود رکھیں جو کام کے لیے ضروری ہو۔
2. **مشاہدے کو عمل سے الگ رکھیں۔** ایجنٹ کو پہلے تلاش، پڑھنے، اور ڈیٹا نکالنے دیں؛ فارم جمع کرانے، پیغامات بھیجنے، سفر بک کرنے، خریداری کرنے، ریکارڈ حذف کرنے، یا اکاؤنٹ کی ترتیبات تبدیل کرنے سے پہلے واضح اجازت کی ضرورت ہو۔
3. **خفیہ معلومات کو پرامپٹس اور ٹریسز سے باہر رکھیں۔** پاس ورڈز، ادائیگی کی تفصیلات، سیشن کوکیز، یا ذاتی ڈیٹا کو ماڈل کے مواد میں شامل نہ کریں۔ صارف کو توثیق کا اختیار دیں اور لاگز سے حساس فیلڈز حذف کریں۔
4. **صفحے کے مواد کو غیر معتبر ان پٹ سمجھیں۔** ایک ویب سائٹ ایسی ہدایات رکھ سکتی ہے جو ایجنٹ کے لیے ہوں، صارف کے لیے نہیں۔ ایجنٹ کو وہ صفحے کا متن نظر انداز کرنا چاہیے جو اس کے ہدف کو تبدیل کرنے، ڈیٹا ظاہر کرنے، حفاظتی اقدامات غیر فعال کرنے، یا غیر متعلقہ سائٹس پر جانے کو کہے۔
5. **خطرناک اقدامات کے اردگرد یقینی چیک استعمال کریں۔** موجودہ URL، صفحے کا عنوان، منتخب آئٹم، قیمت، وصول کنندہ، اور عمل کی تفصیل کو کوڈ کے ساتھ جانچیں، پھر صارف سے حتمی مرحلے کی منظوری لیں۔
6. **بجٹ اور روکنے کی شرائط مقرر کریں۔** ایجنٹ کی کارروائیوں، دوبارہ کوششوں، ٹیبز، اور منٹس کی تعداد محدود کریں۔ جب صفحے کی حالت مبہم ہو تو آگے کلک کرنے کے بجائے روک دیں۔
7. **صرف ضروری شواہد ریکارڈ کریں۔** کارروائی کے خلاصے، ٹائم اسٹیمپس، URLs، منتخب شدہ عنصر کی وضاحت، اور اسکرین شاٹ کے حوالے رکھیں تاکہ ناکامیاں جائزہ لینے کے لیے دستیاب ہوں بغیر غیر ضروری حساس مواد سٹور کیے۔

Airbnb کی مثال میں، محفوظ ڈیفالٹ یہ ہے کہ لسٹنگز تلاش کریں اور قیمتیں نکالیں۔ سائن ان ہونا، میزبان سے رابطہ کرنا، یا بکنگ مکمل کرنا علیحدہ صارف کی منظوری کا عمل ہونا چاہیے۔

### حقیقی دنیا کی ایپلیکیشنز

- سفر کی بکنگ اور قیمت مانیٹرنگ
- ای کامرس قیمت موازنہ اور دستیابی کی جانچ
- متحرک ویب سائٹس سے ساختی استخراج
- وژن سے لیس UI ٹیسٹنگ اور تصدیق
- ویب سائٹ مانیٹرنگ اور الرٹنگ
- ملٹی-اسٹپ فلو میں ذہین فارم بھرائی

## حقیقی دنیا کی مثال: Microsoft Project Opal

جو ایجنٹ آپ اس سبق میں بناتے ہیں وہ ایک چھوٹا، مقامی ورژن ہے **کمپیوٹر یوز ایجنٹ (CUA)** کا — ایسا پروگرام جو براؤزر کو انسان کی طرح چلاتا ہے۔ Microsoft اسی خیال کو کاروبار میں لا رہا ہے **[Project Opal (Frontier)](https://support.microsoft.com/en-us/microsoft-365-copilot/get-started-with-project-opal-frontier)** کے ذریعے، جو Microsoft 365 Copilot کی ایک خصوصیت ہے۔

Project Opal کے ساتھ، آپ ایک کام کی تفصیل دیتے ہیں اور ایجنٹ آپ کی جانب سے **کمپیوٹر یوز کو ایک محفوظ Windows 365 Cloud PC پر استعمال کرتے ہوئے** کام کرتا ہے، جو آپ کی تنظیم کی براؤزر پر مبنی ایپلیکیشنز، سائٹس، اور ڈیٹا میں عمل کرتا ہے۔ یہ **بیک گراونڈ میں غیر متزامن طور پر** کام کرتا ہے، اور آپ کبھی بھی کام کی رہنمائی یا کنٹرول لے سکتے ہیں۔ مثال کے طور پر کام شامل ہیں:

- سیکیورٹی گروپ ممبرشپ درخواستوں کا انتظام
- تعمیل کے جائزوں کے لیے آڈٹ شواہد اکٹھا اور تصدیق کرنا
- IT واقعات کا تجزیہ (ٹکٹ کی صورتحال کو اپ ڈیٹ کرنا، مالکان تفویض کرنا، نقلی کو بند کرنا)
- مالی بند کرنے کی دستاویز کے لیے Excel ڈیٹا مرتب کرنا

Opal ایک کارآمد حوالہ ہے کہ ایک **پروڈکشن-گریڈ، قابل اعتماد** کمپیوٹر یوز ایجنٹ کیسا ہوتا ہے — اور یہ پچھلے سبقوں کے تصورات کو مضبوط کرتا ہے:

| اس کورس کا تصور | Project Opal اس کو کیسے لاگو کرتا ہے |
|-------------------------|------------------------------|
| **Human-in-the-loop** (سبق 06) | Opal لاگ ان کی شناختی تفصیلات، حساس ڈیٹا، یا مبہم ہدایات پر رکتا ہے، اور بغیر واضح تصدیق کے کبھی پاس ورڈز داخل نہیں کرتا یا فارم جمع نہیں کرواتا۔ آپ کام کے دوران *کنٹرول لے* اور *واپس کر* سکتے ہیں۔ |
| **قابل اعتماد اور محفوظ ایجنٹس** (سبق 06 اور 18) | Windows 365 Cloud PC پر الگ تھلگ چلتا ہے، بنیادی طور پر براؤزر-صرف ہے (دیگر کمپیوٹر رسائی بلاک ہے، Intune کے ذریعے نافذ)، *آپ* کی شناخت استعمال کرتا ہے تاکہ صرف وہی رسائی ہو جس کی اجازت دی گئی ہے، اور ہر عمل کو آڈٹ کے لیے لاگ کرتا ہے۔ |
| **منصوبہ بندی اور خود ارادیت** (سبق 07 اور 09) | Opal پہلے کام کے لیے منصوبہ بناتا ہے، پھر ہر قدم پر اپنی منطق کی نگرانی کرتا ہے اور مشکوک سرگرمی پر رکتا ہے۔ |
| **دوبارہ قابل استعمال صلاحیتیں / اوزار** (سبق 04) | **Skills** آپ کو بار بار کے کاموں کے لیے ہدایات لکھنے دیتی ہیں (ایک `.md` فائل سے درآمد یا Opal کے ساتھ تیار کی گئی) اور انہیں گفتگوؤں میں دوبارہ استعمال کیا جا سکتا ہے۔ |

> **دستیابی:** Project Opal فی الحال صارفین کو [Frontier ایڈوانس پروگرام](https://adoption.microsoft.com/copilot/frontier-program/) میں Microsoft 365 Copilot کی سبسکرپشن کے ساتھ دستیاب ہے، اور آپ کے ایڈمنسٹریٹر کو سیٹ اپ مکمل کرنا ہوگا۔ چونکہ یہ ایک تجرباتی Frontier فیچر ہے، اس کی صلاحیتیں وقت کے ساتھ بدل سکتی ہیں۔
