# בדיקות סמוק של סוכן

תיקייה זו מכילה **קטלוגי בדיקות סמוק** עבור הסוכנים שאתם בונים במהלך הקורס.
בדיקת סמוק היא בדיקה זולה ומהירה לכך ש**סוכן מתארח שפורסם במיקרוסופט פאונדרי**
נגיש, מגיב, ועומד בציפיות הבסיסיות ביותר של ההנחיה שלו. זו שער ראשון — לא תחליף לצנרת
המלאה להערכה שאתם לומדים ב-[שיעור 10](../10-ai-agents-production/README.md) ו-
[שיעור 16](../16-deploying-scalable-agents/README.md).


הקטלוגים מנוצלים על ידי הפעולה [AI Smoke Test](https://github.com/marketplace/actions/ai-smoke-test)
של GitHub דרך תהליך העבודה [`.github/workflows/smoke-test.yml`](../../../.github/workflows/smoke-test.yml).


## איך להריץ

1. **פרסו את הסוכן מהשיעור** ב-Microsoft Foundry כסוכן מתארח (ראו את
   שיעור 16 עבור תהליך הפריסה). רשמו את **שם הסוכן** ואת
   **נקודת הקצה של פרויקט פאונדרי** שלכם.
2. הוסיפו סודות ריפוזיטורי אלה (הגדרות → סודות ומשתנים → פעולות):
   `AZURE_CLIENT_ID`, `AZURE_TENANT_ID`, `AZURE_SUBSCRIPTION_ID`. הזהות המאוחדת
   צריכה את תפקיד **משתמש Azure AI** בהיקף **פרויקט Foundry**.
3. בטאב **Actions**, הריצו את **Smoke-test hosted agents** ובחרו את
   `tests_file` לשיעור, ואז הזינו את `agent_name` ו-
   `project_endpoint` התואמים.

## אילו שיעורים כוללים בדיקות סמוק?

בדיקות סמוק חלות על שיעורים שבהם אתם **מפרסמים סוכן** שהתשובות הטקסטואליות שלו יכולות
להיבדק מול תוכן ידוע. שיעורים שהם קונספטואליים, רצים רק במחשב מקומי,
או מייצרים פלט יצירתי בלתי דטרמיניסטי – מוצאים במכוון מן הכלל:

- **שיעור 17 (יצירת סוכני AI מקומיים)** רץ כולו על תחנת העבודה עם
  Foundry Local ואינו חושף נקודת קצה Foundry Responses, לכן פעולה זו אינה רלוונטית.
  אמתו אותו על ידי הרצת המחברת מקומית.
- שיעורי תבניות-עיצוב ותיאוריה (02, 03, 06, 07, 09, 12) אינם מפרסמים סוכן אחד
  שניתן לבדוק באמצעות בדיקת סמוק.

## סכמת קטלוג (עיון מהיר)

כל קטלוג הוא מסמך JSON עם מערך `tests` ברמה העליונה. כל רשומה שולחת
הנחיה אחת ובודקת את התשובה:

| שדה | משמעות |
|-------|---------|
| `id` | מזהה שלב ייחודי שמודפס ביומן. |
| `description` | מטרה קריאה לאדם. |
| `prompt` | ההודעה שנשלחת לסוכן. |
| `assertions.status` | סטטוס HTTP צפוי (ברירת מחדל 200). |
| `assertions.contains_any` | מצליח אם התשובה מכילה כל אחת מתת-המחרוזות האלה. |
| `assertions.contains_all` | מצליח אם התשובה מכילה כל אחת מתת-המחרוזות. |
| `assertions.contains_none` | מצליח אם התשובה אינה מכילה אף אחת מתת-המחרוזות האלה. |
| `save_response_id_as` | שומר את מזהה התשובה לשלב רב-סיבובי מאוחר יותר. |
| `use_previous_response_id` | שולח את הסיבוב הזה כשקשור למזהה תשובה שמור. |

בדיקות ההנחות הן בדיקות תת-מחרוזת שאינן תלויות באותיות רישיות. ראו את
[תיעוד הפעולה](https://github.com/marketplace/actions/ai-smoke-test) עבור
הסכמה המלאה, כולל משאבים לשיחה המנוהלים על ידי Foundry.
