# ⚡ זרמי עבודה מנוהלים מקביליים עם Azure OpenAI (Responses API) (.NET)

## 📋 מדריך לעיבוד מקבילי בעל ביצועים גבוהים

פנקס רשימות זה מציג **תבניות זרימת עבודה מקבילות** באמצעות Microsoft Agent Framework עבור .NET ו-Azure OpenAI (Responses API). תלמד כיצד לבנות זרמי עבודה לעיבוד מקבילי מהיר שממקסמים את התפוקה על ידי הפעלת סוכני AI מרובים בו זמנית תוך שמירה על תיאום ועקביות בנתונים.

## 🎯 מטרות הלמידה

### 🚀 **יסודות העיבוד המקבילי**
- **הרצת סוכנים במקביל**: הפעל סוכני AI מרובים בו זמנית לביצועים מקסימליים

- **ניהול משאבים**: נהל ביעילות את משאבי מודל ה-AI בין פעולות מקבילות

### 🏢 **יישומים ארגוניים מקבילים**

## 💻 הפעלת הקוד

המימוש המלא זמין ב-`03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. קובץ זה מדגים **זרימת עבודה מקבילה בסגנון Fan-Out/Fan-In** לתכנון טיולים:

### 🏗️ **ארכיטקטורת זרימת העבודה**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**רכיבים מרכזיים:**

1. **ConcurrentStartExecutor**: משדר את בקשת המשתמש לכל הסוכנים בו זמנית
2. **סוכן חוקר**: מנתח יעדים ואטרקציות במקביל
3. **סוכן מתכנן**: יוצר תוכניות טיול מפורטות במקביל
4. **ConcurrentAggregationExecutor**: אוסף וממזג תוצאות משני הסוכנים

### 🎯 **תבנית Fan-Out/Fan-In**

זרימת עבודה זו מציגה את תבנית ה-**Fan-Out/Fan-In** הקלאסית:
- **Fan-Out**: הודעה אחת משודרת למספר סוכנים בו זמנית
- **עיבוד מקבילי**: מספר סוכנים עובדים במקביל על אותה משימה
- **Fan-In**: תוצאות מכל הסוכנים נאספות ומאוחדות לפלט יחיד

### 🔍 הבנת Custom Executors

**ConcurrentStartExecutor:**
- מיישם `IMessageHandler<string>` לקבלת קלט מחרוזת
- משדר הודעות לכל הסוכנים המחוברים
- שולח `TurnToken` כדי להפעיל עיבוד מקבילי

**ConcurrentAggregationExecutor:**
- מיישם `IMessageHandler<ChatMessage>` לקבלת תגובות סוכן
- אוסף הודעות באופן בטוח להרצה במקביל
- אוגר כאשר כל התגובות הצפויות מגיעות
- מחזיר פלט סופי באמצעות `context.YieldOutputAsync()`

### ⚡ יתרונות ביצועים

**מקבילי לעומת סדרתי:**
- סדרתי: סוכן1 (30 שניות) → סוכן2 (30 שניות) = **60 שניות בסה"כ**
- מקבילי: סוכן1 (30 שניות) || סוכן2 (30 שניות) = **30 שניות בסה"כ**

**שיפור בתפוקה**: עד N× מהיר יותר עבור N סוכנים מקבילים (בהתאם לעומס ולמשאבים)
