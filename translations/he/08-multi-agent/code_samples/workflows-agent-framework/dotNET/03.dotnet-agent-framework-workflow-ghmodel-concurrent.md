# ⚡ זרימות עבודה מקבילות עם Azure OpenAI (Responses API) (.NET)

## 📋 מדריך לעיבוד מקבילי בעל ביצועים גבוהים

פנקס זה מדגים **תבניות זרימות עבודה מקביליות** באמצעות Microsoft Agent Framework ל-.NET ו-Azure OpenAI (Responses API). תלמד כיצד לבנות זרימות עבודה מקבילות בעלות ביצועים גבוהים שממקסמות תפוקה באמצעות הפעלת סוכני AI מרובים בו-זמנית תוך שמירה על תיאום ועקביות נתונים.

## 💻 הרצת הקוד

מימוש מלא זמין בקובץ `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. קובץ זה מדגים **זרימת עבודה מקבילה מסוג Fan-Out/Fan-In** לתכנון נסיעות:

### 🏗️ **ארכיטקטורת זרימת העבודה**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**רכיבים מרכזיים:**

1. **ConcurrentStartExecutor**: משדר את בקשת המשתמש לכל הסוכנים בו-זמנית
2. **סוכן חוקר**: מתבצע ניתוח מקבילי של יעדים ואטרקציות
3. **סוכן מתכנן**: יוצר תוכניות נסיעות מפורטות במקביל
4. **ConcurrentAggregationExecutor**: אוסף ומאגד תוצאות משני הסוכנים

### 🎯 **תבנית Fan-Out/Fan-In**

זרימת עבודה זו מדגימה את תבנית **Fan-Out/Fan-In** הקלאסית:
- **Fan-Out**: הודעת קלט אחת משודרת למספר סוכנים בו-זמנית
- **עיבוד מקבילי**: סוכנים מרובים עובדים במקביל על אותה מטלה
- **Fan-In**: התוצאות מכל הסוכנים נאספות ומאוגדות לפלט יחיד

    .AddFanInEdge(aggregationExecutor, sources: [researcherAgent, plannerAgent, budgetAgent])
    .WithOutputFrom(aggregationExecutor)
    .Build();

### 🔍 הבנת מבצעים מותאמים אישית

**ConcurrentStartExecutor:**
- מממש `IMessageHandler<string>` לקבלת קלט מסוג מחרוזת
- משדר הודעות לכל הסוכנים המחוברים
- שולח `TurnToken` כדי לגרום לעיבוד מקבילי להתחיל

**ConcurrentAggregationExecutor:**
- מממש `IMessageHandler<ChatMessage>` לקבלת תגובות סוכנים
- אוסף הודעות בצורה בטוחה לתהליכים
- מבצע אגרגציה כשכל התגובות הצפויות מגיעות
- מחזיר פלט סופי באמצעות `context.YieldOutputAsync()`

### ⚡ יתרונות ביצועים

**מקבילי לעומת סדרתי:**
- סדרתי: סוכן1 (30 שניות) → סוכן2 (30 שניות) = **סה"כ 60 שניות**
- מקבילי: סוכן1 (30 שניות) || סוכן2 (30 שניות) = **סה"כ 30 שניות**

**שיפור בתפוקה**: עד N פעמים מהיר יותר עבור N סוכנים מקבילים (בהתאם לעומס עבודה ומשאבים)
