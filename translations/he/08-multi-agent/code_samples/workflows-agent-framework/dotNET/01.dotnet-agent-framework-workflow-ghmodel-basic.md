# 🔄 זרימות עבודה בסיסיות של סוכן עם Azure OpenAI (Responses API) (.NET)

## 📋 מדריך לאורקסטרציית זרימות עבודה

פנקס עבודה זה מדגים כיצד לבנות **זרימות עבודה מתקדמות של סוכן** באמצעות Microsoft Agent Framework עבור .NET ו-Azure OpenAI (Responses API). תלמדו ליצור תהליכים עסקיים מרובי שלבים שבהם סוכני AI משתפים פעולה כדי להשלים משימות מורכבות באמצעות דפוסי אורקסטרציה מובנים.

## 🎯 מטרות הלמידה

### 🏗️ **יסודות ארכיטקטורת זרימות עבודה**
- **בונה זרימות עבודה**: תכנון ואורקסטרציה של תהליכי AI מורכבים מרובי שלבים
- **קואורדינציית סוכנים**: תיאום בין סוכנים מיוחדים מרובים בזרימות עבודה
- **Azure OpenAI (Responses API)**: שימוש ב-Azure OpenAI Responses API בזרימות עבודה
- **עיצוב ויזואלי של הזרימה**: יצירה והמחשה של מבני זרימות עבודה להבנה טובה יותר

### 🔄 **דפוסי אורקסטרציה של תהליכים**
- **עיבוד סדרתי**: שרשור מספר משימות סוכן בסדר לוגי
- **ניהול מצב**: שמירת הקשר וזרימת נתונים בין שלבי הזרימה
- **טיפול בשגיאות**: יישום התאוששות שגיאות חסינה ועמידות בזרימות עבודה
- **אופטימיזציית ביצועים**: עיצוב זרימות עבודה יעילות לפעילות בקנה מידה ארגוני

### 🏢 **יישומי זרימות עבודה ארגוניות**
- **אוטומציה של תהליכים עסקיים**: אוטומציה של זרימות עבודה ארגוניות מורכבות
- **צינור הפקת תוכן**: זרימות עבודה עריכתיות עם שלבי סקירה ואישור
- **אוטומציה של שירות לקוחות**: פתרון פניות לקוחות מרובי שלבים
- **זרימות עבודה לעיבוד נתונים**: זרימות ETL עם טרנספורמציה מונעת AI

## ⚙️ דרישות מוקדמות והגדרה

### 📦 **חבילות NuGet דרושות**

הדגמת זרימת העבודה הזו משתמשת בכמה חבילות מפתח של .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **הגדרת Azure OpenAI**

**הגדרת סביבה (קובץ .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**גישה ל-Azure OpenAI:**
1. צור משאב Azure OpenAI בפורטל Azure
2. פרוס דגם (לדוגמה, `gpt-5-mini`) ורשום את שם הפריסה
3. היכנס עם `az login` והגדר את משתני הסביבה כפי שמוצג למעלה

### 🏗️ **סקירת ארכיטקטורת זרימות עבודה**

```mermaid
graph TD
    A[בונה זרימת עבודה] --> B[רישום סוכנים]
    B --> C[מנוע ביצוע זרימות עבודה]
    C --> D[Agent 1: יוצר תוכן]
    C --> E[Agent 2: בודק תוכן] 
    D --> F[תוצאות זרימת עבודה]
    E --> F
    G[Azure OpenAI (ממשק API לתשובות)] --> D
    G --> E
```

**רכיבים מרכזיים:**
- **WorkflowBuilder**: מנוע האורקסטרציה הראשי לתכנון זרימות עבודה
- **AIAgent**: סוכנים יחידים ומיוחדים עם יכולות ספציפיות
- **Azure OpenAI Client**: אינטגרציית Azure OpenAI Responses API
- **Execution Context**: מנהל מצב וזרימת נתונים בין שלבי הזרימה

## 💻 הרצת הקוד

המימוש המלא זמין בקובץ `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. קובץ זה מדגים:

1. **הגדרת סביבה** - טעינת הגדרות Azure OpenAI מקובץ `.env`
2. **הגדרת לקוח Azure OpenAI** - קביעת תצורת הלקוח לשימוש ב-Azure OpenAI Responses API
3. **יצירת סוכן** - הגדרת סוכנים מיוחדים (דלפק הקבלה וקונסיירז')
4. **בונה זרימות עבודה** - יצירת זרימת עבודה עם מספר סוכנים ועיבוד סדרתי
5. **הפעלה זרימת עבודה** - הרצת הזרימה עם תוצאות בשידור חי
