# ⏩ זרימות עבודה סידוריות של סוכנים עם Azure OpenAI (Responses API) (.NET)

## 📋 מדריך מתקדם לעיבוד סידורי

מחברת זו מדגימה **תבניות זרימת עבודה סידוריות** באמצעות Microsoft Agent Framework ל-.NET ו-Azure OpenAI (Responses API). תלמד כיצד לבנות צינורות עיבוד מתוחכמים שלב אחר שלב שבהם סוכנים מבצעים בסדר מסוים, כאשר כל שלב בונה על התוצאות של השלב הקודם.

## 🎯 מטרות למידה

### 🔄 **ארכיטקטורת עיבוד סידורי**
- **עיצוב זרימת עבודה לינארית**: יצירת צינורות עיבוד שלב אחר שלב עם תלות ברורה
- **ניהול מצב**: שמירה על הקשר וזרימת נתונים לאורך שלבי הזרימה הסידורית
- **Azure OpenAI (Responses API)**: ניצול דגמי Azure OpenAI בזרימות עבודה מרובות שלבים ב-.NET
- **תבניות צינורות ארגוניות**: בניית מערכות עיבוד סידוריות מוכנות לייצור

### 🏗️ **תבניות סידוריות מתקדמות**
- **עיבוד בשערי שלב**: יישום נקודות אימות בין שלבי זרימת העבודה
- **שימור הקשר**: שמירה על מצב וידע מצטבר לאורך כל השלבים
- **הפצת שגיאות**: טיפול בכשלונות באופן תקין בשרשראות עיבוד סידוריות
- **אופטימיזציית ביצועים**: ביצוע סידורי יעיל עם עומס מינימלי

### 🏢 **יישומי סדר ארגוניים**
- **צינור עיבוד מסמכים**: ניתוח, טרנספורמציה ואימות מסמכים מרובי שלבים
- **זרימות עבודה של איכות**: סקירה, אימות ואישור סדירים
- **צינור הפקת תוכן**: מחקר → כתיבה → עריכה → סקירה → פרסום
- **אוטומציה של תהליכים עסקיים**: זרימות עבודה עסקיות מרובות שלבים עם תלות שלבים ברורה

## ⚙️ דרישות מוקדמות והגדרה

### 📦 **חבילות NuGet נדרשות**

חבילות חיוניות לזרימות עבודה סידוריות ב-.NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **הגדרת Azure OpenAI**

**הגדרת סביבה (קובץ .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**ניהול הגדרות:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **ארכיטקטורת זרימת עבודה סידורית**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: סוכן ניתוח]
    B --> C[נקודת בדיקה 1]
    C --> D[Stage 2: סוכן עיבוד]
    D --> E[נקודת בדיקה 2]
    E --> F[Stage 3: סוכן אימות]
    F --> G[פלט סופי]
    
    H[הקשר מצב] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (ממשק תגובות)] --> B
    I --> D
    I --> F
```

**רכיבי מפתח:**
- **סוכנים סידוריים**: סוכנים מיוחדים לכל שלב עיבוד
- **הקשר מצב**: שומר על נתונים והחלטות מצטברים לאורך השלבים
- **נקודות ביקורת**: נקודות אימות בין שלבים להבטחת איכות ועקביות
- **לקוח Azure OpenAI**: גישה עקבית למודל AI בכל שלבי הזרימה

## 💻 הרצת הקוד

המימוש המלא זמין בקובץ `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. קובץ זה מדגים **זרימת עבודה לשלושה שלבים לניתוח רהיטים**:

1. **שלב 1 - סוכן מכירות**: מנתח תמונות רהיטים ומספק המלצות רכישה
2. **שלב 2 - סוכן מחיר**: מספק פירוטי תמחור ואפשרויות תקציב
3. **שלב 3 - סוכן הצעות מחיר**: יוצר מסמך הצעת מחיר מקצועי בפורמט Markdown

### 🏗️ **ארכיטקטורת זרימת עבודה**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

- מקבל את הפלט מהשלב הקודם כהקשר
- בונה על ניתוח קודם עם מומחיות ממוקדת
- שומר על רצף זרימת העבודה באמצעות ניהול מצב

### 📝 פלט צפוי

זרימת העבודה תבצע:
1. **סוכן מכירות**: זיהוי פריטי רהיטים מהתמונה ומתן המלצות
2. **סוכן מחיר**: הוספת ניתוח תמחור מפורט עם רמות תקציב והמלצות קנייה
3. **סוכן הצעות מחיר**: יצירת מסמך הצעה מעוצב עם כל המידע המשולב

הפלט הסופי יהיה הצעת מחיר מקיפה ומקצועית לרהיטים מבוססת על ניתוח התמונה.

### 🔧 אפשרויות התאמה אישית

**שינוי התנהגות הסוכן:**
```csharp
// Adjust agent instructions to change their focus
const string SalesAgentInstructions = "Your custom instructions...";
```

**שינוי זרימה סידורית:**
```csharp
// Add or reorder workflow stages
var workflow = new WorkflowBuilder(salesagent)
    .AddEdge(salesagent, priceagent)
    .AddEdge(priceagent, quoteagent)
    .AddEdge(quoteagent, newAgent)  // Add another stage
    .Build();
```

**שימוש בקלט שונה:**
