# ⏩ זרימות עבודה סדרתיות עם Azure OpenAI (Responses API) (.NET)

## 📋 מדריך לעיבוד סדרתי מתקדם

פנקס הערות זה מדגים **תבניות זרימות עבודה סדרתיות** באמצעות Microsoft Agent Framework עבור .NET ו-Azure OpenAI (Responses API). תלמד איך לבנות צנרת עיבוד מתוחכמת שלב אחר שלב, שבה סוכנים מבצעים פעולה בסדר מסוים, כאשר כל שלב בנוי על תוצאות השלב הקודם.

## 🎯 מטרות הלמידה

### 🔄 **ארכיטקטורת עיבוד סדרתי**
- **עיצוב זרימת עבודה ליניארית**: יצירת צנרות עיבוד שלב אחר שלב עם תלות ברורה
- **ניהול מצב**: שמירת הקשר וזרימת נתונים לאורך שלבי זרימת העבודה הסדרתית
- **Azure OpenAI (Responses API)**: שימוש במודלים של Azure OpenAI בזרימות עבודה מרובות שלבים ב-.NET
- **תבניות צנרת ארגוניות**: בניית מערכות עיבוד סדרתיות מוכנות לייצור

### 🏗️ **תבניות סדרתיות מתקדמות**
- **עיבוד מסוג Stage-Gate**: יישום נקודות בדיקה בין שלבי זרימת העבודה
- **שימור הקשר**: שמירת המצב והידע הצטבר לאורך כל השלבים
- **הפצת שגיאות**: טיפול בכשלונות בצורה חלקה בשרשראות עיבוד סדרתיות
- **אופטימיזציה של ביצועים**: ביצוע סדרתי יעיל עם מינימום עומס

### 🏢 **יישומים סדרתיים ארגוניים**
- **צנרת עיבוד מסמכים**: ניתוח, המרה ואימות של מסמכים במספר שלבים
- **זרימות עבודה לאבטחת איכות**: סקירה, אימות ואישור סדרתי
- **צנרת יצירת תוכן**: מחקר → כתיבה → עריכה → סקירה → פרסום
- **אוטומציה של תהליכים עסקיים**: זרימות עבודה עסקיות רב-שלביות עם תלות ברורה בין השלבים

## ⚙️ דרישות מקדימות והגדרה

### 📦 **חבילות NuGet נדרשות**

חבילות חיוניות לזרימות עבודה סדרתיות ב-.NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **הגדרת Azure OpenAI**

**הגדרת סביבה (קובץ .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**ניהול הגדרות:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **ארכיטקטורת זרימת עבודה סדרתית**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: סוכן ניתוח]
    B --> C[נקודת בדיקה 1]
    C --> D[Stage 2: סוכן עיבוד]
    D --> E[נקודת בדיקה 2]
    E --> F[Stage 3: סוכן אימות]
    F --> G[פלט סופי]
    
    H[הקשר מצב] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (ממשק תגובות)] --> B
    I --> D
    I --> F
```

**רכיבים מרכזיים:**
- **סוכנים סדרתיים**: סוכנים מתמחים לכל שלב בעיבוד
- **הקשר מצב**: שומר נתונים והחלטות מצטברים לאורך השלבים
- **נקודות בדיקה**: נקודות אימות בין השלבים להבטחת איכות ועקביות
- **לקוח Azure OpenAI**: גישה עקבית למודל ה-AI לאורך כל שלבי זרימת העבודה

// Add or reorder workflow stages
var workflow = new WorkflowBuilder(salesagent)
