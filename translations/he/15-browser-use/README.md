# בניית סוכנים לשימוש מחשב (CUA)

סוכני שימוש מחשב יכולים לתקשר עם אתרי אינטרנט באותה צורה שאדם היה עושה זאת: על ידי פתיחת דפדפן, בדיקת הדף, ולקיחת הפעולה הטובה הבאה ממה שהם רואים. בשיעור זה, תבנה סוכן אוטומציה לדפדפן המחפש ב-Airbnb, מחלץ נתוני רשימות מובנים, ומזהה את השהייה הזולה ביותר בסטוקהולם.

השיעור משלב Browser-Use לניווט מונחה AI, Playwright ופרוטוקול Chrome DevTools (CDP) לשליטה בדפדפן, Azure OpenAI להסקת דעות מבוססת ראייה, ו-Pydantic לחליצה מובנית.

## מבוא

בשיעור זה יכוסו:

- הבנת מתי סוכני שימוש מחשב מתאימים יותר מאוטומציה המבוססת רק על API
- שילוב Browser-Use עם Playwright ו-CDP לניהול אמין של מחזור חיי הדפדפן
- שימוש בראיית Azure OpenAI ופלט מובנה של Pydantic לחלוץ נתוני רשימות מדפי אינטרנט דינמיים
- קבלת החלטה מתי להשתמש בסוכן-ראשון, שחקן-ראשון או תהליך אוטומציה היברידי של דפדפן

## מטרות למידה

לאחר השלמת שיעור זה, תדע כיצד:

- להגדיר Browser-Use עם Azure OpenAI ו-Playwright
- לבנות תהליך אוטומציה בדפדפן הניווט באתר אמיתי ומתמודדת עם אלמנטים דינמיים בממשק
- לחלץ תוצאות טיפוסיות מתוכן הדף הנראה ולהפכן ללוגיקה עסקית בהמשך
- לבחור בין דפוסי סוכן ושחקן בהתאם לחיזוי המשימה בדפדפן

## דוגמת קוד

בשיעור זה יש מדריך פנקס אחד:

- [15-browser-user.ipynb](./15-browser-user.ipynb): מפעיל סשן Chrome דרך CDP, מחפש ב-Airbnb רשימות בסטוקהולם, מחלץ מחירים עם ראיית Browser-Use, ומחזיר את האפשרות הזולה ביותר כנתונים מובנים.

## דרישות מוקדמות

- Python 3.12+
- פריסת Azure OpenAI מוגדרת בסביבתך
- Chrome או Chromium מותקן מקומית
- תלות ב-Playwright מותקנת
- היכרות בסיסית עם Python אסינכרוני

## הגדרה

התקן את החבילות שמשמשות בפנקס:

```bash
pip install browser_use playwright python-dotenv
playwright install chromium
```

הגדר את משתני הסביבה של Azure OpenAI שמשמשים בפנקס:

```bash
AZURE_OPENAI_ENDPOINT=...
AZURE_OPENAI_API_KEY=...
AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=...
# אפשרי: ברירת מחדל לגרסת ה-API העדכנית ביותר כאשר לא מצוין
AZURE_OPENAI_API_VERSION=...
```

## סקירת ארכיטקטורה

הפנקס מדגים תהליך אוטומציה היברידי של דפדפן:

1. Chrome מתחיל עם CDP מופעל כך שגם Playwright וגם Browser-Use יכולים לשתף את אותה סשן דפדפן.
2. סוכן Browser-Use מטפל במשימות ניווט פתוחות כמו פתיחת Airbnb, סגירת חלונות קופצים וחיפוש אחר סטוקהולם.
3. הדף הפעיל נבדק עם סכמת Pydantic מובנית כדי לחלץ כותרות רשימות, מחירי לילה, דירוגים וקישורים.
4. לוגיקת Python משווה בין הרשימות החולצו ומדגישה את התוצאה הזולה ביותר.

גישה זו שומרת על הסקת דעות גמישה מבוססת ראייה ש-Browser-Use מתמחה בה, ומעניקה לך שליטה דטרמיניסטית בדפדפן כאשר אתה זקוק לה.

## מסקנות עיקריות ופרקטיקות מיטביות

### מתי להשתמש בסוכן לעומת שחקן

| תרחיש | להשתמש בסוכן | להשתמש בשחקן |
|----------|-----------|-----------|
| פריסות דינמיות | כן, AI יכול להתאים לשינויים בדף | לא, בוחרים שבירים עלולים להישבר |
| מבנה ידוע | לא, סוכן איטי יותר מבקרה ישירה | כן, מהיר ומדויק |
| מציאת אלמנטים | כן, שפה טבעית עובדת טוב | לא, נדרשים בוחרים מדויקים |
| שליטה בזמנים | לא, פחות צפוי | כן, שליטה מלאה על המתנות וניסיונות חוזרים |
| תהליכים מורכבים | כן, מטפל במצבים בלתי צפויים בממשק המשתמש | לא, דורש הסתעפויות מפורשות |

### פרקטיקות מיטביות ל-Browser-Use

1. התחל עם סוכן לחקירה וניווט דינמי.
2. עבור לבקרה ישירה של הדף כשהאינטראקציה הופכת לניבויה.
3. השתמש במודלים לפלט מובנה כדי שהנתונים החולצים ייבדקו ויהיו בטוחים טיפוסית.
4. הוסף השהיות באופן אסטרטגי לאחר פעולות שמפעילות שינויים גלויים בממשק המשתמש.
5. תעד צילומי מסך בזמן איטרציה כדי לייעל איתור שגיאות.
6. צפה שהאתרים משתנים ועצב אסטרטגיות גיבוי לחלונות קופצים ושינויים בפריסות.
7. שלב דפוסי סוכן ושחקן לקבלת גמישות ודיוק גם יחד.

### קווים מנחים לבטיחות לסוכני דפדפן

סוכני דפדפן פועלים באינטרנט חי, ולכן הם זקוקים לגבולות הדוקים יותר מאשר סקריפט שמפעיל רק API ידוע. לפני המעבר מהדגמת פנקס לתהליך עובד אמיתי, הגדר את הבקרות סביב מה שהסוכן יכול לראות, ללחוץ ולהגיש.

1. **הגדר את סביבת הגלישה.** הרץ את הסוכן בפרופיל דפדפן ייעודי או בסנדבוקס, והגבילו לדומיינים הנדרשים למשימה.
2. **הפרד התבוננות מפעולה.** אפשר לסוכן לחפש, לקרוא ולחלץ נתונים תחילה; דרוש אישור מפורש לפני שהוא מגיש טפסים, שולח הודעות, מבצע הזמנות, מבצע רכישות, מוחק רשומות או משנה הגדרות חשבון.
3. **שמור סודות מחוץ לטריגרים ולעקבות.** אל תכניס סיסמאות, פרטי תשלום, קוקיז של מושב, או נתונים אישיים גלויים להקשר המודל. אפשר למשתמש לקחת על עצמו את תהליך האימות ולמחוק שדות רגישים מלוגים.
4. **טפל בתוכן הדף כקלט לא מהימן.** אתר אינטרנט יכול להכיל הנחיות המיועדות לסוכן, לא למשתמש. הסוכן צריך להתעלם מטקסט הדף שמבקש לשנות את מטרתו, לחשוף נתונים, לבטל אמצעי הגנה או לבקר באתרים לא קשורים.
5. **השתמש בבדיקות דטרמיניסטיות סביב צעדים מסוכנים.** אשר את ה-URL הנוכחי, כותרת הדף, הפריט הנבחר, המחיר, הנמען, ותמצית הפעולה עם קוד לפני בקשת אישור מהמשתמש על השלב הסופי.
6. **הגדר תקציבים ותנאי עצירה.** הגבל את מספר הפעולות, הניסיונות החוזרים, הכרטיסיות, והדקות שהסוכן יכול להשתמש בהן. עצור כאשר מצב הדף לא חד-משמעי במקום להמשיך ללחוץ.
7. **תעד ראיות שימושיות, לא הכל.** שמור תמציות פעולה, חותמות זמן, URLים, תיאורי האלמנטים שנבחרו, והפניות לצילומי מסך כדי שניתן יהיה לבדוק כשלים בלי לאחסן תוכן דף רגיש מיותר.

בדוגמת Airbnb, ברירת המחדל הבטוחה היא לחפש רשימות ולחלץ מחירים. כניסה, יצירת קשר עם מארח, או השלמת הזמנה צריכים להיות פעולה נפרדת המאושרת על ידי המשתמש.

### יישומים בעולם האמיתי

- הזמנת טיולים ומעקב מחירים
- השוואת מחירים זמינות במסחר אלקטרוני
- חילוץ מובנה מאתרים דינמיים
- בדיקות ומבחני ממשק מבוססי ראייה
- ניטור אתרים והתראות
- מילוי טפסים חכם בזרימות רב-שלביות

## דוגמה מהעולם האמיתי: Microsoft Project Opal

הסוכן שתבנה בשיעור זה הוא גרסה קטנה ומקומית של **סוכן שימוש מחשב (CUA)** — תוכנה שמפעילה דפדפן כמו אדם. מיקרוסופט מביאה את הרעיון הזה לארגונים עם **[Project Opal (Frontier)](https://support.microsoft.com/en-us/microsoft-365-copilot/get-started-with-project-opal-frontier)**, יכולת במסגרת Microsoft 365 Copilot.

עם Project Opal, אתה מתאר משימה והסוכן פועל מטעמך באמצעות **שימוש מחשב ב-Windows 365 Cloud PC מאובטח**, הפועל על פני יישומי, אתרי אינטרנט ונתונים מבוססי דפדפן בארגון שלך. הוא פועל **באופן אסינכרוני ברקע**, ואתה יכול לכוון את העבודה או לקחת שליטה בכל רגע. דוגמאות לתפקידים כוללות:

- ניהול בקשות חברות בקבוצות אבטחה
- איסוף ואימות ראיות ביקורת לסקירות תאימות
- מיון תקריות IT (עדכון סטטוס כרטיסים, הקצאת בעלים, סגירת כפילויות)
- עריכת נתוני Excel ללוח סגירה פיננסי

Opal הוא רפרנס שימושי לסוכן שימוש מחשב **אמין ורמת ייצור** — ומחזק מושגים משיעורים קודמים:

| מושג בקורס זה | איך Project Opal מיישם אותו |
|------------------------|-----------------------------|
| **אדם בלולאה** (שיעור 06) | Opal עוצר לקבלת אישורי כניסה, נתונים רגישים או הוראות עמומות, ואף פעם לא מזין סיסמאות או מגיש טפסים בלי אישור מפורש. אפשר *לקחת שליטה* ו*להחזיר שליטה* באמצע המשימה. |
| **סוכנים אמינים ומאובטחים** (שיעורים 06 ו-18) | פועל ב-Windows 365 Cloud PC מבודד, משרת דפדפן בלבד כבררת מחדל (גישה אחרת למחשב חסומה, מאוכפת דרך Intune), משתמש ב*זהות שלך* ולכן ניגש רק למה שאתה מורשה לו, ומתעד כל פעולה לצורך ביקורת. |
| **תכנון ומטא-קוגניציה** (שיעורים 07 ו-09) | Opal מייצר תוכנית למשימה תחילה, ואז מפקח על ההסקה העצמית שלו בכל שלב ועוצר אם מזהה פעילות חשודה. |
| **יכולות / כלים לשימוש חוזר** (שיעור 04) | **מיומנויות** מאפשרות לך לכתוב הוראות למשימות חוזרות (מיובאות מקובץ `.md` או נכתבות עם Opal) ולשחזר אותן בשיחות. |

> **זמינות:** Project Opal זמין כעת למשתמשים בתוכנית גישה מוקדמת [Frontier](https://adoption.microsoft.com/copilot/frontier-program/) עם מנוי Microsoft 365 Copilot, ומנהל המערכת שלך צריך להשלים את ההגדרה. מכיוון שמדובר בתכונת Frontier ניסיונית, היכולות עשויות להשתנות עם הזמן.
