# סוכני AI למתחילים - מדריך ללימוד

השתמש במדריך זה כמשלים מעשי בזמן ההתקדמות שלך בקורס. הוא
אינו מיועד להחליף את השיעורים. הוא עוזר לך להחליט מאיפה להתחיל, מה
לחפש בכל שיעור, ואיך לחבר את הרעיונות יחד לדמו קטן של סוכן עובד.


אם זו הפעם הראשונה שלך כאן, התחל פשוט:

1. קרא את [הכנת הקורס](./00-course-setup/README.md).
2. השלם את השיעורים 01-06 לפי הסדר.
3. שמור במחשבה רעיון דמו קטן בזמן הלמידה.
4. לאחר כל שיעור, שאל: "מה הסוכן שלי יכול לעשות עכשיו שלא היה יכול
   לפני כן?"

## דמו פשוט לשמור בזכרון

דרך טובה ללמוד על סוכנים היא לעקוב אחרי רעיון דמו אחד לאורך הקורס.

דמו לדוגמה: **סוכן עזר לקורס**.

המשתמש שואל:

> "אני רוצה ללמוד איך סוכנים משתמשים בכלים. מצא את השיעורים הנכונים, סכם מה
> עליי לקרוא קודם, ותן לי משימת תרגול קצרה."

צ'אטבוט רגיל יכול לענות ממה שהוא כבר יודע. סוכן יכול לעשות יותר:

1. **לקרא או לחפש בקבצי הקורס** כדי למצוא את השיעורים המתאימים.
2. **להשתמש בכלים** כדי לשלוף קישורי שיעורים, דוגמאות או חומרי תמיכה.
3. **לתכנן** מסלול למידה קצר במקום לתת תשובה ארוכה אחת.
4. **להשתמש בהקשר** מהשיחה הנוכחית כדי להישאר ממוקד במטרת הלומד.
5. **לזכור העדפות שימושיות** אם היישום תומך בזיכרון.
6. **להראות עקבות, הפניות או לוגים** כדי שהמשתמש יבין מה קרה.
7. **להחיל כללי בטיחות** לפני נקיטת פעולות מסוכנות או שימוש בנתונים רגישים.


יוסיף?


## למה אתה בונה לקראת

שמחברות את החלקים האלה:

| חלק | משמעות בשפת יום-יום | בדמו |

| כלים | פונקציות, APIs, קבצים, דפדפנים או שירותים שהסוכן יכול להשתמש בהם | מחפש בריפו או שולף תוכן שיעור |
| ידע | מסמכים או נתונים שמשמשים לייסוד התשובה | קבצי README של הקורס וחומר השיעור |

## מודלים וספקים

דוגמאות הקוד בקורס משתמשות ב**מסגרת סוכן של מיקרוסופט (MAF)** ומכוונות ל**Azure OpenAI Responses API** — ה-API המומלץ להמשך, שמשלב השלמות שיחה, קריאות כלים, קלט מולטימודלי ושיחות עם מצב בתוך ממשק API יחיד. אתה מתחבר דרך פרויקט **Microsoft Foundry** (עם `FoundryChatClient`) או ישירות ל-Azure OpenAI (עם `OpenAIChatClient`).


בזמן שאתה עובד דרך השיעורים, יש לך כמה אפשרויות לספק:

- **Microsoft Foundry / Azure OpenAI (Responses API)** — הנתיב הראשי שבו משתמשים לאורך השיעורים. התחבר עם `az login` לאימות Entra ID ללא מפתח.
- **Foundry Local** — הפעל מודלים במלואם על המכשיר דרך API תואם OpenAI (ללא ענן, ללא מפתחות API). אידיאלי לניסויים לא מקוונים או ללא עלות. ראה [Course Setup](./00-course-setup/README.md).
- **MiniMax** — ספק תואם OpenAI עם מודלים של הקשר רחב, ניתן להשתמש בו כאלטרנטיבה קלה להחלפה.

> **הערה:** GitHub Models יוצא משירות (יפסיק ביולי 2026) ואינו תומך ב-Responses API. הדוגמאות עודכנו לשימוש ב-Azure OpenAI / Microsoft Foundry במקום.

## רעיונות מרכזיים במונחים ידידותיים למתחילים

### כלים

כלי הוא משהו שהסוכן יכול לקרוא לו כדי לבצע עבודה מחוץ לדגם. כלי טוב
יש לו שם ברור, משימה צרה, קלטים ממומשים, פלט צפוי, ודרך בטוחה
להיכשל.

להדגמת העוזר של הקורס, כלי עשוי להיות:

- `search_lessons(query)`
- `read_lesson(path)`
- `create_practice_task(topic)`

### RAG וידע

RAG עוזר לסוכן לענות מחומר מקור במקום לנחש. בקורס זה
חומר המקור יכול להיות README של שיעורים, דוגמאות קוד, או משאבים חיצוניים
המקושרים מהשיעורים.

השתמש ב-RAG כאשר התשובה צריכה להתבסס על מסמכים, נתונים, או קבצי פרויקט נוכחיים.


### תכנון

תכנון שימושי כשלב הבקשה כולל יותר מצעד אחד. שמור על התכניות קצרות ו
מספיק גלויות כדי שמפתח או משתמש יוכלו לבדוק.

להדגמה, תוכנית עשויה להיות:

1. מצא שיעורים הקשורים לשימוש בכלים.
2. סכם את השיעורים הרלוונטיים ביותר.
3. המלץ על משימת תרגול אחת.

### הקשר

הקשר הוא מה שהדגם רואה כרגע. מעט מדי הקשר יכול לגרום לסוכן
לפספס פרטים חשובים. יותר מדי הקשר יכול להאט את הסוכן, להיות יקר יותר,
או להפוך אותו לקל יותר לבלבול.

הנדסת הקשר טובה פירושה לבחור את המידע הנכון לקריאת הדגם הבאה.


### זיכרון

זיכרון הוא מידע שנשמר להמשך. אל תשמור הכל. שמור מידע
רק כשהוא שימושי, בטוח, וקל לעדכון או למחיקה.

לדוגמה, לזכור "הלומד מעדיף דוגמאות בפייתון" עשוי להיות שימושי.
בדרך כלל לא כדאי לזכור מידע אישי רגיש.

### הערכה וצפייה

הערכה שואלת: האם הסוכן עשה את הדבר הנכון?

צפייה שואלת: האם אפשר לראות איך זה קרה?

עבור סוכנים בפרודקשן, עקוב אחרי קריאות הדגם, קריאות לכלים, הקשר שהובא,
השהייה, עלות, כישלונות ומשוב משתמש.

### אמון ואבטחה

סוכנים אמינים צריכים יותר מאשר בקשה מועילה. השתמש בכלים עם מינימום הרשאות,
אישור אנושי לפעולות בעלות השפעה גבוהה, מחיקת נתונים לפי הצורך, ויומנים או
קבלות לפעולות שצריך לבדוק אותן.

## שגרה של 15 דקות לסקירה

השתמש בשגרה הזו לאחר כל שיעור:

1. **סכם את השיעור במשפט אחד.**
2. **תן שם ליכולת החדשה של הסוכן.** למשל: שימוש בכלים, אחזור,
   תכנון, זיכרון, צפייה, או אבטחה.
3. **הוסף אותו להדגמה של עוזר הקורס.** מה משתנה בהדגמה עכשיו?
4. **מצא את הסיכון.** מה עלול להשתבש אם משתמשים ביכולת זו לרעה?
5. **כתוב שאלה לבחינה אחת.** איך תבדוק שהסוכן מתנהג בצורה טובה?

## בדיקה עצמית מהירה

לפני שממשיכים, נסה לענות על השאלות האלו:

1. מה סוכן יכול לעשות שצ'אטבוט רגיל לא יכול לעשות בעצמו?
2. איזה כלי הסוכן שלך צריך ראשון, ולמה?
3. מאיזה מקור ידע כדאי שהתשובה של הסוכן תתבסס?
4. איזה הקשר צריך להיכלל בקריאת הדגם הבאה?

5. מה הסוכן צריך לזכור, ומה עליו להימנע מאחסון?
6. מתי על הסוכן לבקש אישור אנושי?
7. אילו יומנים, עקבות או קבלות יסייעו לך לאבחן או לבדוק את הסוכן מאוחר יותר?


## תרגיל קפסטון מוצע

בסוף הקורס, לבנות סוכן קטן שעוזר ללומד לנווט ב
המאגר הזה.

גרסת מינימום:

- לקבל נושא מהמשתמש.
