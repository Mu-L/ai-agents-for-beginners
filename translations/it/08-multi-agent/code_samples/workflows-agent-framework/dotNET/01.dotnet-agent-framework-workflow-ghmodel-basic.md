# 🔄 Workflow di Base per Agenti con Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial di Orchestrazione del Workflow

Questo notebook dimostra come costruire sofisticati **workflow per agenti** utilizzando il Microsoft Agent Framework per .NET e Azure OpenAI (Responses API). Imparerai a creare processi aziendali multi-step in cui agenti AI collaborano per svolgere compiti complessi attraverso pattern di orchestrazione strutturati.

## 🎯 Obiettivi di Apprendimento

### 🏗️ **Fondamenti dell'Architettura del Workflow**
- **Workflow Builder**: Progettare e orchestrare processi AI multi-step complessi
- **Coordinamento degli Agenti**: Coordinare più agenti specializzati all'interno dei workflow
- **Azure OpenAI (Responses API)**: Sfruttare l'API Responses di Azure OpenAI nei workflow
- **Progettazione Visuale del Workflow**: Creare e visualizzare strutture di workflow per una migliore comprensione

### 🔄 **Pattern di Orchestrazione dei Processi**
- **Elaborazione Sequenziale**: Collegare più attività di agenti in ordine logico
- **Gestione dello Stato**: Mantenere il contesto e il flusso dei dati tra le fasi del workflow
- **Gestione degli Errori**: Implementare un recupero robusto dagli errori e resilienza del workflow
- **Ottimizzazione delle Prestazioni**: Progettare workflow efficienti per operazioni su scala enterprise

### 🏢 **Applicazioni Enterprise dei Workflow**
- **Automazione dei Processi Aziendali**: Automatizzare workflow organizzativi complessi
- **Pipeline di Produzione dei Contenuti**: Workflow editoriali con fasi di revisione e approvazione
- **Automazione del Servizio Clienti**: Risoluzione multi-step delle richieste dei clienti

```xml
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Configurazione di Azure OpenAI**

**Configurazione dell'Ambiente (.env file):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Accesso ad Azure OpenAI:**
1. Crea una risorsa Azure OpenAI nel portale Azure
2. Distribuisci un modello (per esempio, `gpt-5-mini`) e annota il nome della distribuzione
3. Accedi con `az login` e configura le variabili di ambiente come mostrato sopra

## 💻 Esecuzione del Codice

L'implementazione completa è disponibile in `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Questo file dimostra:

1. **Configurazione dell'Ambiente** - Caricamento della configurazione Azure OpenAI dal file `.env`
2. **Configurazione del Client Azure OpenAI** - Configurazione del client per utilizzare l'API Responses di Azure OpenAI
3. **Creazione degli Agenti** - Definizione di agenti specializzati (Reception e Concierge)
4. **Workflow Builder** - Creazione di un workflow multi-agente con elaborazione sequenziale
5. **Esecuzione del Workflow** - Esecuzione del workflow con risultati in streaming

### 🚀 Esecuzione dell'Esempio

```bash
# Rendi lo script eseguibile (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Esegui il flusso di lavoro
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Oppure su Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Output Previsto

Il workflow:
1. Accetterà la tua richiesta di destinazione di viaggio ("Vorrei andare a Parigi")
2. L'agente Reception fornirà una raccomandazione iniziale
3. L'agente Concierge esaminerà e perfezionerà la raccomandazione
4. L'output finale mostrerà l'intero flusso della conversazione
