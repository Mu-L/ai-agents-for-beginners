# ⏩ Flussi di lavoro sequenziali degli agenti con Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial avanzato di elaborazione sequenziale

Questo notebook dimostra **modelli di flusso di lavoro sequenziali** utilizzando il Microsoft Agent Framework per .NET e Azure OpenAI (Responses API). Imparerai come costruire pipeline di elaborazione sofisticate, passo dopo passo, in cui gli agenti eseguono in un ordine specifico, con ogni fase che si basa sui risultati della fase precedente.

## 🎯 Obiettivi di apprendimento

### 🔄 **Architettura di elaborazione sequenziale**
- **Progettazione di flusso di lavoro lineare**: Crea pipeline di elaborazione passo dopo passo con dipendenze chiare
- **Gestione dello stato**: Mantieni il contesto e il flusso dei dati attraverso le fasi sequenziali del flusso di lavoro
- **Azure OpenAI (Responses API)**: Sfrutta i modelli Azure OpenAI in workflow .NET a più fasi
- **Modelli di pipeline aziendali**: Costruisci sistemi di elaborazione sequenziale pronti per la produzione

### 🏗️ **Modelli sequenziali avanzati**
- **Elaborazione a fasi con gate**: Implementa checkpoint di validazione tra le fasi del flusso di lavoro
- **Conservazione del contesto**: Mantieni lo stato e la conoscenza accumulata attraverso tutte le fasi
- **Propagazione degli errori**: Gestisci i fallimenti in modo elegante nelle catene di elaborazione sequenziali
- **Ottimizzazione delle prestazioni**: Esecuzione sequenziale efficiente con overhead minimo

### 🏢 **Applicazioni aziendali sequenziali**
- **Pipeline di elaborazione documenti**: Analisi, trasformazione e validazione di documenti a più fasi
- **Flussi di lavoro per assicurazione qualità**: Processi di revisione, validazione e approvazione sequenziali
- **Pipeline di produzione contenuti**: Ricerca → Scrittura → Modifica → Revisione → Pubblicazione
- **Automazione dei processi aziendali**: Flussi di lavoro aziendali a più passaggi con dipendenze di fase chiare

**Componenti chiave:**
- **Agenti sequenziali**: Agenti specializzati per ogni fase di elaborazione

- **Checkpoint**: Punti di validazione tra le fasi per garantire qualità e coerenza
- **Client Azure OpenAI**: Accesso coerente al modello AI in tutte le fasi del flusso di lavoro

## 💻 Esecuzione del codice

L'implementazione completa è disponibile in `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Questo file dimostra un **flusso di lavoro di analisi dei mobili a tre fasi**:

1. **Fase 1 - Agente vendite**: Analizza immagini di mobili e fornisce suggerimenti per l'acquisto
2. **Fase 2 - Agente prezzi**: Fornisce dettagliati scorpori dei prezzi e opzioni di budget
3. **Fase 3 - Agente preventivi**: Genera un documento di preventivo professionale in formato Markdown

### 🏗️ **Architettura del flusso di lavoro**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Ogni agente:
- Riceve l'output dalla fase precedente come contesto
- Si basa sull'analisi precedente con competenze specializzate
- Mantiene la continuità del flusso di lavoro tramite la gestione dello stato

### 🚀 Esecuzione dell'esempio

**Prerequisiti:**
- Posiziona un'immagine di mobili in `../imgs/home.png` (o aggiorna la variabile `imgPath`)
- Configura il tuo file `.env` con il tuo endpoint e deployment Azure OpenAI, quindi accedi con `az login`

```bash
# Rendi eseguibile lo script (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Esegui il flusso di lavoro sequenziale
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Oppure su Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Output previsto

Il flusso di lavoro:
1. **Agente vendite**: Identifica gli elementi di mobili dall'immagine e fornisce raccomandazioni
2. **Agente prezzi**: Aggiunge analisi dettagliata dei prezzi con fasce di budget e consigli per gli acquisti
3. **Agente preventivi**: Genera un documento preventivo formattato con tutte le informazioni sintetizzate

L'output finale sarà un preventivo professionale e completo per mobili basato sull'analisi dell'immagine.

### 🎯 Applicazioni nel mondo reale

Questo modello sequenziale è ideale per:
- **E-commerce**: Analisi prodotto → Prezzi → Generazione preventivi
- **Immobiliare**: Analisi proprietà → Valutazione → Creazione inserzioni
- **Assicurazioni**: Analisi reclami → Valutazione → Generazione preventivi

### 🔍 Comprendere il flusso di stato

Ogni agente nella sequenza riceve:
- **Input originale**: Il messaggio iniziale dell'utente (immagine + testo)
- **Output degli agenti precedenti**: Tutte le risposte precedenti degli agenti nella cronologia della conversazione
- **Contesto accumulato**: Stato completo mantenuto durante l'intero flusso di lavoro

Questo permette elaborazioni multi-fase sofisticate, dove ogni agente si basa su un contesto completo di tutte le fasi precedenti.
