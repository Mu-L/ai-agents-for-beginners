# ⏩ Flussi di lavoro sequenziali con Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial avanzato per l'elaborazione sequenziale

Questo notebook dimostra **modelli di flusso di lavoro sequenziali** utilizzando il Microsoft Agent Framework per .NET e Azure OpenAI (Responses API). Imparerai come costruire pipeline di elaborazione sofisticate e passo-passo in cui gli agenti eseguono in un ordine specifico, con ogni fase che si basa sui risultati della fase precedente.

## 🎯 Obiettivi di apprendimento

### 🔄 **Architettura di elaborazione sequenziale**
- **Progettazione lineare del flusso di lavoro**: Crea pipeline di elaborazione passo-passo con dipendenze chiare
- **Gestione dello stato**: Mantieni il contesto e il flusso di dati attraverso le fasi sequenziali del flusso di lavoro
- **Azure OpenAI (Responses API)**: Sfrutta i modelli Azure OpenAI in flussi di lavoro .NET multi-fase
- **Modelli di pipeline aziendali**: Costruisci sistemi di elaborazione sequenziali pronti per la produzione

### 🏗️ **Modelli sequenziali avanzati**
- **Elaborazione Stage-Gate**: Implementa checkpoint di validazione tra le fasi del flusso di lavoro
- **Preservazione del contesto**: Mantieni lo stato e la conoscenza accumulata tra tutte le fasi
- **Propagazione degli errori**: Gestisci i fallimenti in modo elegante nelle catene di elaborazione sequenziale
- **Ottimizzazione delle prestazioni**: Esecuzione sequenziale efficiente con overhead minimo

### 🏢 **Applicazioni aziendali sequenziali**
- **Pipeline di elaborazione documenti**: Analisi, trasformazione e validazione documenti multi-fase
- **Flussi di lavoro di assicurazione qualità**: Revisione sequenziale, validazione e processi di approvazione
- **Pipeline di produzione contenuti**: Ricerca → Scrittura → Editing → Revisione → Pubblicazione
- **Automazione di processi aziendali**: Flussi di lavoro aziendali multi-passo con dipendenze chiare tra fasi

### 🏗️ **Architettura del flusso di lavoro sequenziale**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Agente di Analisi]
    B --> C[Punto di Controllo 1]
    C --> D[Stage 2: Agente di Elaborazione]
    D --> E[Punto di Controllo 2]
    E --> F[Stage 3: Agente di Validazione]
    F --> G[Output Finale]
    
    H[Contesto dello Stato] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API Risposte)] --> B
    I --> D
    I --> F
```

**Componenti chiave:**
- **Agenti sequenziali**: Agenti specializzati per ogni fase di elaborazione
- **Contesto di stato**: Mantiene dati e decisioni accumulate tra le fasi
- **Checkpoint**: Punti di validazione tra le fasi per garantire qualità e coerenza
- **Client Azure OpenAI**: Accesso coerente ai modelli AI in tutte le fasi del flusso di lavoro

## 💻 Esecuzione del codice

L'implementazione completa è disponibile in `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Questo file dimostra un **flusso di lavoro di analisi del mobile a tre fasi**:

1. **Fase 1 - Agente vendite**: Analizza immagini di mobili e fornisce suggerimenti di acquisto
2. **Fase 2 - Agente prezzi**: Fornisce dettagliate ripartizioni dei prezzi e opzioni di budget
3. **Fase 3 - Agente preventivi**: Genera un documento di preventivo professionale in formato Markdown

### 🏗️ **Architettura del flusso di lavoro**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Ogni agente:
- Riceve l'output della fase precedente come contesto
- Si basa sull'analisi precedente con competenze specializzate
- Mantiene la continuità del flusso di lavoro tramite la gestione dello stato

### 🚀 Esecuzione dell'esempio

**Prerequisiti:**
- Posiziona un'immagine di mobili in `../imgs/home.png` (o aggiorna la variabile `imgPath`)
- Configura il tuo file `.env` con l'endpoint e il deployment Azure OpenAI, quindi effettua l'accesso con `az login`

### 📝 Output previsto

Il flusso di lavoro:
1. **Agente vendite**: identifica gli elementi di mobilio dall'immagine e fornisce raccomandazioni
2. **Agente prezzi**: aggiunge analisi dettagliate dei prezzi con fasce di budget e consigli per gli acquisti

Questo modello sequenziale è ideale per:
- **E-commerce**: Analisi prodotto → Prezzi → Generazione preventivi
- **Immobiliare**: Analisi proprietà → Valutazione → Creazione annunci

### 🔍 Comprendere il flusso dello stato

Ogni agente nella sequenza riceve:
- **Input originale**: Il messaggio iniziale dell'utente (immagine + testo)
- **Output degli agenti precedenti**: Tutte le risposte degli agenti precedenti nella cronologia della conversazione
- **Contesto accumulato**: Stato completo mantenuto durante tutto il flusso di lavoro

Questo permette un'elaborazione multi-fase sofisticata dove ogni agente si basa su un contesto completo di tutte le fasi precedenti.
