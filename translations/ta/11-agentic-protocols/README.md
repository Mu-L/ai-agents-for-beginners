# முகவர்கள் கட்டமைப்பைக் கொள்கைகள் பயன்படுத்துதல் (MCP, A2A மற்றும் NLWeb)

[![Agentic Protocols](../../../translated_images/ta/lesson-11-thumbnail.b6c742949cf1ce2a.webp)](https://youtu.be/X-Dh9R3Opn8)

> _(இந்த பாடத்தின் வீடியோவை பார்க்க மேல் உள்ள படத்தை செய்க)_

AI முகவர்களின் பயன்பாடு அதிகரிக்கும்போதும், தரநிலைபடுத்தல், பாதுகாப்பு மற்றும் திறந்த புதுமைகளை ஆதரிக்கும் கட்டமைப்புகளுக்கான தேவையும் அதிகரிக்கிறது. இந்த பாடத்தில், இந்த தேவையை பூர்த்தி செய்ய நாடும் 3 கட்டமைப்புகளை நாம் பார்க்கப்போகிறோம் - மாதிரி உள்ளடக்கிய கட்டமைப்பு (MCP), முகவரிடமிருந்து முகவருக்கு (A2A) மற்றும் இயற்கை மொழி வலை (NLWeb).

## அறிமுகம்

இந்த பாடத்தில், நாம்:

• எப்படி **MCP** AI முகவர்களுக்கு வெளியூர்களைக் கொண்டுள்ள கருவிகள் மற்றும் தரவுகளுக்கு அணுகுவதை அனுமதிக்கிறது என்பதைப் பற்றி.

•  எப்படி **A2A** வெவ்வேறு AI முகவர்களுக்கு இடையேயான தொடர்பு மற்றும் ஒத்துழைப்பை வழங்குகிறது என்பதைப் பற்றி.

• எப்படி **NLWeb** ஒவ்வொரு வலைதளத்திற்கும் இயற்கை மொழி இடைமுகங்களை கொண்டு வருகிறது, AI முகவர்கள் அங்குள்ள உள்ளடக்கத்தை கண்டுபிடித்து தொடர்பு கொள்ள உதவுகிறது என்பதைப் பற்றி.

## கற்றல் இலக்குகள்

• AI முகவர்களின் சூழலில் MCP, A2A மற்றும் NLWeb இன் அடிப்படை நோக்கம் மற்றும் நன்மைகளை **அறிதல்**.

• ஒவ்வொரு கட்டமைப்பும் LLMக்கள், கருவிகள் மற்றும் மற்ற முகவர்களிடையே தொடர்பு மற்றும் இடையிலான தொடர்பை எவ்வாறு எளிமைப்படுத்துகிறது என்பதை **விளக்குதல்**.

• சிக்கலான முகவர் அமைப்புகளை கட்ட بناء செய்ய ஒவ்வொரு கட்டமைப்பும் வகிக்கும் தனித்துவமான பங்குகளை **தெரிமுகம் செய்தல்**.

## மாதிரி உள்ளடக்கக் கொள்கை

**Model Context Protocol (MCP)** என்பது ஒரு திறந்த நிலையானது, செயலிகளுக்கு LLMகளுக்கு உள்ளடக்கம் மற்றும் கருவிகளை வழங்கும் தரநிலை வழியை வழங்குகிறது. இது AI முகவர்கள் இணைக்கக்கூடிய விதமாக வெவ்வேறு தரவு மூலங்கள் மற்றும் கருவிகளுக்கு ஒரு "பொதுவான இணைப்பாளர்" ஆக செயல்படுவதை சாத்தியமாக்குகிறது.

MCP உருவக பகுதிகளை, நேரடி API பயன்பாட்டுடன் ஒப்பிடும் நன்மைகளை, மற்றும் AI முகவர்களின் MCP சேவையகத்தை எவ்வாறு பயன்படுத்தக்கூடும் என்பதற்கான உதாரணத்தை பார்ப்போம்.

### MCP மூல கூறுகள்

MCP என்பது **வாடிக்கையாளர்-சேவையகம் கட்டமைப்பு** ஆக செயல்படுகிறது, மற்றும் அதன் முக்கிய கூறுகள்:

• **அதிபர்கள்** என்பது MCP சேவையகத்துடன் இணைப்புகளை துவக்கும் LLM செயலிகள் (உதாரணமாக VSCode போன்ற குறியீட்டு தொகுப்பி).

• **வாடிக்கையாளர்கள்** என்பது அதிபர் செயலியில் உள்ள கூறுகள், அவை சேவையகத்துடன் ஒரே நேரத்தில் தொடர்பை பராமரிக்கின்றன.

• **சேவையகங்கள்** என்பது குறிப்பிட்ட திறன்களை வெளிப்படுத்தும் எளிய இருப்புக்கள்.

இந்த கொள்கைக்குள் உள்ளே மூன்று அடிப்படை கூறுகள் உள்ளன, அவை MCP சேவையகத்தின் திறன்கள்:

• **கருவிகள்**: AI முகவர் செயல் செய்ய அழைக்கக்கூடிய தனித்துவமான நடவடிக்கைகள் அல்லது செயல்பாடுகள். உதாரணமாக, வானிலை சேவை "வானிலை பெற" கருவியை தெரிவிக்கலாம், அல்லது ஒரு மின் வணிக சேவையகம் "பொருள் வாங்குதல்" கருவியை வெளிப்படுத்தலாம். MCP சேவையகங்கள் ஒவ்வொரு கருவியின் பெயர், விவரம் மற்றும் உள்ளீடு/வெளியீட்டு அட்டவணையை திறன்களில் அறிவிக்கின்றன.

• **வளங்கள்**: வாசிக்க மட்டும் பயன்படும் தரவு பொருட்கள் அல்லது ஆவணங்கள், MCP சேவையகம் வழங்கக்கூடியவை மற்றும் வாடிக்கையாளர்கள் அவற்றை தேவைக்கேற்ப பெறக்கூடியவை. உதாரணமாக கோப்பு உள்ளடக்கம், தரவுத்தள பதிவுகள், அல்லது செயலிழப்பு கோப்புகள். வளங்கள் உரை (குறியீடு அல்லது JSON போன்றவை) அல்லது பைனரி (படங்கள் அல்லது PDFகள் போன்றவை) வடிவத்தில் இருக்கலாம்.

• **கேள்விமுறைகள்**: முன்கூட்டியே தயாரிக்கப்பட்ட வார்ப்புருக்கள், பரிந்துரைக்கப்பட்ட கேள்விகள், மேலும் சிக்கலான பணிகளுக்கான வழிகளை வழங்குகின்றன.

### MCP நன்மைகள்

MCP AI முகவர்களுக்கு முக்கியமான பல நன்மைகளை வழங்குகிறது:

• **மாறிலியான கருவி கண்டறிதல்**: முகவர்கள் சேவையகத்திலிருந்து என்ன கருவிகள் உள்ளன என இலகுவாகப் பெறலாம், அவற்றின் விளக்கத்துடன். இது பாரம்பரிய APIகளை ஒப்பிடுகையில் வேறுபாடு அளிக்கிறது, ஏனெனில் பாரம்பரிய APIகள் ஒருமுறை எழுதப்பட்ட குறியீட்டை மட்டுமே பயன்படுத்த தேவைபடும், உப மாற்றங்கள் குறியீடு மாற்றங்களை வேண்டுமாகும். MCP "ஒருமுறை இணைப்பு" என்ற அணுகுமுறையை வழங்குகிறது, இது மேம்படுத்துதலை அதிகரிக்கிறது.

• **பல்வேறு LLMகளுக்கு இடையிலான இணக்தி**: MCP பலவித LLMகளுடன் வேலை செய்கிறது, சிறந்த செயல்திறனை மதிப்பிட வேறு மாதிரிகளுக்கு மாற்றத்தை எளிமைப்படுத்துகிறது.

• **தரநிலைபடுத்தப்பட்ட பாதுகாப்பு**: MCP ஒரு தரநிலை அங்கீகார முறையை கொண்டுள்ளது, இது கூடுதல் MCP சேவையகங்களுக்கான அணுகலை விரிவுசெய்ய எளிதாக்குகிறது. இது வெவ்வேறு பாரம்பரிய APIகளுக்கான சிக்கலான விசைகள் மற்றும் அங்கீகார வகைகளை நிர்வகிப்பதைவிட எளிமையானது.

### MCP உதாரணம்

![MCP Diagram](../../../translated_images/ta/mcp-diagram.e4ca1cbd551444a1.webp)

ஒருவர் MCP சக்திவாய்ந்த AI உதவியாளரைப் பயன்படுத்தி ஒரு uçinerary புக் செய்ய விரும்புகிறார் என்று கண்டாயிடுவோம்.

1. **இணைப்பு**: AI உதவியாளர் (MCP வாடிக்கையாளர்) ஒரு விமான நிறுவனத்தால் வழங்கப்படும் MCP சேவையகத்துடன் இணைக்கப்படுகிறது.

2. **கருவி கண்டறிதல்**: வாடிக்கையாளர் விமான நிறுவனத்தின் MCP சேவையகத்திடம், "உங்களுக்கு என்ன கருவிகள் உள்ளன?" என்று கேட்கின்றது. சேவையகம் "சர்ச்ச் பிளைட்ஸ்" மற்றும் "புக் பிளைட்ஸ்" போன்ற கருவிகளை பதிலளிக்கின்றது.

3. **கருவி அழைக்கும் செயல்பாடு**: நீங்கள் பின்னர் AI உதவியாளரிடம், "போர்ட்லாந்த் இருந்து ஹொனோலுலு விமானத்தை தேடுங்கள்" என்று கேட்கின்றீர்கள். AI உதவியாளர் அதன் LLM மூலம் 'சர்ச்ச் பிளைட்ஸ்' கருவியை அழைக்க முடியும் என உணர்கிறது மற்றும் தேவையான அளவுருக்களை MCP சேவையகத்திற்கு அனுப்புகிறது.

4. **செயற்பாடு மற்றும் பதில்**: MCP சேவையகம் விமான நிறுவனத்தின் உள்ளமைப்பு புக்கிங் APIயை அழைத்து, விமானத் தகவலை (உதாரணமாக JSON தரவு) பெற்று அதை AI உதவியாளருக்கு அனுப்புகிறது.

5. **மேலும் தொடர்பு கொள்ளல்**: AI உதவியாளர் விமானத் தேர்வுகளை வழங்குகிறது. நீங்கள் ஒரு விமானத்தை தேர்வு செய்தவுடன், உதவியாளர் அதே MCP சேவையகத்தில் "புக் பிளைட்" கருவியை அழைக்கலாம், புக்கிங்கை நிறைவேற்றி.

## முகவரிடமிருந்து முகவருக்கு கொள்கை (A2A)

MCP LLMக்களை கருவிகளுடன் இணைப்பதற்கு கவனம் செலுத்தியபோது, **Agent-to-Agent (A2A) கொள்கை** அதை ஒரு படி மேலே கொண்டு செல்லுகிறது, வெவ்வேறு AI முகவர்களுக்கு இடையேயான தொடர்பு மற்றும் ஒத்துழைப்பை வழங்குகிறது. A2A வெவ்வேறு நிறுவனங்கள், சூழல்களும் தொழில்நுட்ப மேடைகளும் உள்ள AI முகவர்களை இணைத்து பகிர்ந்து பணிகளை நிறைவேற்ற உதவுகிறது.

A2A உறுப்புகள் மற்றும் நன்மைகளைப் பார்ப்போம், மேலும் நமது பயண பயன்பாட்டில் அதன் பயன்முறையை ஆராய்வோம்.

### A2A மூல கூறுகள்

A2A முகவர்களுக்கு இடையேயான தொடர்பை வழங்குவதே மற்றும் அவர்கள் ஒன்றுடன் ஒன்று இணைந்து பயனரின் துணை காரியங்களை முடிப்பதில் துணைசெய்வதே இலக்காகும். ஒவ்வொரு கூறும் இதற்கு பங்களிக்கிறது:

#### முகவர் அட்டை

MCP சேவையகம் கருவிகளின் பட்டியலை பகிர்ந்துவிடுவது போல, ஒரு முகவர் அட்டையில் உள்ளன:
- முகவரின் பெயர்.
- அது முடிக்கும் பொதுவான பணிகள் பற்றிய **விவரம்**.
- மற்ற முகவர்கள் (அல்லது மனித பயனர்களும்) அழைக்கும் காரணம் மற்றும் நேரம் புரிந்துகொள்ள உதவும் **கூடுதலான திறன்களின் பட்டியல்** மற்றும் விளக்கங்கள்.
- முகவரின் **தற்போதைய இறுதிக் குறிப்பு URL**.
- முகவரின் **பதிப்பு** மற்றும் **திறன்கள்** (ஸ்ட்ரீமிங் பதில்கள், புஷ் அறிவிப்புகள் போன்றவை).

#### முகவர் செயல்படுத்தி

முகவர் செயல்படுத்தி **பயனர் உரையாடல் உள்ளடக்கத்தை தொலை முகவருக்கு வழங்கும் பொறுப்பு** வாய்ந்தது, தொலை முகவர் பணி புரிய இதைத் தேவைப்படுகிறது. A2A சேவையகத்தில், ஒரு முகவர் தனது சொந்த LLM கொண்டு வரும் கோரிக்கைகளைப் படித்து உள்வாங்கும் மற்றும் தனது சொந்த கருவிகளுடன் பணிகளை நிறைவேற்றும்.

#### படைப்பு

தொலை முகவர் கோரிய பணியை முடித்த பிறகு, அதன் வேலை தயாரிப்பு ஒரு படைப்பாக உருவாகிறது. படைப்பு **முகவரின் பணிக்கு விளைவாகும் முடிவு** மற்றும் **முடிக்கப்பட்ட பணிகள் பற்றிய விளக்கம்** மற்றும் கொள்கையின் வழியாக அனுப்பப்படும் **உரைச் சூழல்** ஆகியவற்றைக் கொண்டுள்ளது. படைப்பு அனுப்பப்பட்ட பிறகு, தொலை முகவருடன் தொடர்பு மூடப்படும் மற்றும் மீண்டும் தேவைப்படும் வரைக்கும் திறக்கப்படாது.

#### நிகழ்வு வரிசை

இந்த கூறு **புதுப்பிப்புகளையும் செய்திகள் அனுப்புவதை** கையாள்கிறது. முகவர் அமைப்ப்களில் வெளியீட்டு நிலைப்பாட்டைப் பாதுகாக்க அது முக்கியம், குறிப்பாக பணிகள் நிறைவேறும் நேரம் நீண்டதாக இருக்கும் போது, முகவர்களுக்கிடையேயான தொடர்பு பணி முடிவுக்கு முன் மூடப்படாமல் இருப்பதை உறுதி செய்ய உதவுகிறது.

### A2A நன்மைகள்

• **மேம்பட்ட ஒத்துழைப்பு**: வெவ்வேறு விற்பனையாளர்கள் மற்றும் மேடைகளிலிருந்து முகவர்கள் இணைந்து, சூழலை பகிர்ந்து, ஒருங்கிணைந்த தொழில்முறைகளை இயக்க உதவுகிறது.

• **மாதிரி தேர்வு சுதந்திரம்**: ஒவ்வொரு A2A முகவரும் தமது கோரிக்கைகளை சேவையாற்ற எந்த LLMஐ பயன்படுத்துவதாக விரும்புகிறீர்கள் என்பதைத் தெரிவு செய்யலாம், இது MCPவைப்பாட்டில் ஒரே LLM இணைப்பை விட சிறந்ததாகும்.

• **உள்ளமைக்கப்பட்ட அங்கீகாரம்**: A2A கொள்கையில் அங்கீகாரம் நேரடியாக அடங்கியிருக்கிறது, முகவர் தொடர்புகளுக்கு வலுவான பாதுகாப்பு முறையை வழங்குகிறது.

### A2A உதாரணம்

![A2A Diagram](../../../translated_images/ta/A2A-Diagram.8666928d648acc26.webp)

நாம் முன்னர் பார்த்த பயண புக்கிங் நிலையை, இப்போது A2A பயன்படுத்தி விரிவாக்குவோம்.

1. **பயனர் கோரிக்கை பல முகவர்களுக்கு**: பயனர் "Travel Agent" என்ற A2A வாடிக்கையாளர்/முகவருடன் தொடர்பு கொண்டு, "வருகிற வாரம் ஹொனோலுலுக்கான முழு பயணத்தை புக் செய்யவும், விமானங்கள், ஓட்டல் மற்றும் வாட்டக கார் உட்பட" என்று கூறுகிறார்.

2. **பயண முகவரின் ஒருங்குப்பு**: பயண முகவர் இந்த சிக்கலான கோரிக்கையை பெறுகிறது. தன் LLMஐ பயன்படுத்தி பணியை ஆய்வு செய்து, பல நுட்ப முகவர்களுடன் தொடர்பு கொள்ள வேண்டும் என்பதை தீர்மானிக்கிறது.

3. **முகவர்களிடையேயான தொடர்பு**: பயண முகவர் A2A கொள்கையைப் பயன்படுத்தி, "விமான முகவர்", "ஒட்டல் முகவர்" மற்றும் "வாடகக் கார் முகவர்" போன்ற வித்தியாசமான நிறுவனங்களின் முகவர்களுடன் இணைக்கிறது.

4. **துணைப்பணி செயல்படுத்தல்**: பயண முகவர் குறிப்பிட்ட பணிகளை (உதா: "ஹொனோலுலுக்கான விமானங்கள் தேடு", "ஒட்டல் புக் செய்", "வாடகக் கார் புக் செய்") இந்நுட்ப முகவர்களுக்கு அனுப்புகிறது. இவை தங்கள் சொந்த LLMகளையும் MCP சேவையகங்களையும் பயன்படுத்தி தங்களது பங்குகளை நிறைவேற்றுகின்றன.

5. **ஒன்றிணைந்த பதில்**: அனைத்து துணை முகவர்களும் பணிகளை முடித்ததும், பயண முகவர் விவரங்கள் (விமான விவரம், ஓட்டல் உறுதி, கார் வாடகை) ஒன்றிணைத்து, பயனருக்கு ஒரு வாய்சாடல் வடிவத்தில் பதிலளிக்கின்றது.

## இயற்கை மொழி வலை (NLWeb)

இணையத்தில் தகவல்களையும் தரவுகளையும் அணுக பயனர்கள் பல ஆண்டுகளாக வலைத்தளங்களை முதன்மையாகப் பயன்படுத்தி வருகின்றனர்.

NLWeb இன் கூறுகள், நன்மைகள் மற்றும் நமது பயண பயன்பாட்டின் உதாரணமாக எப்படி செயல்படுகிறது என்பதைக் காண்போம்.

### NLWeb கூறுகள்

- **NLWeb செயலி (முகப்பு சேவை குறியீடு)**: இயற்கை மொழி கேள்விகளை செயலாக்கும் அமைப்பு. இது மையப் பகுதிகளை இணைத்து பதில்களை உருவாக்குகிறது. இது வலைதளத்தின் இயற்கை மொழி அம்சங்களை இயக்கும் **எந்திரம்** எனக் நினைக்கலாம்.

- **NLWeb கொள்கை**: இது ஒரு **வலைத்தளத்துடன் இயற்கை மொழி தொடர்புத்திறனைக்கான அடிப்படைக் கொள்கைகள்**. இது JSON வடிவத்தில் பதிலை அனுப்புகிறது (பொதுவாக Schema.org பயன்படுத்தி). இது “AI வலை” என்ற ஆயத்தமான அடித்தளத்தை உருவாக்கும் நோக்கத்தைக் கொண்டுள்ளது, HTML போன்றவைகளின் வாழ்க்கை வழியாக ஆவணங்களை பகிர்வது போல.

- **MCP சேவையக (மாதிரி உள்ளடக்கக் கொள்கை இறுதி செல்லுபடியாக்கல்)**: ஒவ்வொரு NLWeb அமைப்பும் ஒரு **MCP சேவையகமாக** செயல்படுகிறது. இதனால் அது பிற AI அமைப்புகளுக்கு கருவிகள் (அதாவது “ask” முறை) மற்றும் தரவை பகிர முடியும். நடைமுறையில், இந்த வசதி வலைத்தள உள்ளடக்கத்தையும் திறன்களையும் AI முகவர்களால் பயன்படுத்தக்கூடியதாக மாற்றி, அத்தளத்தை "முகவர் சூழல்" இன் பகுதியாக்கிறது.

- **ஏதுமாற்று மாதிரிகள்**: இவை வலைத்தள உள்ளடக்கத்தை அங்கீகாரம் செய்யும் எண்ணிக்கைப்படுத்தப்பட்ட வடிவங்களாக (செயல்முறை) மாற்றப் பயன்படும். இந்த வடிவங்கள் பொருள் விளக்கத்தைக் கம்ப்யூட்டர்களால் ஒப்பிட்டு தேட உதவும். இவை ஒரு குறிப்பிட்ட தரவுத்தளத்தில் சேமிக்கப்படுகின்றன, பயனர்கள் விரும்பிய ஏதுமாற்று மாதிரியை தேர்வு செய்யலாம்.

- **வெக்டர் தரவுத்தளம் (வழங்கும் இயந்திரம்)**: இது வலைத்தள உள்ளடக்கத்தின் ஏதுமாற்றுகள் சேமிக்கப்பட்ட தரவுத்தளம் ஆகும். யாராவது கேள்வியினை கேட்கும் போது, NLWeb வேகமாக பொருத்தமான தகவலைக் கண்டறிந்து தர வேக வடிவ மென்பொருள் தரவுத்தளத்துடன் வேலை செய்கிறது. இது Qdrant, Snowflake, Milvus, Azure AI Search மற்றும் Elasticsearch போன்ற பல்வேறு தரவுத்தளங்களோடு வேலை செய்கிறது.

### NLWeb உதாரணம்

![NLWeb](../../../translated_images/ta/nlweb-diagram.c1e2390b310e5fe4.webp)

மறுபடி நமது பயண புக்கிங் இணையதளத்தைப் பார்ப்போம், இப்போது அது NLWeb மூலம் இயங்குகிறது.

1. **தரவு ஏற்றுதல்**: பயண இணையதளத்தின் தயாரிப்பு பட்டியல்கள் (விமான பட்டியல்கள், ஓட்டல் விவரங்கள், சுற்றுலா தொகுப்புகள்) Schema.org மூலம் வடிவமைக்கப்பட்டுள்ளன அல்லது RSS மூலம் ஏற்றப்பட்டுள்ளன. NLWeb கருவிகள் இந்த கட்டமைக்கப்பட்ட தரவுகளை ஏற்று, ஏதுமாற்றுகள் உருவாக்கி உள்ளூர் அல்லது தொலைவகடல் வெக்டர் தரவுத்தளத்தில் சேமிக்கின்றன.

2. **இயற்கை மொழி கேள்வி (மனிதர்)**: பயனர் இணையதளத்தை சென்றடைந்து, மெனுக்களை சுற்றி செல்காமல், உரையாடல் இடைமுகத்தில் "அடுத்த வாரம் ஹொனோலுலு உடன் உட்புகுந்த குடும்ப நட்பு ஓட்டலுக்கு ஒரு இடம் தேடு" என்று உள்ளிடுகிறார்.

3. **NLWeb செயலாக்கம்**: NLWeb செயலி இந்த கேள்வியைக் பெறுகிறது. அதை LLMக்கு ஒன்றாக அனுப்பி புரிந்துகொள்ளச் செய்கிறது மற்றும் அதே நேரத்தில் வெக்டர் தரவுத்தளத்தில் பொருத்தமான ஓட்டல் பட்டியலைத் தேடுகிறது.

4. **தனிச்சிறப்பு முடிவுகள்**: LLM தரவுத்தள தேடலின் முடிவுகளைப் புரிந்து கொண்டு, "குடும்ப நட்பு," "குளியல் குளம்," மற்றும் "ஹொனோலுலு" என்ற நிபந்தனைகளுக்கு சிறந்த பொருத்தங்களை கண்டறிந்து இயற்கை மொழி பதிலை உருவாக்குகிறது. முக்கியமாக பதில் இணையதளத்தின் உண்மையான ஓட்டல்கள் பட்டியலை குறிப்பிடுகிறது, கற்பனை செய்த தகவல் அல்ல.

5. **AI முகவர் தொடர்பு**: NLWeb MCP சேவையகமாக செயல்படுவதால், வெளிப்புறத்தின் AI பயண முகவரும் இந்த இணையதளத்தின் NLWeb பதிப்புடன் இணைக்க முடியும். AI முகவர் `ask` MCP முறையை பயன்படுத்தி நேரடியாக கேள்வி கேட்கலாம்: `ask("ஹொனோலுலு பகுதியில் வில்லங்க நண்பர்கள் உணவகங்கள் உள்ளதா?")`. NLWeb இதை செயலாக்கி, அதன் உணவக தரவுத்தளத்தை பயன்படுத்தி, உருவமைக்கப்பட்ட JSON பதிலை திருப்பி வழங்கும்.

### MCP/A2A/NLWeb பற்றி மேலும் கேள்விகள் உள்ளதா?

மற்ற கற்றலாளர்களைச் சந்திக்க, அலுவலக நேரத்தில் கலந்துகொள்ள மற்றும் உங்கள் AI முகவர் கேள்விகளுக்கு பதிலளிக்க [Microsoft Foundry Discord](https://discord.com/invite/ATgtXmAS5D) இல் சேர்ந்திருங்கள்.
