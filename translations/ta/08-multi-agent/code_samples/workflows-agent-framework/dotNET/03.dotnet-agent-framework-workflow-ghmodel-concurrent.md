# ⚡ Azure OpenAI (Responses API) உடன் இணைந்து செயல்படும் முகவர் பணிச்சூழல்கள் (.NET)

## 📋 உயர் செயல்திறன் உடனடியாக செயலாக்க பயிற்சி

இந்த குற்றவியல் Microsoft Agent Framework for .NET மற்றும் Azure OpenAI (Responses API) பயன்படுத்தி **இணைந்து செயல்படும் பணிச்சூழல் வடிவமைப்புகளை** காட்சியளிக்கிறது. நீங்கள் பல AI முகவர்களை சீராக ஒரே நேரத்தில் இயக்குவதன் மூலம் பரிமாண திறனை அதிகரிப்பதற்கான உயர் செயல்திறன், ஒத்துழைப்பு மற்றும் தரவு ஒருமைத்தன்மையை பாதுகாப்பதான இணைந்த செயல்முறை பணிச்சூழல்களை உருவாக்க கற்றுக்கொள்ளப் போகிறீர்கள்.

## 🎯 கற்றல் குறிக்கோள்கள்

### 🚀 **இணைந்த செயலாக்க அடிப்படைகள்**
- **இணைந்த முகவர் இயங்கு**: அதிகபட்ச செயல்திறனுக்கான பல AI முகவர்களை ஒரே நேரத்தில் இயக்குதல்
- **Async/Await வடிவமைப்புகள்**: திறமையான இணைச்சேர்க்கை க்காக .NET இன் async நிரலாக்க மாதிரியை பயன்படுத்துதல்
- **Azure OpenAI (Responses API)**: Azure OpenAI Responses API-க்கு பல இணைந்த அழைப்பு இயக்கங்களை ஒருங்கிணைத்தல்
- **வள மேலாண்மை**: இணைந்த செயல்பாடுகளின் போது AI மாதிரி வளங்களை திறமையாக நிர்வகித்தல்

### 🏗️ **அமேம்பட்ட இணைச்சேர்க்கை கட்டமைப்பு**
- **தலைப்பு அடிப்படையிலான இணைச்சேர்க்கை**: சிறந்த செயலாக்கத்திற்காக .NET Task Parallel Library பயன்படுத்துதல்
- **இணைச்சேர்க்கை வடிவமைப்புகள்**: இரண்டாவது வரைவைத் தவிர்த்து ஒருங்கிணைந்த முகவர்களை ஒருங்கிணைத்தல்
- **பணிவிதானம்**: பெற்றுக்கொள்ளக்கூடிய இணைச்சேர்ந்த செயலாக்க திறன் முழுமையாக பகிர்தல்
- **பிழைத் தவிர்ப்பு**: தனிப்பட்ட முகவர் தோல்விகளைச் செயல்முறையை நிறுத்தாமல் கையாளுதல்

### 🏢 **கூட்டு நிறுவன பயன்பாடுகள்**
- **உயர் அளவிலான ஆவண செயலாக்கம்**: பல ஆவணங்களை ஒரே நேரத்தில் செயலாக்கம்
- **நேரடி உள்ளடக்கம் ஆய்வு**: வரும் தரவு ஓடைகளின் இணைந்த ஆய்வு
- **கூட்ட செயலாக்க மேம்பாடு**: பெரிய அளவிலான தரவு செயலாக்கத்தை அதிகபட்சம் செய்யல்
- **பல-வகை ஆய்வு**: வெவ்வேறு உள்ளடக்கங்கள் மற்றும் வடிவங்களில் இணைந்த செயலாக்கம்

## 💻 குறியீட்டை இயக்குதல்

முழுமையான செயலாக்கம் `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` கோப்பில் கிடைக்கிறது. இந்த கோப்பு பயணத் திட்டமிடலுக்கான **Fan-Out/Fan-In இணைந்த பணிச்சூழலை** காட்சி செய்கிறது:

### 🏗️ **பணிச்சூழல் கட்டமைப்பு**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**முக்கிய கூறுகள்:**

1. **ConcurrentStartExecutor**: பயனர் கோரிக்கையை ஒரே நேரத்தில் அனைத்து முகவர்களுக்கும் பரப்புகிறது
2. **Researcher Agent**: இடங்கள் மற்றும் பார்வை இடங்களை இணைந்து பகுப்பாய்வு செய்கிறது
3. **Planner Agent**: விரிவான பயணத் திட்டங்களை இணைந்து உருவாக்குகிறது
4. **ConcurrentAggregationExecutor**: இரு முகவர்களின் முடிவுகளை சேகரித்து ஒருங்கிணைக்கிறது

### 🎯 **Fan-Out/Fan-In வடிவமைப்பு**

இந்த பணிச்சூழல் வழக்கமான **Fan-Out/Fan-In** வடிவமைப்பைப் பயிற்றுவிக்கிறது:
- **Fan-Out**: ஒரே உள்ளீட்டு செய்தி பல முகவர்களுக்கு ஒரே நேரத்தில் பரவுகிறது
- **இணைந்த செயலாக்கம்**: பல முகவர்கள் ஒரே பணியில் இணைந்து வேலை செய்கின்றனர்
- **Fan-In**: அனைத்து முகவர்களின் முடிவுகள் சேகரிக்கப்பட்டு ஒருங்கிணைக்கப்படுகிறது

### 📝 எதிர்பார்க்கப்படும் வெளிச்சம்

பணிச்சூழல் பின்வருமாறு செயல்படும்:
1. **கோரிக்கை பரப்பல்**: "டிசெம்பர் மாதம் சீயாட்டிளுக்கு பயணம் திட்டமிடுக" என்பதை இரு முகவர்களுக்கும் அனுப்புதல்
2. **இணைந்த செயலாக்கம்**: இரு முகவர்களும் ஒரே நேரத்தில் பணி செய்தல்:
   - ஆராய்ச்சியாளர் இடங்கள் மற்றும் விவரங்களை கண்டறிதல்
   - திட்டமையாளர் பயண அட்டவணை மற்றும் பயண ஏற்பாடுகளை உருவாக்குதல்
3. **முடிவுக் கூட்டல்**: இரு பதில்களையும் ஒன்றிணைத்து விரிவான வெளிச்சம் தருதல்
4. **முடிவுகளை காண்பி**: ஒருங்கிணைக்கப்பட்ட பயணத் திட்டத்தைக் காண்பித்தல்

    .WithOutputFrom(aggregationExecutor)

### 🎯 நிஜ உலக பயன்பாடுகள்

இந்த இணைந்த வடிவமைப்பு பின்வருவனுக்கு சிறந்தது:
- **உள்ளடக்க உருவாக்கம்**: பல எழுத்தாளர்கள் வேறுபட்ட பகுதிகளை ஒரே நேரத்தில் உருவாக்குதல்
- **குறியீட்டு மதிப்பீடு**: பல மதிப்பாய்வாளர்கள் பன்முக பார்வையில் குறியீட்டை பரிசீலனை செய்தல்
- **சந்தை ஆராய்ச்சி**: வெவ்வேறு சந்தைப் பிரிவுகளின் இணைந்த பகுப்பாய்வு
- **ஆவண செயலாக்கம்**: இணைந்து எடுத்துக்கொள்ளல், பகுப்பாய்வு மற்றும் சரிபார்ப்பு
- **பல பார்வை பகுப்பாய்வு**: ஒரே உள்ளீட்டிற்கான பல்வேறு பார்வைகளைப் பெறுதல்

### 🔍 தனிப்பயன் நிறைவேற்றுநரைப் புரிதல்

**ConcurrentStartExecutor:**
- `IMessageHandler<string>`ஐ நடைமுறைப்படுத்துகிறது, எழுத்து உள்ளீட்டைப் பெறுகிறது
- அனைத்து இணைந்த முகவர்களுக்கும் செய்திகளை பரப்புகிறது
- இணைந்த செயலாக்கத்தைத் துவக்க `TurnToken` அனுப்புகிறது

**ConcurrentAggregationExecutor:**
- `IMessageHandler<ChatMessage>`ஐ நடைமுறைப்படுத்தி முகவர் பதில்களைப் பெறுகிறது
- thread-safe முறையில் செய்திகளை சேகரிக்கிறது
- எதிர்பார்க்கப்படும் எல்லா பதில்களும் வந்ததும் முடிவுகளைச் சங்கதிப்பது
- `context.YieldOutputAsync()` மூலம் இறுதி விளைவினை வழங்குகிறது

### ⚡ செயல்திறன் நன்மைகள்

**இணைந்து மற்றும் தொடர்ச்சியாக:**
- தொடர்ச்சியாக: Agent1 (30s) → Agent2 (30s) = **மொத்தம் 60 விநாடிகள்**
- இணைந்து: Agent1 (30s) || Agent2 (30s) = **மொத்தம் 30 விநாடிகள்**

**தெளிவாக்க மேம்பாடு**: N இணைந்த முகவர்களுக்கு N மடங்கு வேகம் (பணி மற்றும் வளங்களுக்கு இணங்க)

### 🛡️ பிழை கையாளுதல்

பணிச்சூழல் தனிப்பட்ட முகவர் தோல்விகளை மென்மையாக கையாள்கிறது:
- ஒருவர் தோல்வியடைந்தால், மற்றவர்கள் செயல்முறையை தொடர்ந்து செய்கின்றனர்
- சங்ககலிப்பாளர் கால அவகாச நெறிமுறையை நடைமுறைப்படுத்தலாம்
- தேவையானால் பகுதி முடிவுகள் திருப்பி வழங்கப்படலாம்
