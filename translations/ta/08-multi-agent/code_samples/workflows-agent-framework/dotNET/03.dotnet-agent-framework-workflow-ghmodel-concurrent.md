இந்த நோட்புக் .NET கிடைக்கும் மைக்ரோசாஃப்ட் முகவர் கட்டமைப்பையும் அசுரன் திறந்த AI (பதில் API)யைப் பயன்படுத்தி **ஒரே நேரத்தில் பணிசெய்யும் செயல்முறை மாதிரிகள்** ஐ காண்பிக்கிறது. பல AI முகவர்களை ஒரே நேரத்தில் ஒருங்கிணைத்தல் மற்றும் தரவு ஒருங்கிணைக்கும் போது அதிகபட்ச முறைப்பாடு மூலம் உயர் செயல்திறன், இணை பணியிட செயல்முறை ஒழுங்குகளை உருவாக்கி கற்றுக்கொள்வீர்கள்.

- **வள மேலாண்மை**: ஒரே நேர செயலாக்க நடவடிக்கைகளுக்கு AI மாதிரி வளங்களை திறம்பட நிர்வகிக்கவும்

## 💻 குறியீடு இயக்கல்

முழுமையான செயல்பாடு `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` இல் கிடைக்கிறது. இந்த கோப்பு பயணம் திட்டமிடுவதற்கான **Fan-Out/Fan-In ஒரே நேரப் பணிசெய்தி** மூலம் காட்சி அளிக்கிறது:

### 🏗️ **பணிசெய்தி கட்டமைப்பு**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**முக்கிய கூறுகள்:**

1. **ConcurrentStartExecutor**: பயனர் கோரிக்கையை அனைத்து முகவர்களுக்கும் ஒரே நேரத்தில் பிரசாரம் செய்கிறது
2. **ஆராய்ச்சி முகவர்**: இலக்குகளையும் ஈர்க்கும் இடங்களையும் ஒரே நேரத்தில் பகுப்பாய்வு செய்கிறது
3. **திட்டமிடுவான் முகவர்**: பயணம் திட்டங்களை விரிவாக ஒரே நேரத்தில் உருவாக்கும்
4. **ConcurrentAggregationExecutor**: இரண்டு முகவர்களின் முடிவுகளை சேகரித்து இணைக்கும்

### 🎯 **Fan-Out/Fan-In மாதிரி**

இந்த பணிசெய்தி பாரம்பரியமான **Fan-Out/Fan-In** மாதிரியை வெளிப்படுத்துகிறது:
- **Fan-Out**: ஒரு உள்ளீட்டு செய்தி பல முகவர்களுக்கு ஒரே நேரத்தில் பிரசாரம் செய்யப்படுகிறது
- **ஒரே நேர செயலாக்கம்**: பல முகவர்கள் ஒரே வேலைக்கு சமத்துவமாக செயல்படுகிறார்கள்
- **Fan-In**: அனைத்து முகவர்களிடமிருந்தும் முடிவுகள் சேகரிக்கப்பட்டு ஒரே வெளியீட்டியில் இணைக்கப்படும்

   - திட்டமிடுபவர் பயண அட்டவணை மற்றும் இயக்கத் திட்டத்தை உருவாக்குகிறார்
3. **ஒன்றுமை**: இரு பதில்களையும் ஒருங்கிணைத்து முழுமையான வெளியீடு உருவாக்கப்படுகிறது
4. **முடிவுகளை காட்டு**: இணைக்கப்பட்ட பயணம் திட்டத்துடன் அனைத்து தகவல்களையும் காட்டுகிறது

### 🎯 நிஜ உலக பயன்பாடுகள்

இந்த ஒரே நேர மாதிரி சிறந்தது:
- **உள்ளடக்க உருவாக்கம்**: பல எழுத்தாளர்கள் வெவ்வேறு பகுதிகளை ஒரே நேரத்தில் உருவாக்குதல்
- **குறியீடு மதிப்பாய்வு**: பல மதிப்பாய்வாளர்கள் வெவ்வேறு பார்வைகளில் குறியீட்டை பகுப்பாய்வு செய்தல்
- **மார்க்கெட் ஆராய்ச்சி**: வெவ்வேறு சந்தை பிரிவுகளின் இணை பகுப்பாய்வு
- **ஆவண செயலாக்கம்**: ஒரே நேரத்தில் பெறுதல், பகுப்பாய்வு மற்றும் உறுதிப்படுத்தல்
- **பல பார்வைகள் பகுப்பு**: ஒரே உள்ளீட்டிற்கு வெவ்வேறு கருத்துரைகளைப் பெறுதல்

### 🔍 தனிப்பயன் பணியாளர்களை புரிந்துகொள்

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` ஐ செயல்படுத்தி சரியான வரிகளை ஏற்றுக்கொள்கிறது
- இணைக்கப்பட்ட அனைத்து முகவர்களுக்கும் செய்திகளை பிரசாரம் செய்கிறது
- ஒரே நேர செயல்முறையைத் தூண்டும் `TurnToken` ஐ அனுப்புகிறது

**ConcurrentAggregationExecutor:**
- `IMessageHandler<ChatMessage>` ஐ செயல்படுத்தி முகவர் பதில்களை பெறுகிறது
- திரட் பாதுகாப்பான முறையில் செய்திகளை சேகரிக்கிறது
- எதிர்பார்க்கப்படும் அனைத்து பதில்களும் வந்தவுடன் தொகுக்கிறது
- `context.YieldOutputAsync()` மூலம் இறுதி வெளியீட்டை வழங்குகிறது

### ⚡ செயல்திறன் நன்மைகள்

**ஒரே நேர vs வரிசைப்படி:** 

- வரிசைப்படி: முகவர்1 (30s) → முகவர்2 (30s) = **மொத்தம் 60 வினாடிகள்**
- ஒன்றாக நடத்துதல்: முகவர்1 (30s) || முகவர்2 (30s) = **மொத்தம் 30 வினாடிகள்**

**திறன் மேம்பாடு**: N ஒரே நேரத்தில் முகவர்களுக்கான N மடங்கு வேகம் (பணி மற்றும் வளங்களின் அடிப்படையில்)

### 🛡️ பிழை கையாளுதல்

வேலைநடை தனித்தனியான முகவர் தோல்விகளை நன்றாக கையாள்கிறது:
- ஒரு முகவர் தோல்வியடைந்தால், மற்றவர்கள் செயல்பாட்டை தொடர்கின்றனர்
- தொகுப்பாளர் காலாவதி அறிவிப்பை செயல்படுத்தலாம்
- தேவையானபோது பகுதி முடிவுகள் வழங்கப்படலாம்
