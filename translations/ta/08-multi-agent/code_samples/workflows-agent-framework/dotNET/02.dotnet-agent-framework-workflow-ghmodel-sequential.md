# ⏩ Azure OpenAI (Responses API) உடன் தொடர் முகவர் வேலைப்புழக்கங்கள் (.NET)

## 📋 மேம்பட்ட தொடர் செயலாக்கம் பாடநெறி

இந்த நோட்புக் Microsoft Agent Framework ஐ பாவித்து .NET மற்றும் Azure OpenAI (Responses API) உடன் **தொடர் வேலைப்புழக்க மாதிரிகள்** எப்படி உருவாக்குவது என்பதை காட்டுகிறது. முகவர்கள் ஒரு குறிப்பிட்ட வரிசையில் செயல்படுகிறார்கள், ஒவ்வொரு கட்டமும் முந்தைய கட்டத்தின் முடிவுகளின் அடிப்படையில் நடைபெறுகின்றன என்பதைக் கற்றுக் கொள்வீர்கள்.

## 🎯 கற்றல் குறிக்கோள்கள்

### 🔄 **தொடர் செயலாக்கக் கட்டமைப்பு**
- **நேரியல் வேலைப்புழக்க வடிவமைப்பு**: தெளிவான சார்புகளுடன் கட்ட step-by-step செயலாக்க குழாய்களை உருவாக்குதல்
- **நிலை மேலாண்மை**: தொடர் வேலைப்புழக்க கட்டங்களுக்கிடையேயான சூழல் மற்றும் தரவு ஓட்டத்தை பராமரித்தல்
- **Azure OpenAI (Responses API)**: பல கட்ட .NET வேலைப்புழக்கங்களில் Azure OpenAI மாதிரிகளை பயன்படுத்துதல்
- **உற்பத்தி குழாய் மாதிரிகள்**: தொழில்துறை பயன்பாட்டிற்கு தயாரான தொடர் செயலாக்க அமைப்புகளை கட்டமைத்தல்

### 🏗️ **மேம்பட்ட தொடர் மாதிரிகள்**
- **கட்ட-கேட் செயலாக்கம்**: வேலைப்புழக்கக் கட்டங்களிடையே சரிபார்ப்பு கட்டங்களை செயல்படுத்தல்
- **சூழல் பாதுகாப்பு**: அனைத்து கட்டங்களிலும் நிலை மற்றும் சேகரிக்கப்பட்ட அறிவை பராமரித்தல்
- **பிழை பரவல்**: தொடராக செயல்படும் சங்கிலிகளில் தோல்விகளை மென்மையாக கையாளுதல்
- ** செயல்திறன் மேம்பாடு**: குறைந்த பளுக்கு உடன் திறம்பட தொடர் செயல்பாடு

### 🏢 **தொழில்துறை தொடர் பயன்பாடுகள்**
- **ஆவண செயலாக்க குழாய்**: பல கட்ட ஆவண பகுப்பாய்வு, மாற்றம் மற்றும் சரிபார்ப்பு
- **தரத்தை உறுதிப்படுத்தும் வேலைப்புழக்கங்கள்**: தொடர் ஆய்வு, சரிபார்ப்பு மற்றும் அங்கீகார செயல்முறை
- **உள்ளடக்க உற்பத்தி குழாய்**: ஆய்வு → எழுதுதல் → தொகுப்பு → ஆய்வு → வெளியீடு
- **பணியியல் செயல்பாட்டு தானியக்கம்**: தெளிவான கட்ட சார்புகளுடன் பல கட்ட வணிக வேலைப்புழக்கங்கள்

### 📦 **தேவையான NuGet தொகுப்புகள்**

.NET தொடர் வேலைப்புழக்கங்களுக்கான முக்கிய தொகுப்புகள்:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->

<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Azure OpenAI கட்டமைப்பு**

**சூழல் அமைப்பு (.env கோப்பு):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**கட்டமைப்பு மேலாண்மை:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: பகுப்பாய்வு முகவர்]
    B --> C[பரிசோதனை புள்ளி 1]
    C --> D[Stage 2: செயலாக்க முகவர்]

    H[நிலை சூழல்] --> B

    I[Azure OpenAI (மறுசெயல்கள் API)] --> B
```

**முக்கிய பகுதிகள்:**
- **தொடர் முகவர்கள்**: ஒவ்வொரு செயலாக்க கட்டத்திற்குமான நிபுணத்துவமிக்க முகவர்கள்
- **நிலை சூழல்**: கட்டங்களுக்கிடையில் சேகரிக்கப்பட்ட தரவு மற்றும் முடிவுகளை பராமரிக்கிறது
- **சரிபார்ப்பு புள்ளிகள்**: தரமும் ஒத்திசைவும்த் பெற கட்டங்களுக்கிடையில் சரிபார்ப்பு புள்ளிகள்
- **Azure OpenAI கிளையண்ட்**: அனைத்து வேலைப்புழக்க கட்டங்களிலும் ஒரே மாதிரி AI அணுகல்

இறுதி வெளியீடு பட பகுப்பாய்வின் அடிப்படையில் ஒரு முழுமையான, தொழில்முறை இருக்கை மேற்கோள் ஆக இருக்கும்.

### 🎯 நடைமுறை பயன்பாடுகள்

இந்த தொடர் மாதிரி ஏற்றது:
- **மின்வணிகம்**: பொருள் பகுப்பு → விலை நிர்ணயம் → மேற்கோள் உருவாக்கம்
- **உள்துறை சொத்து**: சொத்து பகுப்பு → மதிப்பீடு → பட்டியல் உருவாக்கம்
- **காப்பீடு**: கோரிக்கை பகுப்பு → மதிப்பீடு → மேற்கோள் உருவாக்கம்
- **உள்ளடக்க உருவாக்கம்**: ஆய்வு → எழுதுதல் → தொகுப்பு → வெளியீடு

### 🔍 நிலை ஓட்டத்தை புரிந்து கொள்வது

தொடரில் உள்ள ஒவ்வொரு முகவரும் பெறுகிறது:
- **மூல உள்ளீடு**: தொடக்க பயனர் தகவல் (படம் + உரை)
- **முந்தைய முகவர் வெளிப்பாடுகள்**: உரையாடல் வரலாற்றில் உள்ள அனைத்து முந்தைய முகவர் பதில்களும்
- **சேகரிக்கப்பட்ட சூழல்**: முழு வேலைப்புழக்கத்தில் பராமரிக்கப்படும் நிலை

இது சிக்கலான பல கட்ட செயலாக்கத்தைக் கையாள உதவுகிறது, ஒவ்வொரு முகவரும் அனைத்து முந்தைய கட்டங்களின் முழுமையான சூழல் தகவலின் அடிப்படையில் செயல்படுகிறான்.
