# கணினி பயனாளர் முகவர்களை (CUA) கட்டமைத்தல்

கணினி பயனாளர் முகவர்கள், ஒரு மனிதன் செய்வது போல, ஒரு உலாவியை திறந்து, பக்கத்தை பார்வையிட்டு, அங்கிருந்து அடுத்த சிறந்த நடவடிக்கையை எடுத்துக் கொள்ளும் முறையில் வலைத்தளங்களுடன் தொடர்பு கொள்ள முடியும். இந்த பாடத்தில், நீங்கள் ஒரு உலாவி தானியங்கி முகவரை உருவாக்கப் போகிறீர்கள், அது Airbnb-யை தேடி, கட்டமைக்கப்பட்ட பட்டியல் தரவை எடுத்துக்கொண்டு, ஸ்டாக்ஹோல்ம் நகரில் மிகக் குறைந்த விலையிலான தங்கும் இடத்தை கண்டுபிடிக்கும்.

இந்த பாடம் செயற்கை நுண்ணறிவினால் இயக்கப்படும் திரை உலாவல், Playwright மற்றும் Chrome DevTools Protocol (CDP) உலாவி கட்டுப்பாடு, Azure OpenAI காட்சியுடன் செயல்படுதல், மற்றும் Pydantic மூலம் கட்டமைக்கப்பட்ட வெளியீடு ஆகியவற்றை இணைக்கிறது.

## அறிமுகம்

இந்த பாடம்:

- கணினி பயனாளர் முகவர்கள் API-ஐ மட்டும் பயன்படுத்தும் தானியங்கத்துக்கு ஏற்றதா என்பதை புரிந்து கொள்வது
- நம்பகமான உலாவி உயிர் உயர் மேலாண்மைக்காக Browser-Use ஐ Playwright மற்றும் CDP உடன் இணைத்தல்
- Azure OpenAI காட்சி மற்றும் கட்டமைக்கப்பட்ட Pydantic வெளியீட்டை பயன்படுத்தி இயக்குவிக்கும் வலைப்பக்கங்களில் பட்டியல் தரவை எடுப்பது
- எப்போது முகவர் முதன்மை, நடிகர் முதன்மை அல்லது கலவை உலாவி தானியங்கி வேலைநிலை செயல்முறை பயன்படுத்தவேண்டும் என்பதை தீர்மானித்தல்

## கற்றல் இலக்குகள்

இந்த பாடத்தை முடித்தவுடன், நீங்கள் எப்படி தெரியும்:

- Azure OpenAI மற்றும் Playwright உடன் Browser-Use ஐ அமைக்க
- ஒரு உண்மையான வலைத்தளத்தில் வழிசெலுத்தும் உலாவி தானியங்கி வேலைநில் அமைக்கவும் இயக்கவும், மற்றும் இயக்கத்தில் உள்ள UI கூறுகளை கையாளவும்
- பக்கத்தில் தோன்றும் உள்ளடக்கத்திலிருந்து typed விளைவுகளை எடுத்து வணிக திருப்புகளுக்கு மாற்றவும்
- உலாவி பணியை எவ்வளவு எதிர்பார்க்கத்தக்கது என்பதற்கேற்ப முகவர் மற்றும் நடிகர் மாதிரிகளில் தேர்வு செய்யவும்

## குறியீடு உதாரணம்

இந்த பாடத்தில் ஒரு நோட்புக் பயிற்சி உள்ளது:

- [15-browser-user.ipynb](./15-browser-user.ipynb): CDP மூலம் Chrome அமர்வை துவக்குகிறது, Airbnb இல் ஸ்டாக்ஹோல்ம் பட்டியல்களை தேடுகிறது, Browser-Use காட்சி மூலம் விலைகளை எடுக்கும், மற்றும் மிகக் குறைந்த விலை ஆகும் விருப்பத்தை கட்டமைக்கப்பட்ட தரவாக காண்பிக்கிறது.

## முன்னோக்கிய தேவைகள்

- Python 3.12+
- Azure OpenAI தளவமைவு உங்கள் சுற்றுச்சூழலில் உள்ளமைக்கப்பட்டிருத்தல்
- Chrome அல்லது Chromium உள்ளூர் நிறுவல்
- Playwright சார்ந்தவைகள் நிறுவல்
- அசிங்க் Python பற்றிய அடிப்படைக் கவனம்

## அமைப்பு

நோட்புக்கில் பயன்படுத்தப்படும் தொகுதிகளை நிறுவவும்:

```bash
pip install browser_use playwright python-dotenv
playwright install chromium
```

நோட்புக்கால் பயன்படுத்தப்படும் Azure OpenAI சுற்றுச் சூழல் மாறிலிகள் அமைக்க:

```bash
AZURE_OPENAI_ENDPOINT=...
AZURE_OPENAI_API_KEY=...
AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=...
# விருப்பமானது: தவிர்க்கப்பட்டபோது புதிய API பதிப்புக்கு இயல்புநிலை அமைக்கப்படும்
AZURE_OPENAI_API_VERSION=...
```

## கட்டமைப்பு நோக்கு

நோட்புக் ஒரு கலவை உலாவி தானியங்கி வேலைநிலை செயல்முறையை காட்டுகிறது:

1. Playwright மற்றும் Browser-Use இரண்டும் ஒரே உலாவி அமர்வை பகிர அவர்கள் CDP இயங்கும் முறையில் Chrome துவங்குகிறது.
2. ஒரு Browser-Use முகவர் Airbnb-ஐத் திறந்து, பாப்ப்-அப்களை மறுக்கும், மற்றும் ஸ்டாக்ஹோல்ம் தேடும் போன்ற திறந்த தொலைநோக்குப் பணி செய்கிறது.
3. செயல்பட்டு இருக்கும் பக்கம் கட்டமைக்கப்பட்ட Pydantic ஸ்கீமானுடன் பார்வையிட்டு, பட்டியல் தலைப்புகள், இரவில் விலைகள், மதிப்பீடுகள் மற்றும் URL-களை எடுக்கிறது.
4. பைதான் தர்க்கம் எடுத்துக்கொண்ட பட்டியல்களை ஒப்பிட்டு, மிகக் குறைந்த விலை விளைவுக்கு முன்னுரிமை அளிக்கிறது.

இந்த அணுகுமுறை, Browser-Use நன்கு செய்யக்கூடிய நுண்ணறிவுடன் இயக்கப்படும் நோக்குகாட்சியலை வைத்திருக்கும் போது நிரூபிக்கப்பட்ட உலாவி கட்டுப்பாட்டையும் வழங்குகிறது.

## முக்கிய வெற்றிக் குறிப்புகள் மற்றும் சிறந்த நடைமுறைகள்

### முகவர் மற்றும் நடிகர் எப்போது பயன்படுத்துவது

| சந்தரம் | முகவர் பயன்படுத்தவும் | நடிகர் பயன்படுத்தவும் |
|----------|-----------|-----------|
| இயக்குநிலை முறை | ஆம், செயற்கை நுண்ணறிவு பக்க மாற்றங்களுக்கு ஏற்ப தகுவாகும் | இல்லை, பதிலைத் தேர்வு செய்தல் உடைக்கக்கூடும் |
| பரிச்சயமான கட்டமைப்பு | இல்லை, முகவர் நேரடி கட்டுப்பாட்டிலும் மெதுவானவர் | ஆம், விரைவும் துல்லியமும் |
| எலெஃமெண்ட்கள் கண்டுபிடித்தல் | ஆம், இயற்கை மொழி சிறப்பாக செயல்படும் | இல்லை, துல்லியமான தேர்வாளர்கள் அவசியம் |
| நேர கட்டுப்பாடு | இல்லை, குறைத்துக்கூடியது | ஆம், நன்கு முழுமையான கட்டுப்பாடு மற்றும் மீண்டும் முயற்சி செய்யும் நேர நிர்ணயம் |
| சிக்கலான வேலைநிலைகள் | ஆம், எதிர்பாராத UI நிலைகளை கையாளும் | இல்லை, தெளிவான கிளை மறுகிறிப்பு தேவை |

### Browser-Use சிறந்த நடைமுறைகள்

1. ஆராய்ச்சிக்கும் இயக்குநிலை மாற்றங்களுக்கு முகவருடன் துவங்கவும்.
2. தொடர்பு எதிர்பார்க்கப்படும் போது நேரடி பக்க கட்டுப்பாட்டிற்கு மாறவும்.
3. எடுத்துக்கொள்ளப்பட்ட தரவுக்கு உகந்த கட்டமைக்கப்பட்ட வெளிப்படுத்தும் மாதிரிகள் பயன்படுத்தவும்.
4. தெரியமான UI மாற்றங்களைத் தூண்டும் செயல்களுக்கு பிறகு கால விவகாரங்களை திட்டமிடவும்.
5. தோல்விகளை எளிதாக்கக் காட்சிகளைப் பதிவு செய்யவும்.
6. வலைத்தளங்கள் மாற்றப்படக்கூடும் என்பதை எதிர்பார்த்து பாப்-அப் மற்றும் அமைப்பு மாற்றங்களுக்கு மாற்று முறைகளை வடிவமைக்கவும்.
7. எளிதாக இயக்கமுடியும் மற்றும் துல்லியமான முகவர் மற்றும் நடிகர் மாதிரிகளை கலக்கவும்.

### உலாவி முகவர்களுக்கான பாதுகாப்பு வழிகாட்டுதல்கள்

உலாவி முகவர்கள் நேரடி வலைத்தளங்களில் செயல்படுவதால், அவர்கள் பார்வையிட முடியும், கிளிக் செய்ய முடியும் மற்றும் சமர்ப்பிக்க முடியும் என்ற கட்டுப்பாடுகளைத் திட்டமிடுவதே அவசியம். ஒரு நோட்புக் டெமோவில் இருந்து உண்மையான வேலைநிலைக்கு மாறுவதற்கு முன், இந்த கட்டுப்பாடுகளை வரையறுக்கவும்.

1. **உலாவல் சூழலை வரையறுக்கவும்.** முகவரைக் குறித்த உலாவி சுயவிவரத்தில் அல்லது பாதுகாப்பாக்கத்தில் இயக்கவும், வேலைக்கு தேவையான டொமைன்களுக்கு மட்டுப்படுத்தவும்.
2. **பரிசோதனையிலிருந்து நடவடிக்கையை பிரிக்கவும்.** முகவர் முதலில் தேடவும், படிக்கவும் மற்றும் தரவை எடுத்துக்கொள்ளவும்; இதன் பிறகு கூடுதல் நியமன செயலுக்கு முன்னே அனுமதி பெறவும்.
3. **கடவுச்சொற்கள், பாக்கெட் விவரங்கள், அமர்வு குக்கீகள் போன்றவற்றை மாடல் Contex-ல் வைக்காமல் பாதுகாப்புக்கு கவனம் செலுத்தவேண்டியது.** அங்கீகாரத்திற்கான நடவடிக்கையில் பயனர் மேலாண்மை ஏற்கப்பட வேண்டும் மற்றும் நுணுக்கமான விவரங்கள் பதிவு தொடர் புறக்கணிக்கப்பட வேண்டும்.
4. **பக்க உள்ளடக்கத்தை நம்பப்படாத உள்ளீடாக கருதவும்.** வலைத்தளம் முகவருக்கு மட்டும் அமைக்கப்பட்ட அறிவுரைகளைக் கொண்டிருக்கலாம், முகவர் அது போன்ற உரைகளை புறக்கணிக்க வேண்டும்.
5. **ஆபத்தான படிகளுக்கு முன் நிரூபித்தல் செய்து உறுதிசெய்யவும்.** இப்போதைய URL, பக்க தலைப்பு, தேர்ந்தெடுக்கப்பட்ட பொருள், விலை, பெறுநர் மற்றும் செயல்கள் ஆகியதைக் கோடுடன் சரிபார்க்க பயனரிடம் இறுதி அங்கீகாரத்திற்கு முன் கேட்கவும்.
6. **நடவடிக்கை மற்றும் நிறுத்து நிபந்தனைகள் அமைக்கவும்.** முகவர் பயன்படுத்தும் செயல்கள், மீண்டும் முயற்சிகள், தாவல்கள் மற்றும் நிமிட எண்ணிக்கைகளை வரையறுக்கவும். பக்க நிலை தெளிவாக இல்லாவிட்டால் கிளிக்கச் செய்காமல் நிறுத்தவும்.
7. **அனைத்தையும் பதிவுச்செய்யாமல் பயன்படுத்தக்கூடிய சான்றுகளை பதிவு செய்யவும்.** செயலின் சுருக்கங்கள், நேரம், URL கள், தேர்ந்தெடுக்கப்பட்ட பகுதி விளக்கங்கள் மற்றும் ஸ்கிரீன்ஷாட் குறிப்புகள் மட்டுமே பதிவு செய்யப்படவேண்டும்.

Airbnb உதாரணத்தில், பாதுகாப்பான இயல்பு குறைந்த விலை மார்க்கெட்டின் விலைகளை தேடுவது மற்றும் எடுப்பது தான். உள்நுழையவும், கோரிக்கை அனுப்பவும் அல்லது முன்பதிவு செய்யவும் அரை கட்டுப்பாடான பயனர் செயல்கள்.

### உண்மை உலகில் பயன்பாடுகள்

- பயண முன்பதிவு மற்றும் விலை கண்காணிப்பு
- மின்கழிவு விலை ஒப்பிடல் மற்றும் கிடைக்கும் நிலை பரிசோதனை
- இயக்குநிலை வலைத்தளங்களிலிருந்து கட்டமைக்கப்பட்ட தரவை எடுத்துக்கொள்வது
- காட்சி அறிவியலுடன் UI சோதனை மற்றும் சரிபார்ப்பு
- வலைத்தள கண்காணிப்பு மற்றும் அறிவிப்பு
- பல படி வடிவங்கள் மீதான புத்திசாலி படிவ பேணி நிரப்புதல்

## உண்மை உலகின் உதாரணம்: Microsoft Project Opal

இந்த பாடத்தில் நீங்கள் கட்டமைப்பது ஒரு சிறிய உள்ளூர் பதிப்பு **கணினி பயனாளர் முகவர் (CUA)** — ஒரு மனிதன் செய்வது போல உலாவியை இயக்கும் ஒரு திட்டம். Microsoft இதே யோசனையை Microsoft 365 Copilot-இல் உள்ள **[Project Opal (Frontier)](https://support.microsoft.com/en-us/microsoft-365-copilot/get-started-with-project-opal-frontier)** மூலம் நிறுவனங்களுக்கு கொண்டு வருகிறது.

Project Opal-இல், நீங்கள் ஒரு பணியை விவரிக்கிறீர்கள், முகவர் உங்கள் சார்பாக **பாதுகாப்பான Windows 365 Cloud PC-வில் கணினி பயன்பாட்டை** பயன்படுத்தி, உங்கள் நிறுவனத்தின் உலாவி அடிப்படையிலான பயன்பாடுகள், தளங்கள் மற்றும் தரவுகளின் மீது இயங்குகிறது. இது **பின்னணியில் ஒத்திசைக்கப்பட்ட முறையில்** செயல்படுகிறது, நீங்கள் எந்த நேரத்திலும் பணியின் வழிகாட்டுதல் அல்லது கட்டுப்பாட்டை எடுத்துக் கொள்ளலாம். உதாரண வேலைகள்:

- பாதுகாப்பு குழு உறுப்பினர் கோரிக்கைகளை நிர்வகித்தல்
- ஒத்திசைவு மதிப்பீடுகளுக்கு ஆய்வு ஆதாரங்கள் சேகரித்தல் மற்றும் சரிபார்த்தல்
- IT சம்பவங்களை முகாமித்தல் (டிக்கெட் நிலையை புதுப்பித்தல், உரிமையாளர்களை நியமித்தல், நகல்களை மூடல்)
- Excel தரவை நிதி முடிவு அறிக்கைக்கான தொகுப்பாகச் சேர்த்தல்

Opal ஒரு **உற்பத்தி தரம், நம்பகமான** கணினி பயனாளர் முகவரின் சிறப்பினை எடுத்துக்காட்டும் மற்றும் முன்இருந்த பாடங்களின் கருத்துக்களை ஊக்குவிக்கிறது:

| இந்த பாடத்தில் உள்ள கருத்து | Project Opal அதை எப்படி பயன்படுத்துகிறது |
|------------------------|-----------------------------|
| **மனிதர் இணைந்த (Human-in-the-loop)** (பாடம் 06) | Opal உள்நுழைவு சான்றுகள், நுணுக்கமான தரவுகள், அல்லது தெளிவில்லாத கருத்துகளுக்கு இடைவெளி கொடுக்கும், கடவுச்சொற்களை எப்போதும் உள்ளிடாது அல்லது படிவங்களை அனுப்பாது. நீங்கள் *கட்டுப்பாட்டை ஏற்று* மற்றும் *மீண்டும் ஒப்படைக்க* முடியும். |
| **நம்பகமான மற்றும் பாதுகாப்பான முகவர்கள்** (பாடங்கள் 06 & 18) | தனிமைப்படுத்தப்பட்ட Windows 365 Cloud PC-வில் இயக்கப்படுவதாகும், இயல்பாக உலாவி மட்டும் (மற்ற கணினி அணுகலை தடுக்கும், Intune மூலம் கட்டாயப்படுத்தப்படும்), உங்கள் அடையாளத்தை பயன்படுத்துவதால் நீங்கள் அனுமதிக்கப்பட்டதையே அணுகுகிறது, மற்றும் ஒவ்வொரு நடவடிக்கையும் ஆய்வுக்குறிப்பாக பதிவாகும். |
| **திட்டமிடல் மற்றும் மேல்முக்கிய அறிவு** (பாடங்கள் 07 & 09) | Opal முதலில் வேலைக்கான திட்டம் உருவாக்கி, பிறகு ஒவ்வொரு படியிலும் தன் நியாயத்தைக் கண்காணித்து, சந்தேகத்திற்கும் இடைவெளி கொடுக்கும். |
| **மீண்டும் பயன்படுத்தக்கூடிய திறன்கள் / கருவிகள்** (பாடம் 04) | **திறன்கள்** மீண்டும் செய்யக்கூடிய பணிகளுக்கான வழிமுறைகளை எழுத அனுமதிக்கும் (`.md` கோப்பில் இருந்து இறக்குமதி செய்வதோ அல்லது Opal-ல் எழுதுவதோ), மற்றும் உரையாடல்களில் மீண்டும் பயன்படுத்தவும். |

> **கிடைக்குமிடம்:** Project Opal தற்போது [Frontier ஆரம்ப நுழைவுத் திட்டத்தில்](https://adoption.microsoft.com/copilot/frontier-program/) Microsoft 365 Copilot சந்தாவுடன் என்ற பயனர்களுக்கு கிடைக்கிறது, உங்கள் நிர்வாகி அமைப்பை முடிக்க வேண்டும். இது ஒரு பரிசோதனையான Frontier அம்சமாக இருப்பதால், திறன்கள் காலப்போக்கில் மாறக்கூடும்.
