# ⏩ Azure OpenAI (Responses API) ile Sıralı Ajan İş Akışları (.NET)

## 📋 İleri Düzey Sıralı İşleme Eğitimi

Bu not defteri, Microsoft Agent Framework for .NET ve Azure OpenAI (Responses API) kullanarak **sıralı iş akışı desenlerini** göstermektedir. Ajanların belirli bir sırayla yürütüldüğü, her aşamanın önceki aşamanın sonuçları üzerine inşa edildiği gelişmiş, adım adım işleme boru hatları nasıl oluşturulur öğreneceksiniz.

## 🎯 Öğrenme Hedefleri

### 🔄 **Sıralı İşleme Mimarisi**
- **Doğrusal İş Akışı Tasarımı**: Net bağımlılıklarla adım adım işleme boru hatları oluşturun
- **Durum Yönetimi**: Sıralı iş akışı aşamaları arasında bağlam ve veri akışını sürdürün
- **Azure OpenAI (Responses API)**: Çok aşamalı .NET iş akışlarında Azure OpenAI modellerinden yararlanın
- **Kurumsal Boru Hattı Desenleri**: Üretime hazır sıralı işleme sistemleri oluşturun

### 🏗️ **İleri Düzey Sıralı Desenler**
- **Aşama-Kapı İşleme**: İş akışı aşamaları arasında doğrulama kontrol noktaları uygulayın
- **Bağlam Koruma**: Tüm aşamalar boyunca durum ve birikmiş bilgiyi koruyun
- **Hata Yayılımı**: Sıralı işleme zincirlerindeki hataları nazikçe yönetin
- **Performans Optimizasyonu**: Minimum ek yük ile verimli sıralı yürütme

### 🏢 **Kurumsal Sıralı Uygulamalar**
- **Belge İşleme Boru Hattı**: Çok aşamalı belge analizi, dönüşümü ve doğrulama
- **Kalite Güvence İş Akışları**: Sıralı inceleme, doğrulama ve onay süreçleri
- **İçerik Üretim Boru Hattı**: Araştırma → Yazım → Düzenleme → İnceleme → Yayınlama
- **İş Süreci Otomasyonu**: Net aşama bağımlılıkları olan çok adımlı iş akışları

## ⚙️ Önkoşullar ve Kurulum

### 📦 **Gerekli NuGet Paketleri**

.NET sıralı iş akışları için gerekli paketler:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Azure OpenAI Yapılandırması**

**Ortam Kurulumu (.env dosyası):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Yapılandırma Yönetimi:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

## 💻 Kodu Çalıştırma

Tam uygulama `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs` dosyasında mevcuttur. Bu dosya **üç aşamalı mobilya analiz iş akışını** göstermektedir:

1. **Aşama 1 - Satış Ajanı**: Mobilya görsellerini analiz eder ve satın alma önerileri sunar
2. **Aşama 2 - Fiyat Ajanı**: Detaylı fiyatlandırma dökümleri ve bütçe seçenekleri sağlar
3. **Aşama 3 - Teklif Ajanı**: Markdown formatında profesyonel teklif belgesi oluşturur

### 🏗️ **İş Akışı Mimarisi**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Her ajan:
- Önceki aşamanın çıktısını bağlam olarak alır
- Özel uzmanlıkla önceki analizlerin üzerine inşa eder
- Durum yönetimi sayesinde iş akışı sürekliliğini sağlar

### 🚀 Örneği Çalıştırma

**Önkoşullar:**
- `../imgs/home.png` dosyasına bir mobilya görseli yerleştirin (veya `imgPath` değişkenini güncelleyin)
- Azure OpenAI uç noktanız ve dağıtımınızla `.env` dosyanızı yapılandırın, ardından `az login` ile oturum açın

```bash
# Betiği çalıştırılabilir yap (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Sıralı iş akışını çalıştır
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Veya Windows'ta:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Beklenen Çıktı

İş akışı şunları yapacak:
1. **Satış Ajanı**: Görselden mobilya öğelerini tanımlayacak ve öneriler sunacak
2. **Fiyat Ajanı**: Bütçe katmanları ve alışveriş önerileri ile detaylı fiyat analizi ekleyecek
3. **Teklif Ajanı**: Tüm bilgiler sentezlenmiş biçimlendirilmiş teklif belgesi oluşturacak
