# ⏩ Azure OpenAI ile Ardışık Agent İş Akışları (Responses API) (.NET)

## 📋 İleri Seviye Ardışık İşlem Eğitimi

Bu not defteri, Microsoft Agent Framework for .NET ve Azure OpenAI (Responses API) kullanarak **ardışık iş akışı desenlerini** gösterir. Her bir aşamanın önceki aşamanın sonuçları üzerine kurulduğu, ajanların belirli bir sırayla çalıştığı gelişmiş, adım adım işleme boru hatları nasıl oluşturulur öğreneceksiniz.

## 🎯 Öğrenme Hedefleri

### 🔄 **Ardışık İşleme Mimarisi**
- **Doğrusal İş Akışı Tasarımı**: Net bağımlılıklarla adım adım işleme boru hatları oluşturma
- **Durum Yönetimi**: Ardışık iş akış aşamaları arasında bağlam ve veri akışını sürdürme
- **Azure OpenAI (Responses API)**: Çok aşamalı .NET iş akışlarında Azure OpenAI modellerini kullanma
- **Kurumsal Boru Hattı Desenleri**: Üretime hazır ardışık işleme sistemleri kurma

### 🏗️ **İleri Seviyede Ardışık Desenler**
- **Aşama-Kapı İşlemleri**: İş akışı aşamaları arasında doğrulama kontrol noktaları uygulama
- **Bağlam Koruma**: Tüm aşamalar arasında durum ve birikmiş bilgiyi sürdürme
- **Hata Yayılımı**: Ardışık işleme zincirlerinde hataları zarifçe ele alma
- **Performans Optimizasyonu**: Minimum ek yükle etkili ardışık yürütme

### 🏢 **Kurumsal Ardışık Uygulamalar**
- **Doküman İşleme Boru Hattı**: Çok aşamalı doküman analizi, dönüştürme ve doğrulama
- **Kalite Güvence İş Akışları**: Ardışık inceleme, doğrulama ve onay süreçleri
- **İçerik Üretim Boru Hattı**: Araştırma → Yazım → Düzenleme → İnceleme → Yayınlama
- **İş Süreçleri Otomasyonu**: Net aşama bağımlılıklarına sahip çok adımlı iş akışları

## ⚙️ Önkoşullar & Kurulum

### 📦 **Gerekli NuGet Paketleri**

.NET ardışık iş akışları için gerekli paketler:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Azure OpenAI Yapılandırması**

**Ortam Kurulumu (.env dosyası):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Yapılandırma Yönetimi:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

## 💻 Kodu Çalıştırma

Tam uygulama `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs` dosyasında mevcuttur. Bu dosya **üç aşamalı mobilya analiz iş akışını** gösterir:

1. **Aşama 1 - Satış Ajanı**: Mobilya görüntülerini analiz eder ve satın alma önerileri sunar
2. **Aşama 2 - Fiyat Ajanı**: Ayrıntılı fiyatlandırma dökümleri ve bütçe seçenekleri sağlar
3. **Aşama 3 - Teklif Ajanı**: Markdown formatında profesyonel bir teklif belgesi üretir

### 🏗️ **İş Akışı Mimarisi**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Her ajan:
- Önceki aşamanın çıktısını bağlam olarak alır
- Uzmanlığıyla önceki analiz üzerine inşa eder
- Durum yönetimiyle iş akışı sürekliliğini sağlar

### 🚀 Örneği Çalıştırma

**Önkoşullar:**
- Bir mobilya görüntüsünü `../imgs/home.png` konumuna yerleştirin (ya da `imgPath` değişkenini güncelleyin)
- `.env` dosyanızı Azure OpenAI uç noktası ve dağıtımı ile yapılandırın, sonra `az login` ile giriş yapın

```bash
# Betik çalıştırılabilir hale getirin (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Sıralı iş akışını çalıştırın
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Ya da Windows üzerinde:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Beklenen Çıktı

İş akışı:
1. **Satış Ajanı**: Görüntüden mobilya öğelerini tanımlar ve öneriler sunar
2. **Fiyat Ajanı**: Bütçe katmanları ve alışveriş önerileriyle ayrıntılı fiyat analizi ekler
3. **Teklif Ajanı**: Tüm bilgileri sentezleyerek formatlanmış bir teklif belgesi üretir

Nihai çıktı, görüntü analizine dayalı kapsamlı, profesyonel bir mobilya teklifi olacaktır.
