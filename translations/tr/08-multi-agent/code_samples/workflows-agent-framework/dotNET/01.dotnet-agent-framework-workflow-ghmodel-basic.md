# 🔄 Azure OpenAI (Responses API) ile Temel Ajan İş Akışları (.NET)

## 📋 İş Akışı Orkestrasyonu Eğitimi

Bu not defteri, Microsoft Agent Framework for .NET ve Azure OpenAI (Responses API) kullanarak gelişmiş **ajan iş akışları** oluşturmayı nasıl yapacağınızı gösterir. AI ajanlarının yapılandırılmış orkestrasyon desenleri aracılığıyla karmaşık görevleri başarmak için işbirliği yaptığı çok adımlı iş süreçleri oluşturmayı öğreneceksiniz.

## 🎯 Öğrenme Hedefleri

### 🏗️ **İş Akışı Mimarisi Temelleri**
- **İş Akışı Oluşturucu**: Karmaşık çok adımlı AI süreçlerini tasarlayın ve orkestre edin
- **Ajan Koordinasyonu**: İş akışlarında birden çok uzmanlaşmış ajanları koordine edin
- **Azure OpenAI (Responses API)**: İş akışlarında Azure OpenAI Responses API'yi kullanın
- **Görsel İş Akışı Tasarımı**: Daha iyi anlamak için iş akışı yapılarını oluşturun ve görselleştirin

### 🔄 **Süreç Orkestrasyon Desenleri**
- **Sıralı İşleme**: Birden çok ajan görevini mantıksal sırada zincirleyin
- **Durum Yönetimi**: İş akışı aşamalarında bağlamı ve veri akışını koruyun
- **Hata Yönetimi**: Güçlü hata kurtarma ve iş akışı dayanıklılığı uygulayın
- **Performans Optimizasyonu**: Kurumsal ölçekli operasyonlar için verimli iş akışları tasarlayın

### 🏢 **Kurumsal İş Akışı Uygulamaları**
- **İş Süreci Otomasyonu**: Karmaşık organizasyonel iş akışlarını otomatikleştirin
- **İçerik Üretim Hattı**: İnceleme ve onay aşamaları ile editoryal iş akışları
- **Müşteri Hizmetleri Otomasyonu**: Çok adımlı müşteri sorgusu çözümü
- **Veri İşleme İş Akışları**: AI destekli dönüşüm ile ETL iş akışları

## ⚙️ Önkoşullar ve Kurulum

### 📦 **Gerekli NuGet Paketleri**

Bu iş akışı gösterimi birkaç önemli .NET paketi kullanır:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI Yapılandırması**

**Ortam Ayarı (.env dosyası):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Azure OpenAI Erişimi:**
1. Azure portalında bir Azure OpenAI kaynağı oluşturun
2. Bir model dağıtın (örneğin, `gpt-5-mini`) ve dağıtım adını not edin
3. `az login` ile oturum açın ve yukarıda gösterildiği gibi ortam değişkenlerini yapılandırın

### 🏗️ **İş Akışı Mimarisi Genel Bakış**

```mermaid
graph TD

    C --> E[Agent 2: İçerik İnceleyicisi 
    D --> F[İş Akışı Sonuçları]
    E --> F
    G[Azure OpenAI (Yanıtlar API)] --> D
    G --> E
```

**Ana Bileşenler:**
- **WorkflowBuilder**: İş akışlarını tasarlamak için ana orkestrasyon motoru
- **AIAgent**: Belirli yeteneklere sahip bireysel uzman ajanlar
- **Azure OpenAI Client**: Azure OpenAI Responses API entegrasyonu
- **Execution Context**: İş akışı aşamaları arasında durumu ve veri akışını yönetir

## 💻 Kodu Çalıştırma

Tam uygulama `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` içinde mevcuttur. Bu dosya şunları gösterir:

1. **Ortam Yapılandırması** - `.env` dosyasından Azure OpenAI yapılandırmasının yüklenmesi
2. **Azure OpenAI İstemcisi Kurulumu** - Azure OpenAI Responses API kullanan istemcinin ayarlanması
3. **Ajan Oluşturma** - Uzmanlaşmış ajanların tanımlanması (Resepsiyon ve Konsiyerj)
4. **İş Akışı Oluşturucu** - Sıralı işleme ile çok ajanlı iş akışı oluşturma
5. **İş Akışı Yürütme** - Akışı akışlı sonuçlarla çalıştırma

### 🚀 Örneği Çalıştırmak

```bash
# Betik çalıştırılabilir hale getirin (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# İş akışını çalıştırın
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Veya Windows üzerinde:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```
