# ⚡ Azure OpenAI (Responses API) ile Eşzamanlı Ajan İş Akışları (.NET)

## 📋 Yüksek Performanslı Paralel İşleme Eğitimi

Bu not defteri, .NET için Microsoft Agent Framework ve Azure OpenAI (Responses API) kullanarak **eşzamanlı iş akışı kalıplarını** göstermektedir. Birden fazla AI ajanını aynı anda çalıştırarak verimliliği maksimize eden yüksek performanslı paralel işleme iş akışlarının nasıl oluşturulacağını, koordinasyon ve veri tutarlılığını korurken öğreneceksiniz.

## 💻 Kodu Çalıştırma

Tam uygulama `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` dosyasında mevcuttur. Bu dosya seyahat planlaması için **Fan-Out/Fan-In eşzamanlı iş akışını** göstermektedir:

### 🏗️ **İş Akışı Mimarisi**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Ana Bileşenler:**

1. **ConcurrentStartExecutor**: Kullanıcı isteğini tüm ajanlara eşzamanlı yayınlar
2. **Researcher Agent**: Gidecek yerleri ve cazibe merkezlerini eşzamanlı analiz eder
3. **Planner Agent**: Ayrıntılı seyahat planları oluşturur eşzamanlı olarak
4. **ConcurrentAggregationExecutor**: Her iki ajandan gelen sonuçları toplar ve birleştirir

### 🎯 **Fan-Out/Fan-In Kalıbı**

Bu iş akışı klasik **Fan-Out/Fan-In** kalıbını göstermektedir:
- **Fan-Out**: Bir giriş mesajı birden çok ajana eşzamanlı olarak yayınlanır
- **Eşzamanlı İşleme**: Birden fazla ajan aynı görev üzerinde paralel çalışır
- **Fan-In**: Tüm ajanların sonuçları toplanıp tek bir çıktı olarak birleştirilir

### 🔍 Özel Yürütücüleri Anlamak

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` implementasyonu ile string girişi kabul eder
- Mesajları bağlı tüm ajanlara yayınlar
- Eşzamanlı işleme başlatmak için `TurnToken` gönderir

**ConcurrentAggregationExecutor:**
- Ajan yanıtlarını almak için `IMessageHandler<ChatMessage>` implementasyonu
- Mesajları thread-safe şekilde toplar
- Tüm beklenen yanıtlar geldiğinde birleşim yapar
- Son çıktıyı `context.YieldOutputAsync()` kullanarak verir

### ⚡ Performans Faydaları

**Eşzamanlı vs Ardışık:**
- Ardışık: Agent1 (30s) → Agent2 (30s) = **Toplam 60 saniye**
- Eşzamanlı: Agent1 (30s) || Agent2 (30s) = **Toplam 30 saniye**

**Verimlilik artışı**: N eşzamanlı ajan için N× daha hızlı (iş yükü ve kaynaklara bağlı)

### 🛡️ Hata Yönetimi

İş akışı, bireysel ajan hatalarını nazikçe yönetir:
- Bir ajan başarısız olursa diğerleri işlemeye devam eder
- Toplayıcı zaman aşımı mantığı uygulayabilir
- Gerekirse kısmi sonuçlar döndürülebilir
