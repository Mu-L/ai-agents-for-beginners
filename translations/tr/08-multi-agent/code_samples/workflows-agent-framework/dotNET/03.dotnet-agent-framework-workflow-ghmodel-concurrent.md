# ⚡ Azure OpenAI ile Eşzamanlı Ajan İş Akışları (Yanıtlar API) (.NET)

## 📋 Yüksek Performanslı Paralel İşleme Eğitimi

Bu not defteri, Microsoft Agent Framework for .NET ve Azure OpenAI (Yanıtlar API) kullanarak **eşzamanlı iş akışı desenlerini** gösterir. Çoklu AI ajanlarını aynı anda çalıştırarak maksimum verimliliğe sahip, yüksek performanslı paralel işleme iş akışlarının nasıl oluşturulacağını öğreneceksiniz; koordinasyon ve veri tutarlılığı sağlanacaktır.

## 💻 Kodu Çalıştırma

Tam implementasyon `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` dosyasında mevcuttur. Bu dosya seyahat planlama için bir **Fan-Out/Fan-In eşzamanlı iş akışı** gösterir:

### 🏗️ **İş Akışı Mimarisi**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Temel Bileşenler:**

1. **ConcurrentStartExecutor**: Kullanıcı isteğini tüm ajanlara aynı anda iletir
2. **Araştırmacı Ajan**: Hedef ve cazibe noktalarını eşzamanlı analiz eder
3. **Planlayıcı Ajan**: Ayrıntılı seyahat planları eşzamanlı oluşturur
4. **ConcurrentAggregationExecutor**: Her iki ajandan gelen sonuçları toplar ve birleştirir

### 🎯 **Fan-Out/Fan-In Deseni**

Bu iş akışı klasik **Fan-Out/Fan-In** desenini gösterir:
- **Fan-Out**: Tek bir giriş mesajı aynı anda çoklu ajanlara yayımlanır
- **Eşzamanlı İşleme**: Birden fazla ajan aynı görevi paralel şekilde yürütür
- **Fan-In**: Tüm ajanlardan gelen sonuçlar toplanır ve tek bir çıktı haline getirilir

### 🔍 Özel Executor’ların Anlaşılması

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` arayüzünü uygular, string girdi kabul eder
- Mesajları bağlı tüm ajanlara yayar
- Eşzamanlı işlemi tetiklemek için `TurnToken` gönderir

**ConcurrentAggregationExecutor:**
- `IMessageHandler<ChatMessage>` arayüzünü uygular, ajan yanıtlarını alır
- Mesajları iş parçacığı güvenli şekilde toplar
- Beklenen tüm yanıtlar geldiğinde toplama yapar
- Son çıktıyı `context.YieldOutputAsync()` kullanarak verir

### ⚡ Performans Avantajları

**Eşzamanlı vs Ardışık:**
- Ardışık: Ajan1 (30s) → Ajan2 (30s) = **Toplam 60 saniye**
- Eşzamanlı: Ajan1 (30s) || Ajan2 (30s) = **Toplam 30 saniye**

**Verimlilik Artışı**: N eşzamanlı ajan için N× kadar hızlı (iş yükü ve kaynaklara bağlı olarak)

### 🛡️ Hata Yönetimi

İş akışı, bireysel ajan hatalarını nazikçe yönetir:
- Bir ajan başarısız olursa diğerleri işleme devam eder
- Toplayıcı zaman aşımı mantığı uygulayabilir
- Gerekirse kısmi sonuçlar döndürülebilir
