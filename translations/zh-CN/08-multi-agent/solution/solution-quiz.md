# 知识检测答案

## 问题1

**答案：A2。** 当任务具有明确的专业角色、不同的工具或权限，以及需要协调的清晰交接时，多代理是有用的。一个简单的支持机器人通常可以保持为单一代理。大量相同请求主要是一个扩展问题，本身并不需要多代理设计。

## 问题2

**答案：A1。** 当一套说明和工具能够处理任务时，单一代理通常更易于测试、观察和操作。访问多个工具并不自动要求多个代理。不同权限和审计要求的不同角色可能证明多代理设计是合理的。
