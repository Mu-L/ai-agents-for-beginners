## 💻 运行代码

完整实现位于 `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` 文件。本文件演示了用于旅行规划的<strong>扇出/扇入并发工作流</strong>：

### 🏗️ <strong>工作流架构</strong>

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**关键组件：**

1. **ConcurrentStartExecutor**：同时向所有代理广播用户请求
2. **Researcher Agent**：并发分析目的地和景点
3. **Planner Agent**：并发创建详细旅行计划
4. **ConcurrentAggregationExecutor**：收集并合并两个代理的结果

### 🎯 **扇出/扇入模式**

本工作流展示经典的<strong>扇出/扇入</strong>模式：
- <strong>扇出</strong>：一个输入消息同时广播给多个代理
- <strong>并发处理</strong>：多个代理并行处理相同任务
- <strong>扇入</strong>：收集所有代理结果并聚合成单个输出

### 🔍 理解自定义执行器

**ConcurrentStartExecutor：**
- 实现 `IMessageHandler<string>` 接收字符串输入
- 向所有连接的代理广播消息
- 发送 `TurnToken` 以触发并发处理

**ConcurrentAggregationExecutor：**
- 实现 `IMessageHandler<ChatMessage>` 接收代理响应
- 线程安全地收集消息
- 所有预期响应到达后进行聚合
- 使用 `context.YieldOutputAsync()` 产生最终输出

### ⚡ 性能优势

**并发 vs 顺序：**
- 顺序：代理1（30秒）→ 代理2（30秒）= **共 60 秒**
- 并发：代理1（30秒）|| 代理2（30秒）= **共 30 秒**

<strong>吞吐量提升</strong>：对于 N 个并发代理，速度提升最高可达 N 倍（取决于工作负载和资源）

### 🛡️ 错误处理

工作流优雅地处理个别代理失败：
- 一个代理失败，其他仍继续处理
- 聚合器可以实现超时逻辑
- 必要时返回部分结果
