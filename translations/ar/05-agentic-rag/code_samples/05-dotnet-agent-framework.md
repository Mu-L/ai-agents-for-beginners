# 🔍 استرجاع معزز للمؤسسات باستخدام Microsoft Foundry (.NET)

## 📋 أهداف التعلم

يعرض هذا الدفتر كيفية بناء أنظمة استرجاع معزز للإنتاج من الدرجة المؤسسية باستخدام إطار Microsoft Agent في .NET مع Microsoft Foundry. ستتعلم كيفية إنشاء وكلاء جاهزين للإنتاج يمكنهم البحث عبر المستندات وتقديم استجابات دقيقة وواعية للسياق مع أمان الشركات وقابليتها للتوسع.

**قدرات استرجاع معزز للمؤسسات التي ستبنيها:**
- 📚 **ذكاء المستندات**: معالجة مستندات متقدمة باستخدام خدمات Azure AI
- 🔍 **البحث الدلالي**: بحث متجه عالي الأداء مع ميزات مؤسسية
- 🛡️ **دمج الأمان**: التحكم في الوصول بناءً على الدور ونماذج حماية البيانات
- 🏢 **هندسة قابلة للتوسع**: أنظمة RAG جاهزة للإنتاج مع المراقبة

## 🎯 هندسة استرجاع معزز للمؤسسات

### المكونات الأساسية للمؤسسة
- **Microsoft Foundry**: منصة ذكاء اصطناعي للمؤسسات مُدارة مع الأمان والامتثال
- **وكلاء مستمرون**: وكلاء ذو حالة مع سجل المحادثة وإدارة السياق
- **إدارة مخزن المتجهات**: فهرسة واسترجاع مستندات بمعايير مؤسسية
- **تكامل الهوية**: توثيق Azure AD والتحكم في الوصول بناءً على الدور

### فوائد .NET للمؤسسات
- **سلامة النوع**: التحقق أثناء الترجمة لعمليات واسترجاع البيانات
- **أداء غير متزامن**: معالجة وبحث مستندات غير حاصرة
- **إدارة الذاكرة**: استخدام موارد فعال لمجموعات مستندات كبيرة
- **نماذج التكامل**: تكامل خدمات Azure الأصلية مع حقن الاعتماد

## 🏗️ هندسة تقنية

### خط أنابيب استرجاع معزز للمؤسسات
```
Document Upload → Security Validation → Vector Processing → Index Creation
                      ↓                    ↓                  ↓
User Query → Authentication → Semantic Search → Context Ranking → AI Response
```

### مكونات .NET الأساسية
- **Azure.AI.Agents.Persistent**: إدارة الوكلاء للمؤسسات مع حفظ الحالة
- **Azure.Identity**: توثيق مدمج للوصول الآمن إلى خدمات Azure
- **Microsoft.Agents.AI.AzureAI**: تنفيذ إطار عمل الوكلاء المحسن لـ Azure
- **System.Linq.Async**: عمليات LINQ متزامنة عالية الأداء

## 🔧 ميزات وفوائد المؤسسات

### الأمان والامتثال
- **تكامل Azure AD**: إدارة الهوية المؤسسية والتوثيق
- **الوصول بناءً على الدور**: أذونات دقيقة للوصول إلى المستندات والعمليات
- **حماية البيانات**: التشفير أثناء التخزين والنقل للمستندات الحساسة
- **تسجيل التدقيق**: تتبع شامل للأنشطة لمتطلبات الامتثال

### الأداء وقابلية التوسع
- **تجميع الاتصالات**: إدارة فعالة لاتصالات خدمات Azure
- **المعالجة غير الحاصرة**: عمليات غير متزامنة لسيناريوهات عالية الحمل
- **استراتيجيات التخزين المؤقت**: تخزين مؤقت ذكي للمستندات المكررة الوصول إليها
- **موازنة الحمل**: المعالجة الموزعة للنشر واسع النطاق

### الإدارة والمراقبة
- **فحوصات الصحة**: مراقبة مدمجة لمكونات نظام RAG
- **مؤشرات الأداء**: تحليلات مفصلة على جودة البحث وأوقات الاستجابة
- **معالجة الأخطاء**: إدارة استثناءات شاملة مع سياسات إعادة المحاولة
- **إدارة التهيئة**: إعدادات خاصة بالبيئة مع التحقق الصحيح

## ⚙️ المتطلبات الأساسية والإعداد

**بيئة التطوير:**
- .NET 9.0 SDK أو أحدث
- Visual Studio 2022 أو VS Code مع امتداد C#
- اشتراك Azure مع وصول إلى Microsoft Foundry

**حزم NuGet المطلوبة:**
```xml
<PackageReference Include="Microsoft.Extensions.AI" Version="9.9.0" />
<PackageReference Include="Azure.AI.Agents.Persistent" Version="1.2.0-beta.5" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

**إعداد توثيق Azure:**
```bash
# قم بتثبيت Azure CLI وتسجيل الدخول
az login
az account set --subscription "your-subscription-id"
```

**تهيئة البيئة:**
* تكوين Microsoft Foundry (يتم التعامل معه تلقائيًا عبر Azure CLI)
* تأكد من أنك مصادق على الاشتراك الصحيح في Azure

## 📊 أنماط استرجاع معزز للمؤسسات

### أنماط الأمان
- **أمان على مستوى المستند**: تحكم دقيق في الوصول لكل مستند
- **تصنيف البيانات**: وسم وحماية الحساسية تلقائيًا
- **سجلات التدقيق**: تسجيل شامل لجميع عمليات RAG
- **حماية الخصوصية**: الكشف عن بيانات التعريف الشخصية وإزالتها

## 🔒 ميزات أمان المؤسسات

### التوثيق والتفويض
```csharp
// Azure AD integrated authentication
var credential = new AzureCliCredential();
var agentsClient = new PersistentAgentsClient(endpoint, credential);

// Role-based access validation
if (!await ValidateUserPermissions(user, documentId))
{
    throw new UnauthorizedAccessException("Insufficient permissions");
}
```
