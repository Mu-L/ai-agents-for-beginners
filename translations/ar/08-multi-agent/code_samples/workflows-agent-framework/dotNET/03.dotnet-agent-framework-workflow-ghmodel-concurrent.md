# ⚡ تدفقات أعمال الوكلاء المتزامنة مع Azure OpenAI (Responses API) (.NET)

## 📋 درس المعالجة المتوازية عالية الأداء

يوضح هذا الدفتر **أنماط تدفقات العمل المتزامنة** باستخدام إطار عمل الوكلاء من Microsoft لـ .NET وAzure OpenAI (Responses API). ستتعلم كيفية إنشاء تدفقات عمل معالجة متوازية عالية الأداء تزيد من الإنتاجية عبر تنفيذ وكلاء ذكاء اصطناعي متعددين في نفس الوقت مع الحفاظ على التنسيق وتناسق البيانات.

## 🎯 أهداف التعلم

### 🚀 **أساسيات المعالجة المتزامنة**
- **تشغيل الوكلاء المتوازي**: تشغيل عدة وكلاء ذكاء اصطناعي في نفس الوقت لأداء أقصى
- **أنماط Async/Await**: الاستفادة من نموذج البرمجة غير المتزامنة في .NET لتحقيق تنافسية فعالة
- **Azure OpenAI (Responses API)**: تنسيق عدة مكالمات متزامنة إلى Azure OpenAI Responses API
- **إدارة الموارد**: إدارة موارد نماذج الذكاء الاصطناعي بكفاءة عبر العمليات المتزامنة

### 🏗️ **هندسة التزامن المتقدم**
- **الموازاة المعتمدة على المهام**: استخدام مكتبة المهام المتوازية في .NET لأداء متزامن مثالي
- **أنماط التزامن**: تنسيق الوكلاء المتزامنين مع تجنب حالات السباق
- **موازنة الأحمال**: توزيع العمل بفعالية عبر القدرة المتاحة للمعالجة المتزامنة
- **تحمل الأخطاء**: التعامل مع فشل وكيل فردي دون إيقاف تدفق العمل كله

### 🏢 **تطبيقات المؤسسات المتزامنة**
- **معالجة مستندات عالية الحجم**: معالجة مستندات متعددة في نفس الوقت

## 💻 تشغيل الكود

التنفيذ الكامل متوفر في `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. يوضح هذا الملف **تدفق عمل Fan-Out/Fan-In متزامن** لتخطيط السفر:

### 🏗️ **هندسة تدفق العمل**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**المكونات الأساسية:**

1. **ConcurrentStartExecutor**: يذيع طلب المستخدم إلى كل الوكلاء في نفس الوقت
2. **وكيل الباحث**: يحلل الوجهات والمعالم بشكل متزامن
3. **وكيل المخطط**: يُعد خطط سفر مفصلة متزامنة
4. **ConcurrentAggregationExecutor**: يجمع ويدمج النتائج من كلا الوكلين

### 🎯 **نمط Fan-Out/Fan-In**

يوضح هذا العمل النمط الكلاسيكي **Fan-Out/Fan-In**:
- **Fan-Out**: رسالة إدخال واحدة تبث إلى عدة وكلاء في نفس الوقت
- **المعالجة المتزامنة**: يعمل عدة وكلاء بالتوازي على نفس المهمة
- **Fan-In**: تُجمع نتائج كل الوكلاء وتدمج في إخراج واحد

### 📝 المخرجات المتوقعة

سيقوم تدفق العمل بـ:
1. **بث الطلب**: إرسال "خطط رحلة إلى سياتل في ديسمبر" إلى كل الوكلاء
2. **المعالجة المتزامنة**: يعمل الوكلاء معاً في الوقت ذاته:
   - الباحث يحدد المعالم والتفاصيل
   - المخطط يبدع الخطة واللوجستيات
3. **الدمج**: دمج الردود لنتيجة شاملة
4. **عرض النتائج**: إظهار خطة السفر المدمجة بكل المعلومات

### 🎯 تطبيقات العالم الحقيقي

هذا النمط المتزامن مثالي لـ:
- **إنشاء المحتوى**: عدة كُتاب يكتبون أجزاء مختلفة في نفس الوقت
- **مراجعة الكود**: عدة مراجعين يحللون الكود من وجهات نظر مختلفة
- **أبحاث السوق**: تحليل موازٍ لقطاعات سوق مختلفة
- **معالجة المستندات**: استخراج وتحليل وتحقق متزامن
- **تحليل متعدد المنظور**: الحصول على وجهات نظر متنوعة على نفس المدخل

### 🔍 فهم المنفذين المخصصين

**ConcurrentStartExecutor:**
- ينفذ `IMessageHandler<string>` لقبول إدخال نصي
- يذيع الرسائل إلى كافة الوكلاء المتصلين
- يرسل `TurnToken` لتحفيز المعالجة المتزامنة

**ConcurrentAggregationExecutor:**
- ينفذ `IMessageHandler<ChatMessage>` لتلقي استجابات الوكلاء
- يجمع الرسائل بطريقة آمنة للخيوط
- يجمع عندما تصل كل الردود المتوقعة
- ينتج الإخراج النهائي باستخدام `context.YieldOutputAsync()`

### ⚡ فوائد الأداء

**المتزامن مقابل المتسلسل:**
- متسلسل: الوكيل1 (30ث) → الوكيل2 (30ث) = **60 ثانية إجمالاً**
- متزامن: الوكيل1 (30ث) || الوكيل2 (30ث) = **30 ثانية إجمالاً**

**تحسين الإنتاجية**: أسرع حتى N× مع N وكلاء متزامنين (حسب عبء العمل والموارد)
