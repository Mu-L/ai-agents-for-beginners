# 🔄 سير عمل الوكلاء الأساسي مع Azure OpenAI (Responses API) (.NET)

## 📋 درس تنظيم سير العمل

يوضح هذا الدفتر كيفية بناء **سير عمل وكلاء** متقدم باستخدام إطار عمل Microsoft Agent لـ .NET وAzure OpenAI (Responses API). ستتعلم إنشاء عمليات تجارية متعددة الخطوات حيث يتعاون وكلاء الذكاء الاصطناعي لإنجاز مهام معقدة من خلال أنماط تنظيمية مُهيكلة.

## 🎯 أهداف التعلم

### 🏗️ **أساسيات هندسة سير العمل**
- **منشئ سير العمل**: تصميم وتنظيم عمليات الذكاء الاصطناعي المعقدة متعددة الخطوات
- **تنسيق الوكلاء**: تنسيق عدة وكلاء متخصصين داخل سير العمل
- **Azure OpenAI (Responses API)**: استخدام API ردود Azure OpenAI ضمن سير العمل
- **تصميم سير العمل بصريًا**: إنشاء وتصوير هياكل سير العمل لفهم أفضل

### 🔄 **أنماط تنظيم العمليات**
- **المعالجة التسلسلية**: ربط مهام الوكلاء المتعددة بالترتيب المنطقي
- **إدارة الحالة**: الحفاظ على السياق وتدفق البيانات عبر مراحل سير العمل
- **معالجة الأخطاء**: تنفيذ استرداد قوي من الأخطاء ومرونة سير العمل
- **تحسين الأداء**: تصميم سير عمل فعال لعمليات على نطاق المؤسسات

### 🏢 **تطبيقات سير العمل المؤسسية**
- **أتمتة العمليات التجارية**: أتمتة سير العمل المعقد في المؤسسات
- **خط إنتاج المحتوى**: سير العمل التحريري مع مراحل المراجعة والموافقة
- **أتمتة خدمة العملاء**: حل استفسارات العملاء متعدد الخطوات
- **سير عمل معالجة البيانات**: سير عمل ETL مع تحويل مدعوم بالذكاء الاصطناعي

## ⚙️ المتطلبات والإعداد

### 📦 **حزم NuGet المطلوبة**

يستخدم عرض سير العمل هذا عدة حزم رئيسية لـ .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **تكوين Azure OpenAI**

**إعداد البيئة (ملف .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**الوصول إلى Azure OpenAI:**
1. إنشاء مورد Azure OpenAI عبر بوابة Azure
2. نشر نموذج (مثلاً، `gpt-5-mini`) وتسجيل اسم النشر
3. تسجيل الدخول باستخدام `az login` وتكوين متغيرات البيئة كما هو موضح أعلاه

### 🏗️ **نظرة عامة على هندسة سير العمل**

```mermaid

    C --> E[Agent 2: مراجع المحتوى]
```

**المكونات الرئيسية:**
- **WorkflowBuilder**: محرك التنظيم الرئيسي لتصميم سير العمل
- **AIAgent**: وكلاء متخصصين فرديين مع قدرات محددة
- **عميل Azure OpenAI**: تكامل API ردود Azure OpenAI
- **سياق التنفيذ**: إدارة الحالة وتدفق البيانات بين مراحل سير العمل

## 💻 تشغيل الكود

التنفيذ الكامل متوفر في الملف `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. يوضح هذا الملف:

2. **إعداد عميل Azure OpenAI** - تهيئة العميل لاستخدام Azure OpenAI Responses API
3. **إنشاء الوكيل** - تعريف وكلاء متخصصين (الاستقبال والكونسيرج)
4. **منشئ سير العمل** - إنشاء سير عمل متعدد الوكلاء مع المعالجة التسلسلية
5. **تشغيل سير العمل** - تنفيذ سير العمل مع نتائج متدفقة

### 🚀 تشغيل المثال

