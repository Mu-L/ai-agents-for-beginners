# ⏩ تدفقات عمل الوكيل التسلسلية مع Azure OpenAI (واجهة برمجة الاستجابات) (.NET)

## 📋 برنامج تعليمي متقدم للمعالجة التسلسلية

يوضح هذا الدفتر **أنماط سير العمل التسلسلي** باستخدام إطار عمل الوكيل من مايكروسوفت لـ .NET وAzure OpenAI (واجهة برمجة الاستجابات). ستتعلم كيفية بناء خطوط معالجة متطورة خطوة بخطوة حيث ينفذ الوكلاء بترتيب محدد، مع كل مرحلة تبني على نتائج المرحلة السابقة.

## 🎯 أهداف التعلم

### 🔄 **هندسة المعالجة التسلسلية**
- **تصميم سير العمل الخطي**: إنشاء خطوط معالجة خطوة بخطوة مع تبعيات واضحة
- **إدارة الحالة**: الحفاظ على السياق وتدفق البيانات عبر مراحل سير العمل التسلسلية
- **Azure OpenAI (واجهة برمجة الاستجابات)**: الاستفادة من نماذج Azure OpenAI في سير عمل .NET متعدد المراحل
- **أنماط خطوط الإنتاج المؤسساتية**: بناء أنظمة معالجة تسلسلية جاهزة للإنتاج

### 🏗️ **أنماط تسلسلية متقدمة**
- **معالجة بوابة المرحلة**: تنفيذ نقاط تحقق للتحقق بين مراحل سير العمل
- **حفظ السياق**: الحفاظ على الحالة والمعرفة المتراكمة عبر جميع المراحل
- **انتشار الأخطاء**: التعامل مع الأعطال بسلاسة في سلاسل المعالجة التسلسلية
- **تحسين الأداء**: تنفيذ تسلسلي فعال مع أقل تكلفة إضافية

### 🏢 **تطبيقات مؤسساتية تسلسلية**
- **خط معالجة المستندات**: تحليل متعدد المراحل للمستندات، التحويل، والتحقق
- **سير عمل ضمان الجودة**: عمليات مراجعة، تحقق، وموافقة متتابعة
- **خط إنتاج المحتوى**: البحث → الكتابة → التحرير → المراجعة → النشر
- **أتمتة العمليات التجارية**: سير عمل تجاري متعدد الخطوات مع تبعيات مراحل واضحة

## ⚙️ المتطلبات الأساسية والإعداد

### 📦 **حزم NuGet المطلوبة**

حزم أساسية لسير العمل التسلسلي في .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **تكوين Azure OpenAI**

**إعداد البيئة (ملف .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**إدارة التكوين:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **هندسة سير العمل التسلسلي**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: وكيل التحليل]
    B --> C[نقطة التحقق 1]
    C --> D[Stage 2: وكيل المعالجة]
    D --> E[نقطة التحقق 2]
    E --> F[Stage 3: وكيل التحقق]
    F --> G[الناتج النهائي]
    
    H[سياق الحالة] --> B
    H --> D
    H --> F
    
    I[أزور أوبن إيه آي (واجهة برمجة تطبيقات الردود)] --> B
    I --> D
    I --> F
```

**المكونات الرئيسية:**
- **وكلاء تسلسليون**: وكلاء متخصصون لكل مرحلة معالجة
- **سياق الحالة**: يحافظ على البيانات والقرارات المتراكمة عبر المراحل
- **نقاط تحقق**: نقاط تحقق بين المراحل لضمان الجودة والاتساق
- **عميل Azure OpenAI**: وصول متسق إلى نموذج الذكاء الاصطناعي عبر جميع مراحل سير العمل

## 🏢 **فوائد المؤسساتية التسلسلية**

### 🎯 **الموثوقية والجودة**
- **المعالجة الحتمية**: نتائج متسقة وقابلة للتكرار عبر مراحل منظمة
- **بوابات الجودة**: نقاط تحقق للتحقق من الجودة في كل مرحلة
- **عزل الأخطاء**: مشاكل في مرحلة واحدة لا تنتقل إلى المراحل التالية
- **سجلات التدقيق**: تتبع كامل للقرارات والتحويلات في كل مرحلة

### 📈 **القابلية للتوسع والأداء**
- **تصميم معياري**: يمكن تحسين كل مرحلة بشكل مستقل
- **إدارة الموارد**: تخصيص فعال لموارد نموذج الذكاء الاصطناعي عبر المراحل
- **تحسين الحالة**: نقل الحد الأدنى من الحالة بين المراحل لأداء مثالي
- **مجموعات المراحل المتوازية**: يمكن تشغيل عدة سير عمل تسلسلية بالتوازي

### 🔒 **الأمان والامتثال**
- **أمان مستوى المرحلة**: سياسات أمان مختلفة لمراحل معالجة مختلفة
- **التحقق من البيانات**: ضمان سلامة البيانات والامتثال في كل نقطة تحقق
- **التحكم في الوصول**: أذونات دقيقة لمراحل سير العمل المختلفة
- **الامتثال التنظيمي**: الوفاء بمتطلبات اللوائح من خلال معالجة منظمة

### 📊 **المراقبة والتحليلات**
- **مقاييس مستوى المرحلة**: مراقبة الأداء لكل مرحلة في سير العمل
- **تحديد نقاط الاختناق**: التعرف على المراحل البطيئة وتحسينها
- **مقاييس الجودة**: تتبع جودة ومعدلات النجاح في كل مرحلة
- **تحسين العمليات**: التحسين المستمر بناءً على تحليلات مستوى المرحلة

لنبني خطوط معالجة ذكاء اصطناعي تسلسلية قوية! 🚀

## 💻 تشغيل الشيفرة

التنفيذ الكامل متاح في `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. يوضح هذا الملف **سير عمل تحليل أثاث ثلاثي المراحل**:

1. **المرحلة 1 - وكيل المبيعات**: يحلل صور الأثاث ويقدم اقتراحات شراء
2. **المرحلة 2 - وكيل السعر**: يقدم تفصيلات تسعير مفصلة وخيارات ميزانية
3. **المرحلة 3 - وكيل الاقتباس**: ينشئ وثيقة اقتباس احترافية بتنسيق Markdown

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

كل وكيل:
- يستلم الناتج من المرحلة السابقة كسياق
- يبني على التحليل السابق بخبرة متخصصة
- يحافظ على استمرارية سير العمل من خلال إدارة الحالة

### 🚀 تشغيل المثال

**المتطلبات الأساسية:**
- ضع صورة أثاث في `../imgs/home.png` (أو حدّث متغيّر `imgPath`)
- قم بتكوين ملف `.env` بنقطة نهاية Azure OpenAI والنشر الخاص بك، ثم سجّل الدخول باستخدام `az login`

```bash
# اجعل السكريبت قابلاً للتنفيذ (يونكس/لينكس/ماك أو إس)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# شغّل سير العمل المتسلسل
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

أو على ويندوز:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 الناتج المتوقع

سير العمل سوف:
1. **وكيل المبيعات**: يحدد عناصر الأثاث من الصورة ويقدم توصيات
2. **وكيل السعر**: يضيف تحليل تسعير تفصيلي مع طبقات الميزانية وتوصيات التسوق
3. **وكيل الاقتباس**: ينشئ وثيقة اقتباس منسقة مع جميع المعلومات مجمعة

الناتج النهائي سيكون اقتباس أثاث شامل واحترافي استنادًا إلى تحليل الصورة.


**تغيير التدفق التسلسلي:**
```csharp
// Add or reorder workflow stages
var workflow = new WorkflowBuilder(salesagent)
```

### 🔍 فهم تدفق الحالة

كل وكيل في التسلسل يستلم:
- **الإدخال الأصلي**: رسالة المستخدم الأولية (صورة + نص)
- **مخرجات الوكلاء السابقة**: جميع ردود الوكلاء السابقة في سجل المحادثة
- **السياق المتراكم**: الحالة الكاملة المُحتفظ بها طوال سير العمل

هذا يمكّن معالجة متعددة المراحل متطورة حيث يبني كل وكيل على سياق شامل من جميع المراحل السابقة.
