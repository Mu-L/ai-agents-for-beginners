# 🔀 سير عمل الوكلاء الشرطي باستخدام Microsoft Foundry (.NET)

## 📋 درس تعليمي لسير عمل يعتمد على القرار الذكي

توضح هذه الدفتر **أنماط سير العمل الشرطية** باستخدام Microsoft Foundry وإطار عمل الوكلاء من Microsoft لـ .NET. ستتعلم كيفية بناء سير عمل متقدم يعتمد على القرار الذي يوجه المعالجة بذكاء بناءً على تحليل الذكاء الاصطناعي والقواعد التجارية والظروف الديناميكية لأتمتة على مستوى المؤسسات.

## 🎯 أهداف التعلم

### 🧠 **هيكل القرار الذكي**
- **تنفيذ المنطق الشرطي**: بناء أشجار قرار معقدة مع نقاط تشعب متعددة
- **التوجيه المدعوم بالذكاء الاصطناعي**: استخدام نماذج Microsoft Foundry لاتخاذ قرارات توجيه ذكية
- **تكييف سير العمل الديناميكي**: تعديل سلوك سير العمل بناءً على التحليل والظروف أثناء التشغيل
- **تكامل قواعد المؤسسات**: دمج المنطق التجاري ومتطلبات الامتثال في سير العمل

### 🔀 **أنماط شرطية متقدمة**
- **اتخاذ قرارات متعددة المعايير**: تقييم عوامل متعددة لاتخاذ قرارات التوجيه
- **المعالجة الواعية للسياق**: اتخاذ قرارات بناءً على سياق وتاريخ سير العمل المتراكم
- **تعديل سير العمل التكيفي**: تعديل مسارات المعالجة ديناميكيًا بناءً على الظروف في الوقت الحقيقي
- **تكامل محرك القواعد**: تنفيذ محركات قواعد أعمال متقدمة داخل سير العمل

### 🏢 **تطبيقات شرطية على مستوى المؤسسة**
- **تصنيف وتوجيه الوثائق**: تصنيف الوثائق تلقائيًا وتوجيهها إلى سير العمل المناسب
- **توزيع خدمة العملاء**: توجيه ذكي لاستفسارات العملاء إلى فرق متخصصة
- **المعالجة وفق الامتثال والمخاطر**: تطبيق عمليات تحقق ومراجعة مختلفة بناءً على تقييم المخاطر
- **سير عمل ضمان الجودة**: توجيه المحتوى عبر عمليات مراجعة مناسبة بناءً على مؤشرات الجودة

### 🏗️ **هيكل سير العمل الشرطي**

```mermaid
graph TD
    A[Input: مخطط المحتوى] --> B[منفذ المسودة
    B --> C[منفذ مراجعة المحتوى]
    C --> D{قرار المراجعة}
    
    D -->|نعم - موافق عليه| E[منفذ النشر]
    D -->|لا - مرفوض| F[منفذ إرسال المراجعة]
    
    E --> G[حفظ كملف ماركداون]
    F --> H[إشعار بالمراجعات]
    
    I[مايكروسوفت فاوندري] --> B
    I --> C
    I --> E
    
    J[بينغ جراندنغ] --> B
```

**المكونات الرئيسية:**
- **منفذ المسودات**: وكيل ذكاء اصطناعي ينشئ مسودات مبدئية من المخططات
- **منفذ مراجعة المحتوى**: وكيل ذكاء اصطناعي يقيم جودة المسودات والامتثال
- **التوجيه الشرطي**: منطق القرار الذي يوجه بناءً على نتائج المراجعة
- **مسارات النشر/المراجعة**: مسارات معالجة منفصلة للمحتوى الموافق عليه مقابل المرفوض
- **إدارة الحالة**: يحافظ على سياق المحتوى والمراجعة طوال سير العمل

## 💻 تشغيل الكود

التنفيذ الكامل متوفر في `04.dotnet-agent-framework-workflow-aifoundry-condition.cs`. يوضح هذا سير عمل **إنتاج محتوى مع بوابات جودة**:

### 🏗️ **هيكل سير العمل**

```
Content Outline → Draft Creation → Quality Review → Conditional Routing:
                                                      ├─ Approved (>200 words) → Publish
                                                      └─ Rejected (<200 words) → Review Notification
```

**الوكلاء في سير العمل:**
1. **وكيل التبشير**: ينشئ مسودات الدروس من المخططات مع تدعيم Bing
2. **وكيل مراجعة المحتوى**: يقيم جودة المسودة (عدد الكلمات، الاكتمال)
3. **وكيل النشر**: يحفظ المحتوى الموافق عليه كملفات Markdown مع طابع زمني

**المنفذون المخصصون:**
1. **DraftExecutor**: ينسق إنشاء المسودات
2. **ContentReviewExecutor**: يقوم بتقييم الجودة
3. **PublishExecutor**: يتعامل مع نشر المحتوى الموافق عليه
4. **SendReviewExecutor**: يدير إشعارات المحتوى المرفوض

### 📝 المخرجات المتوقعة

سير العمل سوف:
1. **إنشاء وكلاء**: تهيئة ثلاثة وكلاء متخصصين من Microsoft Foundry
2. **إنشاء المسودة**: وكيل التبشير ينشئ مسودة الدرس من المخطط
3. **مراجعة المحتوى**: وكيل مراجعة المحتوى يقيم جودة المسودة
4. **التوجيه الشرطي**:
   - **إذا تم الموافقة (>200 كلمة)**: منفذ النشر يحفظ الملف كـ Markdown

5. **عرض النتائج**: عرض النتيجة النهائية لسير العمل

### 🔍 فهم المنطق الشرطي

**دالة الشرط:**
```csharp
public Func<object?, bool> GetCondition(string expectedResult) =>
    reviewResult => reviewResult is ReviewResult review && review.Result == expectedResult;
```

تنشئ هذه الدالة شرطًا يتحقق من:
1. إذا كانت النتيجة من نوع `ReviewResult`
2. مقارنة خاصية `Result` بالقيمة المتوقعة
3. تعيد صحيح/خطأ لتحديد التوجيه

**حواف سير العمل مع الشروط:**
```csharp
.AddEdge(contentReviewerExecutor, publishExecutor, condition: GetCondition("Yes"))
.AddEdge(contentReviewerExecutor, sendReviewerExecutor, condition: GetCondition("No"))
```

// Define response structure
public class ReviewResult

.AddEdge(contentReviewerExecutor, publishExecutor, condition: GetCondition("Yes"))

**تنفيذ مراجعة متعددة المستويات:**

```csharp

.AddEdge(editorialReviewer, publishExecutor, condition: GetCondition("EditPass"))
```
