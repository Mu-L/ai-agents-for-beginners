# ⏩ Послідовні робочі процеси агентів з Azure OpenAI (Responses API) (.NET)

## 📋 Поглиблений послідовний підручник з обробки

Цей ноутбук демонструє **шаблони послідовних робочих процесів** із використанням Microsoft Agent Framework для .NET і Azure OpenAI (Responses API). Ви навчитеся створювати складні, поетапні конвеєри обробки, де агенти виконуються у певному порядку, а кожен етап базується на результатах попереднього етапу.

## 🎯 Навчальні цілі

### 🔄 **Архітектура послідовної обробки**
- **Лінійний дизайн робочого процесу**: Створювати покрокові конвеєри обробки з чіткими залежностями
- **Управління станом**: Підтримувати контекст і потік даних через послідовні етапи робочого процесу
- **Azure OpenAI (Responses API)**: Використовувати моделі Azure OpenAI у багатоступеневих робочих процесах .NET
- **Шаблони корпоративних конвеєрів**: Створювати готові до виробництва системи послідовної обробки

### 🏗️ **Поглиблені послідовні шаблони**
- **Обробка з контрольно-пропускними пунктами**: Впроваджувати перевірочні контрольні точки між етапами робочого процесу
- **Збереження контексту**: Підтримувати стан і накопичені знання на всіх етапах
- **Поширення помилок**: Гнучко обробляти збої в послідовних ланцюгах обробки
- **Оптимізація продуктивності**: Ефективне послідовне виконання з мінімальними накладними витратами

### 🏢 **Корпоративні послідовні застосунки**
- **Конвеєр обробки документів**: Багатоступенева аналіз, трансформація та перевірка документів
- **Робочі процеси контролю якості**: Послідовний перегляд, перевірка та затвердження
- **Конвеєр виробництва контенту**: Дослідження → Написання → Редагування → Перегляд → Публікація
- **Автоматизація бізнес-процесів**: Багатокрокові бізнес-робочі процеси з чіткими залежностями етапів

```xml
<!-- Local Agent Framework References -->

```

### 🔑 **Конфігурація Azure OpenAI**

**Налаштування середовища (.env файл):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Управління конфігурацією:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **Архітектура послідовного робочого процесу**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Агент аналізу]
    B --> C[Контрольна точка 1]
    C --> D[Stage 2: Агент обробки]
    D --> E[Контрольна точка 2]
    E --> F[Stage 3: Агент валідації]
    F --> G[Остаточний результат]
    
    H[Контекст стану] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API відповідей)] --> B
    I --> D
    I --> F
```

**Ключові компоненти:**
- **Послідовні агенти**: Спеціалізовані агенти для кожного етапу обробки
- **Контекст стану**: Підтримує накопичені дані та рішення на всіх етапах
- **Контрольно-пропускні пункти**: Точки перевірки між етапами для забезпечення якості та послідовності
- **Клієнт Azure OpenAI**: Послідовний доступ до AI-моделі на всіх етапах робочого процесу

## 💻 Запуск коду

Повна реалізація доступна у файлі `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Цей файл демонструє **триетапний робочий процес аналізу меблів**:

1. **Етап 1 - Агент продажів**: Аналізує зображення меблів і надає рекомендації щодо покупки
2. **Етап 2 - Агент цін**: Надає детальний розподіл цін і варіанти бюджету
3. **Етап 3 - Агент пропозицій**: Генерує професійний документ пропозиції у форматі Markdown

### 🏗️ **Архітектура робочого процесу**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Кожен агент:
- Отримує вихідні дані з попереднього етапу як контекст
- Побудовує аналіз на основі попередніх результатів із спеціалізованими знаннями
- Підтримує безперервність робочого процесу через управління станом

### 🚀 Запуск прикладу

**Передумови:**
- Розмістіть зображення меблів у `../imgs/home.png` (або оновіть змінну `imgPath`)
- Налаштуйте ваш `.env` файл з вашим Azure OpenAI endpoint і deployment, потім увійдіть в систему за допомогою `az login`

```bash
# Зробіть скрипт виконуваним (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Запустіть послідовний робочий процес
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Або у Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Очікуваний результат

Робочий процес:
1. **Агент продажів**: ідентифікує предмети меблів на зображенні та надає рекомендації
2. **Агент цін**: додає детальний аналіз цін із бюджетними категоріями та рекомендаціями для покупок
3. **Агент пропозицій**: генерує форматований документ пропозиції з усією узагальненою інформацією

Підсумковий результат буде комплексною, професійною комерційною пропозицією на основі аналізу зображення меблів.

```csharp
// Add or reorder workflow stages
var workflow = new WorkflowBuilder(salesagent)
    .AddEdge(salesagent, priceagent)
    .AddEdge(priceagent, quoteagent)

    .Build();
```

Кожен агент у послідовності отримує:
- **Початковий вхід**: Початкове повідомлення користувача (зображення + текст)
