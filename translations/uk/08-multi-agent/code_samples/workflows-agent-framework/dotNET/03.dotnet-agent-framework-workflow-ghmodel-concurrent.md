# ⚡ Конкурентні робочі процеси агентів з Azure OpenAI (Responses API) (.NET)

## 📋 Посібник із високопродуктивної паралельної обробки

Цей блокнот демонструє **конкурентні шаблони робочих процесів** за допомогою Microsoft Agent Framework для .NET та Azure OpenAI (Responses API). Ви дізнаєтесь, як створювати високопродуктивні робочі процеси паралельної обробки, що максимізують пропускну здатність шляхом одночасного виконання декількох агентів штучного інтелекту, одночасно підтримуючи координацію та узгодженість даних.

## 💻 Запуск коду

Повна реалізація наведена у файлі `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Цей файл демонструє **робочий процес Fan-Out/Fan-In** для планування подорожей:

### 🏗️ **Архітектура робочого процесу**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Ключові компоненти:**

1. **ConcurrentStartExecutor**: Розсилає запит користувача одночасно всім агентам
2. **Researcher Agent**: Конкурентно аналізує напрямки та атракції
3. **Planner Agent**: Конкурентно створює детальні плани подорожей
4. **ConcurrentAggregationExecutor**: Збирає та об’єднує результати від обох агентів

### 🎯 **Шаблон Fan-Out/Fan-In**

Цей робочий процес демонструє класичний шаблон **Fan-Out/Fan-In**:
- **Fan-Out**: Один вхідний запит розсилається кільком агентам одночасно
- **Конкурентна обробка**: Кілька агентів працюють паралельно над тим самим завданням
- **Fan-In**: Результати всіх агентів зібрані та агреговані в єдиний вихідний результат

### 🔍 Розуміння користувацьких виконавців

**ConcurrentStartExecutor:**
- Реалізує `IMessageHandler<string>` для прийняття рядкових вхідних даних
- Розсилає повідомлення всім підключеним агентам
- Надсилає `TurnToken` для запуску конкурентної обробки

**ConcurrentAggregationExecutor:**
- Реалізує `IMessageHandler<ChatMessage>` для отримання відповідей агентів
- Збирає повідомлення у потокобезпечний спосіб
- Агреґує, коли отримано всі очікувані відповіді
- Повертає фінальний результат через `context.YieldOutputAsync()`

### ⚡ Переваги продуктивності

**Конкурентно проти послідовно:**
- Послідовно: Agent1 (30с) → Agent2 (30с) = **загалом 60 секунд**
- Конкурентно: Agent1 (30с) || Agent2 (30с) = **загалом 30 секунд**

**Покращення пропускної здатності**: До N× швидше для N конкурентних агентів (залежить від навантаження та ресурсів)
