# 🔄 Основні робочі процеси агентів із Azure OpenAI (Responses API) (.NET)

## 📋 Підручник з оркестрації робочих процесів

Цей ноутбук демонструє, як створювати складні **агентські робочі процеси** за допомогою Microsoft Agent Framework для .NET та Azure OpenAI (Responses API). Ви навчитеся створювати багатоступеневі бізнес-процеси, де AI-агенти співпрацюють для виконання складних завдань через структуровані патерни оркестрації.

## 🎯 Цілі навчання

### 🏗️ **Основи архітектури робочих процесів**
- **Побудовник робочих процесів**: Проєктування та оркестрація складних багатоступеневих AI-процесів
- **Координація агентів**: Координування кількох спеціалізованих агентів у робочих процесах
- **Azure OpenAI (Responses API)**: Використання Azure OpenAI Responses API у робочих процесах
- **Візуальне проєктування робочих процесів**: Створення та візуалізація структури робочих процесів для кращого розуміння

### 🔄 **Патерни оркестрації процесів**
- **Послідовна обробка**: Ланцюжок кількох завдань агентів у логічному порядку
- **Управління станом**: Підтримка контексту та потоку даних між етапами робочого процесу
- **Обробка помилок**: Реалізація надійного відновлення після помилок і стійкості робочого процесу
- **Оптимізація продуктивності**: Проєктування ефективних робочих процесів для операцій корпоративного рівня

### 🏢 **Корпоративні застосування робочих процесів**
- **Автоматизація бізнес-процесів**: Автоматизація складних організаційних робочих процесів
- **Потік виробництва контенту**: Редакційні робочі процеси з етапами рецензування та затвердження
- **Автоматизація обслуговування клієнтів**: Багатоступенева обробка запитів клієнтів
- **Обробка даних у робочих процесах**: ETL-процеси з трансформацією на базі AI

## ⚙️ Вимоги та налаштування

### 📦 **Необхідні пакети NuGet**

Для демонстрації цього робочого процесу використовуються кілька ключових пакетів .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Налаштування Azure OpenAI**

**Налаштування середовища (.env файл):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Доступ до Azure OpenAI:**
1. Створіть ресурс Azure OpenAI у порталі Azure
2. Розгорніть модель (наприклад, `gpt-5-mini`) та зафіксуйте назву розгортання
3. Увійдіть за допомогою `az login` та налаштуйте змінні середовища, як зазначено вище

### 🏗️ **Огляд архітектури робочого процесу**

```mermaid
graph TD
    A[Конструктор робочих процесів] --> B[Реєстр агентів]
    B --> C[Двигун виконання робочих процесів]

    C --> E[Agent 2: Рецензент контенту] 
    D --> F[Результати робочого процесу]
    E --> F
    G[Azure OpenAI (Responses API)] --> D
    G --> E
```

**Ключові компоненти:**
- **WorkflowBuilder**: Основний рушій оркестрації для проєктування робочих процесів
- **AIAgent**: Індивідуальні спеціалізовані агенти з конкретними можливостями
- **Клієнт Azure OpenAI**: Інтеграція з Azure OpenAI Responses API
- **Контекст виконання**: Керує станом і потоком даних між етапами робочого процесу

## 💻 Запуск коду

Повна реалізація доступна у файлі `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Цей файл демонструє:

1. **Налаштування середовища** - Завантаження конфігурації Azure OpenAI з файлу `.env`
2. **Налаштування клієнта Azure OpenAI** - Конфігурація клієнта для використання Azure OpenAI Responses API
3. **Створення агентів** - Визначення спеціалізованих агентів (Front Desk і Concierge)
4. **Побудовник робочого процесу** - Створення багатонаповного робочого процесу з послідовною обробкою
5. **Виконання робочого процесу** - Запуск робочого процесу з потоковим виводом результатів

### 🚀 Запуск прикладу

```bash
# Зробіть скрипт виконуваним (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Запустіть робочий процес
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Або на Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Очікуваний результат

Робочий процес:
1. Прийме ваш запит на подорож («Я хотів би поїхати до Парижа»)
2. Агент Front Desk надасть початкову рекомендацію
3. Агент Concierge перегляне і доопрацює рекомендацію
4. Кінцевий вивід відображає повний потік розмови
