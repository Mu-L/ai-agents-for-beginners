# 🔄 Основні робочі процеси агентів із Azure OpenAI (Responses API) (.NET)

## 📋 Підручник з оркестрації робочих процесів

Ця записна книжка демонструє, як створювати складні **робочі процеси агентів** за допомогою Microsoft Agent Framework для .NET і Azure OpenAI (Responses API). Ви навчитеся створювати багатокрокові бізнес-процеси, де агенти зі штучним інтелектом співпрацюють, щоб виконувати складні завдання через структуровані оркестраційні патерни.

## 🎯 Цілі навчання

### 🏗️ **Основи архітектури робочих процесів**
- **Створювач робочих процесів**: Створення та оркестрація складних багатокрокових процесів із ШІ
- **Координація агентів**: Координація кількох спеціалізованих агентів у робочих процесах
- **Azure OpenAI (Responses API)**: Використання Azure OpenAI Responses API у робочих процесах
- **Візуальне проєктування робочих процесів**: Створення та візуалізація структур робочих процесів для кращого розуміння

### 🔄 **Патерни оркестрації процесів**
- **Послідовна обробка**: Ланцюжок кількох завдань агентів у логічному порядку
- **Управління станом**: Підтримання контексту та потоку даних на різних етапах робочого процесу
- **Обробка помилок**: Реалізація надійного відновлення після помилок і стійкості робочих процесів
- **Оптимізація продуктивності**: Проєктування ефективних робочих процесів для масштабування на рівні підприємства

### 🏢 **Застосування робочих процесів у підприємствах**
- **Автоматизація бізнес-процесів**: Автоматизація складних організаційних робочих процесів
- **Процес виробництва контенту**: Редакційні робочі процеси з етапами перегляду та затвердження
- **Автоматизація обслуговування клієнтів**: Багатокрокове вирішення запитів клієнтів
- **Робочі процеси обробки даних**: ETL-процеси з трансформаціями, підсиленими ШІ

## ⚙️ Передумови та налаштування

### 📦 **Необхідні пакети NuGet**

Для демонстрації робочих процесів використовуються кілька ключових .NET-пакетів:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Налаштування Azure OpenAI**

**Налаштування середовища (.env файл):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Доступ до Azure OpenAI:**
1. Створіть ресурс Azure OpenAI у порталі Azure
2. Розгорніть модель (наприклад, `gpt-4.1-mini`) і запам’ятайте ім’я розгортання
3. Увійдіть за допомогою `az login` і налаштуйте змінні середовища, як показано вище

### 🏗️ **Огляд архітектури робочих процесів**

```mermaid
graph TD
    A[Конструктор Робочих Процесів] --> B[Реєстр Агентів]
    B --> C[Двигун Виконання Робочих Процесів]
    C --> D[Agent 1: Генератор Контенту]
    C --> E[Agent 2: Рецензент Контенту] 
    D --> F[Результати Робочого Процесу]
    E --> F
    G[Azure OpenAI (API Відповідей)] --> D
    G --> E
```

**Ключові компоненти:**
- **WorkflowBuilder**: Основний механізм оркестрації для проєктування робочих процесів
- **AIAgent**: Окремі спеціалізовані агенти з конкретними можливостями
- **Azure OpenAI Client**: Інтеграція з Azure OpenAI Responses API
- **Execution Context**: Керує станом і потоком даних між етапами робочого процесу

## 💻 Запуск коду

Повна реалізація доступна у файлі `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Цей файл демонструє:

1. **Налаштування середовища** - Завантаження конфігурації Azure OpenAI з файлу `.env`
2. **Налаштування клієнта Azure OpenAI** - Конфігурація клієнта для використання Azure OpenAI Responses API
3. **Створення агентів** - Визначення спеціалізованих агентів (Рецепція та Консьєрж)
4. **Створювач робочих процесів** - Створення робочого процесу з кількома агентами з послідовною обробкою
5. **Виконання робочого процесу** - Запуск робочого процесу з потоковим виводом результатів

### 🚀 Запуск прикладу

```bash
# Зробіть скрипт виконуваним (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Запустіть робочий процес
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Або на Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Очікуваний результат

Робочий процес:
1. Прийме ваш запит на місце подорожі ("Я хочу поїхати до Парижа")
2. Агент рецепції надасть початкову рекомендацію
3. Агент-консьєрж перегляне і вдосконалить рекомендацію
4. Кінцевий вивід відображатиме повний потік розмови
