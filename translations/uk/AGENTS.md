# AGENTS.md

## Огляд проєкту

Цей репозиторій містить "AI Agents for Beginners" - всебічний навчальний курс, який навчає всьому, що потрібно для створення AI Аґентів. Курс складається з 18 уроків (пронумерованих 00-18), що охоплюють основи, патерни проектування, фреймворки, продуктивне розгортання, локальних/пристосованих агентів та безпеку AI агентів.

**Ключові технології:**
- Python 3.12+
- Jupyter Notebooks для інтерактивного навчання
- AI фреймворки: Microsoft Agent Framework (MAF)
- Azure AI Services: Microsoft Foundry, Microsoft Foundry Agent Service V2

**Архітектура:**
- Структура за уроками (папки 00-15+)
- Кожен урок містить: документацію README, приклади коду (Jupyter notebooks) та зображення
- Підтримка кількох мов через автоматизовану систему перекладу
- По одному Python ноутбуку на урок із Microsoft Agent Framework

## Команди Налаштування

### Передумови
- Python 3.12 або вище
- Підписка Azure (для Microsoft Foundry)
- Встановлений та автентифікований Azure CLI (`az login`)

### Початкове налаштування

1. **Клонувати або форкнути репозиторій:**
   ```bash
   gh repo fork microsoft/ai-agents-for-beginners --clone
   # АБО
   git clone https://github.com/microsoft/ai-agents-for-beginners.git
   cd ai-agents-for-beginners
   ```

2. **Створити та активувати віртуальне середовище Python:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # У Windows: venv\Scripts\activate
   ```

3. **Встановити залежності:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Налаштувати змінні середовища:**
   ```bash
   cp .env.example .env
   # Відредагуйте .env, додавши свої API-ключі та кінцеві точки
   ```

### Необхідні змінні середовища

Для **Microsoft Foundry** (Обов’язково):
- `AZURE_AI_PROJECT_ENDPOINT` - кінцева точка проєкту Microsoft Foundry
- `AZURE_AI_MODEL_DEPLOYMENT_NAME` - ім'я розгортання моделі (наприклад, gpt-4.1-mini)

Для **Azure AI Search** (Урок 05 - RAG):
- `AZURE_SEARCH_SERVICE_ENDPOINT` - кінцева точка Azure AI Search
- `AZURE_SEARCH_API_KEY` - ключ API Azure AI Search

Аутентифікація: Запустіть `az login` перед запуском ноутбуків (використовується `AzureCliCredential`).

## Робочий процес розробки

### Запуск Jupyter Notebooks

Кожен урок має кілька Jupyter ноутбуків для різних фреймворків:

1. **Запустити Jupyter:**
   ```bash
   jupyter notebook
   ```

2. **Перейти до директорії уроку** (наприклад, `01-intro-to-ai-agents/code_samples/`)

3. **Відкрити та виконувати ноутбуки:**
   - `*-python-agent-framework.ipynb` - Використання Microsoft Agent Framework (Python)
   - `*-dotnet-agent-framework.ipynb` - Використання Microsoft Agent Framework (.NET)

### Робота з Microsoft Agent Framework

**Microsoft Agent Framework + Microsoft Foundry:**
- Потрібна підписка Azure
- Використовує `FoundryChatClient` для Agent Service V2 (агенти видимі у порталі Foundry)
- Готовий до продуктивного використання із вбудованою спостережливістю
- Шаблон файлу: `*-python-agent-framework.ipynb`

Дивіться [tests/README.md](./tests/README.md) для відповідності каталогу, уроку та агента. Урок 17 виконується локально з Foundry Local і не має хостованої кінцевої точки, тож він перевіряється безпосередньо шляхом запуску ноутбуку.

1. **Несумісність версії Python:**

   - Потрібна активна підписка Azure AI services
   - Деякі функції залежать від регіону
