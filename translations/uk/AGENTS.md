# AGENTS.md

## Огляд проекту

Цей репозиторій містить "AI Agents for Beginners" — комплексний навчальний курс, який навчає всьому необхідному для створення AI Агентів. Курс складається з 18 уроків (пронумерованих 00-18), що охоплюють основи, шаблони проектування, фреймворки, розгортання в продакшн, локальні/пристрійні агенти та безпеку AI агентів.

**Ключові технології:**
- Python 3.12+
- Jupyter ноутбуки для інтерактивного навчання
- AI фреймворки: Microsoft Agent Framework (MAF)
- Azure AI сервіси: Microsoft Foundry, Microsoft Foundry Agent Service V2

**Архітектура:**
- Структура на основі уроків (каталоги 00-15+)
- Кожен урок містить: README документацію, приклади коду (Jupyter ноутбуки) та зображення
- Підтримка багатьох мов через автоматизовану систему перекладу
- Один Python ноутбук на урок із використанням Microsoft Agent Framework

## Команди для налаштування

### Вимоги
- Python 3.12 або вище
- Підписка Azure (для Microsoft Foundry)
- Встановлений та автентифікований Azure CLI (`az login`)

### Початкове налаштування

1. **Клонувати або форкнути репозиторій:**
   ```bash
   gh repo fork microsoft/ai-agents-for-beginners --clone
   # АБО
   git clone https://github.com/microsoft/ai-agents-for-beginners.git
   cd ai-agents-for-beginners
   ```

2. **Створити та активувати віртуальне середовище Python:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # У Windows: venv\Scripts\activate
   ```

3. **Встановити залежності:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Налаштувати змінні середовища:**
   ```bash
   cp .env.example .env
   # Відредагуйте файл .env, додавши ваші API ключі та кінцеві точки
   ```

### Обов’язкові змінні середовища

Для **Microsoft Foundry** (обов’язково):
- `AZURE_AI_PROJECT_ENDPOINT` - кінцева точка проекту Microsoft Foundry
- `AZURE_AI_MODEL_DEPLOYMENT_NAME` - назва розгортання моделі (наприклад, gpt-5-mini)

Для **Azure AI Search** (Урок 05 - RAG):
- `AZURE_SEARCH_SERVICE_ENDPOINT` - кінцева точка Azure AI Search
- `AZURE_SEARCH_API_KEY` - ключ API Azure AI Search

Аутентифікація: Виконайте `az login` перед запуском ноутбуків (використовується `AzureCliCredential`).

## Робочий процес розробки

### Запуск Jupyter ноутбуків

Кожен урок містить кілька Jupyter ноутбуків для різних фреймворків:

1. **Запустіть Jupyter:**
   ```bash
   jupyter notebook
   ```

### Робота з Microsoft Agent Framework

**Microsoft Agent Framework + Microsoft Foundry:**
- Потрібна підписка Azure
- Використовує `FoundryChatClient` для Agent Service V2 (агенти видимі в порталі Foundry)
- Готовий до продакшн із вбудованою спостережливістю
- Формат файлів: `*-python-agent-framework.ipynb`

## Додаткові нотатки

### Типові проблеми

1. **Невідповідність версії Python:**
   - Переконайтеся, що використовується Python 3.12+
   - Деякі пакети можуть не працювати з старими версіями
   - Використовуйте `python3 -m venv` для явного вказування версії Python

2. **Змінні середовища:**
   - Завжди створюйте `.env` з `.env.example`
   - Не комітьте `.env` файл (він у `.gitignore`)
   - Авторизуйтесь через `az login` для безключової аутентифікації Entra ID

3. **Конфлікти пакетів:**
   - Використовуйте нове віртуальне середовище
   - Встановлюйте залежності з `requirements.txt`, а не окремо
   - Деяким ноутбукам потрібні додаткові пакети, згадані в markdown клітинках

4. **Сервіси Azure:**
   - Сервіси Azure AI вимагають активної підписки
   - Деякі функції специфічні для регіонів
   - Переконайтеся, що ваше розгортання моделі Azure OpenAI підтримує Responses API

## Особливості проекту

### Підтримка багатьох мов

Цей репозиторій використовує автоматизовану систему перекладу:
- Підтримка понад 50 мов
- Переклади в каталогах `/translations/<lang-code>/`
- Робочий процес GitHub Actions оновлює переклади
- Початкові файли англійською мовою на корені репозиторію

### Структура уроку

Кожен урок має постійну структуру:
1. Відеопрев’ю з посиланням
2. Текстовий матеріал уроку (README.md)
3. Приклади коду в різних фреймворках
4. Цілі навчання та передумови
5. Посилання на додаткові ресурси

### Іменування прикладів коду

Формат: `<lesson-number>-python-agent-framework.ipynb`
- `01-python-agent-framework.ipynb` - Урок 1, MAF Python
- `14-sequential.ipynb` - Урок 14, розширені шаблони MAF
- `16-python-agent-framework.ipynb` - Урок 16, продакшн агент обслуговування клієнтів
- `17-local-agent-foundry-local.ipynb` - Урок 17, локальний агент з Foundry Local + Qwen

### Спеціальні каталоги

- `translated_images/` - локалізовані зображення для перекладів
- `images/` - оригінальні зображення для англомовного контенту
- `.devcontainer/` - конфігурація середовища розробки VS Code
- `.github/` - робочі процеси та шаблони GitHub Actions

### Залежності

Основні пакети з `requirements.txt`:
- `agent-framework` - Microsoft Agent Framework
- `a2a-sdk` - підтримка протоколу Agent-to-Agent
- `azure-ai-inference`, `azure-ai-projects` - сервіси Azure AI
- `azure-identity` - автентифікація Azure (AzureCliCredential)
- `azure-search-documents` - інтеграція Azure AI Search
- `mcp[cli]` - підтримка Model Context Protocol
