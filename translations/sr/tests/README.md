# Агентски Smoke тестови

Ова фасцикла садржи **каталоге smoke-тестова** за агенте које правите на курсу.
Smoke тест је јефтена, брза провера да ли је **дејставни Microsoft Foundry хостовани
агент** доступан, одговара и следи своје најосновније очекивања у упиту.
То је први филтер — а не замена за пуну оцену
процедуру коју учите у [Лекцији 10](../10-ai-agents-production/README.md) и
[Лекцији 16](../16-deploying-scalable-agents/README.md).

Каталози се користе у оквиру [AI Smoke Test](https://github.com/marketplace/actions/ai-smoke-test)
GitHub акције преко [`.github/workflows/smoke-test.yml`](../../../.github/workflows/smoke-test.yml)
флоуа.

## Како покренути

1. **Деплојујте агента из лекције** на Microsoft Foundry као хостованог агента (погледајте
   Лекцију 16 за флоу процеса деплоја). Запамтите **име агента** и ваш
   **Foundry крајњи тачку пројекта**.
2. Додајте ове тајне податке репозиторијуму (Settings → Secrets and variables → Actions):
   `AZURE_CLIENT_ID`, `AZURE_TENANT_ID`, `AZURE_SUBSCRIPTION_ID`. Федеративни
   идентитет треба да има улогу **Azure AI User** на **Foundry опсегу пројекта**.
3. Из таба **Actions**, покрените **Smoke-test hosted agents** и одаберите
   `tests_file` за лекцију, затим унесите одговарајуће `agent_name` и
   `project_endpoint`.

## Које лекције имају smoke тестове?

Smoke тестови се примењују на лекције где **деплојујете агента** чији се текстуални одговори могу
проверити у односу на познат садржај. Лекције које су концептуалне, раде само локално,
или производе недетерминистички креативни излаз су намерно изузете:

- **Лекција 17 (Креирање локалних AI агената)** ради у потпуности на вашем рачунару користећи
  Foundry Local и не излаже Foundry Responses крајњу тачку, тако да ова
  акција не важи. Потврдите је тако што ћете покренути ноутбук локално.
- Лекције о дизајн образцима и теорији (02, 03, 06, 07, 09, 12) не испоручују ниједног
  деплојованог агента за smoke-тест.

## Шема каталога (брзи преглед)

Сваки каталог је JSON документ са горњим `tests` низом. Свака ставка шаље
један упит и проверава одговор:

| Поље | Значење |
|-------|---------|
| `id` | Јединствени идентификатор корака штампан у записнику. |
| `description` | Човеку читљива сврха. |
| `prompt` | Порука послата агенту. |
| `assertions.status` | Очекивани HTTP статус (подразумевано 200). |
| `assertions.contains_any` | Прође ако одговор садржи било који од ових подниза. |
| `assertions.contains_all` | Прође ако одговор садржи сваки подниз. |
| `assertions.contains_none` | Прође ако одговор не садржи ниједан од ових подниза. |
| `save_response_id_as` | Сачувај ID одговора за каснији корак са више окретања. |
| `use_previous_response_id` | Пошаљи овај круг уз похрањени ID одговора. |

Тврђења су проверe поднизова без обзира на величину слова. Погледајте
[документацију акције](https://github.com/marketplace/actions/ai-smoke-test) за
потпуну шему, укључујући ресурсе конверзације којима управља Foundry.
