# 🔄 Основни токови агената са Azure OpenAI (Responses API) (.NET)

## 📋 Туторијал за оркестрацију тока рада

Овај нотебоок показује како изградити софистициране **токове агената** користећи Microsoft Agent Framework за .NET и Azure OpenAI (Responses API). Научићете како да креирате пословне процесе са више корака где AI агенти сарађују да би извршили сложене задатке кроз структуиране оркестрацијске шаблоне.

## 🎯 Циљеви учења

### 🏗️ **Основе архитектуре тока рада**
- **Workflow Builder**: Дизајнирање и оркестрација сложених AI процеса са више корака
- **Координација агената**: Координација више специјализованих агената у токовима рада
- **Azure OpenAI (Responses API)**: Користите Azure OpenAI Responses API у токовима рада
- **Визуелни дизајн тока рада**: Креирање и визуализација структура тока ради бољег разумевања

### 🔄 **Обрасци оркестрације процеса**
- **Секвенцијална обрада**: Ланцирање више задатака агената у логичком редоследу
- **Управљање стањем**: Одржавање контекста и тока података кроз фазе тока рада
- **Обрада грешака**: Имплементација робусног опоравка од грешака и отпорности тока рада
- **Оптимизација перформанси**: Дизајнирање ефикасних токова рада за операције на нивоу предузећа

### 🏢 **Апликације тока рада за предузећа**
- **Аутоматизација пословних процеса**: Аутоматизација сложених организационих токова рада
- **Профил производње садржаја**: Уреднички токови рада са фазама прегледа и одобрења
- **Аутоматизација корисничке службе**: Решавање корисничких упита са више корака
- **Токови обраде података**: ETL токови рада са AI покретаном трансформацијом

## ⚙️ Захтеви и подешавање

### 📦 **Потребни NuGet пакети**

Ова демонстрација тока рада користи неколико кључних .NET пакета:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Конфигурација Azure OpenAI**

**Подешавање окружења (.env фајл):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Приступ Azure OpenAI:**
1. Креирајте Azure OpenAI ресурс у Azure порталу
2. Деплојујте модел (на пример, `gpt-5-mini`) и запамтите име деплоја
3. Пријавите се помоћу `az login` и конфигуришите варијабле окружења као што је горе приказано

### 🏗️ **Преглед архитектуре тока рада**

```mermaid
graph TD
    A[Креатор радних токова] --> B[Регистар агената]
    B --> C[Енгин за извршавање радних токова]
    C --> D[Agent 1: Генератор садржаја
    C --> E[Agent 2: Рецензент садржаја 
    D --> F[Резултати радних токова]
    E --> F
    G[Azure OpenAI (Responses API)] --> D
    G --> E
```

**Кључне компоненте:**
- **WorkflowBuilder**: Главни мотор за оркестрацију и дизајн токова рада
- **AIAgent**: Појединачни специјализовани агенти са одређеним способностима
- **Azure OpenAI клијент**: Интеграција Azure OpenAI Responses API-ја
- **Execution Context**: Управља статусом и током података између фаза тока рада

## 💻 Покретање кода

Потпуна имплементација је доступна у `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Овај фајл демонстрира:

1. **Конфигурацију окружења** - Учитавање конфигурације Azure OpenAI из `.env` фајла
2. **Подешавање Azure OpenAI клијента** - Конфигурисање клијента за коришћење Azure OpenAI Responses API
3. **Креирање агената** - Дефинисање специјализованих агената (Пријемна и Консијерж)
4. **Workflow Builder** - Креирање тока рада са више агената и секвенцијалном обрадом
5. **Извршење тока рада** - Покретање тока рада са стриминг резултатима

### 🚀 Покретање примера

```bash
# Направите скрипту извршном (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Покрените ток рада
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Или на Windows-у:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Очекивани излаз

Ток рада ће:
1. Прихватити ваш захтев за дестинацију путовања ("Желео бих да идем у Париз")
2. Агенат Пријемне службе ће дати почетну препоруку
3. Агенат Консијержа ће прегледати и унапредити препоруку
4. Коначан излаз приказује комплетан ток конверзације
