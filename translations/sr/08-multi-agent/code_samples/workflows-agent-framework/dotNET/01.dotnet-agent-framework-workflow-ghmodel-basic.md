# 🔄 Основни токови агента са Azure OpenAI (Responses API) (.NET)

## 📋 Туторијал о оркестрацији токова рада

Овај записник демонстрира како се праве софистицирани **токови агента** користећи Microsoft Agent Framework за .NET и Azure OpenAI (Responses API). Учите како да креирате вишестепене пословне процесе где AI агенти сарађују у извршавању сложених задатака кроз структуриране образце оркестрације.

## 🎯 Циљеви учења

### 🏗️ **Основе архитектуре токова рада**
- **Workflow Builder**: Пројектовање и оркестрација сложених вишестепених AI процеса
- **Agent Coordination**: Координисање више специјализованих агената у токовима рада
- **Azure OpenAI (Responses API)**: Коришћење Azure OpenAI Responses API у токовима рада
- **Visual Workflow Design**: Креирање и визуализација структура токова рада за боље разумевање

### 🔄 **Обрасци оркестрације процеса**
- **Низак поступак обраде**: Ланцирање више агенцијских задатака у логичком редоследу
- **Управљање стањем**: Одржавање контекста и протока података кроз фазе токова рада
- **Обрада грешака**: Имплементација робусног опоравка грешака и отпорности токова рада
- **Оптимизација учинка**: Пројектовање ефикасних токова рада за рад у предузећима

### 🏢 **Апликације пословних токова рада**
- **Аутоматизација пословних процеса**: Аутоматизација сложених организационих токова рада
- **Производни ланац садржаја**: Уреднички токови рада са фазама прегледа и одобрења
- **Аутоматизација корисничке службе**: Вишестепено решавање корисничких упита
- **Токови обраде података**: ETL токови рада са AI-подржаном трансформацијом

## ⚙️ Захтеви и подешавање

### 📦 **Потребни NuGet пакети**

У овој демонстрацији тока рада коришћено је неколико кључних .NET пакета:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Конфигурација Azure OpenAI**

**Подешавање окружења (.env фајл):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Приступ Azure OpenAI:**
1. Креирајте Azure OpenAI ресурс у Azure порталу
2. Расположите модел (на пример, `gpt-4.1-mini`) и запамтите име расподеле
3. Пријавите се са `az login` и конфигуришите променљиве окружења као што је приказано горе

### 🏗️ **Преглед архитектуре токова рада**

```mermaid
graph TD
    A[Изградња рада] --> B[Регистар агената]
    B --> C[Енгин за извршавање радног тока]
    C --> D[Agent 1: Генератор садржаја]
    C --> E[Agent 2: Рецензент садржаја] 
    D --> F[Резултати радног тока]
    E --> F
    G[Azure OpenAI (Responses API)] --> D
    G --> E
```

**Кључне компоненте:**
- **WorkflowBuilder**: Главни механизам за оркестрацију и пројектовање токова рада
- **AIAgent**: Појединачни специјализовани агенти са специфичним могућностима
- **Azure OpenAI Client**: Интеграција Azure OpenAI Responses API
- **Execution Context**: Управљање стањем и протоком података између фаза токова рада

- **Контрола приступа**: Прецизна права за извршење и праћење токова рада
- **Приватност података**: Безбедна обрада осетљивих информација у току целог процеса

### 📊 **Посматрање и управљање**
- **Визуелни дизајн токова рада**: Јасна репрезентација протока и зависности процеса
- **Мониторинг извршења**: Праћење напретка и учинка тока рада у реалном времену
- **Извештавање о грешкама**: Детаљна анализа грешака и могућности за отклањање
- **Аналитика учинка**: Метрике за оптимизацију и планирање капацитета

## 💻 Покретање кода

Комплетна имплементација је доступна у `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Овај фајл демонстрира:

1. **Конфигурација окружења** - Учитавање Azure OpenAI конфигурације из `.env` фајла
2. **Подешавање Azure OpenAI клијента** - Конфигурисање клијента за коришћење Azure OpenAI Responses API
3. **Креирање агента** - Дефинисање специјализованих агената (пријем и консијерж)
4. **Workflow Builder** - Креирање вишестепеног тока са више агената са секвенцијалном обрадом
5. **Извршење тока рада** - Покретање тока рада са резултатима у стриму

### 🚀 Покретање примера

```bash
# Учини скрипту извршном (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Покрени радни ток
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Очекивани резултат

Ток рада ће:
1. Примити ваш захтев за дестинацију путовања ("Желео бих да идем у Париз")
2. Агент пријема ће дати почетну препоруку
3. Консијерж агент ће прегледати и дорадити препоруку
4. Коначни излаз приказује комплетан ток разговора

Можете прилагодити ток рада тако што ћете:
- Мењати упутства агената за промену њиховог понашања
- Додавати више агената и створити сложене вишестепене токове рада
- Мењати корисничку поруку за тестирање различитих сценарија
- Прилагођавати везе у току рада за креирање различитих образаца извршења
