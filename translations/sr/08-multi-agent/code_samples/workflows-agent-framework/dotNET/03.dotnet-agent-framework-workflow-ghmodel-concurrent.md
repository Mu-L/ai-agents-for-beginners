# ⚡ Конкурентни агенцијски токови рада са Azure OpenAI (Responses API) (.NET)

## 📋 Туторијал за високо перформансе паралелне обраде

Овај нотебоок демонстрира **конкурентне обрасце токова рада** користећи Microsoft Agent Framework за .NET и Azure OpenAI (Responses API). Научићете како да направите високо перформантне, паралелне токове рада који максимизују пропусни опсег извршавањем више AI агената истовремено уз одржавање координације и конзистентности података.

## 💻 Покретање кода

Комплетна имплементација доступна је у `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Ова датотека приказује **Fan-Out/Fan-In конкурентни ток рада** за планирање путовања:

### 🏗️ **Архитектура тока рада**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Кључне компоненте:**

1. **ConcurrentStartExecutor**: Истовремено емитује кориснички захтев свим агентима
2. **Researcher Agent**: Конкурентно анализира дестинације и атракције
3. **Planner Agent**: Конкурентно креира детаљне планове путовања
4. **ConcurrentAggregationExecutor**: Прикупља и спаја резултате оба агента

### 🎯 **Fan-Out/Fan-In образац**

Овај ток рада илуструје класичан **Fan-Out/Fan-In** образац:
- **Fan-Out**: Једна улазна порука се истовремено емитује више агената
- **Конкурентна обрада**: Више агената ради паралелно на истом задатку
- **Fan-In**: Резултати свих агената се прикупљају и агрегирају у један излаз

### 📝 Очекивани резултат

Ток рада ће:
1. **Емитује захтев**: Пошаље „Планирај путовање за Сијетл у децембру“ оба агента
2. **Конкурентна обрада**: Оба агента раде истовремено:
   - Истраживач идентификује атракције и детаље
   - Планер креира итинерер и логистику
3. **Агрегација**: Комбинује оба одговора у свеобухватан резултат
4. **Приказ резултата**: Приказује спојени план путовања са свим информацијама

### 🎯 Примењене употребе у стварном свету

Овај конкурентни образац је идеалан за:
- **Креирање садржаја**: Више аутора истовремено пише различите секције
- **Прегледање кода**: Више рецензената анализира код из различитих перспектива
- **Истраживање тржишта**: Паралелна анализа различитих сегмената тржишта
- **Обрада докумената**: Конкурентна екстракција, анализа и валидација
- **Мултиперспективна анализа**: Прикупљање различитих мишљења о истом улазу

### 🔍 Разумевање прилагођених извршитеља

**ConcurrentStartExecutor:**
- Имплементира `IMessageHandler<string>` за прихват улазног текста
- Емитује поруке свим повезаним агентима
- Слање `TurnToken` за покретање конкурентне обраде

**ConcurrentAggregationExecutor:**
- Имплементира `IMessageHandler<ChatMessage>` за примање одговора агената
- Прикупља поруке на безбедан за конекцију начин
- Агрегира када стигну сви очекивани одговори
- Враћа финални излаз коришћењем `context.YieldOutputAsync()`

### ⚡ Предности перформанси

**Конкурентно у односу на секвенцијално:**
- Секвенцијално: Agent1 (30с) → Agent2 (30с) = **укупно 60 секунди**
- Конкурентно: Agent1 (30с) || Agent2 (30с) = **укупно 30 секунди**

**Побољшање пропусности**: До Н× брже за Н конкурената (у зависности од оптерећења и ресурса)

### 🛡️ Руководење грешкама

Ток рада лепо управља појединачним отказима агената:
- Ако један агент не успе, остали настављају обраду
- Агрегатор може применити логичко време чекања (timeout)
- Пуни резултати могу бити враћени уколико је потребно
