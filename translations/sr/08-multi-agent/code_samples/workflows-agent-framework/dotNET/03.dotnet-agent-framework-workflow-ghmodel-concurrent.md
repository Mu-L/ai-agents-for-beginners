# ⚡ Паралелни токови рада агената уз Azure OpenAI (Responses API) (.NET)

## 📋 Туторијал за високопродуктивну паралелну обраду

Овај записник демонстрира **образце паралелних токова рада** користећи Microsoft Agent Framework за .NET и Azure OpenAI (Responses API). Научићете како да креирате високопродуктивне токове рада који максимизирају пропусност извршавањем више AI агената истовремено уз одржавање координације и доследности података.

## 💻 Покретање кода

Комплетна имплементација доступна је у `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Овај фајл демонстрира **Fan-Out/Fan-In паралелни ток рада** за планирање путовања:

### 🏗️ **Архитектура тока рада**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Кључни елементи:**

1. **ConcurrentStartExecutor**: Истовремено шаље кориснички захтев свим агентима
2. **Researcher Agent**: Паралелно анализира дестинације и знаменитости
3. **Planner Agent**: Паралелно креира детаљне планове путовања
4. **ConcurrentAggregationExecutor**: Сакупља и спаја резултате оба агента

### 🎯 **Образац Fan-Out/Fan-In**

Овај ток рада демонстрира класичан **Fan-Out/Fan-In** образац:
- **Fan-Out**: Једна улазна порука се истовремено шаље више агената
- **Паралелна обрада**: Више агената ради паралелно на истом задатку
- **Fan-In**: Резултати свих агената се сакупљају и агрегирају у један излаз

Ток рада ће:
1. **Проследити захтев**: Послати "Планирај путовање у Сијетл у децембру" оба агента
2. **Паралелна обрада**: Оба агента раде истовремено:

### 🎯 Примене у реалном свету

Овај паралелни образац је идеалан за:
- **Креирање садржаја**: Више писаца који истовремено раде на различитим деловима
- **Преглед кода**: Више рецензената који анализирају код из различитих углова
- **Истраживање тржишта**: Паралелна анализа различитих сегмената тржишта
- **Обрада докумената**: Паралелно извлачење, анализа и валидација
- **Анализа из више перспектива**: Добијање различитих гледишта на исти улаз

### 🔍 Разумевање прилагођених Executor-а

**ConcurrentStartExecutor:**
- Имплементира `IMessageHandler<string>` за прихват улазног низа
- Шаље поруке свим повезаним агентима
- Шаље `TurnToken` да покрене паралелну обраду

**ConcurrentAggregationExecutor:**
- Имплементира `IMessageHandler<ChatMessage>` да прими одговоре агената
- Сакупља поруке на безбедан начин за више нити
- Агрегира када стигну сви очекивани одговори
- Враћа коначни излаз користећи `context.YieldOutputAsync()`

### ⚡ Предности перформанси

**Паралелно у односу на секвенцијално:**
- Секвенцијално: Agent1 (30с) → Agent2 (30с) = **укупно 60 секунди**
- Паралелно: Agent1 (30с) || Agent2 (30с) = **укупно 30 секунди**

**Побољшање пропусности**: До N× брже за N паралелних агената (у зависности од оптерећења и ресурса)

### 🛡️ Руководење грешкама

Ток рада сагледава одвајање грешака појединачних агената:
- Ако један агент падне, други настављају са обрадом
- Агрегатор може имплементирати логику временског ограничења (timeout)
- Деловни резултати се могу вратити ако је потребно
