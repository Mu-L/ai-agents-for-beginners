# ⏩ Редоследни радни токови агената са Azure OpenAI (Responses API) (.NET)

## 📋 Напредни туторијал за редоследну обраду

Овај запис демонстрира **шаблоне редоследних радних токова** користећи Microsoft Agent Framework за .NET и Azure OpenAI (Responses API). Научићете како да направите софистициране, корак по корак цевоводе за обраду где агенти извршавају одређени редослед, при чему сваки корак гради на резултатима претходног.

## 🎯 Циљеви учења

### 🔄 **Архитектура редоследне обраде**
- **Линеарни дизајн радног тока**: Креирање корак по корак цевовода са јасним зависностима
- **Управљање стањем**: Одржавање контекста и тока података кроз редоследне фазе радног тока
- **Azure OpenAI (Responses API)**: Коришћење Azure OpenAI модела у вишестепеним .NET радним токовима
- **Предузетнички шаблони цевовода**: Прављење продукцијски спремних редоследних система за обраду

### 🏗️ **Напредни редоследни шаблони**
- **Обрада са контролним тачкама**: Имплементација проверних поена између фаза радног тока
- **Чување контекста**: Одржавање стања и акумулираног знања кроз све фазе
- **Пренос грешака**: Љубазно руковање неуспесима у ланцима редоследне обраде
- **Оптимизација перформанси**: Ефикасно редоследно извршавање са минималним трошковима

### 🏢 **Редоследне апликације за предузећа**
- **Цевовод обраде докумената**: Вишестепена анализа, трансформација и валидација докумената
- **Радни токови за осигурање квалитета**: Редоследни процеси прегледа, валидације и одобрења
- **Цевовод за продукцију садржаја**: Истраживање → Писање → Уређивање → Преглед → Објављивање
- **Аутоматизација пословних процеса**: Вишестепени пословни радни токови са јасним зависностима између фаза

## ⚙️ Захтеви и подешавање

### 📦 **Потребни NuGet пакети**

Есенцијални пакети за .NET редоследне радне токове:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Конфигурација Azure OpenAI**

**Подешавање окружења (фајл .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Управљање конфигурацијом:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **Архитектура редоследног радног тока**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Агент за анализу]
    B --> C[Контролна тачка 1]
    C --> D[Stage 2: Агент за обраду]
    D --> E[Контролна тачка 2]
    E --> F[Stage 3: Агент за валидацију]
    F --> G[Коначни резултат]
    
    H[Контекст стања] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Responses API)] --> B
    I --> D
    I --> F
```

**Кључне компоненте:**
- **Редоследни агенти**: Специјализовани агенти за сваку фазу обраде
- **Контекст стања**: Одржава акумулиране податке и одлуке кроз фазе
- **Контролне тачке**: Тачке валидације између фаза за осигурање квалитета и конзистентности
- **Azure OpenAI клијент**: Конзистентан приступ AI моделу у свим фазама радног тока

## 💻 Покретање кода

Комплетна имплементација доступна је у `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Ова датотека демонстрира **троступењски радни ток анализе намештаја**:

1. **Фаза 1 - Агент продаје**: Анализира слике намештаја и даје предлоге за куповину
2. **Фаза 2 - Агент цене**: Даје детаљне рачуне цена и опције у оквиру буџета
3. **Фаза 3 - Агент понуде**: Генерише професионални документ са понудом у Markdown формату

### 🏗️ **Архитектура радног тока**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Свaki агент:
- Прима излаз претходне фазе као контекст
- Надограђује претходну анализу са специјализованом експертизом
- Одржава континуитет радног тока кроз управљање стањем

### 🚀 Покретање примера

**Захтеви:**
- Ставите слику намештаја у `../imgs/home.png` (или ажурирајте варијаблу `imgPath`)
- Конфигуришите ваш `.env` фајл са Azure OpenAI крајњом тачком и деплојментом, затим се пријавите са `az login`

```bash
# Направите скрипт извршним (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Покрените секвенцијални ток рада
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Или на Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Очекујте излаз

Радни ток ће:
1. **Агент продаје**: Препознати предмете намештаја са слике и дати препоруке
2. **Агент цене**: Додати детаљну анализу цена са нивоима буџета и препорукама за куповину
3. **Агент понуде**: Генерисати форматирани документ са понудом са свим синтетизованим информацијама

Коначни излаз ће бити свеобухватна, професионална понуда намештаја базирана на анализи слике.

### 🎯 Примене у стварном свету

Овај редоследни шаблон је идеалан за:
- **Е-трговина**: Анализа производа → Форматирање цена → Генерисање понуда
- **Нekреtтинe**: Анализа имовине → Процена → Креирање огласа
- **Осигурање**: Анализа потраживања → Оцена → Генерисање понуде
- **Креирање садржаја**: Истраживање → Писање → Уређивање → Објављивање

### 🔍 Разумевање тока стања

Свaki агент у низу добија:
- **Првобитни унос**: Почетна порука корисника (слика + текст)
- **Излазе претходних агената**: Све претходне агентске реакције у историји разговорa
- **Акумулирани контекст**: Потпуно одржавано стање кроз цео радни ток

Ово омогућава софистицирану вишестепену обраду где сваки агент надограђује свеобухватни контекст из свих претходних фаза.
