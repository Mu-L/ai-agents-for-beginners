# ⏩ Секвенцијални токови рада агената са Azure OpenAI (Responses API) (.NET)

## 📋 Напредни туторијал секвенцијалне обраде

Овај нотебоок демонстрира **шаблоне секвенцијалних токова рада** користећи Microsoft Agent Framework за .NET и Azure OpenAI (Responses API). Научићете како да изградите сложене, корак по корак процесне цевоводе где агенти извршавају у одређеном редоследу, при чему свака фаза гради на резултатима претходне фазе.

## 🎯 Циљеви учења

### 🔄 **Архитектура секвенцијалне обраде**
- **Линеарни дизајн тока рада**: Креирајте корак по корак цевоводе са јасним зависностима
- **Управљање стањем**: Одржавајте контекст и ток података кроз секвенцијалне фазе тока рада
- **Azure OpenAI (Responses API)**: Искористите Azure OpenAI моделе у мултифазним .NET токовима рада
- **Шаблони цевовода за предузећа**: Изградите продукционо спремне секвенцијалне системе обраде

### 🏗️ **Напредни секвенцијални шаблони**
- **Обрада са контролним тачкама (Stage-Gate)**: Имплементирајте проверне тачке између фаза тока рада
- **Чување контекста**: Одржавање стања и акумулираног знања кроз све фазе
- **Пропагирање грешака**: Грациозно руковање неуспесима у секвенцијалним ланцима обраде
- **Оптимизација перформанси**: Ефикасно секвенцијално извршавање са минималним оптерећењем

### 🏢 **Секвенцијалне апликације за предузећа**
- **Цевовод за обраду докумената**: Мултифазна анализа, трансформација и валидација докумената
- **Токови рада за контролу квалитета**: Секвенцијални преглед, валидација и одобрење процеса
- **Цевовод производње садржаја**: Истраживање → Писање → Уређивање → Преглед → Објављивање
- **Аутоматизација пословних процеса**: Мултифазни пословни токови са јасним зависностима између фаза

## ⚙️ Захтеви и подешавање

### 📦 **Потребне NuGet пакете**

Основне пакете за .NET секвенцијалне токове рада:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Конфигурација Azure OpenAI**

**Подешавање окружења (.env фајл):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Управљање конфигурацијом:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **Архитектура секвенцијалног тока рада**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Аналитички агент]
    B --> C[Контролна тачка 1]
    C --> D[Stage 2: Процесни агент]
    D --> E[Контролна тачка 2]
    E --> F[Stage 3: Валидациони агент]
    F --> G[Коначни резултат]
    
    H[Контекст стања] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Responses API)] --> B
    I --> D
    I --> F
```

**Кључне компоненте:**
- **Секвенцијални агенти**: Специјализовани агенти за сваку фазу обраде
- **Контекст стања**: Чува акумулиране податке и одлуке кроз фазе
- **Контролне тачке**: Тачке валидације између фаза за осигурање квалитета и конзистентности
- **Azure OpenAI клијент**: Конзистентан приступ AI моделима кроз све фазе тока рада

## 🏢 **Предности секвенцијалног рада у предузећима**

### 🎯 **Поузданост и квалитет**
- **Детерминистичка обрада**: Конзистентни, понављиви резултати кроз структуиране фазе
- **Врата квалитета**: Контролне тачке валидације обезбеђују квалитет у свакој фази
- **Изолација грешака**: Проблеми у једној фази не продиру у наредне фазе
- **Трагови ревизије**: Потпуна праћења одлука и трансформација у свакој фази

### 📈 **Скалабилност и перформансе**
- **Модуларни дизајн**: Сваку фазу може да се оптимизује независно
- **Управљање ресурсима**: Ефикасна алокација AI ресурса кроз фазе
- **Оптимизација стања**: Минималан пренос стања између фаза за оптималне перформансе
- **Паралелне групе фаза**: Више секвенцијалних токова могу радити паралелно

### 🔒 **Безбедност и усаглашеност**
- **Безбедност на нивоу фаза**: Различите безбедносне политике за различите фазе обраде
- **Валидација података**: Обезбеђује интегритет података и усаглашеност на свакој контролној тачки
- **Контрола приступа**: Грануларна права за различите фазе тока рада
- **Регулаторна усаглашеност**: Испуњавање регулаторних захтева кроз структуирану обраду

### 📊 **Праћење и аналитика**
- **Метрике на нивоу фаза**: Праћење перформанси за сваку фазу тока рада
- **Идентификација тешкоћа**: Идентификујте и оптимизујте спора места
- **Метрике квалитета**: Пратите квалитет и стопу успеха у свакој фази
- **Оптимизација процеса**: Континуирано унапређење засновано на аналитикама на нивоу фаза

## 💻 Покретање кода

Комплетна имплементација доступна је у фајлу `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Овај фајл демонстрира **трофазни ток рада за анализу намештаја**:

1. **Фаза 1 - Продајни агент**: Анализира слике намештаја и пружа препоруке за куповину
2. **Фаза 2 - Агент за цену**: Даје детаљну расподелу цена и опције буџета
3. **Фаза 3 - Агент за понуде**: Генерише професионални документ понуде у Markdown формату

### 🏗️ **Архитектура тока рада**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Сваки агент:
- Прима резултат претходне фазе као контекст
- Гради се на претходној анализи са стручним знањем
- Одржава континуитет тока рада путем управљања стањем

### 🚀 Покретање примера

**Предуслови:**
- Поставите слику намештаја на `../imgs/home.png` (или ажурирајте променљиву `imgPath`)
- Конфигуришите свој `.env` фајл са својим Azure OpenAI крајњом тачком и деплојментом, затим пријавите се преко `az login`

```bash
# Направите скрипт извршним (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Покрените секвенцијални ток рада
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Или на Windows-у:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Очекујени излаз

Ток рада ће:
1. **Продајни агент**: Идентификовати предмете намештаја са слике и дати препоруке
2. **Агент за цену**: Додати детаљну анализу цена са буџетским категоријама и препорукама за куповину
3. **Агент за понуде**: Генерисати форматирани документ понуде са сажетком свих информација

Коначан излаз биће свеобухватна, професионална понуда за намештај базирана на анализи слике.

### 🎯 Примене у стварном свету

Овај секвенцијални шаблон је идеалан за:
- **Е-трговина**: Анализа производа → Цена → Генерисање понуде
- **Некретнине**: Анализа имовине → Процена → Креирање огласа
- **Осигурање**: Анализа потраживања → Процена → Генерисање понуде

### 🔍 Разумевање тока стања

Сваки агент у низу прима:
- **Првобитни улаз**: Почетну корисничку поруку (слика + текст)
- **Излазе претходних агената**: Све претходне агентске одговоре у историји конверзације
- **Акумулирани контекст**: Комплетно стање одржано у току целог тока рада

Ово омогућава софистицирану мултифазну обраду где сваки агент гради на свеобухватном контексту из свих претходних фаза.
