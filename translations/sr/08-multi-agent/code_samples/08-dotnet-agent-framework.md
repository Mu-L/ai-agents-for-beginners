# 🤝 Пословни мулти-агентски системи за радне токове (.NET)

## 📋 Циљеви учења

Овај нотебоок показује како направити сложене мулти-агентске системе за пословну употребу користећи Microsoft Agent Framework у .NET-у уз Azure OpenAI (Responses API). Научићете како да оркестрирате више специјализованих агената који раде заједно кроз структуриран радни ток, користећи ентерпрајз функције .NET-а за решења спремна за производњу.

**Могућности мулти-агентског система које ћете изградити:**
- 👥 **Сарадња агената**: Типски безбедна координација агената са проверама у време компајлирања
- 🔄 **Оркестрација радних токова**: Декларативна дефиниција радног тока са async патернима у .NET-у
- 🎭 **Специјализација улога**: Јако типизоване личности агената и области експертизе
- 🏢 **Интеграција у ентерпрајз систем**: Шаблони спремни за производњу са мониторингом и обрадом грешака

## Шта овај пример приказује

Овај мулти-агентски систем радног тока прави услугу препорука путовања за хотел са два специјализована агента:

1. **FrontDesk агент**: Агент путовања који пружа препоруке активности и локација
2. **Concierge агент**: Прегледа препоруке како би обезбедио аутентична, не-туристичка искуства

Агенте раде заједно у радном току где:
- FrontDesk агент прима почетни захтев за путовање
- Concierge агент прегледа и усавршава препоруку
- Радни ток стримује одговоре у реалном времену

## Кључни појмови

### Координација агената
Пример показује типски безбедну координацију агената користећи Microsoft Agent Framework са валидацијом у време компајлирања.

### Оркестрација радних токова
Користи декларативну дефиницију радног тока са async патернима у .NET-у за повезивање више агената у пипелина.

### Стриминг одговора
Имплементира стримовање одговора агената у реалном времену користећи async enumerable и догађајно оријентисану архитектуру.

### Ентерпрајз интеграција
Приказује шаблоне спремне за производњу укључујући:
- Конфигурацију кроз променљиве окружења
- Безбедно управљање акредитивима
- Обраду грешака
- Асинхрону обраду догађаја
