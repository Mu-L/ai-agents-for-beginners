# 🔄 Các Quy trình Tác vụ Cơ bản với Agent sử dụng Azure OpenAI (Responses API) (.NET)

## 📋 Hướng dẫn Điều phối Quy trình Tác vụ

Notebook này trình bày cách xây dựng các **quy trình tác vụ agent** phức tạp sử dụng Microsoft Agent Framework cho .NET và Azure OpenAI (Responses API). Bạn sẽ học cách tạo các quy trình kinh doanh nhiều bước trong đó các agent AI hợp tác để hoàn thành các nhiệm vụ phức tạp thông qua các mẫu điều phối có cấu trúc.

## 🎯 Mục tiêu Học tập

### 🏗️ **Kiến trúc Quy trình Tác vụ Cơ bản**
- **Trình Xây dựng Quy trình**: Thiết kế và điều phối các quy trình AI nhiều bước phức tạp
- **Phối hợp Agent**: Phối hợp nhiều agent chuyên biệt trong các quy trình tác vụ
- **Azure OpenAI (Responses API)**: Tận dụng Azure OpenAI Responses API trong quy trình tác vụ
- **Thiết kế Quy trình Trực quan**: Tạo và trực quan hóa cấu trúc quy trình để hiểu rõ hơn

### 🔄 **Mẫu Điều phối Quy trình**
- **Xử lý Tuần tự**: Xâu chuỗi nhiều nhiệm vụ của agent theo thứ tự logic
- **Quản lý Trạng thái**: Duy trì ngữ cảnh và luồng dữ liệu qua các giai đoạn quy trình tác vụ
- **Xử lý Lỗi**: Triển khai phục hồi lỗi mạnh mẽ và tăng độ bền cho quy trình tác vụ
- **Tối ưu hiệu suất**: Thiết kế các quy trình tác vụ hiệu quả cho quy mô doanh nghiệp

### 🏢 **Ứng dụng Quy trình Tác vụ Doanh nghiệp**
- **Tự động hóa Quy trình Kinh doanh**: Tự động hóa các quy trình tổ chức phức tạp
- **Quy trình Sản xuất Nội dung**: Quy trình biên tập với các giai đoạn đánh giá và phê duyệt
- **Tự động hóa Dịch vụ Khách hàng**: Giải quyết thắc mắc khách hàng đa bước
- **Quy trình Xử lý Dữ liệu**: Quy trình ETL với chuyển đổi hỗ trợ AI

## ⚙️ Yêu cầu & Thiết lập

### 📦 **Các Gói NuGet Cần thiết**

Ví dụ quy trình này sử dụng một số gói .NET chính:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Cấu hình Azure OpenAI**

**Thiết lập Môi trường (tệp .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Truy cập Azure OpenAI:**
1. Tạo một tài nguyên Azure OpenAI trong cổng Azure
2. Triển khai một mô hình (ví dụ, `gpt-5-mini`) và ghi lại tên triển khai
3. Đăng nhập với `az login` và cấu hình biến môi trường như hướng dẫn trên

### 🏗️ **Tổng quan Kiến trúc Quy trình Tác vụ**

```mermaid
graph TD
    A[Trình Xây Dựng Quy Trình Làm Việc] --> B[Đăng Ký Tác Nhân]
    B --> C[Bộ Máy Thực Thi Quy Trình Làm Việc]
    C --> D[Agent 1: Bộ Tạo Nội Dung]
    C --> E[Agent 2: Người Đánh Giá Nội Dung] 
    D --> F[Kết Quả Quy Trình Làm Việc]
    E --> F
    G[Azure OpenAI (API Phản Hồi)] --> D
    G --> E
```

**Các Thành phần Chính:**
- **WorkflowBuilder**: Bộ máy điều phối chính để thiết kế quy trình tác vụ
- **AIAgent**: Các agent chuyên biệt cá nhân với năng lực cụ thể
- **Azure OpenAI Client**: Tích hợp Azure OpenAI Responses API
- **Execution Context**: Quản lý trạng thái và luồng dữ liệu giữa các giai đoạn quy trình tác vụ

## 🎨 **Mẫu Thiết kế Quy trình Doanh nghiệp**

### 📝 **Quy trình Sản xuất Nội dung**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **Quy trình Xử lý Tài liệu**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **Quy trình Thông minh Kinh doanh**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **Tự động hóa Dịch vụ Khách hàng**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **Lợi ích Doanh nghiệp**

### 🎯 **Độ tin cậy & Khả năng mở rộng**
- **Thực thi Xác định**: Kết quả quy trình nhất quán, có thể lặp lại
- **Phục hồi Lỗi**: Xử lý thất bại nhẹ nhàng ở bất kỳ giai đoạn nào của quy trình
- **Giám sát Hiệu suất**: Theo dõi các chỉ số thực thi và cơ hội tối ưu hóa
- **Quản lý Tài nguyên**: Phân bổ và sử dụng hiệu quả tài nguyên mô hình AI

### 🔒 **Bảo mật & Tuân thủ**
- **Xác thực An toàn**: Xác thực Microsoft Entra ID qua `az login` (AzureCliCredential)
- **Theo dõi Đăng nhập**: Ghi lại đầy đủ quá trình thực thi và các điểm quyết định trong quy trình
- **Kiểm soát Truy cập**: Phân quyền chi tiết cho thực thi và giám sát quy trình
- **Bảo mật Dữ liệu**: Xử lý an toàn thông tin nhạy cảm xuyên suốt quy trình tác vụ

### 📊 **Khả năng quan sát & Quản lý**
- **Thiết kế Quy trình Trực quan**: Biểu diễn rõ ràng luồng xử lý và sự phụ thuộc
- **Giám sát Thực thi**: Theo dõi tiến trình và hiệu suất quy trình thời gian thực
- **Báo cáo Lỗi**: Phân tích lỗi chi tiết và khả năng gỡ lỗi
- **Phân tích Hiệu suất**: Chỉ số để tối ưu hóa và lập kế hoạch năng lực
