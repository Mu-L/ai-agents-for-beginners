# ⏩ Các Quy Trình Làm Việc Liên Tiếp với Đại Lý Azure OpenAI (Responses API) (.NET)

## 📋 Hướng Dẫn Xử Lý Liên Tiếp Nâng Cao

Sổ tay này trình bày các **mẫu quy trình làm việc theo trình tự** sử dụng Microsoft Agent Framework cho .NET và Azure OpenAI (Responses API). Bạn sẽ học cách xây dựng các đường ống xử lý phức tạp theo từng bước, trong đó các đại lý thực hiện theo thứ tự cụ thể, với mỗi giai đoạn dựa trên kết quả của giai đoạn trước đó.

## 🎯 Mục Tiêu Học Tập

### 🔄 **Kiến Trúc Xử Lý Liên Tiếp**
- **Thiết Kế Quy Trình Tuyến Tính**: Tạo các đường ống xử lý từng bước với các phụ thuộc rõ ràng
- **Quản Lý Trạng Thái**: Duy trì ngữ cảnh và dòng dữ liệu qua các giai đoạn làm việc liên tiếp
- **Azure OpenAI (Responses API)**: Tận dụng các mô hình Azure OpenAI trong quy trình .NET đa giai đoạn
- **Mẫu Đường Ống Doanh Nghiệp**: Xây dựng hệ thống xử lý liên tiếp sẵn sàng cho sản xuất

### 🏗️ **Mẫu Liên Tiếp Nâng Cao**
- **Xử Lý Qua Cổng Giai Đoạn**: Triển khai các điểm kiểm tra xác nhận giữa các giai đoạn quy trình
- **Bảo Tồn Ngữ Cảnh**: Duy trì trạng thái và kiến thức tích lũy qua tất cả các giai đoạn
- **Phát Tán Lỗi**: Xử lý lỗi một cách nhẹ nhàng trong chuỗi xử lý liên tiếp
- **Tối Ưu Hiệu Suất**: Thực thi liên tiếp hiệu quả với chi phí tối thiểu

### 🏢 **Ứng Dụng Liên Tiếp Doanh Nghiệp**
- **Đường Ống Xử Lý Tài Liệu**: Phân tích, chuyển đổi và xác nhận tài liệu đa giai đoạn
- **Quy Trình Đảm Bảo Chất Lượng**: Quy trình rà soát, xác nhận và phê duyệt theo thứ tự
- **Đường Ống Sản Xuất Nội Dung**: Nghiên cứu → Viết → Chỉnh sửa → Rà soát → Xuất bản
- **Tự Động Hóa Quy Trình Kinh Doanh**: Quy trình kinh doanh đa bước với các phụ thuộc giai đoạn rõ ràng

## ⚙️ Yêu Cầu & Cài Đặt

### 📦 **Các Gói NuGet Cần Thiết**

Các gói thiết yếu cho quy trình làm việc liên tiếp trên .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Cấu Hình Azure OpenAI**

**Cài Đặt Môi Trường (.env file):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Quản Lý Cấu Hình:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **Kiến Trúc Quy Trình Làm Việc Liên Tiếp**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Tác nhân Phân tích]
    B --> C[Điểm kiểm tra 1]
    C --> D[Stage 2: Tác nhân Xử lý]
    D --> E[Điểm kiểm tra 2]
    E --> F[Stage 3: Tác nhân Xác thực]
    F --> G[Kết quả Cuối cùng]
    
    H[Ngữ cảnh Trạng thái] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API Phản hồi)] --> B
    I --> D
    I --> F
```

**Các Thành Phần Chính:**
- **Đại Lý Liên Tiếp**: Đại lý chuyên biệt cho từng giai đoạn xử lý
- **Ngữ Cảnh Trạng Thái**: Duy trì dữ liệu tích lũy và quyết định qua các giai đoạn
- **Điểm Kiểm Tra**: Các điểm xác nhận giữa các giai đoạn đảm bảo chất lượng và tính nhất quán
- **Khách Hàng Azure OpenAI**: Truy cập nhất quán mô hình AI qua tất cả các giai đoạn quy trình

Mẫu liên tiếp này lý tưởng cho:
