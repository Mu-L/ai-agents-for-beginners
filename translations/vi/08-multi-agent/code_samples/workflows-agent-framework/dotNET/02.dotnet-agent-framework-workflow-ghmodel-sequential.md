# ⏩ Quy trình làm việc tuần tự của Agent với Azure OpenAI (Responses API) (.NET)

## 📋 Hướng dẫn xử lý tuần tự nâng cao

Nhật ký này trình bày **mẫu quy trình làm việc tuần tự** sử dụng Microsoft Agent Framework cho .NET và Azure OpenAI (Responses API). Bạn sẽ học cách xây dựng các quy trình xử lý tinh vi, từng bước, nơi các agent thực hiện theo một thứ tự cụ thể, mỗi giai đoạn dựa trên kết quả của giai đoạn trước.

## 🎯 Mục tiêu học tập

### 🔄 **Kiến trúc xử lý tuần tự**
- **Thiết kế Quy trình tuyến tính**: Tạo các quy trình xử lý từng bước với các phụ thuộc rõ ràng
- **Quản lý trạng thái**: Duy trì ngữ cảnh và luồng dữ liệu qua các giai đoạn của quy trình tuần tự
- **Azure OpenAI (Responses API)**: Tận dụng các mô hình Azure OpenAI trong quy trình làm việc đa giai đoạn .NET
- **Mẫu quy trình doanh nghiệp**: Xây dựng các hệ thống xử lý tuần tự sẵn sàng cho sản xuất

### 🏗️ **Mẫu tuần tự nâng cao**
- **Xử lý có điểm kiểm tra**: Triển khai các điểm kiểm tra xác thực giữa các giai đoạn quy trình
- **Bảo tồn ngữ cảnh**: Duy trì trạng thái và kiến thức tích lũy qua tất cả các giai đoạn
- **Lan truyền lỗi**: Xử lý các lỗi một cách uyển chuyển trong chuỗi xử lý tuần tự
- **Tối ưu hiệu năng**: Thực thi tuần tự hiệu quả với chi phí tối thiểu

### 🏢 **Ứng dụng tuần tự trong doanh nghiệp**
- **Quy trình xử lý tài liệu**: Phân tích, chuyển đổi và xác thực tài liệu đa giai đoạn
- **Quy trình đảm bảo chất lượng**: Xem xét, xác thực và phê duyệt theo tuần tự
- **Quy trình sản xuất nội dung**: Nghiên cứu → Viết → Chỉnh sửa → Xem xét → Xuất bản
- **Tự động hóa quy trình kinh doanh**: Các quy trình kinh doanh đa bước với phụ thuộc giai đoạn rõ ràng

## ⚙️ Yêu cầu & Cài đặt

### 📦 **Các gói NuGet cần thiết**

Các gói thiết yếu cho quy trình làm việc tuần tự .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Cấu hình Azure OpenAI**

**Thiết lập môi trường (.env file):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Quản lý cấu hình:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **Kiến trúc Quy trình tuần tự**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Đại lý Phân tích]
    B --> C[Điểm kiểm tra 1]
    C --> D[Stage 2: Đại lý Xử lý]
    D --> E[Điểm kiểm tra 2]
    E --> F[Stage 3: Đại lý Kiểm tra]
    F --> G[Kết quả Cuối cùng]
    
    H[Ngữ cảnh Trạng thái] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API Phản hồi)] --> B
    I --> D
    I --> F
```

**Các thành phần chính:**
- **Agent tuần tự**: Các agent chuyên biệt cho mỗi giai đoạn xử lý
- **Ngữ cảnh trạng thái**: Duy trì dữ liệu tích lũy và các quyết định qua các giai đoạn
- **Điểm kiểm tra**: Các điểm xác thực giữa các giai đoạn đảm bảo chất lượng và nhất quán
- **Client Azure OpenAI**: Truy cập mô hình AI nhất quán xuyên suốt các giai đoạn quy trình

## 🎨 **Mẫu thiết kế Quy trình tuần tự**

### 📝 **Quy trình xử lý tài liệu**
```
Raw Document → Content Extraction → Analysis → Validation → Structured Output
```

### 🎯 **Quy trình tạo nội dung**
```
Brief/Requirements → Research → Content Creation → Review → Final Polish
```

### 🔍 **Quy trình Đảm bảo Chất lượng**
```
Initial Review → Technical Validation → Compliance Check → Final Approval
```

### 💼 **Quy trình Trí tuệ Kinh doanh**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 📝 Kết quả mong đợi

Quy trình sẽ:
1. **Agent Bán hàng**: Nhận diện các món nội thất từ hình ảnh và đưa ra đề nghị
2. **Agent Giá**: Thêm phân tích giá chi tiết với các mức ngân sách và đề xuất mua sắm
3. **Agent Báo giá**: Tạo tài liệu báo giá định dạng chuyên nghiệp với tất cả thông tin tổng hợp

Kết quả cuối cùng sẽ là báo giá nội thất toàn diện, chuyên nghiệp dựa trên phân tích hình ảnh.

### 🎯 Ứng dụng thực tế

Mẫu tuần tự này lý tưởng cho:
- **Thương mại điện tử**: Phân tích sản phẩm → Định giá → Tạo báo giá
- **Bất động sản**: Phân tích bất động sản → Định giá → Tạo danh sách
- **Bảo hiểm**: Phân tích yêu cầu → Đánh giá → Tạo báo giá
- **Tạo nội dung**: Nghiên cứu → Viết → Chỉnh sửa → Xuất bản

### 🔍 Hiểu luồng trạng thái

Mỗi agent trong chuỗi nhận:
- **Đầu vào gốc**: Tin nhắn người dùng ban đầu (hình ảnh + văn bản)
- **Đầu ra các agent trước**: Tất cả phản hồi các agent trước trong lịch sử hội thoại
- **Ngữ cảnh tích lũy**: Trạng thái hoàn chỉnh được duy trì xuyên suốt quy trình làm việc

Điều này cho phép xử lý đa giai đoạn tinh vi khi mỗi agent phát triển dựa trên ngữ cảnh toàn diện từ tất cả các giai đoạn trước đó.
