# AI 代理初學者 - 學習指南

使用此指南作為您學習課程的實用伴侶。此指南
並非用來取代課程，而是幫助您決定從何處開始、在每個課程中
應尋找什麼，以及如何將想法連結成一個小型實作代理
示範。

如果您是第一次來這裡，請從簡單開始：

1. 閱讀 [課程設定](./00-course-setup/README.md)。
2. 按順序完成第01至06課。
3. 在學習時，心中保持一個小型示範構想。
4. 每完成一課，問自己：「我的代理現在能做什麼，是之前
   不能做到的？」

## 讓你記住的一個簡單示範

學習代理的好方法是貫穿整個課程追蹤一個示範構想。

示範範例：<strong>課程助手代理</strong>。

用戶要求：

>「我想學習代理如何使用工具。找出合適的課程，總結我
> 先應該閱讀的內容，並給我一個簡短的練習任務。」

一般聊天機器人會根據它已知的內容回答。代理可以做更多：

1. <strong>閱讀或搜尋課程檔案</strong>，找出合適的課程。
2. <strong>使用工具</strong> 取回課程連結、範例或輔助資料。
3. <strong>規劃</strong> 一條短的學習路徑，而非給出單一長回答。
4. <strong>利用目前對話的上下文</strong>，專注於學習者的目標。
5. <strong>記住實用偏好</strong>，如果應用程式支持記憶功能。
6. **顯示追蹤紀錄、引用或日誌**，使用戶能了解發生了什麼。
7. <strong>套用安全防護措施</strong>，在執行風險操作或使用敏感資料前。

## 您建構的目標

課程結束時，您應該能解釋並建構包含這些部分的代理系統：


|------|------------------------|-------------|
| 模型 | 負責解讀使用者請求的推理引擎 | 理解學習者想找使用工具的課程 |
| 工具 | 代理可用的函式、API、檔案、瀏覽器或服務 | 搜尋代碼庫或擷取課程內容 |
| 知識 | 用以建立答案基礎的文件或資料 | 課程 README 文件與課程教材 |
| 上下文 | 加入下一次模型呼叫的資訊 | 使用者目標及工具結果 |
| 記憶 | 保存以供後續使用的資訊 | 學習者偏好動手操作的 Python 範例 |
| 規劃 | 將較大目標拆分成小步驟 | 找課程、摘要、建議練習 |
| 編排 | 在工具、步驟或代理間路由工作 | 一個規劃者呼叫搜尋工具，然後呼叫摘要工具 |
| 信任 | 安全、保安、評估和可觀察性 | 記錄工具呼叫，執行高風險前詢問 |




**Azure OpenAI Responses API** 為目標 — 這是未來推薦的 API，整合聊天完成、工具呼叫、多模態輸入和有狀態對話於單一 API 界面。您可透過 **Microsoft Foundry** 專案（使用 `FoundryChatClient`）或直接連接至 Azure OpenAI（使用 `OpenAIChatClient`）。

在學習課程時，您有多種提供者選項：

- **Microsoft Foundry / Azure OpenAI (Responses API)** — 是課程中主要路線。使用 `az login` 登入以獲得免金鑰的 Entra ID 認證。
- **Foundry Local** — 利用 OpenAI 相容 API，於裝置本地全程運行模型（無須雲端、無 API 金鑰）。適合離線或免費實驗。詳見 [課程設定](./00-course-setup/README.md)。
- **MiniMax** — 一個支援大上下文模型的 OpenAI 相容提供者，可作為即插即用替代品。

## 新手友好術語的核心概念

### 工具

工具是代理可調用以在模型外執行工作的內容。一個好的工具
具有清楚的名稱、專一職責、類型輸入、可預測輸出及安全的
故障處理方式。

對於課程輔助示範，工具可能是：

- `search_lessons(query)`
- `read_lesson(path)`
- `create_practice_task(topic)`

### RAG 與知識

RAG 幫助代理從原始資料作答，而非猜測。在本
課程中，該資料可能是課程 README、程式碼範例，或課程連結的外部
資源。

當答案應根據文件、資料或目前
專案檔案為基礎時使用 RAG。

### 規劃

請求包含多個步驟時，規劃很有用。保持計劃簡潔且
足夠清晰讓開發者或用戶能檢視。

對於示範，計劃可能是：

1. 找出與工具使用相關的課程。
2. 彙整最相關課程重點。
3. 推薦一個練習任務。

### 上下文

上下文是模型此刻經歷的內容。上下文不夠可能導致代理
遺漏重要細節。上下文過多會使代理變慢、成本更高，
或更易混淆。

良好上下文工程意指選擇下次模型
調用所需的正確資訊。

### 記憶

記憶是為之後儲存的資訊。不要保存所有內容。僅保存
有用、安全且易於更新或刪除的資訊。

例如，記得「學習者偏好 Python 範例」可能有用。
通常不記錄敏感個人資料。

### 評估與可觀察性

評估問：代理是否做了正確事？

可觀察性問：我們能看見它如何運作？

對於生產代理，追蹤模型調用、工具調用、檢索上下文、
延遲、成本、失敗與用戶反饋。

### 信任與安全

值得信賴的代理需要不只是有用的提示。使用最小權限工具、
高影響動作經人類核准、必要的資料遮蔽，以及
必須稽核動作的日誌或收據。

## 15 分鐘複習程序

每節課後使用此程序：

1. **用一句話總結課程。**
2. **說出新代理能力。** 例如：工具使用、檢索、
   規劃、記憶、可觀察性或安全性。
3. **加入課程輔助示範。** 示範現在有什麼變化？
4. **找出風險。** 若該能力被誤用，可能發生什麼？
5. **寫一個測試問題。** 你會如何檢查代理行為良好？

## 快速自我檢查

進行下一步前，先嘗試回答這些問題：

1. 代理能做但單純聊天機器人無法獨立完成什麼？
2. 你的代理首先需要什麼工具？為什麼？
3. 哪個知識來源應該是代理答案的基礎？
4. 下一個模型調用應包含什麼上下文？

5. 代理應該記住什麼，應該避免儲存什麼？
6. 代理應該在什麼時候徵求人工批准？
7. 什麼日誌、追蹤或收據會幫助你稍後調試或審計代理？


## 建議的綜合實踐練習

在課程結束時，建立一個小型代理程式，幫助學習者瀏覽此
存放庫。

最低版本：

- 接受使用者輸入的主題。
- 找出最相關的課程。
- 總結先閱讀的內容。
- 建議一個實作練習任務。
- 顯示使用了哪些課程檔案或連結。

進階版本：

- 記住學習者偏好的程式語言。
- 回答前使用簡單的計劃。
- 在最終回應前加入自我檢查步驟。
- 記錄工具調用和檢索到的來源。
- 在開啟瀏覽器或 UI 自動化任務前詢問確認。

這為你提供一個小而實際的方式，在一個專案中練習工具、RAG、計劃、
上下文、記憶、可觀察性與信任。
