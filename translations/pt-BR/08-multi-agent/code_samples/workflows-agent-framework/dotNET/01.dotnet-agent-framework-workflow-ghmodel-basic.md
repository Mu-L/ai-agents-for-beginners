# 🔄 Fluxos de Trabalho Básicos de Agentes com Azure OpenAI (API de Respostas) (.NET)

## 📋 Tutorial de Orquestração de Fluxos de Trabalho

Este notebook demonstra como construir **fluxos de trabalho sofisticados de agentes** usando o Microsoft Agent Framework para .NET e Azure OpenAI (API de Respostas). Você aprenderá a criar processos de negócios em múltiplas etapas onde agentes de IA colaboram para realizar tarefas complexas por meio de padrões estruturados de orquestração.

## 🎯 Objetivos de Aprendizagem

### 🏗️ **Fundamentos da Arquitetura de Fluxos de Trabalho**
- **Construtor de Fluxos de Trabalho**: Projetar e orquestrar processos complexos de IA em várias etapas
- **Coordenação de Agentes**: Coordenar múltiplos agentes especializados dentro dos fluxos de trabalho
- **Azure OpenAI (API de Respostas)**: Aproveitar a API de Respostas do Azure OpenAI em fluxos de trabalho
- **Design Visual de Fluxos de Trabalho**: Criar e visualizar estruturas de fluxos para melhor compreensão

### 🔄 **Padrões de Orquestração de Processos**
- **Processamento Sequencial**: Encadear múltiplas tarefas de agentes em ordem lógica
- **Gerenciamento de Estado**: Manter contexto e fluxo de dados entre etapas do fluxo de trabalho
- **Tratamento de Erros**: Implementar recuperação robusta de erros e resiliência do fluxo
- **Otimização de Desempenho**: Projetar fluxos eficientes para operações em escala empresarial

### 🏢 **Aplicações Empresariais de Fluxos de Trabalho**
- **Automação de Processos de Negócios**: Automatizar fluxos organizacionais complexos
- **Pipeline de Produção de Conteúdo**: Fluxos editoriais com revisão e aprovação
- **Automação de Atendimento ao Cliente**: Resolução de consultas em múltiplas etapas
- **Fluxos de Trabalho de Processamento de Dados**: Fluxos ETL com transformação alimentada por IA

```xml
<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Configuração do Azure OpenAI**

**Configuração do Ambiente (arquivo .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Acesso ao Azure OpenAI:**
1. Crie um recurso Azure OpenAI no portal do Azure
2. Faça o deploy de um modelo (por exemplo, `gpt-5-mini`) e anote o nome do deploy
3. Entre com `az login` e configure as variáveis de ambiente conforme mostrado acima

**Componentes Principais:**
- **WorkflowBuilder**: Motor principal de orquestração para projetar fluxos de trabalho
- **AIAgent**: Agentes individuais especializados com capacidades específicas
- **Cliente Azure OpenAI**: Integração com a API de Respostas do Azure OpenAI
- **Contexto de Execução**: Gerencia estado e fluxo de dados entre etapas do fluxo

## 💻 Executando o Código

A implementação completa está disponível em `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Este arquivo demonstra:

1. **Configuração do Ambiente** - Carregando configuração do Azure OpenAI do arquivo `.env`
2. **Configuração do Cliente Azure OpenAI** - Configurando o cliente para usar a API de Respostas do Azure OpenAI
3. **Criação de Agentes** - Definindo agentes especializados (Recepção e Concierge)
4. **Construtor de Fluxos** - Criando fluxo multi-agentes com processamento sequencial
5. **Execução do Fluxo** - Rodando o fluxo com resultados em streaming

### 🚀 Executando o Exemplo

```bash
# Torne o script executável (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Execute o fluxo de trabalho
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Ou no Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Saída Esperada

O fluxo irá:
1. Aceitar sua solicitação de destino de viagem ("Eu gostaria de ir para Paris")
2. O agente da Recepção fornece uma recomendação inicial
3. O agente Concierge revisa e aprimora a recomendação
4. A saída final exibe a conversa completa em streaming

### 🔧 Personalização

Você pode personalizar o fluxo de trabalho:
- Modificando as instruções dos agentes para alterar seu comportamento
- Adicionando mais agentes para criar fluxos complexos em várias etapas
- Alterando a mensagem do usuário para testar diferentes cenários
- Ajustando as conexões do fluxo para criar diferentes padrões de execução
