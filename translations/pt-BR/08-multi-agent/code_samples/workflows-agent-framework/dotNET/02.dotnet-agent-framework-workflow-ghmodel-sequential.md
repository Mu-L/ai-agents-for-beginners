# ⏩ Fluxos de Trabalho Sequenciais com Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial de Processamento Sequencial Avançado

Este notebook demonstra **padrões de fluxo de trabalho sequencial** usando o Microsoft Agent Framework para .NET e Azure OpenAI (Responses API). Você aprenderá a construir pipelines sofisticados de processamento passo a passo, onde agentes executam em uma ordem específica, com cada etapa se baseando nos resultados da etapa anterior.

## 🎯 Objetivos de Aprendizagem

### 🔄 **Arquitetura de Processamento Sequencial**
- **Design Linear de Workflow**: Crie pipelines de processamento passo a passo com dependências claras
- **Gerenciamento de Estado**: Mantenha o contexto e fluxo de dados através das etapas sequenciais do fluxo de trabalho
- **Azure OpenAI (Responses API)**: Aproveite os modelos Azure OpenAI em fluxos de trabalho .NET em múltiplas etapas
- **Padrões de Pipeline Empresariais**: Construa sistemas de processamento sequencial prontos para produção

### 🏗️ **Padrões Sequenciais Avançados**
- **Processamento Stage-Gate**: Implemente pontos de validação entre as etapas do fluxo de trabalho
- **Preservação do Contexto**: Mantenha estado e conhecimento acumulado ao longo de todas as etapas
- **Propagação de Erros**: Manipule falhas de forma graciosa em cadeias de processamento sequencial
- **Otimização de Performance**: Execução sequencial eficiente com overhead mínimo

### 🏢 **Aplicações Empresariais Sequenciais**
- **Pipeline de Processamento de Documentos**: Análise, transformação e validação de documentos em múltiplas etapas
- **Fluxos de Trabalho de Garantia de Qualidade**: Processos sequenciais de revisão, validação e aprovação
- **Pipeline de Produção de Conteúdo**: Pesquisa → Escrita → Edição → Revisão → Publicação
- **Automação de Processos de Negócios**: Fluxos de trabalho empresariais com etapas dependentes claras

### 🏗️ **Arquitetura do Fluxo de Trabalho Sequencial**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Agente de Análise]
    B --> C[Ponto de Verificação 1]
    C --> D[Stage 2: Agente de Processamento]
    D --> E[Ponto de Verificação 2]
    E --> F[Stage 3: Agente de Validação]
    F --> G[Resultado Final]
    
    H[Contexto do Estado] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API de Respostas)] --> B
    I --> D
    I --> F
```

**Componentes-chave:**
- **Agentes Sequenciais**: Agentes especializados para cada etapa de processamento
- **Contexto de Estado**: Mantém dados acumulados e decisões em todas as etapas
- **Pontos de Verificação**: Pontos de validação entre etapas para garantir qualidade e consistência
- **Cliente Azure OpenAI**: Acesso consistente ao modelo de IA em todas as etapas do fluxo de trabalho

## 💻 Executando o Código

A implementação completa está disponível em `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Este arquivo demonstra um **fluxo de trabalho de análise de móveis em três etapas**:

1. **Etapa 1 - Agente de Vendas**: Analisa imagens de móveis e fornece sugestões de compra
2. **Etapa 2 - Agente de Preços**: Fornece detalhamento de preços e opções de orçamento
3. **Etapa 3 - Agente de Cotação**: Gera um documento de cotação profissional em formato Markdown

### 🏗️ **Arquitetura do Fluxo de Trabalho**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Cada agente:
- Recebe a saída da etapa anterior como contexto
- Constrói sobre a análise prévia com expertise especializada
- Mantém a continuidade do fluxo de trabalho através do gerenciamento de estado

### 🚀 Executando o Exemplo

**Pré-requisitos:**
- Coloque uma imagem de móveis em `../imgs/home.png` (ou atualize a variável `imgPath`)
- Configure seu arquivo `.env` com seu endpoint e implantação do Azure OpenAI, depois faça login com `az login`

```bash
# Torne o script executável (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Execute o fluxo de trabalho sequencial
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

```powershell

```

### 📝 Saída Esperada

O fluxo de trabalho irá:
1. **Agente de Vendas**: Identificar itens de móveis na imagem e fornecer recomendações
2. **Agente de Preços**: Adicionar análise detalhada de preços com faixas de orçamento e sugestões de compra
3. **Agente de Cotação**: Gerar um documento de cotação formatado com todas as informações sintetizadas

A saída final será uma cotação completa e profissional de móveis baseada na análise da imagem.

### 🔍 Entendendo o Fluxo de Estado

Cada agente na sequência recebe:
- **Entrada Original**: A mensagem inicial do usuário (imagem + texto)
- **Saídas dos Agentes Anteriores**: Todas as respostas anteriores dos agentes no histórico da conversa
- **Contexto Acumulado**: Estado completo mantido durante todo o fluxo de trabalho

Isso possibilita um processamento sofisticado em múltiplas etapas onde cada agente se baseia em um contexto abrangente de todas as etapas anteriores.
