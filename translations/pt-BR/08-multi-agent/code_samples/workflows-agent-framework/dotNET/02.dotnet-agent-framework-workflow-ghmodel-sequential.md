# ⏩ Fluxos de Trabalho Sequenciais com Agentes usando Azure OpenAI (API de Respostas) (.NET)

## 📋 Tutorial Avançado de Processamento Sequencial

Este notebook demonstra **padrões de fluxo de trabalho sequencial** usando o Microsoft Agent Framework para .NET e Azure OpenAI (API de Respostas). Você aprenderá como construir pipelines sofisticados de processamento passo a passo, onde agentes executam em uma ordem específica, com cada etapa se baseando nos resultados da etapa anterior.

## 🎯 Objetivos de Aprendizagem

### 🔄 **Arquitetura de Processamento Sequencial**
- **Design de Fluxo de Trabalho Linear**: Crie pipelines de processamento passo a passo com dependências claras

- **Azure OpenAI (API de Respostas)**: Aproveite modelos Azure OpenAI em fluxos de trabalho .NET de múltiplas etapas
- **Padrões de Pipeline Empresarial**: Construa sistemas de processamento sequencial prontos para produção

### 🏗️ **Padrões Avançados Sequenciais**
- **Processamento Stage-Gate**: Implemente pontos de validação entre as etapas do fluxo de trabalho
- **Preservação de Contexto**: Mantenha estado e conhecimento acumulado por todas as etapas
- **Propagação de Erros**: Trate falhas de forma elegante em cadeias de processamento sequencial
- **Otimização de Performance**: Execução sequencial eficiente com sobrecarga mínima

### 🏢 **Aplicações Empresariais Sequenciais**
- **Pipeline de Processamento de Documentos**: Análise, transformação e validação multiestágio de documentos
- **Fluxos de Trabalho de Garantia de Qualidade**: Processos sequenciais de revisão, validação e aprovação
- **Pipeline de Produção de Conteúdo**: Pesquisa → Redação → Edição → Revisão → Publicação

### 🏗️ **Arquitetura do Fluxo de Trabalho Sequencial**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Agente de Análise]
    B --> C[Ponto de Verificação 1]
    C --> D[Stage 2: Agente de Processamento]
    D --> E[Ponto de Verificação 2]
    E --> F[Stage 3: Agente de Validação]
    F --> G[Resultado Final]
    
    H[Contexto de Estado] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API de Respostas)] --> B
    I --> D
    I --> F
```

**Componentes-Chave:**
- **Agentes Sequenciais**: Agentes especializados para cada etapa do processamento
- **Contexto de Estado**: Mantém dados e decisões acumulados através das etapas
- **Checkpoints**: Pontos de validação entre as etapas para garantir qualidade e consistência
- **Cliente Azure OpenAI**: Acesso consistente ao modelo de IA em todas as etapas do fluxo de trabalho

## 💻 Executando o Código

A implementação completa está disponível em `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Este arquivo demonstra um **fluxo de trabalho de análise de móveis em três etapas**:

1. **Etapa 1 - Agente de Vendas**: Analisa imagens de móveis e fornece sugestões de compra
2. **Etapa 2 - Agente de Preço**: Fornece detalhamentos de preços e opções de orçamento
3. **Etapa 3 - Agente de Cotação**: Gera um documento de cotação profissional em formato Markdown

### 🏗️ **Arquitetura do Fluxo de Trabalho**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Cada agente:
- Recebe a saída da etapa anterior como contexto
- Expande a análise anterior com expertise especializada
- Mantém a continuidade do fluxo de trabalho por meio do gerenciamento de estado

### 🚀 Executando o Exemplo

**Pré-requisitos:**
- Coloque uma imagem de móvel em `../imgs/home.png` (ou atualize a variável `imgPath`)
- Configure seu arquivo `.env` com seu endpoint e implantação do Azure OpenAI, depois faça login com `az login`

### 📝 Saída Esperada

O fluxo de trabalho irá:
1. **Agente de Vendas**: Identificar itens de móveis na imagem e fornecer recomendações
2. **Agente de Preço**: Adicionar análise detalhada de preços com faixas orçamentárias e recomendações de compra
3. **Agente de Cotação**: Gerar um documento de cotação formatado com todas as informações sintetizadas

A saída final será uma cotação abrangente e profissional de móveis baseada na análise da imagem.

### 🔍 Compreendendo o Fluxo de Estado

Cada agente na sequência recebe:
- **Entrada Original**: A mensagem inicial do usuário (imagem + texto)
- **Saídas dos Agentes Anteriores**: Todas as respostas dos agentes anteriores no histórico da conversa
- **Contexto Acumulado**: Estado completo mantido durante todo o fluxo de trabalho

Isso permite um processamento multiestágio sofisticado onde cada agente se baseia em um contexto abrangente de todas as etapas anteriores.
