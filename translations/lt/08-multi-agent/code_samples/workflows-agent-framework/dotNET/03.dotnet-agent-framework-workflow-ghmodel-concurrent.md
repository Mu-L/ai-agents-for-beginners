# ⚡ Vienlaikiniai agentų darbo srautai su Azure OpenAI (Responses API) (.NET)

## 📋 Aukštos spartos lygiagretus apdorojimo vadovas

Šiame užrašų knygyne demonstruojami **vienalaikiai darbo srautų modeliai** naudojant Microsoft Agent Framework .NET kalbai ir Azure OpenAI (Responses API). Sužinosite, kaip kurti aukšto našumo lygiagrečius darbo srautus, kurie maksimaliai padidina pralaidumą vykdant kelis dirbtinio intelekto agentus vienu metu, išlaikant koordinaciją ir duomenų nuoseklumą.

## 🎯 Mokymosi tikslai

### 🚀 **Vienalaikio apdorojimo pagrindai**
- **Lygiagretus agentų vykdymas**: vienu metu vykdykite kelis DI agentus, kad pasiektumėte maksimalų našumą
- **Async/Await modeliai**: naudokite .NET asinchroninio programavimo modelį efektyviai vienalaikei veiklai
- **Azure OpenAI (Responses API)**: koordinuokite kelis lygiagreičius kvietimus Azure OpenAI Responses API
- **Resursų valdymas**: efektyviai tvarkykite DI modelių resursus vykdant tuo pačiu metu

### 🏗️ **Išplėstinė vienalaikio veikimo architektūra**
- **Užduočių pagrindu veikiantis lygiagretumas**: naudokite .NET Task Parallel Library norėdami optimaliai vykdyti vienalaikiai
- **Sinchronizavimo modeliai**: koordinuokite vienalaikius agentus vengiant konkurencijos
- **Darbo paskirstymas**: efektyviai paskirstykite darbą per turimą vienalaikio apdorojimo galią
- **Avarijų tolerancija**: valdykite pavienių agentų gedimus nesustabdydami viso darbo srauto

### 🏢 **Įmonių vienalaikės programos**
- **Didelio apimties dokumentų apdorojimas**: vienu metu apdorokite daugybę dokumentų
- **Tikro laiko turinio analizė**: vienalaikė atvykstančių duomenų srautų analizė
- **Paketo apdorojimo optimizavimas**: maksimaliai padidinkite pralaidumą didelio masto duomenų apdorojimo operacijoms
- **Daugiapakopė analizė**: lygiagretus skirtingų turinio tipų ir formatų apdorojimas

## 💻 Kodo paleidimas

Pilna įgyvendinimo versija yra faile `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Šis failas demonstruoja **Fan-Out/Fan-In vienalaikį darbo srautą** kelionių planavimui:

### 🏗️ **Darbo srauto architektūra**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Pagrindinės sudedamosios dalys:**

1. **ConcurrentStartExecutor**: vienu metu perduoda naudotojo užklausą visiems agentams
2. **Researcher Agent**: vienalaikiai analizuoja paskirties vietas ir atrakcionus
3. **Planner Agent**: vienalaikiai kuria detalius kelionių planus
4. **ConcurrentAggregationExecutor**: surenka ir sujungia abu agentų rezultatus

### 🎯 **Fan-Out/Fan-In modelis**

Šis darbo srautas demonstruoja klasikinių **Fan-Out/Fan-In** modelį:
- **Fan-Out**: vienas įėjimo pranešimas vienu metu perduodamas keliems agentams
- **Vienalaikis apdorojimas**: keli agentai lygiagrečiai dirba su ta pačia užduotimi
- **Fan-In**: iš visų agentų surinkti rezultatai apjungiami į vieną išvestį

### 🎯 Tikro pasaulio taikymai

Šis vienalaikis modelis yra idealus:
- **Turinio kūrimui**: keli rašytojai vienu metu kuria skirtingas dalis
- **Kodo peržiūrai**: keli peržiūrėtojai analizuoja kodą iš skirtingų perspektyvų
- **Rinkos tyrimams**: lygiagretus skirtingų rinkos segmentų analizavimas
- **Dokumentų apdorojimui**: vienalaikis duomenų išgavimas, analizė ir tikrinimas
- **Daugiaperspektyvinė analizė**: įvairovės požiūrių gavimas apie tą patį įvestį

### 🔍 Pasirinktinių vykdytojų supratimas

**ConcurrentStartExecutor:**
- Įgyvendina `IMessageHandler<string>` priimti eilutės įvestį
- Siunčia žinutes visiems prijungtiems agentams
- Siunčia `TurnToken` suvienalaikintam apdorojimui pradėti

**ConcurrentAggregationExecutor:**
- Įgyvendina `IMessageHandler<ChatMessage>` agentų atsakymų priėmimui
- Surenka žinutes saugiai iš kelių gijų
- Atlieka agregavimą gavus visus laukiamus atsakymus
- Gražina galutinį rezultatą naudodamas `context.YieldOutputAsync()`

### ⚡ Našumo privalumai

**Vienalaikis vs Sekos vykdymas:**
- Sekos: Agentas1 (30s) → Agentas2 (30s) = **iš viso 60 sekundžių**
- Vienalaikis: Agentas1 (30s) || Agentas2 (30s) = **iš viso 30 sekundžių**

**Pralaidumo pagerėjimas**: iki N kartų greitesnis N vienalaikių agentų atveju (priklauso nuo darbo krūvio ir resursų)

### 🛡️ Klaidų valdymas

Darbo srautas gražiai tvarko pavienių agentų gedimus:
- Jei vienas agentas sugenda, kiti tęsia darbą
- Agregatorius gali įgyvendinti laiko limitų logiką
- Jei reikia, gali gražinti dalinius rezultatus
