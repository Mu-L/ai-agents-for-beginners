# 🔄 Pagrindiniai agentų darbo eigos srautai su Azure OpenAI (Atsakymų API) (.NET)

## 📋 Darbo eigos srauto orkestravimo pamoka

Šis užrašų knygutė parodo, kaip sukurti pažangius **agentų darbo eigos srautus** naudojant Microsoft Agent Framework .NET bei Azure OpenAI (Atsakymų API). Išmoksite kurti daugiapakopes verslo procesų grandines, kur AI agentai bendradarbiauja, kad atliktų sudėtingas užduotis per struktūruotus orkestravimo modelius.

## 🎯 Mokymosi tikslai

### 🏗️ **Darbo eigos architektūros pagrindai**
- **Darbo eigos kūrimo priemonė**: Projektuoti ir orkestruoti sudėtingus daugiapakopius AI procesus
- **Agentų koordinavimas**: Koordinuoti kelis specializuotus agentus darbo eigos srautuose
- **Azure OpenAI (Atsakymų API)**: Naudoti Azure OpenAI Atsakymų API darbo eigoje
- **Darbo eigos vizualizavimas**: Kurti ir vizualizuoti darbo eigos struktūras geresniam supratimui

### 🔄 **Proceso orkestravimo modeliai**
- **Sekvinis apdorojimas**: Grandininis kelių agentų užduočių vykdymas logine tvarka
- **Būsenos valdymas**: Išlaikyti kontekstą ir duomenų srautą per darbo eigos etapus
- **Klaidų valdymas**: Įgyvendinti atsparų klaidų atkūrimą ir darbo eigos patvarumą
- **Veikimo optimizavimas**: Projektuoti efektyvias darbo eigas įmonių mastui

### 🏢 **Įmonių darbo eigos taikymai**
- **Verslo procesų automatizavimas**: Automatizuoti sudėtingus organizacinius darbo eigos srautus
- **Turinio gamybos pipeline**: Redakciniai darbo eigos srautai su peržiūros ir patvirtinimo etapais
- **Klientų aptarnavimo automatizavimas**: Daugiapakopis klientų užklausų sprendimas
- **Duomenų apdorojimo darbo eigos**: ETL darbo eigos su AI varomais transformavimais

## ⚙️ Reikalavimai ir paruošimas

### 📦 **Reikalingos NuGet paketo bibliotekos**

Šiame darbo eigos pavyzdyje naudojamos kelios pagrindinės .NET bibliotekos:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI konfigūracija**

**Aplinka (.env failas):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Azure OpenAI prieiga:**
1. Sukurkite Azure OpenAI išteklių Azure portale
2. Diegkite modelį (pvz., `gpt-5-mini`) ir užsirašykite diegimo pavadinimą
3. Prisijunkite su `az login` ir konfigūruokite aplinkos kintamuosius kaip parodyta aukščiau

### 🏗️ **Darbo eigos architektūros apžvalga**

```mermaid
graph TD
    A[Darbo eigos kūrėjas] --> B[Agentų registras]
    B --> C[Darbo eigos vykdymo variklis]
    C --> D[Agent 1: Turinys generatorius
    C --> E[Agent 2: Turinys peržiūrėtojas 
    D --> F[Darbo eigos rezultatai]
    E --> F
    G[Azure OpenAI (Atsakymų API)] --> D
    G --> E
```

**Pagrindiniai komponentai:**
- **WorkflowBuilder**: Pagrindinis orkestravimo variklis darbo eigos projektavimui
- **AIAgent**: Individualūs specializuoti agentai su specifinėmis galimybėmis
- **Azure OpenAI Client**: Azure OpenAI Atsakymų API integracija
- **Execution Context**: Valdo būseną ir duomenų srautą tarp darbo eigos etapų

## 🎨 **Įmonių darbo eigos dizaino modeliai**

### 📝 **Turinio gamybos darbo eiga**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **Dokumentų apdorojimo pipeline**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **Verslo intelekto darbo eiga**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **Klientų aptarnavimo automatizavimas**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 💻 Kodo paleidimas

Pilną implementaciją rasite faile `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Šiame faile demonstruojama:

1. **Aplinkos konfigūracija** – Azure OpenAI konfigūracijos įkėlimas iš `.env` failo
2. **Azure OpenAI kliento nustatymas** – Kliento konfigūravimas naudoti Azure OpenAI Atsakymų API
3. **Agentų kūrimas** – Specializuotų agentų apibrėžimas (Recepcijos ir Konsjeržo agentai)
4. **Darbo eigos kūrėjas** – Daugiagentės darbo eigos su sekvenciniu apdorojimu kūrimas
5. **Darbo eigos vykdymas** – Darbo eigos paleidimas su tiesiogine rezultatų transliacija

### 🚀 Pavyzdžio paleidimas

```bash
# Padarykite scenarijų vykdomu (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Vykdyti darbo eigą
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Ar Windows aplinkoje:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Tikėtinas rezultatas

Darbo eiga atliks šiuos veiksmus:
1. Priims jūsų kelionės pageidavimą („Norėčiau vykti į Paryžių“)
2. Recepcijos agentas pateiks pradinę rekomendaciją
3. Konsjeržo agentas peržiūrės ir patikslins rekomendaciją
4. Galutinis rezultatas rodys visą pokalbio srautą

### 🔧 Individualizavimas

Darbo eigą galite pritaikyti:
- Keisdami agentų instrukcijas jų elgesiui pakeisti
- Pridėdami daugiau agentų sudėtingų daugiapakopių darbo eigų kūrimui
- Keisdami vartotojo žinutę, norėdami išbandyti skirtingus scenarijus
- Koreguodami darbo eigos viršūnes norėdami sukurti įvairius vykdymo modelius
