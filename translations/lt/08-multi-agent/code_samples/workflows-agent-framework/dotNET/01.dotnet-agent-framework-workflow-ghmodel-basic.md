# 🔄 Pagrindiniai agentų darbo procesai su Azure OpenAI (Responses API) (.NET)

## 📋 Darbo proceso orkestravimo pamoka

Šis užrašų knygelės pavyzdys demonstruoja, kaip sukurti sudėtingus **agentų darbo procesus** naudojant Microsoft Agent Framework .NET ir Azure OpenAI (Responses API). Išmoksite kurti daugiapakopes verslo procesų sąveikas, kuriose AI agentai bendradarbiauja, kad įvykdytų sudėtingas užduotis taikydami struktūrizuotus orkestracijos šablonus.

## 🎯 Mokymosi tikslai

### 🏗️ **Darbo proceso architektūros pagrindai**
- **Workflow Builder**: Projektuoti ir orkestruoti sudėtingus daugiapakopius AI procesus
- **Agentų koordinavimas**: Koordinuoti kelis specializuotus agentus darbo procesuose
- **Azure OpenAI (Responses API)**: Naudoti Azure OpenAI Responses API darbo procesuose
- **Vizualus darbo proceso dizainas**: Kurti ir vizualizuoti darbo procesų struktūras geresniam supratimui

### 🔄 **Proceso orkestravimo šablonai**
- **Sekveninis apdorojimas**: Susieti kelias agentų užduotis logine tvarka
- **Būsena valdymas**: Palaikyti kontekstą ir duomenų srautą darbo proceso etapuose
- **Klaidų valdymas**: Įgyvendinti tvirtą klaidų atkūrimą ir darbo proceso atsparumą
- **Veiklos optimizavimas**: Projektuoti efektyvius darbo procesus įmonių mastu

### 🏢 **Įmonių darbo proceso taikymai**
- **Verslo procesų automatizavimas**: Automatizuoti sudėtingus organizacinius darbo procesus
- **Turinio gamybos srautas**: Redakciniai darbo procesai su peržiūros ir patvirtinimo etapais
- **Klientų aptarnavimo automatizavimas**: Daugiapakopiai klientų užklausų sprendimai
- **Duomenų apdorojimo darbo procesai**: ETL darbo procesai su AI pagrįstomis transformacijomis

## ⚙️ Reikalavimai ir nustatymai

### 📦 **Reikalingos NuGet paketiniai**

Šiame darbo proceso demonstraciniame pavyzdyje naudojami keli pagrindiniai .NET paketai:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI konfigūracija**

**Aplinkos nustatymas (.env failas):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI prieiga:**
1. Sukurkite Azure OpenAI išteklių Azure portale
2. Įdiekite modelį (pvz., `gpt-4.1-mini`) ir užsirašykite diegimo pavadinimą
3. Prisijunkite naudodami `az login` ir sukonfigūruokite aplinkos kintamuosius kaip parodyta aukščiau

### 🏗️ **Darbo proceso architektūros apžvalga**

```mermaid
graph TD
    A[Darbo eigos kūrėjas] --> B[Agentų registras]
    B --> C[Darbo eigos vykdymo variklis]
    C --> D[Agent 1: Turinys generatorius]
    C --> E[Agent 2: Turinys peržiūrėtojas] 
    D --> F[Darbo eigos rezultatai]
    E --> F
    G[Azure OpenAI (Atsakymų API)] --> D
    G --> E
```

**Pagrindinės sudedamosios dalys:**
- **WorkflowBuilder**: Pagrindinis orkestravimo variklis darbo proceso kūrimui
- **AIAgent**: Pavieniai specializuoti agentai su konkrečiomis galimybėmis
- **Azure OpenAI klientas**: Azure OpenAI Responses API integracija
- **Vykdymo kontekstas**: Valdo būseną ir duomenų srautą tarp darbo proceso etapų

## 🎨 **Įmonių darbo proceso dizaino šablonai**

### 📝 **Turinio gamybos darbo procesas**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **Dokumentų apdorojimo srautas**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **Verslo žvalgybos darbo procesas**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **Klientų aptarnavimo automatizavimas**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

- **Duomenų privatumas**: Saugus jautrios informacijos tvarkymas visuose darbo procesuose

### 📊 **Stebimas ir valdymas**

- **Vykdymo stebėjimas**: Darbo proceso pažangos ir našumo realaus laiko sekimas

- **Veiklos analizė**: Matavimai optimizavimui ir pajėgumų planavimui

## 💻 Kodo vykdymas

Pilną įgyvendinimą rasite faile `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Ši byla demonstruoja:

1. **Aplinkos konfigūracija** - Azure OpenAI konfigūracijos įkėlimas iš `.env` failo
2. **Azure OpenAI kliento nustatymas** - Kliento konfigūracija naudoti Azure OpenAI Responses API
3. **Agentų kūrimas** - Specializuotų agentų (Registratūros ir Konsjeržo) apibrėžimas
4. **Workflow Builder** - Daugiagentės darbo eigos kūrimas su sekvenciniu apdorojimu
5. **Darbo proceso vykdymas** - Vykdymas su srautiniais rezultatais

### 🚀 Pavyzdžio vykdymas

```bash
# Padarykite scenarijų vykdomu (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Paleiskite darbo eigą
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Arba Windows aplinkoje:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Laukiamas rezultatas

Darbo eiga atliks:
1. Priims jūsų kelionės tikslą („Norėčiau keliauti į Paryžių“)
2. Registratūros agentas pateiks pradinį rekomendaciją
3. Konsjeržo agentas peržiūrės ir patobulins rekomendaciją
4. Galutinis rezultatas atvaizduos visą pokalbio srautą

### 🔧 PrCustomizationavimas

Darbo eigą galite pritaikyti:
- Keisdami agentų nurodymus jų elgesiui modifikuoti
- Pridėdami daugiau agentų sudėtingiems daugiapakopiams darbo procesams kurti
- Keisdami vartotojo žinutę, kad išbandytumėte skirtingus scenarijus
- Koreguodami darbo proceso kraštus skirtingų vykdymo šablonų kūrimui
