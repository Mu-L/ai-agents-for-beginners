# ⏩ Sekviniai agentų darbo eiga su Azure OpenAI (Atsakymų API) (.NET)

## 📋 Pažangus sekvencinio apdorojimo vadovas

Ši užduočių knyga demonstruoja **sekvencinių darbo eigos šablonų** naudojimą su „Microsoft Agent Framework“ .NET aplinkoje ir Azure OpenAI (Atsakymų API). Išmoksite sukurti sudėtingas, žingsnis po žingsnio vykdomas apdorojimo grandines, kuriose agentai dirba tam tikra tvarka, kiekvienas etapas remiasi ankstesnio etapo rezultatais.

## 🎯 Mokymosi tikslai

### 🔄 **Sekvencinio apdorojimo architektūra**
- **Linearinis darbo eigos dizainas**: Kurkite žingsnis po žingsnio apdorojimo grandines su aiškiomis priklausomybėmis
- **Būsenos valdymas**: Išlaikyti kontekstą ir duomenų srautą per sekvencinius darbo eigos etapus
- **Azure OpenAI (Atsakymų API)**: Naudokite Azure OpenAI modelius daugiaetapiuose .NET darbo eigos procesuose
- **Įmonės grandinių šablonai**: Kurkite gamybai paruoštas sekvencines apdorojimo sistemas

### 🏗️ **Pažangūs sekvenciniai šablonai**
- **Etapas vartai apdorojimas**: Diegti patikros taškus tarp darbo eigos etapų
- **Konteksto išsaugojimas**: Išlaikyti būseną ir sukauptas žinias visuose etapuose
- **Klaidų plitimas**: Tvarkyti gedimus maloniai sekvencinėse grandinėse
- **Veikimo optimizavimas**: Efektyvus sekvencinis vykdymas su minimaliomis sąnaudomis

### 🏢 **Įmonių sekvencinės programos**
- **Dokumentų apdorojimo grandinė**: Daugiaetapis dokumentų analizavimas, transformavimas ir patikra
- **Kokybės užtikrinimo darbo eiga**: Sekvencinis peržiūrėjimas, patikra ir patvirtinimo procesai
- **Turinio gamybos grandinė**: Tyrimai → Rašymas → Redagavimas → Peržiūra → Leidyba
- **Verslo proceso automatizavimas**: Daugiapakopiai verslo procesai su aiškiomis etapų priklausomybėmis

## ⚙️ Reikalavimai ir paruošimas

### 📦 **Būtini NuGet paketai**

Esminiai paketai .NET sekvencinėms darbo eigoms:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```


### 🏗️ **Sekvencinės darbo eigos architektūra**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Analizės Agentas]
    B --> C[Kontrolinis taškas 1]
    C --> D[Stage 2: Apdorojimo Agentas]
    D --> E[Kontrolinis taškas 2]
    E --> F[Stage 3: Tikrinimo Agentas]
    F --> G[Galutinis rezultatas]
    
    H[Būsenos Kontekstas] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Atsakymų API)] --> B
    I --> D
    I --> F
```

**Pagrindiniai komponentai:**
- **Sekviniai agentai**: Specializuoti agentai kiekvienam apdorojimo etapui
- **Būsenos kontekstas**: Išlaiko sukauptus duomenis ir sprendimus per etapus
- **Patikros taškai**: Validacijos taškai tarp etapų, užtikrinantys kokybę ir nuoseklumą
- **Azure OpenAI klientas**: Pastovi AI modelio prieiga visuose darbo eigos etapuose

## 💻 Kodo paleidimas

Pilnas įgyvendinimas pateiktas faile `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Šis failas demonstruoja **trijų etapų baldų analizės darbo eigą**:

1. **1 etapas - Pardavimų agentas**: analizuoja baldų nuotraukas ir pateikia pirkimo pasiūlymus
2. **2 etapas - Kainų agentas**: pateikia detalius kainų suskaidymus ir biudžeto parinktis
3. **3 etapas - Pasiūlymo agentas**: generuoja profesionalų pasiūlymo dokumentą Markdown formatu

### 🏗️ **Darbo eigos architektūra**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Kiekvienas agentas:
- Gauta ankstesnio etapo išvestis kaip kontekstas
- Remiasi ankstesne analize su specializuotomis žiniomis
- Išlaiko darbo eigos tęstinumą valdomai būsenai

### 🚀 Pavyzdžio paleidimas

**Reikalavimai:**
- Įdėkite baldų nuotrauką į `../imgs/home.png` (arba atnaujinkite `imgPath` kintamąjį)
- Konfigūruokite savo `.env` failą su Azure OpenAI pabaigos tašku ir diegimu, tada prisijunkite su `az login`

```bash
# Padarykite scenarijų vykdomu (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Vykdykite sekvenčią darbų eigą
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Ar Windows aplinkoje:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Tikėtinas rezultatas

Darbo eiga atliks:
1. **Pardavimų agentas**: Nustatys baldų objektus nuotraukoje ir pateiks rekomendacijas
2. **Kainų agentas**: Pridės išsamią kainų analizę su biudžeto lygiais ir pirkimo rekomendacijomis
3. **Pasiūlymo agentas**: Sugeneruos suformuotą pasiūlymo dokumentą su visos informacijos santrauka

Galutinis rezultatas bus išsamus, profesionalus baldų pasiūlymas pagal nuotraukos analizę.
