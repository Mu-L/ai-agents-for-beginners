# ⏩ Nuoseklūs agentų darbo srautai su Azure OpenAI (Atsakymai API) (.NET)

## 📋 Išplėstinis nuosekliojo apdorojimo vadovas

Šis užrašų knygelės pavyzdys demonstruoja **nuoseklaus darbo srautų modelius** naudojant Microsoft Agent Framework .NET ir Azure OpenAI (Atsakymai API). Sužinosite, kaip sukurti sudėtingas, žingsnis po žingsnio vykdomas apdorojimo grandines, kur agentai vykdo veiksmus tam tikra tvarka, o kiekvienas etapas remiasi ankstesnio rezultatais.

## 🎯 Mokymosi tikslai

### 🔄 **Nuoseklaus apdorojimo architektūra**
- **Linijinis darbo srauto dizainas**: Sukurkite žingsnis po žingsnio apdorojimo grandines su aiškiomis priklausomybėmis
- **Būsenos valdymas**: Išlaikyti kontekstą ir duomenų srautą per nuoseklius darbo srauto etapus
- **Azure OpenAI (Atsakymai API)**: Naudokite Azure OpenAI modelius daugiasluoksniuose .NET darbo srautuose
- **Įmonių grandinių modeliai**: Kurkite gamybinės kokybės nuoseklaus apdorojimo sistemas

### 🏗️ **Pažangūs nuoseklūs modeliai**
- **Etapų tikrinimo apdorojimas**: Įgyvendinkite patvirtinimo kontrolinius punktus tarp darbo srauto etapų
- **Konteksto išsaugojimas**: Išlaikykite būseną ir sukauptas žinias visuose etapuose
- **Klaidų propagavimas**: Tvarkykite nesėkmes nuoseklių apdorojimo grandinėse maloniai
- **Veikimo našumo optimizavimas**: Efektyvus nuoseklus vykdymas su minimaliomis sąnaudomis

### 🏢 **Įmonių nuoseklios programos**
- **Dokumentų apdorojimo grandinė**: Daugiapakopis dokumentų analizė, transformavimas ir patvirtinimas
- **Kokybės užtikrinimo darbo srautai**: Nuoseklus peržiūros, patvirtinimo ir patvirtinimo procesai
- **Turinio gamybos grandinė**: Tyrimai → Rašymas → Redagavimas → Peržiūra → Leidyba
- **Verslo procesų automatizavimas**: Daugiapakopiai verslo darbo srautai su aiškiomis etapų priklausomybėmis

## ⚙️ Reikalavimai ir nustatymas

### 📦 **Būtinos NuGet paketai**

Esminiai paketai .NET nuosekliems darbo srautams:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Azure OpenAI konfigūracija**

**Aplinkos nustatymas (.env failas):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Konfigūracijos valdymas:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **Nuoseklaus darbo srauto architektūra**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Analizės agentas]
    B --> C[Kontrolinis taškas 1]
    C --> D[Stage 2: Apdorojimo agentas]
    D --> E[Kontrolinis taškas 2]
    E --> F[Stage 3: Patvirtinimo agentas]
    F --> G[Galutinis rezultatas]
    
    H[Būsena Kontekstas] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Atsakymų API)] --> B
    I --> D
    I --> F
```

**Pagrindinės dalys:**
- **Nuoseklūs agentai**: specializuoti agentai kiekvienam apdorojimo etapui
- **Būsenos kontekstas**: išlaiko sukauptus duomenis ir sprendimus per etapus
- **Kontroliniai punktai**: patvirtinimo taškai tarp etapų kokybei ir nuoseklumui užtikrinti
- **Azure OpenAI klientas**: nuoseklus dirbtinio intelekto modelio prieinamumas visuose darbo srauto etapuose

## 💻 Kodo paleidimas

Pilna įgyvendinimo versija yra faile `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Šis failas demonstruoja **trijų etapų baldų analizės darbo srautą**:

1. **1 etapas - Pardavimų agentas**: analizuoja baldų vaizdus ir teikia pirkimo rekomendacijas
2. **2 etapas - Kainų agentas**: pateikia detalias kainų dalis ir biudžeto parinktis
3. **3 etapas - Sąmatų agentas**: generuoja profesionalų sąmatų dokumentą Markdown formatu

### 🏗️ **Darbo srauto architektūra**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Kiekvienas agentas:
- Gautų ankstesnio etapo rezultatą kaip kontekstą
- Remiasi ankstesnės analizės rezultatais su specializuota kompetencija
- Išlaiko darbo srauto tęstinumą per būsenos valdymą

### 🚀 Pavyzdžio paleidimas

**Reikalavimai:**
- Įdėkite baldų vaizdą į `../imgs/home.png` (arba atnaujinkite `imgPath` kintamąjį)
- Suveskite savo `.env` failą su Azure OpenAI galiniu tašku ir diegimu, tada prisijunkite naudodami `az login`

```bash
# Padarykite skriptą vykdomu (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Paleiskite nuoseklų darbo eigą
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Ar Windows sistemoje:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Tikėtinas rezultatas

Darbo srautas atliks:
1. **Pardavimų agentas**: identifikuos baldų objektus iš vaizdo ir pateiks rekomendacijas
2. **Kainų agentas**: pridės detalią kainų analizę su biudžeto lygiais ir pirkinių pasiūlymais
3. **Sąmatų agentas**: sugeneruos suformatuotą sąmatų dokumentą, kuriame bus sujungta visa informacija

### 🎯 Tikrosios pasaulio taikomosios sritys

Šis nuoseklus modelis tinka:
- **Elektroninė prekyba**: produkto analizė → kainodara → sąmatų sudarymas
- **Nekilnojamasis turtas**: turto analizė → vertinimas → įrašų kūrimas
- **Draudimas**: pretenzijų analizė → įvertinimas → sąmatų sudarymas
- **Turinio kūrimas**: tyrimai → rašymas → redagavimas → leidyba

### 🔍 Būsenos srauto supratimas

Kiekvienas sekos agentas gauna:
- **Originalią įvestį**: pradinį vartotojo pranešimą (vaizdą + tekstą)
- **Ankstesnių agentų išvestis**: visas ankstesnias agentų atsakymus pokalbių istorijoje
- **Sukauptą kontekstą**: visą darbo srauto metu išlaikomą būseną

Tai leidžia sudėtingą daugiasluoksnį apdorojimą, kai kiekvienas agentas remiasi visapusišku visu ankstesnių etapų kontekstu.
