# 🔍 ការបង្កើត Enterprise RAG ជាមួយ Microsoft Foundry (.NET)

## 📋 គោលបំណងរៀន

សៀវភៅ​កំណត់​សម្គាល់នេះបង្ហាញពីរបៀបសាងសង់ប្រព័ន្ធ Retrieval-Augmented Generation (RAG) សម្រាប់សហគ្រាសប្រកបដោយគុណភាពខ្ពស់ ដោយប្រើប្រាស់ Microsoft Agent Framework ក្នុង .NET ជាមួយ Microsoft Foundry។ អ្នកនឹងរៀនពីរបៀបបង្កើតភ្នាក់ងារ​ដែលសម្រាប់ផលិតកម្ម​ដែលអាចស្វែងរកតាមឯកសារ ហើយផ្តល់ឲ្យនូវចម្លើយដែលត្រឹមត្រូវ និងមានបរិបទជាមួយសុវត្ថិភាពនិងការអាចពង្រីកបានសម្រាប់សហគ្រាស។

**សមត្ថភាព Enterprise RAG ដែលអ្នកនឹងបង្កើត៖**
- 📚 **ការយល់ដឹងអំពីឯកសារ**: ការកែប្រែឯកសារដែលមានកម្រិតខ្ពស់ជាមួយសេវាកម្ម Azure AI
- 🔍 **ការស្វែងរករូបមន្ត**: ការស្វែងរកវ៉ិចទ័រដែលមានប្រសិទ្ធភាពខ្ពស់ជាមួយលក្ខណៈសម្បត្តិសហគ្រាស
- 🛡️ **ការភ្ជាប់សុវត្ថិភាព**: រចនាសម្ព័ន្ធចូលប្រើផ្អែកលើតួនាទី និងលីវវិធីការពារទិន្នន័យ
- 🏢 **ស្ថាបត្យកម្មអាចពង្រីកបាន**: ប្រព័ន្ធ RAG សម្រាប់ផលិតកម្មដែលមានការត្រួតពិនិត្យ

## 🎯 ស្ថាបត្យកម្ម Enterprise RAG

### ធាតុសំខាន់សម្រាប់សហគ្រាស
- **Microsoft Foundry**: វេទិកា AI សម្រាប់សហគ្រាសដែលគ្រប់គ្រងជាមួយសុវត្ថិភាពនិងការទៀងទាត់
- **ភ្នាក់ងារជាប់តំណែង**: ភ្នាក់ងារដែលមានស្ថានភាពជាមួយបទបញ្ជាសន្ទនា និងការគ្រប់គ្រងបរិបទ
- **ការគ្រប់គ្រងហាងវ៉ិចទ័រ**: ការរក្សាទុកនិងស្វែងរកឯកសារដែលមានគុណភាពសម្រាប់សហគ្រាស
- **ការភ្ជាប់សម្គាល់អត្តសញ្ញាណ**: ការផ្ទៀងផ្ទាត់ Azure AD និងការគ្រប់គ្រងការចូលប្រើផ្អែកលើតួនាទី

### សក្ដានុពល .NET សម្រាប់សហគ្រាស
- **សុវត្ថិភាពប្រភេទទិន្នន័យ**: ការពិនិត្យតំរូវការពេលរបៀបកំណត់ RAM និងរចនាសម្ព័ន្ធទិន្នន័យ
- **កម្រិតសមត្ថភាព Async**: ការដំណើរការឯកសារនិងស្វែងរកដោយមិនបាច់ឈប់
- **ការគ្រប់គ្រងអង្គចងចាំ**: ការប្រើប្រាស់ធនធានយ៉ាងមានប្រសិទ្ធភាពសម្រាប់ជម្រើសឯកសារធំនៅច្រើន
- **លំដាប់ភ្ជាប់**: ការភ្ជាប់សេវាកម្ម Azure ដោយដើមជាមួយការវាយចាក់ពុម្ព

### ធាតុ .NET ជា​មូលដ្ឋាន
- **Azure.AI.Agents.Persistent**: ការគ្រប់គ្រងភ្នាក់ងារសហគ្រាសជាមួយស្ថានភាពដែលរក្សាទុក
- **Azure.Identity**: ការផ្ទៀងផ្ទាត់ភ្ជាប់សុវត្ថិភាពសម្រាប់ការចូលប្រើសេវាកម្ម Azure
- **Microsoft.Agents.AI.AzureAI**: ការអនុវត្តក្របខ័ណ្ឌភ្នាក់ងារ Azure ឆាប់រហ័ស
- **System.Linq.Async**: ប្រតិបត្តិ LINQ ដំណើរការលឿនមិនធ្វើឲ្យឈប់

## 🔧 លក្ខណៈពិសេស និងអត្ថប្រយោជន៍របស់សហគ្រាស

### សុវត្ថិភាព និងការទៀងទាត់
- **ការភ្ជាប់ Azure AD**: ការគ្រប់គ្រងអត្តសញ្ញាណនិងការផ្ទៀងផ្ទាត់សម្រាប់សហគ្រាស
- **ការចូលប្រើផ្អែកលើតួនាទី**: សិទិ្ធចូលប្រើច្បាស់លាស់សម្រាប់ឯកសារនិងប្រតិបត្តការណ៍
- **ការការពារទិន្នន័យ**: ការអ៊ិនគ្រីបនៅពេលស្តុក និងការបញ្ជូនទិន្នន័យ សម្រាប់ឯកសារមានសំខាន់
- **ការចុះកំណត់ហេតុត្រួតពិនិត្យ**: ការតាមដានសកម្មភាពពេញលេញសម្រាប់ការទៀងទាត់

### សមត្ថភាព និងការអាចពង្រីកបាន
- **ការប្រមូលការតភ្ជាប់**: ការគ្រប់គ្រងការតភ្ជាប់សេវាកម្ម Azure យ៉ាងមានប្រសិទ្ធភាព
- **ការដំណើរការ Async**: ប្រតិបត្តិការមិនបាច់ឈប់សម្រាប់សេណារីយ៏ចរន្តខ្ពស់
- **យុទ្ធសាស្ត្រចងក្រងCACHE**: ការចងក្រងCACHE យ៉ាងឆ្លាតវៃសម្រាប់ឯកសារដែលចូលប្រើជាញឹកញាប់
- **ការប្រើកម្រិតតុល្យភាពបន្ទាញ់**: ការដំណើរការចែកចាយសម្រាប់ការតំឡើងកម្រិតធំនៅរាយ

### ការគ្រប់គ្រងនិងត្រួតពិនិត្យ
- **ការត្រួតពិនិត្យសុខភាព**: ការត្រួតពិនិត្យដ៏ដំឡើងសម្រាប់ធាតុប្រព័ន្ធ RAG
- **តម្រង់ទិសសមត្ថភាព**: វិភាគលម្អិតអំពីគុណភាពស្វែងរក និងពេលឆ្លើយតប
- **ការគ្រប់គ្រងកំហុស**: ការគ្រប់គ្រងករណី Exception ដោយមានគោលនយោបាយ Retry
- **ការគ្រប់គ្រងរចនាសម្ព័ន្ធ**: ការកំណត់លំដាប់នៃបរិយាកាសដែលមានការត្រួតពិនិត្យ

## 📊 លំនាំសម្រាប់ Enterprise RAG

### លំនាំគ្រប់គ្រងឯកសារ
- **ការផ្ទុកដុំ**: ដំណើរការដ៏មានប្រសិទ្ធភាពសម្រាប់ឯកសារជាច្រើន
- **ការធ្វើបច្ចុប្បន្នភាពតាមជំហាន**: ការបន្ថែមនិងកែប្រែឯកសារពេលពិត
- **ការគ្រប់គ្រងកំណែ**: ការតាមដានកំណែក្នុងឯកសារ
- **ការគ្រប់គ្រងមេតាដាតា**: គុណលក្ខណៈនៃឯកសារដ៏សម្បូរបែបនិងការកំណត់ចំណាត់ថ្នាក់

### លំនាំស្វែងរក និងស្វែងយល់
- **ការស្វែងរកផ្សំ**: ភិចផ្សំស្វែងរករូបមន្ត និងពាក្យគន្លឹះសម្រាប់លទ្ធផលល្អសម្រាប់
- **ការស្វែងរកតាមមុខ**: ការត្រង់តាមវិមាត្រផ្សេងគ្នា និងការចាត់ថ្នាក់
- **ការកំណត់ទំរង់សមស្រប**: អាល់ហ្គរីធមគណនាផ្ដល់ពិន្ទុប្ដូរតាមតម្រូវការផ្នែកវិស័យ
- **ការវាយតម្លៃលទ្ធផល**: ការវាយតម្លៃខ្ពស់ជាមួយការភ្ជាប់លក្ខណៈហិរញ្ញវត្ថុ

### លំនាំសុវត្ថិភាព
- **សុវត្ថិភាពកម្រិតឯកសារ**: ការគ្រប់គ្រងសិទ្ធិចូលប្រើជាលំដាប់លម្អិតលើឯកសារ
- **ការបែងចែកទិន្នន័យ**: ការវាយតម្កល់សញ្ញាសំខាន់ដោយស្វ័យប្រវត្តិ និងការការពារ
- **ហត្ថការតាមដាន**: ការចុះកំណត់ហេតុដំណើរការត្រួតពិនិត្យគ្រប់លំដាប់
- **ការការពារឯកជនភាព**: ការស្គាល់ និងការធ្វើសម្អាត PII
