# ⚡ សេចក្ដីផែនការ​អ្នកបញ្ចូល​ពហុ​សកម្មភាព​ជាមួយ Azure OpenAI (Responses API) (.NET)

## 📋 មេរៀន​ដំណើរការសមស្របលឿនជាប្រព័ន្ធ​សមមូល

សៀវភៅកំណត់ត្រានេះបង្ហាញពី **លំនាំការងារសមស្រប** ដោយប្រើ Microsoft Agent Framework សម្រាប់ .NET និង Azure OpenAI (Responses API)។ អ្នក​នឹងរៀន​របៀប​បង្កើតដំណើរការលឿនជាប្រព័ន្ធ​សមមូល​ដែលមានសមត្ថភាពខ្ពស់​ដែលហែតការផលិតដោយការប្រតិបត្តិ​អ្នកជំនួយ AI ពហុជាមួយគ្នាក្នុងពេលតែមួយ ខណៈដែលរក្សាការសម្របសម្រួល និងភាពត្រឹមត្រូវនៃទិន្នន័យ។

## 🎯 គោលបំណងរៀន

### 🚀 **មូលដ្ឋាននៃការដំណើរការសមមូល**
- **ការប្រតិបត្តិនៃភ្នាក់ងារពហុជាមួយគ្នា**៖ ប្រតិបត្តិភ្នាក់ងារជំនួយ AI ច្រើនក្នុងពេលតែមួយដើម្បីសមត្ថភាពខ្ពស់បំផុត
- **លំនាំ Async/Await**៖ ប្រើម៉ូដែលកម្មវិធី async នៃ .NET សម្រាប់ការសមមូលមានប្រសិទ្ធភាព
- **Azure OpenAI (Responses API)**៖ សម្របសម្រួលការហៅពហុ API Responses របស់ Azure OpenAI ជាសមមូល
- **ការគ្រប់គ្រងធនធាន**៖ គ្រប់គ្រងធនធានម៉ូដែល AI ដោយប្រសិទ្ធភាពនៅក្នុងដំណើរការសមមូល

### 🏗️ **ស្ថាបត្យកម្មកម្រិតខ្ពស់នៃសមមូល**
- **Task-Based Parallelism**៖ ប្រើបណ្ណាល័យ Task Parallel Library នៃ .NET សម្រាប់ការប្រតិបត្តិពហុលឿនក្រៅសម័យភាពគ្នា
- **លំនាំសមមូលកំណត់កាលវិភាគ**៖ សម្របសម្រួលភ្នាក់ងារសមមូលដោយចៀសវាងស្ថានភាពប្រណាំង
- **ការបែងចែកបន្ទុក**៖ ចែកចាយការងារយ៉ាងមានប្រសិទ្ធភាពនៅលើសមត្ថភាពដំណើរការសមមូលដែលមាន
- **ការរំលែកកំហុស**៖ ប្រើវិធីសាស្រ្តដោះស្រាយកំហុសអ្នកជំនួយដោយមិនបញ្ឈប់ដំណើរការទាំងមូល

### 🏢 **កម្មវិធីសមមូលសម្រាប់ស្ថាប័នធំ**
- **ដំណើរការឯកសារច្រើនទំហំ**៖ ដំណើរការឯកសារច្រើនក្នុងពេលតែមួយ
- **វិភាគមាតិកាពេលវេលាត្រួតពិនិត្យ**៖ វិភាគសមមូលនៃទិន្នន័យបញ្ចូលបន្ទាត់
- **លំនាំបាល់រួមដំណើរការ**៖ បង្កើនទិន្នន័យធំដំណើរការឆាប់លឿន
- **វិភាគច្រើនទ្រង់ទ្រាយ**៖ ដំណើរការលេចត្រកូលនៃមាតិកាប្រភេទនិងទ្រង់ទ្រាយខុសគ្នា

## 💻 របៀបដំណើរការកូដ

ការអនុវត្តពេញលេញមាននៅក្នុង `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`។ ឯកសារនេះបង្ហាញពី **ដំណើរការសមមូល Fan-Out/Fan-In** សម្រាប់ផែនការជម្រកដំណើរ:

### 🏗️ **ស្ថាបត្យកម្មដំណើរការ**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**ធាតុសំខាន់៖**

1. **ConcurrentStartExecutor**៖ ផ្សាយសារសំណើរបស់អ្នកប្រើទៅកាន់ភ្នាក់ងារទាំងអស់ក្នុងពេលតែមួយ
2. **Researcher Agent**៖ វិភាគកន្លែងទេសចរណ៍និងទំនៀមទំលាប់ស្មើ
3. **Planner Agent**៖ បង្កើតផែនការធ្វើដំណើរយ៉ាងលម្អិតដោយសមមូល
4. **ConcurrentAggregationExecutor**៖ ប្រមូលនិងបញ្ចូលលទ្ធផលពីភ្នាក់ងារទាំងពីរ

### 🎯 **លំនាំ Fan-Out/Fan-In**

ដំណើរការនេះបង្ហាញលំនាំ **Fan-Out/Fan-In** ជាចម្បង:
- **Fan-Out**៖ សារបញ្ចូលមួយត្រូវបានផ្សាយទៅភ្នាក់ងារម៉ាស៊ីនច្រើនក្នុងពេលតែមួយ
- **ការដំណើរការសមមូល**៖ ភ្នាក់ងារច្រើនធ្វើការងារឡើងវិញដោយផ្ទាល់លើភារកិច្ចដូចគ្នា
- **Fan-In**៖ លទ្ធផលពីភ្នាក់ងារទាំងអស់ត្រូវបានប្រមូលនិងបញ្ចូលជា​ចេញ​តែមួយ

- **ពិនិត្យកូដ**៖ អ្នកពិនិត្យច្រើនវិភាគកូដពីចំណុចប្រសើរផ្សេងៗ

- **ដំណើរការឯកសារ**៖ ទាញយក, វិភាគ និងផ្ទៀងផ្ទាត់សមមូល

### 🔍 ការយល់ដឹងអំពី Custom Executors

**ConcurrentStartExecutor:**
- អនុវត្ត `IMessageHandler<string>` ដើម្បីទទួលយកបញ្ចូលជាសរសេរ
- ផ្សាយសារទៅភ្នាក់ងារគ្រប់គ្នាដែលបានភ្ជាប់
- ផ្ញើ `TurnToken` ដើម្បីបើកដំណើរការសមមូល

**ConcurrentAggregationExecutor:**
- អនុវត្ត `IMessageHandler<ChatMessage>` ដើម្បីទទួលសារ ភ្នាក់ងារ
- ប្រមូលសារដោយរបៀបដំបងមានសុវត្ថិភាព
- បញ្ចូលលទ្ធផលនៅពេលទទួលបានឆ្លើយ​តប​ទាំងអស់
- ផ្ដល់លទ្ធផលចុងក្រោយដោយប្រើ `context.YieldOutputAsync()`

### ⚡ អត្ថប្រយោជន៍​សមត្ថភាព

**សមមូល និង ជួរៗ:**
- ជួរៗ: Agent1 (30 វិនាទី) → Agent2 (30 វិនាទី) = **60 វិនាទីសរុប**
- សមមូល: Agent1 (30 វិនាទី) || Agent2 (30 វិនាទី) = **30 វិនាទីសរុប**

**ការ​បង្កើន throughput**៖ លឿន N ដងសម្រាប់ភ្នាក់ងារ N ចំណែក (ផ្អែកលើផ្ទុកការ និងធនធាន)
