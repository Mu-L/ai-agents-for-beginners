# 🔄 ការងារ​ Basic Agent Workflows ជាមួយ Azure OpenAI (Responses API) (.NET)

## 📋 មេរៀនរចនាបទអង្គុយកិច្ចការ Workflow Orchestration Tutorial

សៀវភៅកំណត់នេះបង្ហាញពីរបៀបដែលត្រូវបង្កើត**ការសរសេរប្រតិបត្តិការ agent ដែលមានភាពស្មុគស្មាញ**ដោយប្រើ Microsoft Agent Framework សម្រាប់ .NET និង Azure OpenAI (Responses API)។ អ្នកនឹងរៀនបង្កើតដំណើរការអាជីវកម្មជាច្រើនជំហានដែលអ្នកតំណាង AI សហការជាមួយគ្នាដើម្បីបំពេញភារកិច្ចស្មុគស្មាញតាមរយៈលំនុំសម្ព័ន្ធការគ្រប់គ្រងដែលមានរចនាសម្ព័ន្ធ។

## 🎯 គោលបំណងការរៀន

### 🏗️ **មូលដ្ឋានរចនាសម្ព័នរបស់ Workflow**
- **Workflow Builder**: រចនានិងគ្រប់គ្រងដំណើរការតាមជំហាន AI ដែលស្មុគស្មាញ
- **Agent Coordination**: សម្របសម្រួលអ្នកតំណាងជាច្រើនដែលមានជំនាញពិសេសនៅក្នុង workflow
- **Azure OpenAI (Responses API)**: ប្រើប្រាស់ Azure OpenAI Responses API ក្នុង workflow
- **Visual Workflow Design**: បង្កើតនិងមើលទំនាក់ទំនងរចនាសម្ព័ន្ធ Workflow សម្រាប់ការយល់ដឹងប្រសើរឡើង

### 🔄 **លំនុំសម្ព័ន្ធការគ្រប់គ្រងដំណើរការ Process Orchestration Patterns**
- **Sequential Processing**: ខ្សែភ្ជាប់ភារកិច្ចអ្នកតំណាងជាច្រើនក្នុងលំដាប់តុល្យ
- **State Management**: រក្សាទុកបរិបទនិងចរន្តទិន្នន័យឆ្លងកាត់ជំហាន workflow
- **Error Handling**: អនុវត្តការស្ដារឡើងវិញករណីមានកំហុសយ៉ាងរឹងមាំនិងភាពរឹងមាំនៃ workflow
- **Performance Optimization**: រចនារបៀបប្រតិបត្តិការសម្រួលសម្រាប់ប្រតិបត្តិការអង្គភាពធំៗ

### 🏢 **កម្មវិធី Workflow សម្រាប់សហគ្រាស Enterprise**
- **Business Process Automation**: អូតូម៉ាទិកដំណើរការអង្គភាពស្មុគស្មាញ
- **Content Production Pipeline**: ដំណើរការការងារកែសម្រួលជាមួយការត្រួតពិនិត្យនិងអនុម័ត
- **Customer Service Automation**: ដំណើរការដោះស្រាយសំណួរពហុជំហានរបស់អតិថិជន
- **Data Processing Workflows**: ដំណើរការ ETL ជាមួយការបម្លែងដែលជាគន្លងដោយ AI

### 🔑 **ការតំឡើង Azure OpenAI**

**ការតំឡើងបរិស្ថាន (.env file):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**ការចូលប្រើ Azure OpenAI:**
1. បង្កើតធនធាន Azure OpenAI នៅក្នុងផ្ទៃ Azure portal
2. តម្លើងម៉ូដែល (ឧទាហរណ៍ `gpt-4.1-mini`) ហើយចំណាំឈ្មោះ deployment
3. ចូលប្រើដោយ `az login` ហើយកំណត់អថេរបរិស្ថានដូចបានបង្ហាញខាងលើ

### 🏗️ **ទិដ្ឋភាពទូលំទូលាយរចនាសម្ព័ន្ធ Workflow**

```mermaid
graph TD
    A[អ្នកបង្កើតលំហូរត្រូវការ] --> B[បញ្ជីភ្នាក់ងារ]
    B --> C[ម៉ាស៊ីនអនុវត្តលំហូរត្រូវការ]
    C --> D[Agent 1: អ្នកបង្កើតមាតិកា]
    C --> E[Agent 2: អ្នកវិភាគមាតិកា] 
    D --> F[លទ្ធផលលំហូរត្រូវការ]
    E --> F
    G[Azure OpenAI (Responses API)] --> D
    G --> E
```

**ធាតុសំខាន់ៗ:**
- **WorkflowBuilder**: ម៉ាស៊ីនគ្រប់គ្រងសំខាន់សម្រាប់រចនាបទអង្គុយកិច្ចការ workflow
- **AIAgent**: អ្នកតំណាងជាច្រើនដែលមានជំនាញពិសេសនីមួយៗ
- **Azure OpenAI Client**: ដំណើរការ Azure OpenAI Responses API
- **Execution Context**: គ្រប់គ្រងសភាពនិងចរន្តទិន្នន័យកន្លងផុតជំហាន workflow

## 🎨 **លំនំមអង្គុយការងារ Workflow សម្រាប់សហគ្រាស Enterprise**

### 📝 **ដំណើរការផលិតមាតិកា Content Production Workflow**
```
User Request → Content Generation → Quality Review → Final Output
```

### 🔍 **ដំណើរការប្រមូលផ្ទុកឯកសារ Document Processing Pipeline**
```
Document Input → Analysis → Extraction → Validation → Structured Output
```

### 💼 **ដំណើរការព័ត៌មានអាជីវកម្ម Business Intelligence Workflow**
```
Data Collection → Processing → Analysis → Report Generation → Distribution
```

### 🤝 **ការអូតូមាទិកសេវាអតិថិជន Customer Service Automation**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **អត្ថប្រយោជន៍សម្រាប់សហគ្រាស Enterprise Benefits**

### 🎯 **ភាពរឹងមាំ និងការពង្រីក**
- **Deterministic Execution**: លទ្ធផល workflow ដែលមានភាពឯកជន និងអាចធ្វើរៀងរាល់ដង
- **Error Recovery**: ការកែសម្រួលកំហុសយ៉ាងមានគុណភាពនៅជំហាន workflow ណាមួយ
- **Performance Monitoring**: តាមដានគំនិតប្រតិបត្តិការនិងឱកាសបង្កើនប្រសិទ្ធភាព
- **Resource Management**: ការចែកចាយនិងប្រើប្រាស់ធនធានម៉ូដែល AI យ៉ាងមានប្រសិទ្ធភាព

### 🔒 **សុវត្ថិភាព និងការអនុវត្តតាមច្បាប់**
- **Secure Authentication**: ការផ្ទៀងផ្ទាត់ Microsoft Entra ID តាម `az login` (AzureCliCredential)
- **Audit Trails**: កំណត់ហេតុខលខលទ្ធផល workflow និងចំណុចសំខាន់ៗ
- **Access Control**: សិទ្ធិកំណត់ឯកជនសម្រាប់ការអនុវត្ត workflow និងត្រួតពិនិត្យ
- **Data Privacy**: ការថែរក្សាទិន្នន័យឯកជនតាម workflow ទាំងមូល

### 📊 **ការត្រួតពិនិត្យ និងការគ្រប់គ្រង Observability & Management**
- **Visual Workflow Design**: ការតំណាងច្បាស់លាស់នៃបរិវេណនិងភាពពាក់ព័ន្ធ
- **Execution Monitoring**: តាមដានកាន់តែពេលនេះនៃដំណើរការ workflow និងប្រសិទ្ធភាព
- **Error Reporting**: អនុភាពវិភាគកំហុសលម្អិតនិងសមត្ថភាពស្វែងរកកំហុស
- **Performance Analytics**: គំនិតសម្រាប់បង្កើនប្រសិទ្ធភាពនិងផែនការសមត្ថភាព

## 💻 ការប្រើប្រាស់កូដ

ការអនុវត្តពេញលេញមានក្នុង `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`។ ឯកសារនេះបង្ហាញ៖

1. **ការកំណត់បរិស្ថាន** - ការរត់កំណត់ Azure OpenAI ពី *.env* file
2. **ការកំណត់ Azure OpenAI Client** - កំណត់ client របស់ Azure OpenAI Responses API
3. **ការបង្កើត агент** - កំណត់អ្នកតំណាងពិសេស (Front Desk និង Concierge)
4. **Workflow Builder** - បង្កើត workflow មានអ្នកតំណាងជាច្រើន ជាមួយការចាប់ផ្តើមជាដំណាក់កាល
5. **ការប្រតិបត្តិ Workflow** - រត់ workflow ជាមួយលទ្ធផលផ្សាយបន្តផ្ទាល់

### 🚀 រត់ឧទាហរណ៍

```bash
# បង្កើតឲ្យស្គ្រីបមានសិទ្ធិអនុវត្ត (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# រត់កម្មវិធីធ្វើការ
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

រឺនៅលើ Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 លទ្ធផលដែលរំពឹងទុក

Workflow នឹង:
1. ទទួលសំណើដើម្បីធ្វើដំណើរទៅកាន់កន្លែង ("ខ្ញុំចង់ទៅប៉ារីស")
2. អ្នកតំណាង Front Desk ផ្តល់អនុសាសន៍ដំបូង
3. អ្នកតំណាង Concierge ពិនិត្យនិងកែលម្អអនុសាសន៍នេះ
4. លទ្ធផលចុងក្រោយបង្ហាញផ្ទាល់ជាស្ទ្រីមការសន្ទនា

### 🔧 ការប្តូរតាមបំណង

អ្នកអាចផ្លាស់ប្តូរផ្នែក workflow ដោយ:
- កែប្រែកំណត់របៀបដំណើរការរបស់អ្នកតំណាង
- បន្ថែមអ្នកតំណាងបន្ថែមសម្រាប់បង្កើត workflow ស្មុគស្មាញជាច្រើនជំហាន
- ប្ដូរប្រេកង់សារអ្នកប្រើដើម្បីសាកល្បងស្ថានការណ៍ខុសៗគ្នា
- កែសម្រួលកំណែនរបស់ workflow ដើម្បីបង្កើតលំនាំប្រតិបត្តិការខុសៗគ្នា
