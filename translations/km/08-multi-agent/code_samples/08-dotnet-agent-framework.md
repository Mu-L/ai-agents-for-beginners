# 🤝 ប្រព័ន្ធធ្វើការជាសហការណ៍ដោយភ្នាក់ងារច្រើនសម្រាប់អាជីវកម្ម (Enterprise Multi-Agent Workflow Systems) (.NET)

## 📋 គោលបំណងសិក្សា

សៀវភៅកំណត់ត្រានេះបង្ហាញពីរបៀបបង្កើតប្រព័ន្ធភ្នាក់ងារច្រើនដែលមានកម្រិតអាជីវកម្មកម្រិតខ្ពស់ដោយប្រើ Microsoft Agent Framework ក្នុង .NET ជាមួយ Azure OpenAI (Responses API)។ អ្នកនឹងរៀនពីរបៀបរៀបចំភ្នាក់ងារជាច្រើនដែលមានជំនាញជាក់លាក់សហការគ្នាតាមរបៀបរៀបចំកាន់តែរចនាសម្ព័ន្ធ ដោយបញ្ចេញអត្ថប្រយោជន៍ម៉ាស៊ីនអាជីវកម្មរបស់ .NET សម្រាប់ដំណោះស្រាយដែលប្រើបានក្នុងការផលិត។

**សមត្ថភាពភ្នាក់ងារច្រើនសម្រាប់អាជីវកម្មដែលអ្នកនឹងបង្កើត៖**
- 👥 **សហការភ្នាក់ងារ**: ការសម្របសម្រួលភ្នាក់ងារដោយប្រើប្រភេទខាងសុវត្ថិភាពជាមួយការត្រួតពិនិត្យពេលបង្ហាញកូដ
- 🔄 **ការរៀបចំប្រតិបត្តិការប្រតិបត្តិការ**: ការបញ្ចេញរូបរាងបញ្ជាក់ដំណើរការដោយប្រើ .NET async patterns
- 🎭 **ជំនាញតំណាងតួ**: បុគ្គលិកភាពនៃភ្នាក់ងារដែលបានវាយតម្លៃប្រភេទយ៉ាងខ្លាំង និងដែនជំនាញ
- 🏢 **ការរួមបញ្ចូលអាជីវកម្ម**: គំរូដែលរួចរាល់សម្រាប់ផលិតកម្មជាមួយការត្រួតពិនិត្យនិងការដោះស្រាយកំហុស

## ឧទាហរណ៍កូដ

កូដពេញលេញសម្រាប់មេរៀននេះអាចរកបាននៅក្នុងឯកសារ C# មួយដែលភ្ជាប់មកជាមួយ៖ [`08-dotnet-agent-framework.cs`](../../../../08-multi-agent/code_samples/08-dotnet-agent-framework.cs)

```bash
dotnet run 08-dotnet-agent-framework.cs
```

## តើឧទាហរណ៍នេះបង្ហាញអ្វីខ្លះ

ប្រព័ន្ធហ្វ្លូវើកភ្នាក់ងារច្រើននេះបង្កើតសេវាកម្មផ្ដល់អនុសាសន៍ដំណើរកម្សាន្តសណ្ឋាគារជាមួយភ្នាក់ងារពិសេសពីរនាក់៖

1. **ភ្នាក់ងារ FrontDesk**: ភ្នាក់ងារការធ្វើដំណើរដែលផ្ដល់អនុសាសន៍អំពីសកម្មភាពនិងទីតាំង
2. **ភ្នាក់ងារ Concierge**: ពិនិត្យអនុសាសន៍ដើម្បីធ្វើឱ្យប្រាកដថាបទពិសោធន៍គឺពិតប្រាកដ មិនមែនជាដំណើរកំសាន្តទេ

ភ្នាក់ងារទាំងពីរនាក់រួមគ្នាដំណើរការនៅក្នុងហ្វ្លូដែល៖
- ភ្នាក់ងារ FrontDesk ទទួលសំណើដំណើរកម្សាន្តដំបូង
- ភ្នាក់ងារ Concierge ពិចារណានិងធ្វើការកែប្រែអនុសាសន៍
- ហ្វ្លូផ្តល់ចម្លើយដោយផ្ទាល់ដល់អ្នកប្រើស្របពេលពិត

## គំនិតសំខាន់ៗ

### ការសម្របសម្រួលភ្នាក់ងារ
ឧទាហរណ៍បង្ហាញពីការសម្របសម្រួលភ្នាក់ងារដោយប្រើវេទិកា Microsoft Agent Framework ជាមួយការត្រួតពិនិត្យពេលបង្ហាញកូដ។

### ការរៀបចំហ្វ្លូប្រតិបត្ដិការ
ប្រើការបញ្ជាក់ហ្វ្លូដោយប្រើ .NET async patterns ដើម្បីភ្ជាប់ភ្នាក់ងារជាច្រើនក្នុងបន្ទាត់ផ្សាយ

### ការផ្ទេរចម្លើយបែបស្ទ្រីម
អនុវត្តន៍ការផ្ទេរចម្លើយភ្នាក់ងារដោយផ្ទាល់ពេលវេលាជាមួយ async enumerables និងរចនាសម្ព័ន្ធស្រោចស្រង់ព្រឹត្តិការណ៍

### ការរួមបញ្ចូលអាជីវកម្ម
បង្ហាញគំរូដែលរួចរាល់សម្រាប់ផលិតកម្មរួមមាន៖
- ការកំណត់បរិស្ថានប៉ារ៉ាម៉ែត្រ
- ការគ្រប់គ្រងសក្ដានុពលយ៉ាងសុវត្ថិភាព
- ការជួសជុលកំហុស
- ការប្រតិបត្តិព្រឹត្តិការណ៍អាសិច្រឹម
