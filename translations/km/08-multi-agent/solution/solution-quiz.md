# ចម្លើយសម្រាប់ការត្រួតពិនិត្យចំណេះដឹង

## សំនួរ​ទី 1

**ចម្លើយ៖ A2.** អេនជិនជាច្រើនមានប្រយោជន៍ នៅពេលមានតួនាទីជាពិសេសផ្សេងៗគ្នា ឧបករណ៍ ឬអាជ្ញាបណ្ណបំបែកគ្នា និងការផ្ទេរដែលច្បាស់លាស់ដែលត្រូវការការសម្របសម្រួល។ ប៊ូតមួយសម្រាប់គាំទ្រសាមញ្ញប្រៀបធៀបហើយក៏អាចនៅតែម្នាក់ឯងបាន។ បញ្ហានៃការស្នើសុំដូចគ្នាច្រើន និយាយពីបញ្ហាការពង្រីក និងមិនត្រូវការរចនាឡើងជាអេនជិនជាច្រើនឡើយ។

## សំនួរ​ទី 2

**ចម្លើយ៖ A1.** អេនជិនតែម្នាក់ធម្មតានឹងងាយស្រួលសម្រាប់សាកល្បង សង្កេត និងដំណើរការ នៅពេលមានសំណុំបទបញ្ជា និងឧបករណ៍មួយដែលអាចដោះស្រាយបញ្ហានោះ។ ការចូលដំណើរការទៅកាន់ឧបករណ៍ជាច្រើនមិនបញ្ជាក់ថាត្រូវការ​អេនជិនជាច្រើនទេ។ តួនាទីបំបែកជាមួយអាជ្ញាបណ្ណផ្សេងៗ និងតម្រូវការត្រួតពិនិត្យអាចបញ្ជាក់ពីការរចនាឡើងជាអេនជិនជាច្រើន។
