# 🔄 Fluxos de Trabalho Básicos de Agentes com Azure OpenAI (API de Respostas) (.NET)

## 📋 Tutorial de Orquestração de Fluxos de Trabalho

Este notebook demonstra como construir fluxos de trabalho sofisticados de **agentes** usando o Microsoft Agent Framework para .NET e Azure OpenAI (API de Respostas). Aprenderá a criar processos empresariais multi-etapas onde agentes de IA colaboram para realizar tarefas complexas através de padrões estruturados de orquestração.

## 🎯 Objetivos de Aprendizagem

### 🏗️ **Fundamentos da Arquitectura de Fluxos de Trabalho**
- **Construtor de Fluxos de Trabalho**: Projetar e orquestrar processos complexos de IA multi-etapas
- **Coordenação de Agentes**: Coordenar vários agentes especializados dentro dos fluxos de trabalho
- **Azure OpenAI (API de Respostas)**: Aproveitar a API de Respostas do Azure OpenAI nos fluxos de trabalho
- **Design Visual de Fluxos de Trabalho**: Criar e visualizar estruturas de fluxo para melhor compreensão

### 🔄 **Padrões de Orquestração de Processos**
- **Processamento Sequencial**: Encadear múltiplas tarefas de agentes em ordem lógica
- **Gestão de Estado**: Manter contexto e fluxo de dados ao longo das etapas do fluxo de trabalho
- **Gestão de Erros**: Implementar recuperação robusta de erros e resiliência no fluxo
- **Otimização de Performance**: Projetar fluxos eficientes para operações em escala empresarial

### 🏢 **Aplicações Empresariais de Fluxos de Trabalho**
- **Automação de Processos Empresariais**: Automatizar fluxos de trabalho organizacionais complexos
- **Pipeline de Produção de Conteúdo**: Fluxos editoriais com etapas de revisão e aprovação
- **Automação de Serviço ao Cliente**: Resolução de consultas de clientes em múltiplas etapas
- **Fluxos de Trabalho de Processamento de Dados**: Fluxos ETL com transformação assistida por IA

```xml
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Configuração do Azure OpenAI**

**Configuração do Ambiente (ficheiro .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Acesso ao Azure OpenAI:**
1. Crie um recurso Azure OpenAI no portal Azure
2. Implemente um modelo (por exemplo, `gpt-5-mini`) e anote o nome da implementação
3. Inicie sessão com `az login` e configure as variáveis de ambiente conforme mostrado acima

**Componentes Principais:**
- **WorkflowBuilder**: Motor principal de orquestração para desenhar fluxos de trabalho
- **AIAgent**: Agentes especializados individuais com capacidades específicas
- **Cliente Azure OpenAI**: Integração da API de Respostas do Azure OpenAI
- **Contexto de Execução**: Gere estado e fluxo de dados entre etapas do fluxo de trabalho

## 💻 A Executar o Código

A implementação completa está disponível em `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Este ficheiro demonstra:

1. **Configuração do Ambiente** - Carregar a configuração Azure OpenAI a partir do ficheiro `.env`
2. **Configuração do Cliente Azure OpenAI** - Configurar o cliente para usar a API de Respostas Azure OpenAI
3. **Criação de Agentes** - Definir agentes especializados (Receção e Concierge)
4. **Construtor de Fluxos de Trabalho** - Criar um fluxo multi-agentes com processamento sequencial
5. **Execução do Fluxo** - Executar o fluxo com resultados em streaming

### 🚀 Executar o Exemplo

```bash
# Torne o script executável (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Execute o fluxo de trabalho
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Ou no Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Saída Esperada

O fluxo irá:
1. Aceitar o seu pedido de destino de viagem ("Gostaria de ir a Paris")
2. O agente da Receção fornece uma recomendação inicial
3. O agente Concierge revê e aprimora a recomendação
4. A saída final exibe a conversa completa em fluxo

### 🔧 Personalização

Pode personalizar o fluxo de trabalho ao:
- Modificar as instruções dos agentes para alterar o comportamento deles
- Adicionar mais agentes para criar fluxos multi-etapas complexos
- Mudar a mensagem do utilizador para testar diferentes cenários
- Ajustar as ligações do fluxo para criar diferentes padrões de execução
