# 🔄 Fluxos de Trabalho Básicos de Agentes com Azure OpenAI (API de Respostas) (.NET)

## 📋 Tutorial de Orquestração de Fluxos de Trabalho

Este bloco de notas demonstra como construir **fluxos de trabalho sofisticados de agentes** usando o Microsoft Agent Framework para .NET e Azure OpenAI (API de Respostas). Aprenderá a criar processos empresariais multi-etapas onde agentes de IA colaboram para realizar tarefas complexas através de padrões de orquestração estruturados.

## 🎯 Objetivos de Aprendizagem

### 🏗️ **Fundamentos da Arquitetura de Fluxos de Trabalho**
- **Construtor de Fluxos de Trabalho**: Projetar e orquestrar processos complexos multi-etapas de IA
- **Coordenação de Agentes**: Coordenar múltiplos agentes especializados dentro dos fluxos de trabalho
- **Azure OpenAI (API de Respostas)**: Aproveitar a Azure OpenAI Responses API nos fluxos de trabalho
- **Design Visual de Fluxos de Trabalho**: Criar e visualizar estruturas de fluxos de trabalho para melhor compreensão

### 🔄 **Padrões de Orquestração de Processos**
- **Processamento Sequencial**: Encadear múltiplas tarefas de agentes em ordem lógica
- **Gestão de Estado**: Manter o contexto e o fluxo de dados entre as etapas do fluxo de trabalho
- **Gestão de Erros**: Implementar recuperação robusta de erros e resiliência no fluxo de trabalho
- **Otimização de Performance**: Projetar fluxos de trabalho eficientes para operações em escala empresarial

### 🏢 **Aplicações Empresariais de Fluxos de Trabalho**
- **Automação de Processos de Negócio**: Automatizar fluxos de trabalho organizacionais complexos
- **Pipeline de Produção de Conteúdo**: Fluxos editoriais com etapas de revisão e aprovação
- **Automação de Atendimento ao Cliente**: Resolução de consultas de clientes multi-etapas
- **Fluxos de Trabalho de Processamento de Dados**: Fluxos ETL com transformação potenciada por IA

## ⚙️ Pré-requisitos & Configuração

### 📦 **Pacotes NuGet Requeridos**

Esta demonstração de fluxo de trabalho utiliza diversos pacotes-chave .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Configuração do Azure OpenAI**

**Configuração do Ambiente (ficheiro .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Acesso ao Azure OpenAI:**
1. Crie um recurso Azure OpenAI no portal Azure
2. Faça o deploy de um modelo (por exemplo, `gpt-4.1-mini`) e anote o nome do deployment
3. Inicie sessão com `az login` e configure as variáveis de ambiente conforme mostrado acima

**Componentes Principais:**
- **WorkflowBuilder**: Motor principal de orquestração para projetar fluxos de trabalho
- **AIAgent**: Agentes especializados individuais com capacidades específicas
- **Cliente Azure OpenAI**: Integração com a Azure OpenAI Responses API
- **Contexto de Execução**: Gere o estado e o fluxo de dados entre as etapas do fluxo de trabalho

## 💻 Executar o Código

A implementação completa está disponível em `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Este ficheiro demonstra:

1. **Configuração do Ambiente** - Carregamento da configuração Azure OpenAI a partir do ficheiro `.env`
2. **Configuração do Cliente Azure OpenAI** - Configurar o cliente para usar a Azure OpenAI Responses API
3. **Criação de Agentes** - Definição de agentes especializados (Front Desk e Concierge)
4. **Construtor de Fluxos de Trabalho** - Criação de um fluxo multi-agente com processamento sequencial
5. **Execução do Fluxo de Trabalho** - Execução do fluxo de trabalho com resultados em streaming

### 🚀 Executar o Exemplo

```bash
# Torne o script executável (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Execute o fluxo de trabalho
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Ou no Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Saída Esperada

O fluxo de trabalho irá:
1. Aceitar o seu pedido de destino de viagem ("Gostaria de ir a Paris")
2. O agente Front Desk fornece uma recomendação inicial
3. O agente Concierge revê e melhora a recomendação
4. A saída final mostra a conversa completa em streaming

### 🔧 Personalização

Pode personalizar o fluxo de trabalho:
- Modificando as instruções dos agentes para alterar o seu comportamento
- Adicionando mais agentes para criar fluxos de trabalho multi-etapas complexos
- Alterando a mensagem do utilizador para testar diferentes cenários
- Ajustando as ligações do fluxo de trabalho para criar diferentes padrões de execução
