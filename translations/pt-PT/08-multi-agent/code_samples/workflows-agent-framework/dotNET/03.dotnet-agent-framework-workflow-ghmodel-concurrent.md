# ⚡ Fluxos de Trabalho Concorrentes de Agentes com Azure OpenAI (API de Respostas) (.NET)

## 📋 Tutorial de Processamento Paralelo de Alto Desempenho

## 💻 Executar o Código

A implementação completa está disponível em `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Este ficheiro demonstra um **fluxo de trabalho concorrente Fan-Out/Fan-In** para planeamento de viagens:

### 🏗️ **Arquitetura do Fluxo de Trabalho**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Componentes-Chave:**

1. **ConcurrentStartExecutor**: Transmite o pedido do utilizador a todos os agentes simultaneamente
2. **Agente Investigador**: Analisa destinos e atrações em paralelo
3. **Agente Planeador**: Cria planos de viagem detalhados em paralelo
4. **ConcurrentAggregationExecutor**: Recolhe e funde resultados de ambos os agentes

### 🎯 **Padrão Fan-Out/Fan-In**

Este fluxo de trabalho demonstra o clássico padrão **Fan-Out/Fan-In**:
- **Fan-Out**: Uma mensagem de entrada é transmitida para múltiplos agentes simultaneamente
- **Processamento Concorrente**: Múltiplos agentes trabalham em paralelo na mesma tarefa
- **Fan-In**: Resultados de todos os agentes são recolhidos e agregados numa saída única

### 📝 Saída Esperada

O fluxo de trabalho irá:
1. **Transmitir Pedido**: Enviar "Planeia uma viagem para Seattle em Dezembro" a ambos os agentes
2. **Processamento Concorrente**: Ambos os agentes trabalham simultaneamente:

### 🔍 Entendendo Executores Customizados

**ConcurrentStartExecutor:**
- Implementa `IMessageHandler<string>` para aceitar entrada em string
- Transmite mensagens para todos os agentes conectados
- Envia `TurnToken` para disparar o processamento concorrente

**ConcurrentAggregationExecutor:**
- Implementa `IMessageHandler<ChatMessage>` para receber respostas dos agentes
- Recolhe mensagens de forma thread-safe
- Agrega quando todas as respostas esperadas chegam
- Produz saída final usando `context.YieldOutputAsync()`

### ⚡ Benefícios de Desempenho

**Concorrente vs Sequencial:**
- Sequencial: Agente1 (30s) → Agente2 (30s) = **60 segundos no total**
- Concorrente: Agente1 (30s) || Agente2 (30s) = **30 segundos no total**

**Melhoria do débito**: Até N× mais rápido para N agentes concorrentes (dependendo da carga de trabalho e recursos)

### 🛡️ Gestão de Erros

O fluxo de trabalho lida graciosamente com falhas individuais de agentes:
- Se um agente falhar, os outros continuam a processar
- O agregador pode implementar lógica de timeout
- Resultados parciais podem ser retornados se necessário
