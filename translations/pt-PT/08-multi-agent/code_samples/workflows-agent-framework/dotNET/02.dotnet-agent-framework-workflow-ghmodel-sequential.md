# ⏩ Workflows de Agentes Sequenciais com Azure OpenAI (API de Respostas) (.NET)

## 📋 Tutorial Avançado de Processamento Sequencial

Este notebook demonstra **padrões de workflow sequenciais** utilizando o Microsoft Agent Framework para .NET e Azure OpenAI (API de Respostas). Aprenderá a construir pipelines de processamento sofisticados, passo a passo, onde os agentes executam numa ordem específica, com cada etapa a basear-se nos resultados da etapa anterior.

## 🎯 Objetivos de Aprendizagem

### 🔄 **Arquitetura de Processamento Sequencial**
- **Design Linear do Workflow**: Criar pipelines de processamento passo a passo com dependências claras
- **Gestão de Estado**: Manter contexto e fluxo de dados ao longo das etapas do workflow sequencial
- **Azure OpenAI (API de Respostas)**: Aproveitar modelos Azure OpenAI em workflows .NET multi-etapa
- **Padrões Empresariais de Pipeline**: Construir sistemas de processamento sequencial prontos para produção

### 🏗️ **Padrões Avançados Sequenciais**
- **Processamento em Portões de Etapa**: Implementar pontos de validação entre etapas do workflow
- **Preservação do Contexto**: Manter estado e conhecimento acumulado ao longo de todas as etapas
- **Propagação de Erros**: Gerir falhas de forma elegante nas cadeias de processamento sequencial
- **Otimização de Performance**: Execução sequencial eficiente com overhead mínimo

### 🏢 **Aplicações Empresariais Sequenciais**
- **Pipeline de Processamento de Documentos**: Análise, transformação e validação de documentos em várias etapas

### 🏗️ **Arquitetura do Workflow Sequencial**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Agente de Análise]
    B --> C[Ponto de Verificação 1]
    C --> D[Stage 2: Agente de Processamento]
    D --> E[Ponto de Verificação 2]
    E --> F[Stage 3: Agente de Validação]
```

**Componentes chave:**
- **Agentes Sequenciais**: Agentes especializados para cada etapa de processamento
- **Contexto de Estado**: Mantém dados acumulados e decisões ao longo das etapas
- **Pontos de Verificação**: Pontos de validação entre etapas para assegurar qualidade e consistência
- **Cliente Azure OpenAI**: Acesso consistente ao modelo de IA em todas as etapas do workflow

## 💻 Execução do Código

A implementação completa está disponível em `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Este ficheiro demonstra um **workflow de análise de móveis em três etapas**:

1. **Etapa 1 - Agente de Vendas**: Analisa imagens de móveis e fornece sugestões de compra
2. **Etapa 2 - Agente de Preços**: Fornece detalhamento de preços e opções de orçamento
3. **Etapa 3 - Agente de Orçamento**: Gera um documento de orçamento profissional em formato Markdown

### 🏗️ **Arquitetura do Workflow**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Cada agente:
- Recebe a saída da etapa anterior como contexto
- Baseia-se na análise anterior com expertise especializada
- Mantém a continuidade do workflow através da gestão de estado

### 📝 Saída Esperada

O workflow irá:

### 🔍 Compreender o Fluxo de Estado

Cada agente na sequência recebe:
- **Entrada Original**: A mensagem inicial do utilizador (imagem + texto)
- **Saídas dos Agentes Anteriores**: Todas as respostas anteriores dos agentes no histórico da conversa
- **Contexto Acumulado**: Estado completo mantido ao longo do workflow

Isto permite um processamento sofisticado em múltiplas etapas onde cada agente se baseia no contexto abrangente de todas as etapas anteriores.
