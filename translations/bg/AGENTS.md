# AGENTS.md

## Преглед на проекта

Това хранилище съдържа "AI Агенти за начинаещи" - цялостен образователен курс, който преподава всичко необходимо за изграждането на AI агенти. Курсът се състои от 18 урока (номерирани 00-18), обхващащи основи, проектни модели, рамки, внедряване в продукция, локални/на устройството агенти и сигурността на AI агентите.

**Ключови технологии:**
- Python 3.12+
- Jupyter тетрадки за интерактивно учене
- AI рамки: Microsoft Agent Framework (MAF)
- Azure AI услуги: Microsoft Foundry, Microsoft Foundry Agent Service V2

**Архитектура:**
- Структура на базата на уроци (директории 00-15+)
- Всеки урок съдържа: README документация, примерен код (Jupyter тетрадки) и изображения
- Поддръжка на множество езици чрез автоматизирана система за превод
- По една Python тетрадка на урок, използваща Microsoft Agent Framework

## Команди за настройка

### Предварителни изисквания
- Python 3.12 или по-висока версия
- Абонамент за Azure (за Microsoft Foundry)
- Инсталиран и удостоверен Azure CLI (`az login`)

### Първоначална настройка

1. **Клонирайте или форкнете хранилището:**
   ```bash
   gh repo fork microsoft/ai-agents-for-beginners --clone
   # ИЛИ
   git clone https://github.com/microsoft/ai-agents-for-beginners.git
   cd ai-agents-for-beginners
   ```

2. **Създайте и активирайте Python виртуална среда:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # В Windows: venv\Scripts\activate
   ```

3. **Инсталирайте зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Настройване на променливи на средата:**
   ```bash
   cp .env.example .env
   # Редактирайте .env с вашите API ключове и крайни точки
   ```

### Задължителни променливи на средата

За **Microsoft Foundry** (задължително):
- `AZURE_AI_PROJECT_ENDPOINT` - крайна точка на Microsoft Foundry проекта
- `AZURE_AI_MODEL_DEPLOYMENT_NAME` - име на разгъната модел (напр. gpt-4.1-mini)

За **Azure AI Search** (Урок 05 - RAG):
- `AZURE_SEARCH_SERVICE_ENDPOINT` - крайна точка на Azure AI Search
- `AZURE_SEARCH_API_KEY` - API ключ за Azure AI Search

Удостоверяване: Изпълнете `az login` преди пускане на тетрадките (ползва `AzureCliCredential`).

## Работен процес при разработка

### Работа с Jupyter тетрадки

Всеки урок съдържа няколко Jupyter тетрадки за различни рамки:

1. **Стартирайте Jupyter:**
   ```bash
   jupyter notebook
   ```

2. **Отидете в директорията на урока** (напр. `01-intro-to-ai-agents/code_samples/`)

3. **Отворете и изпълнете тетрадките:**
   - `*-python-agent-framework.ipynb` - Използване на Microsoft Agent Framework (Python)
   - `*-dotnet-agent-framework.ipynb` - Използване на Microsoft Agent Framework (.NET)

### Работа с Microsoft Agent Framework

**Microsoft Agent Framework + Microsoft Foundry:**
- Изисква абонамент за Azure
- Използва `FoundryChatClient` за Agent Service V2 (агентите се виждат в портала на Foundry)
- Готов за продукция с вградена възможност за наблюдение
- Файловият модел: `*-python-agent-framework.ipynb`

### Стартиране на отделни тетрадки

Отворете тетрадки в Jupyter и изпълнявайте клетките по ред. Всяка тетрадка е самостоятелна и включва:
- Импортни изрази
- Зареждане на конфигурация
- Примери за имплементации на агенти
- Очаквани резултати в markdown клетки

### Основно тестване на внедрени агенти

За уроци, където агентът е внедрен като хостван агент на Microsoft Foundry (01, 04, 05, 16), в хранилището има каталози за основно тестване под `tests/`, които се изпълняват чрез workflow `.github/workflows/smoke-test.yml` с action-а [AI Smoke Test](https://github.com/marketplace/actions/ai-smoke-test). Това е леко пост-внедряване, което проверява дали агентът е достъпен и следва базисни очаквания за подсказки, допълвайки оценъчния процес в уроци 10 и 16. Вижте [tests/README.md](./tests/README.md) за съпоставката каталог-урок-агент. Урок 17 се изпълнява локално с Foundry Local и няма хоствана крайна точка, затова се валидира чрез директно пускане на тетрадката.

## Специфичен за проекта контекст

### Поддръжка на множество езици

Това хранилище използва автоматизирана система за превод:
- Поддържат се над 50 езика
- Преводите са в директории `/translations/<lang-code>/`
- Workflow на GitHub Actions управлява актуализации на преводите
- Изходните файлове са на английски в корена на хранилището

### Структура на урока

Всеки урок следва последователен модел:
1. Видеоиконка с линк
2. Писмено съдържание на урока (README.md)
3. Примерен код в няколко рамки
4. Цели на обучението и предварителни изисквания
5. Допълнителни учебни ресурси с линкове

### Именуване на примерен код

Формат: `<номер-на-урока>-python-agent-framework.ipynb`
- `01-python-agent-framework.ipynb` - Урок 1, MAF Python
- `14-sequential.ipynb` - Урок 14, разширени модели в MAF
- `16-python-agent-framework.ipynb` - Урок 16, продукционен агент за клиентска поддръжка
- `17-local-agent-foundry-local.ipynb` - Урок 17, локален агент с Foundry Local + Qwen

### Специални директории

- `translated_images/` - Локализирани изображения за преводи
- `images/` - Оригинални изображения за съдържание на английски
- `.devcontainer/` - Конфигурация на VS Code development container
- `.github/` - Workflow-и и шаблони за GitHub Actions

### Зависимости

Основни пакети от `requirements.txt`:
- `agent-framework` - Microsoft Agent Framework
- `a2a-sdk` - Поддръжка на протокола Agent-to-Agent
- `azure-ai-inference`, `azure-ai-projects` - Azure AI услуги
- `azure-identity` - Azure удостоверяване (AzureCliCredential)
- `azure-search-documents` - Интеграция с Azure AI Search
- `mcp[cli]` - Поддръжка на Model Context Protocol
