# 🤝 Бизнес системи с многоагентни работни потоци (.NET)

## 📋 Учебни цели

Този бележник демонстрира как да се изградят сложни бизнес системи с много агенти, използвайки Microsoft Agent Framework в .NET с Azure OpenAI (Responses API). Ще научите как да оркестрирате множество специализирани агенти, работещи заедно чрез структурирани работни процеси, използвайки възможностите на .NET за предприятия за решения готови за производство.

**Възможности за многоагентни системи, които ще изградите:**
- 👥 **Сътрудничество между агенти**: Координация между агенти със сигурност на типовете и проверка при компилация
- 🔄 **Оркестрация на работни потоци**: Декларативно дефиниране на работни потоци с async шаблони на .NET
- 🎭 **Специализация на роли**: Силно типизирани личности и области на експертиза на агентите
- 🏢 **Интеграция в предприятия**: Шаблони готови за производство с наблюдение и обработка на грешки

## Какво демонстрира този пример

Тази система за многоагентни работни потоци създава препоръчителна услуга за хотелски пътувания с двама специализирани агенти:

1. **Агент на рецепцията**: Агент по пътувания, който дава препоръки за дейности и локации
2. **Консиерж агент**: Преглежда препоръките, за да осигури автентични, нетуристически преживявания

Агенти работят заедно в работен процес, при който:
- Агентът на рецепцията приема първоначалната молба за пътуване
- Консиерж агентът преглежда и усъвършенства препоръката
- Работният процес предава отговори в реално време

## Ключови концепции

### Координация между агенти
Примерът демонстрира координация с типова сигурност на агентите, използвайки Microsoft Agent Framework с проверка при компилация.

### Оркестрация на работни потоци
Използва декларативно дефиниране на работни потоци с async шаблони на .NET за свързване на множество агенти в конвейер.

### Поточно предаване на отговори
Реализира предаване в реално време на отговори на агентите чрез async enumerable и събитийно-ориентирана архитектура.

### Интеграция в предприятия
Показва шаблони готови за производство, включително:
- Конфигурация чрез променливи на средата
- Сигурно управление на креденциали
- Обработка на грешки
- Асинхронна обработка на събития
