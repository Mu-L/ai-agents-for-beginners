# 🔄 Основни работни потоци на агенти с Azure OpenAI (Responses API) (.NET)

## 📋 Урок за оркестрация на работни потоци

Този бележник демонстрира как да изградите сложни **работни потоци на агенти** с помощта на Microsoft Agent Framework за .NET и Azure OpenAI (Responses API). Ще научите как да създавате многостъпкови бизнес процеси, при които AI агентите си сътрудничат за изпълнение на сложни задачи чрез структурирани модели на оркестрация.

## 🎯 Учебни цели

### 🏗️ **Основи на архитектурата на работните потоци**
- **Workflow Builder**: Проектиране и оркестрация на сложни многократни AI процеси
- **Координация на агентите**: Координиране на множество специализирани агенти в работните потоци
- **Azure OpenAI (Responses API)**: Използване на Azure OpenAI Responses API в работни потоци
- **Визуален дизайн на работни потоци**: Създаване и визуализация на структури на работни потоци за по-добро разбиране

### 🔄 **Модели за оркестрация на процеси**
- **Последователна обработка**: Връзване на множество задачи на агенти в логичен ред
- **Управление на състоянието**: Поддържане на контекст и поток от данни през етапите на работния поток
- **Обработка на грешки**: Изграждане на устойчивост и възстановяване при грешки в работния поток
- **Оптимизация на производителността**: Проектиране на ефективни работни потоци за операции в мащаба на предприятия

### 🏢 **Приложения за работни потоци в предприятия**
- **Автоматизация на бизнес процеси**: Автоматизиране на сложни организационни работни потоци
- **Производствена линия за съдържание**: Редакторски работни потоци с етапи за преглед и одобрение
- **Автоматизация на обслужване на клиенти**: Многостъпково разрешаване на запитвания от клиенти
- **Работни потоци за обработка на данни**: ETL потоци с AI-задвижвана трансформация

## ⚙️ Предварителни условия и настройка

### 📦 **Необходими NuGet пакети**

Тази демонстрация на работни потоци използва няколко ключови .NET пакети:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Конфигурация на Azure OpenAI**

**Настройка на средата (.env файл):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Достъп до Azure OpenAI:**
1. Създайте ресурс Azure OpenAI в портала Azure
2. Разположете модел (например `gpt-4.1-mini`) и запишете името на разполагането
3. Влезте с `az login` и конфигурирайте променливите на средата както е показано по-горе

### 🏗️ **Преглед на архитектурата на работния поток**

```mermaid
graph TD
    A[Създател на работни потоци] --> B[Регистър на агенти]
    B --> C[Мотор за изпълнение на работни потоци]
    C --> D[Agent 1: Генератор на съдържание]
    C --> E[Agent 2: Преглед на съдържанието] 
    D --> F[Резултати от работните потоци]
    E --> F
    G[Azure OpenAI (Responses API)] --> D
    G --> E
```

**Ключови компоненти:**
- **WorkflowBuilder**: Основен двигател за оркестрация при проектиране на работни потоци
- **AIAgent**: Отделни специализирани агенти със специфични способности
- **Azure OpenAI Client**: Интеграция с Azure OpenAI Responses API
- **Execution Context**: Управлява състоянието и потока от данни между етапите на работния поток

## 💻 Стартиране на кода

Пълната реализация е налична в `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Този файл показва:

1. **Конфигурация на средата** - Зареждане на конфигурацията за Azure OpenAI от `.env` файл
2. **Настройка на Azure OpenAI клиента** - Конфигуриране на клиента за използване на Azure OpenAI Responses API
3. **Създаване на агенти** - Дефиниране на специализирани агенти (Reception и Concierge)
4. **Workflow Builder** - Създаване на многоагентски работен поток с последователна обработка
5. **Изпълнение на работния поток** - Стартиране на работния поток с резултати на потоци

### 🚀 Стартиране на примера

```bash
# Направете скрипта изпълним (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Стартирайте работния процес
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Или на Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Очакван изход

Работният поток ще:
1. Приеме вашето запитване за дестинация за пътуване („Бих искал да отида в Париж“)
2. Агентът Reception предоставя първоначална препоръка
3. Агентът Concierge преглежда и усъвършенства препоръката
4. Финалният изход показва целия поток на разговора
