# 🔄 Основни работни потоци с агенти с Azure OpenAI (Responses API) (.NET)

## 📋 Урок по оркестрация на работни потоци

Тази тетрадка демонстрира как да изградите сложни **работни потоци с агенти** с помощта на Microsoft Agent Framework за .NET и Azure OpenAI (Responses API). Ще научите как да създавате многоетапни бизнес процеси, където AI агентите си сътрудничат за изпълнение на сложни задачи чрез структурирани оркестрационни модели.

## 🎯 Цели на обучението

### 🏗️ **Основи на архитектурата на работни потоци**
- **Workflow Builder**: Проектиране и оркестрация на сложни многоетапни AI процеси
- **Agent Coordination**: Координиране на множество специализирани агенти в работни потоци
- **Azure OpenAI (Responses API)**: Използване на Azure OpenAI Responses API в работни потоци
- **Visual Workflow Design**: Създаване и визуализиране на структурата на работни потоци за по-добро разбиране

### 🔄 **Модели за оркестрация на процеси**
- **Последователна обработка**: Свързване на множество задачи на агентите в логичен ред
- **Управление на състоянието**: Поддържане на контекста и потока на данни между етапите на работния поток
- **Обработка на грешки**: Имплементиране на здрава възстановяемост и устойчивост на работния поток при грешки
- **Оптимизация на производителността**: Проектиране на ефективни работни потоци за корпоративни операции в голям мащаб

### 🏢 **Корпоративни приложения на работни потоци**
- **Автоматизация на бизнес процеси**: Автоматизиране на сложни организационни работни потоци
- **Производствена линия за съдържание**: Редакционни работни потоци с етапи на преглед и одобрение
- **Автоматизация на обслужване на клиенти**: Многоетапно разрешаване на клиентски запитвания
- **Работни потоци за обработка на данни**: ETL работни потоци с AI-задвижвани трансформации

## ⚙️ Предварителни изисквания и настройка

### 📦 **Необходими NuGet пакети**

Тази демонстрация на работен поток използва няколко ключови .NET пакета:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Конфигурация на Azure OpenAI**

**Настройка на средата (.env файл):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Достъп до Azure OpenAI:**
1. Създайте ресурс Azure OpenAI в портала на Azure
2. Разположете модел (например `gpt-5-mini`) и отбележете името на разполагането
3. Влезте с `az login` и конфигурирайте променливите на средата както е показано по-горе

### 🏗️ **Преглед на архитектурата на работния поток**

```mermaid
graph TD
    A[Конструктор на работни потоци] --> B[Регистър на агенти]
    B --> C[Двигател за изпълнение на работни потоци]
    C --> D[Agent 1: Генератор на съдържание]
    C --> E[Agent 2: Преглед на съдържание] 
    D --> F[Резултати от работния поток]
    E --> F
    G[Azure OpenAI (API за отговори)] --> D
    G --> E
```

**Ключови компоненти:**
- **WorkflowBuilder**: Основният оркестрационен двигател за проектиране на работни потоци
- **AIAgent**: Отделни специализирани агенти с конкретни възможности
- **Azure OpenAI Client**: Интеграция с Azure OpenAI Responses API
- **Execution Context**: Управлява състоянието и потока на данни между етапите на работния поток

## 💻 Стартиране на кода

Пълната имплементация е налична в `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Този файл демонстрира:

1. **Конфигурация на средата** - Зареждане на конфигурацията за Azure OpenAI от `.env` файл
2. **Настройка на Azure OpenAI клиента** - Конфигуриране на клиента за използване на Azure OpenAI Responses API
3. **Създаване на агенти** - Дефиниране на специализирани агенти (Front Desk и Concierge)
4. **Workflow Builder** - Създаване на многоагентен работен поток с последователна обработка
5. **Изпълнение на работния поток** - Стартиране на работния поток с излъчване на резултати в реално време

### 🚀 Стартиране на примера

```bash
# Направете скрипта изпълним (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Стартирайте работния процес
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Или на Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Очакван изход

Работният поток ще:
1. Приеме вашето запитване за пътуване ("Искам да отида в Париж")
2. Агентът на Front Desk дава първоначална препоръка
3. Агентът Concierge преглежда и усъвършенства препоръката
4. Крайният резултат показва пълния поток на разговора

### 🔧 Персонализиране

Можете да персонализирате работния поток чрез:
- Промяна на инструкциите на агентите, за да промените тяхното поведение
- Добавяне на повече агенти за създаване на сложни многоетапни работни потоци
- Смяна на съобщението от потребителя за тестване на различни сценарии
- Регулиране на ръбовете на работния поток за създаване на различни модели на изпълнение
