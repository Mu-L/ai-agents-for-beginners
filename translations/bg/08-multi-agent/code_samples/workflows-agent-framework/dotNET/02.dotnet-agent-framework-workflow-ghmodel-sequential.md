# ⏩ Последователни работни потоци на агент с Azure OpenAI (Responses API) (.NET)

## 📋 Урок за разширена последователна обработка

Тази тетрадка демонстрира **модели на последователни работни потоци** с помощта на Microsoft Agent Framework за .NET и Azure OpenAI (Responses API). Ще научите как да изграждате сложни, стъпка по стъпка обработващи конвейери, където агентите изпълняват в определен ред, като всяка фаза се надгражда върху резултатите от предходната.

## 🎯 Цели на обучението

### 🔄 **Архитектура на последователната обработка**
- **Линеен дизайн на работния поток**: Създаване на обработващи конвейери стъпка по стъпка с ясни зависимости
- **Управление на състоянието**: Поддържане на контекст и поток на данни през последователни етапи на работния поток
- **Azure OpenAI (Responses API)**: Използване на модели Azure OpenAI в многоетапни .NET работни потоци
- **Корпоративни модели на конвейри**: Изграждане на готови за производството последователни системи за обработка

### 🏗️ **Разширени последователни модели**
- **Обработка с контролни точки (Stage-Gate)**: Внедряване на проверки за валидност между етапите на работния поток
- **Запазване на контекста**: Поддържане на състояние и натрупани знания през всички етапи
- **Разпространение на грешки**: Гладко управление на неизправности в последователните вериги за обработка
- **Оптимизация на производителността**: Ефективно последователно изпълнение с минимални разходи

### 🏢 **Корпоративни последователни приложения**
- **Конвейер за обработка на документи**: Многоетапен анализ, трансформация и валидиране на документи
- **Работни потоци за контрол на качеството**: Последователно преглеждане, валидиране и одобрения
- **Конвейер за създаване на съдържание**: Проучване → Писане → Редактиране → Преглед → Публикуване
- **Автоматизация на бизнес процеси**: Многостъпкови бизнес работни потоци с ясни зависимости между етапи

```env
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Управление на конфигурацията:**

### 🏗️ **Архитектура на последователния работен поток**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Анализиращ агент]
    B --> C[Контролна точка 1]
    C --> D[Stage 2: Обработващ агент]
    D --> E[Контролна точка 2]
    E --> F[Stage 3: Валидиращ агент]
    F --> G[Краен резултат]
    
    H[Контекст на състоянието] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Responses API)] --> B
    I --> D
    I --> F
```

**Основни компоненти:**
- **Последователни агенти**: Специализирани агенти за всеки етап от обработката
- **Контекст на състоянието**: Поддържа натрупани данни и решения през етапите
- **Контролни точки**: Точки за валидиране между етапите за гарантиране на качество и съгласуваност
- **Клиент на Azure OpenAI**: Консистентен достъп до AI моделите през всички етапи на работния поток

## 💻 Стартиране на кода

Пълната имплементация е налична в `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Този файл демонстрира **триетапен работен поток за анализ на мебели**:

1. **Етап 1 - Търговски агент**: Анализира изображения на мебели и дава препоръки за покупка
2. **Етап 2 - Ценообразуващ агент**: Предоставя подробни цени и опции за бюджет
3. **Етап 3 - Агент за оферти**: Генерира професионален офертен документ във формат Markdown

### 🏗️ **Архитектура на работния поток**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Всеки агент:
- Получава изхода от предишния етап като контекст
- Надгражда предишния анализ с специализирана експертиза
- Поддържа непрекъснатостта на работния поток чрез управление на състоянието

### 🚀 Стартиране на примера

**Предварителни условия:**
- Поставете изображение на мебели в `../imgs/home.png` (или актуализирайте променливата `imgPath`)
- Конфигурирайте `.env` файла си с крайна точка и разгръщане на Azure OpenAI, след което влезте с `az login`

```bash
# Направете скрипта изпълним (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Стартирайте последователната работна последователност
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Или в Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Очакваният резултат

Работният поток ще:
1. **Търговски агент**: Разпознае мебели на изображението и предостави препоръки
2. **Ценообразуващ агент**: Добави детайлен ценови анализ с бюджетни нива и препоръки за пазаруване
3. **Агент за оферти**: Генерира форматиран офертен документ с цялата синтезирана информация

Крайният резултат ще бъде изчерпателна, професионална мебелна оферта на базата на анализ на изображението.

### 🎯 Приложения в реалния свят

Този последователен модел е идеален за:
- **Електронна търговия**: Анализ на продукти → Ценообразуване → Генериране на оферти
- **Недвижими имоти**: Анализ на имоти → Оценка → Създаване на обява

### 🔍 Разбиране на потока на състоянието

Всеки агент в последователността получава:
- **Първоначален вход**: Първоначалното съобщение от потребителя (изображение + текст)
- **Изходи от предишни агенти**: Всички отговори от предишни агенти в историята на разговора
- **Натрупан контекст**: Пълно състояние, поддържано през целия работен поток

Това позволява сложна многоетапна обработка, където всеки агент изгражда върху цялостния контекст от всички предишни етапи.
