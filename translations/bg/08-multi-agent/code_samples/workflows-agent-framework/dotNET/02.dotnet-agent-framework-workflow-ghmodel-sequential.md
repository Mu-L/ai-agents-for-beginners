# ⏩ Последователни работни потоци с Azure OpenAI (Responses API) (.NET)

## 📋 Напреднал урок за последователна обработка

Тази тетрадка демонстрира **шаблони за последователни работни потоци**, използвайки Microsoft Agent Framework за .NET и Azure OpenAI (Responses API). Ще научите как да изградите сложни, стъпка по стъпка обработки, при които агентите изпълняват в определен ред, като всеки етап се гради върху резултатите от предходния етап.

## 🎯 Цели на обучението

### 🔄 **Архитектура на последователната обработка**
- **Линеен дизайн на работния поток**: Създаване на стъпка по стъпка обработки с ясни зависимости
- **Управление на състоянието**: Поддържане на контекст и поток на данни през етапите на последователния работен поток
- **Azure OpenAI (Responses API)**: Използване на Azure OpenAI модели в многослойни .NET работни потоци
- **Шаблони за корпоративни работни потоци**: Изграждане на производствени последователни системи за обработка

### 🏗️ **Напреднали последователни шаблони**
- **Обработка с контроли на пропускателни пунктове (Stage-Gate)**: Имплементиране на валидиращи контролни точки между етапите на работния поток
- **Запазване на контекст**: Поддържане на състояние и натрупани знания през всички етапи
- **Препращане на грешки**: Обработка на неуспехи по изящен начин в последователни вериги за обработка
- **Оптимизация на производителността**: Ефективно последователно изпълнение с минимален разход

### 🏢 **Корпоративни последователни приложения**
- **Работен поток за обработка на документи**: Многослойна обработка, трансформация и валидиране на документи
- **Работни потоци за контрол на качеството**: Последователен преглед, валидиране и одобрение
- **Работен поток за създаване на съдържание**: Проучване → Писане → Редактиране → Преглед → Публикуване
- **Автоматизация на бизнес процеси**: Многоетапни бизнес работни потоци с ясни зависимости между етапите

### 🏗️ **Архитектура на последователните работни потоци**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Агент за анализ]
    B --> C[Контролна точка 1]
```

**Основни компоненти:**

- **Контролни точки**: Валидиращи точки между етапите за осигуряване на качество и последователност
- **Клиент Azure OpenAI**: Последователен достъп до AI модела за всички етапи

## 💻 Изпълнение на кода

Пълната реализация е налична в `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Този файл демонстрира **тристепенен работен поток за анализ на мебели**:

1. **Етап 1 - Агент продажби**: Анализира изображения на мебели и предоставя предложения за покупка
2. **Етап 2 - Агент за цени**: Предоставя подробен ценови анализ и бюджетни опции
3. **Етап 3 - Агент оферти**: Генерира професионален офертен документ в Markdown формат

### 🏗️ **Архитектура на работния поток**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Всеки агент:
- Получава изхода от предходния етап като контекст
- Изгражда върху предишния анализ със специализирана експертиза
- Поддържа непрекъснатостта на работния поток чрез управление на състоянието

### 🚀 Изпълнение на примера

**Предварителни изисквания:**
- Поставете изображение на мебели в `../imgs/home.png` (или актуализирайте променливата `imgPath`)
- Конфигурирайте вашия `.env` файл с крайна точка на Azure OpenAI и разполагането, след което влезте с `az login`

```bash
# Направете скрипта изпълним (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Стартирайте последователния работен поток
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Или в Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Очакван изход

Работният поток ще:
1. **Агент продажби**: Идентифицира мебелните елементи от изображението и дава препоръки
2. **Агент за цени**: Добавя подробен ценови анализ с бюджетни нива и препоръки за пазаруване
3. **Агент оферти**: Генерира форматиран офертен документ със синтезирана цялата информация

Крайният резултат ще бъде изчерпателна, професионална оферта за мебели, базирана на анализ на изображението.

### 🎯 Приложения в реалния свят

Този последователен шаблон е идеален за:

- **Застраховане**: Анализ на претенция → Оценка → Генериране на оферта

### 🔍 Разбиране на потока на състоянието

Всеки агент в последователността получава:
- **Оригинален вход**: Първоначалното потребителско съобщение (изображение + текст)
- **Изходи на предишните агенти**: Всички предишни отговори на агентите в историята на разговора
- **Натрупан контекст**: Пълното състояние, поддържано през целия работен поток

Това позволява сложна многостепенна обработка, при която всеки агент се позовава на обстоен контекст от всички предишни етапи.
