## 이벤트 이름: LangGraph와 Azure Cosmos DB를 활용한 멀티 에이전트 API

## 설명

멀티 에이전트 AI 애플리케이션의 부상은 지능형 시스템 구축 방식을 변화시키고 있지만, 실제 세계에서 확장성과 성능을 위해 어떻게 설계해야 할까요? 이번 세션에서는 LangGraph로 에이전트 오케스트레이션을, FastAPI로 API 레이어를, Azure Cosmos DB를 상태 관리, 벡터 저장소, 트랜잭션 데이터의 핵심 인프라로 사용한 본격적인 멀티 에이전트 애플리케이션을 심도 있게 살펴봅니다.

상세한 코드 워크스루를 통해 RAG(검색 보강 생성), 메모리 지속성, 동적 상태 전환을 원활하게 통합하는 에이전트 구동 워크플로우 설계 및 구현 과정을 볼 수 있습니다. 다룹니다:

LangGraph를 활용한 구조화된 추론을 위한 에이전트 협력
Azure Cosmos DB를 활용한 실시간 채팅 기록 저장 - ChatGPT에서 채팅 기록을 지원하는 동일한 데이터베이스로, 역사상 가장 빠르게 성장하는 AI 에이전트 애플리케이션입니다
Cosmos DB의 네이티브 임베딩 지원을 활용한 지식 검색 벡터 검색
FastAPI의 비동기 기능을 활용해 상호작용을 반응형 및 확장 가능하게 유지
이 세션을 마치면 현대 AI 및 클라우드 인프라의 힘을 활용하는 확장 가능하고 클라우드 네이티브 멀티 에이전트 애플리케이션 구축 및 배포를 위한 명확한 청사진을 갖게 됩니다. AI 엔지니어, 클라우드 아키텍트, Python 개발자 누구나 다음 세대 AI 기반 애플리케이션 구축을 위한 실용적인 인사이트와 검증된 패턴을 얻을 수 있습니다.
