# ⚡ Azure OpenAI(Responses API)를 사용한 동시 에이전트 워크플로우 (.NET)

## 📋 고성능 병렬 처리 튜토리얼

이 노트북은 Microsoft Agent Framework for .NET과 Azure OpenAI(Responses API)를 사용한 <strong>동시 워크플로우 패턴</strong>을 보여줍니다. 여러 AI 에이전트를 동시에 실행하여 처리량을 극대화하면서 조정과 데이터 일관성을 유지하는 고성능 병렬 처리 워크플로우 구축 방법을 배웁니다.

## 🎯 학습 목표

### 🚀 **동시 처리 기초**
- **병렬 에이전트 실행**: 최대 성능을 위해 여러 AI 에이전트를 동시에 실행
- **Async/Await 패턴**: 효율적인 동시성을 위한 .NET의 비동기 프로그래밍 모델 활용
- **Azure OpenAI (Responses API)**: Azure OpenAI Responses API에 대한 여러 동시 호출 조정
- **리소스 관리**: 동시 작업 간 AI 모델 리소스를 효율적으로 관리

### 🏗️ **고급 동시성 아키텍처**
- **작업 기반 병렬 처리**: 최적의 동시 실행을 위한 .NET Task Parallel Library 사용
- **동기화 패턴**: 경합 상태를 피하면서 동시 에이전트 조정
- **부하 분산**: 사용 가능한 동시 처리 용량 전반에 작업을 효율적으로 분배
- <strong>내결함성</strong>: 전체 워크플로우 중단 없이 개별 에이전트 실패 처리

### 🏢 **엔터프라이즈 동시 애플리케이션**
- **고용량 문서 처리**: 여러 문서를 동시에 처리
- **실시간 콘텐츠 분석**: 들어오는 데이터 스트림 동시 분석
- **배치 처리 최적화**: 대규모 데이터 처리 작업의 처리량 극대화
- **다중 모드 분석**: 다양한 콘텐츠 유형과 형식 병렬 처리

## 💻 코드 실행

전체 구현은 `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`에 있습니다. 이 파일은 여행 계획을 위한 <strong>Fan-Out/Fan-In 동시 워크플로우</strong>를 보여줍니다:

### 🏗️ **워크플로우 아키텍처**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**주요 구성 요소:**

1. **ConcurrentStartExecutor**: 사용자 요청을 모든 에이전트에 동시에 방송
2. **Researcher Agent**: 목적지 및 명소를 동시 분석
3. **Planner Agent**: 상세 여행 계획을 동시 생성
4. **ConcurrentAggregationExecutor**: 두 에이전트 결과 수집 및 병합

### 🎯 **Fan-Out/Fan-In 패턴**

이 워크플로우는 기본적인 **Fan-Out/Fan-In** 패턴을 보여줍니다:
- **Fan-Out**: 하나의 입력 메시지를 여러 에이전트에 동시에 방송
- **동시 처리**: 여러 에이전트가 동일 작업을 병렬 수행
- **Fan-In**: 모든 에이전트의 결과를 수집 후 단일 출력으로 집계

### 🔍 사용자 지정 실행기 이해하기

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` 구현으로 문자열 입력 수신
- 모든 연결된 에이전트에 메시지 방송
- 동시 처리를 트리거하는 `TurnToken` 전송

**ConcurrentAggregationExecutor:**
- `IMessageHandler<ChatMessage>` 구현으로 에이전트 응답 수신
- 스레드 안전 방식으로 메시지 수집
- 모든 예상 응답 도착 시 집계 수행
- `context.YieldOutputAsync()`를 사용하여 최종 출력 반환

### ⚡ 성능 이점

**동시 처리 대 순차 처리:**
- 순차 처리: 에이전트1 (30초) → 에이전트2 (30초) = **총 60초**
- 동시 처리: 에이전트1 (30초) || 에이전트2 (30초) = **총 30초**

**처리량 향상**: N개의 동시 에이전트에 대해 최대 N배 속도 향상(작업량 및 리소스에 따라 다름)

### 🛡️ 오류 처리

워크플로우는 개별 에이전트 실패를 우아하게 처리합니다:
- 한 에이전트가 실패해도 다른 에이전트는 계속 처리
- 집계기는 타임아웃 로직 구현 가능
- 필요시 부분 결과 반환 가능
