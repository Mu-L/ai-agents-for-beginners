# 🔄 Azure OpenAI (Responses API)와 함께 사용하는 기본 에이전트 워크플로우 (.NET)

## 📋 워크플로우 오케스트레이션 튜토리얼

이 노트북에서는 Microsoft Agent Framework for .NET과 Azure OpenAI (Responses API)를 사용하여 정교한 <strong>에이전트 워크플로우</strong>를 구축하는 방법을 시연합니다. AI 에이전트들이 협력하여 다단계 비즈니스 프로세스를 구조화된 오케스트레이션 패턴을 통해 복잡한 작업을 수행하는 방법을 배우게 됩니다.

## 🎯 학습 목표

### 🏗️ **워크플로우 아키텍처 기초**
- **워크플로우 빌더**: 복잡한 다단계 AI 프로세스 설계 및 오케스트레이션
- **에이전트 조정**: 워크플로우 내 여러 전문 에이전트 조정
- **Azure OpenAI (Responses API)**: 워크플로우에서 Azure OpenAI Responses API 활용
- **시각적 워크플로우 설계**: 워크플로우 구조를 만들고 시각화하여 이해 증진

### 🔄 **프로세스 오케스트레이션 패턴**
- **순차 처리**: 여러 에이전트 작업을 논리적 순서로 연쇄 실행
- **상태 관리**: 워크플로우 단계 간 컨텍스트 및 데이터 흐름 유지
- **오류 처리**: 견고한 오류 복구 및 워크플로우 내성 구현
- **성능 최적화**: 엔터프라이즈 규모 작업을 위한 효율적 워크플로우 설계

### 🏢 **엔터프라이즈 워크플로우 응용**
- **비즈니스 프로세스 자동화**: 복잡한 조직 워크플로우 자동화
- **콘텐츠 제작 파이프라인**: 검토 및 승인 단계가 포함된 편집 워크플로우
- **고객 서비스 자동화**: 다단계 고객 문의 해결
- **데이터 처리 워크플로우**: AI 기반 변환을 포함한 ETL 워크플로우

## ⚙️ 필수 조건 및 설정

### 📦 **필요한 NuGet 패키지**

이 워크플로우 데모에는 몇 가지 핵심 .NET 패키지가 사용됩니다:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI 구성**

**환경 설정 (.env 파일):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Azure OpenAI 접근법:**
1. Azure 포털에서 Azure OpenAI 리소스 생성
2. 모델 배포 (예: `gpt-5-mini`) 및 배포 이름 메모
3. `az login`으로 로그인하고 위와 같이 환경 변수 구성

### 🏗️ **워크플로우 아키텍처 개요**

```mermaid
graph TD
    A[워크플로 빌더] --> B[에이전트 등록부]
    B --> C[워크플로 실행 엔진]
    C --> D[Agent 1: 콘텐츠 생성기]
    C --> E[Agent 2: 콘텐츠 검토자] 
    D --> F[워크플로 결과]
    E --> F
    G[Azure OpenAI (응답 API)] --> D
    G --> E
```

**주요 구성 요소:**
- **WorkflowBuilder**: 워크플로우 설계를 위한 주요 오케스트레이션 엔진
- **AIAgent**: 특정 기능을 가진 개별 전문 에이전트
- **Azure OpenAI Client**: Azure OpenAI Responses API 통합
- **Execution Context**: 워크플로우 단계 간 상태 및 데이터 흐름 관리

## 💻 코드 실행

전체 구현은 `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`에 있습니다. 이 파일에서는 다음을 시연합니다:

1. **환경 구성** - `.env` 파일에서 Azure OpenAI 구성 로드
2. **Azure OpenAI 클라이언트 설정** - Azure OpenAI Responses API 사용을 위한 클라이언트 구성
3. **에이전트 생성** - 전문화된 에이전트(Front Desk 및 Concierge) 정의
4. **워크플로우 빌더** - 순차 처리로 다중 에이전트 워크플로우 생성
5. **워크플로우 실행** - 스트리밍 결과와 함께 워크플로우 실행

### 🚀 예제 실행

```bash
# 스크립트를 실행 가능하게 만들기 (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# 워크플로우 실행하기
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Windows에서 실행하려면:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 예상 출력

워크플로우는 다음과 같은 작업을 수행합니다:
1. 여행 목적지 요청 수락 ("파리로 가고 싶습니다")
2. Front Desk 에이전트가 초기 추천 제공
3. Concierge 에이전트가 추천 내용을 검토 및 수정
4. 최종 출력에 전체 대화 스트림 표시

### 🔧 맞춤화

워크플로우를 다음과 같이 맞춤화할 수 있습니다:
- 에이전트 지침을 수정하여 동작 변경
- 더 많은 에이전트를 추가하여 복잡한 다단계 워크플로우 생성
- 사용자 메시지를 바꿔 다양한 시나리오 테스트
- 워크플로우 엣지를 조정하여 다른 실행 패턴 생성
