# ⏩ Azure OpenAI (Responses API)와 함께 하는 순차적 에이전트 워크플로우 (.NET)

## 📋 고급 순차 처리 튜토리얼

이 노트북은 Microsoft Agent Framework for .NET과 Azure OpenAI (Responses API)를 사용한 <strong>순차적 워크플로우 패턴</strong>을 시연합니다. 각 단계가 이전 단계의 결과를 바탕으로 실행되는 정교한 단계별 처리 파이프라인을 구축하는 방법을 배우게 됩니다.

## 🎯 학습 목표

### 🔄 **순차 처리 아키텍처**
- **선형 워크플로우 설계**: 명확한 의존성을 가진 단계별 처리 파이프라인 생성
- **상태 관리**: 순차 워크플로우 단계 간 문맥과 데이터 흐름 유지
- **Azure OpenAI (Responses API)**: 다단계 .NET 워크플로우에서 Azure OpenAI 모델 활용
- **기업용 파이프라인 패턴**: 생산 환경에 적합한 순차 처리 시스템 구축

### 🏗️ **고급 순차 패턴**
- **단계별 검증 처리(Stage-Gate Processing)**: 워크플로우 단계 간 검증 체크포인트 구현
- **문맥 보존**: 모든 단계를 아우르는 상태 및 누적 지식 유지
- **오류 전파 처리**: 순차 처리 체인 내 실패에 대한 우아한 처리
- **성능 최적화**: 최소한의 오버헤드로 효율적인 순차 실행

### 🏢 **기업용 순차 애플리케이션**
- **문서 처리 파이프라인**: 다단계 문서 분석, 변환 및 검증
- **품질 보증 워크플로우**: 순차 리뷰, 검증 및 승인 프로세스
- **콘텐츠 제작 파이프라인**: 조사 → 작성 → 편집 → 검토 → 출판
- **비즈니스 프로세스 자동화**: 명확한 단계 의존성을 가진 다단계 비즈니스 워크플로우

**환경 설정 (.env 파일):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**구성 관리:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **순차 워크플로우 아키텍처**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: 분석 에이전트]
    B --> C[체크포인트 1]
    C --> D[Stage 2: 처리 에이전트]
    D --> E[체크포인트 2]
    E --> F[Stage 3: 검증 에이전트]
    F --> G[최종 출력]
    
    H[상태 컨텍스트] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (응답 API)] --> B
    I --> D
    I --> F
```

**핵심 구성 요소:**
- **순차 에이전트**: 각 처리 단계별 특화된 에이전트
- **상태 문맥**: 단계 간 누적 데이터와 의사결정 유지
- <strong>체크포인트</strong>: 품질과 일관성을 보장하는 단계 간 검증 지점
- **Azure OpenAI 클라이언트**: 모든 워크플로우 단계에서 일관된 AI 모델 접근

## 🏢 **기업용 순차 처리의 이점**

### 🎯 **신뢰성 및 품질**
- **결정적 처리**: 구조화된 단계를 통한 일관되고 반복 가능한 결과
- **품질 게이트**: 각 단계에서 품질을 보장하는 검증 체크포인트
- **오류 격리**: 한 단계의 문제는 이후 단계로 전파되지 않음
- **감사 추적**: 각 단계의 의사결정과 변환 완전 추적

### 📈 **확장성 및 성능**
- **모듈형 설계**: 각 단계를 독립적으로 최적화 가능
- **자원 관리**: 단계별 AI 모델 자원의 효율적 할당
- **상태 최적화**: 최적 성능을 위한 최소 상태 전송
- **병렬 단계 그룹**: 여러 순차 워크플로우를 병렬 실행 가능

### 🔒 **보안 및 규정 준수**
- **단계별 보안**: 각 처리 단계마다 다른 보안 정책 적용
- **데이터 검증**: 각 체크포인트에서 데이터 무결성과 규정 준수 확인
- **접근 제어**: 단계별 세분화된 권한 관리
- **규제 준수**: 구조화된 처리를 통한 규제 요구사항 충족

### 📊 **모니터링 및 분석**
- **단계별 지표**: 각 워크플로우 단계의 성능 모니터링
- **병목 현상 식별**: 느린 단계를 식별하고 최적화
- **품질 지표**: 각 단계의 품질과 성공률 추적
- **프로세스 최적화**: 단계별 분석을 통한 지속적 개선

## 💻 코드 실행하기

전체 구현은 `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`에 있습니다. 이 파일은 <strong>3단계 가구 분석 워크플로우</strong>를 시연합니다:

1. **1단계 - 판매 에이전트**: 가구 이미지를 분석하고 구매 제안 제공
2. **2단계 - 가격 에이전트**: 상세 가격 내역 및 예산 옵션 제공
3. **3단계 - 견적 에이전트**: Markdown 형식의 전문 견적서 생성

### 🏗️ **워크플로우 아키텍처**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

각 에이전트는:
- 이전 단계 결과를 문맥으로 받아 처리
- 특화된 전문지식으로 이전 분석을 확장
- 상태 관리를 통해 워크플로우 연속성 유지

### 📝 예상 출력 결과

워크플로우는 다음 작업을 수행합니다:
1. **판매 에이전트**: 이미지에서 가구 항목을 식별하고 추천 제공
2. **가격 에이전트**: 예산 등급과 쇼핑 추천을 포함한 상세 가격 분석 추가
3. **견적 에이전트**: 모든 정보를 종합한 형식화된 견적서 문서 생성

최종 출력은 이미지 분석을 바탕으로 한 종합적이고 전문적인 가구 견적서가 될 것입니다.
