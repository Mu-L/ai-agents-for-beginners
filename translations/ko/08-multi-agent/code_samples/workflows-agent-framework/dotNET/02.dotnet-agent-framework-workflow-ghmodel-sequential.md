# ⏩ Azure OpenAI(Responses API) (.NET)과 함께하는 순차적 에이전트 워크플로우

## 📋 고급 순차 처리 튜토리얼

이 노트북은 Microsoft Agent Framework for .NET과 Azure OpenAI(Responses API)를 사용한 <strong>순차적 워크플로우 패턴</strong>을 보여줍니다. 에이전트가 특정 순서로 실행되고 각 단계가 이전 단계의 결과를 바탕으로 진행되는 정교한 단계별 처리 파이프라인을 구축하는 방법을 배웁니다.

## 🎯 학습 목표

### 🔄 **순차 처리 아키텍처**
- **선형 워크플로우 설계**: 명확한 의존성을 가진 단계별 처리 파이프라인 생성
- **상태 관리**: 순차 워크플로우 단계 전반에 걸친 컨텍스트 및 데이터 흐름 유지
- **Azure OpenAI (Responses API)**: 다단계 .NET 워크플로우에서 Azure OpenAI 모델 활용
- **엔터프라이즈 파이프라인 패턴**: 프로덕션 수준의 순차적 처리 시스템 구축

### 🏗️ **고급 순차 패턴**
- **스테이지 게이트 처리**: 워크플로우 단계 간 검증 지점 구현
- **컨텍스트 보존**: 모든 단계 전반에 걸친 상태와 누적 지식 유지
- **오류 전파 처리**: 순차 처리 체인의 실패를 원활하게 처리
- **성능 최적화**: 최소한의 오버헤드로 효율적인 순차 실행

### 🏢 **엔터프라이즈 순차 애플리케이션**
- **문서 처리 파이프라인**: 다단계 문서 분석, 변환, 검증
- **품질 보증 워크플로우**: 순차 검토, 검증, 승인 프로세스
- **콘텐츠 제작 파이프라인**: 연구 → 작성 → 편집 → 검토 → 발행
- **비즈니스 프로세스 자동화**: 명확한 단계 의존성을 가진 다단계 비즈니스 워크플로우

- **Azure OpenAI 클라이언트**: 모든 워크플로우 단계에서 일관된 AI 모델 액세스

## 💻 코드 실행

전체 구현은 `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`에 있습니다. 이 파일은 <strong>3단계 가구 분석 워크플로우</strong>를 보여줍니다:

1. **1단계 - 영업 에이전트**: 가구 이미지를 분석하고 구매 제안 제공
2. **2단계 - 가격 에이전트**: 상세 가격 분석 및 예산 옵션 제공
3. **3단계 - 견적 에이전트**: Markdown 형식의 전문 견적 문서 생성

### 🏗️ **워크플로우 아키텍처**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

각 에이전트는:
- 이전 단계에서 나온 출력을 컨텍스트로 받음
- 전문 지식을 활용해 이전 분석을 기반으로 확장
- 상태 관리를 통해 워크플로우 연속성 유지

### 🚀 예제 실행

**사전 요구 사항:**
- `../imgs/home.png`에 가구 이미지를 넣거나 `imgPath` 변수를 업데이트
- `.env` 파일에 Azure OpenAI 엔드포인트 및 배포를 설정하고 `az login`으로 로그인

### 📝 예상 출력

워크플로우는:
1. **영업 에이전트**: 이미지에서 가구 항목을 식별하고 추천 제공
2. **가격 에이전트**: 예산 등급과 쇼핑 추천을 포함한 상세 가격 분석 추가
3. **견적 에이전트**: 모든 정보를 종합한 포맷된 견적 문서 생성

최종 출력은 이미지 분석 기반의 포괄적이고 전문적인 가구 견적입니다.

이 순차 패턴은 다음에 이상적입니다:

- <strong>부동산</strong>: 부동산 분석 → 가치 평가 → 매물 생성

### 🔍 상태 흐름 이해하기

순차 내 각 에이전트는:
- **원본 입력**: 초기 사용자 메시지(이미지 + 텍스트)
- **이전 에이전트 출력**: 대화 기록 내 모든 이전 에이전트 응답
- **누적 컨텍스트**: 워크플로우 전반에 걸쳐 유지되는 완전한 상태

이를 통해 각 에이전트가 이전 모든 단계의 종합적 컨텍스트를 바탕으로 정교한 다단계 처리를 수행할 수 있습니다.
