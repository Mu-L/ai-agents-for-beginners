# 🔄 Mipangilio ya Kazi ya Wakala wa Msingi na Azure OpenAI (API ya Majibu) (.NET)

## 📋 Mafunzo ya Uratibu wa Mchakato wa Kazi

Daftari hili linaonyesha jinsi ya kujenga **mipangilio tata ya wakala** kwa kutumia Microsoft Agent Framework kwa .NET na Azure OpenAI (API ya Majibu). Utafundishwa kuunda michakato ya biashara yenye hatua nyingi ambapo mawakala wa AI hushirikiana kutimiza kazi tata kupitia mifumo ya uratibu iliyoandaliwa.

## 🎯 Malengo ya Kujifunza

### 🏗️ **Misingi ya Muundo wa Mchakato wa Kazi**
- **Mjenzi wa Mchakato wa Kazi**: Buni na ratibu michakato tata ya AI yenye hatua nyingi
- **Ushirikiano wa Wakala**: Ratibu mawakala maalum wengi ndani ya michakato ya kazi
- **Azure OpenAI (API ya Majibu)**: Tumia API ya Majibu ya Azure OpenAI katika michakato ya kazi
- **Ubunifu wa Mchakato wa Kazi kwa Mtazamo wa Picha**: Unda na onyesha miundo ya michakato ya kazi kwa ufahamu bora

### 🔄 **Mifumo ya Uratibu wa Mchakato**
- **Usindikaji wa Mtawaliwa**: Unganisha kazi nyingi za wakala kwa mpangilio wa mantiki
- **Usimamizi wa Hali**: Dumisha muktadha na mtiririko wa data katika hatua za michakato ya kazi
- **Utambuzi wa Makosa**: Tekeleza uwezeshaji bora wa urejeshaji makosa na uimara wa mchakato
- **Uboreshaji wa Utendaji**: Buni michakato bora kwa ajili ya operesheni za kiwango cha biashara

### 🏢 **Programu za Michakato ya Kazi ya Biashara**
- **Uendeshaji wa Michakato ya Biashara**: Kuendesha moja kwa moja michakato tata ya shirika
- **Mtiririko wa Uzalishaji wa Maudhui**: Michakato ya uhariri yenye hatua za mapitio na idhini
- **Uendeshaji wa Huduma kwa Wateja**: Utatuzi wa maswali ya wateja kwa hatua nyingi
- **Michakato ya Usindikaji Data**: Michakato ya ETL yenye mabadiliko yanayofanywa na AI

## ⚙️ Mahitaji na Mpangilio

### 📦 **Pakiti za NuGet Zinazohitajika**

Maonyesho haya ya mchakato yanatumia pakiti kadhaa muhimu za .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Mpangilio wa Azure OpenAI**

**Mpangilio wa Mazingira (faili `.env`):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Ufikiaji wa Azure OpenAI:**
1. Unda rasilimali ya Azure OpenAI katika portal ya Azure
2. Tengeneza mfano (kwa mfano, `gpt-4.1-mini`) na kumbuka jina la usaaji
3. Jisajili kwa `az login` na panga mabadiliko ya mazingira kama ilivyoonyeshwa hapo juu

### 🏗️ **Muhtasari wa Muundo wa Mchakato wa Kazi**

```mermaid
graph TD
    A[Mjenzi wa Mtiririko wa Kazi] --> B[Rejista ya Wakala]
    B --> C[Injini ya Utekelezaji wa Mtiririko wa Kazi]
    C --> D[Agent 1: Mzalishaji wa Yaliyomo]
    C --> E[Agent 2: Mtathmini wa Yaliyomo] 
    D --> F[Matokeo ya Mtiririko wa Kazi]
    E --> F
    G[Azure OpenAI (API za Majibu)] --> D
    G --> E
```

**Vikundi Muhimu:**
- **WorkflowBuilder**: Kifaa kikuu cha uratibu kwa kubuni michakato ya kazi
- **AIAgent**: Mawakala maalum binafsi wenye uwezo maalum
- **Azure OpenAI Client**: Uingizaji API ya Majibu ya Azure OpenAI
- **Execution Context**: Husimamia hali na mtiririko wa data kati ya hatua za mchakato

Document Input → Analysis → Extraction → Validation → Structured Output

Data Collection → Processing → Analysis → Report Generation → Distribution
```

Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **Manufaa ya Biashara**

### 🎯 **Uaminifu na Uwezo wa Kupanuka**
- **Utekenezaji wa Hakika**: Matokeo thabiti na yanayorudiwa ya mchakato wa kazi
- **Urejeshaji wa Makosa**: Utambuzi mzuri wa kushughulikia kushindwa katika hatua yoyote ya mchakato
- **Ufuatiliaji wa Utendaji**: Fuata vipimo vya utekelezaji na fursa za kuboresha
- **Usimamizi wa Rasilimali**: Mpangilio mzuri na matumizi ya rasilimali za mfano wa AI

### 🔒 **Usalama na Uzingatiaji Sheria**
- **Uthibitishaji Salama**: Uthibitishaji kwa kutumia Microsoft Entra ID kupitia `az login` (AzureCliCredential)
- **Rekodi za Ukaguzi**: Kumbukumbu kamili ya utekelezaji wa mchakato na hatua za maamuzi
- **Udhibiti wa Ufikiaji**: Ruhusa za kina kwa utekelezaji na ufuatiliaji wa mchakato
- **Faragha ya Data**: Ushughulikiaji salama wa taarifa nyeti katika michakato yote

### 📊 **Uwezo wa Kuonekana na Usimamizi**
- **Ubunifu wa Mchakato kwa Mtazamo wa Picha**: Uwakilishaji wazi wa mtiririko wa michakato na utegemezi
- **Ufuatiliaji wa Utekelezaji**: Ufuatiliaji wa moja kwa moja wa maendeleo na utendaji wa mchakato
- **Utoaji Ripoti za Makosa**: Uchambuzi wa kina wa makosa na uwezo wa kutatua kasoro
- **Uchanganuzi wa Utendaji**: Vipimo kwa ajili ya kuboresha na kupanga uwezo

Tuanze kujenga mchakato wako wa AI tayari kwa biashara! 🚀

## 💻 Kukimbia Nambari

Utekelezaji kamili upo katika `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Faili hili linaonyesha:

1. **Mpangilio wa Mazingira** - Kupakia mipangilio ya Azure OpenAI kutoka faili `.env`
2. **Mpangilio wa Mteja wa Azure OpenAI** - Kupanga mteja kutumia API ya Majibu ya Azure OpenAI
3. **Uundaji wa Wakala** - Kufafanua mawakala maalum (Front Desk na Concierge)
4. **Mjenzi wa Mchakato wa Kazi** - Kuunda mchakato wa wakala wengi kwa usindikaji mtawaliwa
5. **Utekelezaji wa Mchakato wa Kazi** - Kukimbia mchakato na kupata matokeo ya mtiririko

### 🚀 Kukimbia Mfano

```bash
# Fanya script iweze kutekelezwa (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Endesha mtiririko wa kazi
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Au kwenye Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```
