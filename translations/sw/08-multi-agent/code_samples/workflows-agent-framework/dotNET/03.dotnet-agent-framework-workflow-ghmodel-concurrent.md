# ⚡ Mifumo ya Kazi ya Wakala Kadhaa kwa Wakati Mmoja na Azure OpenAI (API za Majibu) (.NET)

## 📋 Mafunzo ya Usindishaji Sambamba wa Utendaji wa Juu

Daftari hili linaonyesha **mitindo ya mifumo ya kazi kwa wakati mmoja** ukitumia Microsoft Agent Framework kwa .NET na Azure OpenAI (API za Majibu). Utajifunza jinsi ya kujenga mifumo ya kazi ya usindikaji sambamba wenye utendaji wa juu inayoongeza kasi kwa kuendesha mawakala wa AI wengi kwa wakati mmoja huku ikidumisha uratibu na uthabiti wa data.

- **Uendeshaji Sambamba wa Wakala**: Endesha mawakala wa AI wengi kwa wakati mmoja kwa utendaji wa juu

- **Azure OpenAI (API za Majibu)**: Ratibu miito mingi sambamba kwa Azure OpenAI Responses API
- **Usimamizi wa Rasilimali**: Simamia rasilimali za modeli za AI kwa ufanisi katika operesheni sambamba

### 🏗️ **Marekebisho ya Ubora wa Usindikaji Sambamba**

## 💻 Kuwezesha Msimbo

Utekelezaji kamili upo katika `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Faili hili linaonyesha **mfumo wa kazi wa Fan-Out/Fan-In sambamba** kwa mipango ya kusafiri:

### 🏗️ **Ubunifu wa Mifumo ya Kazi**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Vipengele Muhimu:**

1. **ConcurrentStartExecutor**: Hutuma ombi la mtumiaji kwa wakala wote kwa wakati mmoja
2. **Researcher Agent**: Anachambua maeneo na vivutio kwa wakati mmoja
3. **Planner Agent**: Hutengeneza mipango ya kusafiri kwa kina kwa wakati mmoja
4. **ConcurrentAggregationExecutor**: Hukusanya na kuunganisha matokeo kutoka kwa mawakala wote wawili

### 🎯 **Mfumo wa Fan-Out/Fan-In**

Mfumo huu unaonyesha mfano wa zamani wa **Fan-Out/Fan-In**:
- **Fan-Out**: Ujumbe mmoja wa ingizo hutumwa kwa mawakala wengi kwa wakati mmoja
- **Usindikaji Sambamba**: Wakala wengi hufanya kazi sambamba kwenye kazi moja
- **Fan-In**: Matokeo kutoka kwa mawakala wote hukusanywa na kuunganishwa kuwa pato moja

### 🔍 Kuelewa Watekelezaji Maalum

**ConcurrentStartExecutor:**
- Hutekeleza `IMessageHandler<string>` kupokea ingizo la aina ya string
- Hutuma ujumbe kwa mawakala wote wanaounganishwa
- Hutuma `TurnToken` kuanzisha usindikaji sambamba

**ConcurrentAggregationExecutor:**
- Hutekeleza `IMessageHandler<ChatMessage>` kupokea majibu ya wakala
- Hukusanya ujumbe kwa njia salama kwa thread
- Huujuisha wakati majibu yote yanayotarajiwa yamefika
- Hutoa pato la mwisho kwa kutumia `context.YieldOutputAsync()`

### ⚡ Manufaa ya Utendaji

**Sambamba dhidi ya Mfuatano:**
- Mfuatano: Wakala1 (30s) → Wakala2 (30s) = **60 sekunde jumla**
- Sambamba: Wakala1 (30s) || Wakala2 (30s) = **30 sekunde jumla**

**Kuboresha Kiwango cha Kazi**: Hadi mara N× haraka kwa mawakala N sambamba (kulingana na mzigo na rasilimali)
