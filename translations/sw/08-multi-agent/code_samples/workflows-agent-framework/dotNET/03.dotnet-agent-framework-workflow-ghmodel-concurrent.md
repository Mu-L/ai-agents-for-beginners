# ⚡ Mipangilio ya Wakala Wenye Msimulizi Sambamba na Azure OpenAI (API za Majibu) (.NET)

## 📋 Mafunzo ya Usindikaji wa Sambamba wenye Ufanisi Mkubwa

Daftari hili linaonyesha **mipangilio ya mtiririko wa kazi yenye msimulizi sambamba** kwa kutumia Microsoft Agent Framework kwa .NET na Azure OpenAI (API za Majibu). Utafundishwa jinsi ya kujenga mitiririko ya kazi ya usindikaji sambamba yenye ufanisi mkubwa inayoongeza kiwango cha matokeo kwa kuendesha mawakala wa AI wengi kwa wakati mmoja huku ukidumisha uratibu na usawa wa data.

## 🎯 Malengo ya Kujifunza

- **Uendeshaji sambamba wa Wakala**: Endesha mawakala wa AI wengi kwa wakati mmoja kwa ufanisi mkubwa zaidi

- **Azure OpenAI (API za Majibu)**: Ratibu simu nyingi sambamba kwenye API za Majibu za Azure OpenAI

## 💻 Kuendesha Msimulizi

Utekelezaji kamili upo katika `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Faili hili linaonyesha **mtiririko wa kazi wa Fan-Out/Fan-In sambamba** wa kupanga safari:

### 🏗️ **Miundo ya Mtiririko wa Kazi**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Vijumuishi Muhimu:**

1. **ConcurrentStartExecutor**: Inatangaza ombi la mtumiaji kwa mawakala wote kwa wakati mmoja
2. **Researcher Agent**: Huchambua maeneo na vivutio sambamba
3. **Planner Agent**: Hutengeneza mipango ya maandalizi ya safari sambamba
4. **ConcurrentAggregationExecutor**: Hukusanya na kuunganisha matokeo kutoka kwa mawakala wote wawili

### 🎯 **Mfano wa Fan-Out/Fan-In**

Mtiririko huu unaonyesha mfano wa kawaida wa **Fan-Out/Fan-In**:
- **Fan-Out**: Ujumbe mmoja wa ingizo unatangazwa kwa mawakala wengi kwa wakati mmoja
- **Usindikaji Sambamba**: Mawakala wengi hufanya kazi sambamba kwenye kazi ileile
- **Fan-In**: Matokeo kutoka kwa mawakala yote hukusanywa na kuunganishwa kuwa matokeo moja

### 🔍 Kuelewa Watendaji wa Mbinafsisho

**ConcurrentStartExecutor:**
- Inatekeleza `IMessageHandler<string>` kupokea kiingilio cha maandishi
- Huwatangaza ujumbe kwa mawakala wote waliounganishwa
- Hutuma `TurnToken` kuanzisha usindikaji sambamba

**ConcurrentAggregationExecutor:**
- Inatekeleza `IMessageHandler<ChatMessage>` kupokea majibu ya wakala
- Hukusanya ujumbe kwa usalama wa mfululizo
- Hukusanya matokeo wakati majibu yote yanapotegemewa yanapofika
- Hutengeneza matokeo ya mwisho kwa kutumia `context.YieldOutputAsync()`

### ⚡ Manufaa ya Ufanisi

**Sambamba dhidi ya Mlolongo:**
- Mlolongo: Agent1 (30s) → Agent2 (30s) = **muda jumla wa sekunde 60**
- Sambamba: Agent1 (30s) || Agent2 (30s) = **muda jumla wa sekunde 30**

**Kuboresha Kiwango cha Matokeo**: Hadi mara N× haraka kwa mawakala N sambamba (kutegemea mzigo na rasilimali)
