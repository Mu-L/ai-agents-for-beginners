# ⏩ Peräkkäiset agenttityönkulut Azure OpenAI:n (Responses API) kanssa (.NET)

## 📋 Edistynyt peräkkäinen prosessointitutoriaali

Tämä muistikirja esittelee **peräkkäisiä työnkulku­kuvioita** Microsoft Agent Frameworkin .NET-versiolla ja Azure OpenAI:lla (Responses API). Opit rakentamaan monimutkaisia, vaiheittaisia prosessointiputkia, joissa agentit suorittavat tehtäviä tietyssä järjestyksessä, ja jokainen vaihe rakentuu edellisen vaiheen tulosten varaan.

## 🎯 Oppimistavoitteet

### 🔄 **Peräkkäisen prosessoinnin arkkitehtuuri**
- **Lineaarinen työnkulun suunnittelu**: Luo vaiheittaisia prosessointiputkia, joissa on selkeät riippuvuudet
- **Tilan hallinta**: Säilytä konteksti ja tiedon kulku peräkkäisten työnkulun vaiheiden välillä
- **Azure OpenAI (Responses API)**: Hyödynnä Azure OpenAI -malleja monivaiheisissa .NET-työnkuluissa
- **Yritystason putkikuvioita**: Rakenna tuotantovalmiita peräkkäisiä prosessointijärjestelmiä

### 🏗️ **Edistyneet peräkkäiset kuviot**
- **Vaiheiden valvonta**: Toteuta validointipisteitä työnkulun vaiheiden väliin
- **Kontekstin säilyttäminen**: Säilytä tila ja kertynyt tieto kaikissa vaiheissa
- **Virheiden kulku**: Käsittele virheet hallitusti peräkkäisissä prosessointiketjuissa
- **Suorituskyvyn optimointi**: Tehokas peräkkäinen suoritus minimikustannuksin

### 🏢 **Yritystason peräkkäiset sovellukset**
- **Dokumenttien prosessointiputki**: Monivaiheinen dokumenttien analysointi, muokkaus ja validointi
- **Laatutakuun työnkulut**: Peräkkäinen tarkastus, validointi ja hyväksyntäprosessi
- **Sisällöntuotannon putki**: Tutkimus → Kirjoittaminen → Editointi → Tarkastus → Julkaisu
- **Liiketoimintaprosessien automaatio**: Monivaiheiset liiketoiminnan työnkulut selkeillä vaihe-riippuvuuksilla

## ⚙️ Esivaatimukset & asennus

### 📦 **Vaaditut NuGet-paketit**

Olennaiset paketit .NET:n peräkkäisiin työnkulkuihin:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Azure OpenAI -asetukset**

**Ympäristöasetukset (.env-tiedosto):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Konfiguraation hallinta:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **Peräkkäisen työnkulun arkkitehtuuri**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Analyysiedustaja]
    B --> C[Tarkistuspiste 1]
```

- **Peräkkäiset agentit**: Erikoistuneet agentit jokaiselle prosessointivaiheelle

- **Tarkistuspisteet**: Validointipisteet vaiheiden välillä laadun ja johdonmukaisuuden varmistamiseksi
- **Azure OpenAI -asiakas**: Johdonmukainen AI-mallien käyttö kaikissa työnkulun vaiheissa

## 💻 Koodin suorittaminen

Kokonaisimplementaatio löytyy tiedostosta `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Tämä tiedosto demonstroi **kolmivaiheisen huonekalujen analysointityönkulun**:

1. **Vaihe 1 - Myyntiagentti**: Analysoi huonekalukuvia ja tarjoaa osto­ehdotuksia
2. **Vaihe 2 - Hintagentti**: Antaa yksityiskohtaisen hinnan erittelyn ja budjettivaihtoehdot
3. **Vaihe 3 - Tarjousagentti**: Luo ammattimaisen tarjous­dokumentin Markdown-muodossa

### 🏗️ **Työnkulun arkkitehtuuri**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Jokainen agentti:
- Saa edellisen vaiheen tuloksen kontekstina
- Rakentaa edellisen analyysin päälle erikoistunutta osaamista käyttäen
- Säilyttää työnkulun jatkuvuuden tilanhallinnan avulla

### 🚀 Esimerkin suorittaminen

**Esivaatimukset:**
- Aseta huonekalukuva osoitteeseen `../imgs/home.png` (tai päivitä muuttuja `imgPath`)
- Konfiguroi `.env`-tiedostosi Azure OpenAI -päätepisteen ja -käytön mukaan, kirjautumiseen käytä `az login`

```bash
# Tee skriptistä suoritettava (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Suorita peräkkäinen työnkulku
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Tai Windowsilla:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Odotettu tulos

Työnkulku:
1. **Myyntiagentti**: Tunnistaa huonekaluesineet kuvasta ja antaa suosituksia
2. **Hintagentti**: Lisää yksityiskohtaisen hinta-analyysin budjettitasoineen ja ostosuosituksineen
3. **Tarjousagentti**: Luo muotoillun tarjous­dokumentin, joka kokoaa kaikki tiedot

Lopputuloksena on kattava, ammattimainen huonekalutarjous kuvan analyysin pohjalta.

### 🔧 Mukautusvaihtoehdot

**Muokkaa agentin käyttäytymistä:**
```csharp
// Adjust agent instructions to change their focus
const string SalesAgentInstructions = "Your custom instructions...";
```

**Vaihda peräkkäistä työnkulkua:**
```csharp
// Add or reorder workflow stages
var workflow = new WorkflowBuilder(salesagent)
    .AddEdge(salesagent, priceagent)
    .AddEdge(priceagent, quoteagent)
    .AddEdge(quoteagent, newAgent)  // Add another stage
    .Build();
```

### 🔍 Tilan kulun ymmärtäminen

Jokainen agentti sarjassa saa:
- **Alkuperäisen syötteen**: Käyttäjän alkuperäinen viesti (kuva + teksti)
- **Edellisten agenttien vastaukset**: Kaikki aiemmat vastaukset keskusteluhistoriassa
- **Kertynyt konteksti**: Täydellinen tila, joka säilytetään koko työnkulun ajan

Tämä mahdollistaa kehittyneen monivaiheisen prosessoinnin, jossa jokainen agentti rakentaa kokonaisvaltaisen kontekstin varaan kaikista aiemmista vaiheista.
