# ⏩ Peräkkäiset agenttityönkulut Azure OpenAI:n (Responses API) kanssa (.NET)

## 📋 Kehittynyt peräkkäisen prosessoinnin opetusohjelma

Tämä muistikirja esittelee **peräkkäisiä työnkulkumalleja** Microsoft Agent Frameworkin .NET-versiolla ja Azure OpenAI:lla (Responses API). Opit rakentamaan monimutkaisia, vaiheittaisia prosessointiputkia, joissa agentit suorittavat tehtävät tietyssä järjestyksessä, ja jokainen vaihe rakentuu edellisen vaiheen tulosten päälle.

## 🎯 Oppimistavoitteet

### 🔄 **Peräkkäisen prosessoinnin arkkitehtuuri**
- **Lineaarinen työnkulun suunnittelu**: Luo vaiheittaisia prosessointiputkia, joissa on selkeät riippuvuudet
- **Tilanhallinta**: Säilytä konteksti ja tietovirta peräkkäisten työnkulun vaiheiden välillä
- **Azure OpenAI (Responses API)**: Hyödynnä Azure OpenAI -malleja monivaiheisissa .NET-työnkuluissa
- **Yritysputken mallit**: Rakenna tuotantovalmiita peräkkäisiä prosessointijärjestelmiä

### 🏗️ **Kehittyneet peräkkäiset mallit**
- **Vaiheporttien käsittely**: Toteuta validointipisteet työnkulun vaiheiden välille
- **Kontekstin säilytys**: Säilytä tila ja kertynyt tieto kaikkien vaiheiden yli
- **Virheen leviämisen hallinta**: Käsittele virheitä hallitusti peräkkäisissä prosessointiketjuissa
- **Suorituskyvyn optimointi**: Tehokas peräkkäinen suoritus minimialustalla

### 🏢 **Yritysten peräkkäiset sovellukset**
- **Asiakirjan prosessointiputki**: Monivaiheinen asiakirjan analysointi, muuntaminen ja validointi
- **Laadunvarmistuksen työnkulut**: Peräkkäinen tarkastus, validointi ja hyväksyntäprosessi
- **Sisällöntuotantoputki**: Tutkimus → Kirjoitus → Editointi → Tarkastus → Julkaisu
- **Liiketoimintaprosessien automaatio**: Monivaiheiset liiketoiminnan työnkulut selkeillä vaiheriippuvuuksilla


```csharp
// Load environment variables securely
```

## 💻 Koodin ajaminen

Täydellinen toteutus löytyy tiedostosta `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Tämä tiedosto esittelee **kolmivaiheisen huonekaluanalyysin työnkulun**:

1. **Vaihe 1 - Myyntiedustaja**: Analysoi huonekalujen kuvia ja antaa osto-ehdotuksia
2. **Vaihe 2 - Hintavälittäjä**: Tarjoaa yksityiskohtaiset hintajakautumat ja budjettivaihtoehdot
3. **Vaihe 3 - Tarjousagentti**: Luo ammatillisen tarjousasiakirjan Markdown-muodossa

### 🏗️ **Työnkulun arkkitehtuuri**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Kukin agentti:
- Saa edellisen vaiheen tuotoksen kontekstina
- Rakentaa aiemman analyysin päälle erikoisosaamisella
- Säilyttää työnkulun jatkuvuuden tilanhallinnan avulla

### 🚀 Esimerkin ajaminen

**Edellytykset:**
- Sijoita huonekalukuva polkuun `../imgs/home.png` (tai päivitä `imgPath`-muuttuja)
- Määritä `.env`-tiedostosi Azure OpenAI -päätepisteellä ja käyttöönotolla, kirjaudu sitten sisään komennolla `az login`

```bash
# Tee skriptistä suoritettava (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Suorita peräkkäinen työnkulku
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Tai Windowsissa:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Odotettu tulos

Työnkulku:
1. **Myyntiedustaja**: Tunnistaa huonekalut kuvasta ja antaa suosituksia
2. **Hintavälittäjä**: Lisää yksityiskohtainen hintaanalyysi budjettitasoineen ja ostosuosituksineen
3. **Tarjousagentti**: Luo muotoillun tarjousasiakirjan, jossa kaikki tiedot ovat koottuna

Lopputuloksena on kattava, ammattimainen huonekalutarjous kuvan analyysin perusteella.

### 🔍 Tilavirran ymmärtäminen

Jokainen agentti peräkkäin saa:
- **Alkuperäisen syötteen**: Käyttäjän alkuviestin (kuva + teksti)
- **Aiemmat agenttien vastaukset**: Kaikki aiemmat agenttivastaukset keskusteluhistoriassa
- **Kertyneen kontekstin**: Koko työnkulun ajan ylläpidetty tila

Tämä mahdollistaa monimutkaisen monivaiheisen prosessoinnin, jossa kukin agentti rakentaa laajan kontekstin päälle kaikista edellisistä vaiheista.
