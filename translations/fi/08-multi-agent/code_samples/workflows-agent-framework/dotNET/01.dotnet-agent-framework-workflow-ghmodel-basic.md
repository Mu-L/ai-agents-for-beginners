# 🔄 Perusagenttien työnkulut Azure OpenAI:lla (Vastaus-API) (.NET)

## 📋 Työnkulun orkestrointivinkki

Tämä muistikirja näyttää, miten rakennetaan monimutkaisia **agenttien työnkulkuja** Microsoft Agent Frameworkin .NET-versiolla ja Azure OpenAI:lla (Vastaus-API). Opit luomaan monivaiheisia liiketoimintaprosesseja, joissa tekoälyagentit tekevät yhteistyötä suorittaakseen monimutkaisia tehtäviä rakenteellisten orkestrointimallien avulla.

## 🎯 Oppimistavoitteet

### 🏗️ **Työnkulkuarkkitehtuurin perusteet**
- **Työnkulun rakentaja**: Suunnittele ja orkestroi monimutkaisia monivaiheisia tekoälyprosesseja
- **Agenttien koordinointi**: Koordinoi useita erikoistuneita agenteja työnkuluissa
- **Azure OpenAI (Vastaus-API)**: Hyödynnä Azure OpenAI Vastaus-API:a työnkuluissa
- **Visuaalinen työnkulun suunnittelu**: Luo ja visualisoi työnkulkurakenteita paremman ymmärryksen saamiseksi

### 🔄 **Prosessien orkestrointimallit**
- **Peräkkäinen prosessointi**: Ketjuta useita agenttitehtäviä loogisessa järjestyksessä
- **Tilanhallinta**: Säilytä konteksti ja tiedonkulku työnkulun vaiheiden välillä
- **Virheenkäsittely**: Toteuta vikasietoista virheenkorjausta ja työnkulun kestävyyttä
- **Suorituskyvyn optimointi**: Suunnittele tehokkaita työnkulkuja yrityskokoisiin operaatioihin

### 🏢 **Yritysten työnkulkujen sovellukset**
- **Liiketoimintaprosessien automatisointi**: Automatisoi monimutkaisia organisaation työnkulkuja
- **Sisällöntuotantoputki**: Toimitukselliset työnkulut, joissa on tarkistus- ja hyväksyntävaiheita
- **Asiakaspalvelun automatisointi**: Monivaiheinen asiakaskyselyiden ratkaisu
- **Tietojenkäsittelytyönkulut**: ETL-työnkulut tekoälyllä tehostettuun muokkaukseen

## ⚙️ Esivaatimukset ja käyttöönotto

### 📦 **Vaaditut NuGet-paketit**

Tässä työnkulun esimerkissä käytetään useita keskeisiä .NET-paketteja:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI -asetukset**

**Ympäristön asetus (.env-tiedosto):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI -käyttöoikeus:**
1. Luo Azure OpenAI -resurssi Azure-portaalissa
2. Ota käyttöön malli (esimerkiksi `gpt-4.1-mini`) ja kirjaa käyttöönoton nimi
3. Kirjaudu sisään `az login` -komennolla ja määritä ympäristömuuttujat yllä esitetyllä tavalla

### 🏗️ **Työnkulkuarkkitehtuurin yleiskatsaus**

```mermaid
graph TD
    A[Työnkulun rakentaja] --> B[Agenttirekisteri]
    B --> C[Työnkulun suoritusmoottori]
    C --> D[Agent 1: Sisällön luoja]
    C --> E[Agent 2: Sisällön tarkastaja] 
    D --> F[Työnkulun tulokset]
    E --> F
    G[Azure OpenAI (Vastaus-API)] --> D
    G --> E
```

**Keskeiset komponentit:**
- **WorkflowBuilder**: Pääorkestrointimoottori työnkulkujen suunnitteluun
- **AIAgent**: Yksittäiset erikoistuneet agentit tiettyine kykyineen
- **Azure OpenAI Client**: Azure OpenAI Vastaus-API:n integrointi
- **Execution Context**: Hallinnoi tilaa ja tiedonkulkua työnkulun vaiheiden välillä

## 💻 Koodin suorittaminen

Täydellinen toteutus löytyy tiedostosta `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Tämä tiedosto näyttää:

1. **Ympäristön konfigurointi** - Azure OpenAI -asetusten lataaminen `.env`-tiedostosta
2. **Azure OpenAI -asiakkaan asetus** - Asiakkaan määrittely käyttämään Azure OpenAI Vastaus-API:a
3. **Agenttien luominen** - Erikoistuneiden agenttien määrittely (Vastaanotto ja Concierge)
4. **Työnkulun rakentaja** - Moniagenttisen työnkulun luominen peräkkäisellä prosessoinnilla
5. **Työnkulun suoritus** - Työnkulun ajo virtaviivaisilla tuloksilla

### 🚀 Esimerkin suoritus

```bash
# Tee skriptistä suoritettava (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Suorita työnkulku
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Tai Windowsilla:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Odotettu tulos

Työnkulku:
1. Ottaa vastaan matkakohdepyynnön ("Haluaisin mennä Pariisiin")
2. Vastaanotto-agentti antaa alkuperäisen suosituksen
3. Concierge-agentti tarkistaa ja hienosäätää suositusta
4. Lopputuloksena näytetään koko keskusteluketju
