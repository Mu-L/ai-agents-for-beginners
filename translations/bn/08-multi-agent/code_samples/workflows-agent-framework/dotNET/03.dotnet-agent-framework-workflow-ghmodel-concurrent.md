# ⚡ Azure OpenAI (Responses API) সহ সমকালীন এজেন্ট কর্মপ্রবাহ (.NET)

## 📋 উচ্চ-প্রদর্শন সমান্তরাল প্রক্রিয়াকরণ টিউটোরিয়াল

এই নোটবুকটি Microsoft Agent Framework for .NET এবং Azure OpenAI (Responses API) ব্যবহার করে **সমকালীন কর্মপ্রবাহের প্যাটার্ন** প্রদর্শন করে। আপনি শিখবেন কিভাবে একাধিক AI এজেন্টকে একযোগে চালিয়ে সর্বোচ্চ থ্রুপুটের জন্য উচ্চ-প্রদর্শন সমান্তরাল প্রক্রিয়াকরণ কর্মপ্রবাহ তৈরি করবেন, সাথেই সমন্বয় এবং ডেটা সামঞ্জস্যতা বজায় রাখবেন।

## 🎯 শেখার লক্ষ্যসমূহ

### 🚀 **সমকালীন প্রক্রিয়াকরণের মৌলিক বিষয়সমূহ**
- **সমান্তরাল এজেন্ট কার্যকরী**: সর্বোচ্চ কর্মক্ষমতার জন্য একাধিক AI এজেন্টকে একসঙ্গে চালানো
- **Async/Await প্যাটার্ন**: দক্ষ সমকালীনতার জন্য .NET এর async প্রোগ্রামিং মডেল ব্যবহার
- **Azure OpenAI (Responses API)**: Azure OpenAI Responses API এর একাধিক সমকালীন কল সমন্বয় করা
- **রিসোর্স ম্যানেজমেন্ট**: সমকালীন অপারেশন জুড়ে AI মডেল রিসোর্স দক্ষভাবে পরিচালনা করা

### 🏗️ **উন্নত সমকালীন স্থাপত্য**
- **টাস্ক-ভিত্তিক সমান্তরালতা**: সর্বোত্তম সমকালীন কার্যকরীর জন্য .NET Task Parallel Library ব্যবহার
- **সমন্বয় প্যাটার্ন**: প্রতিদ্বন্দ্বিতামূলক পরিস্থিতি এড়িয়ে সমকালীন এজেন্টদের সমন্বয় করা
- **লোড ব্যালান্সিং**: উপলব্ধ সমান্তরাল প্রক্রিয়াকরণ ক্ষমতা জুড়ে কার্য সুষ্ঠুভাবে বিতরণ
- **ফল্ট সহনশীলতা**: সমগ্র কর্মপ্রবাহ বন্ধ না করে পৃথক এজেন্ট ব্যর্থতা পরিচালনা করা

### 🏢 **এন্টারপ্রাইজ সমকালীন অ্যাপ্লিকেশন**
- **উচ্চ-আয়তনের ডকুমেন্ট প্রক্রিয়াকরণ**: একযোগে একাধিক ডকুমেন্ট প্রক্রিয়াকরণ
- **রিয়েল-টাইম বিষয়বস্তু বিশ্লেষণ**: আগত ডেটা স্ট্রিমের সমকালীন বিশ্লেষণ
- **ব্যাচ প্রক্রিয়াকরণ অপ্টিমাইজেশন**: বৃহৎমাত্রার ডেটা প্রক্রিয়াকরণের জন্য সর্বোচ্চ থ্রুপুট
- **মাল্টি-মোডাল বিশ্লেষণ**: বিভিন্ন বিষয়বস্তু প্রকার এবং ফরম্যাটের সমান্তরাল প্রক্রিয়াকরণ

- **Task Parallel Library**: .NET এর বিল্ট-ইন সমকালীন অপারেশনের সাপোর্ট
- **Agent Pool**: সমান্তরাল প্রক্রিয়াকরণের জন্য একাধিক এজেন্ট ইনস্ট্যান্স
- **Result Aggregation**: সমকালীন এজেন্ট ফলাফল সমন্বয় ও একত্রিকরণ
- **Synchronization Points**: সমকালীন অপারেশনের মাধ্যমে ডেটা সামঞ্জস্যতা নিশ্চিতকরণ

### 🏗️ **কর্মপ্রবাহ স্থাপত্য**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**মূল উপাদানসমূহ:**

1. **ConcurrentStartExecutor**: ব্যবহারকারীর অনুরোধ সকল এজেন্টকে একসাথে প্রচার করে
2. **Researcher Agent**: গন্তব্য ও আকর্ষণসমূহ সমান্তরালভাবে বিশ্লেষণ করে
3. **Planner Agent**: বিস্তারিত ভ্রমণ পরিকল্পনা সমান্তরালভাবে প্রস্তুত করে
4. **ConcurrentAggregationExecutor**: দুই এজেন্টের ফলাফল সংগ্রহ ও একত্রিত করে

### 🎯 **ফ্যান-আউট/ফ্যান-ইন প্যাটার্ন**

এই কর্মপ্রবাহ ক্লাসিক **ফ্যান-আউট/ফ্যান-ইন** প্যাটার্ন প্রদর্শন করে:
- **ফ্যান-আউট**: এক ইনপুট মেসেজ একযোগে একাধিক এজেন্টে প্রেরণ করে
- **সমকালীন প্রক্রিয়াকরণ**: একাধিক এজেন্ট সমান্তরালভাবে একই কাজ সম্পাদন করে
- **ফ্যান-ইন**: সকল এজেন্ট থেকে ফলাফল সংগ্রহ করে একক আউটপুটে একত্রিত করা হয়

// Create additional specialized agents

    name: "Budget-Agent", instructions: "Calculate travel costs...");

    .AddFanOutEdge(startExecutor, targets: [researcherAgent, plannerAgent, budgetAgent])
    .AddFanInEdge(aggregationExecutor, sources: [researcherAgent, plannerAgent, budgetAgent])
    .WithOutputFrom(aggregationExecutor)
    .Build();

// Update aggregation count

- **কোড রিভিউ**: একাধিক পর্যালোচক বিভিন্ন দৃষ্টিকোণ থেকে কোড বিশ্লেষণ করে
- **মার্কেট রিসার্চ**: বিভিন্ন বাজার বিভাগ সমান্তরাল বিশ্লেষণ
- **ডকুমেন্ট প্রক্রিয়াকরণ**: সমকালীন তথ্য আহরণ, বিশ্লেষণ ও যাচাই
- **বহু-মত বিশ্লেষণ**: একই ইনপুটের উপর বিভিন্ন দৃষ্টিভঙ্গি পাওয়া

### 🔍 কাস্টম এক্সিকিউটর বোঝা

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` বাস্তবায়ন করে স্ট্রিং ইনপুট গ্রহণের জন্য
- সংযুক্ত সকল এজেন্টকে মেসেজ প্রচার করে
- সমকালীন প্রক্রিয়াকরণের জন্য `TurnToken` প্রেরণ করে

**ConcurrentAggregationExecutor:**
- `IMessageHandler<ChatMessage>` বাস্তবায়ন করে এজেন্ট প্রতিক্রিয়া গ্রহণের জন্য
- থ্রেড-নিরাপদ পদ্ধতিতে মেসেজ সংগ্রহ করে
- প্রত্যাশিত সকল প্রতিক্রিয়া এলে একত্রিকরণ করে
- `context.YieldOutputAsync()` ব্যবহার করে চূড়ান্ত আউটপুট প্রদান করে

### ⚡ কর্মক্ষমতা সুবিধাসমূহ

**সমকালীন বনাম ধারাবাহিক:**
- ধারাবাহিক: Agent1 (৩০ সেকেন্ড) → Agent2 (৩০ সেকেন্ড) = **মোট ৬০ সেকেন্ড**
- সমকালীন: Agent1 (৩০ সেকেন্ড) || Agent2 (৩০ সেকেন্ড) = **মোট ৩০ সেকেন্ড**

**থ্রুপুট উন্নতি**: ন সমকালীন এজেন্টের জন্য N গুণ দ্রুত (কাজের পরিমাণ ও রিসোর্স অনুযায়ী)

### 🛡️ ত্রুটি পরিচালনা

কর্মপ্রবাহ পৃথক এজেন্ট ব্যর্থতা মার্জিতভাবে পরিচালনা করে:
- এক এজেন্ট ব্যর্থ হলে, অন্যান্য এজেন্ট কাজ চালিয়ে যায়
- একত্রিকরণকারী টাইমআউট যুক্ত করতে পারে
- প্রয়োজনে আংশিক ফলাফল ফেরত দেয়া যেতে পারে
