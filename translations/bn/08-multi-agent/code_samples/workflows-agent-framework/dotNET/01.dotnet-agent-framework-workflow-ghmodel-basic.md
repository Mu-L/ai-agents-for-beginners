# 🔄 Azure OpenAI (Responses API) সহ বেসিক এজেন্ট ওয়ার্কফ্লো (.NET)

## 📋 ওয়ার্কফ্লো অর্কেস্ট্রেশন টিউটোরিয়াল

এই নোটবুকটি দেখায় কীভাবে Microsoft Agent Framework for .NET এবং Azure OpenAI (Responses API) ব্যবহার করে উন্নত **এজেন্ট ওয়ার্কফ্লো** তৈরি করতে হয়। আপনি শিখবেন কীভাবে মাল্টি-স্টেপ ব্যবসায়িক প্রক্রিয়া তৈরি করতে হয় যেখানে AI এজেন্টরা কাঠামোবদ্ধ অর্কেস্ট্রেশন প্যাটার্নের মাধ্যমে জটিল কাজ সমাধানে একসাথে কাজ করে।

## 🎯 শেখার লক্ষ্যসমূহ

### 🏗️ **ওয়ার্কফ্লো আর্কিটেকচার মৌলিক বিষয়সমূহ**
- **ওয়ার্কফ্লো বিল্ডার**: জটিল মাল্টি-স্টেপ AI প্রক্রিয়া ডিজাইন ও অর্কেস্ট্রেট করা
- **এজেন্ট সমন্বয়**: ওয়ার্কফ্লোতে একাধিক বিশেষায়িত এজেন্টকে সমন্বয় করা
- **Azure OpenAI (Responses API)**: ওয়ার্কফ্লোতে Azure OpenAI Responses API ব্যবহার করা
- **ভিজ্যুয়াল ওয়ার্কফ্লো ডিজাইন**: ভাল বোঝার জন্য ওয়ার্কফ্লো কাঠামো তৈরি ও ভিজ্যুয়ালাইজ করা

### 🔄 **প্রসেস অর্কেস্ট্রেশন প্যাটার্নসমূহ**
- **ক্রমাগত প্রক্রিয়াকরণ**: যুক্তিসঙ্গত ক্রমে একাধিক এজেন্ট টাস্ককে চেইন করা
- **স্টেট ব্যবস্থাপনা**: ওয়ার্কফ্লো পর্যায়সমূহ জুড়ে প্রসঙ্গ এবং ডেটার প্রবাহ বজায় রাখা
- **ত্রুটি হ্যান্ডলিং**: দৃঢ় ত্রুটি পুনরুদ্ধার ও ওয়ার্কফ্লো স্থিতিস্থাপকতা বাস্তবায়ন
- **কর্মক্ষমতা অপ্টিমাইজেশন**: এন্টারপ্রাইজ স্কেলের অপারেশনগুলোর জন্য দক্ষ ওয়ার্কফ্লো ডিজাইন

### 🏢 **এন্টারপ্রাইজ ওয়ার্কফ্লো অ্যাপ্লিকেশনসমূহ**
- **ব্যবসায়িক প্রক্রিয়া অটোমেশন**: জটিল সাংগঠনিক ওয়ার্কফ্লো গুলোর অটোমেশন
- **কন্টেন্ট প্রোডাকশন পাইপলাইন**: পর্যালোচনা ও অনুমোদন ধাপসহ সম্পাদকীয় ওয়ার্কফ্লো
- **গ্রাহক সেবা অটোমেশন**: দীর্ঘ ধাপের গ্রাহক অনুসন্ধান সমাধান
- **ডেটা প্রসেসিং ওয়ার্কফ্লো**: AI-চালিত রূপান্তর সহ ETL ওয়ার্কফ্লো

## ⚙️ প্রয়োজনীয়তা ও সেটআপ

### 📦 **প্রয়োজনীয় NuGet প্যাকেজসমূহ**

এই ওয়ার্কফ্লো প্রদর্শনীতে কয়েকটি গুরুত্বপূর্ণ .NET প্যাকেজ ব্যবহার করা হয়েছে:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI কনফিগারেশন**

**পরিবেশ সেটআপ (.env ফাইল):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI অ্যাক্সেস:**
1. Azure পোর্টালে একটি Azure OpenAI রিসোর্স তৈরি করুন
2. একটি মডেল ডিপ্লয় করুন (উদাহরণস্বরূপ, `gpt-4.1-mini`) এবং ডিপ্লয়মেন্টের নাম নোট করুন
3. `az login` এর মাধ্যমে সাইন ইন করুন এবং উপরে দেওয়া মতো পরিবেশ ভেরিয়েবল কনফিগার করুন

### 🏗️ **ওয়ার্কফ্লো আর্কিটেকচারের সংক্ষিপ্ত বিবরণ**

```mermaid
graph TD
    A[ওয়ার্কফ্লো বিল্ডার] --> B[এজেন্ট রেজিস্ট্রি]
    B --> C[ওয়ার্কফ্লো এক্সিকিউশন ইঞ্জিন]
    C --> D[Agent 1: বিষয়বস্তু নির্মাতা
    C --> E[Agent 2: বিষয়বস্তু পর্যালোচক 
    D --> F[ওয়ার্কফ্লো ফলাফল]
    E --> F
    G[আজুর ওপেনএআই (রেসপন্সেস এপিআই)] --> D
    G --> E
```

**মূল উপাদানসমূহ:**
- **WorkflowBuilder**: ওয়ার্কফ্লো ডিজাইনের জন্য প্রধান অর্কেস্ট্রেশন ইঞ্জিন
- **AIAgent**: নির্দিষ্ট দক্ষতাসম্পন্ন পৃথক বিশেষায়িত এজেন্ট
- **Azure OpenAI ক্লায়েন্ট**: Azure OpenAI Responses API ইন্টিগ্রেশন
- **Execution Context**: ওয়ার্কফ্লো পর্যায়গুলোতে স্টেট এবং ডেটা প্রবাহ পরিচালনা করে

## 🏢 **এন্টারপ্রাইজের সুবিধাসমূহ**

### 🎯 **বিশ্বাসযোগ্যতা ও স্কেলযোগ্যতা**
- **নির্ধারিত এবং পুনরাবৃত্তিমূলক এক্সিকিউশন**: ধারাবাহিক, পুনরাবৃত্তিযোগ্য ওয়ার্কফ্লো ফলাফল
- **ত্রুটি পুনরুদ্ধার**: যে কোন ওয়ার্কফ্লো পর্যায়ে ব্যর্থতা gracefully হ্যান্ডলিং
- **পারফরম্যান্স মনিটরিং**: এক্সিকিউশন মেট্রিকস এবং অপ্টিমাইজেশনের সুযোগ ট্র্যাক করা
- **রিসোর্স ম্যানেজমেন্ট**: AI মডেলের রিসোর্সের দক্ষ বরাদ্দ ও ব্যবহার

### 🔒 **সিকিউরিটি ও কমপ্লায়েন্স**
- **সুরক্ষিত প্রমাণীকরণ**: `az login` (AzureCliCredential) এর মাধ্যমে Microsoft Entra ID প্রমাণীকরণ
- **অডিট ট্রেইলস**: ওয়ার্কফ্লো এক্সিকিউশন এবং সিদ্ধান্তের সম্পূর্ণ লগিং
- **অ্যাক্সেস নিয়ন্ত্রণ**: ওয়ার্কফ্লো এক্সিকিউশন ও মনিটরিংয়ের জন্য সূক্ষ্ম অনুমতি
- **ডেটা প্রাইভেসি**: ওয়ার্কফ্লো জুড়ে সংবেদনশীল তথ্যের সুরক্ষিত হ্যান্ডলিং

### 📊 **দক্ষতা পর্যবেক্ষণ ও ব্যবস্থাপনা**
- **ভিজ্যুয়াল ওয়ার্কফ্লো ডিজাইন**: প্রসেস ফ্লো ও নির্ভরশীলতার স্পষ্ট উপস্থাপনা
- **এক্সিকিউশন মনিটরিং**: ওয়ার্কফ্লো অগ্রগতি ও কর্মক্ষমতার রিয়েল-টাইম ট্র্যাকিং
- **ত্রুটি রিপোর্টিং**: বিশদ ত্রুটি বিশ্লেষণ ও ডিবাগ ক্ষমতা
- **পারফরম্যান্স অ্যানালিটিক্স**: অপ্টিমাইজেশন ও ক্যাপাসিটি পরিকল্পনার জন্য মেট্রিক্স

চলুন আপনার প্রথম এন্টারপ্রাইজ-রেডি AI ওয়ার্কফ্লো তৈরি করি! 🚀

## 💻 কোড চালানো

সম্পূর্ণ ইমপ্লিমেন্টেশন `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` ফাইলে রয়েছে। এই ফাইলটি প্রদর্শন করে:

1. **পরিবেশ কনফিগারেশন** - `.env` ফাইল থেকে Azure OpenAI কনফিগারেশন লোড করা
2. **Azure OpenAI ক্লায়েন্ট সেটআপ** - ক্লায়েন্টকে Azure OpenAI Responses API ব্যবহার করতে কনফিগার করা
3. **এজেন্ট সৃষ্টি** - বিশেষায়িত এজেন্ট সংজ্ঞায়িত করা (Front Desk এবং Concierge)
4. **ওয়ার্কফ্লো বিল্ডার** - ক্রমান্বয়ে প্রক্রিয়াকরণ সহ মাল্টি-এজেন্ট ওয়ার্কফ্লো তৈরি করা
5. **ওয়ার্কফ্লো এক্সিকিউশন** - স্ট্রিমিং ফলাফলের সঙ্গে ওয়ার্কফ্লো চালানো

### 🚀 উদাহরণ চালানো

```bash
# স্ক্রিপ্টটি চালনীয় করুন (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# ওয়ার্কফ্লো চালান
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

অথবা Windows এ:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 প্রত্যাশিত আউটপুট

ওয়ার্কফ্লো হবে:
1. আপনার ভ্রমণ গন্তব্য অনুরোধ গ্রহণ করবে ("আমি প্যারিস যেতে চাই")
2. Front Desk এজেন্ট একটি প্রাথমিক সুপারিশ দেবে
3. Concierge এজেন্ট সুপারিশ পর্যালোচনা এবং সংশোধন করবে
4. চূড়ান্ত আউটপুট সম্পূর্ণ কথোপকথন স্ট্রিম দেখাবে

### 🔧 কাস্টমাইজেশন

আপনি ওয়ার্কফ্লো কাস্টমাইজ করতে পারেন:
- এজেন্টের নির্দেশনা পরিবর্তন করে তাদের আচরণ পরিবর্তন করা
- আরো এজেন্ট যুক্ত করে জটিল মাল্টি-স্টেপ ওয়ার্কফ্লো তৈরি করা
- বিভিন্ন পরিস্থিতি পরীক্ষা করার জন্য ব্যবহারকারী বার্তা পরিবর্তন করা
- বিভিন্ন এক্সিকিউশন প্যাটার্ন তৈরি করতে ওয়ার্কফ্লো এজগুলো সমন্বয় করা
