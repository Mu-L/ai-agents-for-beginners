# 🔄 Azure OpenAI (Responses API) ( .NET ) এর সাথে মৌলিক এজেন্ট ওয়ার্কফ্লো

## 📋 ওয়ার্কফ্লো অর্কেস্ট্রেশন টিউটোরিয়াল

এই নোটবুকটি প্রদর্শন করে কীভাবে Microsoft Agent Framework for .NET এবং Azure OpenAI (Responses API) ব্যবহার করে জটিল **এজেন্ট ওয়ার্কফ্লো** তৈরি করতে হয়। আপনি শিখবেন কীভাবে বহু-ধাপের ব্যবসায়িক প্রক্রিয়া তৈরি করতে হয় যেখানে AI এজেন্টরা গঠনমূলক অর্কেস্ট্রেশন প্যাটার্নের মাধ্যমে সহযোগিতা করে জটিল কাজ সম্পন্ন করে।

## 🎯 শেখার উদ্দেশ্যসমূহ

### 🏗️ **ওয়ার্কফ্লো স্থাপত্যের ভিত্তি**
- **ওয়ার্কফ্লো বিল্ডার**: জটিল বহু-ধাপ AI প্রক্রিয়া ডিজাইন এবং অর্কেস্ট্রেট করা
- **এজেন্ট সমন্বয়**: ওয়ার্কফ্লোতে একাধিক বিশেষায়িত এজেন্টদের সমন্বয় করা
- **Azure OpenAI (Responses API)**: ওয়ার্কফ্লোতে Azure OpenAI Responses API ব্যবহার করা
- **ভিজ্যুয়াল ওয়ার্কফ্লো ডিজাইন**: ভাল বোঝার জন্য ওয়ার্কফ্লো কাঠামো তৈরি ও চাক্ষুষ করা

### 🔄 **প্রক্রিয়া অর্কেস্ট্রেশন প্যাটার্ন**
- **ক্রমাগত প্রক্রিয়াকরণ**: যৌক্তিক ক্রমে একাধিক এজেন্ট টাস্ক সংযোগ করা
- **স্টেট ম্যানেজমেন্ট**: ওয়ার্কফ্লোর ধাপ জুড়ে প্রসঙ্গ এবং ডেটা প্রবাহ বজায় রাখা
- **ত্রুটি হ্যান্ডলিং**: দৃঢ় ত্রুটি পুনরুদ্ধার এবং ওয়ার্কফ্লো স্থিতিস্থাপকতা প্রয়োগ করা
- **সংশ্লেষণ অপটিমাইজেশন**: এন্টারপ্রাইজ-মাপের অপারেশনের জন্য দক্ষ ওয়ার্কফ্লো ডিজাইন করা

### 🏢 **এন্টারপ্রাইজ ওয়ার্কফ্লো অ্যাপ্লিকেশন**
- **ব্যবসায়িক প্রসেস অটোমেশন**: জটিল সংগঠনিক ওয়ার্কফ্লো স্বয়ংক্রিয়করণ
- **কন্টেন্ট উৎপাদন পাইপলাইন**: পর্যালোচনা এবং অনুমোদন ধাপ সহ সম্পাদনাগত ওয়ার্কফ্লো
- **গ্রাহক সেবা অটোমেশন**: বহু-ধাপ গ্রাহক অনুসন্ধান সমাধান
- **ডেটা প্রক্রিয়াকরণ ওয়ার্কফ্লো**: AI-চালিত রূপান্তরসহ ETL ওয়ার্কফ্লো

### 📦 **প্রয়োজনীয় NuGet প্যাকেজসমূহ**

এই ওয়ার্কফ্লো ডেমোতে কয়েকটি গুরুত্বপূর্ণ .NET প্যাকেজ ব্যবহার করা হয়েছে:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI কনফিগারেশন**

**পরিবেশ সেটআপ (.env ফাইল):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Azure OpenAI অ্যাক্সেস:**
1. Azure পোর্টালে একটি Azure OpenAI রিসোর্স তৈরি করুন
2. একটি মডেল ডিপ্লয় করুন (উদাহরণস্বরূপ, `gpt-5-mini`) এবং ডিপ্লয়মেন্ট নাম নোট করুন
3. `az login` দিয়ে লগইন করুন এবং উপরে দেখানো মতো পরিবেশ পরিবর্তনশীলগুলি কনফিগার করুন

### 🏗️ **ওয়ার্কফ্লো স্থাপত্য ওভারভিউ**

```mermaid
graph TD
    A[ওয়ার্কফ্লো বিল্ডার] --> B[এজেন্ট রেজিস্ট্রি]
    B --> C[ওয়ার্কফ্লো এক্সিকিউশন ইঞ্জিন]
    C --> D[Agent 1: কনটেন্ট জেনারেটর]
    C --> E[Agent 2: কনটেন্ট রিভিউয়ার] 
    D --> F[ওয়ার্কফ্লো ফলাফল]
    E --> F
    G[আজুর ওপেনএআই (রেসপন্সেস API)] --> D
    G --> E
```

**মূল উপাদানসমূহ:**
- **WorkflowBuilder**: ওয়ার্কফ্লো ডিজাইন করার প্রধান অর্কেস্ট্রেশন ইঞ্জিন
- **AIAgent**: নির্দিষ্ট ক্ষমতা সহ ব্যক্তিগতকৃত বিশেষায়িত এজেন্ট
- **Azure OpenAI Client**: Azure OpenAI Responses API ইন্টিগ্রেশন
- **Execution Context**: ওয়ার্কফ্লোর ধাপগুলোর মধ্যে স্টেট এবং ডেটা প্রবাহ পরিচালনা করে

## 🏢 **এন্টারপ্রাইজ সুবিধাসমূহ**

### 🎯 **বিশ্বস্ততা এবং স্কেলযোগ্যতা**
- **নির্ধারিত এক্সিকিউশন**: ধারাবাহিক, পুনরাবৃত্তিযোগ্য ওয়ার্কফ্লো ফলাফল
- **ত্রুটি পুনরুদ্ধার**: ওয়ার্কফ্লোর যেকোনো ধাপে ব্যর্থতা সুষ্ঠুভাবে পরিচালনা করা
- **পারফরম্যান্স মনিটরিং**: এক্সিকিউশন মেট্রিক্স এবং অপ্টিমাইজেশন সুযোগ নজর রাখা
- **রিসোর্স ম্যানেজমেন্ট**: AI মডেল রিসোর্সের দক্ষ বরাদ্দ ও ব্যবহার

### 🔒 **সিকিউরিটি ও কমপ্লায়েন্স**
- **নিরাপদ প্রমাণীকরণ**: Microsoft Entra ID এর মাধ্যমে `az login` (AzureCliCredential)
- **অডিট ট্রেইলস**: ওয়ার্কফ্লো এক্সিকিউশন এবং সিদ্ধান্ত গুলোর সম্পূর্ণ লগিং
- **অ্যাক্সেস কন্ট্রোল**: ওয়ার্কফ্লো এক্সিকিউশন এবং মনিটরিং এর জন্য সূক্ষ্ম পারমিশন
- **ডেটা প্রাইভেসি**: ওয়ার্কফ্লো জুড়ে সংবেদনশীল তথ্যের নিরাপদ হ্যান্ডলিং

### 📊 **অবজারভেবিলিটি এবং ব্যবস্থাপনা**
- **ভিজ্যুয়াল ওয়ার্কফ্লো ডিজাইন**: প্রক্রিয়া প্রবাহ এবং নির্ভরশীলতার স্পষ্ট উপস্থাপনা
- **এক্সিকিউশন মনিটরিং**: ওয়ার্কফ্লোর অগ্রগতি এবং কর্মক্ষমতার রিয়েল-টাইম ট্র্যাকিং
- **ত্রুটি রিপোর্টিং**: বিস্তারিত ত্রুটি বিশ্লেষণ এবং ডিবাগিং ক্ষমতা
- **পারফরম্যান্স অ্যানালিটিক্স**: অপ্টিমাইজেশন এবং ক্যাপাসিটি প্ল্যানিং এর মেট্রিক্স

চলুন আপনার প্রথম এন্টারপ্রাইজ-রেডি AI ওয়ার্কফ্লো তৈরি করি! 🚀

## 💻 কোড চালানো

সম্পূর্ণ বাস্তবায়ন `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` ফাইলে পাওয়া যাবে। এই ফাইলটি প্রদর্শন করে:

1. **পরিবেশ কনফিগারেশন** - `.env` ফাইলে থেকে Azure OpenAI কনফিগারেশন লোড করা
2. **Azure OpenAI ক্লায়েন্ট সেটআপ** - Azure OpenAI Responses API ব্যবহারের জন্য ক্লায়েন্ট কনফিগার করা
3. **এজেন্ট সৃষ্টিকরণ** - বিশেষায়িত এজেন্ট (ফ্রন্ট ডেস্ক এবং কনসিয়ার্জ) সংজ্ঞায়িত করা
4. **ওয়ার্কফ্লো বিল্ডার** - ক্রমানুসার প্রক্রিয়াকরণ সহ বহু-এজেন্ট ওয়ার্কফ্লো তৈরি করা
5. **ওয়ার্কফ্লো এক্সিকিউশন** - স্ট্রিমিং ফলাফল সহ ওয়ার্কফ্লো চালানো

### 🚀 উদাহরণ চালানো

```bash
# স্ক্রিপ্টটি executable করুন (ইউনিক্স/লিনাক্স/ম্যাকওএস)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# ওয়ার্কফ্লো চালান

```

অথবা উইন্ডোজে:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 প্রত্যাশিত আউটপুট

ওয়ার্কফ্লো করবে:
1. আপনার ভ্রমণের গন্তব্য অনুরোধ গ্রহণ করবে ("আমি প্যারিস যেতে চাই")
2. ফ্রন্ট ডেস্ক এজেন্ট একটি প্রাথমিক সুপারিশ প্রদান করবে
3. কনসিয়ার্জ এজেন্ট সুপারিশ পর্যালোচনা ও পরিমার্জন করবে
4. চূড়ান্ত আউটপুট সম্পূর্ণ কথোপকথন স্ট্রিম প্রদর্শন করবে

### 🔧 কাস্টমাইজেশন

আপনি ওয়ার্কফ্লো কাস্টমাইজ করতে পারেন:
- এজেন্ট নির্দেশাবলী পরিবর্তন করে তাঁদের আচরণ পরিবর্তন করা
- আরো এজেন্ট যোগ করে জটিল বহু-ধাপের ওয়ার্কফ্লো তৈরি করা
- ব্যবহারকারী মেসেজ পরিবর্তন করে বিভিন্ন পরিস্থিতি পরীক্ষা করা
- ওয়ার্কফ্লো এজ পরিবর্তন করে ভিন্ন এক্সিকিউশন প্যাটার্ন তৈরি করা
