# ⏩ Sekwencyjne przepływy pracy Agentów z Azure OpenAI (Responses API) (.NET)

## 📋 Zaawansowany samouczek przetwarzania sekwencyjnego

Ten notatnik demonstruje **sekwencyjne wzorce przepływu pracy** z użyciem Microsoft Agent Framework dla .NET oraz Azure OpenAI (Responses API). Nauczysz się, jak budować zaawansowane, krok po kroku potoki przetwarzania, w których agenci wykonują zadania w określonym porządku, gdzie każdy etap opiera się na wynikach poprzedniego.

## 🎯 Cele nauki

### 🔄 **Architektura przetwarzania sekwencyjnego**
- **Projektowanie liniowego przepływu pracy**: Tworzenie krok po kroku potoków przetwarzania z jasnymi zależnościami
- **Zarządzanie stanem**: Utrzymanie kontekstu i przepływu danych między etapami sekwencyjnego przepływu pracy
- **Azure OpenAI (Responses API)**: Wykorzystanie modeli Azure OpenAI w wieloetapowych przepływach .NET
- **Wzorce potoków dla przedsiębiorstw**: Budowa systemów przetwarzania sekwencyjnego gotowych do produkcji

### 🏗️ **Zaawansowane wzorce sekwencyjne**
- **Przetwarzanie typu Stage-Gate**: Implementacja punktów kontrolnych walidacji pomiędzy etapami przepływu
- **Zachowanie kontekstu**: Utrzymywanie stanu i zgromadzonej wiedzy na wszystkich etapach
- **Propagacja błędów**: Eleganckie zarządzanie niepowodzeniami w łańcuchach przetwarzania sekwencyjnego
- **Optymalizacja wydajności**: Wydajne przetwarzanie sekwencyjne z minimalnym narzutem

### 🏢 **Sekwencyjne aplikacje dla przedsiębiorstw**
- **Potok przetwarzania dokumentów**: Wieloetapowa analiza, transformacja i walidacja dokumentów
- **Przepływy zapewniania jakości**: Sekwencyjny przegląd, walidacja i procesy zatwierdzania
- **Potok produkcji treści**: Badania → Pisanie → Redakcja → Przegląd → Publikacja
- **Automatyzacja procesów biznesowych**: Wieloetapowe przepływy biznesowe z jasno określonymi zależnościami etapów

## ⚙️ Wymagania wstępne i konfiguracja

### 📦 **Wymagane pakiety NuGet**

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
```

### 🔑 **Konfiguracja Azure OpenAI**

**Konfiguracja środowiska (plik .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Zarządzanie konfiguracją:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **Architektura przepływu pracy sekwencyjnego**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Agent Analizy]
    B --> C[Punkt kontrolny 1]
    C --> D[Stage 2: Agent Przetwarzania]
    D --> E[Punkt kontrolny 2]
    E --> F[Stage 3: Agent Walidacji]
    F --> G[Ostateczny wynik]
    
    H[Kontekst stanu] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API odpowiedzi)] --> B
    I --> D
    I --> F
```

**Kluczowe komponenty:**
- **Sekwencyjni agenci**: Specjalizowani agenci dla każdego etapu przetwarzania
- **Kontekst stanu**: Utrzymuje zgromadzone dane i decyzje na wszystkich etapach
- **Punkty kontrolne**: Punkty walidacji między etapami zapewniające jakość i spójność
- **Klient Azure OpenAI**: Spójny dostęp do modelu AI na wszystkich etapach przepływu pracy

## 💻 Uruchamianie kodu

Kompletną implementację znajdziesz w `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Plik ten demonstruje **trójetapowy przepływ analizy mebli**:

1. **Etap 1 - Agent sprzedaży**: Analizuje zdjęcia mebli i dostarcza sugestie zakupowe
2. **Etap 2 - Agent cenowy**: Podaje szczegółowe analizy cen i opcje budżetowe
3. **Etap 3 - Agent ofertowy**: Generuje profesjonalny dokument oferty w formacie Markdown

### 🏗️ **Architektura przepływu pracy**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Każdy agent:
- Otrzymuje wyjście z poprzedniego etapu jako kontekst
- Opiera się na poprzedniej analizie poprzez specjalistyczną wiedzę
- Utrzymuje ciągłość przepływu pracy poprzez zarządzanie stanem

### 🚀 Uruchamianie przykładu

**Wymagania wstępne:**
- Umieść zdjęcie mebli w `../imgs/home.png` (lub zaktualizuj zmienną `imgPath`)
- Skonfiguruj plik `.env` z punktem końcowym Azure OpenAI i wdrożeniem, następnie zaloguj się za pomocą `az login`

```bash
# Uczyń skrypt wykonywalnym (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Uruchom sekwencyjny przepływ pracy
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Lub na Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Oczekiwany wynik

Przepływ pracy:
1. **Agent sprzedaży**: Zidentyfikuje meble na obrazie i dostarczy rekomendacje
2. **Agent cenowy**: Doda szczegółową analizę cen z poziomami budżetu i rekomendacjami zakupowymi
3. **Agent ofertowy**: Wygeneruje sformatowany dokument oferty zawierający całą zsyntetyzowaną informację

Końcowym wynikiem będzie kompleksowa, profesjonalna oferta mebli oparta na analizie obrazu.

Każdy agent w sekwencji otrzymuje:
- **Oryginalne wejście**: Początkową wiadomość użytkownika (obraz + tekst)
- **Wyjścia poprzednich agentów**: Wszystkie poprzednie odpowiedzi agentów w historii konwersacji

To umożliwia zaawansowane wieloetapowe przetwarzanie, gdzie każdy agent buduje na podstawie pełnego kontekstu ze wszystkich wcześniejszych etapów.
