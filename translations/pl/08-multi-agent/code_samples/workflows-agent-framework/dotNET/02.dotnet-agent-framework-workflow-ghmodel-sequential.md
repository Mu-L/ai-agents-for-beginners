# ⏩ Sekwencyjne przepływy pracy agentów z Azure OpenAI (Responses API) (.NET)

## 📋 Zaawansowany samouczek przetwarzania sekwencyjnego

Ten notatnik demonstruje **wzorce sekwencyjnych przepływów pracy** wykorzystujące Microsoft Agent Framework dla .NET oraz Azure OpenAI (Responses API). Nauczysz się jak budować zaawansowane, krok po kroku, potoki przetwarzania, gdzie agenci wykonują się w określonej kolejności, a każdy etap opiera się na wynikach poprzedniego.

## 🎯 Cele nauki

- **Liniowy projekt przepływu pracy**: Tworzenie krokowych potoków przetwarzania z jasnymi zależnościami

- **Azure OpenAI (Responses API)**: Wykorzystanie modeli Azure OpenAI w wieloetapowych przepływach pracy .NET
- **Wzorce korporacyjnych potoków**: Budowanie gotowych do produkcji sekwencyjnych systemów przetwarzania

### 🏗️ **Zaawansowane wzorce sekwencyjne**
- **Przetwarzanie etapowe (Stage-Gate)**: Implementacja punktów walidacji pomiędzy etapami przepływu
- **Zachowanie kontekstu**: Utrzymanie stanu i zgromadzonej wiedzy na wszystkich etapach
- **Propagacja błędów**: Eleganckie obsługiwanie awarii w łańcuchach sekwencyjnego przetwarzania
- **Optymalizacja wydajności**: Efektywne sekwencyjne wykonywanie z minimalnym narzutem

- **Potok produkcji treści**: Badania → Pisanie → Redakcja → Przegląd → Publikacja

```env

AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Zarządzanie konfiguracją:**

### 🏗️ **Architektura sekwencyjnego przepływu pracy**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Agent analizy]
    B --> C[Punkt kontrolny 1]
    C --> D[Stage 2: Agent przetwarzania]
    D --> E[Punkt kontrolny 2]
    E --> F[Stage 3: Agent walidacji]
    F --> G[Końcowy wynik]
    
    H[Kontekst stanu] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Responses API)] --> B
    I --> D
    I --> F
```

**Kluczowe komponenty:**
- **Sekwencyjni agenci**: Specjalistyczni agenci dla każdego etapu przetwarzania
- **Kontekst stanu**: Utrzymuje zgromadzone dane i decyzje na przestrzeni etapów
- **Punkty kontrolne**: Punkty walidacji pomiędzy etapami zapewniające jakość i spójność
- **Klient Azure OpenAI**: Spójny dostęp do modelu AI na wszystkich etapach przepływu pracy

## 💻 Uruchamianie kodu

Pełna implementacja znajduje się w `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Ten plik demonstruje **trzyetapowy przepływ pracy analizy mebli**:

1. **Etap 1 - Agent sprzedażowy**: Analizuje obrazy mebli i dostarcza sugestie zakupowe
2. **Etap 2 - Agent cenowy**: Dostarcza szczegółowe rozbicia cenowe i opcje budżetowe
3. **Etap 3 - Agent ofertowy**: Generuje profesjonalny dokument oferty w formacie Markdown

### 🏗️ **Architektura przepływu pracy**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Każdy agent:
- Otrzymuje wynik z poprzedniego etapu jako kontekst
- Buduje na podstawie poprzednich analiz dzięki specjalistycznej wiedzy
- Utrzymuje ciągłość przepływu pracy przez zarządzanie stanem

### 🚀 Uruchamianie przykładu

**Wymagania wstępne:**
- Umieść obraz mebli pod ścieżką `../imgs/home.png` (lub zmodyfikuj zmienną `imgPath`)
- Skonfiguruj plik `.env` z punktem końcowym i wdrożeniem Azure OpenAI, następnie zaloguj się za pomocą `az login`

```bash
# Uczyń skrypt wykonywalnym (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Uruchom sekwencyjny przepływ pracy
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Lub na Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Oczekiwany wynik

Przepływ pracy:
1. **Agent sprzedażowy**: Zidentyfikuje meble na obrazie i dostarczy rekomendacje
2. **Agent cenowy**: Doda szczegółową analizę cen z poziomami budżetu i sugestiami zakupowymi
3. **Agent ofertowy**: Wygeneruje sformatowany dokument oferty z całą zgromadzoną informacją

Końcowy wynik będzie kompleksową, profesjonalną ofertą mebli na podstawie analizy obrazu.

### 🔍 Zrozumienie przepływu stanu

Każdy agent w sekwencji otrzymuje:
- **Oryginalne dane wejściowe**: Początkowa wiadomość użytkownika (obraz + tekst)
- **Wyjścia poprzednich agentów**: Wszystkie poprzednie odpowiedzi agentów w historii rozmowy
- **Zgromadzony kontekst**: Kompletny stan utrzymywany w całym przepływie pracy

To umożliwia zaawansowane przetwarzanie wieloetapowe, gdzie każdy agent buduje się na podstawie kompleksowego kontekstu ze wszystkich poprzednich etapów.
