# ⚡ Współbieżne przepływy pracy agentów z Azure OpenAI (Responses API) (.NET)

## 📋 Samouczek wysokowydajnego przetwarzania równoległego

Ten notatnik demonstruje **wzorce współbieżnych przepływów pracy** z użyciem Microsoft Agent Framework dla .NET i Azure OpenAI (Responses API). Nauczysz się, jak budować wysokowydajne, równoległe przepływy pracy, które maksymalizują przepustowość poprzez jednoczesne uruchamianie wielu agentów AI przy zachowaniu koordynacji i spójności danych.

- **Azure OpenAI (Responses API)**: Koordynacja wielu współbieżnych wywołań Azure OpenAI Responses API

## 💻 Uruchamianie kodu

Pełna implementacja dostępna jest w `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Ten plik demonstruje **współbieżny przepływ pracy fan-out/fan-in** dla planowania podróży:

### 🏗️ **Architektura przepływu**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Kluczowe komponenty:**

1. **ConcurrentStartExecutor**: Wysyła żądanie użytkownika jednocześnie do wszystkich agentów
2. **Agent badawczy**: Równocześnie analizuje destynacje i atrakcje
3. **Agent planista**: Równocześnie tworzy szczegółowe plany podróży
4. **ConcurrentAggregationExecutor**: Zbiera i łączy wyniki od obu agentów

### 🎯 **Wzorzec Fan-Out/Fan-In**

Ten przepływ pracy demonstruje klasyczny wzorzec **Fan-Out/Fan-In**:
- **Fan-Out**: Jedna wiadomość wejściowa jest nadawana jednocześnie do wielu agentów
- **Przetwarzanie współbieżne**: Wielu agentów pracuje równolegle nad tym samym zadaniem
- **Fan-In**: Wyniki od wszystkich agentów są zbierane i agregowane do pojedynczego wyjścia

### 🎯 Praktyczne zastosowania

Ten wzorzec współbieżny jest idealny dla:
- **Tworzenie treści**: Wielu pisarzy jednocześnie tworzących różne sekcje
- **Przegląd kodu**: Wielu recenzentów analizujących kod z różnych perspektyw
- **Badania rynkowe**: Równoległa analiza różnych segmentów rynku
- **Przetwarzanie dokumentów**: Współbieżne wydobywanie, analiza i walidacja
- **Analiza wieloperspektywiczna**: Uzyskiwanie różnorodnych punktów widzenia na ten sam input

### 🔍 Zrozumienie niestandardowych executorów

**ConcurrentStartExecutor:**
- Implementuje `IMessageHandler<string>` do akceptowania wejścia tekstowego
- Rozgłasza wiadomości do wszystkich połączonych agentów
- Wysyła `TurnToken` do uruchomienia współbieżnego przetwarzania

**ConcurrentAggregationExecutor:**
- Implementuje `IMessageHandler<ChatMessage>` do odbioru odpowiedzi agentów
- Zbiera wiadomości w sposób bezpieczny dla wątków
- Agreguje, gdy nadejdą wszystkie oczekiwane odpowiedzi
- Zwraca końcowy wynik za pomocą `context.YieldOutputAsync()`

### ⚡ Korzyści wydajnościowe

**Współbieżne vs sekwencyjne:**
- Sekwencyjne: Agent1 (30s) → Agent2 (30s) = **60 sekund łącznie**
- Współbieżne: Agent1 (30s) || Agent2 (30s) = **30 sekund łącznie**

**Poprawa przepustowości**: Nawet N× szybsze dla N agentów współbieżnych (zależnie od obciążenia i zasobów)

### 🛡️ Obsługa błędów

Przepływ pracy obsługuje awarie pojedynczych agentów w sposób łagodny:
- Jeśli jeden agent ulegnie awarii, inni kontynuują przetwarzanie
- Agregator może wdrożyć logikę limitu czasu
- Częściowe wyniki mogą być zwrócone w razie potrzeby
