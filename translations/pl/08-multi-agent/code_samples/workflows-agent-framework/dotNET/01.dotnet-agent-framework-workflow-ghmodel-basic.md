# 🔄 Podstawowe przepływy pracy agentów z Azure OpenAI (Responses API) (.NET)

## 📋 Samouczek orkiestracji przepływów pracy

Ten notatnik pokazuje, jak tworzyć zaawansowane **przepływy pracy agentów** za pomocą Microsoft Agent Framework dla .NET i Azure OpenAI (Responses API). Nauczysz się tworzyć wieloetapowe procesy biznesowe, gdzie agenci AI współpracują, aby realizować złożone zadania poprzez uporządkowane wzorce orkiestracji.

## 🎯 Cele nauki

### 🏗️ **Podstawy architektury przepływów pracy**
- **Workflow Builder**: Projektowanie i orkiestracja złożonych, wieloetapowych procesów AI
- **Koordynacja agentów**: Koordynowanie wielu wyspecjalizowanych agentów w przepływach pracy
- **Azure OpenAI (Responses API)**: Wykorzystanie Azure OpenAI Responses API w przepływach pracy
- **Wizualne projektowanie przepływów pracy**: Tworzenie i wizualizacja struktur przepływów pracy dla lepszego zrozumienia

### 🔄 **Wzorce orkiestracji procesów**
- **Przetwarzanie sekwencyjne**: Łączenie wielu zadań agentów w logicznej kolejności
- **Zarządzanie stanem**: Utrzymywanie kontekstu i przepływu danych pomiędzy etapami przepływu pracy
- **Obsługa błędów**: Implementacja solidnej obsługi błędów i odporności przepływu pracy
- **Optymalizacja wydajności**: Projektowanie efektywnych przepływów pracy dla operacji na poziomie przedsiębiorstwa

### 🏢 **Aplikacje przepływów pracy dla przedsiębiorstw**
- **Automatyzacja procesów biznesowych**: Automatyzacja złożonych przepływów organizacyjnych
- **Proces produkcji treści**: Redakcyjne przepływy pracy z etapami przeglądu i zatwierdzenia
- **Automatyzacja obsługi klienta**: Wieloetapowe rozwiązywanie zapytań klientów
- **Przepływy pracy przetwarzania danych**: ETL z transformacją wspieraną przez AI

## ⚙️ Wymagania wstępne i konfiguracja

### 📦 **Wymagane pakiety NuGet**

Ten demonstracyjny przepływ pracy korzysta z kilku kluczowych pakietów .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Konfiguracja Azure OpenAI**

**Konfiguracja środowiska (plik .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Dostęp do Azure OpenAI:**
1. Utwórz zasób Azure OpenAI w portalu Azure
2. Wdróż model (np. `gpt-4.1-mini`) i zanotuj nazwę wdrożenia
3. Zaloguj się za pomocą `az login` i skonfiguruj zmienne środowiskowe jak pokazano powyżej

### 🏗️ **Przegląd architektury przepływów pracy**

```mermaid
graph TD
    A[Kreator przepływów pracy] --> B[Rejestr agentów]
    B --> C[Silnik wykonawczy przepływów pracy]
    C --> D[Agent 1: Generator treści]
    C --> E[Agent 2: Recenzent treści] 
    D --> F[Wyniki przepływu pracy]
    E --> F
    G[Azure OpenAI (API odpowiedzi)] --> D
    G --> E
```

**Kluczowe komponenty:**
- **WorkflowBuilder**: Główny silnik orkiestracji do projektowania przepływów pracy
- **AIAgent**: Pojedyncze, wyspecjalizowane agenty o określonych zdolnościach
- **Azure OpenAI Client**: Integracja z Azure OpenAI Responses API
- **Execution Context**: Zarządza stanem i przepływem danych między etapami przepływu pracy

## 💻 Uruchamianie kodu

Pełna implementacja jest dostępna w `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Ten plik demonstruje:

1. **Konfigurację środowiska** - ładowanie konfiguracji Azure OpenAI z pliku `.env`
2. **Konfigurację klienta Azure OpenAI** - ustawienie klienta do korzystania z Azure OpenAI Responses API
3. **Tworzenie agentów** - definiowanie wyspecjalizowanych agentów (Recepcja i Konsjerż)
4. **Workflow Builder** - tworzenie wieloagentowego przepływu pracy z przetwarzaniem sekwencyjnym
5. **Wykonanie przepływu pracy** - uruchamianie przepływu z wynikami w streamingu

### 🚀 Uruchamianie przykładu

```bash
# Uczyń skrypt wykonywalnym (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Uruchom przepływ pracy
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Lub na Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Oczekiwany wynik

Przepływ pracy:
1. Przyjmie Twoje żądanie dotyczące miejsca podróży ("Chciałbym pojechać do Paryża")
2. Agent Recepcji poda wstępną rekomendację
3. Agent Konsjerża przejrzy i udoskonali rekomendację
4. Końcowy wynik wyświetli kompletny strumień rozmowy
