# 🔄 Podstawowe przepływy pracy agentów z Azure OpenAI (Responses API) (.NET)

## 📋 Samouczek orkiestracji przepływów pracy

Ten notatnik demonstruje, jak budować zaawansowane **przepływy pracy agentów** przy użyciu Microsoft Agent Framework dla .NET oraz Azure OpenAI (Responses API). Nauczysz się tworzyć wieloetapowe procesy biznesowe, w których agenci AI współpracują, aby wykonać złożone zadania za pomocą uporządkowanych wzorców orkiestracji.

## 🎯 Cele nauki

### 🏗️ **Podstawy architektury przepływów pracy**
- **Workflow Builder**: Projektuj i orkiestruj złożone wieloetapowe procesy AI
- **Koordynacja agentów**: Koordynuj wielu wyspecjalizowanych agentów w przepływach pracy
- **Azure OpenAI (Responses API)**: Wykorzystaj Azure OpenAI Responses API w przepływach pracy
- **Wizualny projekt przepływów pracy**: Twórz i wizualizuj struktury przepływów dla lepszego zrozumienia

### 🔄 **Wzorce orkiestracji procesów**
- **Przetwarzanie sekwencyjne**: Łącz wiele zadań agentów w logicznym porządku
- **Zarządzanie stanem**: Utrzymuj kontekst i przepływ danych między etapami przepływu pracy
- **Obsługa błędów**: Implementuj niezawodne odzyskiwanie po błędach i odporność przepływu pracy
- **Optymalizacja wydajności**: Projektuj efektywne przepływy pracy dla operacji na poziomie przedsiębiorstwa

### 🏢 **Zastosowania przepływów pracy w przedsiębiorstwach**
- **Automatyzacja procesów biznesowych**: Automatyzuj złożone przepływy pracy organizacji
- **Linia produkcji treści**: Redakcyjne przepływy pracy z etapami przeglądu i zatwierdzania
- **Automatyzacja obsługi klienta**: Wieloetapowe rozwiązywanie zapytań klientów
- **Przepływy pracy przetwarzania danych**: Przepływy ETL z transformacjami wspomaganymi przez AI

## ⚙️ Wymagania wstępne i konfiguracja

### 📦 **Wymagane pakiety NuGet**

Ta demonstracja przepływu pracy wykorzystuje kilka kluczowych pakietów .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Konfiguracja Azure OpenAI**

**Konfiguracja środowiska (plik .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Dostęp do Azure OpenAI:**
1. Utwórz zasób Azure OpenAI w portalu Azure
2. Wdróż model (na przykład `gpt-5-mini`) i zapamiętaj nazwę wdrożenia
3. Zaloguj się za pomocą `az login` i skonfiguruj zmienne środowiskowe jak powyżej

- **Klient Azure OpenAI**: Integracja z Azure OpenAI Responses API
- **Kontekst wykonania**: Zarządza stanem i przepływem danych między etapami przepływu pracy

## 💻 Uruchamianie kodu

Pełna implementacja jest dostępna w `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Ten plik demonstruje:

1. **Konfigurację środowiska** - Ładowanie konfiguracji Azure OpenAI z pliku `.env`
2. **Konfigurację klienta Azure OpenAI** - Konfiguracja klienta do użycia Azure OpenAI Responses API
3. **Tworzenie agentów** - Definiowanie wyspecjalizowanych agentów (Front Desk i Concierge)
4. **Workflow Builder** - Tworzenie przepływu pracy z wieloma agentami z przetwarzaniem sekwencyjnym
5. **Wykonanie przepływu pracy** - Uruchomienie przepływu z wynikami w strumieniu

### 🚀 Uruchamianie przykładu

```bash
# Uczyń skrypt wykonywalnym (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Uruchom przepływ pracy
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Lub na Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Oczekiwany wynik

Przepływ pracy:
1. Przyjmie Twój wniosek o miejsce podróży ("Chciałbym pojechać do Paryża")
2. Agent Front Desk poda wstępną rekomendację
3. Agent Concierge przejrzy i udoskonali rekomendację
4. Końcowy output wyświetli pełny strumień rozmowy
