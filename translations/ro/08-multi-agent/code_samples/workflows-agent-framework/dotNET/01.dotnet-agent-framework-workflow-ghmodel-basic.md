# 🔄 Fluxuri de lucru de bază pentru agenți cu Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial de Orchestrare a Fluxului de Lucru

Acest notebook demonstrează cum să construiești **fluxuri de lucru sofisticate pentru agenți** folosind Microsoft Agent Framework pentru .NET și Azure OpenAI (Responses API). Vei învăța să creezi procese de afaceri în mai multe etape unde agenții AI colaborează pentru a îndeplini sarcini complexe prin tipare structurate de orchestrare.

## 🎯 Obiective de Învățare

### 🏗️ **Fundamentele Arhitecturii Fluxului de Lucru**
- **Workflow Builder**: Proiectează și orchestrează procese AI complexe în mai multe etape
- **Coordonarea Agenților**: Coordonează mai mulți agenți specializați în cadrul fluxurilor de lucru
- **Azure OpenAI (Responses API)**: Utilizează Azure OpenAI Responses API în fluxuri de lucru
- **Proiectare Vizuală a Fluxului de Lucru**: Creează și vizualizează structuri de flux pentru o mai bună înțelegere

### 🔄 **Tipare de Orchestrare a Proceselor**
- **Procesare Secvențială**: Leagă multiple sarcini ale agenților în ordine logică
- **Gestionarea Stării**: Menține contextul și fluxul de date între etapele fluxului
- **Gestionarea Erorilor**: Implementează recuperare robustă din erori și reziliență a fluxului
- **Optimizarea Performanței**: Proiectează fluxuri eficiente pentru operațiuni la scară enterprise

### 🏢 **Aplicații Enterprise pentru Fluxuri de Lucru**
- **Automatizarea Proceselor de Afaceri**: Automatizează fluxuri organizaționale complexe
- **Flux de Producție de Conținut**: Fluxuri editoriale cu revizuire și etape de aprobare
- **Automatizarea Serviciului Clienți**: Rezolvare în mai multe etape a solicitărilor clienților
- **Fluxuri de Procesare a Datelor**: Fluxuri ETL cu transformare asistată de AI

## ⚙️ Cerințe și Configurare

### 📦 **Pachete NuGet Necesare**

Această demonstrație de flux de lucru folosește mai multe pachete cheie .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Configurarea Azure OpenAI**

**Configurare Mediu (.env file):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Acces Azure OpenAI:**
1. Creează o resursă Azure OpenAI în portalul Azure
2. Fă deploy unui model (de exemplu, `gpt-4.1-mini`) și notează numele deploy-ului
3. Autentifică-te cu `az login` și configurează variabilele de mediu așa cum este prezentat mai sus

## 💻 Rularea Codului

Implementarea completă este disponibilă în `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Acest fișier demonstrează:

1. **Configurarea Mediului** - Încărcarea configurației Azure OpenAI din fișier `.env`
2. **Setarea Clientului Azure OpenAI** - Configurarea clientului pentru a folosi Azure OpenAI Responses API
3. **Crearea Agenților** - Definirea agenților specializați (Front Desk și Concierge)
4. **Workflow Builder** - Crearea unui flux multi-agent cu procesare secvențială
5. **Executarea Fluxului** - Rularea fluxului cu rezultate în streaming

### 🚀 Rularea Exemplului

```bash
# Fă scriptul executabil (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Rulează fluxul de lucru
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Sau pe Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Rezultatul Așteptat

Fluxul va:
1. Accepta cererea ta de destinație de călătorie („Aș dori să merg la Paris”)
2. Agentul Front Desk oferă o recomandare inițială
3. Agentul Concierge revizuiește și rafinează recomandarea
4. Ieșirea finală afișează fluxul complet al conversației

### 🔧 Personalizare

Poți personaliza fluxul de lucru prin:
- Modificarea instrucțiunilor agenților pentru a schimba comportamentul lor
- Adăugarea mai multor agenți pentru a crea fluxuri complexe în mai multe etape
- Schimbarea mesajului utilizatorului pentru a testa scenarii diferite
- Ajustarea muchiilor fluxului pentru a crea modele diferite de execuție
