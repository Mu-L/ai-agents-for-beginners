# 🔄 Fluxuri de lucru de bază pentru agenți cu Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial pentru orchestrarea fluxurilor de lucru

Acest caiet demonstrează cum să construiești fluxuri de lucru **sofisticate pentru agenți** folosind Microsoft Agent Framework pentru .NET și Azure OpenAI (Responses API). Vei învăța să creezi procese de afaceri în mai mulți pași în care agenții AI colaborează pentru a îndeplini sarcini complexe prin modele de orchestrare structurate.

## 🎯 Obiective de învățare

### 🏗️ **Fundamentele arhitecturii fluxurilor de lucru**
- **Workflow Builder**: Proiectează și orchestrează procese AI complexe în mai mulți pași
- **Coordonarea Agenților**: Coordonează mai mulți agenți specializați în cadrul fluxurilor de lucru
- **Azure OpenAI (Responses API)**: Valorifică API-ul Azure OpenAI Responses în fluxurile de lucru
- **Design vizual al fluxurilor de lucru**: Creează și vizualizează structurile fluxurilor pentru o înțelegere mai bună

### 🔄 **Modele de orchestrare a proceselor**
- **Procesare secvențială**: Leagă multiple sarcini ale agenților într-o ordine logică
- **Gestionarea stării**: Menține contextul și fluxul de date între etapele fluxului de lucru
- **Gestionarea erorilor**: Implementează recuperare robustă și reziliență a fluxului de lucru
- **Optimizarea performanței**: Proiectează fluxuri eficiente pentru operațiuni la scară enterprise

### 🏢 **Aplicații enterprise pentru fluxurile de lucru**
- **Automatizarea proceselor de afaceri**: Automatizează fluxuri complexe organizaționale
- **Linie de producție conținut**: Fluxuri editoriale cu etape de revizuire și aprobare
- **Automatizarea serviciului clienți**: Rezolvarea interogărilor clienților în mai mulți pași
- **Fluxuri de procesare a datelor**: Fluxuri ETL cu transformare asistată de AI

## ⚙️ Cerințe și configurare

### 📦 **Pachete NuGet necesare**

Această demonstrație a fluxului de lucru utilizează câteva pachete .NET esențiale:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Configurarea Azure OpenAI**

**Configurarea mediului (.env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Acces Azure OpenAI:**
1. Creează o resursă Azure OpenAI în portalul Azure
2. Desfășoară un model (de exemplu, `gpt-5-mini`) și notează numele desfășurării
3. Autentifică-te cu `az login` și configurează variabilele de mediu conform exemplului de mai sus

### 🏗️ **Prezentare generală a arhitecturii fluxului de lucru**

```mermaid
graph TD
    A[Constructor de fluxuri de lucru] --> B[Registrul agenților]
    B --> C[Motorul de execuție al fluxului de lucru]

    C --> E[Agent 2: Recenzor de conținut]
```

**Componente cheie:**
- **WorkflowBuilder**: Motorul principal de orchestrare pentru proiectarea fluxurilor de lucru
- **AIAgent**: Agenți individuali specializați cu capabilități specifice
- **Client Azure OpenAI**: Integrare cu API-ul Azure OpenAI Responses
- **Context de execuție**: Gestionează starea și fluxul de date între etapele fluxului de lucru

## 💻 Rularea codului

Implementarea completă este disponibilă în `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Acest fișier demonstrează:

1. **Configurarea mediului** - Încărcarea configurației Azure OpenAI din fișierul `.env`
2. **Configurarea clientului Azure OpenAI** - Configurarea clientului pentru a folosi Azure OpenAI Responses API
3. **Crearea agenților** - Definirea agenților specializați (Front Desk și Concierge)
4. **Workflow Builder** - Crearea unui flux de lucru cu mai mulți agenți și procesare secvențială
5. **Executarea fluxului de lucru** - Rularea fluxului cu rezultate în streaming

### 🚀 Rularea exemplului

```bash
# Fă scriptul executabil (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Rulează fluxul de lucru
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Sau pe Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Rezultat așteptat

Fluxul de lucru va:
1. Prelua cererea ta de destinație de călătorie ("Aș dori să merg la Paris")
2. Agentul Front Desk oferă o recomandare inițială
3. Agentul Concierge revizuiește și rafinează recomandarea
4. Ieșirea finală afișează fluxul complet al conversației

### 🔧 Personalizare

Poți personaliza fluxul de lucru prin:
- Modificarea instrucțiunilor agenților pentru a schimba comportamentul lor
- Adăugarea mai multor agenți pentru a crea fluxuri complexe în mai mulți pași
- Schimbarea mesajului utilizatorului pentru a testa scenarii diferite
- Ajustarea muchiilor fluxului pentru a crea modele diferite de execuție
