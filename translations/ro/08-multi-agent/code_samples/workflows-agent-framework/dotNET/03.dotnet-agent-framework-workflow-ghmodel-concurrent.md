# ⚡ Fluxuri de lucru concurente cu agenți în Azure OpenAI (API Răspunsuri) (.NET)

## 📋 Tutorial procesare paralelă de înaltă performanță

Acest notebook demonstrează **modele de fluxuri de lucru concurente** folosind Microsoft Agent Framework pentru .NET și Azure OpenAI (API Răspunsuri). Veți învăța cum să construiți fluxuri de lucru de procesare paralelă, de înaltă performanță, care maximizează debitul executând simultan mai mulți agenți AI, menținând coordonarea și consistența datelor.

## 💻 Rularea codului

Implementarea completă este disponibilă în `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Acest fișier demonstrează un **flux de lucru concurent Fan-Out/Fan-In** pentru planificare de călătorii:

### 🏗️ **Arhitectură flux de lucru**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Componente cheie:**

1. **ConcurrentStartExecutor**: Transmite solicitarea utilizatorului simultan tuturor agenților
2. **Agent Cercetător**: Analizează destinațiile și atracțiile în paralel
3. **Agent Planificator**: Creează planuri detaliate de călătorie concurent
4. **ConcurrentAggregationExecutor**: Colectează și combină rezultatele ambilor agenți

### 🎯 **Modelul Fan-Out/Fan-In**

Acest flux de lucru demonstrează modelul clasic **Fan-Out/Fan-In**:
- **Fan-Out**: Un mesaj de intrare este transmis simultan mai multor agenți
- **Procesare concurentă**: Mai mulți agenți lucrează în paralel la aceeași sarcină
- **Fan-In**: Rezultatele de la toți agenții sunt colectate și agregate într-un singur output

### 📝 Output așteptat

Fluxul de lucru va:
1. **Transmitere solicitare**: Trimite "Planifică o călătorie la Seattle în decembrie" ambilor agenți
2. **Procesare concurentă**: Ambii agenți lucrează simultan:

   - Planificatorul creează itinerariul și logistica
3. **Agregare**: Combină ambele răspunsuri într-un output cuprinzător
4. **Afișare rezultate**: Arată planul de călătorie îmbinat cu toate informațiile

### 🔍 Înțelegerea executorilor personalizați

**ConcurrentStartExecutor:**
- Implementează `IMessageHandler<string>` pentru a accepta input sub formă de string
- Difuzează mesaje tuturor agenților conectați
- Trimite `TurnToken` pentru a declanșa procesarea concurentă

**ConcurrentAggregationExecutor:**
- Implementează `IMessageHandler<ChatMessage>` pentru a primi răspunsuri de la agenți
- Colectează mesajele într-un mod thread-safe
- Agregă când toate răspunsurile așteptate sosesc
- Produce output final folosind `context.YieldOutputAsync()`

### ⚡ Beneficii de performanță

**Concurent vs Secvențial:**
- Secvențial: Agent1 (30s) → Agent2 (30s) = **60 de secunde total**
- Concurent: Agent1 (30s) || Agent2 (30s) = **30 de secunde total**

**Îmbunătățire debit**: Până la N× mai rapid pentru N agenți concurenți (în funcție de încărcare și resurse)

### 🛡️ Tratarea erorilor

Fluxul de lucru gestionează cu grație eșecurile agenților individuali:
- Dacă un agent eșuează, ceilalți continuă procesarea
- Aggregatorul poate implementa logica de timeout
- Pot fi returnate rezultate parțiale dacă este nevoie

### 📊 Caracteristici avansate

**Număr dinamic de agenți:**
Modifică logica de agregare pentru a susține număr variabil de agenți:

```csharp
private int _expectedAgentCount;
private readonly List<ChatMessage> _messages = [];

public async ValueTask HandleAsync(ChatMessage message, IWorkflowContext context)
{
    this._messages.Add(message);
    if (this._messages.Count == _expectedAgentCount)
    {
        // Process aggregation
    }
}
```

Acest model de flux de lucru concurent este esențial pentru construirea sistemelor de agenți AI scalabili și de înaltă performanță!
