# ⚡ Fluxuri de Lucru Agent Concurrent cu Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial de Procesare Paralelă de Înaltă Performanță

Acest notebook demonstrează **modele de fluxuri de lucru concurente** folosind Microsoft Agent Framework pentru .NET și Azure OpenAI (Responses API). Veți învăța cum să construiți fluxuri de lucru performante, de procesare paralelă, care maximizează debitul prin executarea simultană a mai multor agenți AI, menținând în același timp coordonarea și consistența datelor.

## 💻 Rularea Codului

Implementarea completă este disponibilă în `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Acest fișier demonstrează un **flux de lucru concurent Fan-Out/Fan-In** pentru planificare de călătorie:

### 🏗️ **Arhitectura Fluxului de Lucru**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Componente Cheie:**

1. **ConcurrentStartExecutor**: Transmite cererea utilizatorului tuturor agenților simultan
2. **Agent Cercetător**: Analizează destinațiile și atracțiile în paralel
3. **Agent Planificator**: Creează planuri de călătorie detaliate concurent
4. **ConcurrentAggregationExecutor**: Colectează și combină rezultatele ambilor agenți

### 🎯 **Modelul Fan-Out/Fan-In**

Acest flux de lucru demonstrează modelul clasic **Fan-Out/Fan-In**:
- **Fan-Out**: Un mesaj de intrare este transmis simultan către mai mulți agenți
- **Procesare Concurentă**: Mai mulți agenți lucrează în paralel la aceeași sarcină
- **Fan-In**: Rezultatele tuturor agenților sunt colectate și agregate într-un singur output

Fluxul de lucru va:

### 🔍 Înțelegerea Executorilor Personalizați

**ConcurrentStartExecutor:**
- Implementarea `IMessageHandler<string>` pentru a accepta input de tip string
- Transmite mesaje către toți agenții conectați
- Trimite `TurnToken` pentru a declanșa procesarea concurentă

**ConcurrentAggregationExecutor:**
- Implementarea `IMessageHandler<ChatMessage>` pentru a primi răspunsurile agenților
- Colectează mesajele într-un mod thread-safe
- Agregă când sosesc toate răspunsurile așteptate
- Produce output-ul final folosind `context.YieldOutputAsync()`

### ⚡ Beneficii de Performanță

**Concurent vs Secvențial:**
- Secvențial: Agent1 (30s) → Agent2 (30s) = **60 secunde total**
- Concurent: Agent1 (30s) || Agent2 (30s) = **30 secunde total**

**Îmbunătățire a debitului**: De până la N× mai rapid pentru N agenți concurenți (în funcție de sarcină și resurse)

### 🛡️ Tratarea Erorilor

Fluxul de lucru gestionează elegant eșecurile agenților individuali:
- Dacă un agent eșuează, ceilalți continuă procesarea
- Agregatorul poate implementa logică de timeout
- Pot fi returnate rezultate parțiale dacă este necesar
