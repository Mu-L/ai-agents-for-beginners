# ⏩ Fluxuri de lucru secvențiale ale agenților cu Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial avansat de procesare secvențială

Acest notebook demonstrează **modele de fluxuri de lucru secvențiale** folosind Microsoft Agent Framework pentru .NET și Azure OpenAI (Responses API). Veți învăța cum să construiți conducte sofisticate de procesare pas cu pas unde agenții execută într-o anumită ordine, fiecare etapă bazându-se pe rezultatele etapei precedente.

## 🎯 Obiective de învățare

### 🔄 **Arhitectura procesării secvențiale**
- **Design linear al fluxului de lucru**: Creați conducte de procesare pas cu pas cu dependențe clare
- **Gestionarea stării**: Mențineți contextul și fluxul de date pe parcursul etapelor secvențiale ale fluxului de lucru
- **Azure OpenAI (Responses API)**: Folosiți modelele Azure OpenAI în fluxuri multi-etape .NET
- **Modele pentru conducte enterprise**: Construiți sisteme de procesare secvențiale pregătite pentru producție

### 🏗️ **Modele secvențiale avansate**
- **Procesare cu praguri (Stage-Gate)**: Implementați puncte de validare între etapele fluxului de lucru
- **Păstrarea contextului**: Mențineți starea și cunoștințele acumulate pe toate etapele
- **Propagarea erorilor**: Gestionați eșecurile cu grație în lanțurile de procesare secvențială
- **Optimizarea performanței**: Execuție secvențială eficientă cu un overhead minim

### 🏢 **Aplicații secvențiale enterprise**
- **Conductă de procesare documente**: Analiză, transformare și validare multi-etapă a documentelor
- **Fluxuri de lucru pentru asigurarea calității**: Procese secvențiale de revizuire, validare și aprobare
- **Conductă de producție conținut**: Cercetare → Scriere → Editare → Revizuire → Publicare
- **Automatizarea proceselor de afaceri**: Fluxuri multi-pași cu dependențe clare între etape


### 🔑 **Configurarea Azure OpenAI**

**Configurarea mediului (.env file):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Gestionarea configurației:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **Arhitectura fluxului de lucru secvențial**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Agent de analiză]
    B --> C[Punct de control 1]
    C --> D[Stage 2: Agent de procesare]
    D --> E[Punct de control 2]
    E --> F[Stage 3: Agent de validare]
    F --> G[Rezultat final]
    
    H[Context de stare] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API Răspunsuri)] --> B
    I --> D
    I --> F
```

**Componente cheie:**
- **Agenți secvențiali**: Agenți specializați pentru fiecare etapă de procesare
- **Contextul stării**: Menține datele acumulate și deciziile pe parcursul etapelor
- **Puncte de control**: Puncte de validare între etape pentru asigurarea calității și consistenței
- **Client Azure OpenAI**: Acces consistent la modelul AI pe toate etapele fluxului de lucru

## 💻 Executarea codului

Implementarea completă este disponibilă în `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Acest fișier demonstrează un **flux de lucru de analiză mobilier în trei etape**:

1. **Etapa 1 - Agent de vânzări**: Analizează imagini de mobilier și oferă sugestii de cumpărare
2. **Etapa 2 - Agent de prețuri**: Oferă detalii despre defalcările prețurilor și opțiunile de buget
3. **Etapa 3 - Agent de ofertă**: Generează un document profesional de ofertă în format Markdown

### 🏗️ **Arhitectura fluxului de lucru**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Fiecare agent:
- Primiți ieșirea de la etapa anterioară ca context
- Construiește pe baza analizei anterioare cu expertiză specializată
- Menține continuitatea fluxului prin gestionarea stării

### 🚀 Rularea exemplului

**Prerechizite:**
- Puneți o imagine de mobilier la `../imgs/home.png` (sau actualizați variabila `imgPath`)
- Configurați fișierul `.env` cu endpoint-ul și implementarea Azure OpenAI, apoi conectați-vă cu `az login`

```bash
# Faceți scriptul executabil (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Rulați fluxul de lucru secvențial
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Sau pe Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Rezultate așteptate

Fluxul de lucru va:
1. **Agent de vânzări**: Identifică piesele de mobilier din imagine și oferă recomandări
2. **Agent de prețuri**: Adaugă analiza detaliată a prețurilor cu niveluri de buget și recomandări de cumpărare
3. **Agent de ofertă**: Generează un document de ofertă formatat cu toate informațiile sintetizate

Rezultatul final va fi o ofertă profesională cuprinzătoare pentru mobilier bazată pe analiza imaginii.
