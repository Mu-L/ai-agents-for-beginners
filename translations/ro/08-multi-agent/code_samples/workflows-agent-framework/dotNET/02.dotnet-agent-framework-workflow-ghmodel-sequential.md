# ⏩ Fluxuri de lucru secvențiale cu agenți folosind Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial Avansat de Procesare Secvențială

Acest notebook demonstrează **modele de fluxuri de lucru secvențiale** folosind Microsoft Agent Framework pentru .NET și Azure OpenAI (Responses API). Veți învăța cum să construiți conducte sofisticate de procesare pas cu pas, în care agenții execută într-o anumită ordine, fiecare etapă bazându-se pe rezultatele etapei precedente.

## 🎯 Obiective de Învățare

### 🔄 **Arhitectura Procesării Secvențiale**
- **Design liniar al fluxului de lucru**: Creați conducte de procesare pas cu pas cu dependențe clare
- **Gestionarea Stării**: Mențineți contextul și fluxul de date prin etapele fluxului secvențial
- **Azure OpenAI (Responses API)**: Utilizați modelele Azure OpenAI în fluxuri de lucru .NET multi-etapă
- **Modele de conducte pentru întreprinderi**: Construiți sisteme de procesare secvențială pregătite pentru producție

### 🏗️ **Modele secvențiale avansate**
- **Procesarea Stage-Gate**: Implementați puncte de validare între etapele fluxului
- **Păstrarea contextului**: Mențineți starea și cunoștințele acumulate prin toate etapele
- **Propagarea erorilor**: Gestionează eșecurile cu grație în lanțurile de procesare secvențială
- **Optimizarea performanței**: Execuție secvențială eficientă cu un minim de costuri

### 🏢 **Aplicații secvențiale pentru întreprinderi**
- **Conductă de procesare documente**: Analiză, transformare și validare documente în mai multe etape
- **Fluxuri de lucru pentru asigurarea calității**: Revizuire, validare și aprobare secvențială
- **Conductă de producție de conținut**: Cercetare → Scriere → Editare → Revizuire → Publicare
- **Automatizarea proceselor de afaceri**: Fluxuri de lucru multi-pas cu dependențe clare între etape

## ⚙️ Cerințe și Configurare

### 📦 **Pachete NuGet Necesare**

Pachete esențiale pentru fluxuri de lucru secvențiale în .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Configurare Azure OpenAI**

**Configurarea mediului (.env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Managementul configurației:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

### 🏗️ **Arhitectura fluxului secvențial**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Agent de analiză]
    B --> C[Punct de control 1]
    C --> D[Stage 2: Agent de procesare]
    D --> E[Punct de control 2]
    E --> F[Stage 3: Agent de validare]
    F --> G[Rezultat final]
    
    H[Context stare] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API Răspunsuri)] --> B
    I --> D
    I --> F
```

**Componente cheie:**
- **Agenți secvențiali**: Agenți specializați pentru fiecare etapă de procesare
- **Contextul stării**: Menține date acumulate și decizii prin etape
- **Puncte de control**: Puncte de validare între etape pentru a asigura calitatea și consistența
- **Client Azure OpenAI**: Acces consistent la modelul AI prin toate etapele fluxului

## 💻 Rularea Codului

Implementarea completă este disponibilă în `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Acest fișier demonstrează un **flux de analiză pentru mobilă în trei etape**:

1. **Etapa 1 - Agent de vânzări**: Analizează imagini de mobilă și oferă sugestii de cumpărare
2. **Etapa 2 - Agent de preț**: Oferă detalii despre structura prețurilor și opțiuni de buget
3. **Etapa 3 - Agent de ofertă**: Generează un document de ofertă profesional în format Markdown

### 🏗️ **Arhitectura fluxului de lucru**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Fiecare agent:
- Primește rezultatul etapei precedente ca context
- Construiește pe baza analizei anterioare cu expertiză specializată
- Menține continuitatea fluxului prin gestionarea stării

### 🚀 Rularea Exemplului

**Prerechizite:**
- Puneți o imagine de mobilă la `../imgs/home.png` (sau actualizați variabila `imgPath`)
- Configurați fișierul `.env` cu punctul final Azure OpenAI și implementarea, apoi autentificați-vă cu `az login`

```bash
# Fă scriptul executabil (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Rulează fluxul de lucru secvențial
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Sau pe Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Output așteptat

Fluxul de lucru va:
1. **Agentul de vânzări**: Identifică articolele de mobilă din imagine și oferă recomandări
2. **Agentul de preț**: Adaugă analiza detaliată a prețurilor cu niveluri de buget și recomandări de cumpărături
3. **Agentul de ofertă**: Generează un document de ofertă formatat cu toate informațiile sintetizate

Output-ul final va fi o ofertă de mobilă profesională și cuprinzătoare, bazată pe analiza imaginii.
