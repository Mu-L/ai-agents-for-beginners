# ⏩ Sekvensielle agentarbeidsflyter med Azure OpenAI (Responses API) (.NET)

## 📋 Avansert veiledning i sekvensiell behandling

Denne notatboken demonstrerer **sekvensielle arbeidsflytmønstre** ved bruk av Microsoft Agent Framework for .NET og Azure OpenAI (Responses API). Du vil lære hvordan du bygger sofistikerte, trinnvise behandlingsrørledninger der agenter utfører i en bestemt rekkefølge, hvor hvert trinn bygger på resultatene fra det forrige trinnet.

## 🎯 Læringsmål

### 🔄 **Sekvensiell behandlingsarkitektur**
- **Lineært arbeidsflytdesign**: Lag trinnvise behandlingsrørledninger med klare avhengigheter
- **Tilstandshåndtering**: Oppretthold kontekst og dataflyt gjennom sekvensielle arbeidsflyttrinn
- **Azure OpenAI (Responses API)**: Utnytt Azure OpenAI-modeller i flerstegs .NET-arbeidsflyter
- **Bedriftsrørledningsmønstre**: Bygg produksjonsklare sekvensielle behandlingssystemer

### 🏗️ **Avanserte sekvensielle mønstre**
- **Stage-Gate behandling**: Implementer valideringskontroller mellom arbeidsflyttrinn
- **Kontekstbevaring**: Oppretthold tilstand og akkumulert kunnskap gjennom alle trinn
- **Feilpropagasjon**: Håndter feil på en smidig måte i sekvensielle behandlingskjeder
- **Ytelsesoptimalisering**: Effektiv sekvensiell utførelse med minimal overhead

### 🏢 **Sekvensielle bedriftsapplikasjoner**
- **Dokumentbehandlingsrørledning**: Flertrinns dokumentanalyse, transformasjon og validering

- **Innholdsproduksjonsrørledning**: Forskning → Skriving → Redigering → Gjennomgang → Publisering
- **Automatisering av forretningsprosesser**: Flertrinns arbeidsflyter med klare trinnavhengigheter

## 💻 Kjøre koden

Den komplette implementeringen er tilgjengelig i `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Denne filen demonstrerer en **treskikts møbelanalyse-arbeidsflyt**:

1. **Trinn 1 - Salgsagent**: Analyserer møbelbilder og gir kjøpsforslag
2. **Trinn 2 - Prisagent**: Gir detaljerte prisoversikter og budsjettalternativer
3. **Trinn 3 - Tilbudsagent**: Genererer et profesjonelt tilbudsdokument i Markdown-format

### 🏗️ **Arbeidsflytarkitektur**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Hver agent:
- Mottar output fra forrige trinn som kontekst
- Bygger videre på forrige analyse med spesialisert ekspertise
- Opprettholder kontinuiteten i arbeidsflyten gjennom tilstandshåndtering

### 🚀 Kjøre eksemplet

**Forutsetninger:**
- Sett inn et møbelbilde på `../imgs/home.png` (eller oppdater `imgPath`-variabelen)
- Konfigurer `.env`-filen med din Azure OpenAI-endepunkt og distribusjon, så logg inn med `az login`

```bash
# Gjør skriptet kjørbart (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Kjør den sekvensielle arbeidsflyten
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Eller på Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Forventet output

Arbeidsflyten vil:
1. **Salgsagent**: Identifisere møbelobjekter fra bildet og gi anbefalinger
2. **Prisagent**: Legge til detaljert prisanalyse med budsjettlag og handleforslag
3. **Tilbudsagent**: Generere et formatert tilbudsdokument med all informasjon sammenfattet

Det endelige resultatet vil være et omfattende, profesjonelt møbeltilbud basert på bildeanalyse.

### 🎯 Virkelige bruksområder

Dette sekvensielle mønsteret er ideelt for:
- **E-handel**: Produktsanalyse → Prissetting → Tilbudsgenerering
- **Eiendom**: Eiendomsvurdering → Verdivurdering → Oppretting av annonser

### 🔍 Forståelse av tilstandsstrøm

Hver agent i sekvensen mottar:
- **Opprinnelig input**: Den initielle brukermeldingen (bilde + tekst)
- **Forrige agents svar**: Alle tidligere agentsvar i samtalehistorikken
- **Akkumulert kontekst**: Full tilstand opprettholdt gjennom hele arbeidsflyten

Dette gjør mulig sofistikert flerstegs behandling der hver agent bygger på en omfattende kontekst fra alle tidligere trinn.
