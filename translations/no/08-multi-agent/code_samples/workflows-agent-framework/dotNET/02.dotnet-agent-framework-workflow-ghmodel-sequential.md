# ⏩ Sekvensielle agentarbeidsflyter med Azure OpenAI (Responses API) (.NET)

## 📋 Avansert veiledning for sekvensiell behandling

Denne notatboken viser **sekvensielle arbeidsflytmønstre** ved bruk av Microsoft Agent Framework for .NET og Azure OpenAI (Responses API). Du vil lære hvordan du bygger sofistikerte, steg-for-steg behandlingsrørledninger der agenter utfører oppgaver i en bestemt rekkefølge, med hvert trinn som bygger på resultatene fra det forrige trinnet.

## 🎯 Læringsmål

### 🔄 **Sekvensiell behandlingsarkitektur**
- **Lineært arbeidsflytdesign**: Lag steg-for-steg behandlingsrørledninger med klare avhengigheter

- **Azure OpenAI (Responses API)**: Utnytt Azure OpenAI-modeller i flertrinns .NET-arbeidsflyter
- **Enterprise-pipeline-mønstre**: Bygg produksjonsklare sekvensielle behandlingssystemer

### 🏗️ **Avanserte sekvensielle mønstre**
- **Stage-Gate-behandling**: Implementer valideringspunkter mellom arbeidsflyttrinn
- **Kontekstopprettholdelse**: Bevar tilstand og akkumulert kunnskap på tvers av alle trinn
- **Feilsmittehåndtering**: Håndter feil grasiøst i sekvensielle behandlingskjeder
- **Ytelsesoptimalisering**: Effektiv sekvensiell utførelse med minimal overhead

### 🏢 **Enterprise sekvensielle applikasjoner**

## 💻 Kjøre koden

Den komplette implementasjonen er tilgjengelig i `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Denne filen demonstrerer en **tretrinns møbelanalysearbeidsflyt**:

1. **Trinn 1 - Salgsagent**: Analyserer møbelbilder og gir kjøpsforslag
2. **Trinn 2 - Prisantagent**: Gir detaljert prisoversikt og budsjettalternativer
3. **Trinn 3 - Tilbudsagent**: Genererer et profesjonelt tilbudsdokument i Markdown-format

### 🏗️ **Arbeidsflytarkitektur**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Hver agent:
- Mottar utdata fra forrige trinn som kontekst
- Bygger videre på tidligere analyser med spesialisert ekspertise
- Opprettholder kontinuitet i arbeidsflyten gjennom tilstandsadministrasjon

### 🚀 Kjøre eksemplet

**Forutsetninger:**
- Plasser et møbelbilde på `../imgs/home.png` (eller oppdater variabelen `imgPath`)
- Konfigurer `.env`-filen din med Azure OpenAI-endepunkt og distribusjon, og logg inn med `az login`

```bash
# Gjør skriptet kjørbart (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Kjør den sekvensielle arbeidsflyten
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Eller på Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Forventet resultat

Arbeidsflyten vil:
1. **Salgsagent**: Identifisere møbelobjekter fra bildet og gi anbefalinger
2. **Prisantagent**: Legge til detaljert prisanalyse med budsjettnivåer og handleanbefalinger
3. **Tilbudsagent**: Generere et formatert tilbudsdokument med all informasjon syntetisert

Sluttresultatet blir et omfattende, profesjonelt møbeltilbud basert på bildeanalyse.

Hver agent i sekvensen mottar:
- **Opprinnelig input**: Den innledende brukermeldingen (bilde + tekst)
- **Tidligere agentutdata**: Alle tidligere agentrespons i samtalehistorikken
- **Akkumulert kontekst**: Fullstendig tilstand opprettholdt gjennom hele arbeidsflyten

Dette muliggjør avansert flertrinns behandling hvor hver agent bygger på omfattende kontekst fra alle tidligere trinn.
