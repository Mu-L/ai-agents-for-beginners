# 🔄 Grunnleggende Agentarbeidsflyter med Azure OpenAI (Responses API) (.NET)

## 📋 Veiledning for arbeidsflytorkestrering

Denne notatblokken demonstrerer hvordan du kan bygge sofistikerte **agentarbeidsflyter** ved å bruke Microsoft Agent Framework for .NET og Azure OpenAI (Responses API). Du vil lære å lage flertrinns forretningsprosesser der AI-agenter samarbeider for å utføre komplekse oppgaver gjennom strukturerte orkestreringsmønstre.

## 🎯 Læringsmål

### 🏗️ **Fundamenter for arbeidsflytarkitektur**
- **Workflow Builder**: Utform og orkestrer komplekse flertrinns AI-prosesser
- **Agentkoordinering**: Koordiner flere spesialiserte agenter innen arbeidsflyter
- **Azure OpenAI (Responses API)**: Utnytt Azure OpenAI Responses API i arbeidsflyter
- **Visuell arbeidsflytdesign**: Lag og visualiser arbeidsflytstrukturer for bedre forståelse

### 🔄 **Orkestreringsmønstre for prosesser**
- **Sekvensiell behandling**: Kjed flere agentoppgaver i logisk rekkefølge
- **Statusadministrasjon**: Oppretthold kontekst og dataflyt gjennom arbeidsflytens stadier
- **Feilhåndtering**: Implementer robust feilgjenoppretting og arbeidsflyttilpasning
- **Ytelsesoptimalisering**: Design effektive arbeidsflyter for drift i stor skala

### 🏢 **Forretningsapplikasjoner med arbeidsflyter**
- **Automatisering av forretningsprosesser**: Automatiser komplekse organisatoriske arbeidsflyter
- **Innholdsproduksjonspipeline**: Redaksjonelle arbeidsflyter med gjennomgangs- og godkjenningsfaser
- **Automatisering av kundeservice**: Flertrinnsløsning for kundehenvendelser
- **Databehandlingsarbeidsflyter**: ETL-arbeidsflyter med AI-drevet transformasjon

## ⚙️ Forutsetninger og oppsett

### 📦 **Nødvendige NuGet-pakker**

Denne arbeidsflytdemonstrasjonen bruker flere viktige .NET-pakker:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI-konfigurasjon**

**Miljøoppsett (.env-fil):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI-tilgang:**
1. Opprett en Azure OpenAI-ressurs i Azure-portalen
2. Distribuer en modell (for eksempel, `gpt-4.1-mini`) og merk distribusjonsnavnet
3. Logg inn med `az login` og konfigurer miljøvariabler som vist ovenfor

## 💻 Kjøring av koden

Den komplette implementasjonen finnes i `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Denne filen demonstrerer:

1. **Miljøkonfigurasjon** - Laste Azure OpenAI-konfigurasjon fra `.env`-fil
2. **Azure OpenAI-klientoppsett** - Konfigurere klienten for å bruke Azure OpenAI Responses API
3. **Agentopprettelse** - Definere spesialiserte agenter (resepsjon og concierge)
4. **Workflow Builder** - Lage en fleragent-arbeidsflyt med sekvensiell behandling
5. **Arbeidsflytekjøring** - Kjøre arbeidsflyten med strømmende resultater

### 🚀 Kjøre eksemplet

```bash
# Gjør skriptet kjørbart (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Kjør arbeidsflyten
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Eller på Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Forventet utdata

Arbeidsflyten vil:
1. Godta forespørselen om reisemål ("Jeg vil gjerne dra til Paris")
2. Resepsjonsagenten gir en første anbefaling
3. Concierge-agenten gjennomgår og forbedrer anbefalingen
4. Endelig utdata viser hele samtalestrømmen

### 🔧 Tilpasning

Du kan tilpasse arbeidsflyten ved å:
- Endre agentinstruksjoner for å justere atferden deres
- Legge til flere agenter for å lage komplekse flertrinns arbeidsflyter
- Endre brukermeldingen for å teste ulike scenarier
- Justere arbeidsflytkantene for å lage forskjellige kjøringsmønstre
