# 🔄 Grunnleggende arbeidsflyter for agenter med Azure OpenAI (Responses API) (.NET)

## 📋 Veiledning for arbeidsflytorkestrering

Denne notatblokken viser hvordan du bygger sofistikerte **agentarbeidsflyter** ved hjelp av Microsoft Agent Framework for .NET og Azure OpenAI (Responses API). Du lærer å lage flertrinns forretningsprosesser der AI-agenter samarbeider om å utføre komplekse oppgaver gjennom strukturerte orkestreringsmønstre.

## 🎯 Læringsmål

### 🏗️ **Grunnleggende om arbeidsflytarkitektur**
- **Workflow Builder**: Design og orkestrer komplekse flertrinns AI-prosesser
- **Agentkoordinering**: Koordiner flere spesialiserte agenter i arbeidsflyter
- **Azure OpenAI (Responses API)**: Utnytt Azure OpenAI Responses API i arbeidsflyter
- **Visuell arbeidsflytdesign**: Lag og visualiser arbeidsflytstrukturer for bedre forståelse

### 🔄 **Mønstre for prosessorkestrering**
- **Sekvensiell behandling**: Kjed flere agentoppgaver i logisk rekkefølge
- **Tilstandshåndtering**: Oppretthold kontekst og dataflyt gjennom arbeidsflytens faser
- **Feilhåndtering**: Implementer robust feilutvinning og motstandsdyktighet i arbeidsflyten
- **Ytelsesoptimalisering**: Design effektive arbeidsflyter for virksomhetsskala

### 🏢 **Arbeidsflyt-applikasjoner for virksomheter**
- **Automatisering av forretningsprosesser**: Automatiser komplekse organisatoriske arbeidsflyter
- **Innholdsproduksjonspipeline**: Redaksjonelle arbeidsflyter med gjennomgang og godkjenningsstadier
- **Automatisering av kundeservice**: Flertrinns kundesaksbehandling
- **Databehandlingsarbeidsflyter**: ETL-arbeidsflyter med AI-drevet omforming

## ⚙️ Forutsetninger og oppsett

### 📦 **Nødvendige NuGet-pakker**

Denne arbeidsflytdemonstrasjonen bruker flere viktige .NET-pakker:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Azure OpenAI-konfigurasjon**

**Miljøoppsett (.env-fil):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Tilgang til Azure OpenAI:**
1. Opprett en Azure OpenAI-ressurs i Azure-portalen
2. Distribuer en modell (for eksempel `gpt-5-mini`) og noter distribusjonsnavnet
3. Logg inn med `az login` og konfigurer miljøvariablene som vist ovenfor

- **Utførelseskontekst**: Administrerer tilstand og dataflyt mellom arbeidsflytens faser

## 💻 Kjøring av koden

Den komplette implementeringen finnes i `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Denne filen viser:

1. **Miljøkonfigurasjon** - Laste Azure OpenAI-konfigurasjon fra `.env`-fil
2. **Azure OpenAI-klientoppsett** - Konfigurere klienten for å bruke Azure OpenAI Responses API
3. **Agentopprettelse** - Definere spesialiserte agenter (Front Desk og Concierge)
4. **Workflow Builder** - Lage en fleragent arbeidsflyt med sekvensiell behandling
5. **Arbeidsflytutførelse** - Kjøre arbeidsflyten med strømmede resultater

### 🚀 Kjøre eksemplet

```bash
# Gjør skriptet kjørbart (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Kjør arbeidsflyten
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Eller på Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Forventet resultat

Arbeidsflyten vil:
1. Godta din reisemålsforespørsel ("Jeg vil gjerne dra til Paris")
2. Front Desk-agenten gir en første anbefaling
3. Concierge-agenten gjennomgår og forbedrer anbefalingen
4. Sluttresultatet viser hele samtalestrømmen
