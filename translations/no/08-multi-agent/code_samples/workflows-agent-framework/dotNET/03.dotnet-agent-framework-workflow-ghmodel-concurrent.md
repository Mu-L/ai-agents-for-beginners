# ⚡ Samtidige agentarbeidsflyter med Azure OpenAI (Responses API) (.NET)

## 📋 Veiledning for høyytelses parallellbehandling

Denne notatblokken demonstrerer **samtidige arbeidsflytmønstre** ved bruk av Microsoft Agent Framework for .NET og Azure OpenAI (Responses API). Du vil lære hvordan du bygger høyytelses arbeidsflyter for parallell behandling som maksimerer gjennomstrømning ved å kjøre flere AI-agenter samtidig, samtidig som koordinering og datakonsistens opprettholdes.

### 🚀 **Grunnleggende om samtidig behandling**

- **Lastbalansering**: Fordel arbeidet effektivt på tilgjengelig samtidig behandlingskapasitet

## 💻 Kjøre koden

Den komplette implementeringen finnes i `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Denne filen demonstrerer en **Fan-Out/Fan-In samtidig arbeidsflyt** for reiseplanlegging:

### 🏗️ **Arkitektur for arbeidsflyt**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Nøkkelkomponenter:**

1. **ConcurrentStartExecutor**: Sender brukerforespørselen til alle agenter samtidig
2. **Researcher Agent**: Analyserer reisemål og attraksjoner parallelt
3. **Planner Agent**: Lager detaljert reiseplan samtidig
4. **ConcurrentAggregationExecutor**: Samler og slår sammen resultater fra begge agenter

### 🎯 **Fan-Out/Fan-In-mønster**

Denne arbeidsflyten demonstrerer det klassiske **Fan-Out/Fan-In**-mønsteret:
- **Fan-Out**: Én inngangsmelding sendes ut til flere agenter samtidig
- **Samtidig behandling**: Flere agenter arbeider parallelt på samme oppgave
- **Fan-In**: Resultater fra alle agenter samles og aggregeres til ett enkelt utdata

### 🔍 Forståelse av egendefinerte eksekutører

**ConcurrentStartExecutor:**
- Implementerer `IMessageHandler<string>` for å ta imot strenginput
- Sender meldinger til alle tilkoblede agenter
- Sender `TurnToken` for å trigge samtidig behandling

**ConcurrentAggregationExecutor:**
- Implementerer `IMessageHandler<ChatMessage>` for å motta agentresponser
- Samler meldinger på trådsikker måte
- Agregerer når alle forventede svar er mottatt
- Returnerer endelig resultat med `context.YieldOutputAsync()`

### ⚡ Ytelsesfordeler

**Samtidig kontra sekvensiell:**
- Sekvensiell: Agent1 (30s) → Agent2 (30s) = **60 sekunder totalt**
- Samtidig: Agent1 (30s) || Agent2 (30s) = **30 sekunder totalt**

**Forbedret gjennomstrømning**: Opptil N× raskere for N samtidige agenter (avhengig av arbeidsmengde og ressurser)

### 🛡️ Feilhåndtering

Arbeidsflyten håndterer individuelle agentfeil elegant:
- Hvis én agent feiler, fortsetter de andre behandlingen
- Aggregator kan implementere tidsavbruddslogikk
- Delvise resultater kan returneres hvis nødvendig
