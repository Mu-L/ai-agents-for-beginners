# AGENTS.md

ਇਸ ਰਿਪੋਜ਼ਿਟਰੀ ਵਿੱਚ "ਨਵਸੀਖਿਆਈਆਂ ਲਈ AI ਏਜੰਟ" ਸ਼ਾਮਲ ਹਨ - ਇੱਕ ਵਿਸਤਾਰਪੂਰਨ ਸਿੱਖਿਆਤਮਕ ਕੋਰਸ ਜੋ AI ਏਜੰਟ ਬਣਾਉਣ ਲਈ ਲੋੜੀਂਦੇ ਸਾਰੇ ਗੁਣ ਸਿਖਾਉਂਦਾ ਹੈ। ਕੋਰਸ ਵਿੱਚ 18 ਪਾਠਾਂ (ਨੰਬਰ 00-18) ਹਨ ਜਿਨ੍ਹਾਂ ਵਿੱਚ ਮੂਲ ਭੂਤਕ, ਡਿਜ਼ਾਇਨ ਪੈਟਰਨ, ਫ੍ਰੇਮਵਰਕ, ਉਤਪਾਦਨ ਤੈਯਾਰੀ, ਸਥਾਨਕ/ਡਿਵਾਈਸ ਉੱਤੇ ਏਜੰਟ ਅਤੇ AI ਏਜੰਟਾਂ ਦੀ ਸੁਰੱਖਿਆ ਆਉਂਦੀ ਹੈ।

**ਮੁੱਖ ਤਕਨੀਕਾਂ:**

- ਇੰਟਰਐਕਟਿਵ ਸਿੱਖਣ ਲਈ ਜੂਪਾਈਟਰ ਨੋਟਬੁੱਕ
- AI ਫ੍ਰੇਮਵਰਕ: ਮਾਈਕ੍ਰੋਸਾਫਟ ਏਜੰਟ ਫ੍ਰੇਮਵਰਕ (MAF)
- ਐਜ਼ਿਊਰ AI ਸੇਵਾਵਾਂ: ਮਾਈਕ੍ਰੋਸਾਫਟ ਫਾਉਂਡਰੀ, ਮਾਈਕ੍ਰੋਸਾਫਟ ਫਾਉਂਡਰੀ ਏਜੰਟ ਸੇਵਾ V2

**ਆਰਕੀਟੈਕਚਰ:**

- Microsoft Agent Framework ਵਰਤੋਂ ਕਰਨ ਵਾਲਾ ਹਰ ਪਾਠ ਲਈ ਇੱਕ ਪਾਇਥਨ ਨੋਟਬੁੱਕ

## ਸੈਟਅੱਪ ਕਮਾਂਡ

### ਜਰੂਰੀ ਸ਼ਰਤਾਂ
- ਪਾਇਥਨ 3.12 ਜਾਂ ਇਸ ਤੋਂ ਉੱਚਾ
- ਐਜ਼ਿਊਰ ਸਬਸਕ੍ਰਿਪਸ਼ਨ (Microsoft Foundry ਲਈ)
- ਐਜ਼ਿਊਰ CLI ਇੰਸਟੋਲਡ ਅਤੇ ਪ੍ਰਮਾਣਤ (`az login`)

### ਮੂਲ ਸੈਟਅੱਪ

1. **ਰੀਪੋਜ਼ਿਟਰੀ ਕਲੋਨ ਜਾਂ ਫੋਰਕ ਕਰੋ:**
   ```bash
   gh repo fork microsoft/ai-agents-for-beginners --clone
   # ਜਾਂ
   git clone https://github.com/microsoft/ai-agents-for-beginners.git
   cd ai-agents-for-beginners
   ```

2. **ਪਾਇਥਨ ਵਰਚੁਅਲ ਵਾਤਾਵਰਨ ਬਣਾਓ ਅਤੇ ਚਾਲੂ ਕਰੋ:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # ਵਿਂਡੋਜ਼ 'ਤੇ: venv\Scripts\activate
   ```

3. **ਨਿਰਭਰਤਾਵਾਂ ਇੰਸਟਾਲ ਕਰੋ:**
   ```bash
   pip install -r requirements.txt
   ```

4. **ਵਾਤਾਵਰਨ ਪਰਿਵਰਤਨਾਂ ਸੈੱਟ ਕਰੋ:**
   ```bash
   cp .env.example .env
   # ਆਪਣੀਆਂ API ਕੁੰਜੀਆਂ ਅਤੇ ਐਂਡਪੌਇੰਟਸ ਨਾਲ .env ਸੋਧੋ
   ```

### ਜਰੂਰੀ ਵਾਤਾਵਰਨ ਪਰਿਵਰਤਨ

Microsoft Foundry ਲਈ (ਲਾਜ਼ਮੀ):
- `AZURE_AI_PROJECT_ENDPOINT` - Microsoft Foundry ਪ੍ਰੋਜੈਕਟ ਐਂਡਪੌਇੰਟ
- `AZURE_AI_MODEL_DEPLOYMENT_NAME` - ਮਾਡਲ ਡਿਪਲੋਇਮੈਂਟ ਦਾ ਨਾਮ (ਜਿਵੇਂ gpt-5-mini)

Azure AI Search ਲਈ (ਪਾਠ 05 - RAG):
- `AZURE_SEARCH_SERVICE_ENDPOINT` - Azure AI Search ਐਂਡਪੌਇੰਟ
- `AZURE_SEARCH_API_KEY` - Azure AI Search API ਕੁੰਜੀ

ਪ੍ਰਮਾਣੀਕਰਣ: ਨੋਟਬੁੱਕ ਚਲਾਉਣ ਤੋਂ ਪਹਿਲਾਂ `az login` ਚਲਾਓ (ਇਸ ਵਿੱਚ `AzureCliCredential` ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ).

## ਵਿਕਾਸ ਵਰਕਫਲੋ

### ਜੂਪਾਈਟਰ ਨੋਟਬੁੱਕ ਚਲਾਉਣਾ

1. **ਜੂਪਾਈਟਰ ਸ਼ੁਰੂ ਕਰੋ:**
   ```bash
   jupyter notebook
   ```

2. **ਇੱਕ ਪਾਠ ਡਾਇਰੈਕਟਰੀ 'ਚ ਜਾਓ** (ਉਦਾਹਰਣ ਲਈ: `01-intro-to-ai-agents/code_samples/`)

3. **ਨੋਟਬੁੱਕ ਖੋਲ੍ਹੋ ਅਤੇ ਚਲਾਓ:**
   - `*-python-agent-framework.ipynb` - Microsoft Agent Framework (ਪਾਇਥਨ) ਵਰਤੋਂ ਕਰਦੇ ਹੋਏ
   - `*-dotnet-agent-framework.ipynb` - Microsoft Agent Framework (.NET) ਵਰਤੋਂ ਕਰਦੇ ਹੋਏ

### Microsoft Agent Framework ਨਾਲ ਕੰਮ ਕਰਨਾ

Microsoft Agent Framework + Microsoft Foundry:
- ਐਜ਼ਿਊਰ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਲੋੜੀਂਦਾ ਹੈ
- Agent Service V2 ਲਈ `FoundryChatClient` ਵਰਤਦਾ ਹੈ (ਏਜੰਟ ਫਾਉਂਡਰੀ ਪੋਰਟਲ ਵਿੱਚ ਦਿਖਾਈ ਦਿੰਦੇ ਹਨ)
- ਬਿਲਟ-ਇਨ ਨਿਰੀਖਣ ਯੋਗਤਾ ਨਾਲ ਉਤਪਾਦਨ ਤਯਾਰ
- ਫਾਈਲ ਪੈਟਰਨ: `*-python-agent-framework.ipynb`


2. **ਨੋਟਬੁੱਕ ਚੱਲਾਉਣ ਦੀ ਜਾਂਚ ਕਰੋ:**

### ਵੱਖ-ਵੱਖ ਨੋਟਬੁੱਕ ਚਲਾਉਣਾ

- ਸੰਰਚਨਾ ਲੋਡਿੰਗ
- ਉਦਾਹਰਣ ਦੇ ਤੌਰ ਤੇ ਏਜੰਟਾਂ ਦੇ ਅਮਲ
- ਮਾਰਕਡਾਊਨ ਸੈੱਲ ਵਿੱਚ ਉਮੀਦ ਵਾਲੇ ਨਤੀਜੇ

### ਤੈਯਾਰ ਕੀਤੇ ਗਏ ਏਜੰਟਾਂ ਦੀ ਸਿਗਰਟ ਜਾਂਚ

ਜਿਨ੍ਹਾਂ ਪਾਠਾਂ ਵਿੱਚ ਮਾਈਕ੍ਰੋਸਾਫਟ ਫਾਉਂਡਰੀ ਹੋਸਟ ਕੀਤਾ ਏਜੰਟ ਹੁੰਦਾ ਹੈ (01, 04, 05, 16), ਰਿਪੋ ਵਿੱਚ `tests/` ਦੇ ਹੇਠਾਂ ਸਿਗਰਟ-ਟੈਸਟ ਕੈਟਾਲੌਗਜ਼ ਮੌਜੂਦ ਹਨ ਜੋ `.github/workflows/smoke-test.yml` ਵਰਕਫਲੋ ਰਾਹੀਂ [AI Smoke Test](https://github.com/marketplace/actions/ai-smoke-test) ਕਾਰਵਾਈ ਵਰਤ ਕੇ ਚਲਾਏ ਜਾਂਦੇ ਹਨ। ਇਹ ਹਲਕੇ ਫੁਲਕੇ ਗੇਟ ਹਨ ਮੂਲ ਤੈਯਾਰੀ ਤੋਂ ਬਾਅਦ (ਕੀ ਏਜੰਟ ਪੁੱਜਣਯੋਗ ਹੈ ਅਤੇ ਮੂਲ ਪ੍ਰੈਂਪਟ ਉਮੀਦਾਂ ਨੂੰ ਪਾਲਣ ਕਰ ਰਿਹਾ ਹੈ?), ਜੋ ਪਾਠ 10 ਅਤੇ 16 ਵਿੱਚ ਮੁੱਲਾਂਕਣ ਪਾਈਪਲਾਈਨ ਨੂੰ ਪੂਰਾ ਕਰਦੇ ਹਨ। ਕੈਟਾਲੌਗ-ਤੋਂ-ਪਾਠ-ਤੋਂ-ਏਜੰਟ ਨਕਸ਼ਾ ਲਈ [tests/README.md](./tests/README.md) ਵੇਖੋ। ਪਾਠ 17 ਸਥਾਨਕ Foundry Local ਨਾਲ ਚੱਲਦਾ ਹੈ ਅਤੇ ਇਸ ਦਾ ਕੋਈ ਹੋਸਟ ਕੀਤਾ ਐਂਡਪੌਇੰਟ ਨਹੀਂ ਹੈ, ਇਸ ਲਈ ਇਹ ਆਪਣਾ ਨੋਟਬੁੱਕ ਸਿੱਧਾ ਚਲਾ ਕੇ ਜਾਂਚਿਆ ਜਾਂਦਾ ਹੈ।

### ਆਮ ਮੁੱਦੇ

1. **ਪਾਇਥਨ ਵਰਜਨ ਦਾ ਅੰਤਰ:**
   - ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਪਾਇਥਨ 3.12+ ਵਰਤ ਰਹੇ ਹੋ

   - `.env` ਫਾਈਲ ਕਮਿਟ ਨਾ ਕਰੋ (ਇਹ `.gitignore` ਵਿੱਚ ਹੈ)

3. **ਪੈਕੇਜ ਟਕਰਾਅ:**
   - ਤਾਜ਼ਾ ਵਰਚੁਅਲ ਵਾਤਾਵਰਨ ਵਰਤੋਂ
   - ਇਕ ਇਕ ਕਰਕੇ ਪੈਕੇਜਾਂ ਦੀ ਥਾਂ `requirements.txt` ਤੋਂ ਇੰਸਟਾਲ ਕਰੋ
   - ਕੁਝ ਨੋਟਬੁੱਕ ਲਈ ਜਿਵੇਂ ਮਾਰਕਡਾਊਨ ਸੈੱਲਾਂ ਵਿੱਚ ਜ਼ਿਕਰ ਕੀਤੇ ਵਾਧੂ ਪੈਕੇਜ ਦੀ ਲੋੜ ਹੋ ਸਕਦੀ ਹੈ

   - ਯਕੀਨ ਕਰੋ ਕਿ ਤੁਹਾਡਾ ਐਜ਼ਿਊਰ ਓਪਨAI ਮਾਡਲ ਡਿਪਲੋਇਮੈਂਟ Responses API ਦਾ ਸਮਰਥਨ ਕਰਦਾ ਹੈ
