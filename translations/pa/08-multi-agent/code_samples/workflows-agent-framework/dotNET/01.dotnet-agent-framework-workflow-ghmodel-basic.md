# 🔄 ਬੇਸਿਕ ਏਜੰਟ ਵਰਕਫਲੋਜ਼ ਵਿਥ ਏਜ਼ਯੂਰ ਓਪਨਐਆਈ (Responses API) (.NET)

## 📋 ਵਰਕਫਲੋ ਓਰਚੈਸਟਰਏਸ਼ਨ ਟਿਊਟੋਰਿਅਲ

ਇਹ ਨੋਟਬੁੱਕ ਦਿਖਾਉਂਦੀ ਹੈ ਕਿ ਕਿਵੇਂ ਮਾਈਕ੍ਰੋਸੌਫਟ ਏਜੰਟ ਫਰੇਮਵਰਕ ਫਾਰ .NET ਅਤੇ ਏਜ਼ਯੂਰ ਓਪਨਐਆਈ (Responses API) ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਸੁਧਰੇ ਹੋਏ **ਏਜੰਟ ਵਰਕਫਲੋਜ਼** ਬਣਾਏ ਜਾਣ। ਤੁਸੀਂ ਮਲਟੀ-ਸਟੈਪ ਬਿਜ਼ਨਸ ਪ੍ਰੋਸੈਸ ਬਣਾਉਣਾ ਸਿੱਖੋਗੇ ਜਿੱਥੇ AI ਏਜੰਟ ਸੰਰਚਿਤ ਓਰਚੈਸਟਰਏਸ਼ਨ ਪੈਟਰਨਜ਼ ਰਾਹੀਂ ਮਿਲ ਕੇ ਮੁਸ਼ਕਲ ਕੰਮ ਪੂਰੇ ਕਰਦੇ ਹਨ।

## 🎯 ਸਿੱਖਣ ਦੇ ਉਦੇਸ਼

### 🏗️ **ਵਰਕਫਲੋ ਢਾਂਚਾ ਦੇ ਮੂਲ ਤੱਤ**
- **ਵਰਕਫਲੋ ਬਿਲਡਰ**: ਸੁਧਰੇ ਹੋਏ ਮਲਟੀ-ਸਟੈਪ AI ਪ੍ਰਕਿਰਿਆਵਾਂ ਨੂੰ ਡਿਜ਼ਾਈਨ ਅਤੇ ਓਰਚੈਸਟਰ ਕਰੋ
- **ਏਜੰਟ ਸਮਨਵਯ**: ਵਰਕਫਲੋਜ਼ ਵਿੱਚ ਕਈ ਕੁਸ਼ਲ ਏਜੰਟਾਂ ਦਾ ਢੰਗ ਨਾਲ ਸਹਿਯੋਗ
- **ਏਜ਼ਯੂਰ ਓਪਨ ਐਆਈ (Responses API)**: ਵਰਕਫਲੋਜ਼ ਵਿੱਚ ਏਜ਼ਯੂਰ ਓਪਨਐਆਈ Responses API ਦੀ ਵਰਤੋਂ
- **ਦ੍ਰਿਸ਼ਟੀਗਤ ਵਰਕਫਲੋ ਡਿਜ਼ਾਈਨ**: ਵਰਕਫਲੋ ਢਾਂਚਿਆਂ ਦੀ ਰਚਨਾ ਅਤੇ ਵਿਜ਼ੂਅਲ ਪ੍ਰਸਤੁਤੀ ਲਈ

### 🔄 **ਪ੍ਰਕਿਰਿਆ ਓਰਚੈਸਟਰਏਸ਼ਨ ਪੈਟਰਨਜ਼**
- **ਕ੍ਰਮਬੱਧ ਸੂਚਨਾ ਪ੍ਰਕਿਰਿਆਵਾਂ**: ਤਰਕ ਸੂਚੀਬੱਧ ਕਈ ਏਜੰਟ ਕੰਮਾਂ ਨੂੰ ਜੁੜਨਾ
- **ਸਟੇਟ ਪ੍ਰਬੰਧਨ**: ਵਰਕਫਲੋ ਮੰਚਾਂ ਵਿੱਚ ਸੰਦਰਭ ਅਤੇ ਡਾਟਾ ਪ੍ਰਵਾਹ ਸਥਾਪਿਤ ਕਰਨਾ
- **ਗਲਤੀ ਸੰਭਾਲਣ**: ਮਜ਼ਬੂਤ ਗਲਤੀ ਮੁਆਵਜ਼ਾ ਅਤੇ ਵਰਕਫਲੋ ਲਚੀਲਾਪਣ ਲਾਗੂ ਕਰਨਾ
- **ਕਾਰਗੁਜ਼ਾਰੀ ਸੁਧਾਰ**: ਐਂਟਰਪ੍ਰਾਈਜ਼-ਪੱਧਰੀ ਕਾਰਜਾਂ ਲਈ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਵਰਕਫਲੋਜ਼ ਢਾਂਚਾ

### 🏢 **ਐਂਟਰਪ੍ਰਾਈਜ਼ ਵਰਕਫਲੋ ਐਪਲੀਕੇਸ਼ਨਜ਼**
- **ਬਿਜ਼ਨਸ ਪ੍ਰੋਸੈਸ ਆਟੋਮੇਸ਼ਨ**: ਸਰਕਾਰੀ ਪੱਧਰ ਦੇ ਕਠਿਨ ਵਰਕਫਲੋਜ਼ ਦਾ ਆਟੋਮੇਸ਼ਨ
- **ਸਮੱਗਰੀ ਉਤਪਾਦਨ ਪਾਈਪਲਾਈਨ**: ਸੰਪਾਦਕੀ ਵਰਕਫਲੋਜ ਵਿਖੇ ਸਮੀਖਿਆ ਅਤੇ ਮਨਜ਼ੂਰੀ ਮੰਚ
- **ਗਾਹਕ ਸੇਵਾ ਆਟੋਮੇਸ਼ਨ**: ਬਹੁ-ਪਦਵੀ ਗਾਹਕ ਪੁੱਛ-ਗਿੱਛ ਹੱਲ ਕਰਨ ਵਾਲੀ ਪ੍ਰਕਿਰਿਆ
- **ਡਾਟਾ ਪ੍ਰੋਸੈਸਿੰਗ ਵਰਕਫਲੋਜ਼**: AI-ਸক্ষম ਪਰਿਵਰਤਨ ਵਾਲੇ ETL ਵਰਕਫਲੋਜ਼

## ⚙️ ਪਹਿਲਾਂ ਤਿਆਰੀ ਅਤੇ ਸੈਟਅੱਪ

### 📦 **ਜ਼ਰੂਰੀ NuGet ਪੈਕੇਜ**

ਇਸ ਵਰਕਫਲੋ ਪ੍ਰਦਰਸ਼ਨ ਵਿੱਚ ਕਈ ਮੁੱਖ .NET ਪੈਕੇਜ ਵਰਤੇ ਗਏ ਹਨ:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **ਏਜ਼ਯੂਰ ਓਪਨਐਆਈ ਸੰਰਚਨਾ**

**ਮਾਹੌਲ ਸੈਟਅੱਪ (.env ਫਾਇਲ):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**ਏਜ਼ਯੂਰ ਓਪਨਐਆਈ ਐਕਸੈੱਸ:**
1. ਏਜ਼ਯੂਰ ਪੋਰਟਲ ਵਿੱਚ ਏਜ਼ਯੂਰ ਓਪਨਐਆਈ ਸਰੋਤ ਬਣਾਓ
2. ਇੱਕ ਮਾਡਲ ਜਿਵੇਂ `gpt-4.1-mini` ਤਾਇਨਾਤ ਕਰੋ ਅਤੇ ਤਾਇਨਾਤੀ ਨਾਮ ਲਿਖੋ
3. `az login` ਨਾਲ ਸਾਈਨ ਇਨ ਕਰੋ ਅਤੇ ਉਪਰ ਦਿੱਤੇ ਤਰ੍ਹਾਂ ਮਾਹੌਲ ਵੇਰੀਏਬਲ ਸੰਰਚਿਆ ਕਰੋ

### 🏗️ **ਵਰਕਫਲੋ ਢਾਂਚਾ ਦਾ ਜਾਇਜ਼ਾ**

```mermaid
graph TD
    A[ਵਰਕਫਲੋ ਬਿਲਡਰ] --> B[ਏਜੰਟ ਰਜਿਸਟਰੀ]

    G[ਅਜ਼ੂਰ ਓਪਨਏਆਈ (ਜਵਾਬਾਂ API)] --> D
```

**ਮੁੱਖ ਭਾਗ:**

- **AIAgent**: ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਵਾਲੇ ਵਿਅਕਤੀਗਤ ਏਜੰਟ
- **ਏਜ਼ਯੂਰ ਓਪਨਐਆਈ ਕਲਾਇੰਟ**: ਏਜ਼ਯੂਰ ਓਪਨਐਆਈ Responses API ਇਕਤੀਬੱਧਤਾ
- **ਕਾਰਜ ਸੰਦਰਭ**: ਵਰਕਫਲੋ ਮੰਚਾਂ ਵਿਚਕਾਰ ਸਥਿਤੀ ਅਤੇ ਡਾਟਾ ਪ੍ਰਵਾਹ ਸੰਭਾਲਦਾ ਹੈ

```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

- **ਨਿਯਤਕਾਰਿਤ ਕਾਰਜਨਵਾਈ**: ਲਗਾਤਾਰ, ਦੁਹਰਾਏ ਜਾ ਸਕਣ ਵਾਲੇ ਵਰਕਫਲੋ ਨਤੀਜੇ
- **ਗਲਤੀ ਮੁਆਵਜ਼ਾ**: ਕਿਸੇ ਵੀ ਵਰਕਫਲੋ ਮੰਚ 'ਤੇ ਨਾਕਾਮੀਆਂ ਦਾ ਨਰਮ ਸੰਭਾਲ
- **ਕਾਰਗੁਜ਼ਾਰੀ ਨਿਗਰਾਨੀ**: ਕਾਰਜਨਵਾਈ ਮਾਪਦੰਡ ਅਤੇ ਸੁਧਾਰ ਦੇ ਮੌਕੇ ਟਰੈਕ ਕਰੋ
- **ਸੰਸਾਧਨ ਪ੍ਰਬੰਧ**: AI ਮਾਡਲ ਸਾਂਝੇਦਾਰੀ ਲਈ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਵਰਤੋਂ ਅਤੇ ਵਰਤੋਂ

### 🔒 **ਸੁਰੱਖਿਆ ਅਤੇ ਅਨੁਕੂਲਤਾ**
- **ਸੁਰੱਖਿਅਤ ਪ੍ਰਮਾਣੀਕਰਨ**: Microsoft Entra ID ਪ੍ਰਮਾਣੀਕਰਨ `az login` (AzureCliCredential) ਰਾਹੀਂ
- **ਆਡਿਟ ਟ੍ਰੇਲਜ਼**: ਵਰਕਫਲੋ ਕਾਰਜਨਵਾਈ ਅਤੇ ਫੈਸਲੇ ਦੇ ਬਿੰਦੂਆਂ ਦੀ ਪੂਰੀ ਲਾਗਿੰਗ
- **ਐਕਸੈੱਸ ਕੰਟਰੋਲ**: ਵਰਕਫਲੋ ਕਾਰਜਨਵਾਈ ਅਤੇ ਨਿਗਰਾਨੀ ਲਈ ਵਿਸ਼ੇਸ਼ ਅਧਿਕਾਰ
- **ਡਾਟਾ ਪ੍ਰਾਈਵੇਸੀ**: ਵਰਕਫਲੋਜ਼ ਸਮੇਂ ਸੰਵੇਦਨਸ਼ੀਲ ਜਾਣਕਾਰੀ ਦਾ ਸੁਰੱਖਿਅਤ ਸੰਭਾਲ

- **ਕਾਰਜਨਵਾਈ ਨਿਗਰਾਨੀ**: ਵਰਕਫਲੋ ਪ੍ਰਗਤੀ ਅਤੇ ਕਾਰਗੁਜ਼ਾਰੀ ਦੀ ਪ੍ਰਤੀਕ੍ਰਮਿਕ ਨਿਗਰਾਨੀ
- **ਗਲਤੀ ਰਿਪੋਰਟਿੰਗ**: ਵਿਸਥਾਰਤ ਗਲਤੀ ਵਿਸ਼ਲੇਸ਼ਣ ਅਤੇ ਡੀਬੱਗਿੰਗ ਸਮਰੱਥਾ
- **ਕਾਰਗੁਜ਼ਾਰੀ ਵਿਸ਼ਲੇਸ਼ਣ**: ਸੁਧਾਰ ਅਤੇ ਸਮਰੱਥਾ ਯੋਜਨਾ ਲਈ ਮਾਪਦੰਡ

## 💻 ਕੋਡ ਚਲਾਉਣਾ

ਪੂਰੀ ਲਾਗੂਆਤ `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` ਵਿੱਚ ਉਪਲਬਧ ਹੈ। ਇਹ ਫਾਇਲ ਦਰਸਾਉਂਦੀ ਹੈ:

1. **ਮਾਹੌਲ ਸੰਰਚਨਾ** - `.env` ਫਾਇਲ ਤੋਂ ਏਜ਼ਯੂਰ ਓਪਨਐਆਈ ਸੰਰਚਨਾ ਲੋਡ ਕਰਨਾ
2. **ਏਜ਼ਯੂਰ ਓਪਨਐਆਈ ਕਲਾਇੰਟ ਸੈਟਅੱਪ** - ਕਲਾਇੰਟ ਨੂੰ ਏਜ਼ਯੂਰ ਓਪਨਐਆਈ Responses API ਦੀ ਵਰਤੋਂ ਲਈ ਸੰਰਚਿਤ ਕਰਨਾ
3. **ਏਜੰਟ ਸਿਰਜਣਾ** - ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਵਾਲੇ ਏਜੰਟ (ਫਰੰਟ ਡੈਸਕ ਅਤੇ ਕਾਂਸੀਅਰਜ) ਨੂੰ ਪਰਿਭਾਸ਼ਿਤ ਕਰਨਾ
4. **ਵਰਕਫਲੋ ਬਿਲਡਰ** - ਕ੍ਰਮਬੱਧ ਪ੍ਰਕਿਰਿਆਂ ਨਾਲ ਇੱਕ ਬਹੁ-ਏਜੰਟ ਵਰਕਫਲੋ ਬਣਾਉਣਾ
5. **ਵਰਕਫਲੋ ਚਾਲੂ** - ਸਟ੍ਰੀਮਿੰਗ ਨਤੀਜਿਆਂ ਨਾਲ ਵਰਕਫਲੋ ਚਲਾਉਣਾ

### 🚀 ਉਦਾਹਰਨ ਚਲਾਉਣਾ

```bash
# ਸਕ੍ਰਿਪਟ ਨੂੰ ਚਲਾਉਣਯੋਗ ਬਣਾਓ (ਯੂਨਿਕਸ/ਲਿਨਕਸ/ਮੈਕਓਐਸ)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# ਵਰਕਫਲੋ ਚਲਾਓ
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

ਜਾਂ ਵਿੰਡੋਜ਼ 'ਤੇ:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```
