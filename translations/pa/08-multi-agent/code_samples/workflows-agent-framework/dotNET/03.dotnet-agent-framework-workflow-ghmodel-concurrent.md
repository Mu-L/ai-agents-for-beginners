# ⚡ ਅਜ਼ੂਰੇ ਓਪਨਏਆਈ (Responses API) ਨਾਲ ਸਾਂਝਾ ਏਜੰਟ ਵਰਕਫਲੋ (.NET)

## 📋 ਹਾਈ-ਪਰਫਾਰਮੈਂਸ ਸਮਾਂਤਰ ਪ੍ਰੋਸੈਸਿੰਗ ਟਿਊਟੋਰਿਅਲ

ਇਹ ਨੋਟਬੁੱਕ **ਸਾਂਝੇ ਵਰਕਫਲੋ ਪੈਟਰਨ** ਦਿਖਾਉਂਦਾ ਹੈ ਜੋ ਕਿ .NET ਲਈ ਮਾਈਕ੍ਰੋਸੌਫਟ ਏਜੰਟ ਫਰੇਮਵਰਕ ਅਤੇ ਅਜ਼ੂਰੇ ਓਪਨਏਆਈ (Responses API) ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ। ਤੁਸੀਂ ਸਿੱਖੋਗੇ ਕਿ ਕਿਵੇਂ ਹਾਈ-ਪਰਫਾਰਮੈਂਸ, ਸਮਾਂਤਰ ਪ੍ਰੋਸੈਸਿੰਗ ਵਰਕਫਲੋ ਬਣਾਏ ਜਾ ਸਕਦੇ ਹਨ ਜੋ ਬਹੁਤ ਸਾਰੇ ਏਆਈ ਏਜੰਟਾਂ ਨੂੰ ਇੱਕਸਾਥ ਚਲਾਉਂਦੇ ਹੋਏ ਸਾਂਝਾ ਸਮਨਵਯ ਅਤੇ ਡਾਟਾ ਸਟੀਰਤਾ ਨੂੰ ਬਣਾਈ ਰੱਖਦੇ ਹਨ।

## 🎯 ਸਿੱਖਣ ਦੇ ਲਕੜੀ

### 🚀 **ਸਾਂਝੀ ਪ੍ਰੋਸੈਸਿੰਗ ਦੇ ਮੁੱਲ-ਤੱਤ**
- **ਸਮਾਂਤਰ ਏਜੰਟ ਨਿਭਾਉਣਾ**: ਵੱਧ ਤੋਂ ਵੱਧ ਕਾਰਗੁਜ਼ਾਰੀ ਲਈ ਕਈ ਏਆਈ ਏਜੰਟ ਇੱਕੱਠੇ ਚਲਾਉਣਾ
- **Async/Await ਪੈਟਰਨ**: ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਸਮਾਂਤਰਤਾ ਲਈ .NET ਦੇ async ਪ੍ਰੋਗ੍ਰਾਮਿੰਗ ਮਾਡਲ ਦੀ ਵਰਤੋਂ ਕਰਨਾ
- **ਅਜ਼ੂਰੇ ਓਪਨਏਆਈ (Responses API)**: ਅਜ਼ੂਰੇ ਓਪਨਏਆਈ Responses API ਨੂੰ ਕਈ ਸਾਂਝੇ ਕਾਲਾਂ ਨਾਲ ਕੂਆਰਡੀਨੇਟ ਕਰਨਾ
- **ਸੰਸਾਧਨ ਪ੍ਰਬੰਧਨ**: ਸਾਂਝੇ ਓਪਰੇਸ਼ਨਾਂ ਵਿੱਚ ਏਆਈ ਮਾਡਲ ਸੰਸਾਧਨਾਂ ਦਾ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਪ੍ਰਬੰਧਨ

### 🏗️ **ਉੱਨਤ ਸਮਾਂਤਰਤਾ ਆਰਕੀਟੈਕਚਰ**
- **ਟਾਸਕ ਆਧਾਰਿਤ ਸਮਾਂਤਰਤਾ**: ਅਦ terbaik concurrent execution ਲਈ .NET ਟਾਸਕ ਪੈਰਲਲ ਲਾਇਬ੍ਰੇਰੀ ਦੀ ਵਰਤੋਂ
- **ਸੰਕਲਨ ਪੈਟਰਨ**: ਰੇਸ ਕੰਡੀਸ਼ਨਾਂ ਤੋਂ ਬਚਦਿਆਂ concurrent ਏਜੰਟਾਂ ਦਾ ਸਮਨਵਯ
- **ਲੋਡ ਬੈਲੈਂਸਿੰਗ**: ਉਪਲਬਧ concurrent ਪ੍ਰੋਸੈਸਿੰਗ ਸਮਰੱਥਾ ਵਿੱਚ ਕੰਮ ਦੀ ਬਕਰੀ
- **ਫੌਲਟ ਟੋਲਰੈਂਸ**: ਪੂਰੇ ਵਰਕਫਲੋ ਨੂੰ ਰੋਕਣ ਬਗੈਰ ਅਲੱਗ-ਅਲੱਗ ਏਜੰਟ ਫੇਲਿਅਰਾਂ ਨੂੰ ਹੈਂਡਲ ਕਰਨਾ

### 🏢 **ਏਂਟਰਪ੍ਰਾਈਜ਼ Concurrent ਐਪਲੀਕੇਸ਼ਨਸ**
- **ਉੱਚ-ਪ੍ਰਮਾਣ ਡੌਕਿਊਮੈਂਟ ਪ੍ਰੋਸੈਸਿੰਗ**: ਕਈ ਡੌਕਿਊਮੈਂਟ ਇੱਕੱਠੇ ਪ੍ਰੋਸੈਸ ਕਰਨਾ
- **ਰੀਅਲ-ਟਾਈਮ ਸਮੱਗਰੀ ਵਿਸ਼ਲੇਸ਼ਣ**: ਆਉਂਦੇ ਡਾਟਾ ਸਟ੍ਰੀਮਾਂ ਦੀ concurrent ਵਿਸ਼ਲੇਸ਼ਣ
- **ਬੈਚ ਪ੍ਰੋਸੈਸਿੰਗ ਦਾ ਆਪਟੀਮਾਈਜ਼ਸ਼ਨ**: ਵੱਡੇ ਪੱਧਰ ਦੇ ਡਾਟਾ ਪ੍ਰੋਸੈਸਿੰਗ ਲਈ throughput ਵਧਾਉਣਾ
- **ਮਲਟੀ-ਮੋਡਲ ਵਿਸ਼ਲੇਸ਼ਣ**: ਵੱਖ-ਵੱਖ ਸਮੱਗਰੀ ਦੀਆਂ ਕਿਸਮਾਂ ਅਤੇ ਫਾਰਮੈਟਾਂ ਦੀ concurrent ਪ੍ਰੋਸੈਸਿੰਗ

## 🏢 **ਏਂਟਰਪ੍ਰਾਈਜ਼ ਪ੍ਰਦਰਸ਼ਨ ਲਾਭ**

### ⚡ **ਥਰੂਪੁੱਟ ਅਤੇ ਸਕੇਲਬਿਲਟੀ**
- **ਰੇਖੀਅਨ ਪ੍ਰਦਰਸ਼ਨ ਸਕੇਲਿੰਗ**: throughput ਵਧਾਉਣ ਲਈ ਹੋਰ concurrent ਏਜੰਟ ਸ਼ਾਮਲ ਕਰੋ
- **ਸੰਸਾਧਨ ਵਰਤੋਂ**: ਉਪਲਬਧ ਏਆਈ ਮਾਡਲ ਸਮਰੱਥਾ ਦੀ ਵੱਧ ਤੋਂ ਵੱਧ ਕਾਰਗੁਜ਼ਾਰੀ
- **ਪ੍ਰੋਸੈਸਿੰਗ ਸਮੇਂ ਵਿੱਚ ਕਮੀ**: ਸਮਾਂਤਰ ਕਾਰਜਨ ਨਾਲ ਮਹੱਤਵਪੂਰਨ ਸਮਾਂ ਘਟਾਅ
- **ਵਿਆਪਕ ਸਕੇਲਿੰਗ**: ਕੰਮ ਦੇ ਭਾਰ ਦੇ ਅਧਾਰ 'ਤੇ concurrent ਏਜੰਟਾਂ ਦੀ ਗਿਣਤੀ ਨੂੰ ਗਤਿਸੀਲ ਤੌਰ 'ਤੇ ਢਾਲੋ

### 🛡️ **ਭਰੋਸੇਯੋਗਤਾ ਅਤੇ ਲਚਕੀਲਾਪਨ**
- **ਫੌਲਟ ਆਈਸੋਲੇਸ਼ਨ**: ਇੱਕ ਏਜੰਟ ਦੀ ਅਸਫਲਤਾ ਹੋਣ 'ਤੇ ਹੋਰ concurrent ਓਪਰੇਸ਼ਨਾਂ 'ਤੇ ਅਸਰ ਨਹੀਂ ਪੈਂਦਾ
- **ਨਰਮ ਘਟਾਅ**: 시스템 ਘੱਟ ਏਜੰਟ ਸਮਰੱਥਾ ਨਾਲ ਵੀ ਕੰਮ ਜਾਰੀ ਰੱਖਦਾ ਹੈ
- **ਅਤੇਕਮ ਸੁਧਾਰ**: ਗਲਤ concurrent ਓਪਰੇਸ਼ਨਾਂ ਲਈ ਸਵੈਚਾਲਿਤ ਮੁੜ-ਕੋਸ਼ਿਸ਼ ਕਿਰਿਆਵਾਂ
- **ਲੋਡ ਵੰਡ**: ਉਪਲਬਧ ਏਜੰਟਾਂ ਵਿੱਚ ਕੰਮ ਦੀ ਸਮਾਂਤਰ ਵੰਡ

### 📊 **ਪ੍ਰਦਰਸ਼ਨ ਨਿਗਰਾਨੀ**
- **Concurrent ਨਿਭਾੳੁਂ ਮੈਟ੍ਰਿਕਸ**: ਸਾਰੇ ਸਮਾਂਤਰ ਓਪਰੇਸ਼ਨਾਂ ਦਾ ਪ੍ਰਦਰਸ਼ਨ ਟ੍ਰੈਕ ਕਰੋ
- **ਸੰਸਾਧਨ ਵਰਤੋਂ ਵਿਸ਼ਲੇਸ਼ਣ**: CPU, ਮੈਮੋਰੀ, ਅਤੇ ਨੈਟਵਰਕ ਦੀ ਵਰਤੋਂ ਦੀ ਨਿਗਰਾਨੀ
- **ਥਰੂਪੁੱਟ ਵਿਸ਼ਲੇਸ਼ਣ**: concurrent ਪ੍ਰੋਸੈਸਿੰਗ ਤੋਂ ਕਾਰਗੁਜ਼ਾਰੀ ਵਿੱਚ ਲਾਭ ਦੀ ਮਾਪ
- **ਬੋਟਲਨੇਕ ਪਛਾਣ**: ਪ੍ਰਦਰਸ਼ਨ ਰੁਕਾਵਟਾਂ ਦੀ ਪਛਾਣ ਅਤੇ ਹੱਲ

### 🔧 **ਡਿਵੈਲਪਮੈਂਟ ਅਤੇ ਓਪਰੇਸ਼ਨਸ**
- **Async ਪ੍ਰੋਗ੍ਰਾਮਿੰਗ ਮਾਡਲ**: .NET ਦੇ ਮੈਚੂਰ async/await ਪੈਟਰਨ ਦੀ ਵਰਤੋਂ
- **ਟਾਸਕ ਕੋਆਰਡੀਨੇਸ਼ਨ**: ਜਮ੍ਹਾਂਟਾਸਕ ਪ੍ਰਬੰਧਨ ਅਤੇ ਸਮਨਵਯ ਸਮਰੱਥਾਵਾਂ
- **Exception ਹੈਂਡਲਿੰਗ**: concurrent ਓਪਰੇਸ਼ਨਾਂ ਲਈ ਪੂਰੀ ਤਰ੍ਹਾਂ ਗੜਬੜ ਸੰਭਾਲ
- **ਡੀਬੱਗਿੰਗ ਸਹਾਇਤਾ**: concurrent ਵਰਕਫਲੋ ਲਈ ਵਿਜੁਅਲ ਸਟੂਡੀਓ ਡੀਬੱਗਿੰਗ ਟੂਲਜ਼

## 💻 ਕੋਡ ਚਲਾਉਣਾ

ਪੂਰੀ ਇੰਪਲੀਮੈਂਟੇਸ਼ਨ `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs` ਵਿੱਚ ਉਪਲਬਧ ਹੈ। ਇਹ ਫ਼ਾਈਲ ਯਾਤਰਾ ਯੋਜਨਾ ਲਈ ਇੱਕ **Fan-Out/Fan-In concurrent ਵਰਕਫਲੋ** ਦਿਖਾਉਂਦੀ ਹੈ:

### 🏗️ **ਵਰਕਫਲੋ ਆਰਕੀਟੈਕਚਰ**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**ਮੁੱਖ ਹਿੱਸੇ:**

1. **ConcurrentStartExecutor**: ਯੂਜ਼ਰ ਦੀ ਬੇਨਤੀ ਸਭ ਏਜੰਟਾਂ ਨੂੰ ਇੱਕੱਠੇ ਭੇਜਦਾ ਹੈ
2. **Researcher Agent**: concurrent ਤੌਰ 'ਤੇ ਮੰਜ਼ਿਲਾਂ ਅਤੇ ਆਕਰਸ਼ਣਾਂ ਦਾ ਵਿਸ਼ਲੇਸ਼ਣ ਕਰਦਾ ਹੈ
3. **Planner Agent**: concurrent ਯਾਤਰਾ ਯੋਜਨਾਵਾਂ ਬਣਾਉਂਦਾ ਹੈ
4. **ConcurrentAggregationExecutor**: ਦੋਹਾਂ ਏਜੰਟਾਂ ਤੋਂ ਨਤੀਜੇ ਇਕੱਠੇ ਕਰਦਾ ਅਤੇ ਜੋੜਦਾ ਹੈ

### 🎯 **Fan-Out/Fan-In ਪੈਟਰਨ**

ਇਹ ਵਰਕਫਲੋ ਪ੍ਰਚਲਿਤ **Fan-Out/Fan-In** ਪੈਟਰਨ ਦਿਖਾਉਂਦਾ ਹੈ:
- **Fan-Out**: ਇੱਕ ਇਨਪੁੱਟ ਸੁਨੇਹਾ ਕਈ ਏਜੰਟਾਂ ਨੂੰ ਇੱਕੱਠੇ ਭੇਜਿਆ ਜਾਂਦਾ ਹੈ
- **Concurrent ਪ੍ਰੋਸੈਸਿੰਗ**: ਕਈ ਏਜੰਟ ਇਕੱਠੇ ਉਸੇ ਕੰਮ 'ਤੇ ਕੰਮ ਕਰਦੇ ਹਨ
- **Fan-In**: ਸਾਰੇ ਏਜੰਟਾਂ ਤੋਂ ਨਤੀਜੇ ਇਕੱਠੇ ਕਰਕੇ ਇੱਕ ਹੈਠਾਂ ਨਤੀਜੇ ਵਿੱਚ ਜੋੜੇ ਜਾਂਦੇ ਹਨ

4. **ਨਤੀਜੇ ਦਿਖਾਓ**: ਸਾਰੇ ਜਾਣਕਾਰੀ ਸਮੇਤ ਜੋੜੀ ਹੋਈ ਯਾਤਰਾ ਯੋਜਨਾ ਦਿਖਾਓ

### 🔧 ਕਸਟਮਾਈਜ਼ੇਸ਼ਨ ਵਿਕਲਪ

**ਹੋਰ concurrent ਏਜੰਟ ਸ਼ਾਮਲ ਕਰੋ:**
```csharp

    .WithOutputFrom(aggregationExecutor)
    .Build();

// Update aggregation count
```

### 🎯 ਅਸਲੀ ਦੁਨੀਆ ਦੇ ਐਪਲੀਕੇਸ਼ਨਸ

ਇਹ concurrent ਪੈਟਰਨ ਲਈ ਵਧੀਆ ਹੈ:
- **ਸਮੱਗਰੀ ਬਣਾਉਣਾ**: ਕਈ ਲੇਖਕ ਵੱਖ-ਵੱਖ ਭਾਗ ਇਕਸਾਥ ਬਣਾਉਂਦੇ ਹਨ
- **ਕੋਡ ਸਮੀਖਿਆ**: ਕਈ ਸਮੀਖਿਆਕਾਰ ਵੱਖ-ਵੱਖ ਦਰਸ਼ਿਕੋਣਾਂ ਤੋਂ ਕੋਡ ਦੇਖਦੇ ਹਨ
- **ਮਾਰਕੀਟ ਖੋਜ**: ਵੱਖਰੇ ਮਾਰਕੀਟ ਖੰਡਾਂ ਦੀ ਸਮਾਂਤਰ ਵਿਸ਼ਲੇਸ਼ਣ
- **ਡੌਕਿਊਮੈਂਟ ਪ੍ਰੋਸੈਸਿੰਗ**: concurrent ਤੌਰ 'ਤੇ ਨਿਕਾਸ਼, ਵਿਸ਼ਲੇਸ਼ਣ ਅਤੇ ਸਹੀ ਟੈਸਟ ਕਰਨਾ
- **ਮਲਟੀ-ਦ੍ਰਿਸ਼ਟੀਕੋਣ ਵਿਸ਼ਲੇਸ਼ਣ**: ਇੱਕੋ ਇਨਪੁੱਟ 'ਤੇ ਵੱਖ-ਵੱਖ ਰਾਏ ਲੈਣਾ

### 🔍 ਕਸਟਮ ਐਕਜ਼ਿਕਿਊਟਰ ਨੂੰ ਸਮਝਣਾ

**ConcurrentStartExecutor:**
- `IMessageHandler<string>` ਲਾਗੂ ਕਰਦਾ ਹੈ ਜਿਸ ਨਾਲ ਸਟਰਿੰਗ ਇਨਪੁੱਟ ਮਿਲਦਾ ਹੈ
- ਜੁੜੇ ਹੋਏ ਸਾਰੇ ਏਜੰਟਾਂ ਨੂੰ ਸੁਨੇਹੇ ਭੇਜਦਾ ਹੈ
- concurrent ਪ੍ਰੋਸੈਸਿੰਗ ਨੂੰ ਸ਼ੁਰੂ ਕਰਨ ਲਈ `TurnToken` ਭੇਜਦਾ ਹੈ

**ConcurrentAggregationExecutor:**
- `IMessageHandler<ChatMessage>` ਲਾਗੂ ਕਰਦਾ ਹੈ ਜਿਹੜਾ ਏਜੰਟ ਜਵਾਬ ਲੈਂਦਾ ਹੈ
- ਧਾਗਾ-ਸੁਰੱਖਿਅਤ ਢੰਗ ਨਾਲ ਸੁਨੇਹੇ ਇਕੱਠੇ ਕਰਦਾ ਹੈ
- ਜਦ ਸਾਰੇ ਉਮੀਦਵਾਰ ਜਵਾਬ ਆ ਜਾਂਦੇ ਹਨ ਤਾਂ ਉਹਨਾਂ ਦਾ ਸੰਕਲਨ ਕਰਦਾ ਹੈ
- ਆਖਰੀ ਨਤੀਜਾ `context.YieldOutputAsync()` ਨਾਲ ਦਿੰਦਾ ਹੈ

### ⚡ ਪ੍ਰਦਰਸ਼ਨ ਲਾਭ

**ਸਾਂਝਾ vs ਲੜੀਵਾਰ:**
- ਲੜੀਵਾਰ: Agent1 (30s) → Agent2 (30s) = **ਕੁੱਲ 60 ਸਕਿੰਟ**
- concurrent: Agent1 (30s) || Agent2 (30s) = **ਕੁੱਲ 30 ਸਕਿੰਟ**

**Throughput ਵਿੱਚ ਸੁਧਾਰ**: N concurrent ਏਜੰਟਾਂ ਲਈ N× ਤੇਜ਼ (ਭਾਰ ਅਤੇ ਸਾਂਝੇ ਸੰਸਾਧਨਾਂ ਦੇ ਅਨੁਸਾਰ)

### 🛡️ ਤਰੁੱਟੀ ਸੰਭਾਲ

ਵਰਕਫਲੋ ਅਲੱਗੇ ਅਲੱਗੇ ਏਜੰਟ ਫੇਲਿਅਰਾਂ ਨੂੰ ਸੁਖਦਾ ਹੈ:
- ਜੇ ਇੱਕ ਏਜੰਟ ਫੇਲ੍ਹ ਹੋ ਜਾਵੇ, ਤਾਂ ਬਾਕੀ concurrent ਪ੍ਰੋਸੈਸਿੰਗ ਜਾਰੀ ਰਹਿੰਦੀ ਹੈ
- ਐਗਰੀਗੇਟਰ ਵਿੱਚ ਟਾਈਮਆਉਟ ਲਾਜਿਕ ਲਾਗੂ ਹੋ ਸਕਦਾ ਹੈ
- ਜ਼ਰੂਰਤ ਪੈਣ 'ਤੇ ਹਿਸਸਾ ਨਤੀਜੇ ਵਾਪਸ ਕੀਤੇ ਜਾ ਸਕਦੇ ਹਨ

### 📊 ਉੱਨਤ ਫੀਚਰ

**ਗਤਿਸ਼ੀਲ ਏਜੰਟ ਗਿਣਤੀ:**
ਸੰਮਿਲਿਤ ਤਰਕ ਨੂੰ ਸੋਧੋ ਤਾਂ ਜੋ ਵੱਖ-ਵੱਖ ਏਜੰਟ ਗਿਣਤੀਆਂ ਸਮਰਥਿਤ ਹੋਣ:

```csharp
    }
}
```
