# ਕம்பਿੂਟਰ ਯੂਜ ਏਜੰਟ ਬਣਾਉਣਾ (CUA)

ਕம்பਿਊਟਰ ਯੂਜ਼ ਏਜੰਟ ਉਹੀ ਤਰ੍ਹਾਂ ਵੈਬਸਾਈਟਾਂ ਨਾਲ ਇੰਟਰੈਕਟ ਕਰ ਸਕਦੇ ਹਨ ਜਿਵੇਂ ਕੋਈ ਇਨਸਾਨ ਕਰਦਾ ਹੈ: ਬ੍ਰਾਊਜ਼ਰ ਖੋਲ੍ਹ ਕੇ, ਪੰਨੇ ਦੀ ਜਾਂਚ ਕਰਕੇ, ਅਤੇ ਉਹਨਾਂ ਚੀਜ਼ਾਂ ਵਿੱਚੋਂ ਸਭ ਤੋਂ ਵਧੀਆ ਅਗਲਾ ਕਦਮ ਚੁਣ ਕੇ ਜੋ ਉਹ ਦੇਖਦੇ ਹਨ। ਇਸ ਪਾਠ ਵਿੱਚ, ਤੁਸੀਂ ਇੱਕ ਬ੍ਰਾਊਜ਼ਰ ਆਟੋਮੇਸ਼ਨ ਏਜੰਟ ਬਣਾਓਗੇ ਜੋ Airbnb ਨੂੰ ਖੋਜੇਗਾ, ਸੰਰਚਿਤ ਲਿਸਟਿੰਗ ਡਾਟਾ ਨਿਕਾਲੇਗਾ, ਅਤੇ ਸਟਾਕਹੋਮ ਵਿੱਚ ਸਭ ਤੋਂ ਸਸਤੀ ਰਹਿਣ ਦੀ ਥਾਂ ਦਾ ਪਤਾ ਲਗਾਏਗਾ।

ਇਹ ਪਾਠ Browser-Use ਨੂੰ AI-ਚਲਿਤ ਨੈਵੀਗੇਸ਼ਨ ਲਈ, Playwright ਅਤੇ Chrome DevTools Protocol (CDP) ਨੂੰ ਬ੍ਰਾਊਜ਼ਰ ਕੰਟਰੋਲ ਲਈ, Azure OpenAI ਨੂੰ ਵਿਜ਼ਨ-ਸਮਰੱਥ reasoning ਲਈ, ਅਤੇ Pydantic ਨੂੰ ਸੰਰਚਿਤ extraction ਲਈ ਜੋੜਦਾ ਹੈ।

## ਜਾਣਪਹਿਚਾਣ

ਇਸ ਪਾਠ ਵਿੱਚ ਇਹ ਗੱਲਾਂ ਕਵਰ ਕੀਤੀ ਜਾਣਗੀਆਂ:

- ਸਮਝਣਾ ਕਿ ਕਦੋਂ ਕம்பਿਊਟਰ ਯੂਜ਼ ਏਜੰਟ API-ਮਾਤਰ ਆਟੋਮੇਸ਼ਨ ਨਾਲੋਂ ਵਧੀਆ ਹੁੰਦੇ ਹਨ
- Browser-Use ਨੂੰ Playwright ਅਤੇ CDP ਨਾਲ ਜੋੜ ਕੇ ਭਰੋਸੇਮੰਦ ਬ੍ਰਾਊਜ਼ਰ ਲਾਈਫਸਾਈਕਲ ਪ੍ਰਬੰਧਨ
- Azure OpenAI ਵਿਜ਼ਨ ਅਤੇ ਸੰਰਚਿਤ Pydantic ਆਉਟਪੁੱਟ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਡਾਇਨੈਮਿਕ ਵੈੱਬ ਪੇਜਾਂ ਤੋਂ ਲਿਸਟਿੰਗ ਡਾਟਾ ਕੱਢਣਾ
- ਫੈਸਲਾ ਕਰਨਾ ਕਿ ਕਦੋਂ ਏਜੰਟ-ਪਹਿਲਾਂ, ਅਦਾਕਾਰ-ਪਹਿਲਾਂ ਜਾਂ ਹਾਈਬ੍ਰਿਡ ਬ੍ਰਾਊਜ਼ਰ ਆਟੋਮੇਸ਼ਨ ਵਰਕਫ਼ਲੋ ਦੀ ਵਰਤੋਂ ਕਰਨੀ ਹੈ

## ਸਿੱਖਣ ਦੇ ਟੀਚੇ

ਇਸ ਪਾਠ ਨੂੰ ਪੂਰਾ ਕਰਨ ਤੋਂ ਬਾਦ, ਤੁਸੀਂ ਜਾਣੋਗੇ ਕਿ ਕਿਵੇਂ:

- Browser-Use ਨੂੰ Azure OpenAI ਅਤੇ Playwright ਨਾਲ ਕਨਫਿਗਰ ਕਰਨਾ
- ਇੱਕ ਬ੍ਰਾਊਜ਼ਰ ਆਟੋਮੇਸ਼ਨ ਵਰਕਫਲੋ ਬਣਾਉਣਾ ਜੋ ਕਿਸੇ ਅਸਲ ਵੈੱਬਸਾਈਟ ਦੀ ਨੈਵੀਗੇਸ਼ਨ ਕਰਦਾ ਹੈ ਅਤੇ ਡਾਇਨੈਮਿਕ UI ਤੱਤਾਂ ਨੂੰ ਸੰਭਾਲਦਾ ਹੈ
- ਵੇਖਣਯੋਗ ਪੰਨਾ ਸਮੱਗਰੀ ਤੋਂ ਟਾਈਪ ਕੀਤੇ ਨਤੀਜੇ ਨਿਕਾਲਣਾ ਅਤੇ ਉਨ੍ਹਾਂ ਨੂੰ ਬਿਜ਼ਨਸ ਲੋਜਿਕ ਵਿੱਚ ਬਦਲਣਾ
- ਬ੍ਰਾਊਜ਼ਰ ਦੇ ਕੰਮ ਦੀ ਭਵਿੱਖਬਾਣੀ ਦੇ ਅਧਾਰ ਤੇ ਏਜੰਟ ਅਤੇ ਅਦਾਕਾਰ ਪੈਟਰਨਜ਼ ਵਿੱਚ ਚੋਣ ਕਰਨਾ

## ਕੋਡ ਨਮੂਨਾ

ਇਸ ਪਾਠ ਵਿੱਚ ਇਕ ਨੋਟਬੁੱਕ ਟਿਊਟੋਰਿਯਲ ਸ਼ਾਮਲ ਹੈ:

- [15-browser-user.ipynb](./15-browser-user.ipynb): CDP ਉੱਤੇ ਇੱਕ Chrome ਸੈਸ਼ਨ ਸ਼ੁਰੂ ਕਰਦਾ ਹੈ, Airbnb ਤੇ ਸਟਾਕਹੋਮ ਲਿਸਟਿੰਗਾਂ ਨੂੰ ਖੋਜਦਾ ਹੈ, Browser-Use ਵਿਜ਼ਨ ਨਾਲ ਕੀਮਤਾਂ ਕੱਢਦਾ ਹੈ, ਅਤੇ ਸਭ ਤੋਂ ਸਸਤੇ ਵਿਕਲਪ ਨੂੰ ਸੰਰਚਿਤ ਡਾਟਾ ਵਜੋਂ ਵਾਪਸ ਕਰਦਾ ਹੈ।

## ਲੋੜੀਂਦੀਆਂ ਚੀਜ਼ਾਂ

- Python 3.12+
- Azure OpenAI ਡਿਪਲੌਇਮੈਂਟ ਤੁਹਾਡੇ ماحول ਵਿੱਚ ਕਨਫਿਗਰ ਹੈ
- ਤੁਹਾਡੇ ਸਥਾਨਕ ਕੰਪਿਊਟਰ 'ਤੇ Chrome ਜਾਂ Chromium ਇੰਸਟਾਲ ਹੈ
- Playwright ਦੀਆਂ ਡਿਪੈਂਡੈਂਸੀਜ਼ ਇੰਸਟਾਲ ਕੀਤੀਆਂ ਹੋਣ
- async Python ਦਾ ਮੂਲ ਜਾਣਕਾਰ

## ਸੈਟਅੱਪ

ਨੋਟਬੁੱਕ ਵਿੱਚ ਵਰਤੀ ਜਾਣ ਵਾਲੀਆਂ ਪੈਕੇਜਾਂ ਨੂੰ ਇੰਸਟਾਲ ਕਰੋ:

```bash
pip install browser_use playwright python-dotenv
playwright install chromium
```

ਨੋਟਬੁੱਕ ਦੁਆਰਾ ਵਰਤੀ ਜਾਣ ਵਾਲੀਆਂ Azure OpenAI ਇਨਵਾਇਰੰਮੈਂਟ ਵੈਰੀਏਬਲ ਸੈੱਟ ਕਰੋ:

```bash
AZURE_OPENAI_ENDPOINT=...
AZURE_OPENAI_API_KEY=...
AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=...
# ਵਿਕਲਪੀ: ਛੱਡਣ 'ਤੇ ਅਖੀਰਲਾ API ਵਰਜਨ ਡਿਫ਼ੋਲਟ ਹੁੰਦਾ ਹੈ
AZURE_OPENAI_API_VERSION=...
```

## ਸਰੰਚਨਾ ਦਾ ਜਾਇਜ਼ਾ

ਨੋਟਬੁੱਕ ਇੱਕ ਹਾਈਬ੍ਰਿਡ ਬ੍ਰਾਊਜ਼ਰ ਆਟੋਮੇਸ਼ਨ ਵਰਕਫਲੋ ਨੂੰ ਵਿਖਾਉਂਦਾ ਹੈ:

1. Chrome CDP ਸਹੂਲਤ ਨਾਲ ਸ਼ੁਰੂ ਹੁੰਦਾ ਹੈ ਤਾਂ ਜੋ Playwright ਅਤੇ Browser-Use ਦੋਹਾਂ ਇੱਕੋ ਬ੍ਰਾਊਜ਼ਰ ਸੈਸ਼ਨ ਨੂੰ ਸਾਂਝਾ ਕਰ ਸਕਣ।
2. ਇੱਕ Browser-Use ਏਜੰਟ ਖੁੱਲ੍ਹੇ ਸਮਾਪਤੀ ਨੈਵੀਗੇਸ਼ਨ ਕੰਮ ਸੰਭਾਲਦਾ ਹੈ ਜਿਵੇਂ ਕਿ Airbnb ਖੋਲ੍ਹਣਾ, ਪੋਪ-ਅੱਪਜ਼ ਨੂੰ ਬੰਦ ਕਰਨਾ, ਅਤੇ ਸਟਾਕਹੋਮ ਲਈ ਖੋਜ ਕਰਨਾ।
3. ਸਰਗਰਮ ਪੰਨਾ ਇੱਕ ਸੰਰਚਿਤ Pydantic ਸਕੀਮਾ ਨਾਲ ਜਾਂਚਿਆ ਜਾਂਦਾ ਹੈ ਤਾਂ ਜੋ ਲਿਸਟਿੰਗ ਸਿਰਲੇਖ, ਰਾਤਾਂ ਦੀ ਕੀਮਤ, ਰੇਟਿੰਗਾਂ ਅਤੇ URLs ਕੱਢੇ ਜਾ ਸਕਣ।
4. Python ਲੋਜਿਕ ਕੱਢੀ ਗਈਆਂ ਲਿਸਟਿੰਗਾਂ ਦੀ ਤੁਲਨਾ ਕਰਦਾ ਹੈ ਅਤੇ ਸਭ ਤੋਂ ਸਸਤੇ ਨਤੀਜੇ ਨੂੰ ਉਜਾਗਰ ਕਰਦਾ ਹੈ।

ਇਹ ਤਰੀਕਾ Browser-Use ਦੀ ਲਚਕੀਲੀ, ਵਿਜ਼ਨ-ਆਧਾਰਿਤ reasoning ਨੂੰ ਬਣਾਈ ਰੱਖਦਾ ਹੈ ਜਦੋਂ ਕਿ ਜਰੂਰਤ ਪੈਣ 'ਤੇ ਤੁਹਾਨੂੰ ਨਿਰਧਾਰਿਤ ਬ੍ਰਾਊਜ਼ਰ ਕੰਟਰੋਲ ਵੀ ਮਿਲਦਾ ਹੈ।

## ਮੁੱਖ ਸਿੱਖਾਂ ਅਤੇ ਵਧੀਆ ਅਭਿਆਸ

### ਏਜੰਟ ਅਤੇ ਅਦਾਕਾਰ ਵਿੱਚ ਕਦੋਂ ਵਰਤਣਾ ਹੈ

| ਦਰਸ਼ਨ | ਏਜੰਟ ਦੀ ਵਰਤੋਂ ਕਰੋ | ਅਦਾਕਾਰ ਦੀ ਵਰਤੋਂ ਕਰੋ |
|----------|----------------|-----------|
| ਡਾਇਨੈਮਿਕ ਲੇਆਉਟ | ਹਾਂ, AI ਪੰਨੇ ਦੇ ਬਦਲਾਵਾਂ ਦੇ ਅਨੁਕੂਲ ਹੋ ਸਕਦਾ ਹੈ | ਨਹੀਂ, ਨਾਜੁਕ ਚੁਣੌਤੀਆਂ ਟੁੱਟ ਸਕਦੀਆਂ ਹਨ |
| ਜਾਣੀ-ਪਹਿਚਾਣ ਵਾਲੀ ਸਰਚਨਾ | ਨਹੀਂ, ਏਜੰਟ ਸਿੱਧੇ ਕੰਟਰੋਲ ਨਾਲੋਂ ਧੀਮਾ ਹੁੰਦਾ ਹੈ | ਹਾਂ, ਤੇਜ਼ ਅਤੇ ਸਹੀ |
| ਤੱਤ ਲੱਭਣਾ | ਹਾਂ, ਕੁਦਰਤੀ ਭਾਸ਼ਾ ਚੰਗੀ ਤਰ੍ਹਾਂ ਕੰਮ ਕਰਦੀ ਹੈ | ਨਹੀਂ, ਸਹੀ ਚੁਣੀ ਹੋਈ ਸਿਲੈਕਟਰਾਂ ਦੀ ਲੋੜ ਹੈ |
| ਟਾਈਮਿੰਗ ਕੰਟਰੋਲ | ਨਹੀਂ, ਘੱਟ ਭਵਿੱਖਬਾਣੀਯੋਗ | ਹਾਂ, ਇੰਤਜ਼ਾਰ ਅਤੇ ਫੇਰ ਕੋਸ਼ਿਸ਼ 'ਤੇ ਪੂਰਾ ਕੰਟਰੋਲ |
| ਜਟਿਲ ਵਰਕਫਲੋਜ਼ | ਹਾਂ, ਅਣਉਮੀਦ ਅਤੇ UI ਦੀ ਹਾਲਤਾਂ ਨੂੰ ਸੰਭਾਲਦਾ ਹੈ | ਨਹੀਂ, ਖੁੱਲ੍ਹੇ ਤੌਰ ਤੇ ਸ਼ਾਖਾ ਬੰਦੀ ਦੀ ਲੋੜ ਹੋਂਦੀ ਹੈ |

### Browser-Use ਦੇ ਵਧੀਆ ਅਭਿਆਸ

1. ਖੋਜ ਅਤੇ ਡਾਇਨੈਮਿਕ ਨੈਵੀਗੇਸ਼ਨ ਲਈ ਪਹਿਲਾਂ ਏਜੰਟ ਨਾਲ ਸ਼ੁਰੂ ਕਰੋ।
2. ਜਦੋਂ ਇੰਟਰਐਕਸ਼ਨ ਭਵਿੱਖਬਾਣੀਯੋਗ ਹੋ ਜਾਏ ਤਾਂ ਸਿੱਧੇ ਪੰਨੇ ਕੰਟਰੋਲ ਤੇ ਸਵਿੱਚ ਕਰੋ।
3. ਸੰਰਚਿਤ ਆਉਟਪੁੱਟ ਮਾਡਲਾਂ ਦੀ ਵਰਤੋਂ ਕਰੋ ਤਾਂ ਜੋ ਕੱਢਿਆ ਡਾਟਾ ਪ੍ਰਮਾਣਿਤ ਅਤੇ ਕਿਸਮ-ਸੁਰੱਖਿਅਤ ਹੋਵੇ।
4. ਐਕਸ਼ਨਾਂ ਤੋਂ ਬਾਅਦ ਜੋ ਦਿੱਸਣ ਯੋਗ UI ਤਬਦੀਲੀਆਂ ਨੂੰ ਪ੍ਰੇਰਿਤ ਕਰਦੇ ਹਨ, ਸਮਝਦਾਰੀ ਨਾਲ ਡਿਖੇਰਤਾਂ ਸ਼ਾਮਲ ਕਰੋ।
5. ਤਸਵੀਰਾਂ ਕੈਪਚਰ ਕਰੋ ਜਦੋਂ ਤੱਕੜਾਅ ਹੋ ਰਹੇ ਹਨ ਤਾਂ ਜੋ ਅਸਫਲਤਾਵਾਂ ਦੀ ਡੀਬੱਗਿੰਗ ਆਸਾਨ ਹੋਵੇ।
6. ਉਮੀਦ ਕਰੋ ਕਿ ਵੈੱਬਸਾਈਟਾਂ ਬਦਲਣਗੀਆਂ ਅਤੇ ਪੋਪ-ਅੱਪ ਅਤੇ ਲੇਆਉਟ ਬਦਲਾਵਾਂ ਲਈ ਫਾਲਬੈਕ ਯੋਜਨਾਵਾਂ ਬਣਾਓ।
7. ਲਚਕੀਲਾਪਣ ਅਤੇ ਸਹੀਤਾ ਦੋਹਾਂ ਲਈ ਏਜੰਟ ਅਤੇ ਅਦਾਕਾਰ ਪੈਟਰਨਜ਼ ਨੂੰ ਮਿਲਾਓ।

### ਬ੍ਰਾਊਜ਼ਰ ਏਜੰਟਾਂ ਲਈ ਸੁਰੱਖਿਆ ਗਾਰਡਰੇਲਜ਼

ਬ੍ਰਾਊਜ਼ਰ ਏਜੰਟ ਲਾਈਵ ਵੈੱਬਸਾਈਟਾਂ 'ਤੇ ਕੰਮ ਕਰਦੇ ਹਨ, ਇਸ ਲਈ ਉਹਨਾਂ ਨੂੰ ਉਸ ਸਕ੍ਰਿਪਟ ਨਾਲੋਂ ਵਧੇਰੇ ਕੜੀਆਂ ਸੀਮਾਵਾਂ ਦੀ ਲੋੜ ਹੋਂਦੀ ਹੈ ਜੋ ਸਿਰਫ਼ ਜਾਣੇ-ਮाने API ਨੂੰ ਕਾਲ ਕਰਦਾ ਹੈ। ਨੋਟਬੁੱਕ ਡੈਮੋ ਤੋਂ ਅਸਲ ਵਰਕਫਲੋ ਵਿੱਚ ਜਾਣ ਤੋਂ ਪਹਿਲਾਂ, ਇਹ ਨਿਰਧਾਰਤ ਕਰੋ ਕਿ ਏਜੰਟ ਕੀ ਦੇਖ ਸਕਦਾ ਹੈ, ਕਿੱਥੇ ਕਲਿੱਕ ਕਰ ਸਕਦਾ ਹੈ, ਅਤੇ ਕੀ ਸਬਮਿਟ ਕਰ ਸਕਦਾ ਹੈ।

1. **ਬ੍ਰਾਊਜ਼ਿੰਗ ਵਾਤਾਵਰਨ ਦੀ ਸੀਮਾ ਨਿਰਧਾਰਤ ਕਰੋ।** ਏਜੰਟ ਨੂੰ ਸਮਰਪਿਤ ਬ੍ਰਾਊਜ਼ਰ ਪ੍ਰੋਫਾਈਲ ਜਾਂ ਸੈਂਡਬਾਕਸ ਵਿੱਚ ਚਲਾਓ, ਅਤੇ ਕੰਮ ਲਈ ਲੋੜੀਂਦੇ ਡੋਮੇਨਾਂ ਤੱਕ ਸੀਮਿਤ ਰੱਖੋ।
2. **ਅਵਲੋਕਨ ਨੂੰ ਕਾਰਵਾਈ ਤੋਂ ਵੱਖਰਾ ਕਰੋ।** ਏਜੰਟ ਨੂੰ ਪਹਿਲਾਂ ਖੋਜ, ਪੜ੍ਹਾਈ ਅਤੇ ਡਾਟਾ ਕੱਢਣ ਦਿਓ; ਫਾਰਮਾਂ ਭੇਜਣ, ਸੰਦੇਸ਼ ਭੇਜਣ, ਯਾਤਰਾ ਬੁਕ ਕਰਨ, ਖਰੀਦਦਾਰੀ ਕਰਨ, ਰਿਕਾਰਡ ਮਿਟਾਉਣ ਜਾਂ ਖਾਤਾ ਸੈਟਿੰਗਜ਼ ਬਦਲਣ ਤੋਂ ਪਹਿਲਾਂ ਖੁੱਲ੍ਹੇ ਸਵਿਕਾਰ ਦੀ ਲੋੜ ਰੱਖੋ।
3. **ਪ੍ਰੌੰਪਟਸ ਅਤੇ ਟ੍ਰੇਸਜ਼ ਤੋਂ ਸਿਕ੍ਰੇਟਸ ਬਾਹਰ ਰੱਖੋ।** ਮਾਡਲ ਸੰਦਰਭ ਵਿੱਚ ਪਾਸਵਰਡ, ਭੁਗਤਾਨ ਵੇਰਵੇ, ਸੈਸ਼ਨ ਕੁਕੀਜ਼ ਜਾਂ ਕੱਚਾ ਨਿੱਜੀ ਡਾਟਾ ਨਾ ਰੱਖੋ। ਪ੍ਰਮਾਣੀਕਰਨ ਲਈ ਉਪਭੋਗਤਾ ਨੂੰ ਅਦੇਹੀ ਕਰਨ ਦਿਓ ਅਤੇ ਲੋਗਜ਼ ਵਿੱਚ ਸੰਵੇਦਨਸ਼ੀਲ ਖੇਤਰਾਂ ਨੂੰ ਛੱਡੋ।
4. **ਪੰਨਾ ਸਮੱਗਰੀ ਨੂੰ ਬਿਨਾਂ ਭਰੋਸੇ ਦੇ ਇਨਪੁੱਟ ਵੱਜੋਂ ਲਓ।** ਇੱਕ ਵੈੱਬਸਾਈਟ ਵਿੱਚ ਏਜੰਟ ਲਈ ਨਿਰਦੇਸ਼ ਹੋ ਸਕਦੇ ਹਨ ਜੋ ਉਪਭੋਗਤਾ ਲਈ ਨਹੀਂ ਹਨ। ਏਜੰਟ ਨੂੰ ਉਹ ਪੰਨਾ ਲਿਖਤ ਅਣਦੇਖਾ ਕਰਨੀ ਚਾਹੀਦੀ ਹੈ ਜੋ ਉਸਨੂੰ ਆਪਣਾ ਲਕੜੀ ਬਦਲਣ, ਡਾਟਾ ਖੋਲ੍ਹਣ, ਸੁਰੱਖਿਆ ਗਾਰਡਰੇਲ ਹਟਾਉਣ ਜਾਂ ਬੇਈਨ ਵੈੱਬਸਾਈਟਾਂ ਨੂੰ ਜਾਉਣ ਲਈ ਕਹਿੰਦਾ ਹੈ।
5. **ਖਤਰਨਾਕ ਕਦਮਾਂ ਦੇ ਆਲੇ-ਦੁਆਲੇ ਨਿਰਧਾਰਿਤ ਜਾਂਚਾਂ ਦੀ ਵਰਤੋਂ ਕਰੋ।** ਵੇਰਵਾ ਕੋਡ ਨਾਲ ਮੌਜੂਦਾ URL, ਪੰਨਾ ਸਿਰਲੇਖ, ਚੁਣੀ ਹੋਈ ਚੀਜ਼, ਕੀਮਤ, ਪ੍ਰਾਪਤਕਰਤਾ ਅਤੇ ਕਾਰਵਾਈ ਸਾਰ ਨੂੰ ਜਾਂਚੋ ਪਹਿਲਾਂ ਕਿ ਉਪਭੋਗਤਾ ਅੰਤਿਮ ਕਦਮ ਲਈ ਮਨਜ਼ੂਰੀ ਦੇਵੇ।
6. **ਬਜਟ ਅਤੇ ਰੋਕਣ ਦੀਆਂ ਸ਼ਰਤਾਂ ਸੈੱਟ ਕਰੋ।** ਕਾਰਵਾਈਆਂ, ਦੁਹਰਾਅਵਾਂ, ਟੈਬਾਂ ਅਤੇ ਮਿੰਟਾਂ ਦੀ ਗਿਣਤੀ ਸੀਮਿਤ ਕਰੋ ਜੋ ਏਜੰਟ ਵਰਤ ਸਕਦਾ ਹੈ। ਜਦੋਂ ਪੰਨਾ ਦੀ ਹਾਲਤ ਗੁੰਝਲਦਾਰ ਹੁੰਦੀ ਹੈ ਤਾਂ ਕਲਿੱਕ ਕਰਨਾ ਬੰਦ ਕਰੋ।
7. **ਸਾਰਥਕ ਸਬੂਤ ਦਰਜ ਕਰੋ, ਸਾਰਾ ਕੁਝ ਨਹੀਂ।** ਕਾਰਵਾਈ ਦੇ ਸਾਰੇ ਸਾਰਾਂਸ਼, ਸਮੇਂ ਦੇ ਟੈਂਪ, URLs, ਚੁਣੇ ਹੋਏ ਤੱਤਾਂ ਦੇ ਵਰਣਨ, ਅਤੇ ਸਕ੍ਰੀਨਸ਼ਾਟ ਦਿਸ਼ਾ-ਨਿਰਦੇਸ਼ ਰੱਖੋ ਤਾਂ ਜੋ ਅਸਫਲਤਾਵਾਂ ਦੀ ਸਮੀਖਿਆ ਬਿਨਾਂ ਬੇਲੋੜਾ ਸੰਵੇਦਨਸ਼ੀਲ ਪੰਨਾ ਸਮੱਗਰੀ ਸਟੋਰ ਕੀਤੇ ਕੀਤੀ ਜਾ ਸਕੇ।

Airbnb ਦੇ ਨਮੂਨੇ ਵਿੱਚ, ਸੁਰੱਖਿਅਤ ਡਿਫੌਲਟ ਲਿਸਟਿੰਗਾਂ ਖੋਜਣ ਅਤੇ ਕੀਮਤਾਂ ਕੱਢਣ ਦਾ ਹੈ। ਸਾਈਨ ਇਨ, ਮਾਲਿਕ ਨਾਲ ਸੰਪਰਕ ਕਰਨ ਜਾਂ ਬੁਕਿੰਗ ਪੂਰੀ ਕਰਨ ਲਈ ਵੱਖਰਾ ਉਪਭੋਗਤਾ-ਸਵੀਕਾਰਿਤ ਕਦਮ ਹੋਣਾ ਚਾਹੀਦਾ ਹੈ।

### ਅਸਲ ਸੰਸਾਰ ਦੀਆਂ ਐਪਲੀਕੇਸ਼ਨ

- ਯਾਤਰਾ ਦੀ ਬੁਕਿੰਗ ਅਤੇ ਕੀਮਤ ਨਿਗਰਾਨੀ
- ਈ-ਕਾਮਰਸ ਕੀਮਤ ਦੀ ਤੁਲਨਾ ਅਤੇ ਉਪਲਬਧਤਾ ਦੀ ਜਾਂਚ
- ਡਾਇਨੈਮਿਕ ਵੈੱਬਸਾਈਟਾਂ ਤੋਂ ਸੰਰਚਿਤ ਨਿਕਾਸ
- ਵਿਜ਼ਨ-ਸੂਚਿਤ UI ਟੈਸਟਿੰਗ ਅਤੇ ਪੁਸ਼ਟੀ
- ਵੈੱਬਸਾਈਟ ਨਿਗਰਾਨੀ ਅਤੇ ਅਲਰਟਿੰਗ
- ਬਹੁ-ਕਦਮੀ ਪ੍ਰਵਾਹਾਂ ਵਿੱਚ ਬੁੱਧਿਮਾਨ ਫਾਰਮ ਭਰਾਈ

## ਅਸਲ ਸੰਸਾਰ ਦਾ ਉਦਾਹਰਣ: Microsoft ਪ੍ਰੋਜੈਕਟ ਓਪਲ

ਇਸ ਪਾਠ ਵਿੱਚ ਤੁਸੀਂ ਜੋ ਏਜੰਟ ਬਣਾਉਂਦੇ ਹੋ ਉਹ ਇੱਕ ਛੋਟਾ, ਸਥਾਨਕ ਵਰਜਨ ਹੈ ਇੱਕ **ਕਪਿਊਟਰ ਯੂਜ਼ ਏਜੰਟ (CUA)** ਦਾ — ਇੱਕ ਪ੍ਰੋਗਰਾਮ ਜੋ ਬ੍ਰਾਊਜ਼ਰ ਨੂੰ ਉਸ ਤਰ੍ਹਾਂ ਚਲਾਉਂਦਾ ਹੈ ਜਿਵੇਂ ਕੋਈ ਇਨਸਾਨ ਕਰਦਾ। Microsoft ਇਸੇ ਵਿਚਾਰ ਨੂੰ ਉਦਯੋਗ ਵਿੱਚ ਲਿਆ ਰਿਹਾ ਹੈ **[Project Opal (Frontier)](https://support.microsoft.com/en-us/microsoft-365-copilot/get-started-with-project-opal-frontier)** ਰਾਹੀਂ, ਜੋ Microsoft 365 Copilot ਵਿੱਚ ਇੱਕ ਸਮਰੱਥਾ ਹੈ।

ਪ੍ਰੋਜੈਕਟ ਓਪਲ ਨਾਲ, ਤੁਸੀਂ ਇੱਕ ਕੰਮ ਦਾ ਵਰਣਨ ਕਰਦੇ ਹੋ ਅਤੇ ਏਜੰਟ ਤੁਹਾਡੇ ਵਾਸਤੇ ਕੰਮ ਕਰਦਾ ਹੈ ਜੋ **ਕੰਪਿਊਟਰ ਯੂਜ਼ ਸੁਰੱਖਿਅਤ Windows 365 Cloud PC 'ਤੇ** ਚਲਦਾ ਹੈ, ਅਤੇ ਤੁਹਾਡੇ ਸੰਗਠਨ ਦੀਆਂ ਬ੍ਰਾਊਜ਼ਰ-ਅਧਾਰਿਤ ਐਪਲੀਕੇਸ਼ਨਾਂ, ਸਾਈਟਾਂ, ਅਤੇ ਡਾਟਾ 'ਤੇ ਕੰਮ ਕਰਦਾ ਹੈ। ਇਹ **ਪਿੱਛੇਲੇ ਪਾਸੇ ਅਸਿੰਕ੍ਰੋਨਸ ਤੌਰ 'ਤੇ ਕੰਮ ਕਰਦਾ ਹੈ**, ਅਤੇ ਤੁਸੀਂ ਕਿਸੇ ਵੀ ਸਮੇਂ ਕੰਮ ਨੂੰ ਮਾਰਗਦਰਸ਼ਨ ਦੇ ਸਕਦੇ ਹੋ ਜਾਂ ਕੰਟਰੋਲ ਲੈ ਸਕਦੇ ਹੋ। ਉਦਾਹਰਣ ਬੰਦੋਬਸਤਾਂ ਵਿੱਚ ਸ਼ਾਮਲ ਹਨ:

- ਸੁਰੱਖਿਆ ਗਰੁੱਪ ਮੈਂਬਰਸ਼ਿਪ ਬੇਨਤੀ ਪ੍ਰਬੰਧਨ
- ਅਡਿਟ ਸਮੀਖਿਆਵਾਂ ਲਈ ਸਬੂਤ ਇਕੱਠਾ ਅਤੇ ਪ੍ਰਮਾਣਿਤ ਕਰਨਾ
- IT ਘਟਨਾਵਾਂ ਦਾ ਪ੍ਰਬੰਧ (ਟਿਕਟ ਸਥਿਤੀ ਅੱਪਡੇਟ ਕਰਨਾ, ਮਾਲਿਕ ਐਸਾਈਨ ਕਰਨਾ, ਡੁਪਲਿਕੇਟ ਬੰਦ ਕਰਨਾ)
- Excel ਡਾਟਾ ਨੂੰ ਇੱਕ ਵਿੱਤੀ ਬੰਦ ਡੈੱਕ ਵਿੱਚ ਸੰਕਲਿਤ ਕਰਨਾ

ਓਪਲ ਇੱਕ ਲਾਭਦਾਇਕ ਸੰਦਰਭ ਹੈ ਕਿ ਐਡਵਾਂਸ, ਭਰੋਸੇਮੰਦ ਕੰਪਿਊਟਰ ਯੂਜ਼ ਏਜੰਟ ਕਿਵੇਂ ਹੁੰਦਾ ਹੈ — ਅਤੇ ਇਹ ਪਹਿਲਾਂ ਦੇ ਪਾਠਾਂ ਵਿਚਲੇ ਵਿਚਾਰਾਂ ਨੂੰ ਮਜ਼ਬੂਤ ਕਰਦਾ ਹੈ:

| ਇਸ ਕੋਰਸ ਵਿੱਚ ਸੋਚ | ਪ੍ਰੋਜੈਕਟ ਓਪਲ ਇਸਨੂੰ ਕਿਵੇਂ ਲਾਗੂ ਕਰਦਾ ਹੈ |
|------------------------|-----------------------------|
| **ਹਿਊਮਨ-ਇਨ-ਦ-ਲੂਪ** (ਪਾਠ 06) | ਓਪਲ ਲਾਗਿਨ ਸਰਟੀਫਿਕੇਟ, ਸੰਵੇਦਨਸ਼ੀਲ ਡਾਟਾ, ਜਾਂ ਗੁੰਝਲਦਾਰ ਨਿਰਦੇਸ਼ਾਂ ਲਈ ਰੁਕਦਾ ਹੈ, ਅਤੇ ਕਦੇ ਵੀ ਬਿਨਾਂ ਸਪਸ਼ਟ ਪੁਸ਼ਟੀ ਦੇ ਪਾਸਵਰਡ ਜਾਂ ਫਾਰਮ ਸਬਮਿਟ ਨਹੀਂ ਕਰਦਾ। ਤੁਸੀਂ *Take Control* ਅਤੇ *Return Control* ਵਿਚਕਾਰ ਕੰਮ ਨੂੰ ਚਾਲੂ ਰੱਖ ਸਕਦੇ ਹੋ। |
| **ਭਰੋਸੇਮੰਦ ਅਤੇ ਸੁਰੱਖਿਅਤ ਏਜੰਟ** (ਪਾਠ 06 ਅਤੇ 18) | ਵੱਖ-ਵੱਖ Windows 365 Cloud PC 'ਤੇ ਚਲਦਾ ਹੈ, ਡਿਫੌਲਟ ਤੌਰ 'ਤੇ ਸਿਰਫ ਬ੍ਰਾਊਜ਼ਰ ਹੈ (ਹੋਰ ਕੰਪਿਊਟਰ ਐਕਸੈਸ ਰੋਕਿਆ ਗਿਆ ਹੈ, Intune ਰਾਹੀਂ ਲਾਗੂ), *ਤੁਹਾਡੀ* ਪਹਿਚਾਣ ਵਰਤਦਾ ਹੈ ਤਾਂ ਜੋ ਸਿਰਫ ਅਧਿਕਾਰਤ ਬਰਾਬਰ ਹੀ ਪਹੁੰਚ ਹੋਵੇ, ਅਤੇ ਹਰ ਕਾਰਵਾਈ ਨੂੰ ਆਡੀਟ ਕਰਨ ਲਈ ਲਾਗ ਕਰਦਾ ਹੈ। |
| **ਯੋਜਨਾ ਅਤੇ ਮੈਟਾਕੋਗਨਿਸ਼ਨ** (ਪਾਠ 07 ਅਤੇ 09) | ਓਪਲ ਪਹਿਲਾਂ ਕੰਮ ਲਈ ਯੋਜਨਾ ਬਣਾਉਂਦਾ ਹੈ, ਫਿਰ ਹਰ ਕਦਮ 'ਤੇ ਆਪਣੇ reasoning ਦੀ ਨਿਗਰਾਨੀ ਕਰਦਾ ਹੈ ਅਤੇ ਸ਼ੱਕੀ ਕਾਰਵਾਈ ਦੇਖ ਕੇ ਰੁਕਦਾ ਹੈ। |
| **ਪੁਨਰਵਰਤਨਯੋਗ ਸਮਰੱਥਾਵਾਂ / ਸੰਦ** (ਪਾਠ 04) | **Skills** ਤੁਹਾਨੂੰ ਦੁਹਰਾਵਾਂ ਵਾਲੇ ਕੰਮਾਂ ਲਈ ਨਿਰਦੇਸ਼ ਲਿਖਣ ਦਿੰਦੇ ਹਨ (ਜੋ `.md` ਫਾਇਲ ਤੋਂ ਲਿਆ ਜਾਂਦਾ ਹੈ ਜਾਂ ਓਪਲ ਨਾਲ ਬਣਾਇਆ ਜਾਂਦਾ ਹੈ) ਅਤੇ ਗੱਲਬਾਤ ਵਿੱਚ ਫਿਰੋਂ ਵਰਤਿਆ ਜਾ ਸਕਦਾ ਹੈ। |

> **ਉਪਲਬਧਤਾ:** ਪ੍ਰੋਜੈਕਟ ਓਪਲ ਇਸ ਸਮੇਂ [Frontier early access program](https://adoption.microsoft.com/copilot/frontier-program/) ਵਿੱਚ ਉਪਲਬਧ ਹੈ ਜਿੱਥੇ Microsoft 365 Copilot ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਦੀ ਲੋੜ ਹੈ, ਅਤੇ ਤੁਹਾਡੇ ਪ੍ਰਬੰਧਕ ਨੂੰ ਸੈਟਅੱਪ ਪੂਰਾ ਕਰਨਾ ਹੋਵੇਗਾ। ਕਿਉਂਕਿ ਇਹ ਪ੍ਰਯੋਗਸ਼ੀਲ Frontier ਫੀਚਰ ਹੈ, ਸਮਰੱਥਾਵਾਂ ਸਮੇਂ ਦੇ ਨਾਲ ਬਦਲ ਸਕਦੀਆਂ ਹਨ।
