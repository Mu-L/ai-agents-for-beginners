### ਨੋਟਸ ਅਤੇ ਜਾਣੀਆਂ ਸੀਮਾਵਾਂ

- **ਚਲਾਇਆ/ਕੰਪਾਈਲ ਨਹੀਂ ਕੀਤਾ।** ਇਹ ਸਿੱਖਣ ਵਾਲੇ ਨਮੂਨੇ ਹਨ ਜੋ API/ਨਾਮਕਰਨ ਸਹੀਤਾ ਲਈ ਅਪਡੇਟ ਕੀਤੇ ਗਏ ਹਨ; ਇਹ ਸਥਿਰ ਅਜ਼ੂਰੇ ਸਰੋਤਾਂ ਦੇ ਖਿਲਾਫ ਨਹੀਂ ਚਲਾਏ ਗਏ ਸਨ, ਅਤੇ .NET ਨਮੂਨੇ ਇਸ ਮਾਹੌਲ ਵਿੱਚ ਕੰਪਾਈਲ ਨਹੀਂ ਕੀਤੇ ਗਏ। ਆਪਣੇ ਖੁਦ ਦੇ Microsoft Foundry / Azure OpenAI ਡਿਪਲੌਇਮੈਂਟ ਦੇਖ ਚੱਕ ਕਰੋ।

- **ਏਜੰਟ-ਫਰੇਮਵਰਕ ਵਰਜਨ।** ਨਮੂਨੇ ਨਵੀਨਤਮ MAF (`>=1.10.0`) ਨੂੰ ਨਿਸ਼ਾਨਾ ਬਣਾਉਂਦੇ ਹਨ। ਕੈਨੀਨੀਕਲ ਏਜੰਟ-ਤਿਆਰ ਕਰਨ ਵਾਲੀ ਕਾਲ `client.as_agent(...)` ਹੈ; API ਦੀ ਪੁਸ਼ਟੀ ਫਰੇਮਵਰਕ ਦੇ ਪ੍ਰਕਾਸ਼ਿਤ ਦਸਤਾਵੇਜ਼ਾਂ ਅਤੇ ਇੰਸਟਾਲ ਕੀਤੇ ਬਿਲਡ ਨਾਲ ਕੀਤੀ ਗਈ ਸੀ। ਜੇ ਤੁਸੀਂ ਵੱਖ-ਵੱਖ ਵਰਜਨ ਨੂੰ ਚੁਣਦੇ ਹੋ ਤਾਂ ਵਿਧੀ ਉਪਲਬਧਤਾ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ (`as_agent` ਵਿਰੁੱਧ `create_agent`)।
- **ਲੈਸਨ 08 ਵਰਕਫਲੋ ਨੋਟਬੁੱਕ 04** ਇਰਾਦਾਤੰਤਰਿਕ ਤੌਰ 'ਤੇ `AzureAIAgentClient` (ਜੋ `agent-framework-azure-ai` ਤੋਂ ਹੈ) ਰੱਖਦਾ ਹੈ ਕਿਉਂਕਿ ਇਹ Microsoft Foundry ਏਜੰਟ ਸਰਵਿਸ ਹੋਸਟ ਕੀਤੇ ਸੰਦ (ਬਿੰਗ ਗ੍ਰਾਊਂਡਿੰਗ, ਕੋਡ ਇੰਟਰਪ੍ਰਿਟਰ) ਵਰਤਦਾ ਹੈ; ਇਹ ਪਹਿਲਾਂ ਹੀ Responses-ਅਧਾਰਿਤ ਹੈ।
- **.NET ਡਿਫਾਲਟ ਡਿਪਲੌਇਮੈਂਟ।** ਦੋ ਲੈਸਨ 08 dotNET ਵਰਕਫਲੋ ਨਮੂਨੇ ਪਹਿਲਾਂ ਇੱਕ ਖਾਸ ਮਾਡਲ ਨੂੰ ਹਾਰਡ-ਕੋਡ ਕਰਦੇ ਸਨ; ਹੁਣ ਇਹ `AZURE_OPENAI_DEPLOYMENT` (`gpt-4.1-mini`) ਤੇ ਡਿਫਾਲਟ ਹਨ। ਜੇ ਨਮੂਨਾ ਮਲਟੀਮੋਡਲ/ਦ੍ਰਿਸ਼ ਇਨਪੁੱਟ 'ਤੇ ਆਧਾਰਿਤ ਹੈ ਤਾਂ `AZURE_OPENAI_DEPLOYMENT` ਨੂੰ ਇੱਕ ਉਚਿਤ ਮਾਡਲ 'ਤੇ ਸੈੱਟ ਕਰੋ।
- **Foundry Local** ਇੱਕ OpenAI-ਸੰਗਤ **ਚੈਟ ਕਮਪਲੀਸ਼ਨਜ਼** ਐਂਡਪੋਇੰਟ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ ਅਤੇ ਇਹ ਸਥਾਨਕ ਵਿਕਾਸ ਲਈ ਬਣਾਇਆ ਗਿਆ ਹੈ; ਪੂਰੇ Responses API ਫੀਚਰ ਸੈੱਟ ਲਈ Azure OpenAI / Microsoft Foundry ਦਾ ਉਪਯੋਗ ਕਰੋ।
