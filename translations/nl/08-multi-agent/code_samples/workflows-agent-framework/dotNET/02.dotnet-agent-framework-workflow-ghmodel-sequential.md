# ⏩ Sequentiële Agent Workflows met Azure OpenAI (Responses API) (.NET)

## 📋 Geavanceerde Tutorial voor Sequentiële Verwerking

Deze notebook demonstreert **sequentiële workflowpatronen** met het Microsoft Agent Framework voor .NET en Azure OpenAI (Responses API). Je leert hoe je geavanceerde, stapsgewijze verwerkingspipelines bouwt waarbij agenten in een specifieke volgorde uitvoeren, waarbij elke fase voortbouwt op de resultaten van de vorige fase.

## 🎯 Leerdoelen

## 💻 Code Uitvoeren

De volledige implementatie is beschikbaar in `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Dit bestand demonstreert een **driefasen meubelanalyse-workflow**:

1. **Fase 1 - Sales Agent**: Analyseert meubelafbeeldingen en geeft aankoopaanbevelingen
2. **Fase 2 - Prijsagent**: Biedt gedetailleerde prijsopsplitsingen en budgetopties
3. **Fase 3 - Offerteagent**: Genereert een professioneel offertedocument in Markdown-formaat

### 🏗️ **Workflowarchitectuur**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Elke agent:
- Ontvangt de output van de vorige fase als context
- Bouwt voort op vorige analyse met gespecialiseerde expertise
- Behoudt workflowcontinuïteit via statusbeheer

**Vereisten:**
- Plaats een meubelafbeelding op `../imgs/home.png` (of werk de variabele `imgPath` bij)

### 📝 Verwachte Uitvoer

De workflow zal:
1. **Sales Agent**: Identificeer meubelstukken op de afbeelding en geef aanbevelingen
2. **Prijsagent**: Voeg gedetailleerde prijsanalyse toe met budgetlagen en winkeladviezen
3. **Offerteagent**: Genereer een geformatteerd offertedocument met alle gesynthetiseerde informatie

De uiteindelijke uitvoer zal een uitgebreide, professionele meubelofferte zijn op basis van afbeeldinganalyse.

- **Vastgoed**: Vastgoedanalyse → Waardering → Aanmaak van advertenties

### 🔍 Begrip van de Statusstroom

Elke agent in de reeks ontvangt:
- **Oorspronkelijke Invoer**: Het initiële gebruikersbericht (afbeelding + tekst)
- **Outputs van Vorige Agenten**: Alle eerdere agentantwoorden in het gespreksverloop
- **Verzamelde Context**: Volledige status die gedurende de workflow wordt vastgehouden

Dit maakt geavanceerde multi-stage verwerking mogelijk waarbij elke agent voortbouwt op uitgebreide context uit alle voorgaande fasen.
