# ⏩ Sequentiële Agent Workflows met Azure OpenAI (Responses API) (.NET)

## 📋 Geavanceerde Sequentiële Verwerking Tutorial

Deze notebook demonstreert **sequentiële workflowpatronen** met het Microsoft Agent Framework voor .NET en Azure OpenAI (Responses API). Je leert hoe je geavanceerde, stapsgewijze verwerkingspijplijnen bouwt waarin agenten in een specifieke volgorde uitvoeren, waarbij elke fase voortbouwt op de resultaten van de vorige fase.

## 💻 De Code Uitvoeren

De volledige implementatie is beschikbaar in `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Dit bestand toont een **driefasen meubelanalyseworkflow**:

1. **Fase 1 - Verkoopagent**: Analyseert meubelafbeeldingen en geeft aankoopadviezen
2. **Fase 2 - Prijsagent**: Biedt gedetailleerde prijsopgaven en budgetopties
3. **Fase 3 - Offerteagent**: Genereert een professioneel offerte-document in Markdown-formaat

### 🏗️ **Workflowarchitectuur**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Elke agent:
- Ontvangt de output van de vorige fase als context
- Bouwt voort op eerdere analyses met gespecialiseerde expertise
- Behoudt workflowcontinuïteit via statusbeheer

### 🚀 Het Voorbeeld Uitvoeren

**Vereisten:**
- Plaats een meubelafbeelding op `../imgs/home.png` (of update de variabele `imgPath`)
- Configureer je `.env` bestand met je Azure OpenAI endpoint en deployment, log vervolgens in met `az login`

```bash
# Maak het script uitvoerbaar (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Voer de sequentiële workflow uit
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Of op Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Verwachte Uitvoer

De workflow zal:
1. **Verkoopagent**: Meubelstukken identificeren uit de afbeelding en aanbevelingen geven
2. **Prijsagent**: Gedetailleerde prijsanalyse toevoegen met budgetniveaus en shopaanbevelingen
3. **Offerteagent**: Een geformatteerd offerte-document genereren met alle samengevoegde informatie

De uiteindelijke output is een uitgebreide, professionele meubelofferte gebaseerd op afbeeldinganalyse.

### 🔍 Inzicht in Statusstroom

Elke agent in de reeks ontvangt:
- **Oorspronkelijke Invoer**: Het initiële gebruikersbericht (afbeelding + tekst)
- **Outputs van Vorige Agenten**: Alle eerdere agentantwoorden in de gespreksgeschiedenis
- **Opgebouwde Context**: Volledige status die gedurende de workflow wordt behouden

Dit maakt geavanceerde meervoudige faseverwerking mogelijk waarbij elke agent voortbouwt op een uitgebreide context van alle voorgaande fases.
