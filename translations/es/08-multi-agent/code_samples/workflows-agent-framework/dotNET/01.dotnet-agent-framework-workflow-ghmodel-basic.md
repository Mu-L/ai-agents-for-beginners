# 🔄 Flujos de trabajo básicos de agentes con Azure OpenAI (API de Respuestas) (.NET)

## 📋 Tutorial de Orquestación de Flujos de Trabajo

Este cuaderno demuestra cómo construir **flujos de trabajo de agentes** sofisticados utilizando el Microsoft Agent Framework para .NET y Azure OpenAI (API de Respuestas). Aprenderás a crear procesos empresariales de múltiples pasos donde agentes de IA colaboran para lograr tareas complejas mediante patrones de orquestación estructurados.

## 🎯 Objetivos de Aprendizaje

### 🏗️ **Fundamentos de Arquitectura de Flujos de Trabajo**
- **Constructor de Flujos de Trabajo**: Diseñar y orquestar procesos complejos de IA con múltiples pasos
- **Coordinación de Agentes**: Coordinar múltiples agentes especializados dentro de los flujos de trabajo
- **Azure OpenAI (API de Respuestas)**: Aprovechar la API de Respuestas de Azure OpenAI en flujos de trabajo
- **Diseño Visual de Flujos de Trabajo**: Crear y visualizar estructuras de flujos para mejor comprensión

### 🔄 **Patrones de Orquestación de Procesos**
- **Procesamiento Secuencial**: Enlazar múltiples tareas de agentes en orden lógico
- **Gestión de Estado**: Mantener el contexto y el flujo de datos a través de las etapas del flujo de trabajo
- **Manejo de Errores**: Implementar recuperación robusta y resiliencia en el flujo de trabajo
- **Optimización de Rendimiento**: Diseñar flujos de trabajo eficientes para operaciones a escala empresarial

### 🏢 **Aplicaciones Empresariales de Flujos de Trabajo**
- **Automatización de Procesos de Negocio**: Automatizar flujos de trabajo organizacionales complejos
- **Cadena de Producción de Contenido**: Flujos editoriales con etapas de revisión y aprobación
- **Automatización de Servicio al Cliente**: Resolución de consultas de clientes en múltiples pasos
- **Flujos de Trabajo de Procesamiento de Datos**: Flujos ETL con transformación potenciada por IA

```xml

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Configuración de Azure OpenAI**

**Configuración del entorno (archivo .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Acceso a Azure OpenAI:**
1. Crea un recurso de Azure OpenAI en el portal de Azure
2. Despliega un modelo (por ejemplo, `gpt-4.1-mini`) y anota el nombre de despliegue
3. Inicia sesión con `az login` y configura las variables de entorno como se muestra arriba

**Componentes clave:**
- **WorkflowBuilder**: Motor principal de orquestación para diseñar flujos de trabajo
- **AIAgent**: Agentes especializados individuales con capacidades específicas
- **Cliente Azure OpenAI**: Integración con la API de Respuestas de Azure OpenAI
- **Contexto de Ejecución**: Gestiona estado y flujo de datos entre etapas del flujo de trabajo

## 💻 Ejecutando el Código

La implementación completa está disponible en `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Este archivo demuestra:

1. **Configuración del Entorno** - Cargando la configuración de Azure OpenAI desde el archivo `.env`
2. **Configuración del Cliente Azure OpenAI** - Configurando el cliente para usar la API de Respuestas de Azure OpenAI
3. **Creación de Agentes** - Definiendo agentes especializados (Recepción y Concierge)
4. **Constructor de Flujos de Trabajo** - Creando un flujo de trabajo con múltiples agentes y procesamiento secuencial
5. **Ejecución del Flujo de Trabajo** - Ejecutando el flujo con resultados en streaming

### 🚀 Ejecutando el Ejemplo

```bash
# Hacer el script ejecutable (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Ejecutar el flujo de trabajo
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

O en Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Salida Esperada

El flujo de trabajo hará lo siguiente:
1. Aceptar tu solicitud de destino de viaje ("Me gustaría ir a París")
2. El agente de Recepción ofrece una recomendación inicial
3. El agente Concierge revisa y refina la recomendación
4. La salida final muestra la conversación completa en streaming

### 🔧 Personalización

Puedes personalizar el flujo de trabajo:
- Modificando las instrucciones de los agentes para cambiar su comportamiento
- Añadiendo más agentes para crear flujos de trabajo complejos de múltiples pasos
- Cambiando el mensaje del usuario para probar diferentes escenarios
- Ajustando las conexiones del flujo para crear diferentes patrones de ejecución
