# 🔄 Flujos de trabajo básicos de agentes con Azure OpenAI (API de respuestas) (.NET)

## 📋 Tutorial de orquestación de flujos de trabajo

Este cuaderno demuestra cómo construir **flujos de trabajo de agentes** sofisticados utilizando el Microsoft Agent Framework para .NET y Azure OpenAI (API de respuestas). Aprenderás a crear procesos empresariales de varios pasos donde los agentes de IA colaboran para llevar a cabo tareas complejas mediante patrones de orquestación estructurados.

## 🎯 Objetivos de aprendizaje

### 🏗️ **Fundamentos de arquitectura de flujos de trabajo**
- **Constructor de flujos de trabajo**: Diseñar y orquestar procesos de IA complejos en múltiples pasos
- **Coordinación de agentes**: Coordinar múltiples agentes especializados dentro de flujos de trabajo
- **Azure OpenAI (API de respuestas)**: Aprovechar la API de respuestas de Azure OpenAI en flujos de trabajo
- **Diseño visual de flujos de trabajo**: Crear y visualizar estructuras de flujos de trabajo para mejor comprensión

### 🔄 **Patrones de orquestación de procesos**
- **Procesamiento secuencial**: Encadenar múltiples tareas de agentes en orden lógico
- **Gestión de estado**: Mantener contexto y flujo de datos a lo largo de las etapas del flujo de trabajo
- **Manejo de errores**: Implementar recuperación robusta de errores y resistencia del flujo de trabajo
- **Optimización de rendimiento**: Diseñar flujos de trabajo eficientes para operaciones a escala empresarial

### 🏢 **Aplicaciones empresariales de flujos de trabajo**
- **Automatización de procesos empresariales**: Automatizar flujos de trabajo organizacionales complejos
- **Cadena de producción de contenido**: Flujos editoriales con etapas de revisión y aprobación
- **Automatización de servicio al cliente**: Resolución de consultas de clientes en múltiples pasos
- **Flujos de trabajo de procesamiento de datos**: Flujos ETL con transformación potenciada por IA

```xml

<!-- Azure OpenAI (Responses API) -->

<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Configuración de Azure OpenAI**

**Configuración del entorno (archivo .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Acceso a Azure OpenAI:**
1. Crear un recurso Azure OpenAI en el portal de Azure
2. Desplegar un modelo (por ejemplo, `gpt-5-mini`) y anotar el nombre del despliegue
3. Iniciar sesión con `az login` y configurar las variables de entorno como se muestra arriba


**Componentes clave:**
- **WorkflowBuilder**: Motor principal de orquestación para diseñar flujos de trabajo
- **AIAgent**: Agentes individuales especializados con capacidades específicas
- **Cliente Azure OpenAI**: Integración con la API de respuestas de Azure OpenAI
- **Contexto de ejecución**: Gestiona estado y flujo de datos entre etapas del flujo de trabajo

## 💻 Ejecución del código

La implementación completa está disponible en `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Este archivo demuestra:

1. **Configuración del entorno** - Carga de configuración de Azure OpenAI desde archivo `.env`
2. **Configuración del cliente Azure OpenAI** - Configuración del cliente para usar la API de respuestas de Azure OpenAI
3. **Creación de agentes** - Definición de agentes especializados (Recepción y Conserjería)
4. **Constructor de flujos de trabajo** - Creación de un flujo con varios agentes con procesamiento secuencial
5. **Ejecución del flujo de trabajo** - Ejecución del flujo con resultados en streaming

### 🚀 Ejecutando el ejemplo

```bash
# Hacer el script ejecutable (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Ejecutar el flujo de trabajo
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

O en Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Resultado esperado

El flujo de trabajo:
1. Aceptará tu solicitud de destino de viaje ("Me gustaría ir a París")
2. El agente de Recepción proporciona una recomendación inicial
3. El agente de Conserjería revisa y refina la recomendación
4. La salida final muestra la conversación completa en streaming

### 🔧 Personalización

Puedes personalizar el flujo de trabajo:
- Modificando las instrucciones de los agentes para cambiar su comportamiento
- Añadiendo más agentes para crear flujos de trabajo complejos de múltiples pasos
- Cambiando el mensaje de usuario para probar diferentes escenarios
- Ajustando las conexiones del flujo de trabajo para crear diferentes patrones de ejecución
