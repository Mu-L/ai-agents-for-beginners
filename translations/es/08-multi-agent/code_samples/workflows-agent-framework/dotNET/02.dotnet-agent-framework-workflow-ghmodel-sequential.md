# ⏩ Flujos de Trabajo Secuenciales de Agentes con Azure OpenAI (API de Respuestas) (.NET)

## 📋 Tutorial Avanzado de Procesamiento Secuencial

Este cuaderno demuestra **patrones de flujo de trabajo secuencial** utilizando el Microsoft Agent Framework para .NET y Azure OpenAI (API de Respuestas). Aprenderás a construir tuberías de procesamiento sofisticadas, paso a paso, donde los agentes se ejecutan en un orden específico, y cada etapa se basa en los resultados de la etapa anterior.

### 🏗️ **Arquitectura de Flujo de Trabajo Secuencial**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Agente de Análisis]
    B --> C[Punto de Control 1]
    C --> D[Stage 2: Agente de Procesamiento]
    D --> E[Punto de Control 2]
    E --> F[Stage 3: Agente de Validación]
    F --> G[Resultado Final]
    
    H[Contexto de Estado] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API de Respuestas)] --> B
    I --> D
    I --> F
```

**Componentes Clave:**
- **Agentes Secuenciales**: Agentes especializados para cada etapa de procesamiento
- **Contexto de Estado**: Mantiene datos acumulados y decisiones a lo largo de las etapas
- **Puntos de Control**: Puntos de validación entre etapas para asegurar calidad y consistencia
- **Cliente Azure OpenAI**: Acceso consistente al modelo de IA en todas las etapas del flujo de trabajo

## 💻 Ejecución del Código

La implementación completa está disponible en `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Este archivo demuestra un **flujo de trabajo de análisis de muebles de tres etapas**:

1. **Etapa 1 - Agente de Ventas**: Analiza imágenes de muebles y proporciona sugerencias de compra
2. **Etapa 2 - Agente de Precios**: Proporciona desgloses detallados de precios y opciones de presupuesto
3. **Etapa 3 - Agente de Cotización**: Genera un documento de cotización profesional en formato Markdown

### 🏗️ **Arquitectura del Flujo de Trabajo**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Cada agente:
- Recibe la salida de la etapa previa como contexto
- Construye sobre el análisis anterior con especialización
- Mantiene la continuidad del flujo mediante la gestión de estado

### 🚀 Ejecución del Ejemplo

**Requisitos previos:**
- Colocar una imagen de muebles en `../imgs/home.png` (o actualizar la variable `imgPath`)
- Configurar tu archivo `.env` con tu endpoint y despliegue de Azure OpenAI, luego iniciar sesión con `az login`

```bash
# Hacer el script ejecutable (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Ejecutar el flujo de trabajo secuencial
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

O en Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Salida Esperada

El flujo de trabajo:
1. **Agente de Ventas**: Identificará los muebles en la imagen y proporcionará recomendaciones
2. **Agente de Precios**: Añadirá un análisis detallado de precios con niveles de presupuesto y recomendaciones de compra
3. **Agente de Cotización**: Generará un documento de cotización formateado con toda la información sintetizada

La salida final será una cotización de muebles completa y profesional basada en el análisis de la imagen.

### 🔍 Entendiendo el Flujo de Estado

Cada agente en la secuencia recibe:
- **Entrada Original**: El mensaje inicial del usuario (imagen + texto)
- **Salidas de Agentes Anteriores**: Todas las respuestas previas de agentes en el historial de la conversación
- **Contexto Acumulado**: Estado completo mantenido a lo largo del flujo de trabajo

Esto posibilita un procesamiento sofisticado de múltiples etapas donde cada agente se basa en un contexto comprensivo de todas las etapas anteriores.
