# ⏩ Flujos de Trabajo Secuenciales con Agentes y Azure OpenAI (API de Respuestas) (.NET)

## 📋 Tutorial Avanzado de Procesamiento Secuencial

Este cuaderno demuestra **patrones de flujo de trabajo secuencial** usando el Marco de Agentes de Microsoft para .NET y Azure OpenAI (API de Respuestas). Aprenderás a construir pipelines sofisticados de procesamiento paso a paso donde los agentes se ejecutan en un orden específico, con cada etapa basándose en los resultados de la etapa anterior.

- **Azure OpenAI (API de Respuestas)**: Aprovechar modelos Azure OpenAI en flujos de trabajo .NET multi-etapa

### 🏗️ **Patrones Secuenciales Avanzados**

- **Propagación de Errores**: Manejar fallas de manera elegante en cadenas de procesamiento secuencial

```env

```

**Gestión de configuración:**

### 🏗️ **Arquitectura del Flujo de Trabajo Secuencial**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Agente de Análisis]
    B --> C[Punto de Control 1]
    C --> D[Stage 2: Agente de Procesamiento]
    D --> E[Punto de Control 2]
    E --> F[Stage 3: Agente de Validación]
    F --> G[Resultado Final]
    
    H[Contexto de Estado] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API de Respuestas)] --> B
    I --> D
    I --> F
```

**Componentes Clave:**
- **Agentes Secuenciales**: Agentes especializados para cada etapa de procesamiento
- **Contexto de Estado**: Mantiene datos acumulados y decisiones a través de las etapas
- **Puntos de Control**: Puntos de validación entre etapas para asegurar calidad y coherencia
- **Cliente Azure OpenAI**: Acceso consistente a modelos de IA en todas las etapas del flujo

## 💻 Ejecución del Código

La implementación completa está disponible en `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Este archivo demuestra un **flujo de trabajo de análisis de muebles de tres etapas**:

1. **Etapa 1 - Agente de Ventas**: Analiza imágenes de muebles y proporciona sugerencias de compra
2. **Etapa 2 - Agente de Precio**: Proporciona desglose detallado de precios y opciones de presupuesto
3. **Etapa 3 - Agente de Cotización**: Genera un documento de cotización profesional en formato Markdown

### 🏗️ **Arquitectura del Flujo de Trabajo**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Cada agente:
- Recibe la salida de la etapa anterior como contexto
- Construye sobre el análisis previo con experiencia especializada
- Mantiene la continuidad del flujo mediante la gestión del estado

### 🚀 Ejecución del Ejemplo

**Requisitos previos:**
- Coloca una imagen de muebles en `../imgs/home.png` (o actualiza la variable `imgPath`)
- Configura tu archivo `.env` con tu endpoint y despliegue de Azure OpenAI, luego inicia sesión con `az login`

```bash
# Hacer el script ejecutable (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Ejecutar el flujo de trabajo secuencial
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

O en Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Salida Esperada

El flujo de trabajo realizará:
1. **Agente de Ventas**: Identificar ítems de muebles en la imagen y proporcionar recomendaciones
2. **Agente de Precio**: Añadir análisis detallado de precios con niveles de presupuesto y recomendaciones de compra
3. **Agente de Cotización**: Generar un documento de cotización formateado con toda la información sintetizada

La salida final será una cotización profesional y completa de muebles basada en el análisis de imagen.

### 🔍 Entendiendo el Flujo del Estado

Cada agente en la secuencia recibe:
- **Entrada Original**: El mensaje inicial del usuario (imagen + texto)
- **Salidas Previas de Agentes**: Todas las respuestas anteriores de agentes en el historial de la conversación
- **Contexto Acumulado**: Estado completo mantenido a lo largo del flujo de trabajo

Esto permite un procesamiento sofisticado multi-etapa donde cada agente se basa en un contexto completo de todas las etapas anteriores.
