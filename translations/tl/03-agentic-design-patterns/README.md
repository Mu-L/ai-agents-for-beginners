# Mga Prinsipyo ng Disenyo ng AI Agentic

## Panimula

Maraming mga paraan para pag-isipan ang pagbuo ng AI Agentic Systems. Dahil ang pagiging malabo ay isang katangian at hindi error sa disenyo ng Generative AI, minsan mahirap para sa mga inhinyero na malaman kung saan magsisimula. Nakagawa kami ng hanay ng mga prinsipyo sa human-centric UX Design upang payagan ang mga developer na bumuo ng customer-centric na mga agentic system para tugunan ang kanilang mga pangangailangan sa negosyo. Ang mga prinsipyong ito sa disenyo ay hindi isang tiyak na arkitektura kundi isang panimulang punto para sa mga koponan na nagdedeklara at bumubuo ng karanasan ng agent.

Sa pangkalahatan, ang mga agent ay dapat:

- Palawakin at palakihin ang kakayahan ng tao (pag-iisip, paglutas ng problema, awtomasyon, atbp.)
- Punan ang mga puwang sa kaalaman (pabilisin ako sa mga domain ng kaalaman, pagsasalin, atbp.)
- Pasimplihin at suportahan ang kolaborasyon sa mga paraan na nais ng tao na makatrabaho ang iba
- Gawin tayong mas mabuting bersyon ng ating mga sarili (hal., life coach/task master, pagtulong sa pag-aaral ng emosyonal na regulasyon at mindfulness skills, pagbuo ng katatagan, atbp.)

## Ang mga Prinsipyo ng Agentic Design

![Prinsipyo ng Agentic Design](../../../translated_images/tl/agentic-design-principles.1cfdf8b6d3cc73c2.webp)

### Agent (Espasyo)

Ito ang kapaligiran kung saan gumagana ang agent. Ang mga prinsipyong ito ay nagpapabatid kung paano natin idinisenyo ang mga agent para makilahok sa pisikal at digital na mga mundo.

- **Pagkonekta, hindi pagbagsak** – tumulong na ikonekta ang mga tao sa ibang tao, mga kaganapan, at kaalaman na may aksyon upang mapadali ang kolaborasyon at koneksyon.
- Tinutulungan ng mga agent na ikonekta ang mga kaganapan, kaalaman, at mga tao.
- Pinapalapit ng mga agent ang mga tao sa isa't isa. Hindi sila idinisenyo upang palitan o maliitin ang mga tao.
- **Madaling ma-access ngunit paminsan-minsan di-nakikita** – pangunahin gumagana ang agent sa background at hinihimas lang tayo kapag ito ay may kaugnayan at nararapat.
  - Madaling mahanap at ma-access ang agent para sa awtorisadong mga gumagamit sa anumang device o plataporma.
  - Sumasuporta ang agent sa multimodal na input at output (tunog, boses, teksto, atbp.).
  - Maaaring lumipat ang agent nang walang putol sa pagitan ng foreground at background; sa pagitan ng proactive at reactive, depende sa nararamdaman nitong pangangailangan ng gumagamit.
  - Maaaring gumana ang agent sa di-nakikitang anyo, ngunit ang background process path nito at kolaborasyon sa ibang mga Agent ay transparent at kontrolado ng gumagamit.

### Agent (Panahon)

Ganito gumagana ang agent sa paglipas ng panahon. Ang mga prinsipyong ito ay gabay sa pagdidisenyo ng mga agent na nakikipag-ugnayan sa nakaraan, kasalukuyan, at hinaharap.

- **Nakaraan**: Pagmumuni-muni sa kasaysayan na may kasamang estado at konteksto.
  - Nagbibigay ang agent ng mas may kabuluhang resulta batay sa pagsusuri ng mas mayamang historikal na datos lampas lamang sa kaganapan, mga tao, o estado.
  - Lumilikha ang agent ng mga koneksyon mula sa mga nakaraang kaganapan at aktibong nagmumuni-muni sa memorya upang makipag-ugnayan sa kasalukuyang mga sitwasyon.
- **Ngayon**: Hikayat higit pa sa pagbibigay-alam.
  - Pinanghahawakan ng agent ang isang komprehensibong paraan ng pakikipag-ugnayan sa mga tao. Kapag naganap ang isang kaganapan, lampas ang Agent sa statikong abiso o iba pang pormalidad. Kaya nitong pasimplehin ang mga daloy o dinamiko na bumuo ng mga pahiwatig upang idirekta ang atensyon ng gumagamit sa tamang sandali.
  - Nagbibigay ang agent ng impormasyon base sa kontekstuwal na kapaligiran, mga pagbabago sa panlipunan at kultural at iniangkop sa intensyon ng gumagamit.
  - Maaaring maging unti-unti ang pakikipag-ugnayan ng agent, lumalago o lumalalim ang kumplikasyon upang bigyan ng kapangyarihan ang mga gumagamit sa pangmatagalan.
- **Hinaharap**: Pagsasaayos at pag-unlad.
  - Inaangkop ng agent sa iba't ibang mga device, plataporma, at modalidad.
  - Inaangkop ng agent sa pag-uugali ng gumagamit, pangangailangan sa accessibility, at malayang nako-customize.
  - Huhubugin at uuunlad ang agent sa pamamagitan ng tuloy-tuloy na interaksyon ng gumagamit.

### Agent (Pangunahing Bahagi)

Ito ang mga pangunahing elemento sa puso ng disenyo ng isang agent.

- **Tangkilikin ang kawalang-katiyakan ngunit magtatag ng tiwala**.
  - Inaasahan ang isang antas ng kawalang-katiyakan ng Agent. Ang kawalang-katiyakan ay isang mahalagang elemento ng disenyo ng agent.
  - Tiwala at transparency ay pundamental na mga patong ng disenyo ng Agent.
  - Kontrolado ng tao kung kailan bukas/sara ang Agent at malinaw na nakikita ang status ng Agent sa lahat ng oras.

## Mga Patnubay para Ipapatupad ang Mga Prinsipyong Ito

Kapag ginagamit ang mga naunang prinsipyong disenyo, gamitin ang mga sumusunod na patnubay:

1. **Transparency**: Ipabatid sa gumagamit na may kasali na AI, kung paano ito gumagana (kabilang ang mga nakaraang aksyon), at kung paano magbigay ng puna at baguhin ang sistema.
2. **Kontrol**: Payagan ang gumagamit na i-customize, tukuyin ang mga kagustuhan at personalization, at magkaroon ng kontrol sa sistema at mga katangian nito (kabilang ang kakayahang kalimutan).
3. **Konsistensi**: Sikaping magkaroon ng tuloy-tuloy, multi-modal na karanasan sa mga device at endpoints. Gamitin ang mga pamilyar na UI/UX na elemento kung maaari (hal., icon ng mikropono para sa pakikipag-usap sa boses) at bawasan ang kognitibong bigat ng customer hangga't maaari (hal., sikaping maging maikli ang mga sagot, mga visual aid, at ‘Matuto Pa’ na nilalaman).

## Paano Magdisenyo ng Travel Agent gamit ang Mga Prinsipyo at Patnubay na Ito

Isipin na nagdidisenyo ka ng Travel Agent, ganito mo maiisip ang paggamit ng Mga Prinsipyo at Patnubay sa Disenyo:

1. **Transparency** – Ipabatid sa gumagamit na ang Travel Agent ay isang AI-enabled Agent. Magbigay ng ilang pangunahing tagubilin kung paano magsimula (hal., isang mensaheng “Hello”, mga halimbawa ng prompt). Maliwanag na idokumento ito sa pahina ng produkto. Ipakita ang listahan ng mga prompt na tanong na nagawa ng gumagamit noon. Linawin kung paano magbigay ng puna (thumbs up at down, Button na Magpadala ng Puna, atbp.). Ipahayag ng malinaw kung may mga limitasyon ang Agent sa paggamit o paksa.
2. **Kontrol** – Siguraduhing malinaw kung paano mababago ng gumagamit ang Agent pagkatapos itong malikha gamit ang mga bagay tulad ng System Prompt. Payagan ang gumagamit na piliin kung gaano karami ang sasabihin ng Agent, ang istilo ng pagsulat nito, at kahit anong babala sa kung ano ang hindi dapat pag-usapan ng Agent. Payagan ang gumagamit na makita at tanggalin ang mga kaugnay na file o data, prompt, at mga nakaraang pag-uusap.
3. **Konsistensi** – Siguraduhin na ang mga icon para sa Share Prompt, pagdaragdag ng file o larawan at pag-tag sa isang tao o bagay ay standard at madaling makilala. Gamitin ang paperclip icon para ipakita ang pag-upload/ibahagi ng file sa Agent, at icon ng larawan para ipakita ang pag-upload ng mga graphics.
