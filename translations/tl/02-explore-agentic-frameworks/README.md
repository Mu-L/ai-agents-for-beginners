# Tuklasin ang AI Agent Frameworks

Ang AI agent frameworks ay mga software platform na dinisenyo upang pasimplehin ang paggawa, pag-deploy, at pamamahala ng mga AI agent. Nagbibigay ang mga framework na ito ng mga paunang-buong bahagi, mga abstraksyon, at mga kasangkapan sa mga developer na nagpapadali sa pagbuo ng mga komplikadong AI system.

Tinutulungan ng mga framework na ito ang mga developer na magpokus sa natatanging aspeto ng kanilang mga aplikasyon sa pamamagitan ng pagbibigay ng mga standardized na paraan sa mga karaniwang hamon sa pag-develop ng AI agent. Pinapataas nila ang scalability, accessibility, at kahusayan sa pagbuo ng mga AI system.

## Ano ang AI Agent Frameworks at ano ang napapayagan nitong gawin ng mga developer?

Maaaring makatulong ang mga tradisyonal na AI Frameworks na isama ang AI sa iyong mga app at pagandahin ang mga ito sa mga sumusunod na paraan:

- **Personalization**: Kayang suriin ng AI ang ugali at mga kagustuhan ng user para magbigay ng mga personalized na rekomendasyon, nilalaman, at karanasan.
Halimbawa: Ginagamit ng mga streaming services tulad ng Netflix ang AI para magmungkahi ng mga pelikula at palabas base sa kasaysayan ng panonood, na nagpapataas ng pakikipag-ugnayan at kasiyahan ng user.
- **Automation at Kahusayan**: Kayang i-automate ng AI ang mga paulit-ulit na gawain, pasimplehin ang mga workflow, at pagbutihin ang kahusayan sa operasyon.
Halimbawa: Gumagamit ang mga customer service apps ng mga AI-powered chatbots para sagutin ang mga karaniwang tanong, na nagpapababa ng oras ng pagtugon at nagbibigay-daan sa mga human agent na tutukan ang mas kumplikadong mga isyu.
- **Pinahusay na Karanasan ng User**: Pinagaganda ng AI ang kabuuang karanasan ng user sa pamamagitan ng pagbigay ng mga intelihenteng tampok tulad ng voice recognition, natural language processing, at predictive text.
Halimbawa: Ginagamit ng mga virtual assistant tulad ng Siri at Google Assistant ang AI upang maintindihan at tumugon sa mga voice commands, na nagpapadali sa pakikipag-ugnayan ng mga user sa kanilang mga device.

### Ang ganda naman ng lahat ng iyon, pero bakit kailangan natin ang AI Agent Framework?

Ang AI Agent frameworks ay hindi lang basta AI frameworks. Dinisenyo ang mga ito para payagan ang paggawa ng mga intelligent agent na kayang makipag-ugnayan sa mga user, ibang agent, at sa kapaligiran upang makamit ang mga tiyak na layunin. Kayang magpakita ng autonomous behavior, gumawa ng mga desisyon, at mag-adapt sa nagbabagong kondisyon ang mga agent na ito. Tingnan natin ang ilang mga pangunahing kakayahang naibibigay ng AI Agent Frameworks:

- **Kolaborasyon at Koordinasyon ng Agent**: Pinapahintulutan ang paggawa ng maramihang AI agent na maaaring magtrabaho nang sabay-sabay, makipag-ugnayan, at mag-coordinate para lutasin ang mga komplikadong gawain.
- **Automation at Pamamahala ng Gawain**: Nagbibigay ng mga mekanismo para sa pag-automate ng mga multi-step na workflow, pagtatalaga ng gawain, at dynamic na pamamahala ng mga gawain sa pagitan ng mga agent.
- **Pag-unawa sa Konteksto at Adaptasyon**: Nilalapatan ang mga agent ng kakayahang maintindihan ang konteksto, mag-adapt sa nagbabagong kapaligiran, at gumawa ng desisyon batay sa real-time na impormasyon.

Kaya sa kabuuan, pinapayagan ka ng mga agent na gawin ang higit pa, iangat ang automation sa susunod na antas, at lumikha ng mas intelihenteng mga sistema na maaaring mag-adapt at matuto mula sa kanilang kapaligiran.

## Paano mabilis na makabuo ng prototype, mag-iterate, at pagbutihin ang kakayahan ng agent?

Mabilis ang takbo ng larangang ito, pero may ilang mga pangkaraniwang bagay sa karamihan ng AI Agent Frameworks na makakatulong sa mabilis na paggawa ng prototype at pag-iterate gaya ng module components, kolaboratibong kasangkapan, at real-time na pagkatuto. Talakayin natin ang mga ito:

- **Gamitin ang Modular Components**: Nag-aalok ang mga AI SDK ng paunang-buong mga bahagi tulad ng AI at Memory connectors, function calling gamit ang natural language o code plugins, mga prompt templates, at iba pa.
- **Samantalahin ang Kolaboratibong Kasangkapan**: Disenyuhin ang mga agent na may partikular na mga tungkulin at gawain, na nagpapahintulot sa mga ito na subukan at pahusayin ang mga collaborative workflow.
- **Matuto nang Real-Time**: Magpatupad ng mga feedback loop kung saan natututo ang mga agent mula sa mga interaksyon at inaayos ang kanilang pag-uugali nang dinamik.

### Gamitin ang Modular Components

Nag-aalok ang mga SDK tulad ng Microsoft Agent Framework ng paunang-buong mga bahagi tulad ng AI connectors, tool definitions, at agent management.

**Paano magagamit ng mga koponan ito**: Madaling mapagsama-sama ng mga koponan ang mga bahagi na ito upang makagawa ng functional prototype nang hindi nagsisimula mula sa wala, na nagbibigay-daan sa mabilis na eksperimento at iterasyon.

**Paano ito gumagana sa praktika**: Maaari kang gumamit ng paunang-buong parser para kunin ang impormasyon mula sa user input, isang memory module para mag-imbak at kumuha ng data, at isang prompt generator para makipag-ugnayan sa mga user, lahat nang hindi kailangang bumuo ng mga bahagi na ito mula sa simula.

**Halimbawang code**. Tingnan natin ang isang halimbawa kung paano mo magagamit ang Microsoft Agent Framework gamit ang `FoundryChatClient` upang magpa-responde ang modelo sa user input na may tool calling:

``` python
# Microsoft Agent Framework Halimbawa sa Python

import asyncio
import os

from agent_framework import tool
from agent_framework.foundry import FoundryChatClient
from azure.identity import AzureCliCredential


# Tukuyin ang isang halimbawa ng tool na function para mag-book ng biyahe
@tool(approval_mode="never_require")
def book_flight(date: str, location: str) -> str:
    """Book travel given location and date."""
    return f"Travel was booked to {location} on {date}"


async def main():
    provider = FoundryChatClient(
        project_endpoint=os.environ["AZURE_AI_PROJECT_ENDPOINT"],
        model=os.environ["AZURE_AI_MODEL_DEPLOYMENT_NAME"],
        credential=AzureCliCredential(),
    )
    agent = provider.as_agent(
        name="travel_agent",
        instructions="Help the user book travel. Use the book_flight tool when ready.",
        tools=[book_flight],
    )

    response = await agent.run("I'd like to go to New York on January 1, 2025")
    print(response)
    # Halimbawa ng output: Ang iyong flight papuntang New York sa Enero 1, 2025 ay matagumpay na na-book. Maligtas na paglalakbay! ✈️🗽


if __name__ == "__main__":
    asyncio.run(main())
```

Makikita mo sa halimbawang ito kung paano ka maaaring gumamit ng paunang-buong parser para kunin ang mga pangunahing impormasyon mula sa input ng user, tulad ng pinagmulan, destinasyon, at petsa ng flight booking request. Pinapayagan ka ng modular na paraan na ito na magpokus sa mataas na antas ng lohika.

### Samantalahin ang Kolaboratibong Kasangkapan

Pinapadali ng mga framework tulad ng Microsoft Agent Framework ang paggawa ng maraming agent na maaaring magtrabaho nang magkakasama.

**Paano magagamit ng mga koponan ito**: Maaaring magdisenyo ang mga koponan ng mga agent na may partikular na tungkulin at gawain, na nagpapahintulot sa kanila na subukan at pahusayin ang kolaboratibong workflow at paigtingin ang kahusayan ng sistema.

**Paano ito gumagana sa praktika**: Maaari kang gumawa ng isang koponan ng mga agent kung saan bawat isa ay may espesyal na tungkulin, tulad ng pagkuha ng data, pagsusuri, o paggawa ng desisyon. Makikipag-ugnayan at magsasalo-salo ang mga agent na ito upang makamit ang isang karaniwang layunin, tulad ng pagsagot sa tanong ng user o pagtapos ng gawain.

**Halimbawang code (Microsoft Agent Framework)**:

```python
# Lumilikha ng maraming ahente na nagtutulungan gamit ang Microsoft Agent Framework

import os
from agent_framework.foundry import FoundryChatClient
from azure.identity import AzureCliCredential

provider = FoundryChatClient(
    project_endpoint=os.environ["AZURE_AI_PROJECT_ENDPOINT"],
    model=os.environ["AZURE_AI_MODEL_DEPLOYMENT_NAME"],
    credential=AzureCliCredential(),
)

# Ahente ng Pagkuha ng Datos
agent_retrieve = provider.as_agent(
    name="dataretrieval",
    instructions="Retrieve relevant data using available tools.",
    tools=[retrieve_tool],
)

# Ahente ng Pagsusuri ng Datos
agent_analyze = provider.as_agent(
    name="dataanalysis",
    instructions="Analyze the retrieved data and provide insights.",
    tools=[analyze_tool],
)

# Patakbuhin ang mga ahente nang sunud-sunod sa isang gawain
retrieval_result = await agent_retrieve.run("Retrieve sales data for Q4")
analysis_result = await agent_analyze.run(f"Analyze this data: {retrieval_result}")
print(analysis_result)
```

Makikita mo sa naunang code kung paano gumawa ng isang gawain na kinabibilangan ng maramihang agent na nagtutulungan para suriin ang data. Gumagawa ang bawat agent ng isang tiyak na tungkulin, at pinapatupad ang gawain sa pamamagitan ng koordinasyon ng mga agent para makamit ang nais na resulta. Sa pamamagitan ng paggawa ng dedikadong mga agent na may spesyal na mga tungkulin, mapapabuti mo ang kahusayan at performance ng gawain.

### Matuto nang Real-Time

Nagbibigay ang mga advanced na framework ng mga kakayahan para sa pag-unawa sa konteksto at adaptasyon sa real-time.

**Paano magagamit ng mga koponan ito**: Maaaring magpatupad ang mga koponan ng mga feedback loop kung saan natututo ang mga agent mula sa mga interaksyon at inaayos ang kanilang pag-uugali nang dinamiko, na nagreresulta sa tuloy-tuloy na pagbuti at pagpapahusay ng mga kakayahan.

**Paano ito gumagana sa praktika**: Maaari suriin ng mga agent ang feedback ng user, datos ng kapaligiran, at resulta ng mga gawain upang i-update ang kanilang knowledge base, ayusin ang mga algorithm sa paggawa ng desisyon, at pagbutihin ang performance sa pagdaan ng panahon. Pinapayagan ng proseso ng paulit-ulit na pagkatuto na ito ang mga agent na mag-adapt sa nagbabagong kondisyon at mga kagustuhan ng user, na nagpapabuti sa kabuuang bisa ng sistema.

## Ano ang mga pagkakaiba ng Microsoft Agent Framework at Microsoft Foundry Agent Service?

Maraming paraan para ihambing ang mga approaches na ito, ngunit tingnan natin ang ilang mga pangunahing pagkakaiba sa kanilang disenyo, kakayahan, at target na gamit:

## Microsoft Agent Framework (MAF)

Ang Microsoft Agent Framework ay nagbibigay ng isang streamlined SDK para sa paggawa ng AI agent gamit ang `FoundryChatClient`. Pinapayagan nito ang mga developer na gumawa ng mga agent na gumagamit ng mga Azure OpenAI modelo na may kasamang built-in tool calling, pamamahala ng pag-uusap, at enterprise-grade na seguridad sa pamamagitan ng Azure identity.

**Mga Gamit**: Pagbuo ng production-ready AI agents na may gamit ng mga tool, multi-step workflows, at mga enterprise integration scenario.

Narito ang ilang mahahalagang pangunahing konsepto ng Microsoft Agent Framework:

- **Agents**. Ang isang agent ay nililikha gamit ang `FoundryChatClient` at naisasetup gamit ang pangalan, mga instruksyon, at mga tool. Kayang gawin ng agent:
  - **Proseso ng mga mensahe ng user** at gumawa ng mga tugon gamit ang Azure OpenAI models.
  - **Awtomatikong tumawag sa mga tool** batay sa konteksto ng pag-uusap.
  - **Panatilihin ang estado ng pag-uusap** sa maraming interaksyon.

  Narito ang isang code snippet na nagpapakita kung paano gumawa ng agent:

    ```python
    import os
    from agent_framework.foundry import FoundryChatClient
    from azure.identity import AzureCliCredential

    provider = FoundryChatClient(
        project_endpoint=os.environ["AZURE_AI_PROJECT_ENDPOINT"],
        model=os.environ["AZURE_AI_MODEL_DEPLOYMENT_NAME"],
        credential=AzureCliCredential(),
    )
    agent = provider.as_agent(
        name="my_agent",
        instructions="You are a helpful assistant.",
    )

    response = await agent.run("Hello, World!")
    print(response)
    ```

- **Tools**. Sinusuportahan ng framework ang pag-defina ng mga tool bilang mga Python function na maaaring tawagin ng agent nang awtomatiko. Nire-register ang mga tool kapag ginagawa ang agent:

    ```python
    def get_weather(location: str) -> str:
        """Get the current weather for a location."""
        return f"The weather in {location} is sunny, 72\u00b0F."

    agent = provider.as_agent(
        name="weather_agent",
        instructions="Help users check the weather.",
        tools=[get_weather],
    )
    ```

- **Multi-Agent Coordination**. Maaari kang gumawa ng maramihang agent na may iba't ibang espesyalisasyon at i-coordinate ang kanilang trabaho:

    ```python
    planner = provider.as_agent(
        name="planner",
        instructions="Break down complex tasks into steps.",
    )

    executor = provider.as_agent(
        name="executor",
        instructions="Execute the planned steps using available tools.",
        tools=[execute_tool],
    )

    plan = await planner.run("Plan a trip to Paris")
    result = await executor.run(f"Execute this plan: {plan}")
    ```

- **Azure Identity Integration**. Ginagamit ng framework ang `AzureCliCredential` (o `DefaultAzureCredential`) para sa secure, keyless authentication, na inaalis ang pangangailangan na i-manage ang API keys nang direktang.

## Microsoft Foundry Agent Service

Ang Microsoft Foundry Agent Service ay isang mas bagong karagdagan, ipinakilala sa Microsoft Ignite 2024. Pinapayagan nito ang paggawa at pag-deploy ng mga AI agent gamit ang mas flexible na mga modelo, gaya ng direktang pagtawag sa open-source LLMs tulad ng Llama 3, Mistral, at Cohere.

Nagbibigay ang Microsoft Foundry Agent Service ng mas matibay na mekanismo ng enterprise security at mga paraan ng data storage, kaya angkop ito para sa mga application ng enterprise. 

Gumagana ito agad-agad sa Microsoft Agent Framework para sa paggawa at pag-deploy ng mga agent.

Ang serbisyong ito ay kasalukuyang nasa Public Preview at sumusuporta sa Python at C# para sa paggawa ng mga agent.

Sa paggamit ng Microsoft Foundry Agent Service Python SDK, makakagawa tayo ng agent na may user-defined na tool:

```python
import asyncio
from azure.identity import DefaultAzureCredential
from azure.ai.projects import AIProjectClient

# Tukuyin ang mga function ng tool
def get_specials() -> str:
    """Provides a list of specials from the menu."""
    return """
    Special Soup: Clam Chowder
    Special Salad: Cobb Salad
    Special Drink: Chai Tea
    """

def get_item_price(menu_item: str) -> str:
    """Provides the price of the requested menu item."""
    return "$9.99"


async def main() -> None:
    credential = DefaultAzureCredential()
    project_client = AIProjectClient.from_connection_string(
        credential=credential,
        conn_str="your-connection-string",
    )

    agent = project_client.agents.create_agent(
        model="gpt-4.1-mini",
        name="Host",
        instructions="Answer questions about the menu.",
        tools=[get_specials, get_item_price],
    )

    thread = project_client.agents.create_thread()

    user_inputs = [
        "Hello",
        "What is the special soup?",
        "How much does that cost?",
        "Thank you",
    ]

    for user_input in user_inputs:
        print(f"# User: '{user_input}'")
        message = project_client.agents.create_message(
            thread_id=thread.id,
            role="user",
            content=user_input,
        )
        run = project_client.agents.create_and_process_run(
            thread_id=thread.id, agent_id=agent.id
        )
        messages = project_client.agents.list_messages(thread_id=thread.id)
        print(f"# Agent: {messages.data[0].content[0].text.value}")


if __name__ == "__main__":
    asyncio.run(main())
```

### Pangunahing konsepto

May mga sumusunod na pangunahing konsepto ang Microsoft Foundry Agent Service:

- **Agent**. Ang Microsoft Foundry Agent Service ay nakikisalamuha sa Microsoft Foundry. Sa loob ng Microsoft Foundry, ang isang AI Agent ay gumaganap bilang isang "smart" microservice na maaaring gamitin upang sagutin ang mga tanong (RAG), magsagawa ng mga aksyon, o pangganap ng buong automation ng mga workflow. Nakakamit ito sa pamamagitan ng pagsasama ng kapangyarihan ng generative AI models sa mga tool na nagpapahintulot dito na ma-access at makipag-ugnayan sa mga totoong data source. Narito ang isang halimbawa ng isang agent:

    ```python
    agent = project_client.agents.create_agent(
        model="gpt-4.1-mini",
        name="my-agent",
        instructions="You are helpful agent",
        tools=code_interpreter.definitions,
        tool_resources=code_interpreter.resources,
    )
    ```

    Sa halimbawang ito, nilikha ang isang agent gamit ang modelong `gpt-4.1-mini`, pangalan na `my-agent`, at mga instruksyon na `You are helpful agent`. Ang agent ay nilagyan ng mga tool at resources upang magsagawa ng mga gawain sa pag-interpret ng code.

- **Thread at mga mensahe**. Ang thread ay isa pang mahalagang konsepto. Kinakatawan nito ang pag-uusap o interaksyon sa pagitan ng agent at user. Maaaring gamitin ang mga thread upang subaybayan ang progreso ng pag-uusap, mag-imbak ng konteksto, at pamahalaan ang estado ng interaksyon. Narito ang isang halimbawa ng thread:

    ```python
    thread = project_client.agents.create_thread()
    message = project_client.agents.create_message(
        thread_id=thread.id,
        role="user",
        content="Could you please create a bar chart for the operating profit using the following data and provide the file to me? Company A: $1.2 million, Company B: $2.5 million, Company C: $3.0 million, Company D: $1.8 million",
    )
    
    # Hilingin sa ahente na magsagawa ng gawain sa thread
    run = project_client.agents.create_and_process_run(thread_id=thread.id, agent_id=agent.id)
    
    # Kunin at i-log lahat ng mga mensahe upang makita ang tugon ng ahente
    messages = project_client.agents.list_messages(thread_id=thread.id)
    print(f"Messages: {messages}")
    ```

    Sa naunang code, nilikha ang isang thread. Pagkatapos, isang mensahe ang ipinadala sa thread. Sa pagtawag sa `create_and_process_run`, hinihiling sa agent na magsagawa ng trabaho sa thread. Sa huli, kinukuha at nilalagay sa log ang mga mensahe upang makita ang tugon ng agent. Ipinapakita ng mga mensahe ang progreso ng pag-uusap sa pagitan ng user at agent. Mahalaga ring maunawaan na maaaring magkakaiba ang uri ng mga mensahe gaya ng teksto, larawan, o file, na resulta ng trabaho ng mga agent, halimbawa isang larawan o isang tugon na teksto. Bilang developer, maaari mong gamitin ang impormasyong ito para sa karagdagang pagproseso ng tugon o pagpipresenta nito sa user.

- **Nakasama sa Microsoft Agent Framework**. Gumagana nang maayos ang Microsoft Foundry Agent Service kasama ang Microsoft Agent Framework, ibig sabihin maaari kang gumawa ng mga agent gamit ang `FoundryChatClient` at i-deploy ang mga ito sa pamamagitan ng Agent Service para sa mga production scenario.

**Mga Gamit**: Dinisenyo ang Microsoft Foundry Agent Service para sa mga application ng enterprise na nangangailangan ng secure, scalable, at flexible na pag-deploy ng AI agent.

## Ano ang pagkakaiba ng mga approach na ito?
 
Mukhang mayroong overlap, ngunit may ilang mahahalagang pagkakaiba sa kanilang disenyo, kakayahan, at target na gamit:
 
- **Microsoft Agent Framework (MAF)**: Isang production-ready SDK para sa paggawa ng AI agents. Nagbibigay ito ng streamlined API para sa paggawa ng mga agent na may tool calling, pamamahala ng pag-uusap, at Azure identity integration.
- **Microsoft Foundry Agent Service**: Isang platform at service sa Microsoft Foundry para sa mga agent. Nag-aalok ito ng built-in connectivity sa mga serbisyo tulad ng Azure OpenAI, Azure AI Search, Bing Search, at code execution.

### Mga Gamit
 
Tingnan natin kung makakatulong kami sa pamamagitan ng ilang mga karaniwang gamit:
 
> Q: Gumagawa ako ng production AI agent application at nais kong makapagsimula nang mabilis
>

> A: Ang Microsoft Agent Framework ay isang mahusay na pagpipilian. Nagbibigay ito ng isang simpleng, Pythonic na API sa pamamagitan ng `FoundryChatClient` na nagpapahintulot sa iyo na magdefine ng mga agent na may mga tool at instruksyon sa ilang linya ng code lamang.

> Q: Kailangan ko ng enterprise-grade na deployment na may Azure integrations tulad ng Search at code execution
>
> A: Ang Microsoft Foundry Agent Service ang pinaka-angkop. Ito ay isang platform service na may built-in na kakayahan para sa maramihang mga modelo, Azure AI Search, Bing Search, at Azure Functions. Pinadadali nito ang paggawa ng iyong mga agent sa Foundry Portal at pag-deploy sa malaking sukat.
 
> Q: Nalilito pa rin ako, bigyan mo na lang ako ng isang opsyon
>
> A: Magsimula sa Microsoft Agent Framework upang bumuo ng iyong mga agent, at gamitin ang Microsoft Foundry Agent Service kapag kailangan mong i-deploy at iskalahin ang mga ito sa production. Pinapayagan ka ng approach na ito na mag-iterate nang mabilis sa iyong agent logic habang may malinaw na landas patungo sa enterprise deployment.
 
Ibuod natin ang mga pangunahing pagkakaiba sa isang talahanayan:

| Framework | Pokus | Pangunahing Konsepto | Mga Gamit |
| --- | --- | --- | --- |
| Microsoft Agent Framework | Streamlined agent SDK na may tool calling | Agents, Tools, Azure Identity | Paggawa ng AI agents, gamit ng tool, multi-step workflows |
| Microsoft Foundry Agent Service | Flexible na mga modelo, enterprise security, Code generation, Tool calling | Modularity, Kolaborasyon, Process Orchestration | Secure, scalable, at flexible na pag-deploy ng AI agent |

## Maaari ko bang direktang isama ang aking umiiral na Azure ecosystem tools, o kailangan ko ba ng standalone solutions?


Oo, maaari mong i-integrate ang iyong umiiral na mga Azure ecosystem tools nang direkta sa Microsoft Foundry Agent Service lalo na, dahil ito ay ginawa upang gumana nang maayos sa iba pang mga Azure services. Maaari mong i-integrate ang Bing, Azure AI Search, at Azure Functions bilang halimbawa. Mayroon ding malalim na integrasyon sa Microsoft Foundry.

Ang Microsoft Agent Framework ay nag-iintegrate din sa mga Azure services sa pamamagitan ng `FoundryChatClient` at Azure identity, na nagpapahintulot sa iyo na tawagan ang mga Azure services nang direkta mula sa iyong mga agent tools.
