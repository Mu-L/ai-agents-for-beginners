# ⏩ Sunud-sunod na Mga Workflow ng Ahente gamit ang Azure OpenAI (Responses API) (.NET)

## 📋 Advanced na Tutorial sa Sunud-sunod na Pagpoproseso

Ipinapakita ng notebook na ito ang mga **sunud-sunod na pattern ng workflow** gamit ang Microsoft Agent Framework para sa .NET at Azure OpenAI (Responses API). Matututuhan mo kung paano bumuo ng sopistikadong, hakbang-hakbang na mga pipeline ng pagpoproseso kung saan ang mga ahente ay nagsasagawa sa isang partikular na pagkakasunod-sunod, na bawat yugto ay nakabatay sa mga resulta ng naunang yugto.

## 🎯 Mga Layunin sa Pagkatuto

### 🔄 **Arkitektura ng Sunud-sunod na Pagpoproseso**
- **Disenyo ng Linear Workflow**: Gumawa ng hakbang-hakbang na mga pipeline ng pagpoproseso na may malinaw na mga dependency
- **Pamamahala ng Estado**: Panatilihin ang konteksto at daloy ng datos sa loob ng mga sunud-sunod na yugto ng workflow
- **Azure OpenAI (Responses API)**: Gamitin ang mga Azure OpenAI model sa multi-yugtong mga workflow ng .NET
- **Mga Pattern ng Enterprise Pipeline**: Bumuo ng mga sistemang sunud-sunod na pagpoproseso na handa para sa produksyon

### 🏗️ **Advanced na Mga Pattern ng Sunud-sunod**
- **Stage-Gate Processing**: Magpatupad ng mga validation checkpoint sa pagitan ng mga yugto ng workflow
- **Pagpapanatili ng Konteksto**: Panatilihin ang estado at naipong kaalaman sa lahat ng yugto
- **Pagpapasa ng Mali**: Harapin nang maayos ang mga pagkakamali sa mga sunud-sunod na chain ng pagpoproseso
- **Pag-optimize ng Performance**: Mabisang sunud-sunod na pagpapatupad na may minimal na dagdag na gastos

### 🏢 **Mga Enterprise Sequential Application**
- **Document Processing Pipeline**: Multi-yugtong pagsusuri, pagbabago, at pag-validate ng mga dokumento
- **Quality Assurance Workflows**: Sunud-sunod na pagsusuri, pag-validate, at mga proseso ng pag-apruba
- **Content Production Pipeline**: Pananaliksik → Pagsusulat → Pag-edit → Pagsusuri → Paglalathala
- **Automation ng Proseso ng Negosyo**: Multi-hakbang na mga workflow ng negosyo na may malinaw na dependency sa bawat yugto

## ⚙️ Mga Kinakailangan at Setup

Mahahalagang package para sa mga sunud-sunod na workflow ng .NET:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Konfigurasyon ng Azure OpenAI**

**Setup ng Kapaligiran (.env file):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Pamamahala ng Konfigurasyon:**
```csharp
// Load environment variables securely
```

### 🏗️ **Arkitektura ng Sunud-sunod na Workflow**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Ahente ng Pagsusuri]
    B --> C[Lagusan 1]
    C --> D[Stage 2: Ahente ng Pagpoproseso]
    D --> E[Lagusan 2]
    E --> F[Stage 3: Ahente ng Pagpapatunay]
    F --> G[Panghuling Output]
    
    H[Konteksto ng Estado] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (API ng Mga Sagot)] --> B
    I --> D
    I --> F
```

**Mga Pangunahing Komponent:**
- **Sunud-sunod na Mga Ahente**: Mga espesyal na ahente para sa bawat yugto ng pagpoproseso
- **Konteksto ng Estado**: Nagpapanatili ng naipong datos at mga desisyon sa bawat yugto
- **Mga Checkpoint**: Mga validation point sa pagitan ng mga yugto para matiyak ang kalidad at pagkakapare-pareho
- **Azure OpenAI Client**: Konsistenteng access sa AI model sa lahat ng mga yugto ng workflow

### 📊 **Pagsubaybay at Analitika**

- **Pagkilala ng Bottleneck**: Tukuyin at i-optimize ang mga mabagal na yugto
- **Mga Metric ng Kalidad**: Subaybayan ang kalidad at mga rate ng tagumpay sa bawat yugto

## 💻 Pagpapatakbo ng Code

Ang kompletong implementasyon ay makikita sa `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Ipinapakita ng file na ito ang **tatlong-yugtong workflow ng pagsusuri ng kasangkapan**:

1. **Yugto 1 - Sales Agent**: Sinusuri ang mga larawan ng kasangkapan at nagbibigay ng mga mungkahi sa pagbili
2. **Yugto 2 - Price Agent**: Nagbibigay ng detalyadong breakdown ng presyo at mga opsyon sa badyet
3. **Yugto 3 - Quote Agent**: Gumagawa ng propesyonal na dokumento ng quote sa format na Markdown

### 🏗️ **Arkitektura ng Workflow**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Bawat ahente:
- Tumanggap ng output mula sa naunang yugto bilang konteksto
- Nagtatayo sa naunang pagsusuri gamit ang espesyal na kaalaman
- Pinapanatili ang tuloy-tuloy na daloy ng workflow sa pamamagitan ng pamamahala ng estado

### 🚀 Pagpapatakbo ng Halimbawa

**Mga Kinakailangan:**
- Maglagay ng larawan ng kasangkapan sa `../imgs/home.png` (o i-update ang variable na `imgPath`)

### 📝 Inaasahang Output

Ang workflow ay:
1. **Sales Agent**: Tukuyin ang mga item ng kasangkapan mula sa imahe at magbigay ng mga rekomendasyon
2. **Price Agent**: Magdagdag ng detalyadong pagsusuri sa presyo na may mga tier ng badyet at mga rekomendasyon sa pamimili
3. **Quote Agent**: Lumikha ng isang naka-format na dokumento ng quote na isinalin ang lahat ng impormasyon

Ang panghuling output ay magiging isang komprehensibo, propesyonal na quote para sa kasangkapan batay sa pagsusuri ng imahe.

### 🔍 Pag-unawa sa Daloy ng Estado

Bawat ahente sa sunod-sunod ay tumatanggap ng:
- **Orihinal na Input**: Ang unang mensahe ng user (larawan + teksto)
- **Mga Output ng Nakaraang Ahente**: Lahat ng mga sagot ng naunang ahente sa kasaysayan ng pag-uusap
- **Naipong Konteksto**: Kumpletong estado na pinanatili sa buong workflow

Pinahihintulutan nito ang sopistikadong multi-yugtong pagpoproseso kung saan ang bawat ahente ay nagtutuloy-tuloy sa komprehensibong konteksto mula sa lahat ng naunang yugto.
