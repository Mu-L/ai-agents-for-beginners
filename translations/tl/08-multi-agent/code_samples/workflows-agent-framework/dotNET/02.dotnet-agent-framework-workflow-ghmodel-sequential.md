# ⏩ Sunod-sunod na Mga Daloy ng Gawain ng Ahente gamit ang Azure OpenAI (Responses API) (.NET)

## 📋 Advanced na Tutorial ng Sunod-sunod na Pagpoproseso

Ipinapakita ng notebook na ito ang **mga pattern ng sunod-sunod na daloy ng gawain** gamit ang Microsoft Agent Framework para sa .NET at Azure OpenAI (Responses API). Matututunan mo kung paano bumuo ng mga sopistikadong, hakbang-hakbang na pipelines ng pagpoproseso kung saan ang mga ahente ay magsasagawa sa isang partikular na pagkakasunod-sunod, na bawat yugto ay nakabatay sa mga resulta ng naunang yugto.

## 🎯 Mga Layunin sa Pagkatuto

### 🔄 **Arkitektura ng Sunod-sunod na Pagpoproseso**
- **Linear Workflow Design**: Lumikha ng hakbang-hakbang na mga pipeline ng pagpoproseso na may malinaw na mga dependensya
- **State Management**: Panatilihin ang konteksto at daloy ng datos sa bawat sunod-sunod na yugto ng workflow
- **Azure OpenAI (Responses API)**: Gamitin ang mga modelong Azure OpenAI sa multi-stage na mga workflow sa .NET
- **Enterprise Pipeline Patterns**: Bumuo ng mga sistema ng sunod-sunod na pagpoproseso na handa para sa produksyon

### 🏗️ **Advanced na Mga Pattern ng Sunod-sunod**
- **Stage-Gate Processing**: Ipatupad ang mga validation checkpoint sa pagitan ng mga yugto ng workflow
- **Context Preservation**: Panatilihin ang estado at naipong kaalaman sa lahat ng yugto
- **Error Propagation**: Maayos na humarap sa mga pagkakamali sa sunod-sunod na chains ng pagpoproseso
- **Performance Optimization**: Epektibong sunod-sunod na pagpapatupad na may minimal na overhead

### 🏢 **Enterprise Sequential Applications**
- **Document Processing Pipeline**: Multi-stage na pagsusuri, pagbabago, at pag-validate ng dokumento
- **Quality Assurance Workflows**: Sunod-sunod na pagrepaso, pag-validate, at proseso ng pag-apruba
- **Content Production Pipeline**: Research → Pagsusulat → Pag-edit → Pagrepaso → Paglalathala
- **Business Process Automation**: Multi-hakbang mga workflow ng negosyo na may malinaw na stage dependencies

- **Azure OpenAI Client**: Konsistenteng access sa AI model sa lahat ng yugto ng workflow

## 🏢 **Mga Benepisyo ng Enterprise Sequential**

### 🎯 **Katiyakan at Kalidad**
- **Deterministic Processing**: Konsistenteng, paulit-ulit na mga resulta sa pamamagitan ng mga istrukturadong yugto
- **Quality Gates**: Mga validation checkpoint na nagsisiguro ng kalidad sa bawat yugto
- **Error Isolation**: Hindi naipapasa ang problema sa isang yugto sa mga susunod na yugto
- **Audit Trails**: Kumpletong pagsubaybay ng mga desisyon at pagbabago sa bawat yugto

### 📈 **Scalability at Performance**
- **Modular Design**: Ang bawat yugto ay maaaring i-optimize nang hiwalay
- **Resource Management**: Mahusay na paglalaan ng mga AI model na resources sa bawat yugto
- **State Optimization**: Minimal na paglilipat ng estado sa pagitan ng mga yugto para sa optimal na performance
- **Parallel Stage Groups**: Maraming sunod-sunod na workflows ang pwedeng tumakbo nang sabay-sabay

### 🔒 **Seguridad at Pagsunod**
- **Stage-Level Security**: Iba't ibang polisiya sa seguridad para sa iba't ibang yugto ng pagpoproseso
- **Data Validation**: Siguraduhin ang integridad ng datos at pagsunod sa bawat checkpoint
- **Access Control**: Granular na mga permiso para sa iba't ibang yugto ng workflow
- **Regulatory Compliance**: Matugunan ang mga regulasyong pangangailangan sa pamamagitan ng istrukturadong pagpoproseso

### 📊 **Pagmomonitor at Analitika**
- **Stage-Level Metrics**: Pagmamanman ng performance para sa bawat yugto ng workflow
- **Bottleneck Identification**: Tukuyin at i-optimize ang mabagal na mga yugto
- **Quality Metrics**: Subaybayan ang kalidad at mga rate ng tagumpay sa bawat yugto
- **Process Optimization**: Patuloy na pagpapahusay batay sa analytics sa antas ng yugto

Tara, bumuo tayo ng matatag na mga sequential na AI processing pipelines! 🚀

## 💻 Pagpapatakbo ng Code

Ang kumpletong implementasyon ay makikita sa `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Ipinapakita ng file na ito ang **tatlong yugto ng workflow ng pagsusuri ng kasangkapan**:

1. **Stage 1 - Sales Agent**: Sinusuri ang mga larawan ng kasangkapan at nagbibigay ng mga suhestiyon sa pagbili
2. **Stage 2 - Price Agent**: Nagbibigay ng detalyadong breakdown ng presyo at mga opsyon sa budget
3. **Stage 3 - Quote Agent**: Gumagawa ng propesyonal na dokumento ng quote sa format na Markdown

### 🏗️ **Arkitektura ng Workflow**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Bawat ahente:
- Tumanggap ng output mula sa naunang yugto bilang konteksto
- Nagpatuloy sa naunang pagsusuri gamit ang espesyal na kaalaman
- Pinananatili ang tuloy-tuloy ng workflow sa pamamagitan ng pamamahala ng estado

### 📝 Inaasahang Output

Ang workflow ay:
1. **Sales Agent**: Tukuyin ang mga kasangkapan mula sa larawan at magbigay ng mga rekomendasyon
2. **Price Agent**: Magdagdag ng detalyadong pagsusuri sa presyo na may mga tier ng budget at mga rekomendasyon sa pamimili
3. **Quote Agent**: Gumawa ng naka-format na dokumento ng quote na may lahat ng impormasyong pinagsama-sama

### 🔍 Pag-unawa sa Daloy ng Estado

Bawat ahente sa pagkakasunod-sunod ay tumatanggap ng:
- **Orihinal na Input**: Ang unang mensahe ng gumagamit (larawan + teksto)
- **Mga Output ng Naunang Ahente**: Lahat ng naging tugon ng mga naunang ahente sa kasaysayan ng pag-uusap
- **Naipong Konteksto**: Kumpletong estado na pinananatili sa buong workflow

Ito ay nagpapahintulot sa sopistikadong multi-stage na pagpoproseso kung saan ang bawat ahente ay bumubuo sa komprehensibong konteksto mula sa lahat ng naunang yugto.
