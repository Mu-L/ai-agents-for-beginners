# ⚡ Magkasabay na Mga Daloy ng Trabaho ng Ahente gamit ang Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial sa Mataas na Pagganap na Parallel Processing

Ipinapakita ng notebook na ito ang **magkasabay na mga pattern ng daloy ng trabaho** gamit ang Microsoft Agent Framework para sa .NET at Azure OpenAI (Responses API). Matututuhan mo kung paano bumuo ng mataas na pagganap, parallel na pagpoproseso ng mga daloy ng trabaho na nagpapalaki ng throughput sa pamamagitan ng pagsasagawa ng maramihang AI agents nang sabay-sabay habang pinapanatili ang koordinasyon at pagkakapare-pareho ng datos.

## 💻 Pagpapatakbo ng Code

Ang kumpletong implementasyon ay available sa `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Ipinapakita ng file na ito ang **Fan-Out/Fan-In concurrent workflow** para sa pagpaplano ng paglalakbay:

### 🏗️ **Workflow Architecture**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Pangunahing Mga Sangkap:**

1. **ConcurrentStartExecutor**: Ipinapadala ang kahilingan ng user sa lahat ng ahente nang sabay
2. **Researcher Agent**: Sinasaliksik ang mga destinasyon at atraksyon nang sabay
3. **Planner Agent**: Gumagawa ng detalyadong plano ng paglalakbay nang sabay
4. **ConcurrentAggregationExecutor**: Kinokolekta at pinagsasama ang mga resulta mula sa parehong mga ahente

### 🎯 **Fan-Out/Fan-In Pattern**

Ipinapakita ng workflow na ito ang klasikong **Fan-Out/Fan-In** pattern:
- **Fan-Out**: Isang input na mensahe ang ipinapadala sa maramihang mga ahente nang sabay
- **Concurrent Processing**: Maraming ahente ang sabayang nagtatrabaho sa parehong gawain
- **Fan-In**: Kinokolekta at pinagsasama ang mga resulta mula sa lahat ng ahente sa isang output

### 🔍 Pag-unawa sa Mga Custom Executor

**ConcurrentStartExecutor:**
- Nagpapatupad ng `IMessageHandler<string>` upang tumanggap ng string input
- Nagpapadala ng mga mensahe sa lahat ng nakakonektang mga ahente
- Nagpapadala ng `TurnToken` upang pasimulan ang sabayang pagpoproseso

**ConcurrentAggregationExecutor:**
- Nagpapatupad ng `IMessageHandler<ChatMessage>` upang tumanggap ng mga tugon mula sa ahente
- Kinokolekta ang mga mensahe sa ligtas na paraan mula sa mga thread
- Pinagsasama kapag dumating na ang lahat ng inaasahang tugon
- Nagbibigay ng huling output gamit ang `context.YieldOutputAsync()`

### ⚡ Mga Benepisyo ng Pagganap

**Concurrent kumpara sa Sequential:**
- Sequential: Agent1 (30s) → Agent2 (30s) = **60 segundo kabuuan**
- Concurrent: Agent1 (30s) || Agent2 (30s) = **30 segundo kabuuan**

**Pagsulong sa Throughput**: Hanggang N× na mas mabilis para sa N sabayang mga ahente (depende sa workload at mga resources)

### 🛡️ Paghawak ng Error

Maayos na hinaharap ng workflow ang mga pagkabigo ng indibidwal na ahente:
- Kapag may isang ahente na nabigo, patuloy ang pagproseso ng iba
- Maaaring ipatupad ng aggregator ang timeout logic
- Puwedeng ibalik ang mga bahagyang resulta kung kinakailangan
