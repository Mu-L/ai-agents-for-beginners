## 💻 Pagpapatakbo ng Code

Ang kompletong implementasyon ay matatagpuan sa `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Ipinapakita ng file na ito ang **Fan-Out/Fan-In concurrent workflow** para sa pagpaplano ng paglalakbay:

### 🏗️ **Arkitektura ng Workflow**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Mga Pangunahing Sangkap:**

1. **ConcurrentStartExecutor**: Ipinapadala ang kahilingan ng user sa lahat ng agent nang sabay-sabay
2. **Researcher Agent**: Sabay-sabay na sinusuri ang mga destinasyon at atraksyon
3. **Planner Agent**: Sabay-sabay na gumagawa ng detalyadong plano sa paglalakbay
4. **ConcurrentAggregationExecutor**: Kinokolekta at pinagsasama ang mga resulta mula sa parehong agent

### 🎯 **Fan-Out/Fan-In Pattern**

Ipinapakita ng workflow na ito ang klasikong **Fan-Out/Fan-In** pattern:
- **Fan-Out**: Isang input message ang ipinapadala sa maraming agent nang sabay-sabay
- **Concurrent Processing**: Maraming agent ang nagtatrabaho nang sabay-sabay sa parehong gawain
- **Fan-In**: Kinokolekta at pinagsasama ang mga resulta mula sa lahat ng agent sa isang output

Ang workflow ay:

### 🔍 Pag-unawa sa Mga Custom Executor

**ConcurrentStartExecutor:**
- Nagpapatupad ng `IMessageHandler<string>` para tumanggap ng string input
- Nagpapadala ng mga mensahe sa lahat ng nakakonektang agent
- Nagpapadala ng `TurnToken` upang simulan ang concurrent processing

**ConcurrentAggregationExecutor:**
- Nagpapatupad ng `IMessageHandler<ChatMessage>` para tumanggap ng mga sagot ng agent
- Kinokolekta ang mga mensahe sa isang thread-safe na paraan
- Nagsasama kapag dumating na lahat ng inaasahang sagot
- Nagbabalik ng panghuling output gamit ang `context.YieldOutputAsync()`

### ⚡ Mga Benepisyo sa Performance

**Concurrent kumpara sa Sequential:**
- Sequential: Agent1 (30s) → Agent2 (30s) = **60 segundo kabuuan**
- Concurrent: Agent1 (30s) || Agent2 (30s) = **30 segundo kabuuan**

**Pagpapabuti ng Throughput**: Hanggang N× na mas mabilis para sa N na concurrent na agent (depende sa workload at mga resources)

### 🛡️ Pag-handle ng Error

Ang workflow ay maayos na humaharap sa pagkabigo ng indibidwal na agent:
- Kung may isang agent na pumalya, nagpapatuloy ang ibang mga agent sa pagproseso
- Maaaring magpatupad ng timeout logic ang aggregator
- Puwedeng ibalik ang partial na mga resulta kung kinakailangan
