# 🔄 Mga Pangunahing Daloy ng Trabaho ng Ahente gamit ang Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial sa Orkestrasyon ng Daloy ng Trabaho

Ipinapakita ng notebook na ito kung paano bumuo ng sopistikadong **daloy ng trabaho ng ahente** gamit ang Microsoft Agent Framework para sa .NET at Azure OpenAI (Responses API). Matututuhan mong lumikha ng mga multi-step na proseso ng negosyo kung saan nagsasama-sama ang mga AI agent upang magtagumpay sa mga komplikadong gawain sa pamamagitan ng estrukturadong mga pattern ng orkestrasyon.

## 🎯 Mga Layunin sa Pagkatuto

### 🏗️ **Mga Pangunahing Kaalaman sa Arkitektura ng Daloy ng Trabaho**
- **Tagabuo ng Daloy ng Trabaho**: Magdisenyo at mag-orchestrate ng mga komplikadong multi-step na proseso ng AI
- **Koordinasyon ng Ahente**: Ikoordina ang maraming espesyalisadong ahente sa loob ng mga daloy ng trabaho
- **Azure OpenAI (Responses API)**: Gamitin ang Azure OpenAI Responses API sa mga daloy ng trabaho
- **Biswal na Disenyo ng Daloy ng Trabaho**: Lumikha at biswal na ipakita ang mga estruktura ng daloy ng trabaho para sa mas malinaw na pag-unawa

### 🔄 **Mga Pattern ng Orkestrasyon ng Proseso**
- **Sunud-sunod na Pagpoproseso**: Ikadena ang maraming gawain ng ahente sa lohikal na pagkakasunud-sunod
- **Pamamahala ng Estado**: Panatilihin ang konteksto at daloy ng datos sa buong mga yugto ng daloy ng trabaho
- **Paghawak ng Error**: Magpatupad ng matibay na pag-recover mula sa error at katatagan ng daloy ng trabaho
- **Pag-optimize ng Performance**: Magdisenyo ng mga epektibong daloy ng trabaho para sa mga operasyon sa antas ng enterprise

### 🏢 **Mga Aplikasyon ng Daloy ng Trabaho sa Enterprise**
- **Automation ng Proseso ng Negosyo**: I-automate ang mga komplikadong daloy ng trabaho ng organisasyon
- **Content Production Pipeline**: Mga daloy ng trabaho ng editoryal na may mga yugto ng pagsusuri at pag-apruba
- **Automation ng Serbisyo sa Customer**: Multi-step na resolusyon ng mga query ng customer
- **Data Processing Workflows**: Mga daloy ng ETL na pinalalakas ng AI para sa transformasyon

## ⚙️ Mga Kinakailangan at Setup

### 📦 **Mga Kinakailangang NuGet Package**

Ginagamit ang demonstrasyon ng daloy ng trabaho na ito ng ilang mahahalagang .NET package:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Kumpigurasyon ng Azure OpenAI**

**Pag-setup ng Kapaligiran (.env file):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Pag-access sa Azure OpenAI:**
1. Gumawa ng Azure OpenAI resource sa Azure portal
2. I-deploy ang isang modelo (halimbawa, `gpt-4.1-mini`) at tandaan ang pangalan ng deployment
3. Mag-sign in gamit ang `az login` at i-configure ang mga environment variable tulad ng ipinakita sa itaas

**Pangunahing Mga Bahagi:**

- **Azure OpenAI Client**: Integrasyon ng Azure OpenAI Responses API
- **Execution Context**: Namamahala sa estado at daloy ng datos sa pagitan ng mga yugto ng daloy ng trabaho

## 💻 Pagpapatakbo ng Kodigo

Ang kumpletong implementasyon ay makikita sa `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Ipinapakita ng file na ito:

1. **Pag-configure ng Kapaligiran** - Pag-load ng Azure OpenAI configuration mula sa `.env` file
2. **Pag-setup ng Azure OpenAI Client** - Pag-configure ng client upang gamitin ang Azure OpenAI Responses API
3. **Paglikha ng Ahente** - Pagdeklara ng mga espesyalisadong ahente (Front Desk at Concierge)
4. **Workflow Builder** - Paglikha ng multi-agent workflow na may sunud-sunod na pagpoproseso
5. **Pagpapatupad ng Daloy ng Trabaho** - Pagpapatakbo ng workflow na may mga streaming na resulta

### 🚀 Pagpapatakbo ng Halimbawa

```bash
# Gawing executable ang script (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Patakbuhin ang workflow
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

O sa Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Inaasahang Output

Ang daloy ng trabaho ay:
1. Tatanggap ng iyong kahilingan sa destinasyon ng paglalakbay ("Gusto kong pumunta sa Paris")
2. Magbibigay ang Front Desk agent ng unang rekomendasyon
3. Susuriin at papagandahin ng Concierge agent ang rekomendasyon
4. Ipapakita ang panghuling output ng kumpletong usapan
