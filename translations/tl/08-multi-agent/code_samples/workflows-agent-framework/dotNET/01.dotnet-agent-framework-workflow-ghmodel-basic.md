# 🔄 Mga Pangunahing Daloy ng Trabaho ng Ahente gamit ang Azure OpenAI (Responses API) (.NET)

## 📋 Tutorial sa Pag-oorganisa ng Daloy ng Trabaho

Ipinapakita ng notebook na ito kung paano bumuo ng sopistikadong **daloy ng trabaho ng ahente** gamit ang Microsoft Agent Framework para sa .NET at Azure OpenAI (Responses API). Matututuhan mong gumawa ng mga multi-hakbang na proseso ng negosyo kung saan ang mga AI agent ay nagtutulungan upang matapos ang mga kumplikadong gawain sa pamamagitan ng istrakturang mga pattern ng orkestrasiyon.

## 🎯 Mga Layunin sa Pagkatuto

### 🏗️ **Mga Pangunahing Kaalaman sa Arkitektura ng Daloy ng Trabaho**
- **Tagabuo ng Daloy ng Trabaho**: Magdisenyo at mag-organisa ng komplikadong multi-hakbang na mga proseso ng AI
- **Koordinasyon ng Ahente**: I-coordinate ang maraming espesyal na ahente sa loob ng mga daloy ng trabaho
- **Azure OpenAI (Responses API)**: Gamitin ang Azure OpenAI Responses API sa mga daloy ng trabaho
- **Biswal na Disenyo ng Daloy ng Trabaho**: Gumawa at biswal na ipakita ang mga estruktura ng daloy ng trabaho para sa mas maayos na pag-unawa

### 🔄 **Mga Pattern ng Orkestrasyon ng Proseso**
- **Sunud-sunod na Pagproseso**: I-chain ang maraming gawain ng ahente sa lohikal na pagkakasunod
- **Pamamahala ng Estado**: Panatilihin ang konteksto at daloy ng data sa bawat yugto ng daloy ng trabaho
- **Pag-handle ng Error**: Magpatupad ng matibay na pagkukumpuni ng error at katatagan ng daloy ng trabaho
- **Pag-optimize ng Performance**: Magdisenyo ng episyenteng daloy ng trabaho para sa operasyon sa antas ng enterprise

### 🏢 **Mga Application ng Enterprise Workflow**
- **Pag-automate ng Proseso ng Negosyo**: I-automate ang mga kumplikadong daloy ng trabaho ng organisasyon
- **Pipeline ng Produksyon ng Nilalaman**: Mga editorial workflow na may mga yugto ng pagsusuri at pag-apruba
- **Pag-automate ng Serbisyo sa Customer**: Multi-hakbang na resolusyon ng mga katanungan ng customer
- **Mga Daloy ng Trabaho sa Pagproseso ng Data**: ETL workflows na gamit ang AI-powered na transformasyon

## ⚙️ Mga Kinakailangan at Pagsisimula

### 📦 **Kinakailangang NuGet Packages**

Ang demonstrasyong ito ng daloy ng trabaho ay gumagamit ng ilang pangunahing .NET packages:

```xml
<!-- Core AI Framework -->

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Konfigurasyon ng Azure OpenAI**

**Pagsasaayos ng Kapaligiran (.env file):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Access sa Azure OpenAI:**
1. Gumawa ng isang Azure OpenAI resource sa Azure portal
2. I-deploy ang isang modelo (halimbawa, `gpt-5-mini`) at tandaan ang pangalan ng deployment
3. Mag-sign in gamit ang `az login` at ipasok ang mga environment variable gaya ng ipinakita sa itaas

### 🏗️ **Pangkalahatang-ideya ng Arkitektura ng Daloy ng Trabaho**

```mermaid
graph TD
    A[Tagabuo ng Daloy ng Trabaho] --> B[Rehistro ng Ahente]
    B --> C[Engine ng Pagsasagawa ng Daloy ng Trabaho]

    C --> E[Agent 2: Tagasuri ng Nilalaman]
```

**Pangunahing Mga Bahagi:**
- **WorkflowBuilder**: Pangunahing makina ng orkestrasiyon para sa pagdisenyo ng mga daloy ng trabaho
- **AIAgent**: Mga indibidwal na espesyal na ahente na may partikular na kakayahan
- **Azure OpenAI Client**: Integrasyon sa Azure OpenAI Responses API
- **Execution Context**: Namamahala sa estado at daloy ng data sa pagitan ng mga yugto ng daloy ng trabaho

## 💻 Pagpapatakbo ng Code

Ang kumpletong implementasyon ay makikita sa `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Ipinapakita ng file na ito:

1. **Konfigurasyon ng Kapaligiran** - Pag-load ng Azure OpenAI na konfigurasyon mula sa `.env` file
2. **Setup ng Azure OpenAI Client** - Pagconfigure ng client para gamitin ang Azure OpenAI Responses API
3. **Paglikha ng Ahente** - Pagde-define ng mga espesyal na ahente (Front Desk at Concierge)
4. **Tagabuo ng Daloy ng Trabaho** - Paggawa ng multi-agent workflow na may sunud-sunod na pagproseso
5. **Pagpapatupad ng Daloy ng Trabaho** - Pagpapatakbo ng workflow na may streaming na mga resulta

### 🚀 Pagpapatakbo ng Halimbawa

```bash
# Gawing executable ang script (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Patakbuhin ang workflow
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

O sa Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Inaasahang Output

Ang daloy ng trabaho ay:
1. Tatanggap ng iyong request sa destinasyon ng biyahe ("Gusto kong pumunta sa Paris")
2. Magbibigay ang Front Desk agent ng unang rekomendasyon
3. Susuriin at pagaganda ang rekomendasyon ng Concierge agent
4. Ipapakita ang huling output ng buong daloy ng pag-uusap

- Pagbabago ng mensahe ng user para subukan ang iba't ibang mga senaryo
- Pagsasaayos ng mga gilid ng workflow upang makagawa ng iba't ibang mga pattern ng pagpapatupad
