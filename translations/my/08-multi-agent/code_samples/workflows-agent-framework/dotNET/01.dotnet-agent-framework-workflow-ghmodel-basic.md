# 🔄 Azure OpenAI (Responses API) နှင့်အတူ အခြေခံ အေးဂျင့် လုပ်ငန်းစဉ်များ (.NET)

## 📋 လုပ်ငန်းစဉ် စီမံခန့်ခွဲခြင်း သင်ခန်းစာ

ဒီ notebook က Microsoft Agent Framework for .NET နဲ့ Azure OpenAI (Responses API) ကို အသုံးပြုကာ တွန်းလှန်တဲ့ **အေးဂျင့် လုပ်ငန်းစဉ်များ** တည်ဆောက်နည်းကို ဖော်ပြပေးပါတယ်။ AI အေးဂျင့်များက မျိုးစုံသောအဆင့်ဆင့် စီးပွားရေးလုပ်ငန်းလုပ်ဆောင်မှုတွေကို အဖွဲ့ညီညွတ်ပြီး ကုသိန်းအောင် တည်ဆောက်နိုင်တယ်ဆိုတာ သင်ယူနိုင်မှာ ဖြစ်ပါတယ်။

- **ဖောက်သည် ဝန်ဆောင်မှု အလိုအလျောက်ပြုလုပ်မှု**: ဖောက်သည် မေးခွန်းများ ဖြေရှင်းရန် အဆင့်ဆင့် လုပ်ငန်းစဉ်များ

### 🔑 **Azure OpenAI ဖွဲ့စည်းခြင်း**

**ပတ်ဝန်းကျင် သတ်မှတ်ခြင်း (.env ဖိုင်):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Azure OpenAI ခွင့် ပြုချက်:**
1. Azure ပေါ်တွင် Azure OpenAI အရင်းအမြစ်တစ်ခု ဖန်တီးပါ
2. မော်ဒယ်ကို တပ်ဆင်ပါ (ဥပမာ `gpt-4.1-mini`) နှင့် တပ်ဆင်သော နာမည်ကို မှတ်သားပါ
3. `az login` ဖြင့် လော့ဂ်အင်ဝင်ပြီး အထက်ပါ နေရာတွင်ပြထားသည့် ပတ်ဝန်းကျင် ကြောင့်ဆက်တင်များကို သတ်မှတ်ပါ

### 🏗️ **လုပ်ငန်းစဉ် ဖွဲ့စည်းပုံ အနှစ်ချုပ်**

```mermaid
graph TD
    A[အလုပ်စဉ် တည်ဆောက်သူ] --> B[အေးဂျင့် မှတ်ပုံတင်ဗေဒ]
    B --> C[အလုပ်စဉ် အကောင်အထည်ဖော်မှု အင်ဂျင်]
    C --> D[Agent 1: အကြောင်းအရာ ဖန်တီးသူ]
    C --> E[Agent 2: အကြောင်းအရာ ပြန်လည် သုံးသပ်သူ] 
    D --> F[အလုပ်စဉ် ရလဒ်များ]
    E --> F
    G[Azure OpenAI (ဖြေကြားမှု API)] --> D
    G --> E
```

**အဓိကအစိတ်အပိုင်းများ:**
- **WorkflowBuilder**: လုပ်ငန်းစဉ်များဒီဇိုင်းနှင့် စီမံခန့်ခွဲရန် အဓိက အင်ဂျင်
- **AIAgent**: မှန်ကန်သော စွမ်းရည်များပါရှိသော တစ်ဦးချင်း အေးဂျင့်များ
- **Azure OpenAI Client**: Azure OpenAI Responses API ပေါင်းစပ်မှု
- **Execution Context**: လုပ်ငန်းစဉ် အဆင့်များအကြား အခြေအနေနှင့် ဒေတာ စီးဆင်းမှုကို စီမံခန့်ခွဲခြင်း

## 💻 ကုဒ်ကို ပြေးဆွဲခြင်း

ပြည့်စုံသော အကောင်အထည်ဖော်မှုကို `01.dotnet-agent-framework-workflow-ghmodel-basic.cs` တွင် ဆက်လက် ရယူနိုင်ပါတယ်။ ဒီဖိုင်တွင် ဖော်ပြထားသည်မှာ-

1. **ပတ်ဝန်းကျင် ဖွဲ့စည်းခြင်း** - `.env` ဖိုင်မှ Azure OpenAI ဖွဲ့စည်းမှုများကို ထည့်သွင်းခြင်း
2. **Azure OpenAI Client ပြင်ဆင်ခြင်း** - Azure OpenAI Responses API ကို အသုံးပြုရန် Client ကို ပြင်ဆင်ခြင်း
3. **အေးဂျင့် ဖန်တီးခြင်း** - အထူးပြု အေးဂျင့်များ (Front Desk နှင့် Concierge)
4. **Workflow Builder** - အဆင့်လိုက် စီစဉ်သည့် အေးဂျင့်များစွာပါ ဝတ်တော် များ ဖန်တီးခြင်း
5. **လုပ်ငန်းစဉ် လည်ပတ်ခြင်း** - မှတ်တမ်းများနှင့် အတူ လုပ်ငန်းစဉ် လည်ပတ်ခြင်း

2. Front Desk အေးဂျင့် က ဦးစွာ အကြံပြုချက်တစ်ရပ် ပေးပါလိမ့်မယ်
3. Concierge အေးဂျင့် သည် အကြံပြုချက်ကို ပြန်လည် သုံးသပ်ပြီး တိုးတက်မြှင့်တင်ပါလိမ့်မယ်
4. နောက်ဆုံး ထွက်ပေါက်တွင် စကားဝိုင်း စီးရီးလုံး ဆက်လက်ပြသပါမည်

### 🔧 မိမိလိုအပ်သလို ပြင်ဆင်ခြင်း

သင်လိုအပ်သလို workflow ကို ပြင်ဆင်နိုင်ပါသည်-
- အေးဂျင့်ရဲ့ မှတ်ချက်များကို ပြောင်းလဲဖို့ ညွှန်ကြားချက်များ ပြောင်းလိုက်ပါ
- ပိုပြီး အေးဂျင့်များ ထည့်သွင်းပြီး စပ်ဆက်ပြီး အဆင့်များစွာပါဝင်တဲ့ workflows ဖန်တီးပါ
- အသုံးပြုသူ စာတိုက်ข้อความ တေံ့တိုးခြင်းအတွက် အခြေအနေများကို စမ်းသပ်ရန် ပြောင်းလဲပါ
- လုပ်ငန်းစဉ် လမ်းကြောင်းများကို တည်းဖြတ်ပြီး အမျိုးမျိုးသော လုပ်ဆောင်မှု ပုံစံတွေ ဖန်တီးပါ
