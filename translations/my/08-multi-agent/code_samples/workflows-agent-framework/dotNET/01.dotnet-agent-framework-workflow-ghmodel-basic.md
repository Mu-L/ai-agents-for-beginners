# 🔄 Azure OpenAI (Responses API) နဲ့ အခြေခံအေးဂျင့် လုပ်ငန်းစဉ်များ (.NET)

## 📋 Workflow Orchestration လမ်းညွှန်ချက်

ဒီ notebook က Microsoft Agent Framework for .NET နဲ့ Azure OpenAI (Responses API) ကို အသုံးပြုပြီး ရှုပ်ထွေးတဲ့ **အေးဂျင့်လုပ်ငန်းစဉ်များ** ကို ဘယ်လိုဖန်တီးရမလဲကို ပြသပေးပါတယ်။ AI အေးဂျင့်တွေက စနစ်တကျ စီမံခန့်ခွဲမှု ပုံစံတွေနဲ့ အတူ လုပ်ဆောင်ပြီး ရှုပ်ထွေးတဲ့ လုပ်ငန်းလုပ်ငန်းစဉ်များကို ဆောင်ရွက်နိုင်တဲ့ မျိုးစုံ-အဆင့်လုပ်ငန်းစဉ်တွေ ဖန်တီးတယ်ဆိုတာ သင်ယူရပါမယ်။

## 🎯 သင်ယူရမဲ့ ရည်ရွယ်ချက်များ

### 🏗️ **Workflow ဆောက်လုပ်မှု အခြေခံများ**
- **Workflow Builder**: ရှုပ်ထွေးတဲ့ မျိုးစုံ-အဆင့် AI လုပ်ငန်းစဉ်များကို ဒီဇိုင်းဆွဲခြင်းနှင့် စီမံခန့်ခွဲခြင်း
- **Agent Coordination**: လုပ်ငန်းစဉ်များထဲမှာ အထူးပြု အေးဂျင့်အများအပြားကို ကိုက်ညီစွာ စီမံခန့်ခွဲခြင်း
- **Azure OpenAI (Responses API)**: Workflow များထဲမှာ Azure OpenAI Responses API ကို အသုံးပြုခြင်း
- **Visual Workflow Design**: နားလည်မှုအရည်အသွေးမြင့်အောင် workflow ဖွဲ့စည်းမှုများကို ဖန်တီးပြီး မြင်သာအောင် ပြသခြင်း

### 🔄 **လုပ်ငန်းစဉ် စီမံခန့်ခွဲမှု ပုံစံများ**
- **Sequential Processing**: အေးဂျင့်လုပ်ငန်းတာဝန်များကို မှန်ကန်စွာ တွဲဖက် ချိတ်ဆက်ပေးခြင်း
- **State Management**: Workflow အဆင့်များဆိုင်ရာ context နှင့် ဒေတာလှဲပြောင်းမှုကို ထိန်းသိမ်းခြင်း
- **Error Handling**: စစ်မှုအသစ်တခု တိုးတက်တည်ငြိမ်စွာ ပြန်လည်အသုံးပြုနိုင်မှုရှိသော ချို့တဲ့ချက်င် ရွေ့ပြောင်းမှု
- **Performance Optimization**: လုပ်ငန်းစဉ် အကျိုးသက်သာဆုံးဖြစ်အောင် ဒီဇိုင်းဆွဲခြင်း

### 🏢 **အဖွဲ့အစည်းလုပ်ငန်းစဉ် များ**
- **Business Process Automation**: ရှုပ်ထွေးတဲ့အဖွဲ့အစည်းလုပ်ငန်းစဉ်များကို အလိုအလျောက်လုပ်ဆောင်ခြင်း
- **Content Production Pipeline**: ပြန်လည်သုံးသပ်ခြင်းနှင့် အတည်ပြုခြင်းအဆင့်များပါသည့် ဆောင်းပါးလုပ်ငန်းစဉ်များ
- **Customer Service Automation**: မျိုးစုံ-အဆင့် ဖောက်သည် မေးခွန်းဖြေရှင်းမှု
- **Data Processing Workflows**: AI ဖြင့် ပြောင်းလဲမှုပါဝင်သော ETL Workflow များ

### 🔑 **Azure OpenAI ပတ်ဝန်းကျင် ပြင်ဆင်မှု**

**ပတ်ဝန်းကျင် ပြင်ဆင်မှု (.env ဖိုင်):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Azure OpenAI ဝင်ရောက်ခွင့်:**
1. Azure portal တွင် Azure OpenAI resource ဖန်တီးပါ
2. Model တစ်ခု (ဥပမာ `gpt-5-mini`) ကို တပ်ဆင်ပြီး deployment နာမည် မှတ်သားပါ
3. `az login` ဖြင့် ဝင်ရောက်ပြီး အပေါ်ပါ ပတ်ဝန်းကျင် အနေအထားများကို ပြင်ဆင်ပါ

### 🏗️ **Workflow Architecture အနှိပ်**

```mermaid
graph TD
    A[လုပ်ငန်းစဉ် တည်ဆောက်သူ] --> B[ကိုယ်စားလှယ် မှတ်ပုံတင်]
    B --> C[လုပ်ငန်းစဉ် စီမံခန့်ခွဲရေး အင်ဂျင်]
    C --> D[Agent 1: အကြောင်းအရာ ဖန်တီးသူ]
    C --> E[Agent 2: အကြောင်းအရာ ပြန်လည်သုံးသပ်သူ] 
    D --> F[လုပ်ငန်းစဉ် ရလဒ်များ]
    E --> F
    G[Azure OpenAI (ဖြေကြားချက်များ API)] --> D
    G --> E
```

**အဓိက အစိတ်အပိုင်းများ:**
- **WorkflowBuilder**: Workflow များ ဒီဇိုင်းရေးဆွဲရေး လုပ်ငန်းစဉ် အင်ဂျင်မင်း
- **AIAgent**: အထူးပြု အေးဂျင့်များ၊ တစ်ဦးချင်းစီ၏ တည်ဆောက်မှုများ
- **Azure OpenAI Client**: Azure OpenAI Responses API ပေါင်းစည်းမှု
- **Execution Context**: Workflow အဆင့်များအကြား အခြေအနေနှင့် ဒေတာလှဲပြောင်းမှုကို စီမံခန့်ခွဲခြင်း

## 🎨 **အဖွဲ့အစည်း Workflow ဒီဇိုင်း ပုံစံများ**


### 🤝 **ဖောက်သည် ဝန်ဆောင်မှု အလိုအလျောက်လုပ်ငန်းစဉ်**
```
Customer Inquiry → Classification → Processing → Response Generation → Follow-up
```

## 🏢 **အဖွဲ့အစည်း အကျိုးအမြတ်များ**

### 🎯 **ယုံကြည်စိတ်ချမှုနှင့် ကြီးမားမှု**
- **သေချာကျွန်ုပ်မှု**: workflow အပြီးအပွင့် တည်ငြိမ် တပြန်ပြန်ဖြစ်မှု
- **အမှား ပြန်လည်ကော်ရန်**: workflow အဆင့်တွင် ချို့တဲ့ချက်များကို ဂရုစိုက် ကောင်းစွာ ကိုင်တွယ်ခြင်း
- **Performance ပတ်ကြားခြင်း**: အကောင်အထည်ဖော်မှုပမာဏနှင့် အကောင်းဆုံးရေး ဆန်းစစ်ခြင်း
- **အရင်းအမြစ် စီမံခန့်ခွဲမှု**: AI model အရင်းအမြစ်များကို ထိရောက်စွာ အသုံးပြုခြင်း

### 🔒 **လုံခြုံမှုနှင့် လိုက်နာမှု**
- **လုံခြုံအတည်ပြုရေး**: Microsoft Entra ID authentication ကို `az login` (AzureCliCredential) ဖြင့် အသုံးပြုခြင်း
- **Audit Trails**: Workflow လုပ်ဆောင်မှုနှင့် ဆုံးဖြတ်ချက်များအား စုစည်းမှတ်တမ်းပြုခြင်း
- **Access Control**: Workflow လုပ်ဆောင်မှုနှင့် ကြည့်ရှုမှုတွင် အသေးစိတ်ခွင့်ပြုချက်များ ထုတ်ပေးခြင်း
- **Data Privacy**: workflow များသည့်Sensitive Data ကို လုံခြုံစိတ်ချစွာ ကိုင်တွယ်ခြင်း

### 📊 **မြင်တက်ငံ့မှုနှင့် စီမံခန့်ခွဲမှု**
- **Visual Workflow Design**: လုပ်ငန်းစဉ်စီးဆင်းမှုများနှင့် မူတည်မှုများကို ရှင်းလင်းတောက်ပြောင်စွာ ဖော်ပြခြင်း
- **Execution Monitoring**: Workflow ထွက်ရှိမှုအဆင့်ကို real-time မှာ လိုက်လံကြည့်ရှုခြင်း
- **အမှား အစီရင်ခံ**: အသေးစိတ် အမှား စိစစ်မှုနှင့် debugging အစွမ်းသတ်များ
- **Performance Analytics**: အကောင်းဆုံး ရေးဆွဲမှုနှင့် စွမ်းဆောင်ရည် စီမံခန့်ခွဲမှုအတွက် မီထရစ်များ

မင်းရဲ့ ပထမဆုံး အဖွဲ့အစည်းအသုံးပြုနိုင်တဲ့ AI workflow ကို တည်ဆောက်လိုက်ကြစို့! 🚀

## 💻 ကုဒ် ကို အလုပ်လုပ်စေခြင်း

`01.dotnet-agent-framework-workflow-ghmodel-basic.cs` ဖိုင်ထဲမှာ ပြီးပြည့်စုံတဲ့ အကောင်အထည်ဖော်မှုရှိပြီး၊ ဒီဖိုင်က ပြသတာများမှာ -

1. **ပတ်ဝန်းကျင် ပြင်ဆင်ခြင်း** - `.env` ဖိုင်ထဲမှ Azure OpenAI configuration ကို load ပြုလုပ်ခြင်း
2. **Azure OpenAI Client Setup** - Azure OpenAI Responses API ကို အသုံးပြုဖို့ client ကို စနစ်တကျ ပြင်ဆင်ခြင်း
3. **အေးဂျင့် ဖန်တီးခြင်း** - အထူးပြု အေးဂျင့်များ (Front Desk နှင့် Concierge) ကို သတ်မှတ်ခြင်း
4. **Workflow Builder** - မျိုးစုံအေးဂျင့် များပါဝင်တဲ့ မျိုးစုံ-အဆင့် workflow ဖန်တီးခြင်း
5. **Workflow ဖြင့် လုပ်ဆောင်ခြင်း** - Streaming ရလဒ်တွေနဲ့ workflow ကို အလုပ်လုပ်စေခြင်း

### 📝 ရရှိမယ့် ပြင်ဆင်မှု အချို့

Workflow က -
1. မင်းရဲ့ ခရီးစဉ် တည်နေရာ တောင်းဆိုမှုကို လက်ခံပါလိမ့်မယ် ("I would like to go to Paris")
2. Front Desk အေးဂျင့်က အစပိုင်း အကြံပြုချက် ထုတ်ပေးပါမယ်
3. Concierge အေးဂျင့်က အကြံပြုချက်ကို ပြန်လည် ဆန်းစစ်ပြီး တိုးတက်ကောင်းမွန်အောင် ပြုပြင်ပေးပါမယ်
4. အဆုံးမှာ သွားလာမှု စကားပြော များအားလုံးကို ပြသပါမယ်

- မတူညီတဲ့ လုပ်ဆောင်မှုပုံစံများ ဖန်တီးရန် workflow တွင်လမ်းကြောင်းများ ပြောင်းလဲခြင်း
