# ⏩ Azure OpenAI (Responses API) (.NET) နှင့် အဆက်တွဲ အီးဂျင့်လုပ်ငန်းစဉ်များ

## 📋 အဆင့်မြင့် အဆက်လိုက် လုပ်ဆောင်ခြင်း သင်တန်း

ဒီ notebook က Microsoft Agent Framework for .NET နဲ့ Azure OpenAI (Responses API) ကိုအသုံးပြုပြီး **အဆက်လိုက် workflow ပုံစံများ** ကို ပြသပါတယ်။ အီးဂျင့်တွေက အစဉ်လိုက်အဆင့်စီ အကောင်အထည်ဖော်ပြီး နောက်တစ်ခုရဲ့ရလဒ်အပေါ်မှာ အတည်ပြုပြီး ပိုမိုခိုင်မာတဲ့ အဆင့်တိုင်းအဆင့်တိုင်း ပြုလုပ်မှုစနစ်တွေ ဖန်တီးနည်းနဲ့ အဆင့်ဆင့်လုပ်ဆောင်ပုံတက်ကြမယ်။

## 🎯 သင်ယူရမည့် ရည်မှန်းချက်များ

### 🔄 **အဆက်လိုက် လုပ်ဆောင်မှု ဒီဇိုင်းနှင့် ဖွဲ့စည်းပုံ**
- **လိုင်းနိုက်နေသော workflow ဒီဇိုင်း**: အဆင့်လိုက် ပြုလုပ်ခြင်းကို ရိုးရှင်းပြတ်သားစွာ ဖန်တီးရန်
- **အခြေအနေ စီမံခန့်ခွဲမှု**: အဆက်လိုက် လုပ်ဆောင်မှု အဆင့်များမှာ context နှင့် data ပိုလှောင်မှု ထိန်းသိမ်းရန်
- **Azure OpenAI (Responses API)**: Azure OpenAI မော်ဒယ်များကို .NET တွင် multi-stage workflow များတွင် အသုံးပြုခြင်း
- **အဖွဲ့အစည်း pipeline ပုံစံများ**: ထုတ်လုပ်မှုအဆင့် လိုက်လျားတဲ့ အဆက်လိုက်စနစ်များ ဖန်တီးခြင်း

### 🏗️ **အဆင့်မြင့် အဆက်လိုက် ပုံစံများ**
- **Stage-Gate ပြုလုပ်ခြင်း**: workflow အဆင့်များ အကြား အတည်ပြုချက် checkpoint များ ထည့်သွင်းခြင်း
- **Context ထိန်းသိမ်းမှု**: အဆင့်တိုင်းမှာ အခြေအနေနှင့် အသိပညာ စုဆောင်းထားမှု ထိန်းသိမ်းခြင်း
- **အမှား ကူးပြောင်းမှု**: အဆက်လိုက်လုပ်ဆောင်မှုတွင် အမှားများကို သက်သေအောင် ကိုင်တွယ်ခြင်း
- **စွမ်းဆောင်ရည် တိုးတက်မှု**: အလွန်အသေးစား overhead နှင့် အဆက်လိုက် စီမံရန် အကျိုးရှိသော ပြုလုပ်ခြင်း

### 🏢 **အဖွဲ့အစည်းအဆက်လိုက် အက်ပ်လီကေးရှင်းများ**
- **စာရွက်စာတမ်း ဖြေရှင်းမှု pipeline**: အဆင့်များစွာရှိသော စာရွက်စာတမ်း ဝန်းကျင်နှင့် အတည်ပြုမှု
- **အရည်အသွေး အာမခံ လုပ်ငန်းစဉ်များ**: အဆက်လိုက် ပြန်လည်သုံးသပ်ခြင်း၊ အတည်ပြုခြင်းနှင့် အတည်ပြုချက် လုပ်ငန်းများ
- **အကြောင်းအရာ ထုတ်လုပ်မှု pipeline**: သုတေသန → စာရေးဆွဲခြင်း → တည်းဖြတ်ခြင်း → ပြန်လည်သုံးသပ်ခြင်း → ပုံနှိပ်ခြင်း
- **စီးပွားရေးလုပ်ငန်း အလိုအလျောက် လုပ်ငန်းစဉ်များ**: အဆင့်များစွာ ပါဝင်သည့် စီးပွားရေး workflow များ

### 🏗️ **အဆက်လိုက် Workflow ဖွဲ့စည်းပုံ**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: ချဉ်းကပ်စစ်ဆေးရေး အေးဂျင့်]
    B --> C[စစ်ဆေးနေရာ ၁]
    C --> D[Stage 2: ပြန်လည်စီမံအေးဂျင့်]
    D --> E[စစ်ဆေးနေရာ ၂]
```

**အဓိက ကိရိယာများ:**
- **အဆက်လိုက် အီးဂျင့်များ**: လုပ်ငန်းစဉ် အဆင့်တိုင်းအတွက် အထူးပြု အီးဂျင့်များ
- **အခြေအနေ Context**: အဆင့်အလိုက် အချက်အလက်နှင့် ဆုံးဖြတ်ချက် စုဆောင်းထားမှု
- **Checkpoint များ**: အဆင့်များအကြား အရည်အသွေးနှင့် တိကျမှု စစ်ဆေးချက်များ
- **Azure OpenAI Client**: အဆက်လိုက် လုပ်ငန်းအဆင့်များအတွင်း AI မော်ဒယ် သပ်ရပ်စွာ အသုံးပြုခြင်း

### 📝 မျှော်မှန်းသော ရလဒ်

workflow က -
1. **အရောင်းအီးဂျင့်**: ပရိဘောဂ သတ်မှတ်ပြီး အကြံပြုချက် ပေးသည်
2. **စျေးနှုန်းအီးဂျင့်**: အကြမ်းဖျင်းစျေးနှုန်း ခွဲခြမ်းချက်၊ ဘတ်ဂျက် အဆင့်များ နှင့် ဝယ်ယူမှု အကြံဉာဏ် ပိုမို ပံ့ပိုးသည်
3. **ကိုးထ်အီးဂျင့်**: အချက်အလက်များအားလုံးကို ပေါင်းစပ်ကာ ရေးသားထားသော ကိုးထ် စာတမ်း ပြုလုပ်ပေးသည်

နောက်ဆုံးထွက်သည်မှာ ပရိဘောဂ ဓာတ်ပုံ ခွဲခြမ်းမှုအပေါ် အခြေခံသော စွမ်းဆောင်ရည်မြင့် ပရော်ဖက်ရှင်နယ် furniture quote တစ်ခု ဖြစ်သည်။

### 🎯 လက်တွေ့ လုပ်ငန်းလမ်းကြောင်းများ

ဒီအဆက်လိုက် ပုံစံ သင့်တော်သော -
- **အီးကော်မပ်စ်**: ထုတ်ကုန် ခွဲခြမ်းချက် → စျေးနှုန်းတွက်ချက်ခြင်း → ကိုးထ် ဖန်တီးခြင်း
- **အိမ်ခြံမြေ**: အိမ်ခြံမြေ စိစစ်ခြင်း → တန်ဖိုးညှိနှိုင်းခြင်း → စာရင်းတင်ခြင်း
- **အာမခံ**: မေးခွန်းအစုံအလင် စိစစ်ခြင်း → သုံးသပ်ခြင်း → ကိုးထ်ဖန်တီးခြင်း
- **အကြောင်းအရာ ဖန်တီးခြင်း**: သုတေသန → စာရေး → တည်းဖြတ် → ပုံနှိပ်

### 🔍 အခြေအနေ လယ်လည်လည် စီးဆင်းမှု သိမြင်ခြင်း

အဆက်လိုက် အီးဂျင့် တစ်ခုချင်းစီ လက်ခံရရှိသည် -
- **မူရင်း အချက်အလက်**: စတင်အသုံးပြုသူ စာတိုက် (ဓာတ်ပုံ + စာသား)
- **ယခင်အီးဂျင့်များ ရုပ်ထွက်**: ပြောဆိုခဲ့သော စကားများအတွင်း ဦးဆုံး ထံမှ အားလုံးသော ရုပ်ထွက်များ
- **စုစုပေါင်း Context**: workflow တစ်လျှောက်လုံး ထိန်းသိမ်းထားသော အခြေအနေ အပြည့်အစုံ

ဒါက multi-stage လုပ်ငန်းစဉ်မှာ အဆင့်တိုင်းက နောက်ဆက်တွဲ အဆင့်များ၏ စုစုပေါင်း context တွေကို အသုံးပြု၍ သိပ္ပံနည်းကျ အကဲဖြတ်နိုင်စေသည်။
