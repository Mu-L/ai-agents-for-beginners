### 📦 **လိုအပ်သည့် NuGet Packages**

.NET အဆက်လိုက် workflow အတွက် မရှိမဖြစ် packages များ -

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.*" />

<!-- Azure Identity and Async LINQ Support -->
<PackageReference Include="Azure.Identity" Version="1.15.0" />
<PackageReference Include="System.Linq.Async" Version="6.0.3" />

<!-- Local Agent Framework References -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->
```

### 🔑 **Azure OpenAI နှင့် ကွန်ဖီဂရေးရှင်း**

**ပတ်ဝန်းကျင် တပ်ဆင်မှု (.env ဖိုင်):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**ကွန်ဖီဂရေးရှင်း စီမံခန့်ခွဲမှု:**
```csharp
// Load environment variables securely
Env.Load("../../../.env");
var azureEndpoint = Environment.GetEnvironmentVariable("AZURE_OPENAI_ENDPOINT");
var deployment = Environment.GetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT");
```

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: သုံးသပ်ရေး အေးဂျင့်]
```

- **Azure OpenAI Client**: workflow အဆင့်များ အားလုံးအတွက် တူညီသော AI မော်ဒယ် ချိတ်ဆက်မှု

### 📝 မျှော်မှန်းထားသည့် ထွက်ရှိမှု

workflow သည် -
1. **အရောင်း အေးဂျင့်**: ပုံရိပ်မှ အိမ်ထောင်ပစ္စည်းများကို ရွေးချယ်တင်ပြပြီး အကြံပြုချက် ပေးခြင်း
2. **စျေးနှုန်း အေးဂျင့်**: ဘတ်ဂျက် အဆင့်အတန်းများနှင့် ဝယ်ယူမှု အကြံပြုချက်များ၊ အသေးစိတ် စျေးနှုန်း သုံးသပ်ချက် ထည့်သွင်းခြင်း
3. **ကိုး(Quote) အေးဂျင့်**: အချက်အလက်တိုင်းကို ပေါင်းစပ်၍ ပုံစံသတ်မှတ်ထားသော ကိုး စာရွက် ပြုလုပ်ခြင်း

နောက်ဆုံး ထွက်ရှိမှုမှာ ပုံထုတ် စဉ်းစားပြီး ဖော်ပြထားသော ပရော်ဖက်ရှင်နယ် အိမ်ထောင်ပစ္စည်း ကိုးစာ ဖြစ်ပါမည်။

### 🎯 လက်တွေ့ကမ္ဘာ အတွက် အသုံးချမှုများ

အဆက်လိုက် ပုံစံသည် အထူးသင့်လျော်သောကြောင့် -
- **အီလက်ထရွန်းနစ်ကုန်အရောင်း**: ထုတ်ကုန် သုံးသပ်ခြင်း → စျေးနှုန်း သတ်မှတ်ခြင်း → ကိုး စာဖန်တီးခြင်း
- **အိမ်ခြံမြေ**: ပိုင်ဆိုင်မှု သုံးသပ်ခြင်း → တန်ဖိုး သတ်မှတ်ခြင်း → စာရင်းပြုစုခြင်း
- **အာမခံစာ**: ချိတ်ဆက်ကိစ္စ သုံးသပ်ခြင်း → အကဲဖြတ်ခြင်း → ကိုး စာဖန်တီးခြင်း
- **အကြောင်းအရာ ဖန်တီးမှု**: သုတေသန → စာရေးခြင်း → တည်းဖြတ်ခြင်း → ထုတ်ဝေခြင်း

### 🔍 အခြေအနေ စီးဆင်းမှု နားလည်ခြင်း

အဆက်လိုက် အေးဂျင့်တိုင်းသည် -
- **မူရင်း input**: အစပိုင်း အသုံးပြုသူ မက်ဆေ့(ပုံ + စာသား)
- **ယခင် အေးဂျင့်များ၏ output များ**: စကားပြောထားသော သမိုင်းထဲရှိ အရင်အေးဂျင့် အားလုံး၏ အဖြေများ
- **စုစည်းထားသော အကြောင်းအရာ**: workflow အတွင်း အခြေအနေ လုံးဝ ထိန်းသိမ်းထားခြင်း

၎င်းသည် အဆင့်တိုင်းရှိ အေးဂျင့်တိုင်းသည် ယခင် အဆင့်များမှ စုစည်းထားသော အကြောင်းအရာများ အပေါ် တည်ဆောက်နိုင်သော ဆန်းကြယ်သော multi-stage လုပ်ငန်းစဉ် ဖန်တီးရာတွင် အထောက်အကူပြုသည်။
