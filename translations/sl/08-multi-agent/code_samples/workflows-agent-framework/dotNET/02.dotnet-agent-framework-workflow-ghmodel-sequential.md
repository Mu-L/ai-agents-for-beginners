# ⏩ Zaporedna delovna toka agentov z Azure OpenAI (Responses API) (.NET)

## 📋 Napredno navodilo za zaporedno procesiranje

Ta zvezek prikazuje **vzorce zaporednih delovnih tokov** z uporabo Microsoft Agent Framework za .NET in Azure OpenAI (Responses API). Naučili se boste, kako sestaviti zahtevne, korak za korakom poteke obdelave, kjer agenti izvajajo ukaze v določenem vrstnem redu, pri čemer se vsaka stopnja gradi na rezultatih prejšnje.

## 🎯 Cilji učenja

### 🔄 **Arhitektura zaporednega procesiranja**
- **Linearni dizajn delovnega toka**: Ustvarite korak-po-koraku procesne cevi s jasnimi odvisnostmi
- **Upravljanje stanja**: Ohranjanje konteksta in pretoka podatkov skozi zaporedne stopnje
- **Azure OpenAI (Responses API)**: Izkoristite modele Azure OpenAI v večstopenjskih .NET delovnih tokovih
- **Vzorce cevi v podjetjih**: Zgradite proizvodne zaporedne sisteme obdelave

### 🏗️ **Napredni vzorci zaporedja**
- **Procesiranje skozi kontrolo vstopa**: Implementirajte kontrolne točke za validacijo med stopnjami delovnega toka
- **Ohranjanje konteksta**: Ohranjajte stanje in nabrano znanje skozi vse stopnje
- **Prehod napak**: Obvladovanje napak na uglajen način v zaporednih procesnih verigah
- **Optimizacija zmogljivosti**: Učinkovito zaporedno izvajanje z minimalno režijo

### 🏢 **Podjetniške zaporedne aplikacije**
- **Cev dokumentne obdelave**: Večstopenjska analiza, transformacija in validacija dokumentov

- **Cev za produkcijo vsebine**: Raziskava → Pisanje → Urejanje → Pregled → Objavljanje
- **Avtomatizacija poslovnih procesov**: Večstopenjski poslovni delovni tokovi z jasnimi odvisnostmi stopnje

### 🏗️ **Arhitektura zaporednega delovnega toka**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Analizni agent]
    B --> C[Kontrolna točka 1]
    C --> D[Stage 2: Procesni agent]
    D --> E[Kontrolna točka 2]
    E --> F[Stage 3: Validacijski agent]
    F --> G[Končni rezultat]
    
    H[Kontekst stanja] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Responses API)] --> B
    I --> D
    I --> F
```

**Ključne sestavine:**
- **Zaporedni agenti**: Specializirani agenti za vsako stopnjo obdelave

- **Kontrolne točke**: Točke validacije med stopnjami za zagotovitev kakovosti in skladnosti
- **Azure OpenAI odjemalec**: Dosleden dostop do AI modelov skozi vse stopnje delovnega toka

## 💻 Zagon kode

Celotna implementacija je na voljo v `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Ta datoteka prikazuje **tristopenjsko analizo pohištva**:

1. **Stopnja 1 - prodajni agent**: Analizira slike pohištva in daje predloge za nakup
2. **Stopnja 2 - cenovni agent**: Podaja podrobne razčlenitve cen in možnosti proračuna
3. **Stopnja 3 - agent za ponudbo**: Generira profesionalni dokument ponudbe v Markdown formatu

### 🏗️ **Arhitektura delovnega toka**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Vsak agent:
- Prejme izhod prejšnje stopnje kot kontekst
- Nadgradi prejšnjo analizo s specializiranim znanjem
- Ohranja kontinuiteto delovnega toka z upravljanjem stanja

### 🚀 Zagon primera

**Predpogoji:**
- Postavite sliko pohištva v `../imgs/home.png` (ali posodobite spremenljivko `imgPath`)
- Konfigurirajte svojo `.env` datoteko z vašim Azure OpenAI končnim naslovom in namestitvijo, nato se prijavite z `az login`

```bash
# Naredite skripto izvršljivo (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Zaženite zaporedni potek dela
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Ali na Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Pričakovani izhod

Delovni tok bo:
1. **Prodajni agent**: Prepoznal elemente pohištva na sliki in podal priporočila
2. **Cenovni agent**: Dodal podrobno analizo cen z nivoji proračuna in nakupnimi nasveti
3. **Agent za ponudbo**: Generiral formatiran dokument ponudbe z vsemi sintetiziranimi informacijami

Končni izhod bo obsežna, profesionalna ponudba pohištva na podlagi analize slike.

### 🎯 Praktične uporabe

Ta zaporedni vzorec je idealen za:
- **E-trgovina**: Analiza izdelka → določanje cen → generiranje ponudb
- **Nepremičnine**: Analiza nepremičnin → vrednotenje → ustvarjanje seznamov
- **Zavarovalništvo**: Analiza zahtevkov → ocenjevanje → generiranje ponudb
- **Ustvarjanje vsebine**: Raziskave → Pisanje → Urejanje → Objavljanje

### 🔍 Razumevanje pretoka stanja

Vsak agent v zaporedju prejme:
- **Izvirni vhod**: Začetno uporabniško sporočilo (slika + besedilo)
- **Izhode prejšnjih agentov**: Vse prejšnje odzive agentov v zgodovini pogovora
- **Nabran kontekst**: Popolno stanje ohranjeno skozi cel delovni tok

To omogoča zahtevno večstopenjsko procesiranje, kjer vsak agent gradi na celovitem kontekstu vseh prejšnjih stopenj.
