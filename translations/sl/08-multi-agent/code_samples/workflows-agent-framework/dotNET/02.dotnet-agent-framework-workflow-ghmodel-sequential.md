# ⏩ Zaporedni agentni delovni tokovi z Azure OpenAI (Responses API) (.NET)

## 📋 Napredno učenje zaporednega procesiranja

Ta zvezek prikazuje **vzorce zaporednih delovnih tokov** z uporabo Microsoft Agent Framework za .NET in Azure OpenAI (Responses API). Naučili se boste, kako zgraditi sofisticirane, korak za korakom procesne linije, kjer agenti izvajajo v določenem vrstnem redu, pri čemer vsak korak gradi na rezultatih prejšnjega.

## 🎯 Cilji učenja

### 🔄 **Arhitektura zaporednega procesiranja**
- **Linearni dizajn delovnega toka**: Ustvarite korak za korakom procesne linije z jasnimi odvisnostmi
- **Upravljanje stanja**: Ohranjanje konteksta in toka podatkov skozi zaporedne faze delovnega toka
- **Azure OpenAI (Responses API)**: Izkoristite Azure OpenAI modele v večfaznih .NET delovnih tokovih
- **Vzorci podjetniških procesnih linij**: Gradite proizvodne zaporedne procesne sisteme

### 🏗️ **Napredni zaporedni vzorci**
- **Stage-Gate procesiranje**: Izvedba kontrolnih točk med fazami delovnega toka
- **Ohranjanje konteksta**: Ohranjanje stanja in akumuliranega znanja skozi vse faze
- **Propagacija napak**: Učinkovito ravnanje z napakami v zaporednih procesnih verigah
- **Optimizacija zmogljivosti**: Učinkovito zaporedno izvajanje z minimalnim overheadom

### 🏢 **Podjetniške zaporedne aplikacije**
- **Procesna linija obdelave dokumentov**: Večfazna analiza, transformacija in validacija dokumentov
- **Delovni tokovi za zagotavljanje kakovosti**: Zaporedni procesi pregleda, validacije in odobritve
- **Procesna linija ustvarjanja vsebin**: Raziskovanje → Pisanje → Urejanje → Pregled → Objavljanje
- **Avtomatizacija poslovnih procesov**: Večstopenjski poslovni delovni tokovi z jasnimi odvisnostmi faz

### 🏗️ **Arhitektura zaporednih delovnih tokov**

```mermaid
graph TD
    A[Initial Input] --> B[Stage 1: Analitični agent]
    B --> C[Kontrolna točka 1]
    C --> D[Stage 2: Obdelovalni agent]
    D --> E[Kontrolna točka 2]
    E --> F[Stage 3: Validacijski agent]
    F --> G[Končni rezultat]
    
    H[Kontekst stanja] --> B
    H --> D
    H --> F
    
    I[Azure OpenAI (Responses API)] --> B
    I --> D
    I --> F
```

**Ključne komponente:**
- **Zaporedni agenti**: Specializirani agenti za vsako fazo procesiranja
- **Kontekst stanja**: Ohranja akumulirane podatke in odločitve skozi faze
- **Kontrolne točke**: Validacijske točke med fazami za zagotovitev kakovosti in skladnosti
- **Azure OpenAI klient**: Konsistenten dostop do AI modela skozi vse faze delovnega toka

## 💻 Zagon kode

Celotna implementacija je na voljo v `02.dotnet-agent-framework-workflow-ghmodel-sequential.cs`. Ta datoteka prikazuje **trifazni delovni tok analize pohištva**:

1. **Faza 1 – Prodajni agent**: Analizira slike pohištva in daje predloge za nakup
2. **Faza 2 – Cenovni agent**: Ponuja podrobne cenovne razčlenitve in proračunske možnosti
3. **Faza 3 – Agent za ponudbe**: Ustvari strokovni dokument ponudbe v Markdown formatu

### 🏗️ **Arhitektura delovnega toka**

```
Image Input → Sales Analysis → Price Estimation → Quote Generation → Final Output
```

Vsak agent:
- Prejme izhod iz prejšnje faze kot kontekst
- Gradi na prejšnji analizi s specializiranim znanjem
- Ohranja kontinuiteto delovnega toka preko upravljanja stanja

### 🚀 Zagon primera

**Predpogoji:**
- Postavite sliko pohištva v `../imgs/home.png` (ali posodobite spremenljivko `imgPath`)
- Konfigurirajte svojo `.env` datoteko z vašim Azure OpenAI končnim točko in namestitvijo, nato se prijavite z `az login`

```bash
# Naredite skripto izvršljivo (Unix/Linux/macOS)
chmod +x 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs

# Zaženite zaporedno delovno sekvenco
./02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

Ali na Windows:
```powershell
dotnet run 02.dotnet-agent-framework-workflow-ghmodel-sequential.cs
```

### 📝 Pričakovani izhod

Delovni tok bo:
1. **Prodajni agent**: Prepoznal predmete pohištva na sliki in dal priporočila
2. **Cenovni agent**: Dodal podrobno cenovno analizo s proračunskimi nivoji in nakupnimi priporočili
3. **Agent za ponudbe**: Ustvaril formatiran dokument ponudbe z vsemi združenimi informacijami

Končni izhod bo celovita, strokovna ponudba za pohištvo na osnovi analize slike.

- **Izhode prejšnjih agentov**: Vse prejšnje odzive agentov v zgodovini pogovora
