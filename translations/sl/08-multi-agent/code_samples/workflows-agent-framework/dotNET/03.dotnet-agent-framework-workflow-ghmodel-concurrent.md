# ⚡ Sočasni agentni delovni tokovi z Azure OpenAI (Responses API) (.NET)

## 📋 Vadnica za vzporedno obdelavo z visoko zmogljivostjo

Ta zvezek prikazuje **vzorce sočasnih delovnih tokov** z uporabo Microsoft Agent Framework za .NET in Azure OpenAI (Responses API). Naučili se boste, kako zgraditi visoko zmogljive, vzporedne delovne tokove, ki maksimirajo prepustnost z istočasnim izvajanjem več AI agentov ob vzdrževanju koordinacije in doslednosti podatkov.

## 💻 Zagon kode

Celotna implementacija je na voljo v `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Ta datoteka prikazuje **Fan-Out/Fan-In sočasni delovni tok** za načrtovanje potovanj:

### 🏗️ **Arhitektura delovnega toka**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Ključne sestavine:**

1. **ConcurrentStartExecutor**: Hkrati oddaja uporabniško zahtevo vsem agentom
2. **Researcher Agent**: Sočasno analizira destinacije in znamenitosti
3. **Planner Agent**: Sočasno ustvarja podrobne potovalne načrte
4. **ConcurrentAggregationExecutor**: Zbira in združuje rezultate obeh agentov

### 🎯 **Vzorec Fan-Out/Fan-In**

Ta delovni tok prikazuje klasični **Fan-Out/Fan-In** vzorec:
- **Fan-Out**: Ena vhodna sporočila se hkrati oddajo več agentom
- **Sočasna obdelava**: Več agentov vzporedno dela na isti nalogi
- **Fan-In**: Rezultati vseh agentov se zberejo in združijo v en sam izhod

### 📝 Pričakovani izhod

Delovni tok bo:
1. **Oddaja zahteve**: Poslal "Plan a trip to Seattle in December" obema agentoma
2. **Sočasna obdelava**: Oba agenta delata hkrati:
   - Researcher identificira znamenitosti in podrobnosti
   - Planner ustvari potni načrt in logistiko
3. **Združevanje**: Združi oba odgovora v obsežen izhod
4. **Prikaz rezultatov**: Pokaže združeni načrt potovanja z vsemi informacijami

Ta sočasni vzorec je idealen za:

### 🔍 Razumevanje prilagojenih izvršiteljev

**ConcurrentStartExecutor:**
- Uvaja `IMessageHandler<string>` za sprejemanje vhodnih nizov
- Oddaja sporočila vsem povezanih agentom
- Pošilja `TurnToken` za sprožitev sočasne obdelave

**ConcurrentAggregationExecutor:**
- Uvaja `IMessageHandler<ChatMessage>` za sprejemanje odgovorov agentov
- Varnostno zbira sporočila po nitih
- Združuje, ko prispejo vsi pričakovani odgovori
- Ustvari končni izhod z `context.YieldOutputAsync()`

### ⚡ Koristi zmogljivosti

**Sočasno vs Zaporedno:**
- Zaporedno: Agent1 (30s) → Agent2 (30s) = **60 sekund skupaj**
- Sočasno: Agent1 (30s) || Agent2 (30s) = **30 sekund skupaj**

**Izboljšanje prepustnosti**: Do N× hitrejše za N sočasnih agentov (odvisno od obremenitve in virov)

### 🛡️ Obravnavanje napak

Delovni tok elegantno obvladuje posamezne napake agentov:
- Če eden agent ne uspe, ostali nadaljujejo z obdelavo
- Združitelj lahko izvaja časovno omejitev
- Potencialno se lahko vrnejo delni rezultati, če je potrebno
