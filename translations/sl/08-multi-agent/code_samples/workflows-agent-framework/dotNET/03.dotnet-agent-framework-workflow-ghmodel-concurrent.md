## 📋 Vadnica za paralelno obdelavo z visoko zmogljivostjo

## 💻 Zagon kode

Celotna implementacija je na voljo v `03.dotnet-agent-framework-workflow-ghmodel-concurrent.cs`. Ta datoteka prikazuje **Fan-Out/Fan-In sočni potek dela** za načrtovanje potovanj:

### 🏗️ **Arhitektura poteka dela**

```
User Request → ConcurrentStartExecutor → [Researcher Agent || Planner Agent] → ConcurrentAggregationExecutor → Final Output
```

**Ključne komponente:**

1. **ConcurrentStartExecutor**: Hkrati pošlje uporabnikov zahtevek vsem agentom
2. **Researcher Agent**: Sočasno analizira destinacije in znamenitosti
3. **Planner Agent**: Paralelno ustvarja podrobne potovalne načrte
4. **ConcurrentAggregationExecutor**: Zbira in združuje rezultate obeh agentov

### 🎯 **Vzorec Fan-Out/Fan-In**

Ta potek dela prikazuje klasični **Fan-Out/Fan-In** vzorec:
- **Fan-Out**: Ena vhodna sporočila se hkrati pošlje več agentom
- **Sočasna obdelava**: Več agentov deluje vzporedno na isti nalogi
- **Fan-In**: Rezultati vseh agentov se zberejo in združijo v en izhod

### 📝 Pričakovani izhod

Potek dela bo:
1. **Oddaja zahteve**: Pošlje "Planiraj potovanje v Seattle decembra" obema agentoma
2. **Sočasna obdelava**: Oba agenta delata hkrati:
   - Raziskovalec prepozna znamenitosti in podrobnosti

3. **Združevanje**: Združi oba odgovora v celovit izhod

### 🔍 Razumevanje po meri izdelanih izvajalcev

**ConcurrentStartExecutor:**
- Uvaja `IMessageHandler<string>` za sprejemanje vhodnih nizov
- Oddaja sporočila vsem povezanih agentom
- Pošlje `TurnToken` za sprožitev sočasne obdelave

**ConcurrentAggregationExecutor:**
- Uvaja `IMessageHandler<ChatMessage>` za prejemanje odgovorov agentov
- V varnem načinu zbira sporočila
- Združuje, ko prispejo vsi pričakovani odgovori
- Odda končni izhod z uporabo `context.YieldOutputAsync()`

### ⚡ Koristi zmogljivosti

**Sočasno vs zaporedno:**
- Zaporedno: Agent1 (30s) → Agent2 (30s) = **60 sekund skupaj**
- Sočasno: Agent1 (30s) || Agent2 (30s) = **30 sekund skupaj**

**Izboljšanje prepustnosti**: Do N× hitreje za N sočasnih agentov (odvisno od obremenitve in virov)

### 🛡️ Obravnava napak

Potek dela elegantno obravnava posamezne napake agentov:
- Če en agent odpove, drugi nadaljujejo z obdelavo
- Aggregator lahko izvaja logiko časovne omejitve
- Delne rezultate je mogoče vrniti po potrebi
