# 🔄 Osnovni agentni poteki dela z Azure OpenAI (Responses API) (.NET)

## 📋 Vadnica za orkestracijo poteka dela

Ta zvezek prikazuje, kako zgraditi sofisticirane **agentne poteke dela** z uporabo Microsoft Agent Framework za .NET in Azure OpenAI (Responses API). Naučili se boste ustvarjati večstopenjske poslovne procese, kjer umetni agenti sodelujejo za dosego zahtevnih nalog skozi strukturirane vzorce orkestracije.

## 🎯 Cilji učenja

### 🏗️ **Osnove arhitekture poteka dela**
- **Ustvarjalec poteka dela**: Oblikovanje in orkestracija kompleksnih večstopenjskih AI procesov
- **Koordinacija agentov**: Usklajevanje več specializiranih agentov znotraj potekov dela
- **Azure OpenAI (Responses API)**: Izkoriščanje Azure OpenAI Responses API v potekih dela
- **Vizualno oblikovanje poteka dela**: Ustvarjanje in vizualizacija struktur poteka dela za boljše razumevanje

### 🔄 **Vzorec orkestracije procesov**
- **Zaporedna obdelava**: Zvezanje več agentnih nalog v logičnem vrstnem redu
- **Upravljanje stanja**: Ohranjanje konteksta in pretoka podatkov skozi faze poteka dela
- **Obravnava napak**: Implementacija robustnega okrevanja po napakah in odpornosti poteka dela
- **Optimizacija zmogljivosti**: Oblikovanje učinkovitih potekov dela za operacije na ravni podjetja

### 🏢 **Uporaba poteka dela v podjetjih**
- **Avtomatizacija poslovnih procesov**: Avtomatizacija kompleksnih organizacijskih potekov dela
- **Proizvodna linija vsebine**: Uredniški poteki dela z ocenjevalnimi in potrditvenimi fazami
- **Avtomatizacija storitev za stranke**: Večstopenjsko reševanje strankinih poizvedb
- **Poteki obdelave podatkov**: ETL poteki dela z AI-podprto transformacijo

## ⚙️ Zahteve in nastavitev

### 📦 **Zahtevani NuGet paketi**

Ta prikaz poteka dela uporablja nekaj ključnih .NET paketov:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Konfiguracija Azure OpenAI**

**Nastavitev okolja (datoteka .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-5-mini
```

**Dostop do Azure OpenAI:**
1. Ustvarite Azure OpenAI vir v Azure portalu
2. Namestite model (npr. `gpt-5-mini`) in zabeležite ime namestitve
3. Prijavite se z `az login` in nastavite okoljske spremenljivke kot je prikazano zgoraj

**Ključne komponente:**
- **WorkflowBuilder**: Glavni orkestracijski motor za oblikovanje potekov dela
- **AIAgent**: Posamezni specializirani agenti s specifičnimi zmožnostmi
- **Azure OpenAI Client**: Integracija Azure OpenAI Responses API
- **Execution Context**: Upravljanje stanja in pretoka podatkov med fazami poteka dela

## 💻 Zagon kode

Celotna implementacija je na voljo v `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Ta datoteka prikazuje:

1. **Konfiguracija okolja** - nalaganje konfiguracije Azure OpenAI iz `.env` datoteke
2. **Nastavitev Azure OpenAI odjemalca** - konfiguracija odjemalca za uporabo Azure OpenAI Responses API
3. **Ustvarjanje agentov** - definiranje specializiranih agentov (front desk in concierge)
4. **Ustvarjalec poteka dela** - ustvarjanje večagentnega poteka dela z zaporedno obdelavo
5. **Izvajanje poteka dela** - zagon poteka dela s pretakanjem rezultatov

### 🚀 Zagon primera

```bash
# Naredite skripto izvršljivo (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Zaženite potek dela
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Ali na Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Pričakovani izhod

Potek dela bo:
1. Sprejel vašo zahtevo po potovanju ("Rad bi šel v Pariz")
2. Agent front desk poda začetno priporočilo
3. Agent concierge pregleda in izboljša priporočilo
4. Končni izhod prikaže celoten tok pogovora

### 🔧 Prilagoditev

Potek dela lahko prilagodite z:
- Spreminjanjem navodil agentov za spremembo njihovega vedenja
- Dodajanjem več agentov za ustvarjanje kompleksnih večstopenjskih potekov dela
- Spreminjanjem uporabniškega sporočila za testiranje različnih scenarijev
- Prilagajanjem robov poteka dela za ustvarjanje različnih vzorcev izvajanja
