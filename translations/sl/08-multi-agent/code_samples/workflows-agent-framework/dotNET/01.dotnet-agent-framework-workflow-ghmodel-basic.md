# 🔄 Osnovni delovni procesi agentov z Azure OpenAI (Responses API) (.NET)

## 📋 Vodenje delovnega procesa

Ta zvezek prikazuje, kako zgraditi sofisticirane **delovne procese agentov** z uporabo Microsoft Agent Framework za .NET in Azure OpenAI (Responses API). Naučili se boste ustvarjati večstopenjske poslovne procese, kjer AI agenti sodelujejo za izvedbo zapletenih opravil skozi strukturirane vzorce orkestracije.

## 🎯 Cilji učenja

### 🏗️ **Osnove arhitekture delovnega procesa**
- **Sestavljalec delovnih procesov**: Načrtovanje in orkestracija zapletenih večstopenjskih AI procesov
- **Koordinacija agentov**: Usmerjanje več specializiranih agentov znotraj delovnih procesov
- **Azure OpenAI (Responses API)**: Izraba Azure OpenAI Responses API v delovnih procesih
- **Vizualni dizajn delovnih procesov**: Ustvarjanje in vizualizacija struktur delovnih procesov za boljše razumevanje

### 🔄 **Vzorce orkestracije procesov**
- **Zaporedna obdelava**: Verižna povezava več opravil agentov v logičnem vrstnem redu
- **Upravljanje stanja**: Ohranjanje konteksta in prenosa podatkov skozi stopnje delovnega procesa
- **Ravnanje z napakami**: Uvedba robustnega okrevanja po napakah in odpornosti delovnega procesa
- **Optimizacija učinkovitosti**: Oblikovanje učinkovitih delovnih procesov za poslovne operacije v obsegu

### 🏢 **Poslovne aplikacije delovnih procesov**
- **Avtomatizacija poslovnih procesov**: Avtomatizacija zapletenih organizacijskih delovnih procesov
- **Proizvodnja vsebin**: Uredniški delovni procesi z recenzijskimi in potrditvenimi stopnjami
- **Avtomatizacija službe za stranke**: Večstopenjsko reševanje strankinih povpraševanj
- **Delovni procesi obdelave podatkov**: ETL delovni procesi s transformacijo na osnovi AI

## ⚙️ Predpogoji in nastavitve

### 📦 **Zahtevani NuGet paketi**

Ta prikaz delovnega procesa uporablja več ključnih .NET paketov:

```xml
<!-- Core AI Framework -->
<PackageReference Include="Microsoft.Extensions.AI" Version="10.*" />

<!-- Azure OpenAI (Responses API) -->
<PackageReference Include="Azure.AI.OpenAI" Version="2.1.0" />
<PackageReference Include="Azure.Identity" Version="1.15.0" />

<!-- Agent Framework (Local Development) -->
<!-- Microsoft.Agents.AI.dll - Core agent abstractions -->
<!-- Microsoft.Agents.AI.OpenAI.dll - Azure OpenAI (Responses API) integration -->

<!-- Configuration and Environment -->
<PackageReference Include="DotNetEnv" Version="3.1.1" />
```

### 🔑 **Konfiguracija Azure OpenAI**

**Nastavitev okolja (datoteka .env):**
```env
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4.1-mini
```

**Dostop do Azure OpenAI:**
1. Ustvarite vir Azure OpenAI v Azure portalu
2. Namestite model (na primer, `gpt-4.1-mini`) in zabeležite ime namestitve
3. Prijavite se z `az login` in konfigurirajte spremenljivke okolja, kot je prikazano zgoraj

### 🏗️ **Pregled arhitekture delovnega procesa**

```mermaid
graph TD
    A[Ustvarjalec poteka dela] --> B[Register agentov]
    B --> C[Pogon za izvajanje poteka dela]
    C --> D[Agent 1: Generator vsebine]
    C --> E[Agent 2: Pregledovalec vsebine] 
    D --> F[Rezultati poteka dela]
    E --> F
    G[Azure OpenAI (Responses API)] --> D
    G --> E
```

**Ključne komponente:**
- **WorkflowBuilder**: Glavni orkestracijski mehanizem za načrtovanje delovnih procesov
- **AIAgent**: Posamezni specializirani agenti z določenimi zmogljivostmi
- **Azure OpenAI Client**: Integracija Azure OpenAI Responses API
- **Execution Context**: Upravljanje stanja in pretoka podatkov med stopnjami delovnega procesa

## 💻 Zagon kode

Celotna implementacija je na voljo v `01.dotnet-agent-framework-workflow-ghmodel-basic.cs`. Ta datoteka prikazuje:

1. **Konfiguracija okolja** - nalaganje konfiguracije Azure OpenAI iz datoteke `.env`
2. **Nastavitev odjemalca Azure OpenAI** - konfiguracija odjemalca za uporabo Azure OpenAI Responses API
3. **Ustvarjanje agentov** - definiranje specializiranih agentov (Recepcija in Concierge)
4. **Sestavljalec delovnih procesov** - ustvarjanje večagentnega delovnega procesa z zaporedno obdelavo
5. **Izvajanje delovnega procesa** - zagon delovnega procesa z rezultati v živo (streaming)

### 🚀 Zagon primera

```bash
# Naredite skripto zaganjalno (Unix/Linux/macOS)
chmod +x 01.dotnet-agent-framework-workflow-ghmodel-basic.cs

# Zaženite delovni tok
./01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

Ali v Windows:
```powershell
dotnet run 01.dotnet-agent-framework-workflow-ghmodel-basic.cs
```

### 📝 Pričakovan izhod

Delovni proces bo:
1. Sprejel vašo zahtevo za destinacijo potovanja ("Rad bi šel v Pariz")
2. Agent Recepcije bo podal začetno priporočilo
3. Agent Concierge bo pregledal in izpopolnil priporočilo
4. Končni izhod bo prikazal celoten potek pogovora

### 🔧 Prilagajanje

Delovni proces lahko prilagodite z:
- Spreminjanjem navodil agentov za spremembo njihovega vedenja
- Dodajanjem več agentov za ustvarjanje zapletenih večstopenjskih delovnih procesov
- Spreminjanjem sporočila uporabnika za testiranje različnih scenarijev
- Prilagajanjem povezav delovnega procesa za ustvarjanje različnih vzorcev izvajanja
